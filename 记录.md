记录内容；
- 知识结构
    - 讲座视频与文章中可能用到的操作/技巧/概念，并提取到知识结构
    - 挑战/实验中的思路/技巧/命令/概念/模式，并提取到知识结构
- 速查表
    - 将接触到的主题的核心概念精炼为几句话，进行速览

在材料的模块/主题中将各挑战的内容聚合在开头，也创建概念文档聚合所有材料的信息

材料分类：
- 全文分析，信息密度高的博客文章或教材，保持原文并添加链接。仅提取实操内容。
- 速查手册，仅查看/提取特定的相关内容。

linux & c：
- man2 记录头文件/原型/参数/功能描述到 `c md`，并记录接触到的对应汇编
- 其他 man，记录命令/描述/接触到的参数/简略示例到 `linux md`

网络渗透/后渗透/AD：
- 状态
    - 材料分散在各种官网文档、技术博客中
- 分类
    - 端口协议的服务
    - windows 凭证令牌的模型

归档
- 将分散的文章放入主题大类即完成初步归档。在完成阅读前缺乏内容了解，此时无法精细分类或合并
- 仅对于高度重合的内容进行合并。相关但不重叠的文章作为子主题章节的独立小节

LLM 翻译内容保存
- 图书与几乎不更新的 Web 文档（PTES、手册）作为固定内容保存
- 动态更新的 Web 文档不保存
    - 若再次查看，应当查看最新内容而不是过时内容的翻译
