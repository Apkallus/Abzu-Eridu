# 注入与劫持

尽管 Windows 中有许多与 Linux 相似的概念，但 win32 API 和 Windows 安全模型却截然不同，由此衍生出了一些在 Linux 中并不常见的攻击场景。本模块将探讨几种经典的 Windows 注入与劫持技术，例如 DLL 注入、进程镂空（Process Hollowing）以及线程劫持，这些技术允许攻击者在另一进程的上下文中执行代码。此外，本模块还加入了一个额外的挑战：一个基础版的端点检测与响应（EDR）系统将被用于检测和拦截这些技术。你能否绕过 EDR，在另一进程的上下文中执行你的代码？

注意：本道场的开发进度较为*缓慢*，包括基础设施支持的迭代。相关功能可能会随时调整！

## 资源

### Robdefender EDR 介绍

在本系列挑战开始后不久，我们自定义的 EDR——"robdefender"将在 Windows 虚拟机内启动。随着挑战的推进，robdefender 会不断更新，以检测并阻止各种攻击技术。该 EDR 并非无懈可击，存在多种绕过方式。EDR 的实现文件位于 `C:\challenge\RD` 目录中，仅供了解内部实现。"rd_rules.txt" 文件可以提示启用了哪些钩子。

**注意：ROBDEFENDER 并非这些挑战的目标程序！** 正在运行的挑战进程才是你需要关注的重点。

友情提醒：不建议花费过多时间去逆向分析 EDR 的实现细节，且其实现可能随时更改，恕不另行通知。

## 挑战

### 简陋的计算器

没什么花招，就是一个简单的计算器。你能拿到 flag 吗？
- 注入器附加线程到目标进程：
    - shellcode 汇编指令码
        - 打开并写入文件。得到数据后目标线程可崩溃
        - 写入目标进程内存后，附加进程读取对应位置的内存。需确保附加线程正常返回
    - dll c++
        - 执行库加载
        - 在库文件中使用 c++
