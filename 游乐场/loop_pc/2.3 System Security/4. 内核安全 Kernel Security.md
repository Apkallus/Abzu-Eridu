# 内核安全

内核是操作系统的核心组件，充当软件与硬件之间的桥梁。内核运行在操作系统的最底层，其访问权限极其深刻，甚至可以视同于伪装成系统本身，超越了 root 用户的最高权限。内核安全至关重要，因为一旦这一层面被攻破，攻击者就能以系统的身份行事。漏洞可能导致未经授权的数据访问、系统崩溃或 rootkit 的悄无声息地安装等后果。

为了开始内核挑战，你需要在虚拟机内运行这些挑战。你可以通过在终端中运行命令 `vm start` 来启动此虚拟机。虚拟机将自动加载位于 `/challenge` 目录中的内核模块。然后，你可以使用 `vm connect` 将终端连接到该虚拟机！

```c
    8 @ 0040169c  close(fd)    
    9 @ 004016a6  exit(0)
```
- 子进程在读取 flag 字符串到内存后立即退出
    - 退出时内存进入空闲列表，尝试在内核分配内存以获取残留数据

思路：
- 分配内存页面，得到内存地址 `__get_free_pages(gfp_t gfp_mask, unsigned int order)`
    - `gfp_mask` 使用常用标志 `GFP_KERNEL` 0xcc0
- `__get_free_pages(0xcc0, 0)` 分配 1<<0页
- `strnstr(addr, "pwn.college{", 4096)` c 库函数查找子串
- 循环执行
- `printk("%s", pwn_str)`

调试：
- `./kernel.py && touch /flag && echo "pwn.college{1234567890" > /flag  && cat /flag && hd payload.bin`
- 减小循环次数，使得不至于崩溃而持续修改代码

实行：
- 增加/无限循环，若 hit 则 ret 而不会崩溃，若超过内存上限崩溃则 flag 被删除而需重启环境，最大化循环次数以尽量增加可能性
