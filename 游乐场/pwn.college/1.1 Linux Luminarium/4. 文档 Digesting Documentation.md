## 挑战

### 从文档中学习

典型情况下，您对文档的需求主要是了解如何使用这些程序，其中一个具体场景就是确定需要在命令行中指定哪些参数。本模块将重点探讨这一概念，作为理解程序使用方法的切入点。在后续内容中，您将学习通过各种方式向环境寻求程序帮助，但首先我们将深入探讨阅读文档的概念。

程序的正确使用在很大程度上取决于参数的恰当指定。回顾基础命令模块中「隐藏文件」挑战环节的`ls -a`：其中的`-a`是一个*参数*，它指示`ls`同时列出隐藏文件和非隐藏文件。由于我们*需要*列出隐藏文件，在场景中调用`ls`时附带`-a`参数便是正确的使用方式。

### 阅读手册

本关卡将介绍 `man` 命令。
`man` 是 `manual`（手册）的缩写，用于显示（如果存在）您作为参数传入的命令的手册页。
例如，假设我们想了解 `yes` 命令（*是的*，这确实是一个真实存在的命令）：

```console
hacker@dojo:~$ man yes
```

这将显示 `yes` 命令的手册页，其内容大致如下：


您可以使用方向键和 PgUp/PgDn 键在手册页中滚动浏览。
阅读完毕后，可按 `q` 键退出。

手册页存储在集中式数据库中。
若您感兴趣，该数据库位于 `/usr/share/man` 目录，但您无需直接与之交互：只需通过 `man` 命令进行查询即可。
`man` 命令的参数不是文件路径，而是条目名称本身（例如，查询 `yes` 手册页时应运行 `man yes`，而非 `man /usr/bin/yes`——后者虽是 `yes` 程序的实际路径，但会导致 `man` 显示乱码）。

您必须通过 `challenge` 的手册页来学习此选项！

### 使用帮助参数查看文档

某些程序可能没有手册页，但若使用特定参数调用时，它们会告知运行方法。
通常该参数为 `--help`，但也常见 `-h`，少数情况下可能是 `-?`、`help` 或其他冷门值如 `/?`（后者更常见于 Windows 系统）。

在本关卡中，您将练习通过 `--help` 读取程序的说明文档。
开始尝试吧！

### 使用shell的内置命令查看帮助信息

有些命令并非带有手册页和帮助选项的程序，而是直接内置于 shell 本身。
这些命令称为*内置命令*。
内置命令的调用方式与普通命令相同，但 shell 会在内部处理它们而非启动外部程序。
您可通过运行*内置命令* `help` 获取 shell 内置命令列表，如下所示：

```console
hacker@dojo:~$ help
```

通过将特定内置命令传递给 `help` 内置命令可获取其详细帮助信息。
让我们查看之前已使用过的内置命令 `cd` 的帮助信息：

```console
hacker@dojo:~$ help cd
cd: cd [-L|[-P [-e]] [-@]] [dir]
    Change the shell working directory.
    
    Change the current directory to DIR.  The default DIR is the value of the
    HOME shell variable.
...
```

很有用的信息！
在本挑战中，我们将练习使用 `help` 查询内置命令的帮助信息。
本关卡的 `challenge` 命令是一个 shell 内置命令（而非独立程序）。
与之前类似，您需要查阅其帮助信息以找出需要传递给它的秘密值！
