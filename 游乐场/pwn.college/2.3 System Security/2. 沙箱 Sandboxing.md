# 沙箱

计算机安全中的沙箱是指一种用于隔离潜在恶意代码或不可信程序的技术，确保它们在受限环境中运行，从而无法对更广泛的系统造成危害。通过为这些程序创建一个“沙箱”或受限空间来运行，任何恶意行为都被限制在这个隔离区域内，防止对主机系统造成潜在损害或访问敏感数据。本模块探讨了沙箱技术以及黑客如何逃逸配置不当的沙箱。

## 资源

- chroot 设置绝对路径解析的根目录
    - 若之后未使用 chdir 切换到对应目录
        - 当前工作目录在设置目录外，使用当前工作目录的相对路径遍历
    - 若在 chroot 前拥有打开的外部目录fd
        - 使用 openat 打开目录下的文件
            - 若为使用绝对路径的软链接，则解析时使用 chroot 设置的绝对路径根目录而无法逃逸
        - 调用 fchdir 使用外部目录fd，切换工作目录到外部目录

- seccomp
    - 宽松的策略
        - 大量 syscall
        - 沙箱进程使用 `ptrace()` 控制外部进程
        - `sendmsg()` 传递 fd对象，而不只是 fd数字
        - `prctl()` 
        - `process_vm_writev()` 访问其他进程的内存
    - 系统调用混淆
        - 拥有多个架构接口
            - exit() syscall 60 (mov rax, 60; syscall) on amd64
            - exit() syscall 1 (mov eax, 1; int 0x80) on x86
    - 内核漏洞
    - 数据泄露/带外，逐比特提取信息
        - 运行时间（sleep/无限循环）
        - 正常退出或崩溃
        - 返回值/退出码（exit）

    - 语法功能
        ```c
        // 设置过滤器变量
        scmp_filter_ctx ctx;
        // 设置初始化状态，此处默认为 kill 终止
        ctx = seccomp_init(SCMP_ACT_KILL);
        // 添加过滤规则，此处添加 open 系统调用到 SCMP_ACT_ALLOW 白名单
        seccomp_rule_add(ctx, SCMP_ACT_ALLOW, SCMP_SYS(open), 0)
        // 应用过滤器
        seccomp_load(ctx)
        // 添加额外的 32位架构
        seccomp_arch_add(ctx, SCMP_ARCH_X86);
        ```
        - 若添加额外架构 `seccomp_arch_add`，位于设置规则之后，那么此额外架构仅使用初始化的规则
            - 若初始化为允许，主架构设置禁止条目，之后添加额外架构。那么额外架构不应用主架构的禁止条目，而是全部允许。

- namespace
    - 语法功能
    ```c
    // 调用 unshare 分离共享命名空间
    unshare(CLONE_NEWNS)

    // 创建之后用作新根目录的临时目录 /tmp/jail-XXXXXX
    mkdtemp(new_root)

    // 将根目录 "/" 改为递归私有挂载，修改仅限当前命名空间，不会传播到外面
    mount(NULL, "/", NULL, MS_REC | MS_PRIVATE, NULL) 

    // 将新根目录绑定挂载到自身，将目录创建为挂载点
    mount(new_root, new_root, NULL, MS_BIND, NULL) 

    // 在新目录下设置 /old 目录 /tmp/jail-XXXXXX/old
    mkdir(old_root, 0777)

    // pivot_root 移动进程的根文件系统
    // 新根文件目录 /tmp/jail-XXXXXX
    // 旧根文件目录 /tmp/jail-XXXXXX/old
    syscall(SYS_pivot_root, new_root, old_root)

    // 在新根目录下创建 /tmp/jail-XXXXXX/bin
    mkdir("/bin", 0755)
    // 将旧根文件目录的 /bin 挂在到新根文件目录 /tmp/jail-XXXXXX/bin
    mount("/old/bin", "/bin", NULL, MS_BIND, NULL)

    // 卸载并移除旧根目录
    umount2("/old", MNT_DETACH)
    rmdir("/old") 
    ```

- tips：
    - 执行挑战程序前在 shell 或 pwntool 打开 fd，则可从父进程继承或重定向 fd 到作为子进程的挑战程序中
        - shell 重定向 fd 并打印 `exec {d}</; echo $d`
        - pwntool 设置参数为 `close_fds=False`，调用 `os.open('/', 0)`，似乎默认不继承 fd
    - ns 中使用外部挂载点。若无法直接使用，但拥有 root 权限时：
        - 任意挂载点均可设置目录/文件权限
        - /proc 挂载点可访问其他进程的 fd
        - 若挂载点设置为只读，重新挂载以解除只读限制

### 技巧、诀窍与延伸阅读

一些针对挑战问题的技巧与诀窍！

- 务必仔细理解挑战程序执行的时间线。在 `chroot()` *之前*打开的文件与在 `chroot()` *之后*打开的文件截然不同。操作的顺序至关重要。
- 对 shellcode 没有任何限制（除了系统调用），因此我们强烈建议确保你的 shellcode 能干净地退出。这将使调试更容易。
- 你可以通过编写一个快速的 C 程序来确定诸如 `AT_FDCWD` 等常量的值，该程序包含相关头文件并执行 `printf("%d\n", AT_FDCWD);`。
- 如果你不以 root 身份运行，`chroot()` 将会失败。`strace` 会导致 SUID 位被忽略，因此你必须使用 `sudo strace` 才能正确跟踪这些挑战。当然，这仅在练习模式下可行。
- strace 有一个已知问题，在某些配置下，它无法正确解析 amd64 中 32 位系统调用的系统调用号。使用较新的 Linux 虚拟机有时会有所帮助。如果你使用 `int 0x80` 触发系统调用，那么使用的正是 32 位调用；strace 只是在向你撒谎。
- 关于 32 位系统调用：你不必以 32 位模式汇编你的 shellcode（即，你不需要 `-m32`）。在原本是 64 位的 shellcode 中间直接使用 `int 0x80` 是完全有效的。
- 仔细阅读[此文档](https://www.gnu.org/software/bash/manual/html_node/Redirections.html)，特别是第 3.6.1 节。


## 文件系统沙箱

### 沙箱：chroot

### chroot-逃逸-基础

逃逸一个基础的 chroot 沙箱！

```c
// 创建一个临时目录
char jail_path[] = "/tmp/jail-XXXXXX";
assert(mkdtemp(jail_path) != NULL);

// 使用 chroot 修改根目录为临时目录
assert(chroot(jail_path) == 0);

// 创建临时目录的 /flag，写入假 flag 字符串
int fffd = open("/flag", O_WRONLY | O_CREAT);
write(fffd, "FLAG{FAKE}", 10);
close(fffd);

// 打开参数文件并发送
sendfile(1, open(argv[1], 0), 0, 128);
```
- 并未使用 `chdir()` 切换到临时目录，使用相对路径遍历的参数得到 flag

### chroot-shellcode

通过使用 shellcode 逃逸一个基础的 chroot 沙箱。

```c
// 参数中不可包含 `flag` 字符串
assert(strstr(argv[1], "flag") == NULL);

// 打开传入参数对应的文件，不跟随链接
int fd = open(argv[1], O_RDONLY|O_NOFOLLOW);

// 创建临时目录
char jail_path[] = "/tmp/jail-XXXXXX";
assert(mkdtemp(jail_path) != NULL);

// 使用 chroot 修改根目录为临时目录
assert(chroot(jail_path) == 0);

// 创建临时目录的 /flag，写入假 flag 字符串
int fffd = open("/flag", O_WRONLY | O_CREAT);
write(fffd, "FLAG{FAKE}", 10);
close(fffd);

// 在 0x1337000 创建拥有读写执行的内存页
void *shellcode = mmap((void *)0x1337000, 0x1000, PROT_READ|PROT_WRITE|PROT_EXEC, MAP_PRIVATE|MAP_ANON, 0, 0);
assert(shellcode == (void *)0x1337000);

// 读取载荷到内存页
int shellcode_size = read(0, shellcode, 0x1000);

// 执行内存页的 shellcode
((void(*)())shellcode)();
```
- 在设置临时目录前打开参数文件，已得到逃逸的 fd

思路：
- 设置参数为根目录 `/`，open得到根目录的 fd
    - 分支1：使用 openat 访问根目录下的 flag 文件
    - 分支2：fchdir 修改当前工作目录到根目录，使用 openat/open 访问当前目录的 flag 文件
- 或与之前挑战相同，由于工作目录在 jail 外，设置相对路径


### chroot-规范

使用 shellcode 逃逸一个 chroot 沙箱。


```c
// 调用 chdir 移动工作目录到  chroot 目录
assert(chdir("/") == 0);
```
- 新增 chdir 设置工作目录
    - 无法直接使用相对路径
- 之前挑战代码使用外部 fd，并不依赖相对路径 

## 系统调用沙箱

### seccomp-基础

使用 shellcode 逃逸一个 chroot 沙箱，但这次仅允许使用以下系统调用：`["openat", "read", "write", "sendfile"]`

```c
// 在之前的 chroot 基础上添加 seccomp 过滤器
// 此处默认为 kill，允许4个系统调用
scmp_filter_ctx ctx;

ctx = seccomp_init(SCMP_ACT_KILL);
assert(seccomp_rule_add(ctx, SCMP_ACT_ALLOW, SCMP_SYS(openat), 0) == 0);
assert(seccomp_rule_add(ctx, SCMP_ACT_ALLOW, SCMP_SYS(read), 0) == 0);
assert(seccomp_rule_add(ctx, SCMP_ACT_ALLOW, SCMP_SYS(write), 0) == 0);
assert(seccomp_rule_add(ctx, SCMP_ACT_ALLOW, SCMP_SYS(sendfile), 0) == 0);

assert(seccomp_load(ctx) == 0);

((void(*)())shellcode)();
```
- 之前调用 openat 使用根目录 fd 作为参数，然后使用 sendfile 发送到标准输出。代码仍可使用

### seccomp-linkat

使用 shellcode 逃逸一个 chroot 沙箱，但这次仅允许使用以下系统调用：`["linkat", "open", "read", "write", "sendfile"]`
- 调用 linkat 使用根目录 fd 作为参数，创建硬链接到 jail 目录后读取

### seccomp-fchdir

使用 shellcode 逃逸一个 chroot 沙箱，但这次仅允许使用以下系统调用：`["fchdir", "open", "read", "write", "sendfile"]`
- 调用 fchdir 修改当前目录为根目录

### seccomp-rechroot

使用 shellcode 逃逸一个 chroot 沙箱，但这次仅允许使用以下系统调用：`["chdir", "chroot", "mkdir", "open", "read", "write", "sendfile"]`
- mkdir -> chroot 创造工作目录与 chroot 根目录的差异
    - chdir 此时可使用相对路径逃逸

### seccomp-唯一

使用 shellcode 逃逸一个 chroot 沙箱，但这次仅允许使用以下系统调用：`["openat", "read", "write", "sendfile"]`
- 移除了使用程序参数打开文件的外部 fd 代码，无法使用之前的外部根目录 fd
- 在父进程 shell或pwntool 重定向或打开fd，传递到子进程。
    - pwntool 需设置 `close_fds=False` 以传递 fd

### seccomp-arch32

使用 shellcode 逃逸一个 chroot 沙箱，但这次仅允许使用以下系统调用：`["close", "stat", "fstat", "lstat"]`

```c
scmp_filter_ctx ctx;

// 使用黑名单，除4个系统调用号外设置 kill 终止
ctx = seccomp_init(SCMP_ACT_ALLOW);
for (int i = 0; i < 512; i++)
{
    switch (i)
    {
    case SCMP_SYS(close):
        printf("Allowing syscall: %s (number %i).\n", "close", SCMP_SYS(close));
        continue;
    case SCMP_SYS(stat):
        printf("Allowing syscall: %s (number %i).\n", "stat", SCMP_SYS(stat));
        continue;
    case SCMP_SYS(fstat):
        printf("Allowing syscall: %s (number %i).\n", "fstat", SCMP_SYS(fstat));
        continue;
    case SCMP_SYS(lstat):
        printf("Allowing syscall: %s (number %i).\n", "lstat", SCMP_SYS(lstat));
        continue;
    }
    assert(seccomp_rule_add(ctx, SCMP_ACT_KILL, i, 0) == 0);
}

// 添加额外的 32位架构
seccomp_arch_add(ctx, SCMP_ARCH_X86);

assert(seccomp_load(ctx) == 0);
```
- 设置主架构的过滤规则后添加额外的 32 位架构。此额外的 32 位架构仅应用初始化规则，此处为全部允许
    - 设置载荷为 32位汇编指令字节
    - 之前使用的 push pop 构造字符串指针的方法将被 32位汇编指令截断
        - 使用已知的 mmap 内存页 0x1337000 后的空位置并填充载荷

### seccomp-最小化

使用 shellcode 逃逸一个 chroot 沙箱，但这次仅允许使用以下系统调用：`["read", "exit"]`。注意："write" 被禁用了！你需要一种创造性的方法从你的进程中提取 flag 数据！

```c
int fd = open(argv[1], O_RDONLY|O_NOFOLLOW);
```
- 任意外部字符串作为 open 参数，使用 /flag
- 使用 exit 退出码逐字节泄露，写入 0x1337000 的空位置后遍历

### seccomp-基于时间

使用 shellcode 逃逸一个 chroot 沙箱，但这次仅允许使用以下系统调用：`["read", "nanosleep"]`。注意："write" 被禁用了！你需要一种创造性的方法从你的进程中提取 flag 数据！
- 添加判断指令，遍历所有可能的字符，若匹配则睡眠，pwntool 中设置小于睡眠时间的超时检测
    - nanosleep 需 struct timespec 结构指针写入 0x1337000 后的空位置
    - pwntool 的 poll 检测进程状态，None 对应继续运行
    - 注意汇编代码中从内存中一定1字节到寄存器
- 字符集范围 `string.ascii_letters + string.digits + '-_{}.'`
   

### seccomp-只读

使用 shellcode 逃逸一个 chroot 沙箱，但这次仅允许使用以下系统调用：`["read"]`。注意："write" 被禁用了！你需要一种创造性的方法从你的进程中提取 flag 数据！
- 修改之前程序，若 hit 则无限循环，仍检测超时
    - 爆破优化：从逐字节爆破修改为逐位爆破，提取字节后 test 掩码来区分对应位的有无，再使用 `|` 拼接，8次得到一个字节

### 进程隔离

逃逸一种不同类型的沙箱，其中受限的子进程只能与父进程通信。

```c
void cleanup(int signal)
{
    puts("Time is up: terminating the child and parent!\n");
    kill(child_pid, 9);
    kill(getpid(), 9);
}


//// main
// 关闭 fd 3~9999 
for (int i = 3; i < 10000; i++) close(i);

// 调用 socketpair 创建套接字对
int file_descriptors[2];
assert(socketpair(AF_UNIX, SOCK_STREAM, 0, file_descriptors) == 0);
// socketpair(AF_UNIX, SOCK_STREAM, 0, [3, 4]) = 0
int parent_socket = file_descriptors[0];
int child_socket = file_descriptors[1];

// alarm 1s后调用 cleanup 终止父子进程
alarm(1);
signal(SIGALRM, cleanup);

// fork
child_pid = fork();
if (!child_pid)
{
    // 子进程
    // 关闭标准 fd 与父进程套接字，仅保留子进程套接字
    close(0);
    close(1);
    close(2);
    close(parent_socket);

    // 分配 rwx 内存页
    void *shellcode = mmap((void *)0x1337000, 0x1000, PROT_READ|PROT_WRITE|PROT_EXEC, MAP_PRIVATE|MAP_ANON, 0, 0);
    assert(shellcode == (void *)0x1337000);

    // 设置 seccomp 过滤器，允许3个系统调用
    scmp_filter_ctx ctx;

    ctx = seccomp_init(SCMP_ACT_KILL);
    assert(seccomp_rule_add(ctx, SCMP_ACT_ALLOW, SCMP_SYS(read), 0) == 0);
    assert(seccomp_rule_add(ctx, SCMP_ACT_ALLOW, SCMP_SYS(write), 0) == 0);
    assert(seccomp_rule_add(ctx, SCMP_ACT_ALLOW, SCMP_SYS(exit), 0) == 0);

    assert(seccomp_load(ctx) == 0);

    // 从子进程套接字读取 shellcode
    read(child_socket, shellcode, 0x1000);

    write(child_socket, "print_msg:Executing shellcode!", 128);

    // 执行内存页中的 shellcode
    ((void(*)())shellcode)();
}

else
{
    // 父进程
    // 从标准输入读取 shellcode 到 shellcode buf
    char shellcode[0x1000];
    read(0, shellcode, 0x1000);

    // 发送 shellcode 到子进程
    write(parent_socket, shellcode, 0x1000);

    while (true)
    {
        char command[128] = { 0 };

        // 从父进程套接字读取最多 128 字节到 command buf
        int command_size = read(parent_socket, command, 128);
        command[9] = '\0';

        // 分为命令字符串与参数字符串
        char *command_argument = &command[10];
        int command_argument_size = command_size - 10;

        // 若命令字符串为 print_msg，则打印参数字符串
        if (strcmp(command, "print_msg") == 0)
        {
            puts(command_argument);
        }
        // 否则，若命令字符串为 read_file，则读取参数字符串对应的文件后发送到父进程套接字
        else if (strcmp(command, "read_file") == 0)
        {
            sendfile(parent_socket, open(command_argument, 0), 0, 128);
        }
        else
        {
            puts("Error: unknown command!\n");
            break;
        }
    }
}
```
- 在初始的 shellcode 之后，进入父子进程通信阶段
    - 子进程发送命令字符串与参数字符串到父进程
    - 父进程发送文件或打印参数字符串
- strace 得到，父子进程套接字分别为 3/4

载荷：
- 子进程发送 write(4, 0x1337060, 16)，0x1337060 处存放 'read_file:/flag\x00' 字符串
- 子进程读取 read(4, 0x133707a, 0x40)
- 子进程发送 write(4, 0x1337070, 0x50)，0x1337070 处存放 'print_msg:' 字符串
- 构造：
    - shellcode
    - 填充
    - 'read_file:/flag\x00' 字符串
    - 填充
    - 'print_msg:' 字符串
    - read 读取的 flag 字符串

## 通过命名空间实现沙箱

### 挂载命名空间

了解使用现代命名空间技术实现不同沙箱方式的含义！但如果沙箱非常草率呢？

```c
char hostname[128];

int main(int argc, char **argv, char **envp)
{
    // 关闭从父进程继承的 fd
    for (int i = 3; i < 10000; i++) close(i);

    // 设置新根目录名称字符串与buf
    char new_root[] = "/tmp/jail-XXXXXX";
    char old_root[PATH_MAX];

    // 调用 unshare 分离共享命名空间
    assert(unshare(CLONE_NEWNS) != -1);

    //// 在命名空间中

    // 创建临时目录
    assert(mkdtemp(new_root) != NULL);

    // 将根目录 "/" 改为递归私有挂载，修改仅限当前命名空间，不会传播到外面
    assert(mount(NULL, "/", NULL, MS_REC | MS_PRIVATE, NULL) != -1);

    // 将新根目录绑定挂载到自身，创建目录为挂载点
    assert(mount(new_root, new_root, NULL, MS_BIND, NULL) != -1);

    // 在新目录下设置 /old 目录
    // /tmp/jail-XXXXXX/old
    snprintf(old_root, sizeof(old_root), "%s/old", new_root);
    assert(mkdir(old_root, 0777) != -1);

    // ！！ pivot_root 移动进程的根文件系统
    // 新根文件目录 /tmp/jail-XXXXXX
    // 旧根文件目录 /tmp/jail-XXXXXX/old
    assert(syscall(SYS_pivot_root, new_root, old_root) != -1);

    //// 虽然挂载点是独立的，但对文件本身的更改会传播到父命名空间
    // 在新根目录下创建 /tmp/jail-XXXXXX/bin
    assert(mkdir("/bin", 0755) != -1);
    // 将旧根文件目录的 /bin 挂在到新根文件目录 /tmp/jail-XXXXXX/bin
    assert(mount("/old/bin", "/bin", NULL, MS_BIND, NULL) != -1);

    // 继续挂载 3 个旧根目录到新根目录
    assert(mkdir("/usr", 0755) != -1);
    assert(mount("/old/usr", "/usr", NULL, MS_BIND, NULL) != -1);

    assert(mkdir("/lib", 0755) != -1);
    assert(mount("/old/lib", "/lib", NULL, MS_BIND, NULL) != -1);

    assert(mkdir("/lib64", 0755) != -1);
    assert(mount("/old/lib64", "/lib64", NULL, MS_BIND, NULL) != -1);

    // 为了避免因权限问题导致的异常行为，简化权限设置
    setresuid(0, 0, 0);

    // 将当前工作目录移动到沙盒中
    assert(chdir("/") == 0);

    int fffd = open("/flag", O_WRONLY | O_CREAT);
    write(fffd, "FLAG{FAKE}", 10);
    close(fffd);

    puts("Executing a shell inside the sandbox! Good luck!");
    assert(execl("/bin/bash", "/bin/bash", "-p", NULL) != -1);

    printf("### Goodbye!\n");
}
```
- 在新命名空间中存在旧的根文件目录

### 挂载-绑定挂载

了解使用现代命名空间技术实现不同沙箱方式的含义！挂载一个看似无害的目录可能有什么危害？

```c
// 参数字符串限制：
// 使用 / 开头
// 不含特定字符
assert(argv[1][0] == '/');
assert(strstr(argv[1], ".") == NULL);
assert(strstr(argv[1], "flag") == NULL);
assert(strstr(argv[1], "root") == NULL);
assert(strstr(argv[1], "tmp") == NULL);
assert(strstr(argv[1], "var") == NULL);
assert(strstr(argv[1], "run") == NULL);
assert(strstr(argv[1], "dev") == NULL);
assert(strstr(argv[1], "fd") == NULL);
// 参数字符串若包含 home，则必为 /home/hacker
// 否则确保目录下不包含任何目录
if (strstr(argv[1], "home")) assert(strcmp("/home/hacker", argv[1]) == 0);
else
{
    puts("... to minimize shenanigans, we only support your home dir or a non-writable leaf directory (no subdirs).");
    struct stat statbuf;
    struct dirent *dent;
    char dirpath[1024];
    DIR *dir;

    assert(lstat(argv[1], &statbuf) != -1);
    assert(S_ISDIR(statbuf.st_mode));
    assert(dir = opendir(argv[1]));
    while ((dent = readdir(dir)) != NULL)
    {
        if (strcmp(dent->d_name, ".") == 0 || strcmp(dent->d_name, "..") == 0) continue;
        snprintf(dirpath, 1024, "%s/%s", argv[1], dent->d_name);
        printf("... making sure %s is not a directory\n", dirpath);
        assert(stat(dirpath, &statbuf) != -1);
        assert(!S_ISDIR(statbuf.st_mode));
    }
    closedir(dir);
}

// 切换根目录
assert(syscall(SYS_pivot_root, new_root, old_root) != -1);

// 将使用 / 开头的字符串对应的目录挂载到 /old，即为原目录
// 权限设置为只读，无法尝试修改 chown chmod 
char dirpath[1024];
snprintf(dirpath, 1024, "/old%s", argv[1]);
assert(mkdir("/data", 0755) != -1);
assert(mount(dirpath, "/data", NULL, MS_BIND, NULL) != -1);
assert(mount(NULL, "/data", NULL, MS_REMOUNT|MS_RDONLY|MS_BIND, NULL) != -1);
```

思路：
- 无效：当前用户目录设置了 nosuid 挂载选项，虽然成功设置 suid 到 elf 程序，但内核忽略 suid 位
    - 传递参数 `/home/hacker`
    - 目录以只读 flag 挂载到 ns
    - ns root shellcode 使用 mount 重新挂载

    载荷：
    - `mount(NULL, "/data", NULL, MS_REMOUNT|MS_BIND, NULL)`
    - `chown("/data/mc", 0, 0)`
    - `chmod("/data/mc, 0o4755)`
- 修改上述操作：
    - 第一次启动
        - 传递任意没有 nosuid 挂载选项的目录 `/usr/games`
        - mount 重新挂载目录 `/data`
        - mkdir 创建可写目录 `/usr/games/gg`
            - 然而被 mask 掩码移除了写权限
            - 需在 mkdir 前使用 umask，或 mkdir 后使用 chmod
        - 移动 elf 文件到此目录
    - 第二次启动传递目录参数为 `/usr/games/gg`
        - 设置 suid 权限位后执行

    第一次启动载荷 `/usr/games`：
    - `mount(NULL, "/data", NULL, MS_REMOUNT|MS_BIND, NULL)`
    - `umask(0)` 移除创建文件的 mask
    - `mkdir("/data/gg", 0o777)`
    第二次启动载荷 `/usr/games/gg`：
    - `mount(NULL, "/data", NULL, MS_REMOUNT|MS_BIND, NULL)`
    - `chown("/data/mc", 0, 0)`
    - `chmod("/data/mc, 0o4755)`
