# 竞争条件

想象一下，你正处于一场与时间赛跑的肾上腺素飙升的竞赛中，两个或多个程序正在激烈地竞争访问或更改共享数据。
这就是竞争条件利用的惊险世界！就像大胆的黑客从正在关闭的安全门下滑过一样，这些利用在恰当的毫秒间潜入以篡改数据，导致系统以意想不到且往往是灾难性的方式运行。
在这场数字竞赛中，瞬间便可决定安全与入侵的区别，上演一场在眨眼间展开的高风险戏剧。
通过掌握竞争条件的机制，你不仅是在学习编程，更是在踏入一个时机决定一切的领域，而奖赏则是加固或利用系统漏洞。
你的代码成为系统资源赛道上的高速赛车，理解竞争条件是你通往胜利者圈子的门票！


## 资源

原理：
- 检查/加载 与 执行/使用/保存 的代码并非连续的原子化，而是可被打断或暂停从而存在前后条件或状态不一致的时间窗口

提高竞争攻击成功几率
- 提高目标程序执行时间，复杂命令或超长文本
- （不一定有效）降低程序 cpu/io 优先级，`nice/ionice` 控制程序 cpu/io 优先级
    - 同时运行多个程序
- 增加目录解析时间，设置超长路径（多层目录）
    - 拥有路径长度限制
    - 使用链接以超过 linux 路径长度限制，继续增加路径长度
        - 在同一父目录下，创建超长路径与指向超长路径末尾目录的链接以及目标文件，而末尾目录中设置指向初始父目录的链接。设置的最终到达目标文件的路径中包含大量链接，而每组链接需遍历超长路径
        - 链接也拥有路径解析的数量限制
- 对于拥有线程互斥锁的程序发送信号尝试打断

线程差异：
- setuid：
    - syscall 的 setuid 仅设置对应线程的 uid
    - libc 的 setuid 将使用信号设置所有线程
- exit：
    - syscall 的 exit 仅退出对应线程
    - libc 的 exit 将调用 exit_group() 退出所有线程

工具：
- valgrind 仅在触发竞争条件时有效

提示：
- `__lxstat` 可解析符号，不跟随最后的符号链接，但路径中间的符号链接仍会被解析

### level5.0

利用一个复杂的竞争条件读取flag。
这个竞争条件涉及多个步骤，这使得利用它的可靠性降低！

此挑战将验证文件所在的目录属于root用户，并且其他用户无法在该目录中创建文件。

- 不跟随链接，检查文件本身：路径不含 flag，文件不为链接
- 跟随链接，检查文件路径字符串对应目录的权限：root 为所有者以及组，其他用户不可写

思路：
- 由于此处检查路径字符串对应目录，普通文件与最终的文件链接必然在一层目录中
- root 目录设置同名目录链接或 root 普通文件链接（仅使用字符串而并不检查是否实际为目录）
- 同名目录 + 普通文件
- 同名目录 + flag 文件链接
- 似乎按对应顺序更容易
