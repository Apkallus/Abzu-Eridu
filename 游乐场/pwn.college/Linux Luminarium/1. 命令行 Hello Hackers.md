# Hello Hackers

本模块将教你与命令行交互的**最基础**知识！

**命令行**允许你执行**命令**。
当你启动终端时，它会执行一个命令行“shell”，界面如下所示：

```console
hacker@dojo:~$
```

这被称为“提示符”，它提示你输入命令。
让我们来看看这里的各个部分：

- 提示符中的 `hacker` 是当前用户的**用户名**。
  在 pwn.college 的 DOJO 环境中，用户名是 "hacker"。
- 在上面的示例中，提示符的 `dojo` 部分是 shell 所在机器的**主机名**（例如，如果你是每天需要管理多台机器的系统管理员，这个提示会很有用）。
  在上面的示例中，主机名是 `dojo`，但在 pwn.college 中，它将源自你正在尝试的挑战的名称。
- 我们稍后会介绍 `~` 的含义 :-)
- 提示符末尾的 `$` 表示 `hacker` 不是管理员用户。
  在 pwn.college 后续的许多模块中，当你学会利用漏洞成为管理员用户时，你会看到提示符将显示 `#` 而不是 `$`，那时你就知道自己成功了！

无论如何，提示符正在等待你的命令。
继续完成第一个挑战，学习如何实际执行命令吧！

## 挑战

### 命令简介

在这个挑战中，你将调用你的第一个命令！
当你输入命令并按下回车时，命令将被调用，如下所示：

```console
hacker@dojo:~$ whoami
hacker
hacker@dojo:~$
```

这里，用户执行了 `whoami` 命令，该命令简单地将用户名（`hacker`）打印到终端。
当命令终止后，shell 会再次显示提示符，准备接收下一个命令。

请记住：Linux 中的命令是区分大小写的：`hello` 与 `HELLO` 是不同的。

### 参数介绍

让我们尝试一些更复杂的内容：带有*参数*的命令，参数指的是传递给命令的附加数据。
当你输入一行文本并按下回车时，shell 实际上会将你的输入解析为一个命令及其*参数*。
第一个单词是命令，随后的单词是参数。
观察以下示例：

```console
hacker@dojo:~$ echo Hello
Hello
hacker@dojo:~$
```

在这个例子中，命令是 `echo`，参数是 `Hello`。
`echo` 是一个简单的命令，它会将其所有参数"回显"到终端上，正如你在上面的会话中所见。

让我们看看带有多个参数的 `echo` 命令：

```console
hacker@dojo:~$ echo Hello Hackers!
Hello Hackers!
hacker@dojo:~$
```

在这个例子中，命令是 `echo`，而 `Hello` 和 `Hackers!` 是 `echo` 命令的两个参数。

### 命令历史

你将输入大量命令，而每次都从头开始输入会很麻烦。幸运的是，shell 会保存你调用过的每条命令的历史记录。

你可以使用上下箭头键滚动浏览这些已保存的命令，在其他挑战中，历史记录将包含你运行过的命令日志，因此如果你需要再次运行类似的命令，可以使用箭头键滚动查找！
