# 实验：Token 未与用户会话绑定的 CSRF 漏洞

https://portswigger.net/web-security/csrf/bypassing-token-validation/lab-token-not-tied-to-user-session

本实验的邮箱修改功能易受 CSRF 攻击。它使用 Token 来试图防止 CSRF 攻击，但这些 Token 并未集成到网站的会话处理系统中。

为了解决本实验，请使用你的漏洞利用服务器（exploit server）托管一个 HTML 页面，该页面利用 CSRF 攻击修改查看者的邮箱地址。

你在该应用程序上有两个账户可用于辅助设计攻击。凭据如下：

*   `wiener:peter`
*   `carlos:montoya`

> **提示**
> 你无法注册已被其他用户使用的邮箱地址。如果在测试利用程序时更改了自己的邮箱地址，请确保在向受害者交付最终利用程序时使用另一个不同的邮箱地址。

## 步骤

探测
- 使用邮箱修改功能

    ```http
    POST /my-account/change-email HTTP/2
    Cookie: session=tA6SOwuf0uoShWi6APbhNkfgtTvc7Llv

    email=a%40b&csrf=fvVnhUJROMkwDSrstx4slJiR6LxFDeGF
    ```
    - 存在 csrf 令牌参数

- 尝试移除 csrf 参数
    - 失败
- 尝试重复使用 csrf 令牌
    - 失败
- 开启无痕窗口（不继承当前账户 cookie），登录另一个测试账户 carlos
    - 在 wiener 的会话中使用 carlos 的 csrf 令牌
        - 成功
- 即，应用程序的 csrf 令牌为一次性，但并不验证令牌归属

攻击
- 使用 burp csrf poc 构造载荷
    - 获取一个未被使用的 csrf 令牌
    - 在载荷中设置此令牌
