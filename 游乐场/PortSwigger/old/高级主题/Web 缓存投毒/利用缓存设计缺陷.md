## 串联网页缓存中毒漏洞
实验：结合 web 缓存投毒漏洞
https://portswigger.net/web-security/web-cache-poisoning/exploiting-design-flaws/lab-web-cache-poisoning-combining-vulnerabilities


浏览主页
GET / HTTP/1.1

响应中包含的脚本为
```html
<script>
    data = {"host":"0a9700a00367373f8666005b00c7005b.web-security-academy.net","path":"/"}
</script>

<script type="text/javascript" src="\resources\js\translations.js"></script>

<script>
    initTranslations('//' + data.host + '/resources/json/translations.json');
</script>

```

脚本 translations.js 内容为
```javascript
function initTranslations(jsonUrl)
{
    const lang = document.cookie.split(';')
        .map(c => c.trim().split('='))
        .filter(p => p[0] === 'lang')
        .map(p => p[1])
        .find(() => true);

    const translate = (dict, el) => {
        for (const k in dict) {
            if (el.innerHTML === k) {
                el.innerHTML = dict[k];
            } else {
                el.childNodes.forEach(el_ => translate(dict, el_));
            }
        }
    }

    fetch(jsonUrl)
        .then(r => r.json())
        .then(j => {
            const select = document.getElementById('lang-select');
            if (select) {
                for (const code in j) {
                    const name = j[code].name;
                    const el = document.createElement("option");
                    el.setAttribute("value", code);
                    el.innerText = name;
                    select.appendChild(el);
                    if (code === lang) {
                        select.selectedIndex = select.childElementCount - 1;
                    }
                }
            }

            lang in j && lang.toLowerCase() !== 'en' && j[lang].translations && translate(j[lang].translations, document.getElementsByClassName('maincontainer')[0]);
        });
}
```
当cookie中选择语言lang属性的值非en，且json对象拥有 translations 属性时
将匹配的字符属性的值使用不安全的 innerHTML 复制到html元素 maincontainer类的节点中 :
在主页为
```

|
```

选择语言时将设置cookie
lang=es

数据 translations.json 内容为

添加主页头部为

目标用户使用en
当设置语言为es时，浏览器请求
/setlang/es?

并重定向到 
Location: /?localized=1

那么设置这两个路径，自动触发翻译页面加载
在主页 / 设置路径覆盖，触发cookie中lang设置
X-Original-Url: /setlang/es?

在 /?localized=1 设置从控制页面导入json
X-Forwarded-Host: exploit-0a38007b03c437a985b1ff6f013000c8.exploit-server.net

然而 /setlang/es? 的响应无法缓存
示例解决方案：使用反斜杠触发规格化，实现重定向
> 观察到首页有时使用反斜杠作为文件夹分隔符。注意服务器通过重定向将这些规范化为正斜杠。
> 因此，X-Original-URL: /setlang\es 会触发一个 302 响应，重定向到 /setlang/es
