# 使用 DirBuster 发现 Web 应用

如今的 Web 应用通常由多个目录和文件组成，而这些目录和文件并不总是直接链接在主网站上。它们可能是隐藏的，有时甚至被遗忘，但仍然可以被公众访问，并且可能包含敏感信息或安全漏洞。

因此，对于渗透测试人员来说，评估 Web 服务器上是否存在目录或应用程序非常重要。通常的做法是使用像 DirBuster 这样的发现工具，通过潜在目录和文件名的列表进行枚举，并检查它们是否存在于 Web 服务器上。请记住，结果的质量在很大程度上取决于所使用的目录列表是否完整且是否针对 Web 应用进行了良好定制。

## 如何使用 DirBuster？

DirBuster 附带了 9 种不同场景的目录列表，它们位于 `/usr/share/wordlists/dirbuster` 文件夹中。要在 Kali Linux 上启动 DirBuster，请在终端窗口中执行以下命令：

```bash
dirbuster -u http://10.150.150.11 -l /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt
```

执行上述命令将打开 DirBuster 的图形界面，你可以在其中进行最终配置并开始扫描。DirBuster 随后会通过图形界面中的相应选项卡报告找到的任何文件和目录。
