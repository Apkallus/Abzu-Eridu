# 使用 Nmap NSE 进行漏洞扫描

nmap NSE（Nmap 脚本引擎）是 nmap 最强大的功能之一。它允许任何用户创建或使用脚本来自动化各种网络任务。NSE 可用于网络发现、服务版本检测、漏洞检测、后门检测以及漏洞利用。

## 如何使用 nmap NSE 扫描漏洞？

要使用此功能，你需要在 nmap 命令中添加 `--script` 参数，并指定要用于扫描的脚本或脚本类型。

例如，下面我们指定只使用类别为 `default`（默认）和 `safe`（安全）的脚本，以避免意外破坏正在运行的系统：

```bash
nmap --script "default and safe" -p 80,139,445 hostname/IP
```

![默认和安全脚本扫描结果](/Content/img/lessons/default_safe_nseYYYY10DD1710SS.jpg)

另一种可能性是运行以下命令，针对目标测试所有 `smb-vuln*` 脚本：

```bash
nmap --script "smb-vuln*" hostname/IP
```

![SMB漏洞脚本扫描结果](/Content/img/lessons/smb_etternalYYYY10DD1710SS.jpg)

正如我们所见，两次扫描的结果完全不同，但它们的风险等级也不同。渗透测试人员必须在潜在结果与扫描/攻击可能带来的后果之间取得平衡，但最重要的是，他或她绝不能执行未经许可的操作。

还可以向脚本传递参数：

```bash
nmap --script snmp-sysdescr --script-args creds.snmp=admin 10.150.150.10
```

有关 nmap NSE 用法的更多信息：<https://nmap.org/book/nse-usage.html>
