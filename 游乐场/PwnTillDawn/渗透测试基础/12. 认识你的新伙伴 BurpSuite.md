# 认识你的新伙伴 BurpSuite

BurpSuite 被誉为 Web 应用渗透测试人员的瑞士军刀。它包含多种工具，可以帮助测试 Web 应用程序的过程。在本课中，我们将介绍 BurpSuite 中某些工具的基本用法。

## 入门指南

**下载 BurpSuite**：如果你使用的是 Kali Linux，BurpSuite 应该已经安装好了。否则，你可以[在此处](https://portswigger.net/burp/releases/professional-community-2020-7?requestededition=community)下载免费的 BurpSuite 社区版。（请确保选择正确的操作系统选项）

**配置浏览器**：为简单起见，我们使用 Mozilla Firefox。在地址栏中输入 `about:preferences` 进入设置页面。向下滚动到“网络设置”，点击“设置...”。选择“手动代理配置”，将 IP 设置为 `127.0.0.1`，端口设置为 `8080`。勾选“也为 FTP 和 HTTPS 使用此代理”。点击“确定”。

**安装 Burp 证书**：如果你打算拦截 HTTPS 流量而不仅仅是 HTTP，则需要进行此步骤。要在 Firefox 浏览器中安装 Burp 证书，请按照[此处的说明](https://portswigger.net/support/installing-burp-suites-ca-certificate-in-firefox#:~:text=Click%20on%20%22Preferences%20%2F%20%22Options,Click%20%22View%20Certificates%22.&text=Click%20%22Import%22%2C%20select%20the,saved%20and%20click%20%E2%80%9COpen%E2%80%9D)进行操作。

## 使用 Burp Proxy 拦截流量

**代理**：Burp Proxy 用于拦截 HTTP 流量（请求/响应），并允许你即时修改它们。直接修改 HTTP 请求也是绕过客户端输入验证的一种方式。

在 Proxy > Intercept 选项卡下，点击 **“Intercept is off”** 按钮，确保其显示为 **“Intercept is On”**。

![Burp 拦截开关](/Content/img/lessons/PTD-Burp-InterceptYYYY10DD0910SS.png)

你现在正在拦截 Firefox 浏览器上的所有 Web 流量（**提示**：为了确保不拦截不必要的流量，请关闭 Firefox 浏览器中除目标网页之外的所有其他标签页）。这使你可以检查浏览器发送的每一个请求，并查看传输了哪些信息。

![Burp 请求拦截示例](/Content/img/lessons/PTD-Burp-Req-InterceptYYYY10DD0910SS.png)

BurpSuite 将持有这些请求，允许你在转发或丢弃它们之前进行修改。

### 代理历史

HTTP 历史记录选项卡会记录浏览器与 Web 应用程序之间的所有请求和响应。此外，它还允许我们对 Burp 已“看到”的浏览器执行的请求执行各种操作。下面列出的列表允许通过各个列查看和聚合请求，点击某个条目可以查看请求和响应的详细信息。

![Burp 历史记录](/Content/img/lessons/PTD-Burp-history-reqYYYY10DD0910SS.png)

HTTP 历史记录选项卡中还包括的另一项功能是将各种请求和响应发送到其他工具。默认配置允许发送到：Intruder、Repeater、Sequencer 和 Comparer：

![发送到其他工具](/Content/img/lessons/PTD-Burp-history-sendYYYY10DD0910SS.png)

只需右键单击要发送到你选择的工具的请求，Burp 就会在该工具中准备好该请求。

## 使用 Burp Repeater 修改和重放流量

**Repeater**：Burp Repeater 允许你多次修改和重放 HTTP 请求（这在测试特定网页/功能时非常有用，这样你就不必每次都从 Web 浏览器手动点击触发 HTTP 请求。它对于重放旧的 HTTP 请求也很有用）。

![Burp Repeater](/Content/img/lessons/PTD-Burp-repeatYYYY10DD0910SS.png)
