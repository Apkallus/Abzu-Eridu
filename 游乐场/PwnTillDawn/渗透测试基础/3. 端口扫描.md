# 端口扫描

### 服务发现

渗透测试中最重要的一步是信息收集。盲目地投递漏洞利用既无效也不安全。作为渗透测试人员，你必须明白，尽可能多地收集目标信息对于测试的执行至关重要。这可以通过多种方式完成，但本质上是一个让渗透测试人员发现目标上运行的服务的过程。这些服务是在目标上执行并运行的应用程序，它们开放了监听端口，准备接受连接。通过评估这些开放端口、它们返回的信息或当特定数据发送给它们时的行为，我们不仅可以发现该端口上运行的应用程序名称，还可以发现其版本，有时甚至是配置信息。

适用于此任务的最常用工具是 nmap。Nmap 是一款允许渗透测试人员安全地扫描整个网络以寻找主机及其开放端口的脚本。

### 如何进行端口扫描？

重要的是要知道，即使是 nmap 也可能导致拒绝服务——即导致其正在扫描的应用程序崩溃。这就是为什么 nmap 提供了一系列“开关”，允许用户根据渗透测试人员和环境的需求精确地定制扫描。nmap 的输出差异将取决于这些开关。甚至输出方法也可以更改。

请考虑 nmap 的最基本用法：

`nmap hostname/IP`

![nmap basic scan](https://online.pwntilldawn.com/Content/img/lessons/nmap_basicYYYY10DD1610SS.jpg)

该命令的输出是一个开放端口列表。请注意高亮部分：nmap 告诉我们它没有显示其余的 986 个端口，这意味着（加上列出的 14 个开放端口）它总共扫描了 1000 个端口。这是因为 nmap 内置了一份常见端口列表，当没有指定特定端口列表时，它将默认扫描这些端口。这并不意味着主机上的所有其他端口都关闭了，还有 64535 个端口需要检查，但 nmap 没有检查它们，因为我们没有指示它这样做。继续看实际开放端口列表，第一行说明这些端口是 TCP 类型，第二行显示列出的端口处于开放状态。这意味着主机上有一个使用该端口的服务正在运行。默认情况下，nmap 会提供服务列，但对于如此基本的扫描，这些只是默认使用该特定端口的最常见应用程序的名称。要获取更详细的信息，我们必须使用带开关的 nmap。

我们将讨论的第一个开关是将连接类型（nmap 默认扫描 TCP 开放端口）更改为 UDP：

`nmap -sU hostname/IP`

![nmap udp scan](https://online.pwntilldawn.com/Content/img/lessons/nmap_udpYYYY10DD1610SS.jpg)

我们可以看到输出发生了变化，nmap 将端口报告为 `open|filtered`，因为 UDP 是一种无状态协议，不会发送回传来确认已收到传输。在测试过程中，务必记得检查 UDP 端口。你还可以注意到，UDP 扫描耗时更长。

第二次扫描向我们展示了不同的信息，但仍然仅限于 1000 个最常见端口。它还提供了在这些端口上运行的默认应用程序。让我们接着看本次入门的最后几个开关：

`nmap -Pn -sV -p hostname/IP -oN/-oX/-oS/-oG filename`

![nmap advanced scan](https://online.pwntilldawn.com/Content/img/lessons/nmap_advancedYYYY10DD1610SS.jpg)

让我们按命令行中出现的顺序讨论这些开关。

`-Pn` 告诉 nmap 忽略对主机的初始 ping 扫描，该扫描旨在确定机器是否确实在网络上存在并在线。这在企业网络中特别有用，因为防火墙可能设置为阻止并非来自管理/网络团队子网的 ICMP 流量。

`-p-` 或 `-p[数字范围]` 告诉 nmap 扫描所有端口（`-p-`）或指定范围的端口（例如 `-p69-666`）。

`-oN/-oX/-oS/-oG` 告诉 nmap 分别以正常、XML、脚本小子风格和 Grepable 格式保存扫描输出（这里我们可以看到 `ls` 命令返回了我们正常格式的文件）。

nmap 具有大量的功能和附加脚本，所有这些都可以在 nmap 手册页中看到。

### 总结

始终扫描你正在测试的环境，但需注意，即使是在扫描时，某些端口/服务也可能表现异常或直接崩溃。务必检查 nmap 的选项，并在使用前确保理解它们。
