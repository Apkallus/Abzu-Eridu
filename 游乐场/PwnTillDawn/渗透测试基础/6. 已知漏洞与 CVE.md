# 已知漏洞与 CVE

一旦识别出目标系统上运行的软件版本，接下来的任务是检查该版本是否存在已知漏洞。这可以通过查看该软件版本是否存在 CVE 来实现。

CVE 是 MITRE 的“通用漏洞与暴露”（Common Vulnerabilities and Exposures）的缩写，是一个公开已知漏洞的字典。每个条目都有一个唯一的 ID，由三个元素组成：CVE 前缀 + 年份 + 数字序列。

## 如何搜索已知漏洞和 CVE？

有许多网站包含已知漏洞列表，其中最著名的是 <https://www.cvedetails.com>。根据漏洞的性质、影响的服务或组织，漏洞可能仅对供应商披露、部分公开披露，或者在 MITRE 网站或披露方的网站上完全公开披露。

## 该版本存在已知漏洞，接下来怎么办？

一旦你确定了影响目标服务版本的已知漏洞，就需要查找是否存在公开的漏洞利用代码。

一种选择是使用通用搜索引擎，在私人公司网站和/或博客上查找公告。例如，在 <https://www.wizlynxgroup.com/security-research-advisories/vuln/WLX-2020-011> 上有一个负责任的漏洞披露流程示例，我们可以在其中找到完整的概念验证代码。

另一种选择是使用 Exploit-DB：<https://www.exploit-db.com/>，或者它的离线版本——SearchSploit 命令行工具。SearchSploit 是 Exploit-DB 的命令行搜索工具，允许你随时随地携带 Exploit 数据库的副本。SearchSploit 让你能够通过本地检出的存储库副本进行详细的离线搜索。

搜索影响特定产品和版本的漏洞：

```bash
searchsploit apache 2.2
```

查看特定漏洞利用代码的内容：

```bash
searchsploit -x 4445
```

有关 SearchSploit 的更多信息：<https://www.exploit-db.com/searchsploit>
