# 渗透测试概述

渗透测试领域对新手来说可能显得庞大、复杂，甚至有些令人生畏。尽管在线资源很多，但如果没有与道德黑客社区已有成员的联结，很难决定从哪个主题开始，甚至不知道如何将不同主题串联起来。因此，PwnTillDawn 团队创建了这个入门课程，帮助新手学习执行渗透测试的一种方式，或者至少掌握这个过程的基础知识。本课程传授在 PwnTillDawn 网络中测试和训练你道德黑客技能所需的基本功。你会发现许多实施了真实漏洞的不同系统供你利用。

如果要用一句话来解释什么是渗透测试，我们想到了下面这句：

> 渗透测试是一种通过发起经授权的、真实的网络攻击，并评估所实施的防御机制，来评估计算机系统安全态势的方法。

由于主题的复杂性，结合实际案例动手实践所学内容非常重要。我们建议你在 PwnTillDawn 网络中进行练习。

### 渗透测试方法论

一般来说，渗透测试是一个多步骤的过程。在这次简短的培训中，你最需要了解的是以下几个步骤：

- 侦察
- 漏洞识别
- 漏洞利用
- 报告

#### **侦察**

作为一名渗透测试人员，你通常一开始只有一个 IP 地址、一个 URL 或一个目标网络范围，而没有其他信息。然后，你需要收集关于目标组织的信息，并识别底层组件，如操作系统、正在运行的服务、软件版本等。以下是能帮助你提高成功机率的、一份非详尽的项目清单：

- 公开域名搜索
- DNS 调查
- 公开信息搜索（搜索引擎、社交网络、新闻组等）
- 网络枚举
- 端口扫描、操作系统指纹识别及版本扫描
- 防火墙枚举

#### **漏洞识别**

在第一步之后，你对系统如何运作有了一个整体的了解。现在，识别可能的漏洞至关重要。例如，你可以使用 nmap 内置的脚本引擎（nmap NSE）来枚举目标上的漏洞，或者你可以搜索系统组件及其版本已公开披露的漏洞（Google、[Exploit Database](https://www.exploit-db.com/)、CVE）。如果你的目标是一个 Web 应用程序，你可以评估已发现组件的可能攻击面（例如：登录处的 SQL 注入、论坛中的 XSS、文件上传中的 Webshell、默认用户名和密码等）。总的来说，在这个阶段，你必须释放你内心的“侦探”特质……不放过任何一个细节！

#### **漏洞利用**

既然你发现了一个漏洞，现在是时候通过利用该漏洞来证明它的真实性了。你可以手动创建一个针对目标的漏洞利用程序，也可以修改在 [Exploit Database](https://www.exploit-db.com/) 上找到的现有利用程序，使其适用于你目标的环境并运行它。你可以尝试使用你发现的默认凭据登录系统，或者在登录的密码字段使用 SQL 注入字符串来绕过网站的授权。无论你发现了什么，你都必须利用它来排除任何可能的误报。

密切监视目标对你利用尝试的反应非常重要，因为你可能走在正确的道路上，但仍可能缺少成功利用的某些条件。渗透测试人员常常会在尝试利用一个漏洞数小时后才暂时放弃，然后再改天回来尝试不同的方法。

关键要记住，很多时候，你可能不被允许利用客户网络中发现的漏洞。潜在的漏洞可能带来的后果越严重，你就必须越谨慎，并查阅《工作约定规则》以确认客户是否希望你对它们进行利用。如果不确定，务必在尝试利用你的发现前咨询客户。

#### **报告**

由于撰写报告不属于 PwnTillDawn 平台的一部分，我们在此仅作简要说明。在渗透测试期间，为你所做的一切做好笔记和截图非常重要，因为你必须为客户提供所有必要的工具，以便他们能够通过追溯你所做的步骤来独立验证漏洞。最终，你提供的这些信息就是漏洞存在的证据。

### 渗透测试基础课程

现在你应该对如何进行渗透测试有了一个总体概念，并准备好学习渗透测试人员可能使用的工具。在浏览课程时，请尝试将你正在使用的工具归类到三个步骤之一：侦察、漏洞识别、漏洞利用。
