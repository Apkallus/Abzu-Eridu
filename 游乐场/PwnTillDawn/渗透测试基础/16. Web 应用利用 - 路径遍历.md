# Web 应用利用 - 路径遍历

路径遍历是 Web 应用程序中非常常见的漏洞，它允许攻击者读取正常 Web 服务器目录之外的文件。它通过路径字符串向上遍历目录，从而访问系统文件。利用此漏洞可能获得非常有用的信息，例如配置问题、明文存储的凭据、目标上运行的服务、可用的用户名、服务器密钥、历史文件等。

在渗透测试过程中，寻找直接访问底层文件系统的地方并测试可能的路径遍历漏洞非常重要。

## 如何利用路径遍历？

利用路径遍历漏洞有多种方法。实际上，payload 很大程度上取决于目标系统的上下文，并根据所实施的防火墙、WAF、编码等而变化。

互联网上有多种 payload 列表。最简单的方法是通过父目录向后遍历。

例如，可以通过以下恶意 URL 获取目标系统的系统用户：

```
http://{vulnerablewebsite}/proc/../../../../../etc/passwd
```

根据 Web 应用程序的情况，可能需要对 URL 进行编码。这种情况下，`%2e%2e%2f` 代表 `../`。下面的示例展示了如何使用编码后的 URL 获取目标系统的系统用户：

```
http://{vulnerablewebsite}/proc/2e%2e%2f%2e%2e%2f%2e%2e%2f%2e%2e%2fetc%2fpasswd
```

在某些操作系统中，空字节 `%00` 可用于终止字符串。当 Web 应用程序将文件扩展名附加到文件名时，这非常有用：

```
http://{vulnerablewebsite}/proc/../../../../../var/www/dangerous.txt%00
```
