- cloud.binary.ninja 等静态工具的使用：
    - 无法识别压栈传递的数据
    - `Show Variable Types At Top of Function` 除高级语言类型外在顶部显示所有变量
    - `Type` 选项卡中新增类型
        - 设置未知结构为字符数组展示结构字段
        - 似乎 `Frame offset` 为到达返回地址的整个函数帧偏移 `ret - n`，而汇编显示函数内的 `rsp` 偏移 `rbp - n`
    - 双击函数或数据字段跳转到对应的汇编代码（包含跳转表/字符串/数据段）
    - 右键复制地址
    - 右键变量名以修改类型与名称，修改类型后将自动调整引用或索引
        - 初始，buf 类型为 `int64`：
           
            ```c
            17 @ 004014a9  read(0, &buf, 5)

            26 @ 00401521  buf:2.b = var_12   
            27 @ 00401528  var_12 = buf:2.b
            ```
            - `buf:2.b` >> `2.` 索引2，`b` = 1字节

        - 修改，buf 类型 到 `char[5]`：

            ```c
            27 @ 0040150f  char t = buf[2];   
            28 @ 00401521  buf[2] = buf[4];   
            29 @ 00401528  buf[4] = t;
            ```

跳转：
短跳编码：opcode + imm8（2 bytes）  
近跳编码：0F opcode + imm32（6 bytes）  

- je    `74`    `0F 84`
- jne   `75`    `0F 85`
- jg    `7F cb` `0F 8F cd`
- jge   `7D cb`	`0F 8D cd`
