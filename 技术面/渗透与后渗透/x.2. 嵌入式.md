## 概念
串行接口
- uart 通用异步收发器
	- 使用3个引脚，通常为4引脚形式
	- JTAGulator 工具识别标签引脚
		- 在电路板寻找可能为 uart 的焊点
		- 万用表测量电路板，或识别ic并查表得到电压
		- 测量接地与引脚之间的电阻，寻找接近0的引脚
		- 连接JTAGulator与电路板
		- 查看JTAGulator的固件版本并更新 
		- 启用uart模式，并设置目标电压
		- 运行uart识别测试
		- 查找回车换行符或可读文本以识别对应波特率
		- 以 pass-thru 模式输入识别的波特率
		- 若成功则连接到串行控制台，并显示大量信息
	- 确定布局后 使用Bus Pirate等工具连接焊盘，从而与嵌入式设备通信
- spi 串行外设接口
	- 允许多设备通信，使用主从架构，最简单使用4引脚
		- 专用芯片选择方式
			- 主设备使用专用引脚连接到从设备
			- 增加设备需增加引脚
		- 串联方式
			- 主设备无接收连接，从设备之间收发
			- 拥有芯片数量的延迟
	- 工具
		- Bus Pirate
		- flashrom
- iic 集成电路
	- 多主多从分组化，使用两个引脚
		- 主设备发送地址包，后随数据包
	- 工具
		- Bus Pirate

调试接口
- jtag 电路板/硬件级调试
- swd 对arm架构的调试协议

软件
- 引导加载器：初始化处理器并启动系统软件
	- 流程
		- 微处理器/控制器根据引导模式从外部设备加载小型程序
		- 小型程序加载引导程序的其余部分到内存
		- 引导程序初始化设备后加载主程序，并转移执行流到此程序
- 系统软件
	- 无操作系统
	- 实时操作系统
	- 通用操作系统


## 攻击

静态分析 linux
- 下载更新包
- 使用 binwalk 工具提取更新包
	- 移除现存 binwalk 工具
	- 从 github 项目获取特定版本，运行依赖项安装脚本，安装工具
	- 运行工具提取固件并识别
	- 查看内核镜像与文件系统
- 浏览文件
	- 查找web服务器
	- 查找对应web服务器的配置文件
	- 查看引用链以定位最终二进制执行文件
- 静态分析漏洞，示例为命令注入型
	- 示例目标：sprintf 可控字符串到 cmd，之后使用 system 执行
	- 使用 strings 搜索二进制文件中的命令字符串，得到wget命令
	- 静态反编译工具中
		- 搜索wget字符串，定位后看到被传递到system
		- 追踪wget 的url的控制方式，得到命令与参数字符串
	- rootfs
		- 搜索参数字符串，找到源地址
- 动态分析，示例为命令注入
	- ettercap 工具进行 arp 中间人攻击，从而捕获流量
	- 触发参数字符串对应操作
	- ettercap 过滤器篡改参数字符串添加 reboot 命令
	- 监控 ping 的同时确认执行操作，若成功则设备重启，ping暂时无响应后恢复
- 仿真器动态分析，firmae 工具
	- 安装依赖项并下载
	- 运行并设置制造商与固件文件以进行仿真，得到模拟设备的ip地址
	- 连接到模拟设备ip地址
