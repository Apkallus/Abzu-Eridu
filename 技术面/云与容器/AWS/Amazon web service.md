aws
- 概念：将各种传统功能组件拆分为 aws 服务
- 授权方式
	- aws web 控制台 → 管理员登录
	- aws 命令行接口 →用户、服务、计算机：使用编程访问密钥在系统执行操作
	- aws iam 身份和访问管理 → 用户、服务：授权和权限模型，默认全部拒绝
		- 授权状态、服务操作、资源编号
- 最佳实践
	- 堡垒机
		- 概念：拥有 iam 角色的计算实例称为堡垒机。
		- 使用 shh 控制，仅管理员持有密钥
		- 之后引入的 ssm 系统管理产品允许使用不同的带外管理器
	- 文件储存权限
		- 概念：简单存储服务 s3 在云中存储各种文件
		- 权限配置出错导致储存桶暴露在互联网上
		- 储存桶内数据为明文
- 身份验证控制
	- 方式
		- 标准控制台登录
		- 安全声明标记语言
		- oidc 基于 Oauth2 的 open id connect 身份验证，连接到 kubernetes
		- 用户附加 api 密钥
	- 密钥
		- akia 访问密钥，asia 临时访问密钥
	- 查找密钥
		- 泄露
			- 源码管理系统 github
			- 浏览器代码
			- 移动应用程序硬编码
			- 项目管理环境 trello
		- 身份元数据查询
			- 概念：目标机器的内部ip查询周知身份请求服务地址时 169.254.169.254，将得到元数据。
			- 方式：控制目标机器或服务器端请求伪造
			- 目标：元数据的 meta-data 与 user-data 目录
			- 使用反向dns 查询目标机器ip地址，以判断其设备类型
			- 逐层获取目录下数据直到得到凭证
				- http://169.254.169.254/latest/meta-data/iam/security-credentials
				- http://169.254.169.254/latest/meta-data/iam/security-credentials/获取的标识符
				- 得到密钥与凭证
			- 注
				- imdsv1易受攻击，imdsb2额外增加put方法而阻止服务器端请求伪造攻击
				- 攻击向量需混淆以避免防火墙
				- aws guardduty 检测密钥是否在云内使用
			- 编辑 .aws/credentials 与 .aws/config 两个文件，添加之前获取的凭证
			- 配置 aws 文件后，调用 asw api 查询 asw api key 信息
	- 工具
		- Python 的 boto 库
		- pacu 框架
		- redboto 脚本集合
		- 功能为枚举 iam 权限、提权、禁用保护设置、植入后门等
	- 持久工具
		- userdataswap
			- 基于事件触发的持久化控制
