# Content security policy（内容安全策略）

在本节中，我们将解释什么是内容安全策略，并描述如何利用 CSP 来缓解一些常见攻击。

## 什么是 CSP（内容安全策略）？

CSP 是一种浏览器安全机制，旨在缓解 XSS 和其他一些攻击。它的工作原理是限制页面可以加载的资源（如脚本和图片），以及限制页面是否可以被其他页面嵌入。

要启用 CSP，响应需要包含一个名为 `Content-Security-Policy` 的 HTTP 响应头，其值包含策略。策略本身由一个或多个指令组成，以分号分隔。

## 使用 CSP 缓解 XSS 攻击

以下指令只允许从页面本身的[同源](/web-security/cors/same-origin-policy)加载脚本：

```
script-src 'self'
```

以下指令只允许从特定域加载脚本：

```
script-src https://scripts.normal-website.com
```

允许从外部域加载脚本时应格外小心。如果攻击者有任何方式控制从外部域提供的内容，那么他们可能能够发起攻击。例如，不使用按客户 URL 的内容分发网络（CDN），如 `ajax.googleapis.com`，不应被信任，因为第三方可以在其域上放置内容。

除了将特定域加入白名单之外，内容安全策略还提供了另外两种指定可信资源的方式：随机数（nonce）和哈希（hash）：

- CSP 指令可以指定一个随机数（一个随机值），加载脚本的标签中必须使用相同的值。如果值不匹配，脚本将不会执行。要作为有效的控制措施，随机数必须在每次页面加载时安全生成，并且不能被攻击者猜测。
- CSP 指令可以指定可信脚本内容的哈希值。如果实际脚本的哈希值与指令中指定的值不匹配，脚本将不会执行。如果脚本内容发生变化，当然需要更新指令中指定的哈希值。

CSP 阻止 `script` 等资源是很常见的。然而，许多 CSP 允许图片请求。这意味着你通常可以使用 `img` 元素向外部服务器发送请求，例如为了泄露 CSRF 令牌。

某些浏览器（如 Chrome）具有内置的悬挂标记缓解机制，会阻止包含某些字符（如原始未编码的换行符或尖括号）的请求。

有些策略更加严格，会阻止所有形式的外部请求。然而，仍然有可能通过[诱导一些用户交互来绕过这些限制](https://portswigger.net/research/evading-csp-with-dom-based-dangling-markup)。要绕过这种形式的策略，你需要注入一个 HTML 元素，当被点击时，会存储并发送被注入元素所包含的所有内容到外部服务器。

## 使用 CSP 缓解悬挂标记攻击

以下指令只允许从页面本身的同源加载图片：

```
img-src 'self'
```

以下指令只允许从特定域加载图片：

```
img-src https://images.normal-website.com
```

请注意，这些策略将阻止某些悬挂标记攻击，因为一种无需用户交互即可捕获数据的简单方式是使用 `img` 标签。然而，它不会阻止其他攻击，例如注入带有悬挂 `href` 属性的锚点标签的攻击。

## 通过 HTTP 响应头部的策略注入绕过 CSP

你可能会遇到一个网站将输入反射到实际策略中，很可能是在 `report-uri` 指令中。如果网站反射了一个你可控制的参数，你可以注入分号来添加你自己的 CSP 指令。通常，这个 `report-uri` 指令是列表中的最后一个。这意味着你需要覆盖现有指令才能利用此漏洞并绕过策略。
- `report-uri` 设置浏览器在违反 CSP 策略时，发送 POST 报告的 URL

通常，覆盖现有的 `script-src` 指令是不可能的。然而，浏览器最近引入了脚本细粒度控制指令：
- `script-src-elem 'unsafe-inline'` 
    控制**脚本元素**（`<script>`）
- `script-src-attr 'unsafe-inline'` 
    控制**事件处理器**（`onload`）

至关重要的是，这些新指令允许你[覆盖现有的 `script-src` 指令（portswigger 研究）](https://portswigger.net/research/bypassing-csp-with-policy-injection)。
- 思路是尝试使用类似功能的其他字段覆盖，或使用无效语法使得整个 CSP 无效，或使用冗余字段


## 使用 CSP 防御点击劫持

以下指令只允许页面被同源的其他页面嵌入：

```
frame-ancestors 'self'
```

以下指令将完全禁止嵌入：

```
frame-ancestors 'none'
```

使用内容安全策略来防御点击劫持比使用 `X-Frame-Options` 头更灵活，因为你可以指定多个域并使用通配符。例如：

```
frame-ancestors 'self' https://normal-website.com https://*.robust-website.com
```

CSP 还会验证父框架层次结构中的每个框架，而 `X-Frame-Options` 仅验证顶层框架。

建议使用 CSP 来防御点击劫持攻击。你也可以将其与 `X-Frame-Options` 头结合使用，以便在不支持 CSP 的旧版浏览器（如 Internet Explorer）上提供保护。
