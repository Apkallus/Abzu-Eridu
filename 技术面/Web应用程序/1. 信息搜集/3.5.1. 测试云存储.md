# 测试云存储

## 概述

云存储服务允许 Web 应用程序和服务在存储服务中存储和访问对象。然而，访问控制配置不当可能导致敏感信息泄露、数据篡改或未经授权的访问。

一个众所周知的例子是 Amazon S3 存储桶配置错误，尽管其他云存储服务也可能面临类似风险。默认情况下，所有 S3 存储桶都是私有的，只能由被明确授予访问权限的用户访问。用户不仅可以授予对存储桶本身的公共访问权限，还可以授予对该存储桶内存储的单个对象的公共访问权限。这可能导致未经授权的用户能够上传新文件、修改或读取存储的文件。

## 测试目标

- 评估存储服务的访问控制配置是否妥善到位。

## 如何测试

首先，识别用于访问存储服务中数据的 URL，然后考虑以下测试：

- 读取未经授权的数据
- 上传任意新文件

您可以使用 curl 进行测试，使用以下命令并查看是否可以成功执行未经授权的操作。

测试读取对象的能力：

```bash
curl -X GET https://<cloud-storage-service>/<object>
```

测试上传文件的能力：

```bash
curl -X PUT -d 'test' 'https://<cloud-storage-service>/test.txt'
```

在上述命令中，建议在 Windows 机器上运行时将单引号（'）替换为双引号（"）。

### 测试 Amazon S3 存储桶错误配置

Amazon S3 存储桶 URL 命名约定遵循两种格式之一：虚拟主机风格或路径风格。

- 虚拟主机风格访问

```text
https://bucket-name.s3.Region.amazonaws.com/key-name
```

在以下示例中，`my-bucket` 是存储桶名称，`us-west-2` 是区域，`puppy.png` 是键名：

```text
https://my-bucket.s3.us-west-2.amazonaws.com/puppy.png
```

- 路径风格访问

```text
https://s3.Region.amazonaws.com/bucket-name/key-name
```

如上，在以下示例中，`my-bucket` 是存储桶名称，`us-west-2` 是区域，`puppy.png` 是键名：

```text
https://s3.us-west-2.amazonaws.com/my-bucket/puppy.png
```

对于某些区域，可以使用未指定区域特定端点的旧版全局端点。其格式同样为虚拟主机风格或路径风格。

- 虚拟主机风格访问

```text
https://bucket-name.s3.amazonaws.com
```

- 路径风格访问

```text
https://s3.amazonaws.com/bucket-name
```

注意：在 Amazon S3 中，每个存储桶名称必须在 S3 全局所有现有存储桶名称中唯一。

#### 识别存储桶 URL

对于黑盒测试，S3 URL 可以在 HTTP 消息中找到。以下示例显示了一个存储桶 URL 在 HTTP 响应的 `img` 标签中发送。

```html
...
<img src="https://my-bucket.s3.us-west-2.amazonaws.com/puppy.png">
...
```

对于灰盒测试，您可以从 Amazon 的 Web 界面、文档、源代码和任何其他可用来源获取存储桶 URL。

#### DNS 发现 S3 存储桶

确定网站是否托管在 Amazon S3 存储桶上可以通过一系列调查步骤实现，主要利用 DNS 查询工具。其特定 AWS 区域可以帮助理解站点的托管环境。

例如，考虑对 `flaws.cloud` 等域进行 DNS 查询。

```bash
host flaws.cloud
```

该命令显示解析为 “52.218.201.195” 的几个 IP 地址。访问此地址时，它将重定向到 “aws.amazon.com/s3”，表明 flaws.cloud 托管在 S3 存储桶上。

接下来，你可以获取存储桶的区域，我们可以通过 DNS 请求发现 IP 来执行 dig 和 nslookup：

```bash
nslookup 52.218.201.195
```

DNS 查询解析为 “s3-website-us-west-2.amazonaws.com”。它确认托管在 AWS 区域 “us-west-2”。这是 AWS 用于分布其服务的地理区域之一。因此，该站点也可使用以下域访问：

```
flaws.cloud.s3-website-us-west-2.amazonaws.com
```

或者，你可以直接在 `flaws.cloud.s3-us-west-2.amazonaws.com` 访问存储桶内容。在此替代格式中，省略了 s3-website 部分，可用于访问存储桶，而不是托管的网站。

#### 使用 Google Dorks 识别 S3 存储桶

我们还可以使用被动枚举技术，如 Google Dorks，来识别与特定域关联的配置错误的 S3 存储桶。这种方法可能有助于发现敏感文件。例如，以下查询可用于搜索与 `Paypal.com` 关联的 AWS 存储桶。

```
site:s3.amazonaws.com paypal.com
```

要搜索特定文件扩展名，我们使用 filetype 命令：

```
site:s3.amazonaws.com filetype:xls password
```

注意：如果 S3 存储桶可公开访问，其内容可能已被搜索引擎索引或被 Web 爬虫存档，或保留在第三方数据聚合器中。这意味着敏感数据可能缓存在这些平台上，即使在更改存储桶权限后仍可访问。

以下是一些可用于此目的的常见 Google Dorks 示例：

| 命令 | 描述 |
|------|------|
| `site:s3.amazonaws.com filetype:txt password` | 搜索包含单词 “password” 的文本文件的 S3 存储桶 |
| `site:s3.amazonaws.com filetype:sql` | 搜索包含 SQL 文件的 S3 存储桶 |
| `site:s3.amazonaws.com inurl:backup` | 搜索 URL 中包含 “backup” 的 S3 存储桶 |
| `site:s3.amazonaws.com intext:apikey` | 搜索内容中包含 “apikey” 的 S3 存储桶 |
| `site:s3.amazonaws.com ext:log` | 搜索包含日志文件的 S3 存储桶 |

#### 利用配置错误的 AWS S3 存储桶

有时，S3 存储桶可能配置错误，导致它们可公开访问。要确定 S3 存储桶是否为公开，你可以在 Web 浏览器中输入其 URL。如果收到 “Access Denied” 响应，则存储桶是私有的。相反，公共存储桶将显示其存储的前 1000 个对象列表。

要与 S3 存储桶交互，可以使用 “aws s3” 命令行工具，使用 `s3://` URI 方案。

```bash
# 列出公开储存桶内容
aws s3 ls s3://demo-bucket.redseclabs.com/ --no-sign-request --region us-east-1

# 转储存储桶的内容
aws s3 sync s3://demo-bucket.redseclabs.com/. --region us-east-1 --no-sign-request

# 上传文件
aws s3 cp arbitrary-file s3://bucket-name/path-to-save

## 上传成功时
$ aws s3 cp test.txt s3://bucket-name/test.txt

## 上传失败时
$ aws s3 cp test.txt s3://bucket-name/test.txt
upload failed: ./test2.txt to s3://bucket-name/test2.txt An error occurred (AccessDenied) when calling the PutObject operation: Access Denied

# 删除对象
aws s3 rm s3://bucket-name/object-to-remove
```

#### 利用 S3 身份验证用户组配置错误

访问控制列表（ACL）和存储桶策略是管理对 S3 存储桶资源访问的机制。这些设置中的配置错误可能会无意中授予未经授权用户读取或写入权限。例如，考虑以下配置，其中访问控制列表错误地配置为 “Authenticated users group”，意味着任何拥有 AWS 账户的人都可以读取和写入 AWS 存储桶。

当尝试匿名访问存储桶时，返回 “Access denied” 错误，这通常表示存储桶是私有的：

```bash
aws s3 sync s3://demo-bucket.redseclabs.com/. --region us-east-1 --no-sign-request
```

然而，由于允许任何拥有 AWS 账户的人访问存储桶内容的错误配置，让我们看看如何使用 AWS CLI 实现这一点。

要访问存储桶，我们需要通过配置 AWS 访问密钥 ID 和秘密访问密钥，通过 AWS CLI 进行身份验证。这样做后，你将它们与你机器上的 AWS CLI 关联。CLI 将使用这些凭据来认证你对 AWS 服务的 API 请求。这些细节可以从 AWS 管理控制台检索      
    <https://us-east-1.console.aws.amazon.com/iamv2/home?region=us-east-1#/security_credentials/access-key-wizard>

要配置两个密钥，我们将使用 configure 命令。此命令将提示你输入 AWS 访问密钥、秘密密钥和用于身份验证的默认区域。

```bash
aws configure
```

完成这些配置（包括设置 AWS 凭据和配置 AWS CLI）后，用户可以成功访问存储桶。此访问将根据 ACL 或存储桶策略中设置的权限进行。完成后，可以使用以下命令通过身份验证访问 S3 存储桶。

```bash
aws s3 ls s3://demo-bucket.redseclabs.com/ --region us-east-1
```
