
# 指纹识别 Web 服务器

## 概要

Web 服务器指纹识别是识别目标正在运行的 Web 服务器类型和版本的任务，确定该应用程序是否容易受到攻击。特别是，运行未及时更新安全补丁的旧版本软件的服务器可能容易受到已知的特定版本漏洞的攻击。

## 测试目标

- 确定正在运行的 Web 服务器的版本和类型，以便进一步发现任何已知漏洞。

## 如何测试

用于 Web 服务器指纹识别的技术包括
- banner 抓取
- 诱导对畸形请求的响应
- 使用自动化工具执行结合多种策略的更健壮的扫描

所有这些技术运行的基本前提是相同的。它们都试图从 Web 服务器获取某种响应，然后可以将该响应与已知响应和行为的数据库进行比较，从而匹配到已知的服务器类型。

许多Web服务器公开与Web服务器软件本身和所安装组件有关的详细版本信息。例如，HTTP Server消息头
除Server消息头外，下列位置也可能揭露有关软件类型和版本的信息：
- 建立HTML页面的模板
- 定制的HTTP消息头
- URL查询字符串参数

### Banner 抓取

Banner 抓取：向 Web 服务器发送 HTTP 请求并检查其[响应头](https://developer.mozilla.org/en-US/docs/Glossary/Response_header)

工具
- 用于 HTTP 请求的 `telnet`，
- 用于 TLS/SSL 请求的 `openssl`。

服务器类型和版本可能被暴露。然而，服务器返回的任何信息都可加以定制或进行有意伪造来混淆其服务器信息

在服务器信息被混淆的情况下，测试人员可以根据头部字段的顺序来猜测服务器类型。但同样，许多不同的 Web 服务器可能共享相同的字段顺序，并且字段可以被修改或删除

### 发送畸形请求

Web 服务器可以通过检查其错误响应以及默认错误页面（在没有自定义的情况下）来识别。迫使服务器呈现这些的一种方法是发送故意不正确或畸形的请求。

由于默认错误页面在 Web 服务器类型之间提供了许多差异因素，因此即使在服务器头字段被混淆的情况下，检查它们也是一种有效的指纹识别方法。

默认错误页面信息汇总：
- https://0xdf.gitlab.io/cheatsheets/404

### 使用自动化扫描工具

Web 服务器指纹识别通常作为自动化扫描工具的一项功能包含在内：

- [Netcraft](https://toolbar.netcraft.com/site_report)，一个在线工具，用于扫描站点以获取信息，包括 Web 服务器详细信息。
- [Nikto](https://github.com/sullo/nikto)，一个开源的命令行扫描工具。
- [Nmap](https://nmap.org/)，一个开源的命令行工具，也有 GUI 版本 [Zenmap](https://nmap.org/zenmap/)。

## 修复措施

虽然暴露的服务器信息本身不一定是漏洞，但它可以帮助攻击者利用可能存在的其他漏洞。暴露的服务器信息还可能使攻击者发现特定版本的服务器漏洞，这些漏洞可用于利用未打补丁的服务器。因此，建议采取一些预防措施。这些措施包括：

- 混淆头部中的 Web 服务器信息，例如使用 Apache 的 [mod_headers 模块](https://httpd.apache.org/docs/current/mod/mod_headers.html)。
- 使用加固的[反向代理服务器](https://en.wikipedia.org/wiki/Proxy_server#Reverse_proxies)在 Web 服务器和互联网之间创建额外的安全层。
- 确保 Web 服务器始终更新为最新的软件和安全补丁。
