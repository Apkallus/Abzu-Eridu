## 额外探索：路径简化 —— 省略 `file:` 协议

[XML 文档定义](https://www.xml.com/pub/a/98/08/xmlqna2.html)：
- System Identifier 需使用 URI

[URI 文档定义](https://datatracker.ietf.org/doc/html/rfc3986#section-3.3)：
- URI 可使用 `/` 开头的绝对路径格式

[microsoft .NET 文档](https://learn.microsoft.com/en-us/dotnet/api/system.uri?view=net-10.0#implicit-file-path-support)
- 提供省略 `file://` 的隐式路径支持

[URI 解析文档定义](https://datatracker.ietf.org/doc/html/rfc2396#section-5.2)
- >3) 如果定义了 scheme 组件，说明该引用以方案名称开头，则该引用被解释为绝对 URI，解析到此结束。否则，引用 URI 的 scheme 继承自 base URI 的 scheme 组件。
- >4) 如果定义了 authority 组件，则该引用是一个网络路径引用（network-path），直接跳至第 7 步。否则，引用 URI 的 authority 继承自 base URI 的 authority 组件（如果 URI scheme 不使用 authority 组件，则该值同样为未定义）。
- >5) 如果 path 组件以斜杠字符（"/"）开头，则该引用是一个绝对路径引用（absolute-path），直接跳至第 7 步。
- >6) 如果执行到此步骤，说明我们正在解析一个相对路径引用（relative-path reference）。该相对路径需要与 base URI 的路径进行合并（merge）。虽然合并方式有多种，但我们将使用一个独立的字符串缓冲区来描述一种简单方法。
- >7) 将最终得到的各 URI 组件（包括从 base URI 继承而来的组件）重新组合（recombine），以生成该 URI 引用的绝对形式。
- 即，路径组件中若为 `/` 开头的绝对路径，则不进行路径合并

- [非标准库的 URI 对象](https://github.com/openjdk/jdk/blob/0866799ca8f9540de5b2eee2720960160995a10f/src/java.xml/share/classes/com/sun/org/apache/xerces/internal/util/URI.java#L314) 
    ```java
    public URI(URI p_base, String p_uriSpec) throws MalformedURIException {
        initialize(p_base, p_uriSpec);
    }

    private void initialize(URI p_base, String p_uriSpec)
                            throws MalformedURIException {
        ...
        // 使用 SYSTEM 字符串设置 m_path
        initializePath(uriSpec, index);

        // Resolve relative URI to base URI - see RFC 2396 Section 5.2
        // 当 SYSTEM 字符串为绝对路径时，直接作为路径，并不拼接到基础路径后
        if (p_base != null) {
            absolutize(p_base);
        }
    }

    private void initializePath(String p_uriSpec, int p_nStartIndex)
                    throws MalformedURIException {
        ...
        // 使用 SYSTEM 字符串设置 m_path
        m_path = p_uriSpec.substring(start, index);
        ...
    }

    public void absolutize(URI p_base) {
        ...
        // check for scheme - RFC 2396 5.2 #3
        // if we found a scheme, it means absolute URI, so we're done
        // 由于 SYSTEM 字符串为绝对路径，此时 m_scheme 仍为 null
        // 设置 m_scheme 为（继承） baseSystemId 文件路径的 file 方法
        // ⚠⚠ 此处继承的是当前 XXE 注入的 XML 文件的方法
        // ⚠⚠ 通常 XML 文件直接存在于服务器本地，于是通常为 file 方法
        if (m_scheme == null) {
            m_scheme = p_base.getScheme();
        } else {
            return;
        }
        ...
        // check for absolute path - RFC 2396 5.2 #5
        // 若 SYSTEM 字符串以绝对路径 `/` 开头直接返回
        if (m_path.length() > 0 &&
                m_path.startsWith("/")) {
            return;
        }
        ...
    }

    public String toString() {
        final StringBuilder uriSpecString = new StringBuilder();
        // 返回 `方法:路径` 的 URI 字符串
        if (m_scheme != null) {
            uriSpecString.append(m_scheme);
            uriSpecString.append(':');
        }
        uriSpecString.append(getSchemeSpecificPart());
        return uriSpecString.toString();
    }
    ```

于是本地文件格式
`<!ENTITY xxe SYSTEM "file:///etc/passwd" >`
在双平台均可能被简化为
`<!ENTITY xxe SYSTEM "/etc/passwd" >`
