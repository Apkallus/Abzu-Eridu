# 注入攻击

针对 Web 应用的最广为人知的攻击类型之一是 SQL 注入。SQL 注入是一种专门针对 SQL 数据库的注入攻击，它允许恶意用户要么向现有 SQL 查询提供自己的参数，要么逃逸 SQL 查询并提供自己的查询。自然，由于默认赋予 SQL 解释器较高的权限，这通常会导致数据库被入侵。

SQL 注入是最常见的注入形式，但并非唯一形式。注入攻击有两个主要组成部分：一个**解释器**和一个来自用户的**载荷**，该载荷以某种方式被读入解释器。这意味着注入攻击可能发生在命令行工具（如视频压缩工具 FFMPEG）以及数据库（如传统的 SQL 注入情况）上。

让我们看几种注入攻击形式，以便更好地理解此类攻击需要什么样的应用架构，以及如何构造和传递针对易受攻击 API 的载荷。
