# API 损坏的功能级授权

|ID          |
|------------|
|WSTG-APIT-04|

## 概述

当 API 未能正确限制用户访问某些功能或操作时，就会发生损坏的功能级授权（BFLA）。此漏洞允许攻击者调用他们未被授权执行的敏感功能，如管理功能或其他高权限操作。

BFLA 通常发生在 API 暴露多个为不同用户角色（例如普通用户 vs 管理员）服务的端点，但未能根据用户的授权级别限制对这些功能的访问时。

利用 BFLA 可能导致严重后果，如权限提升、未经授权访问敏感功能（例如管理操作），或暴露仅应特定用户角色可访问的关键功能。

## 测试目标

- 本测试的目标是确定 API 是否强制执行**基于角色或权限的访问控制**，以限制用户访问或执行他们未被授权使用的功能。这确保了功能级安全边界得到正确实施。

## 如何测试

### 识别功能级端点

审查 API 文档（例如 OpenAPI 规范）、流量，或使用拦截代理（例如 **Burp Suite**、**ZAP**）来识别不同的功能级端点。这些可能包括：

- **管理功能**（例如 `/api/admin/deleteUser`、`/api/admin/getAllUsers`）
- **基于角色的操作**（例如 `/api/admin/promoteUser`、`/api/user/createOrder`）
- **用户的关键功能**（例如 `/api/user/withdrawFunds`）

关注不同用户角色（例如普通用户、管理员、合作伙伴、访客）之间的**功能差异**以及提供更敏感功能的端点。

### 操纵基于角色的访问控制

尝试访问或执行 API 端点中暴露的敏感操作，这些操作应根据用户角色受到限制。

以低权限用户（例如访客或普通用户）身份登录，并向执行保留给高权限角色（例如管理员）的敏感操作的端点发送请求。

示例：作为**普通用户**，向以下管理端点发送请求以删除一个随机用户：

```http
POST /api/admin/deleteUser
Authorization: Bearer <普通用户令牌>

{ "userId": "12345" }
```

### 使用不同的 HTTP 方法测试功能级访问

测试各种 **HTTP 方法** 是否存在 BFLA 漏洞：

- **GET**：尝试访问仅高权限用户（例如管理员）可用的信息。
    - 示例：`GET /api/admin/getAllUsers`
- **POST/PUT/PATCH**：尝试修改或创建敏感资源（例如更改用户角色、创建或删除系统关键数据）。
    - 示例：`POST /api/admin/promoteUser { "userId": "12345", "newRole": "admin" }`
- **DELETE**：尝试删除敏感资源，例如删除用户账户或数据。
    - 示例：`DELETE /api/admin/deleteUser/12345`

### 在 GraphQL API 中测试 BFLA

在 **GraphQL API** 中，测试用户是否可以通过修改 GraphQL 查询来调用仅限于高权限角色的功能。

示例：`mutation { deleteUser(id: "12345") { success } }`。

## BFLA 的指标

- **成功利用**：如果低权限用户（例如普通用户或访客）可以执行高权限功能或执行保留给其他角色（例如管理员）的操作。
- **错误响应**：一般来说，配置正确的 API 在调用受限功能时应返回 `403 Forbidden` 或 `401 Unauthorized`，而不是 `200 OK` 响应。
- **不一致的强制执行**：某些端点强制执行基于角色的限制，而其他端点则没有，这表明安全控制不一致。

## 修复措施

为防止 BFLA 漏洞，请实施以下缓解措施：

- **强制执行基于角色的访问控制（RBAC）**：确保 API 在允许访问某些操作之前，在**功能级别**检查用户角色和权限。只应允许授权角色调用敏感功能。
- **最小权限原则**：应用最小权限原则，确保用户只能访问其角色所需的最少功能集。
- **集中式访问控制逻辑**：使用集中式访问控制逻辑以确保所有 API 端点的一致性。这可以避免某些功能可能缺乏适当访问检查的漏洞。

## 工具

- **ZAP**：使用自动化扫描器和手动代理工具来检查 API 请求和响应中是否存在 BFLA 漏洞。
- **Burp Suite**：使用 **Repeater** 或 **Intruder** 以低权限用户身份发送请求，以测试是否实施了功能级限制。
- **Postman**：以不同用户角色手动发送 API 请求并观察响应。
- **模糊测试工具**：使用模糊测试器测试不同的功能参数和方法，以识别潜在的授权弱点。
