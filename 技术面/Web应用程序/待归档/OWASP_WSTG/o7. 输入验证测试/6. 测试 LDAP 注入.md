# 测试 LDAP 注入

|ID          |
|------------|
|WSTG-INPV-06|

## 概述

轻量级目录访问协议（LDAP）用于存储关于用户、主机和许多其他对象的信息。[LDAP 注入](https://wiki.owasp.org/index.php/LDAP_injection)是一种服务器端攻击，可能导致 LDAP 结构中表示的用户和主机的敏感信息被泄露、修改或插入。这是通过操纵随后传递给内部搜索、添加和修改函数的输入参数来实现的。

Web 应用程序可以使用 LDAP 来让用户认证或搜索公司结构内其他用户的信息。LDAP 注入攻击的目标是将 LDAP 搜索过滤器元字符注入到将由应用程序执行的查询中。

[Rfc2254](https://www.ietf.org/rfc/rfc2254.txt) 定义了如何在 LDAPv3 上构建搜索过滤器的语法，并扩展了 [Rfc1960](https://www.ietf.org/rfc/rfc1960.txt)（LDAPv2）。

LDAP 搜索过滤器使用波兰表示法构建，也称为[波兰表示法前缀表示法](https://en.wikipedia.org/wiki/Polish_notation)。

这意味着像这样的搜索过滤器上的伪代码条件：

`find("cn=John & userPassword=mypass")`

将被表示为：

`find("(&(cn=John)(userPassword=mypass))")`

可以通过使用以下元字符将布尔条件和组聚合应用于 LDAP 搜索过滤器：

| 元字符 |  含义              |
|----------|-----------------------|
| &        |  布尔 AND          |
| \|       |  布尔 OR           |
| !        |  布尔 NOT          |
| =        |  等于               |
| ~=       |  近似               |
| >=       |  大于等于         |
| <=       |  小于等于            |
| *        |  任意字符        |
| ()       |  分组括号 |

关于如何构建搜索过滤器的更完整示例可在相关 RFC 中找到。

成功利用 LDAP 注入漏洞可能允许测试人员：

- 访问未经授权的内容
- 规避应用程序限制
- 收集未经授权的信息
- 在 LDAP 树结构中添加或修改对象

## 测试目标

- 识别 LDAP 注入点。
- 评估注入的严重性。

## 如何测试

### 示例 1：搜索过滤器

假设我们有一个使用如下搜索过滤器的 Web 应用程序：

`searchfilter="(cn="+user+")"`

该过滤器由如下 HTTP 请求实例化：

`https://www.example.com/ldapsearch?user=John`

如果将值 `John` 替换为 `*`，通过发送请求：

`https://www.example.com/ldapsearch?user=*`

过滤器将如下所示：

`searchfilter="(cn=*)"`

这将匹配所有具有 'cn' 属性等于任何内容的对象。

如果应用程序容易受到 LDAP 注入攻击，它将显示部分或全部用户的属性，具体取决于应用程序的执行流程和 LDAP 连接用户的权限。

测试人员可以使用试错法，在参数中插入 `(`、`|`、`&`、`*` 和其他字符，以检查应用程序是否出现错误。

### 示例 2：登录

如果 Web 应用程序在登录过程中使用 LDAP 检查用户凭据，并且容易受到 LDAP 注入攻击，则可以通过注入一个始终为真的 LDAP 查询来绕过身份验证检查（类似于 SQL 和 XPATH 注入）。

假设一个 Web 应用程序使用过滤器来匹配 LDAP 用户/密码对。

`searchlogin= "(&(uid="+user+")(userPassword={MD5}"+base64(pack("H*",md5(pass)))+"))";`

通过使用以下值：

```txt
user=*)(uid=*))(|(uid=*
pass=password
```

搜索过滤器将结果如下：

`searchlogin="(&(uid=*)(uid=*))(|(uid=*)(userPassword={MD5}X03MO1qnZdYdgyfeuILPmQ==))";`

这是正确的且始终为真。这样，测试人员将以 LDAP 树中第一个用户的身份获得登录状态。
