# API 损坏的对象级授权

|ID          |
|------------|
|WSTG-APIT-02|

## 概述

当 API 未正确强制执行对客户端访问的每个对象的授权检查时，就会发生损坏的对象级授权（BOLA）。攻击者可以操纵 API 请求中的对象标识符（如 ID、GUID 或令牌），以访问或修改他们未被授权的资源。由于 API 直接访问底层对象以及 API 在现代应用程序中的普遍性，此漏洞在 API 中至关重要。

利用 BOLA 可能导致未经授权访问敏感数据、用户冒充、水平权限提升（访问其他用户的资源）和垂直权限提升（获得未经授权的管理员级访问权限）。

## 测试目标

- 本测试的目标是识别 API 是否强制执行正确的**对象级授权**检查，确保用户只能访问和操作他们被授权交互的对象。

## 如何测试

### 了解 API 端点和对象引用

审查 API 文档（例如 OpenAPI 规范）、流量，或使用拦截代理（例如 **Burp Suite**、**ZAP**）来识别接受相关对象标识符的端点。这些标识符可能以 **ID**、**UUID** 或其他引用的形式存在。

示例：

- `GET /api/users/{user_id}`
- `GET /api/orders/{order_id}`
- `POST /graphql`\
        `query: {user(id: "123") }`

利用上一步获得的知识，审查并收集第三方对象标识符（例如用户 ID、订单 ID 等），这些标识符随后可用于对象标识符操纵。

此外，生成用于暴力破解的潜在对象标识符列表。例如，如果 API 从已认证用户检索采购订单，则生成各种采购订单 ID 以进行测试。

### 操纵 API 请求中的对象标识符

目标是确定用户是否可以通过更改 API 请求中的对象标识符来访问或修改他们不拥有的对象，方法是更改 URL 或请求正文中的对象标识符（例如用户 ID、订单 ID）。

示例：将类似 `GET /api/users/123/profile`（其中 123 是当前用户 ID）的请求修改为 `GET /api/users/124/profile`（其中 124 是另一个用户的 ID）。

根据应用程序上下文，使用两个不同的账户执行测试。使用账户 A 创建仅属于该账户的资源（例如采购订单），并使用账户 B 尝试访问账户 A 的资源（例如采购订单）。

### 使用不同的 HTTP 方法测试对象级访问

测试各种 **HTTP 方法** 是否存在 BOLA 漏洞：

- **GET**：尝试通过操纵请求中的对象 ID 来访问未经授权的对象。
- **POST/PUT/PATCH**：尝试创建或修改属于其他用户的对象。
- **DELETE**：尝试删除其他用户拥有的对象。

### 在 GraphQL API 中测试 BOLA

对于 **GraphQL API**，在查询参数中使用修改后的对象 ID 发送查询（请参阅[测试 GraphQL](https://owasp.org/www-project-web-security-testing-guide/stable/4-Web_Application_Security_Testing/12-API_Testing/01-Testing_GraphQL)）：

示例：`query { user(id: "124") { name, email } }`。

### 测试批量对象访问

测试 API 是否允许未经授权的**批量对象访问**。这可能发生在返回对象列表的端点中。

示例：`GET /api/users` 返回所有用户的数据，而不仅仅是已认证用户的数据。

## BOLA 的指标

- **成功利用**：如果修改请求中的对象 ID 返回了属于其他用户的对象的数据或允许对这些对象执行操作，则 API 存在 BOLA 漏洞。
- **错误响应**：一般来说，配置正确的 API 在未经授权的对象访问时会返回 `403 Forbidden` 或 `401 Unauthorized`。如果对其他用户的对象返回 `200 OK` 响应，则表明存在 BOLA。
- **响应不一致**：如果某些端点强制执行授权而其他端点不强制执行，则表明安全控制不完整或不一致。

## 修复

- **对象所有权检查**：确保对每个 API 请求执行对象级授权检查。始终验证发出请求的用户是否有权访问所请求的对象。
- **基于角色的访问控制（RBAC）**：实施 RBAC 策略，定义哪些角色可以访问或修改特定对象。
- **最小权限原则**：应用最小权限原则，确保用户只能访问其角色所需的最少对象集。
- **使用 UUID 或非连续 ID**：优先使用不可预测、非连续的对象标识符（例如 **UUID** 而不是简单整数），以使枚举和暴力破解攻击更加困难。

## 工具

- **ZAP**：自动化扫描器或手动代理工具可帮助测试 API 请求中的对象引用。
- **Burp Suite**：使用 **Repeater** 或 **Intruder** 工具来操纵对象 ID 并发送多个请求以测试访问控制。
- **Postman**：发送带有更改对象 ID 的请求并观察响应。
- **模糊测试工具**：使用模糊测试器暴力破解对象 ID 并检查未经授权的访问。
