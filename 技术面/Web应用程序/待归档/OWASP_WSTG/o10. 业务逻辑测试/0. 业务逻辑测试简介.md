# 业务逻辑测试简介

在多功能的动态 Web 应用程序中测试业务逻辑缺陷需要以非常规方法进行思考。如果应用程序的身份验证机制被设计为按特定顺序执行步骤 1、2、3 来验证用户身份，那么如果用户直接从步骤 1 跳到步骤 3 会发生什么？在这个简单的示例中，应用程序是开放失败（允许访问）、拒绝访问，还是仅仅以 500 错误信息退出？

可以列举许多示例，但一个永恒的经验教训是“跳出常规思维”。这类漏洞无法被漏洞扫描器检测到，依赖于渗透测试人员的技能和创造力。此外，这类漏洞通常最难检测，且通常是特定于应用程序的，但同时也是对应用程序危害最大的漏洞之一。

业务逻辑缺陷的分类研究不足；尽管业务缺陷的利用在实际系统中经常发生，并且许多应用漏洞研究人员对其进行了研究。最大的关注点在于 Web 应用程序。关于这些问题是否代表特别新的概念，还是仅仅是众所周知原则的变体，社区内存在争论。

业务逻辑缺陷的测试类似于功能测试人员使用的测试类型，这些测试侧重于逻辑或有限状态测试。这类测试要求安全专业人员以不同方式思考，开发滥用和误用案例，并采用功能测试人员所使用的许多测试技术。业务逻辑滥用案例的自动化是不可能的，并且仍然是一门依赖测试人员技能及其对完整业务流程及其规则了解的手工艺术。

## 业务限制与限制条件

考虑应用程序所提供的业务功能的规则。是否存在对人们行为的任何限制或约束？然后考虑应用程序是否强制执行这些规则。如果您熟悉该业务，通常很容易确定测试和分析用例来验证应用程序。如果您是第三方测试人员，那么您将不得不运用常识，或向业务方询问应用程序是否应允许不同的操作。

有时，在非常复杂的应用程序中，测试人员在开始时不会完全了解应用程序的每个方面。在这些情况下，最好让客户带领测试人员了解应用程序，以便在实际测试开始之前更好地了解应用程序的限制和预期功能。此外，在测试期间如果对应用程序功能有任何疑问，与开发人员直接沟通（如果可能）将大有帮助。

## 逻辑测试的挑战

自动化工具难以理解上下文，因此需要人工来执行此类测试。以下两个示例将说明理解应用程序功能、开发人员意图以及一些创造性的“跳出框框”思维如何能够破坏应用程序的逻辑。第一个示例从简单的参数操纵开始，而第二个示例是一个真实世界的多步骤流程案例，导致应用程序被完全颠覆。

**示例 1**：

假设一个电子商务网站允许用户选择要购买的商品、查看汇总页面，然后进行销售结算。如果攻击者能够返回到汇总页面，保持相同的有效会话，并注入较低的商品价格以完成交易，然后结账，会怎样？

**示例 2**：

持有/锁定资源并阻止其他人在线购买这些商品，可能导致攻击者以较低价格购买商品。解决此问题的对策是实施超时和机制，以确保只能收取正确的价格。

**示例 3**：

如果用户能够启动与其俱乐部/忠诚度账户关联的交易，然后在积分已添加到其账户后取消交易，会怎样？积分/信用额是否仍会应用到其账户？

## 工具

虽然有用于测试和验证业务流程在有效情况下正常工作的工具，但这些工具无法检测逻辑漏洞。例如，工具无法检测用户是否能够通过编辑参数、预测资源名称或提升权限来访问受限资源，从而规避业务流程流；也没有任何机制帮助人工测试人员怀疑这种状况。

以下是一些有助于识别业务逻辑问题的常见工具类型。

安装附加组件时，应始终谨慎考虑它们请求的权限以及您的浏览器使用习惯。

### 拦截代理

用于观察 HTTP 流量的请求和响应块

- [Zed Attack Proxy (ZAP)](https://www.zaproxy.org)
- [Burp Proxy](https://portswigger.net/burp)

### Web 浏览器插件

用于查看和修改 HTTP/HTTPS 头部、POST 参数，并观察浏览器的 DOM

- [Tamper Data for FF Quantum](https://addons.mozilla.org/en-US/firefox/addon/tamper-data-for-ff-quantum)
- [Tamper Chrome (for Google Chrome)](https://chrome.google.com/webstore/detail/tamper-chrome-extension/hifhgpdkfodlpnlmlnmhchnkepplebkb)

## 其他测试工具

- [Web Developer 工具栏](https://chrome.google.com/webstore/detail/bfbameneiokkgbdmiekhjnmfkcnldhhm)
    - Web Developer 扩展在浏览器中添加了一个工具栏按钮，提供各种 Web 开发者工具。这是 Firefox 的 Web Developer 扩展的官方移植版。
- [Chrome 的 HTTP Request Maker](https://chrome.google.com/webstore/detail/kajfghlhfkcocafkcjlajldicbikpgnp)
- [Firefox 的 HTTP Request Maker](https://addons.mozilla.org/en-US/firefox/addon/http-request-maker)
    - Request Maker 是一个渗透测试工具。使用它可以轻松捕获网页发出的请求、篡改 URL、头部和 POST 数据，当然，还可以发出新请求。
- [Chrome 的 Cookie Editor](https://chrome.google.com/webstore/detail/fngmhnnpilhplaeedifhccceomclgfbg)
- [Firefox 的 Cookie Editor](https://addons.mozilla.org/en-US/firefox/addon/cookie-editor)
    - Cookie Editor 是一个 Cookie 管理器。您可以添加、删除、编辑、搜索、保护和阻止 Cookie。
