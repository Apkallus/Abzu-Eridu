# 测试文件权限

## 概述

当资源被赋予的权限设置提供了超出必要范围的行为者访问权限时，可能导致敏感信息泄露，或资源被非预期方修改。当资源与程序配置、执行或敏感用户数据相关时，这种风险尤为危险。

一个明显的例子是未经授权的用户可以执行的可执行文件。另一个例子是用于访问 API 的账户信息或令牌值。这些在现代 Web 服务和微服务中越来越常见，可能存储在配置文件中，而在安装时默认具有世界可读权限。此类敏感数据可能被主机系统内的恶意内部人员或远程攻击者暴露。后者可能通过其他漏洞入侵了服务，同时仅获得了普通用户权限。

## 测试目标

- 审查并识别任何不当的文件权限。

## 如何测试

在 Linux 中，使用 `ls` 命令检查文件权限。或者，也可使用 `namei` 递归列出文件权限。

`$ namei -l /PathToCheck/`

需要进行文件权限测试的文件和目录包括但不限于以下内容：

- Web 文件/目录
- 配置文件/目录
- 敏感文件（加密数据、密码、密钥）/目录
- 日志文件（安全日志、操作日志、管理日志）/目录
- 可执行文件（脚本、EXE、JAR、class、PHP、ASP）/目录
- 数据库文件/目录
- 临时文件/目录
- 上传文件/目录

## 修复

正确设置文件和目录的权限，以防止未经授权的用户访问关键资源。
