# 测试 HTTP 严格传输安全

## 概述

HTTP 严格传输安全（HSTS）功能使 Web 服务器能够通过特殊的响应头部告知用户的浏览器，永远不应与指定域名的服务器建立未加密的 HTTP 连接。相反，它应自动将所有访问该站点的连接请求通过 HTTPS 建立。这还可以防止用户覆盖证书错误。

鉴于该安全措施的重要性，务必验证站点是否使用了此 HTTP 头部，以确保 Web 浏览器和服务器之间的所有数据均以加密方式传输。

HTTP 严格传输安全头部使用三个特定指令：

- `max-age`：指示浏览器应将所有 HTTP 请求自动转换为 HTTPS 的秒数。
- `includeSubDomains`：指示所有相关子域名必须使用 HTTPS。
- `preload`（非官方）：指示域名位于预加载列表中，浏览器绝不应在未使用 HTTPS 的情况下连接。
    - 虽然所有主流浏览器都支持此指令，但它并非规范的正式部分。（详见 [hstspreload.org](https://hstspreload.org/)）

以下是 HSTS 头部实现示例：

`Strict-Transport-Security: max-age=31536000; includeSubDomains`

必须检查此头部的存在，因为其缺失可能导致以下安全问题：

- 攻击者截获并访问通过未加密网络通道传输的信息。
- 攻击者利用接受不受信任证书的用户，实施中间人（MITM）攻击。
- 用户在浏览器中错误地输入 HTTP 地址而非 HTTPS，或点击 Web 应用程序中错误使用 HTTP 协议的链接。

## 测试目标

- 审查 HSTS 头部及其有效性。

## 如何测试

- 通过拦截代理检查服务器的响应，确认 HSTS 头部的存在。
- 使用 curl 如下：

```bash
$ curl -s -D- https://owasp.org | grep -i strict-transport-security:
Strict-Transport-Security: max-age=31536000
```
