# 测试应用平台配置

## 概述

即使最简单的Web服务器也带有大量控制其行为的配置选项。各种系统通常带有通用配置，这些配置可能无法很好地适应其在特定站点上所需执行的任务。

虽然典型的 Web 服务器和应用服务器安装会包含大量功能（例如示例应用、文档、测试页面），但在部署之前，应移除所有非必需的内容，以防止安装后的漏洞利用。

## 测试目标

- 确保默认文件和已知文件已被移除。
- 验证生产环境中未遗留调试代码或扩展。
- 审查应用程序已建立的日志记录机制。

## 如何测试

### 黑盒测试

#### 示例文件和已知文件及目录

在默认安装中，许多 Web 服务器和应用服务器会提供示例应用程序和文件，以方便开发人员在安装后立即测试服务器是否正常运行。然而，许多默认的 Web 服务器应用程序后来被发现存在漏洞。例如，CVE-1999-0449（IIS 中安装了 Exair 示例站点时的拒绝服务）、CAN-2002-1744（Microsoft IIS 5.0 中 CodeBrws.asp 的目录遍历漏洞）、CAN-2002-1630（Oracle 9iAS 中 sendmail.jsp 的使用）或 CAN-2003-1172（Apache Cocoon 中 view-source 示例的目录遍历）等。

CGI 扫描器通常包含详细的已知文件和目录样本列表，这些样本来自不同的 Web 或应用服务器，可快速确定这些文件是否存在。

##### 默认内容

大多数Web服务器中含有可用于攻击服务器自身或主目标应用程序的默认内容与功能。以下是一些可能有用的默认内容。

- 为管理员设计的调试与测试功能。
- 用于演示某些常见任务的样本功能。
- 本应禁止公众访问，但无意中允许公众访问的强大功能。
- 包含仅在安装时有用的信息的Web服务器手册。

1.  调试功能

    通常，为方便管理员进行诊断而设计的功能对攻击者极其有用，因为其中包含与服务器和它上面运行的应用程序的配置及与运行状态有关的重要信息。

    如：许多Apache版本中都含有默认页面phpinfo.ph。这个页面运行PHP函数phpinfo()并返回其结果。页面中包含大量与PHP环境、配置设置、Web服务器模块和文件路径有关的信息。

2.  样本功能

    许多服务器默认包含各种样本脚本与页面，其目的在于演示某些Web服务器功能与API的用法。

    - 许多样本脚本包含安全漏洞，可被攻击者用于执行脚本作者不希望执行的操作。
    - 许多样本脚本甚至执行可被攻击者直接利用的功能。

    历史示例：
    - Jetty版本7.0.0中包含的Dump Servlet。此Servlet可以通过/test/jsp/dump.jsp之类的URL访问。一旦被访问，它会打印Jetty安装及当前请求的各种详情，包括请求查询字符串。因此，攻击者只需简单的在URL中包含脚本标签即可实施跨站点脚本攻击。

    - Apache Tomcat中的Sessions Example脚本。此脚本可用于获取并设置任意会话变量。如果在服务器上运行的应用程序将敏感数据保存在用户会话中，攻击者就可以查看这些数据，将通过修改它的值来破坏应用程序的处理过程(加密解密预言机)。

3.  强大的功能

    许多Web服务器软件包含一些公众无法访问的强大功能，但终端用户通过某种方式可以访问这些功能。许多时候，只要提供正确的管理证书，应用程序服务器都允许通过应用程序本身使用的同一HTTP端口来部署Web档案（WAR文件）。应用程序的这种部署过程是黑客的主要攻击目标。常见的渗透测试框架能够自动完成以下过程：扫描默认证书、上传包含后门的Web档案，然后执行该档案以获取远程系统上的命令外壳

    历史示例：
    -   JMX

        JBoss默认安装的JMX控制台。JMX控制台被描述为“JBoss应用程序服务器微内核的原始视图”。实际上，通过它可以直接访问 JBoss 应用程序服务器中的任何托管 Bean。由于可用功能的数量众多，因此，人们从中发现了大量安全漏洞。其中，最简单的利用方法，是使用 DeploymentFileRepository 中的 store 方法创建包含后门的 WAR 文件

    -   Oracle应用程序

        Oracle应用程序服务器实施的PL/SQL网关，最初主要用于传送数据包和过程，但此后发现它的许多默认功能都包含漏洞。其他Oracle产品，如电子商务套件（E-Business Suite）也采用了这个网关。这项功能提供一个接口，通过它可向一个后端Oracle数据库提出Web请求。

        这项功能本用于将某个数据库执行的业务逻辑转换成用户友好的Web应用程序。但是，由于攻击者能够指定任意过程，因此，他可以利用PL/SQL网关访问数据库中的强大功能。例如，SYS.OWA_UTIL.CELLSPRINT过程可用于执行任意数据库查询，从而获取敏感数据

        为防止这种攻击，Oracle引入一个名为PL/SQL排除列表（Exclusion List）的过滤器。该过滤器检查被访问的包的名称，并阻止攻击者访问特定开头的包

        该过滤器旨在阻止攻击者访问数据库中功能强大的默认功能。但是，排除列表并不全面，无法阻止攻击者访问数据库管理员拥有的其他功能强大的默认过程，如CTXSYS与MDSYS。


渗透测试步骤
（1）Nikto之类的工具可有效确定大多数默认的Web内容。应用程序解析过程应已确定所针对的服务器中的绝大多数默认内容。

（2）使用搜索引擎和其他资源确定与已知应用程序使用的技术有关的默认内容与功能。如果可能，可在本地计算机上安装这些技术，从中查找任何可在渗透测试中利用的默认功能。

### 灰盒测试

#### 配置审查

推荐的配置取决于站点策略以及服务器软件应提供的功能。但在大多数情况下，应遵循配置指南（无论是软件供应商还是外部方提供的），应考虑以下常见准则：

- 仅启用应用程序所需的服务器模块（对于 IIS，即 ISAPI 扩展）。这可以减少攻击面，因为禁用软件模块可以缩小服务器的规模和复杂性。同时，如果漏洞只存在于已禁用的模块中，则可以防止供应商软件中的漏洞影响站点。
- 使用自定义错误页面处理服务器错误（40x 或 50x），而不是使用默认的 Web 服务器错误页面。尤其要确保任何应用程序错误都不会返回给最终用户，并且不会通过这些错误泄露任何代码，因为这会帮助攻击者。实际上，这一点常常被忽略，因为开发人员在预生产环境中确实需要这些信息。
- 确保服务器软件以操作系统中的最小权限运行。这可以防止服务器软件中的错误直接危害整个系统，尽管攻击者在以 Web 服务器身份运行代码后仍可能提升权限。
- 切勿向非管理员身份（`NT SERVICE\WMSvc` 除外）授予对 applicationHost.config、redirection.config 和 administration.config 的访问权限（无论是读取还是写入）。
- 切勿通过网络共享 applicationHost.config、redirection.config 和 administration.config。
- 不要在可读的配置文件中存储敏感信息。
- 根据需要加密敏感配置数据。
- 限制对共享配置的访问并适当加以保护。

#### 日志记录

日志记录是应用程序安全的重要组成部分。它支持监控、事件响应和调查。应同时审查平台日志和应用程序日志，以确定日志记录过程是否会引入额外风险。

审查期间，请验证以下事项：

- 安全相关事件被记录。
- 日志不存储秘密或不必要的敏感数据。
- 在存储和显示之前，对记录的数据进行安全处理。
- 对日志的访问仅限于授权人员。
- 日志受到防篡改保护并适当保留。
- 日志记录失败不会泄露敏感调试信息。
