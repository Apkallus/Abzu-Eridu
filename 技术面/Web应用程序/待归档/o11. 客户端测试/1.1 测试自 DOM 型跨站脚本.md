# 测试自 DOM 型跨站脚本

## 概述

自 DOM 型跨站脚本是一种特定攻击，需要具备基于 DOM 的跨站脚本知识和成功的社会工程学手段。这里的“自”指的是用户需要将载荷注入输入字段，从而自行执行该漏洞。该漏洞进一步的特殊之处在于，网站的内容安全策略（CSP）可以阻止脚本的执行。

本场景将按以下方式使用“接收器”一词：在计算中，接收器、事件接收器或数据接收器是一个类或函数，设计用于接收来自其他对象或函数的输入或事件。因此，为了发现可能的漏洞，我们首先需要识别要测试的应用程序的接收器。

## 如何测试

测试自 DOM 型跨站脚本的过程如下：

1. 查找允许用户输入的易受攻击的接收器。
2. 一旦识别出可能的接收器，即可插入载荷。
3. 检查浏览器开发者工具中的错误日志，查看结果并得出结论。
4. 检查攻击者是否能够说服用户插入该载荷，而无需广泛的专业知识。

### 示例

此特定示例来自此 [hackerone 工单](https://hackerone.com/reports/406587)。

在该示例中，以下 JavaScript 函数在网站 `https://example.com` 上执行。

```js
//Marketo Form Code
function strip(html) {
    var tmp = document.createElement("DIV");
    tmp.innerHTML = html;
    return tmp.textContent || tmp.innerText || "";
}

$('form').submit(function() {
    $('textarea').val(function() {
        return strip($(this).val());
    });
});
```

对此功能的滥用可以描述如下：

1. `submit` 事件处理程序将任何 `textarea` 元素的当前值传递给 `strip` 函数。
2. 该函数创建一个新的 `div` 元素，并将 `innerHTML` 属性设置为提供的值。
3. 最后一步，它返回生成的 `div` 的 `textContent` 属性。

此类代码通常用于从字符串中移除 HTML 标签，因为 `textContent` 属性包含浏览器解析 HTML 时渲染的字符串。此特定方法本质上是不安全的，因为它使用了 `innerHTML`。当用户输入提供给 `innerHTML` 属性时，它会被 Web 浏览器解析，因此可能导致恶意 JavaScript 的执行。

以下载荷可用于测试漏洞：`<img src=x onerror=alert(1) />`

开发者控制台将显示两个错误：一个指示 `https://www.example.com/x` 被请求并返回 404（由于 img 标签的 src 属性）。另一个报告违反了网站的 CSP。

第二个错误发生是因为浏览器试图执行 `onerror` 属性中的 JavaScript 代码，但网站的 CSP 阻止了执行。在禁用了 CSP 的浏览器中执行相同的操作，允许了 `onerror` 属性中的 JavaScript 执行。

攻击者可以通过说服用户将恶意载荷粘贴到联系表单的“消息”字段中，然后点击“发送消息”按钮来利用此漏洞。此攻击可以通过说服用户使用不支持 CSP 的浏览器版本来增强。
