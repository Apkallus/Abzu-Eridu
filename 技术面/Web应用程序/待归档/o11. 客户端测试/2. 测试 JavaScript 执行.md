# 测试 JavaScript 执行

|ID          |
|------------|
|WSTG-CLNT-02|

## 概述

JavaScript 注入漏洞是跨站脚本（XSS）的一个子类型，涉及在受害者浏览器内向应用程序注入任意 JavaScript 代码并执行的能力。此漏洞可能产生许多后果，例如泄露用户会话 Cookie，这些 Cookie 可用于冒充受害者，或更一般地，允许攻击者修改受害者看到的页面内容或应用程序的行为。

当应用程序缺乏适当的用户输入和输出验证时，就可能发生 JavaScript 注入漏洞。由于 JavaScript 用于动态填充网页，此注入发生在内容处理阶段，因此会影响受害者。

在测试此漏洞时，请注意某些字符在不同浏览器中的处理方式不同。有关参考，请参见[基于 DOM 的 XSS](https://owasp.org/www-community/attacks/DOM_Based_XSS)。

以下是一个未对变量 `rr` 进行任何验证的脚本示例。该变量通过查询字符串包含用户提供的输入，并且同样未应用任何形式的编码：

```js
var rr = location.search.substring(1);
if(rr) {
    window.location=decodeURIComponent(rr);
}
```

这意味着攻击者只需提交以下查询字符串即可注入 JavaScript 代码：`www.victim.com/?javascript:alert(1)`。

## 测试目标

- 识别接收器和可能的 JavaScript 注入点。

## 如何测试

考虑以下示例：[DOM XSS 练习](http://www.domxss.com/domxss/01_Basics/04_eval.html)

页面包含以下脚本：

```html
<script>
function loadObj(){
    var cc=eval('('+aMess+')');
    document.getElementById('mess').textContent=cc.message;
}

if(window.location.hash.indexOf('message')==-1) {
    var aMess='({"message":"Hello User!"})';
} else {
    var aMess=location.hash.substr(window.location.hash.indexOf('message=')+8)
}
</script>
```

上述代码包含一个由攻击者控制的源 `location.hash`，攻击者可以直接在 `message` 值中注入 JavaScript 代码，以控制用户浏览器。
