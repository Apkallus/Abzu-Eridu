类型：
- suid
    - 以所有者身份作为访问控制身份的 euid 运行的程序
    - 缺乏细粒度控制，违背最小特权原则
- 守护进程/服务
    - 在后台运行的特权用户程序

## suid

攻击面：
- 显式用户输入
    - 换行/分割符等的意外多行数据/多命令执行
        - system 的参数 command 最终作为 shell 的输入执行，存在分隔符注入的额外命令执行漏洞（违背数据代码分离原则）
- 系统输入
    - 可被篡改的目标路径文件
- 隐式环境变量
    - 用户控制
    - 非完整路径时，使用环境变量 PATH 查找路径
        - execlp(), execvp(), execvpe()  
            如果指定的文件名不包含斜杠 (`/`) 字符，这些函数将在 `PATH` 环境变量中指定的冒号分隔的目录路径名列表中查找该文件。
- 权限泄露
    - 降权后未销毁之前的特权能力
        - fd
