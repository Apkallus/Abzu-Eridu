错误使用堆的方式
- 未释放内存
    - 内存占用
    - 敏感数据长期留在内存里
- 释放已经释放的内存
    - 重复释放/或 freelist 被破坏，会让 allocator 的“空闲块目录”出现重复/伪造条目，从而让两次 malloc 返回同一地址或返回异常地址。
- 使用已经释放的内存
    - 若该 chunk 已进入 freelist，其 payload 被当作链表节点字段，继续写可能破坏 freelist；继续读可能读到“分配器写入的指针/结构信息”或新对象数据。
- 堆缓冲区越界
    - 改坏相邻对象数据或 chunk 元数据；或泄露相邻对象/元数据内容。

块结构：
- 元数据
    1. `prev_size` (前一块大小)
        - 在 tcache 和 fastbin 中不被使用
    2. `size` (当前块大小 & 标志位)
        - 存储当前 chunk 的总大小（包括元数据），由于大小是 8 字节对齐的，所以最后 3 个比特位可以借用来作为标志位
        - 标志位 (Flags)：
            1. PREV_INUSE (第 0 位)：最关键的一位。如果为 1，表示前一个相邻的 chunk 正在被使用；如果为 0，则表示前一个 chunk 是空闲的。分配器使用此位判断能否合并前一个空闲块
            2. IS_MMAPPED (第 1 位)：标记该 chunk 是否由 mmap 系统调用分配。如果是，则不参与常规的 bins 管理。
            3. NON_MAIN_ARENA (第 2 位)：标记该 chunk 是否属于非主分配区（thread arena）。
- 用户数据区
    - tcache/fast bins
        - 线程间 tcache 独立，确保在同一个线程进行分配与计算
        1. `*next` 指针到下一块的用户数据区开头
            - 从 glibc 2.32 开始，next 不是明文指针，而是和 heap 地址 XOR 混淆过的值；
            - safe-linking:
                - *next = 下一块的指针^（当前块的 next 槽位地址>>12） 混淆后的值
                    - 右移12位导致
                        - 有效的高3个半字节不变（异或0）
                        - 对于在`同一页`内的堆块，使用相同混淆操作数（两个地址的低3半字节的差异消失）
                        - 由不变的高位即可从同一页相同混淆操作数混淆后的值中恢复真实地址（低位使用高位的值进行异或，回推得到更多解码的高位值，最终得到所有半字节）
                            - 每轮恢复3个半字节：
                                - 混淆值^(混淆值>>12)^(混淆值>>24)^(混淆值>>36)
                                    - 或 mid = 混淆值^(混淆值>>12)
                                        res = mid^(mid>>24)
                                - 或精确的恢复半字节后使用到之后的混淆值
                    - 若下一块指针为 null，则立即得到了混淆操作数
                - 最低半字节必为 `0x0`
                - 若解除混淆后的地址未对齐（最低半字节非 `0x0`），可能表示当前块地址与 next 指向的地址分配到了不同页。尝试额外分配块，以调整块位置

                ```py
                def de_mg(mg_int):
                    # 对于同一页的 next 指针与块中的 next 槽位地址
                    # 由于高位相同，逐步解码还原
                    mid = mg_int^(mg_int>>12)
                    next_ptr = mid^(mid>>24)
                    return next_ptr    

                def mg(next_ptr, next_slot_ptr):
                    # 使用 next 指针与块中的 next 槽位地址混淆
                    mg_int = next_ptr^(next_slot_ptr>>12)
                    return mg_int            
                ```
        2. `key` 安全字段，用于检测 `double free` 错误或攻击
            - `free` 时取 `key` 与进程固定的 `tcache_key` 进行比较，若匹配则阻止释放。攻击时只需修改任意篡改此值即可通过双重释放的检测。
                - 双重释放后， `*next` 指针指向当前地址
                - malloc 将消除 key 值
    - small bins 以上
        - 双向循环链表
        1. `*fd` head -> tail 前向指针到元数据
            - 链尾非 NULL，而是指向表头   
        2. `*bk` tail -> head 后向指针到元数据
            - 链头非 NULL，而是指向表头
    - large bins
        1. `*fd_nextsize` 前向大小分类指针
        2. `*bk_nextsize` 后向大小分类指针

列表：
- 链表长度
    - 线程单向空闲链表 tcache 每个类最多 7 个空闲块
    - 全局缓存单向空闲链表 fast bins 无限块
    - 全局双向空闲链表 small bins 无限块
    - 全局懒缓存双向空闲链表 unsorted bin 无计数
    - 全局双向空闲链表 large bins 无计数
- 列表大小，使用包含元数据的总大小标注
    - tcache <= 0x410 字节
    - fast bins 最高 88（0x58） 字节
    - small bins < 1024（0x400） 字节
    - unsorted bin 不分大小块，任何未进入 small bins 的块
    - large bins >= 1024（0x400） 字节，按照非恒定的大小范围分桶
- 特性
    - tcache 
        - P 位固定置位，无合并
        - 所有块检查双重释放
        - 新块链表头，分配链表头
    - fast bins
        - P 位固定置位，暂时不合并，大型分配将触发延迟合并
        - 仅检查链头块的双重释放
    - small bins
        - 常规清空 P 位，合并
        - 当 small bins 块数量 >1 且 tcache 未满时，分配 small bins 的操作除得到预期的块外，还将触发 small bins 的剩余块进入 tcache
    - unsorted bin
        - 懒归档：分配时检查对应大小的块
            - 若不恰好对应（无论过大还是过小），则将当前块归档至对应大小的列表，并继续查看下一块
            - 若恰好对应则返回
            - 若仍无对应，则查看归档操作后的 bins 是否拥有大于等于目标大小的块
                - 若在其他 bins 中拆分块，则交付对应大小的块，将剩余块放回 unsorted bin
        - 新块到链表头，分配从链表尾
    - large bins
        - 释放时相同大小的新块到当前链表头块之后的第二位置
            - 归档顺序 0/1/2/3 时，从头到尾为 0/3/2/1 
        - 分配时从链表尾取块，即相同大小的块拿第二旧块（第一旧块在头）
        - 尺寸链
            - 链内的相同大小块头的 fd_nextsize 被用来跳转当前链中的下一个大小的块头
            - 小于最大块的其他块进入链尾
            - 除头块外不保存尺寸链指针，即伪造块设置 size 槽位 NULL 可通过检测
- 释放链表
    1. tcache
    2. fast bins
    3. Flags 置位表示 mmap 的 M 则 munmap
    4. 块 > 65KB 则 清理合并后进入 unsorted bin
    5. 块 <= 65KB 则进入 unsorted bin    
- 分配链表
    1. tcache
    2. fast bins
    3. small bins
    4. 清理合并 fast bins 后查看 unsorted bin 
    5. large bins
    - mmap
    - 从荒野分配未使用块
- 合并：
    - 分配释放时若 Flags 未置位表示前一块使用状态的 P 则合并
    - 并非使用桶中的前后向指针，而是使用元数据的 prev_size 定位前一块
    - 当分配大于1024字节（small bings），或释放65KB（魔数）的块时清理归档 fast bins
    - 位于全局堆尾的普通块在释放时与荒野块合并
- 更新
    - 释放/合并
        - 本块的前后向指针字段
        - 堆相邻后块的 prev_size 字段
        - large bins 中当前链表的下一大小块头中的 bk_nextsize 设置下一尺寸头块地址
    - 分配/合并
        - 将从链表移除的块的链表前块的后向指针，链表后块的前向指针 
- 分配检查
    - 内存位置大小：当前块大小，是否等于内存前一块的元数据中保存的下一块大小字段值
        - 仅在块空闲时拥有下一块大小字段，而前一块是否空闲检查当前块的 P 位
            - 设置 P 位以避免内存位置大小检测
            - 设置前一块在尾部的元数据下一块大小字段值
    - 双向链表指针：当前块指针，是否等于链表前一块的后向指针值，以及等于链表后一块的前向指针值

技巧：
- `house of spirit`：
    1. 构造块结构的元数据头部
        - 伪造的 chunk 头为用户数据大小+头部16字节后的16字节对齐，设置前一块是否存在的 flag 为 1 可避免对前一块进行一致性检查；并且避免合并空闲块
        - 猜测：当前一块是否使用的flag置位时，头部按照 +0x8计算后进行16字节对齐
    2. 释放结构中对应用户数据区位置的指针，此时构造块进入空闲列表
    3. 下一次分配将使用此栈中构造块


- tcache poisoning：
    1. 可控块进入空闲列表或已存在于空闲列表的块
    2. 修改已存在于空闲列表中的空闲块的元数据，通常为 *next 指针
        - 新版 glibc 中 next 不是明文指针，而是和 heap 地址 XOR 混淆过的值；
    3. 下一次分配将使用设置的地址

- 重叠分配：
    - 顺序分配块1/2，并分配目标块在块2之后
    - 在块1溢出修改块2 元数据 size，并释放
        - 若在释放后修改，由于此时块已按照原大小进入小桶，于是分配时对应原大小
        - 而在使用中修改，则可进入修改大小后对应的桶，于是分配时对应修改后的大小
    - 修改大小后的分配得到的块将包含目标块，拥有块2相同的位置，以及扩展的大小

触发分配：
- 直接使用 malloc
- 间接使用 printf/scanf 等调用 malloc 的函数
    - setbuf 控制是否使用缓冲区
    - scanf 写入前调用 `malloc_usable_size`，而此函数预期传入的是 malloc 返回的堆块用户指针，若 scanf 的地址不包含块构造则可能崩溃
        - 对于 bss. 段其中包含大量0，于是大小检测时移动指针到元数据位置读取的当前块大小、之前块是否被使用的flag值很可能为0。
            - “可能” 就此返回而无进一步验证，从而可控
        - 对于已拥有值的字段可能对应一个很大的块大小并且 flag 位为已使用
            - “可能” 继续进行之后的块验证或处理导致异常崩溃

可能的错误：
- 篡改后由于 count 计数不足而导致新分配块而不是得到目标块
- 篡改地址无法通过 tcache 16字节对齐检测
- 篡改地址的 safe-linking 异或校验
- 分配释放顺序：空闲链表的新块加入头部，分配时从头部移除
- tcache 向不可写段写入
- 分配释放后 %s 空值：恰好被截断，额外分配块以调整地址
- tcache 的 safe-linking 还原时两个地址不同页：
    - 额外分配块以调整地址

线程：
- 每个线程都有其独有 tcache，分配/计算地址时确保使用同一个线程
    - 仍在内存区域，可被其他线程访问，但分配释放使用其空间
    - 线程a分配的块，位于线程a的线程堆
        - 当线程b释放位于线程a的线程堆地址的块时，此块将进入线程b的空闲链表中
    - 若每个线程的堆在一定次数后被严重污染，那么每次得到 next 槽位地址时使用当前线程获取少量数据后打开全新的线程
- 初始块地址包含 `0x..00...` 导致尝试获取 next 指针时使用 %s 无法打印
    - 额外分配块以调整地址
- 线程空间堆中拥有：
    - 指向主空间堆的指针
    - 指向所在地址的堆指针
    - 指向 fs_base 的线程区域指针
    - 遗留的 libc 指针


tips：
- free 将块放入空闲链表时，对数据区的 *next 与 key 槽的更新，可用来覆盖
    - 从表头取出实际地址进行 safe-linking 的加密，保存到本块的 *next 槽中
- malloc 将空闲块移出空闲链表时，对数据区的 key 槽进行清0，可用来覆盖
    - 对 *next 槽位进行 safe-linking 的解密，保存到空闲链表头
- safe-linking 的最低半字节对齐限制
    - 目标数据在非对齐地址时无法精准定位
    - 目标数据若使用分配的堆，那么地址必然16字节对齐，从而可遍历堆地址的访问
- 使用污染空闲链表的技巧获得泄露后，不是 free 这个在检查块元数据会出错的地址，而是使用新块
    - 使用过一次的目标块不可被释放，每次使用不同的目标块
- 计算出 safe-linking 的 next 槽位地址后，保持对基准块的控制。当需获得更多消耗块时
    - 在开头添加更多分配释放的空闲块，同时确保基准块位于空闲链表头
        - 分配 0/1/2，释放 0/1/2。修改为分配 0/1/2/3/4，释放 0/3/4/1/2
        - 保持 2->1 的栈地址计算不变，即基准块不变
    - 或每次在使用前仅添加一个新块：分配基准块后再分配新块，释放新块后释放基准块。保持基准块在空闲链表头，每次对新块进行修改，从而实现重复使用。整个过程持续对基准块进行分配释放的操控
        - 使用劫持的新块时：分配基准块，分配新块以定位到目标地址，再释放基准块到空闲链表。从而保持后续流程拥有相同操作
- 劫持空闲链表并分配后，空闲链表头保存劫持地址的8字节。即，目标值作为指针泄露
    - 释放1块到链表后，此时next槽中保存使用 safe-linking 混淆后的目标地址的8字节数据
- 字符串型写入读取原语中，在代码中加入 safe-linking 混淆后的空字节或空格等排除字符的断言，以在遇到时立即崩溃而快速进入下一次尝试
- 多个小块触发合并后得到目标大小块
- 若尝试保留控制块，则分配一个保护块
    - 不会被合并的 tcache
    - 相邻内存空间的分配块
- 分配块时若余<=8字节，则与后块的元数据 size 字段 16字节对齐（使用 prev_size 位），在释放块后将大小写入后块的 prev_size 处。例：
    - 分配 0x66，size = 0x70，0x8头部 + 0x68用户段
    - 分配 0x6a, size = 0x80, 0x8头部 + 0x70用户段
- 荒野块拥有 size 字段显示堆剩余分配空间
- tcache 指向用户区基址，其他表指向块基址
- 检测
    - tcache 
        - next 指针 safe-linking
    - fast bins 
        - fd 指针 safe-linking
        - 目标块 size 字段
    - 双向链表
        - 前后指针指回自身
        - 目标块 size 字段

- tcache
    - 释放后使用 UAF(use after free)
        ```py
        # 块0作为控制块，块1仅作为目标块的示意简称

        def get_more_block(ctrl_block=0, new_block):
            # 设置分配释放顺序以持续操控块0作为控制块
            # 块0作为链表头，之后写入新块
            分配块0
            分配新块
            释放新块
            释放块0

        def get_heap_addr()
            # 得到空闲块的 *next 指针
            分配块0
            释放块0
            读取块0

        def set_goal_addr(payload)
            # 劫持空闲列表
            分配块0
            释放块0
            写入块0的 *next 槽位

        def get_goal(ctrl_block=0, target_block)
            # 劫持空闲列表后，读取块1的值
            分配块0/1
            释放块0
            读取块1

        def set_goal(ctrl_block=0, target_block, payload)
            # 劫持空闲列表后，写入块1的值
            分配块0/1
            释放块0
            写入块1

        def get_ctrl_block_page_base()
            # 得到控制块0的页基址在之后用作混淆

            # 得到混淆后的堆地址或 null 的 next
            mg_heap_addr_bytes = get_heap_addr()
            mg_heap_addr_int = int.from_bytes(mg_heap_addr_bytes, 'little')<<12 # 示例为 null 的 next，直接对应页基址

            # ctrl_block_page_base = de_mg(mg_heap_addr_int) # 若为同页混淆指针则解码
            ctrl_block_page_base = mg_heap_addr_int # 示例为 null 的 next，直接对应页基址
            return ctrl_block_page_base

        def set_get(goal_addr, ctrl_block, target_block):
            # 读原语
            # 使用控制块页基址混淆目标地址，写入 *next 槽位，读取目标值
            goal_addr_mg = mg(goal_addr, ctrl_block_page_base)
            goal_addr_mg_bytes = goal_addr_mg.to_bytes(8, 'little')

            set_goal_addr(goal_addr_mg_bytes)
            goal_bytes = get_goal(ctrl_block, target_block)
            return goal_bytes

        def set_scanf(goal_addr, ctrl_block, target_block, payload):
            # 写原语
            # 使用控制块页基址混淆目标地址，写入 *next 槽位，设置目标值
            goal_addr_mg = mg(goal_addr, ctrl_block_page_base)
            goal_addr_mg_bytes = goal_addr_mg.to_bytes(8, 'little')

            set_goal_addr(goal_addr_mg_bytes)
            goal_bytes = set_goal(ctrl_block, target_block, payload)
            return goal_bytes
        ```
- large bins 
    - 释放后使用：控制块0 bk_nextsize 篡改到伪块，以修改伪块的 +0x20 fd_nextsize 字段到触发块2 [large_bin_attack](https://github.com/shellphish/how2heap/blob/master/glibc_2.35/large_bin_attack.c)
        - 分配：
            - 0xc20未来的链表头尺寸头块，控制块0
            - 保护块1
            - 0xc10未来的链表尾尺寸头块，触发块2
            - 保护块3
        - 释放控制块0到 unsorted bins
        - 执行大于控制块0的分配，触发懒归档控制块0到 large bins
        - 修改控制块0的 bk_nextsize 字段到目标地址 -0x20（元数据基址 fd_nextsize 字段对应偏移）作为伪块
        - 释放触发块2到 unsorted bins
        - 执行大于块0的分配，懒归档触发块2到 large bins
        - 此时 large bins 中的大小头块的模拟顺序为：控制块0 - 伪造块 - 触发块2
            - 控制块0的 bk_nextsize 所指伪造块更新其 fd_nextsize 到触发块2地址

- small bins
    - 释放后使用，calloc: small bins 入 tcache [tcache_stashing_unlink_attack](https://github.com/shellphish/how2heap/blob/master/glibc_2.35/tcache_stashing_unlink_attack.c)
        - 设置伪块 +0x18 的 bk 为 +0x10 偏移
                - 之后的 bk->fd 将写入到 +0x20 偏移
        - 分配9个0x90块，再释放
            - 7个填充块到 tcache
            - 2个不相邻而不会被合并的触发块0/控制块2到 unsorted bins
        - 分配一个大块将 unsorted bins 的2个块懒归档到 small bins 
        - 从 tcache 中分配2个填充块以留出空间
        - 设置控制块2的 bk 到伪块（控制块位于首先被分配的链表尾，bk 指向第二个被分配的块）
        - 使用 calloc(0x90) 从 small bins 中取触发块0
            - 控制块2被移动到 tcache
            - 而控制块2 bk 指向的伪块也进入 tcache

- fast bins
    - 释放后使用，safe-linking: fast bins 入 tcache [fastbin_reverse_into_tcache](https://github.com/shellphish/how2heap/blob/master/glibc_2.35/fastbin_reverse_into_tcache.c)
        - 分配块0~13
        - 释放0~6块到 tcache
        - 释放7~13块到 fast bins
        - 设置控制块7的 fd 到 safe-linking 编码后的伪块
        - 分配 tcache 中的块6~0
        - 分配 fast bins 中的块13，触发剩余12~7块与伪块到 tcache

    - 释放后使用，safe-linking: 劫持空闲链表 [fastbin_dup_into_stack](https://github.com/shellphish/how2heap/blob/master/glibc_2.35/fastbin_dup_into_stack.c)
        - 释放分配7块，填满 tcache
        - 寻找 +0x8 处置位 0x1 的地址作为伪块元数据基址，若对应 calloc 需控制数据大小避免巨量清空
        - 分配释放控制块0
            - 得到控制块0的 fd 指针槽位基址，之后用于 safe-linking 编码
            - 使用伪块元数据基址编码后的值，设置控制块 fd 槽位以污染空闲链表
        - 劫持并复用
            - 分配控制块0，拿回控制块
            - 分配劫持块1，劫持空闲链表到伪块
            - 释放控制块0，到 fast bins 链表头之后复用

    - 释放悬空指针后使用任意链表劫持：
        - 此处使用 fast bins 到 tcache
            - 获取堆地址
                - 分配0x10块0，释放块0，分配块0，读取 NULL safe-linking 得到同页基址，释放块0
            - 顺序分配并释放 0x10 大小的填充块0~6到 tcache、构造块7/悬空块8到 fast bins
            - 创建悬空块
                - 分配0x400的超大新构造块7触发 fast bins 中构造块7/悬空块8合并
                - 此时悬空块8的指针在构造块内 +0x20 偏移的用户段
            - 构造块7设置悬空块8以通过 free 检测
                - 用户区 0~0x10 为空
                - +0x10 对应悬空块元数据基址（重点是 size 的 P 位）
                    - prev_size = 0
                    - size = 0x21
                    - next = mg(target_addr, next_page_base)
                    - key = 0
            - 分配 tcache 填充块6/5，此时 tcache 计数为 5
            - 释放悬空块8到 tcache 空闲链表头
            - 再次写入悬空块8以劫持 tcache 空闲链表
            - 分配悬空块8/劫持块9
            - 劫持块9到目标地址


- 双向链表：
    - 相邻块元数据覆盖，拥有指向自身的全局指针 [unsafe_unlink.c](https://github.com/shellphish/how2heap/blob/master/glibc_2.35/unsafe_unlink.c)，前半部分为前向伪块合并，最终伪块合并使用前后向指针覆盖全局指针 
        - 分配两个 0x420 的伪块构造块0、合并触发块1
        - 伪块构造块0的用户数据区基址处构建伪块元数据（对应伪块元数据区）
            - size 字段为 0x431-0x10
            - 前后向指针指向保存用户区基址的全局指针（对应伪块元数据区）
                - fd 设置对应 bk 偏移的 全局指针-0x18 
                - bk 设置对应 fd 偏移的 全局指针-0x10 
        - 修改合并触发块1的元数据 
            - prev_size 为伪块大小 0x420（原为块0的 0x430）
            - size 为 0x430，字段的 flag 位中设置 P 位为0（表示前块未使用而可被合并）
        - 释放合并触发块1，触发其与伪块的合并
            - 双向链表尝试更新伪块前后向指针对应的虚构块中的元数据，即两次写全局指针到其自身之前的地址
                - 先修改 fd 的 fd->bk，后修改 bk 的 bk->fd，而 bk->fd = 全局指针-0x18 偏移
        - 此时全局指针+0x18偏移对应其本身的地址
            - 对全局指针的解引用即从其 -0x18 处开始
            - 若为全局指针数组，无论前后至少设置一个指针到指针数组本身以复用

    - 相邻块元数据写入额外1空字节：前向合并重叠块的水坑攻击：
        - large bins 获取堆地址
            - 顺序分配基准块0 0x510，隔离块1，信息块2 0x520，隔离块3（信息块2相对基准块0稍大）
                - 释放基准块0/信息块2到 unsorted bins
                - 分配超大块，懒归档基准块0/信息块2到 large bins
                    - 此时较大的信息块2在表头，fd指向基准块0
                - 分配信息块2
                - puts 信息块2得到基准块0地址
            - 释放所有块到荒野
        - 分配重叠块0 0x428(任意低半字节为 0x8)、合并触发块1 0x4f0或0x4f8(总大小 0x500 的低字节为 0x00)，保护块2
            - 重叠块0 低半字节为 0x8，重叠下一块元数据的 prev_size 字段并溢出1空字节到 size 字段
            - 合并触发块1 总大小低字节为 0x00，使得 size 元数据被覆盖时效果为清空 flag 而不是破坏大小
        - 重叠块0的用户数据区基址处构建重叠元数据
            - 前后向指针指向保存元数据基址的指针
                - fd 设置对应 bk 偏移的 指针-0x18 
                - bk 设置对应 fd 偏移的 指针-0x10 
                - fd_nextsize 大小分类指针设置0以跳过检测
                - bk_nextsize 大小分类指针设置0以跳过检测
            - 之后放入指向自身元数据基址的指针
        - 覆盖合并触发块1的元数据 
            - prev_size 为重叠块总大小 (0x428+0x8)
            - 1个空字节溢出设置 P 位为0（表示前块未使用而可被合并）
        - 释放合并触发块1，触发其与重叠块的合并
        - note：
            - 7块 tcache 填充后，可用于 small bins
