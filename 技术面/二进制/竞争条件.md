原理：
- 检查/加载 与 执行/使用/保存 的代码并非连续的原子化，而是可被打断或暂停从而存在前后条件或状态不一致的时间窗口

提高竞争攻击成功几率
- 提高目标程序执行时间，复杂命令或超长文本
- （不一定有效）降低程序 cpu/io 优先级，`nice/ionice` 控制程序 cpu/io 优先级
    - 同时运行多个程序
- 增加目录解析时间，设置超长路径（多层目录）
    - 拥有路径长度限制
    - 使用链接以超过 linux 路径长度限制，继续增加路径长度
        - 在同一父目录下，创建超长路径与指向超长路径末尾目录的链接以及目标文件，而末尾目录中设置指向初始父目录的链接。设置的最终到达目标文件的路径中包含大量链接，而每组链接需遍历超长路径
        - 链接也拥有路径解析的数量限制
- 对于拥有线程互斥锁的程序发送信号尝试打断

线程差异：
- setuid：
    - syscall 的 setuid 仅设置对应线程的 uid
    - libc 的 setuid 将使用信号设置所有线程
- exit：
    - syscall 的 exit 仅退出对应线程
    - libc 的 exit 将调用 exit_group() 退出所有线程

工具：
- valgrind 仅在触发竞争条件时有效

提示：
- `__lxstat` 可解析符号，不跟随最后的符号链接，但路径中间的符号链接仍会被解析
- `fprintf` 这类库函数的实现并非原子化：
    - 猜测1：在**隐式**得到长度后调用底层类似 write 的系统函数输出，实际上可输出空字节后的内容而不是在空字节终止
    - 猜测2：或者 fprintf 共用一个缓存，导致缓存被覆盖

    - 猜测1正确，fprintf 为 vfprintf 的包装函数，而 vfprintf 在 strnlen 获取长度后输出，从而存在竞争条件的时间窗口：
        - 平面文字源码 <https://git.musl-libc.org/cgit/musl/tree/src/stdio/vfprintf.c>
        - 语法高亮源码 <https://elixir.bootlin.com/musl/v1.2.5/source/src/stdio/vfprintf.c>
        
        ```c
        static int printf_core(FILE *f, const char *fmt, va_list *ap, union arg *nl_arg, int *nl_type)
            switch(t) {
                case 's':
                    a = arg.p ? arg.p : "(null)";
                    z = a + strnlen(a, p<0 ? INT_MAX : p);
                    if (p<0 && *z) goto overflow;
                    p = z-a;
                    fl &= ~ZERO_PAD;
                    break;
                }

            out(f, a, z-a);

        static void out(FILE *f, const char *s, size_t l)
        {
            if (!ferror(f)) __fwritex((void *)s, l, f);
        }
        ```
