大多数杀毒软件使用称为签名（signatures）的数据位和规则来识别样本中恶意代码的特征。这些签名被加载到杀毒引擎中，引擎扫描磁盘存储和正在运行的进程以查找匹配项。当发现匹配时，大多数杀毒软件会隔离二进制文件或终止正在运行的进程。

为了规避基于特征匹配的杀毒软件，我们可以创建独特的 payload，使其不匹配杀毒软件保护系统上的任何现有特征。此外，当我们将 Metasploit payload 作为漏洞利用的一部分发送时，该 payload 将在内存中运行，且不会向硬盘写入任何数据——这也是 Metasploit 框架试图避免检测的另一种方式。

## 网络

### IDS 和 IPS 规避

注重安全的组织使用 IDS 和 IPS 技术来被动监控和主动阻止可疑的网络流量。在第 3 层和第 4 层采用三种特定的策略来干扰或绕过检测：

- 插入传感器可见但目标主机忽略的数据
- 分片数据包，使传感器忽略它们（但稍后会被重组）
- 修改数据包数据（例如，附加数据和设置特定标志）

工具：
- Nmap 的规避功能，包括数据包分片和数据修改，用于掩盖端口扫描（一个受欢迎的功能是 `--data-length`，它向每个数据包附加数据，欺骗基于签名的检测系统）。

### TTL 操纵

通过发送 TTL 在到达目的地之前超时的数据包，攻击者可以向网络流中插入数据（从传感器的视角）。第二种技术是向目的地发送被忽略但被传感器解析的材料（反之亦然）——通过分片和段重叠实现。根本问题是传感器没有足够的上下文来正确执行网络流重组。

### 数据插入和加扰

数据操纵
- 插入虚假载荷（导致传感器解析未被目的地处理的会话数据）
- 提供虚假的序列信息（导致传感器丢失状态并解析错误数据）
- 注入虚假的信令信息（通过插入被目的地忽略但导致传感器认为会话已结束或新会话已建立的 FIN、RST 或 SYN 数据包）

数据包可以通过多种方式进行畸形化和分片——关键在于采用一种方法，使得数据包被传感器处理但被目标主机忽略，或者反之亦然。这些方法中的许多依赖于传感器和目标主机的 IP 栈实现（通过识别不匹配并利用它来获利）。
- 生成被目标主机忽略的数据包
    - 设置数据包的错误校验和
        高吞吐量环境中的传感器通常出于性能原因不计算数据包校验和。如果是这种情况，恶意内容会被传感器解析，但在目标主机收到时被忽略。
- 生成被传感器忽略的数据包
    - 使用不常见的 IP 和 TCP 选项
        传感器可能会忽略在 IP 和 TCP 头中设置了某些标志和选项的数据包，而目标主机在收到时会接受该数据包。

数据包分片和段重叠
    攻击者可以使用这些策略来规避和绕过 IDS 和 IPS 机制，因为分片数据包被传感器重组的方式可能与目标主机不同

双端点探测
    在传感器分隔的两个受控端点进行一系列探测以创建对应的规避策略

## 二进制

### 编码

规避杀毒软件检测的一种方法是确保载荷的签名是唯一的。我们可以通过使用 MSFvenom 对载荷进行编码来创建唯一签名。这将更改可执行文件中的代码，使其看起来与杀毒软件预期的不同，但运行方式相同。

### 打包可执行文件

打包器（Packers）是压缩可执行文件并将其与解压代码合并的工具。当这个新的可执行文件运行时，解压代码会在执行前从压缩代码中重新创建原始可执行文件。这通常是透明发生的，因此压缩后的可执行文件可以像原始文件一样使用。打包过程会产生一个更小的可执行文件，同时保留原始文件的所有功能。

与 MSFvenom 一样，打包器会改变可执行文件的结构。然而，与通常增加可执行文件大小的 MSFvenom 编码过程不同，精心选择的打包器将使用各种算法来压缩和加密可执行文件。

PolyPack 项目展示了使用各种打包器打包已知恶意二进制文件的结果，以及打包前后杀毒检测的有效性。你可以在 Jon Oberheide、Michael Bailey 和 Farnam Jahanian 的论文《PolyPack: An Automated Online Packing Service for Optimal Antivirus Evasion》（https://faculty.cc.gatech.edu/~mbailey/publications/woot09_final.pdf）中阅读相关内容。

### 自定义可执行模板

当我们不修改静态二进制文件本身而进行杀毒规避时，我们总是陷入猫鼠游戏，因为杀毒签名会频繁更新以检测新的和变化的载荷。

通常，当运行 MSFvenom 时，载荷被嵌入到位于 `/usr/share/metasploit-framework/data/templates/template.exe` 的默认 MSFvenom 可执行模板中。尽管此模板偶尔会更改，但杀毒厂商在构建签名时仍然会查找它。然而，MSFvenom 也支持使用任何 Windows 可执行文件替换默认可执行模板，通过 `-x` 选项。在以下示例中，我们使用微软 Sysinternals 套件中的 Process Explorer 作为自定义可执行模板

#### 隐蔽启动载荷

在大多数情况下，当目标用户启动后门可执行文件（比如我们刚刚生成的）时，似乎什么也没有发生，这可能会引起怀疑。为了提高不被目标察觉的机会，你可以在启动载荷的同时继续执行被启动应用程序的正常功能

在选择将载荷嵌入可执行文件时，如果你指定了 `--keep` 标志，应考虑使用基于 GUI 的应用程序。如果你将载荷嵌入到基于控制台的应用程序中，当载荷运行时，它会显示一个控制台窗口，直到你使用完载荷才会关闭。如果你选择基于 GUI 的应用程序并指定 `--keep` 标志，目标在载荷执行时不会看到控制台窗口。对这些小细节的关注可以帮助你在渗透测试中保持隐蔽。

#### 从 Python 文件生成可执行文件

使用 `reverse_https` shell 允许我们加密载荷与 Metasploit 框架之间的通信，因为 HTTPS 流量是加密的。加密很重要，因为即使你成功规避了杀毒系统，你还需要规避网络监控和入侵检测系统（如 Snort），这些系统会检查你的植入物产生的网络流量。加密通信有助于避免这种检测。

你的载荷还必须通过目标的防火墙。幸运的是，许多防火墙需要保持 443 和 53 端口开放，以便机器可以访问 Web 服务器和 DNS 服务器。

要生成可执行文件，你必须在 Windows 机器上工作。你可以选择几个程序来完成此操作，例如 `py2exe` 命令行界面或 `auto-py-to-exe`（一个逐步创建可执行文件的图形用户界面）。你还可以使用 Nullsoft Scriptable Install System（NSIS）创建自定义安装向导。请记住，许多杀毒系统会标记未签名的可执行文件，因此请使用 Microsoft SignTool 对你的二进制文件进行签名。一旦你拥有了签名后的可执行文件，如果你打算在渗透测试中使用它，请不要将其上传到 VirusTotal，因为你不想让该样本被添加到其恶意样本库中。
