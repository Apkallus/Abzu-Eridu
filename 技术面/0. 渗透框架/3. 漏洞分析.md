漏洞分析评估端口和漏洞扫描、通过横幅抓取收集的数据以及在情报收集期间收集的信息。

漏洞扫描器是一种自动化程序，旨在寻找计算机系统、网络和应用程序中的弱点。该程序通过网络向系统发送数据来探测系统。然后将响应与漏洞数据库中的样本进行比较，匹配结果用于枚举目标上的漏洞。

由于网络实现不同，操作系统在接收网络探测时往往会产生不同的响应。这些独特的响应构成了不同版本操作系统的指纹。扫描器使用这些指纹来确定操作系统的身份甚至其补丁级别。漏洞扫描器还可以使用一组给定的用户凭证登录远程系统，枚举软件和服务以确定它们是否打了补丁。根据获得的结果，扫描器会生成一份报告，概述系统上检测到的任何漏洞，这对网络管理员和渗透测试人员都很有用。

漏洞扫描器通常会在网络上产生大量流量，因此在以保持不被发现为目标的隐蔽渗透测试中通常不会使用。然而，如果你正在进行公开渗透测试且隐蔽性不是问题，漏洞扫描器可以让你免于手动探测系统以确定其补丁级别和漏洞。

通常，结合手动和自动化测试会给你带来最佳结果。

## 基本漏洞扫描

横幅抓取是连接到远程网络服务并读取返回的服务标识（横幅）的行为。许多网络服务，如 Web、文件传输和邮件服务器，要么在连接时立即返回其横幅，要么响应特定命令时返回。

有了这些信息，我们可以使用漏洞扫描器来确定此版本的服务器是否存在任何已知漏洞以及此服务器是否已打补丁。

当然，在实践中，事情并没有那么简单。由于系统和应用程序配置的细微差异，漏洞扫描通常包含许多误报（报告存在漏洞但实际上不存在）和漏报（未能记录存在的漏洞）。

漏洞扫描器的好坏取决于其漏洞数据库，并且它们很容易被误导性的横幅或不一致的配置所欺骗。

通用漏洞扫描器
- Nmap 的 NSE 脚本
- [Nuclei](https://github.com/projectdiscovery/nuclei) 
- [Vuls](https://github.com/future-architect/vuls)
- [Tsunami](https://github.com/google/tsunami-security-scanner)
- [OpenVAS Scanner of the Greenbone Community Edition](https://github.com/greenbone/openvas-scanner)

批量扫描实用程序的操作带来了两个挑战：

- 默认的主机发现和扫描策略会引入覆盖范围的空白
- 输出通常包含误报和没有实际意义的数据

为了获得最佳覆盖范围，同时使用多个工具，并比较输出以识别出由扫描配置和主机发现设置引起的任何空白。

## 常见网络服务评估

执行以下操作以发现常见网络服务中的漏洞：

- **指纹识别**：使用 Nmap 版本扫描（`-sV`）和手动技术审查横幅材料并对可用服务进行指纹识别。同时考虑交叉引用操作系统版本和配置以推断某些实现的版本。

- **枚举支持的功能**：使用手动评估技术和 Nmap 脚本列出给定服务支持的功能（例如 DNS 递归或 LDAP 匿名绑定）。某些实现缺陷的成功利用依赖于对特定功能的支持，因此调查非常重要。

- **识别和确认已知弱点**：搜索漏洞数据库，以识别暴露的网络服务中的已知弱点。这些可能包括提供有用数据的信息泄露缺陷。

- **暴力密码猜测**：使用 Hydra 和其他工具对支持认证的暴露服务（包括 FTP、SSH、Telnet、SNMP、LDAP 和 VNC）执行暴力密码猜测。根据你正在测试的系统类型定制字典文件，以减少测试时间和网络流量。

- **调查获取的材料**：FTP、TFTP、SNMP、LDAP 和 Unix RPC 服务通常会生成有用的材料，你可以将这些材料提供给进一步的测试过程（例如可在密码猜测攻击中使用的用户名）。

## 服务加固和对抗措施

在加固网络服务时，请考虑以下对抗措施：

- **尽可能减少网络攻击面**。例如，不要同时提供 FTP、SFTP 和 SCP 进行文件传输，而选择仅使用 SCP。此外，通过禁用对不必要功能的支持，减少每个网络服务中的暴露逻辑和应用程序攻击面。

- **维护服务器软件包和库（例如 NTP、BIND 和 OpenSSL）**，以消除剩余暴露攻击面中的已知弱点。

- **禁用 Telnet、FTP、SNMP、VNC 和其他缺乏加密传输安全的管理协议**。远程维护操作应通过安全的认证连接（例如 VPN 或 SSH）或通过封闭的管理网络提供。

- **如果使用 SNMP，请确保使用强凭据**。使用 ACL 将 SNMP 访问限制在可信源，并防止未经授权的 TFTP 文件传输到你的设备。

- **了解你服务中暴露的认证机制，并确保配置了审计**，以便暴力密码猜测攻击能够被突出显示。

- **加固 SSH 服务器**，如下所示：
  - 强制使用新版，并禁用向后兼容性，以减轻旧版已知弱点。
  - 精简支持的密钥交换机制和密码。
  - 通过为用户禁用密码认证，并强制使用一次性密码（OTP）、公钥或多因素认证（通过 Google Authenticator、Duo Security 等平台）来缓解暴力密码猜测问题。

- **加固 DNS 服务器**：
  - 禁用来自不受信任来源的递归查询支持。
  - 确保区域文件不包含多余或敏感信息。

- **加固 Kerberos 服务器**：
  - 禁用对弱 HMAC 算法的支持。
  - 在 Microsoft 环境中，考虑强制使用最高的域功能级别以缓解降级攻击。

#### 寻找配置不当的 Microsoft SQL Server

配置不当的 Microsoft SQL Server（MS SQL）安装可能提供进入目标网络的初始途径。事实上，一些系统管理员甚至不知道他们的工作站上安装了 MS SQL 服务器，因为该服务是作为某些常见软件（如 Microsoft Visual Studio）的先决条件安装的。这些安装可能未被使用、未打补丁，甚至从未配置过。

当 MS SQL 安装后，它默认监听 TCP 端口 1433 或一个随机的动态 TCP 端口。如果 MS SQL 正在监听动态端口，只需查询 UDP 端口 1434 即可发现是哪个。当然，Metasploit 有一个可以利用此功能的模块：`mssql_ping`。

```bash
msf > use auxiliary/scanner/mssql/mssql_ping
```

扫描器尝试寻找 MS SQL 服务器，并识别实例名称、SQL 服务器版本及其监听的 TCP 端口号。想想看，这种针对 SQL 服务器的定向扫描比在目标子网的所有机器上对所有端口运行 Nmap 以寻找难以捉摸的 TCP 端口节省了多少时间。
