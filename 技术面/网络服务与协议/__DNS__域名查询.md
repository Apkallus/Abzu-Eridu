dns 结构
- dns 将域名与数字相互转化
- 分为多级域，子域服务器在父域中注册，从父域向子域逐级查找

dns 解析路径
1. 本机 dns 文件
    1. /etc/hosts 保存ip地址与主机名映射
    2. /etc/resolv.conf 保存本地 dns 服务器地址
        - 被动态配置协议自动覆盖
        - /etc/resolvconf/resolv.conf.d/head 内的 nameserver 信息将被添加到此文件
2. 本地 dns 服务器
    - 根据 dns 层次结构查找
    - 缓存得到的信息
3. 网络 dns 服务器

工具
- dig 连接dns服务器显示目标域名查询信息，默认使用本机服务器
    - dig @dns服务器 目标域名
    - dig -x 数字 反向查找
- resolvconf 更新配置文件
- tc 流量控制

攻击面
- 拒绝服务攻击
- dns 欺骗攻击
    - 篡改本地 hosts 文件
    - 伪造本地服务器对用户机的查询响应
    - 伪造网络服务器对本地服务器的查询响应，从而污染缓存
    - 在受控服务器响应中携带其他域的伪造地址
		- 被忽略

伪造内容
- 单条域名的查询结果 A
- 对应的域名服务器 NS

攻击位置
- 局域网嗅探
- 远程攻击
    - 对目标服务器发送相同父域但子域随机的请求，以实现对随机域名的持续查询
    - 触发目标服务器查询
    - 对目标服务器发送伪造响应，尝试污染父域 NS记录
		- 源ip拥有多个可能的地址
		- 目标端口随机
		- 查询id随机
		- 域名字符串拥有特殊编码
反向查询
- 将数字ip地址构建为反向查询伪域名格式
    - 低段.高段.in-addr.arpa
- 逐段查询

对于伪造的防御
- dnssec 父子域的公私钥验证
- tsl/ssl CA公私钥验证

拒绝服务攻击
- 层次高的服务器拥有更多分布式冗余服务器，从而拥有更大弹性
- 直到缓存过期时，攻击的影响才可体现
- 当应用程序本身存在速率限制缺陷时，对特定的低层次服务器成功的拒绝服务攻击使得应用程序本身的庞大用户群发送大量查询流量

DNS重绑定
- 概念：浏览器同源的域名部分检测主机名 host 而非数字 IP。而攻击者 NS 设置任意域名映射，且攻击者载荷页面设置操作
- 流程
    - 返回载荷页面
        - 目标用户尝试访问攻击者控制站点
        - 攻击者域名服务器返回载荷页面，并设置 TLL 短过期时间
    - 触发 NS 再连接
        - 载荷页面定时连接攻击者站点
        - 在域名记录过期后，浏览器再次访问攻击者 NS
    - 载荷页面在内网执行操作     
        - 位于外网的攻击者 NS 返回伪造的位于内网目标 ip
        - 载荷页面仍处于同源状态，在内网执行载荷
