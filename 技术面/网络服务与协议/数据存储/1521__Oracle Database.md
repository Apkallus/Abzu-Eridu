
## Oracle数据库

概念：
- 透明网络底层（TNS）协议通过TNS监听器服务代理客户端到Oracle数据库实例的连接，该监听器服务监听TCP端口1521。

指纹识别工具：
- Nmap

监听器服务有自己的认证机制，并在数据库之外进行管理。在默认配置下，可以与TNS监听器交互以检索有用信息，如果未打补丁，还可以利用它拦截数据库流量并远程执行命令。

从网络攻击Oracle数据库实例通常涉及四个步骤：

1. 评估TNS监听器配置并检索有用信息
2. 枚举数据库系统ID（SID）值
3. 掌握有效SID详细信息后，进行暴力破解密码以获取访问权限
4. 认证后，通过可用函数进行权限提升和横向移动

包括TNS监听器在内的组件中存在可利用的缺陷，攻击者可以利用这些缺陷收集数据（例如SID值）并执行任意命令。

### 与TNS监听器交互

工具：
- `tnscmd10g`
- Metasploit `tnscmd` 模块

TNS监听器有时会在探测期间返回错误代码。详尽的列表可在Oracle Database 文档中找到。

| 错误代码  | 原因            |
| ----- | ------------- |
| 1169  | 监听器不识别密码      |
| 1189  | 监听器无法认证用户     |
| 1190  | 用户未被授权执行请求的命令 |
| 12508 | 监听器无法解析命令     |
| 12618 | TNS版本不兼容      |

### Oracle SID暴力破解

SID值在Oracle环境中唯一标识数据库，并在数据库连接和认证期间使用。

使用Metasploit识别SID值 `sid_enum` `sid_brute`。

### 数据库帐户密码暴力破解

Nmap的`oracle-brute`和`oracle-brute-stealth`脚本。`oracle-brute-stealth`脚本利用了Oracle 11g中泄露帐户密码哈希的缺陷。

从命令行复制粘贴Oracle用户密码哈希到文本文件时，请记住使用`\`字符转义`$`字符。

### 认证到Oracle数据库

命令行使用`sqlplus`实用程序与可用数据库交互。

### 权限提升与横向移动

可能希望执行操作系统命令[^19]、进行端口扫描[^20]，并在认证后利用无数权限提升缺陷。例如，David Litchfield公布的2016年1月Oracle关键补丁更新[^21]包含了Oracle数据库中七个关键漏洞的修复。
