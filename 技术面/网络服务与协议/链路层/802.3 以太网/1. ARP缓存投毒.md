ARP 在本地网络中用于将 IPv4 地址映射到底层 MAC 地址。每个主机维护一个最近映射的 IP 和 MAC 地址对的缓存。

查看本地缓存内容命令

    ```bash
    arp -a
    ```

ARP缓存中毒攻击: 
- 概念：ARP 是无状态的且缺乏身份验证。因此，该协议容易通过发送未经请求的 ARP 应答而被投毒。通过将其 MAC 地址注入受害者系统的 ARP 缓存，攻击者可以实现中间人攻击
- 方式：
    1. 在攻击者系统上启用 IP 转发
    2. 发送未经请求的 ARP 应答以投毒
        - 伪造 ARP 响应，设置攻击者的 MAC（即使目标并未查询）
        - 广播 ARP 地址更新
- 自动工具：
    -  [Ettercap 中间人攻击组件](https://github.com/Ettercap/ettercap)


实现 MITM 后，攻击者可以使用许多实用程序获取数据和秘密，包括：

- **sslstrip**[^4]：降级 HTTPS 会话 ❓
- **easy-creds**[^5]：用于从 Ettercap、sslstrip 等中收集凭据 ❓
- **Responder**[^6]：响应名称解析请求
- **Evilgrade**[^7]：通过服务冒充向受害者提供恶意内容 ❓
- **Metasploit**：执行服务冒充和进一步攻击

atips：
- 部分工具已严重过时，策略或不适用于当前环境
