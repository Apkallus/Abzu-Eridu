# TFTP

概念：

- TFTP 使用 UDP 端口 69，无需身份验证——客户端使用 RFC 1350 中概述的数据报格式从服务器读取和写入。
- 由于协议本身的缺陷（即缺乏身份验证和没有传输安全），在公共互联网上很少见到 TFTP 服务器。然而，在大型内部网络中，TFTP 用于向 VoIP 话机和其他设备提供配置文件和 ROM 映像。

攻击面：

- 从服务器获取材料（例如包含秘密的配置文件）
- 绕过控制覆盖服务器上的数据（例如替换 ROM 映像）
- 通过溢出或内存损坏缺陷执行代码

工具
- `tftp` 手动连接 TFTP 服务器并发出读取（`get`）和写入（`put`）请求。
- Nmap 的 `tftp-enum` 脚本使用常见文件名字典发出读取请求以枚举
- Metasploit 的暴力破解模块


有的 TFTP 服务器配置还允许任意文件上传
