# VNC

概念：
- 虚拟网络计算（VNC）基于远程帧缓冲（RFB）协议规范，提供远程访问主机。
- RFB 服务通常监听在 TCP 端口 5900 上，但也可以使用其他端口（例如 4900 和 6000）。
- 该协议可通过任意编码类型进行扩展，这些编码类型支持 UltraVNC 和 TightVNC 等软件包中的文件传输和压缩。连接后，服务器会提供一个协议字符串。

连接并提供一个版本字符串以协商连接后，服务器返回一个安全类型值。最常见的是 VNC 认证，这是一种仅需要密码的 DES 质询-响应机制。

**RFB 安全类型**

| 类型 | 说明 |
|------|------|
| 0 | 无效的安全类型（连接关闭） |
| 1 | 不需要认证（连接已建立） |
| 2 | 通过 DES 质询-响应进行 VNC 认证 |
| 5 | RealVNC Server Enterprise Edition 公钥认证 |
| 6 | |
| 16 | TightVNC 认证 |
| 17 | UltraVNC 认证 |
| 18 | TLS 认证，用于 Ubuntu Linux 发行版 |
| 19 | TLS 认证，用于 Win32 VeNCrypt 软件包 |
| 20 | GTK-VNC SASL 认证 |
| 21 | MD5 哈希认证 |
| 22 | Citrix Xen VNC Proxy（XVP）认证 |
| 30 | Apple OS X 认证 |
| 35 | |

服务指纹识别工具
- Nmap 的 `vnc-info` 脚本，揭示 RFB 协议版本和支持的安全类型。
- telnet 手动连接并查看横幅

攻击面：
- 暴力密码猜测
- 匿名利用已知软件缺陷

暴力密码猜测
    Nmap `vnc-brute` 脚本 和 Hydra 通过 VNC 认证机制（安全类型 2）执行暴力猜测。
    由于依赖 DES，密码最多限制为八个字符，因此字典文件应相应地进行精简。
