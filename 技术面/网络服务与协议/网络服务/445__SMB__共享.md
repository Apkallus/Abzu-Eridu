# SMB

概念：
- 服务器消息块（SMB）协议提供对数据、打印机和服务端点（通过命名管道）的访问。即，拥有多种渠道访问 SMB。两种最常见的渠道是 NetBIOS 会话服务和 SMB Direct 服务。NetBEUI 是传统 Microsoft 网络中使用的不可路由本地协议。

通过 SMB 向客户端暴露各种共享，包括：

- 默认管理共享（例如 `C$`、`D$` 和 `ADMIN$`）
- 进程间通信共享（`IPC$`）
- 域控制器共享（`SYSVOL` 和 `NETLOGON`）
- 共享打印机和传真共享（`PRINT$` 和 `FAX$`）

通常允许对 `IPC$` 共享进行匿名访问。通过 `IPC$` 暴露的 RPC 端点包括 Server 服务、任务计划程序、本地安全机构（LSA）和服务控制管理器（SCM）。认证后，你可以使用这些来枚举用户和系统详细信息、访问注册表和执行命令。


使用匿名空会话，可访问 `IPC$` 共享并通过命名管道与暴露的服务进行交互。可获得信息：
- 操作系统信息
- 父域的详细信息
- 本地用户和组列表
- 可用 SMB 共享的详细信息
- 有效的系统安全策略

Samba 软件套件在 UNIX 系统上实现了服务器消息块（SMB）协议。Samba 还在 `nmbd` 中实现了 NetBIOS 协议。

- smbd(8) 
    `smbd` 守护进程为 SMB 客户端（如 Windows 95/98、Windows NT、Windows for Workgroups 或 LanManager）提供文件和打印服务。该守护进程的配置文件在 `smb.conf(5)` 中有描述。
- smbclient(1)  
    `smbclient` 程序实现了一个简单的类 FTP 客户端。这对于访问其他兼容服务器（如 Windows NT）上的 SMB 共享非常有用，也可用于让 UNIX 计算机打印到连接在任何 SMB 服务器（如运行 Windows NT 的 PC）上的打印机。
