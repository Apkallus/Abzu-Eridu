# DNS

概念
- dns 结构
    - dns 将域名与数字相互转化
    - 分为多级域，子域服务器在父域中注册，从父域向子域逐级查找
- NS 名称服务器使用两个端口：
    - UDP 端口 53 用于处理常规请求（即将名称解析为 IP 地址，反之亦然）
    - TCP 端口 53 用于可靠地发送大量数据，如 DNS 区域文件
- 反向查询
    - 将数字ip地址构建为反向查询伪域名格式
        - 低段.高段.in-addr.arpa

dns 解析路径
1. 本机 dns 文件
    1. /etc/hosts 保存ip地址与主机名映射
    2. /etc/resolv.conf 保存本地 dns 服务器地址
        - 被动态配置协议自动覆盖
        - /etc/resolvconf/resolv.conf.d/head 内的 nameserver 信息将被添加到此文件
2. 本地 dns 服务器
    - 根据 dns 层次结构查找
    - 缓存得到的信息
3. 网络 dns 服务器


服务器端攻击面：
- 拒绝服务，限制名称服务可用性
    - 层次低的服务器拥有更少冗余服务器与弹性
    - 当应用程序本身存在速率限制缺陷时，对特定的低层次服务器成功的拒绝服务攻击使得应用程序本身的庞大用户群发送大流量查询
- 通过服务器软件缺陷导致的内存损坏和代码执行
- 缓存投毒和破坏，损害名称服务的完整性
    - 对目标服务器发送相同父域但子域随机的请求，以实现对随机域名的持续查询
    - 触发目标服务器查询
    - 对目标服务器发送伪造响应，尝试污染父域 NS记录
		- 源ip拥有多个可能的地址
		- 目标端口随机
		- 查询id随机
		- 域名字符串拥有特殊编码

客户端攻击面：
- 局域网嗅探
- dns 欺骗攻击
    - 伪造内容
        - 单条域名的查询结果 A
        - 对应的域名服务器 NS
    - 方式
        - 篡改本地 hosts 文件
        - 伪造本地服务器对用户机的查询响应
        - 伪造网络服务器对本地服务器的查询响应，从而污染缓存
        - 在受控服务器响应中携带其他域的伪造地址
- DNS重绑定
    - 概念：浏览器同源的域名部分检测主机名 host 而非数字 IP。而攻击者 NS 设置任意域名映射，且攻击者载荷页面设置操作
    - 流程
        - 返回载荷页面
            - 目标用户尝试访问攻击者控制站点
            - 攻击者域名服务器返回载荷页面，并设置 TLL 短过期时间
        - 触发 NS 再连接
            - 载荷页面定时连接攻击者站点
            - 在域名记录过期后，浏览器再次访问攻击者 NS
        - 载荷页面在内网执行操作     
            - 位于外网的攻击者 NS 返回伪造的位于内网目标 ip
            - 载荷页面仍处于同源状态，在内网执行载荷


对于伪造的防御
- dnssec 父子域的公私钥验证
- tsl/ssl CA公私钥验证

工具：
- Nmap 指纹识别：发送 `version.bind` 和 NSID 请求，并解析输出以揭示 BIND 版本和服务器标识符。
- dig 连接dns服务器显示目标域名查询信息，默认使用本机服务器
    - dig @dns服务器 目标域名
    - dig -x 数字 反向查找
- resolvconf 更新配置文件
- tc 流量控制

首先对服务进行指纹识别，然后枚举对递归和其他功能的支持

## 测试递归支持

递归是 DNS 的一项基本功能，名称服务器代表客户端转发请求。内部名称服务器通常支持递归；然而，面向互联网的名称服务器不应响应来自不受信任来源的递归查询。公开可访问的名称服务器支持递归可能导致拒绝服务，因为来自伪造源的 UDP 查询会导致流量被放大并发送到任意位置

如果 UDP 源端口或 TXID 值可预测，支持递归的服务器也存在缓存投毒风险（CVE-2008-1447）。

使用 Nmap 的 `dns-recursion`、`dns-random-srcport` 和 `dns-random-txid` 脚本来评估递归支持和采样随机性。
