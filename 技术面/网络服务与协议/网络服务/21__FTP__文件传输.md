# FTP

概念：
- 文件传输协议（FTP）提供远程文件系统访问，通常用于 Web 应用程序的维护。服务器使用两个端口：
    - TCP 端口 21（入站服务器控制端口，处理来自客户端的 FTP 命令）
    - TCP 端口 20（出站数据端口，用于向客户端传输数据）。

- 文件传输通过控制端口（21）进行协调，其中使用 PORT 等命令通过出站数据端口发起数据传输。


攻击面：
- 暴力密码猜测
- 匿名浏览和利用软件缺陷
- 认证后的漏洞利用（需要特定权限）

额外信息：
- TLS 通常用于包装 FTP（即 FTPS）或通过 STARTTLS 命令提供传输安全。

## FTP 服务指纹识别

Nmap 通过 `-A` 标志执行网络服务和操作系统指纹识别，调用 `ftp-anon` 脚本（以及其他脚本），测试匿名访问并在认证后返回服务器目录结构。

    ```bash
    nmap -Pn -sS -A -p21 130.59.10.36
    ```

MSF 的 `auxiliary/scanner/ftp` 目录下模块

获得有效凭据后，手动评估权限。许多 FTP 服务器缺陷是通过在服务端构造恶意文件结构来利用的。

## 已知的 FTP 漏洞

流行的 FTP 服务器包括 Microsoft IIS FTP Server、ProFTPD 和 Pure-FTPd，在指纹识别后应查询 NVD 以了解已知风险。

要评估公开可用的漏洞利用脚本，使用 `searchsploit` 实用程序，搜索漏洞。Metasploit 中的 `search` 指令也会列出相应的模块。
