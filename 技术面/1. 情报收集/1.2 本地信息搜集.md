# 本地网络发现

评估不可路由的本地网络配置/协议（使用数据链路层和本地广播地址）

## 数据链路协议

以太网被广泛用作底层的物理层和数据链路层格式，由 IEEE 802.3 和 802.2 标准工作组定义。

环境中通常采用许多增强功能，这些功能由 IEEE 802.1 工作组批准：
- **802.1D**（生成树协议）
- **802.1Q**（VLAN 桥接）
- **802.1X**（基于端口的网络访问控制）

### 802.3 以太网测试

以太网容易受到被动网络嗅探和主动攻击（主要通过 ARP 缓存投毒和 CAM 表溢出）的影响。

在制造过程中，每个网络适配器都被编程了一个唯一的 48 位 MAC 地址。这些地址被 IEEE 802 网络（包括 802.3 以太网和 802.11 WiFi）中的系统用来相互寻址。因此，它们的接口会处理发往它们的数据。
通过启用**混杂模式**接收所有帧。如 **Wireshark** 这样的工具启用混杂模式并显示捕获的网络流量。

#### 被动网络嗅探

在本地网络评估工作中，一个很好的起点是运行嗅探器并评估本地网络暴露的材料

通过查看捕获的帧和数据包，揭示 IP 范围、子网大小、MAC 地址和主机名的详细信息。如果网络配置错误或交换结构处于压力之下，攻击者可以通过被动网络嗅探捕获敏感材料。

如果交换式以太网配置正确，你将只看到广播帧和发往你 MAC 地址的数据。要破坏其他主机之间发送的流量，你必须考虑主动攻击策略，

#### ARP 缓存投毒

发送未经请求的 ARP 应答而投毒目标主机的 MAC 与 IP 映射，从而实现中间人攻击

见 ARP 主题页面

#### CAM 表溢出

溢出以太网交换机的 MAC 地址和 VLAN 分配映射表，从而触发换机的失败开放：将数据包广播到所有端口。之后使用网络嗅探器捕获泄露的帧。

见 CAM 表溢出 主题页面

### 802.1Q VLAN

VLAN 在企业中用于划分网络（例如数据、语音和管理）并创建单独的广播域。管理员定义任意的 VLAN ID 值（0-4095），用于标记以太网帧并建立网段。除了减少不必要的广播流量外，802.1Q 标记还限制了 ARP 缓存投毒和其他本地攻击的范围。

802.1Q 实现的一些常见风险：
- 滥用动态中继以破坏 VLAN 和数据（交换机欺骗）
- 双标记帧以向其他 VLAN 发送数据
- 绕过私有 VLAN 端口隔离的第 3 层攻击：CVE-2005-4441。

#### 动态中继

若交换机开启动态中继协议，攻击者模拟交换机并接收所有 VLAN 上的流量。

见 动态中继 主题页面

#### 攻击特定 VLAN

有了 VLAN 和 IP 地址值后，配置虚拟接口来攻击每个网络。示例加入 VLAN 250。如果 DHCP 不可用，使用 `ifconfig` 设置静态 IP 地址。然后在第 2 层（例如 ARP 缓存投毒、MITM、扫描）以及第 3 层（例如端口扫描、暴露服务测试）攻击 VLAN 内的系统。

    ```bash
    modprobe 8021q # 加载 VLAN 模块
    vconfig add eth1 250 # 在 eth1 接口添加到 VLAN 250
    dhclient eth1.250 # DHCP 自动分配
    ifconfig eth1.250 # 若 DHCP 不可用，使用 ifconfig 设置静态 IP 地址
    ...

    root@kali:~# arp-scan -I eth1.250 10.121.5.0/24 # 使用 arp-scan 进行 ARP 网络扫描
    ...
    ```

#### 802.1Q 双标记

若用于在交换机之间形成中继的本征 VLAN 暴露给攻击者，则该攻击者可以双标记帧并将内容发送到其他网络。如果受害者可以与攻击者控制的 IP 通信，则有可能建立 TCP 连接

见 双标记 主题页面

#### 第 3 层私有 VLAN 绕过

在访客无线网络和其他环境中，**私有 VLAN**[^14]（也称为端口隔离）设置用于防止对等方之间进行交互（即客户端连接到无线接入点但无法相互寻址）。根据网络 ACL（或缺乏 ACL），可能可以将 IP 数据包发送到路由器，然后路由器将其转发回相邻对等方。

攻击者发送构造的目标 MAC（路由器）和 IP 地址（目标主机）的网络帧，从而绕过链路层寻址隔离。

网关接收帧并将数据包转发到目的地。此攻击可以与双标记相同的方式被利用：如果受害者可以连接到攻击者控制的 IP，则可以与监听端口建立 TCP 会话。


### 802.1X PNAC

**基于端口的网络访问控制（PNAC）** 是一种允许连接到局域网的设备进行身份验证的机制。
未经身份验证的请求者不能参与网络；然而，它们可以发起与后端 RADIUS 服务器的 EAP 对话。

常见的 802.1X 身份验证方法：
- EAP-MD5
    身份验证服务器向请求者发送一个挑战，请求者以 MD5 哈希的形式发送响应（使用标识符、挑战值和用户密码计算）。服务器执行相同的计算，如果哈希匹配，请求者通过身份验证。
- EAP-TLS
    身份验证服务器和请求者建立 TLS 会话。请求者验证身份验证服务器的 X.509 证书，反之亦然。通过使用证书进行相互身份验证，不使用密码。TLS 身份验证在第 11 章中详述。
- PEAP
    请求者和身份验证服务器建立一个经过身份验证的 TLS 隧道。MS-CHAPv2 挑战/响应通过 TLS 使用用户凭据（即用户名和密码）对请求者进行身份验证。 

**EAP-MD5** 方法容易受到离线暴力密码猜测的攻击。**EAP-TLS** 和**受保护的 EAP（PEAP）** 通过 TLS 提供身份验证和传输安全，这减轻了凭据的暴露。然而，在每种情况下，请求者身份都在明文 EAPOL 开始消息中广播。

以下是一些可用于攻击 802.1X 实现的攻击策略：

- 通过 EAP 进行主动暴力密码猜测
- 使用格式错误的 EAP 内容攻击 RADIUS 服务器[^15]
- EAP 消息捕获和离线密码破解（EAP-MD5 和 PEAP）
- 强制 EAP-MD5 身份验证以绕过 TLS 证书验证
- 在使用集线器或类似设备进行身份验证后注入恶意网络流量[^16]

[^15]: 参见 CVE-2008-2441 和 CVE-2013-3466。
[^16]: 这可以通过采用 802.1AE（MACsec）来缓解。

在共享媒体环境（例如 802.11 WiFi）中，EAP 注销帧欺骗也是一个问题

#### EAP 消息捕获和离线攻击

在请求者本地网络的攻击者可以嗅探 EAP-MD5 对话，并使用恶意的 802.1X 认证者来获取 PEAP（MS-CHAPv2）凭据。

- EAP-MD5
    在捕获 PCAP 格式的请求者和身份验证服务器之间的 EAP-MD5 对话后，使用 `eapmd5pass` 工具破解用户凭据

- PEAP
    - 部署一个恶意认证者（`hostapd-wpe` 工具）来捕获通过 PEAP 在 TLS 上发送的 MS-CHAPv2 数据。
    - 使用 `genkeys` 创建索引文件
    - 使用 `asleap` 进行彩虹表破解

#### 强制 EAP-MD5 身份验证

Microsoft IAS、Cisco ACS 和其他身份验证服务器支持 **EAP-MD5**，攻击者可以使用窃取的凭据（即用户名/密码）滥用该协议进行身份验证，而无需提供客户端证书。

配置 `wpa_supplicant` 以通过以太网使用 EAP-MD5 和用户凭据协商 802.1X 会话。

### CDP

支持**Cisco 发现协议（CDP）** 的主机广播以太网多播帧，描述其操作系统和配置。

CDP 帧中发现的数据

| 类型 | 内容 |
|------|------|
| 1 | 设备主机名或序列号，表示为 ASCII 字符串 |
| 2 | 发送帧的主机的第 3 层接口 |
| 3 | 发送 CDP 更新的端口 |
| 4 | 主机的功能能力（例如路由器、交换机、不支持 IGMP） |
| 5 | 包含软件版本的 ASCII 字符串（与 `show version` 相同） |
| 6 | 硬件平台 |
| 7 | 直接连接的 IP 网络前缀列表 |
| 9 | VTP 管理域 |
| 10 | 本征 VLAN |
| 11 | 发送端口的双工设置 |
| 14 | VoIP 话机辅助 VLAN 查询 |
| 15 | VoIP 话机辅助 VLAN 应答 |
| 16 | 设备（如 VoIP 话机）的功耗（毫瓦） |

在 Cisco 环境中通过被动嗅探收集 CDP 帧并收集网络信息。Yersinia 捕获 CDP 帧并显示各个字段

导致意外后果的主动 CDP 攻击包括：
- CDP 泛洪交换机，导致拒绝服务[^17]（例如 99% CPU 占用）
- 广播 CDP 帧以通告特定设备，进而诱导网络管理产品（包括 CiscoWorks LAN Management 和 IBM Tivoli NetView）通过 SNMP（和其他协议）进行入站轮询
- 广播 CDP 帧以将 Cisco IP 话机置于特定 VLAN 中

[^17]: Vonnie Hudson，“Destroying a Cisco Switch with CDP Flooding”，Fixed by Vonnie Blog，2015 年 6 月 23 日。[链接](https://fixedbyvonnie.com/2015/06/destroying-a-cisco-switch-with-cdp-flooding/)

工具：
- `Yersinia` 捕获、修改和制作 CDP 帧
- `Metasploit cdp 模块`
- `Scapy cdp 库`

### 802.1D STP

**生成树协议（STP）** 是一种用于维护无环网络拓扑的机制。交换机使用**网桥协议数据单元（BPDU）** 帧进行自组织，通过将冗余上行链路端口设置为称为**阻塞**的状态来关闭环路。

STP 网络中的端口状态

| 状态 | 描述 |
|------|------|
| Disabled | 电气上处于非活动状态，直到启用 |
| Blocking | 端口丢弃除 BPDU 帧之外的所有数据 |
| Listening | 交换机监听 BPDU 以构建无环树 |
| Learning | 交换机使用帧的源 MAC 地址构建转发表 |
| Forwarding | 端口完全运行，向/从交换机转发流量 |

在选举根桥后，交换机选择根端口和指定端口。根端口是提供到根桥最佳路径的端口，其他端口成为指定端口。
BPDU 选举过程和端口配置的详细信息：
- Kevin Lauerman 和 Jeff King 的论文[^21]，该论文提供了许多底层示例和详细的拓扑图。

[^21]: Kevin Lauerman and Jeff King，“STP MiTM Attack and L2 Mitigation Techniques”，Cisco 白皮书，2010 年。[链接](https://www.cisco.com/c/en/us/support/docs/lan-switching/spanning-tree-protocol/100157-STP-MiTM-Attack.html)

#### 监控 BPDU

BPDU 帧在 Yersinia STP 协议视图中被捕获和显示。如果你无法在接口上捕获 BPDU 帧，则不太可能成功进行 STP 攻击。

#### 根桥接管

在连接到环境中的两个交换机后，使用 `Ettercap` 建立桥接，并使用 `Yersinia` 通过每个接口发送精心制作的 BPDU 帧。交换机之间的流量被破坏。

使用 `macof` 发起 CAM 表溢出攻击并破坏环境中其他交换机之间的流量。最终效果是交换机然后将帧广播到所有端口，进而可捕获这些帧。

## 本地 IP 协议

前一节描述的协议使用以太网多播并在数据链路层上运行。本节描述的协议通过传输层广播地址在 IPv4 和 IPv6 上提供网络发现和配置服务。

常见的本地名称解析和配置机制包括以下内容：
- **动态主机配置协议（DHCP）**
- **预启动执行环境（PXE）**
- **本地名称解析协议（LLMNR、NBT-NS 和 mDNS）**
- **Web 代理自动发现（WPAD）**
- **内部路由协议（例如 HSRP、VRRP、EIGRP 和 OSPF）**
- **IPv6 网络发现协议**

通过第 2 层 MITM 攻击拦截流量后，你可以使用 Ettercap[^22] 伪造 DNS 响应并将用户引导到任意目的地。

[^22]: 有关 Ettercap 的更多信息，请参见 Lakshmanan Ganapathy，“Ettercap Tutorial: DNS Spoofing & ARP Poisoning Examples”，The Geek Stuff Blog，2012 年 5 月 10 日。[链接](https://www.thegeekstuff.com/2012/05/ettercap-tutorial/)

### DHCP

**动态主机配置协议（DHCP）** 通过 UDP 运行，用于配置本地系统并向客户端提供详细信息，包括 IP 地址、子网和默认网关详细信息。

DHCP 消息类型

| 消息 | 发送方 | 描述 |
|------|--------|------|
| DHCPDISCOVER | 客户端 | 广播以从网络请求 DHCP 提供，并可选地提供最后使用的 IP 地址（请求租约） |
| DHCPOFFER | 服务器 | 收到请求后，服务器首先为客户端预留一个 IP 地址，然后准备并发送一个提供，包括地址、子网掩码和 DHCP 选项（例如 DNS 服务器和路由器详细信息） |
| DHCPREQUEST | 客户端 | 客户端可以接受来自多个服务器的 DHCP 提供，因此此消息用于正式请求特定地址 |
| DHCPACK | 服务器 | 服务器的最终确认，包括租约持续时间和额外的 DHCP 选项（例如 WPAD 服务器） |
| DHCPNAK | 服务器 | 指示客户端的网络地址概念不正确（例如客户端已更改子网或其租约已过期） |
| DHCPDECLINE | 客户端 | 指示客户端网络地址已在使用中 |
| DHCPRELEASE | 客户端 | 取消租约并放弃网络地址 |
| DHCPINFORM | 客户端 | 在已配置其本地 IP 地址和子网后，请求额外的网络配置参数 |

识别 DHCP 服务器和配置
- 使用 Nmap的 DHCP 脚本通过广播本地发现消息来枚举 DHCP 服务器。

主动 DHCP 攻击
- 工具：
    - 使用 Responder DHCP 脚本来建立恶意的 DHCP 服务器
    - DHCPig 耗尽服务器资源
- 注：
    设置恶意网关并不理想，因为被劫持的连接只有半双工（即捕获来自客户端的出口数据包，但无法捕获来自合法网关的响应）。因此，应当设置一个恶意的 DNS 或 WPAD 服务器，特别是为了捕获 HTTP 流量和凭据。


### PXE

**预启动执行环境（PXE）** 是一种系统从本地网络加载操作系统映像的方法。
1. DHCP 用于提供网络配置
2. TFTP 用于提供初始启动映像
3. 文件服务协议（通常是 NFS 或 SMB）用于部署操作系统

默认情况下，许多 BIOS 芯片组首先尝试通过网络引导（PXE），然后从本地介质引导。

在 Microsoft 环境中，PXE 以两种模式之一运行：
- **Lite Touch**：需要凭据才能通过 SMB 加载完整的操作系统映像
- **Zero Touch**：无需凭据即可加载完整的操作系统

实际的 PXE 攻击包括以下内容：

- 从可用的操作系统映像中窃取秘密（例如凭据）[^27]
- 提供恶意映像[^28]以捕获用户凭据（通过呈现虚假登录提示）或通过低级别攻击 BIOS[^29]、硬盘控制器[^30]或其他硬件组件实现持久性

[^27]: Dave DeSimone and Dave Kennedy，“Owning One to Rule Them All”在 Defcon 20 Hacking Conference 上展示，内华达州拉斯维加斯，2012 年 7 月 26-29 日。
[^28]: Metasploit `pxeexploit` 模块。
[^29]: Corey Kallenberg and Xeno Kovah，“How Many Million BIOSes Would You Like to Infect?”，Legbacore 白皮书，2015 年 6 月 11 日。
[^30]: Jonas Zaddach 等人，“Implementation and Implications of a Stealth Hard-Drive Backdoor”，在 Annual Computer Security Applications Conference 上展示，路易斯安那州新奥尔良，2013 年 12 月 9-13 日。

如果受害主机不使用全盘加密（FDE），你可以修补操作系统使用的文件[^31]并获得用户密码哈希[^32]。然而，随着企业中 FDE 的普及，这种攻击向量正迅速变得过时。❓

### LLMNR、NBT-NS 和 mDNS

当 DNS 查询失败时，Microsoft 系统使用**链路本地多播名称解析（LLMNR）** 和 **NetBIOS 名称服务（NBT-NS）** 进行本地主机解析。Apple Bonjour 和 Linux 零配置实现使用**多播 DNS（mDNS）** 来发现网络中的系统。这些协议未经身份验证，并通过 UDP 广播消息；因此，攻击者可以利用它们将用户引导到恶意服务。

`Responder` 工具响应通过端口 137（NBT-NS）、5353（mDNS）和 5355（LLMNR）广播的 UDP 查询，将客户端引导到恶意服务（例如 SMB）。受害者使用可以被破解和重放的哈希对服务进行身份验证

### WPAD

许多浏览器使用**Web 代理自动发现（WPAD）** 从网络加载代理设置。WPAD 服务器通过以下任何一种方式被识别后，通过特定 URL（例如 `http://wpad.example.org/wpad.dat`）提供客户端代理设置：

- DHCP，使用代码 252 条目[^34]
- DNS，在本地域中搜索 `wpad` 主机名
- Microsoft LLMNR 和 NBT-NS（在 DNS 查找失败的情况下）

[^34]: 参见 Microsoft TechNet 上的“Creating a WPAD entry in DHCP”。

`Responder` 工具自动化了 WPAD 攻击——运行代理并通过 DHCP、DNS、LLMNR 和 NBT-NS 将客户端引导到恶意的 WPAD 服务器。

#### HSRP 和 VRRP

**热备用路由器协议（HSRP）** 和**虚拟路由器冗余协议（VRRP）** 用于高可用性环境以提供故障转移支持。路由器向本地多播组发送数据包，通告配置和优先级详细信息。

HSRP 是 Cisco 专有协议，没有 RFC，而 VRRP 是标准化的（RFC 3768 和 RFC 5798）。使用网络嗅探器捕获管理流量，以评估环境中的 HSRP 和 VRRP 支持。

工具
- HSRP 
    - `Scapy`
        - 发送配置的默认身份验证字符串 `cisco` 的数据包
    - `Yersinia`
        - 捕获并呈现任意的明文身份验证字符串
        - 使用 `hsrp` 将身份验证字符串包含在 HSRP 数据包中
- VRRP
    - `Loki`
    - `tcpdump` 对于简单身份验证，可以轻松获得 VRRP 凭据（VRRPv3 不支持身份验证）
        - 使用 Scapy 生成 VRRP 数据包

#### RIP

存在三个版本的**路由信息协议（RIP）**——RIP[^48]、RIPv2[^49] 和 RIPng[^50]。RIP 和 RIPv2 通过端口 520 向对等方发送 UDP 数据报，而 RIPng 通过 IPv6 多播将数据报广播到 UDP 端口 521。RIPv2 引入了 MD5 身份验证支持。RIPng 不包含本地身份验证；相反，它依赖于 IPv6 中可选的 IPsec AH 和 ESP 头。

[^48]: 参见 RFC 1058。
[^49]: 参见 RFC 2453。
[^50]: 参见 RFC 2080。

在许多情况下，RIP 对等方会处理未经请求的路由通告。

使用 Nemesis（RIP/RIPv2） 和 Scapy（RIPng），在受害者主机上注入一条路由，设置目标服务器的新网关。

#### EIGRP

**增强型内部网关路由协议（EIGRP）** 是 Cisco 专有的，可以在有或没有身份验证的情况下运行[^52]。`Coly` 工具支持捕获 EIGRP 广播和注入数据包以操纵路由配置

[^52]: 要无身份验证运行，必须使用带密钥的 MD5 或 HMAC-SHA-256。

#### OSPF

大多数**开放最短路径优先（OSPF）**[^53] 实现使用 MD5 在路由器之间提供身份验证。

`Loki` 和 `John the Ripper` 可以捕获并攻击 MD5 哈希以揭示密钥，然后可以使用该密钥通告新的路由。路由参数通过“Injection”选项卡设置，密钥在“Connection”下设置。

[^53]: 参见 RFC 2328（OSPFv2）和 RFC 5340（OSPFv3）。

### IPv6 网络发现

**邻居发现协议（NDP）**[^56] 定义了五种 ICMPv6 消息类型，用于组织和配置本地 IPv6 网络

| 消息 | ICMPv6 类型 | 描述 |
|------|-------------|------|
| 路由器请求 | 133 | 节点用于定位链路上的路由器 |
| 路由器通告 | 134 | 路由器用于通告其存在，以及链路和前缀参数；通告由路由器定期发送，或在响应请求时发送给主机 |
| 邻居请求 | 135 | 节点用于确定 IPv6 邻居的链路地址，并验证邻居是否可通过缓存的地址到达 |
| 邻居通告 | 136 | 用于响应请求消息，提供 IPv6 和链路地址详细信息 |
| 重定向 | 137 | 路由器用于通知节点对于给定目的地前缀有更好的第一跳 |

## 本地网络发现回顾

用于攻击本地协议的在高层次上有两个迭代阶段
- 主动攻击策略（例如 VLAN 跳跃和 ARP 缓存投毒）用于改变网络状态
- 被动嗅探则产生网络拓扑信息和敏感数据

常见的第 2 层主动攻击方法如下：
- **ARP 缓存投毒**：操纵本地主机之间的流量
- **CAM 表溢出**：导致交换机将帧广播到所有端口
- **通过伪造的 BPDU 帧修改 802.1D STP 流量**：改变流量流
- **通过动态中继或双标记访问特权 VLAN**
- **802.1X 凭据的破坏和破解**（需要网络访问）
- **CDP 泛洪**：导致拒绝服务
- **CDP 欺骗**：通告虚假设备，诱导管理软件连接（例如 SNMP）并破坏敏感数据

第 3 层主动攻击策略包括：
- 使用恶意的 DHCP 和 WPAD 服务器修改客户端配置
- 通过 PXE 提供恶意启动映像，以破坏用户凭据并对易受攻击的主机发起硬件层攻击（例如 BIOS 攻击）
- **LLMNR、NBT-NS 和 mDNS 欺骗**：将用户引导到恶意服务
- 通过 **HSRP、EIGRP、OSPF** 和其他协议模拟路由器
- **IPv6 发现协议欺骗**：导致 MITM
- **识别支持 IP 转发的主机**（用于将流量路由到其他地方）


## 本地网络防御措施

MITM 和恶意服务器攻击
- 强制实施传输安全（通过 IPsec 或 TLS）
- 通过证书验证进行强身份验证
    - 始终验证认证者的 X.509 证书
    - 指定有效认证者（RADIUS 服务器）的 CN 值
    - 通过不在安全异常时提示最终用户来实现故障安全  

Cisco 特定数据链路安全特性：
- 启用端口安全以限制分配给一个端口的 MAC 地址数量
- 禁用 CDP 支持以防止针对交换机的拒绝服务攻击
- 启用 `bpdu-guard` 和 `root guard` 以缓解 STP 攻击
- 使用未知流量泛洪控制功能来限制第 2 层广播攻击[^63]
- 通过在不信任的端口上设置 ACL 以丢弃发往 UDP 端口 520 的流量和使用 IP 协议 88 和 89 的数据包，避免因 OSPF、EIGRP 和 RIP 数据包具有优先级而导致 CPU 耗尽

[^63]: 更多信息，请参见“Catalyst 6500 Release 12.25X Software Configuration Guide”，Cisco.com，2013 年 11 月 17 日。

通用的数据链路攻击缓解措施如下：
- 将交换机端口设置为接入模式并禁用动态中继
- 建立 VLAN，防止不受信任的用户获得对敏感系统（例如 IT 运维人员使用的服务器和工作站）的第 2 层访问
- 禁用未使用的以太网端口并将其置于隔离 VLAN 中
- 始终为中继端口使用专用的 VLAN ID
- 尽可能避免使用默认的 VLAN ID 值 “1”
- 尽可能使用私有 VLAN（端口隔离）功能，以防止客户端系统相互交互

网络层和应用层对抗措施：
- 如果未明确需要 IPv6，请禁用以防止覆盖网络攻击
- 禁用 ICMP 重定向支持以缓解 MITM
- 在 Windows 中禁用多播名称解析和基于 TCP/IP 的 NetBIOS
- 在 Apple OS X 和 Linux 中禁用 Bonjour/零配置功能
- 在未使用隔离的端口上建立 ACL，使得私有 VLAN 攻击（通过网关将流量路由到隔离端口）无效
- 在 Web 应用程序中使用 HSTS 以缓解降级 HTTPS 到 HTTP 的 MITM 攻击（例如 `sslstrip`）
- 审查客户端代理设置，使其不通过 WPAD 自动设置[^65]

[^65]: 参见“How to Turn Off (Disable) Web Proxy Auto Discovery (WPAD) in Windows Server 2008 R2”，Stack Overflow，2013 年 2 月 22 日。
