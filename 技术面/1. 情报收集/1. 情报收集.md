在有限的信息状态下，收集关于目标组织信息，并识别底层组件，如操作系统、正在运行的服务、软件版本等。在此阶段收集的信息越多，未来可能使用的攻击向量就越多。情报收集的非详尽项目清单：
- 公开域名搜索
- DNS/WHOIS 查询
- 公开信息搜索（搜索引擎、社交网络、新闻组等）
- 网络枚举（互联网的网络块和内部 IP 地址）
- 端口扫描、操作系统指纹识别及版本扫描
- 防火墙枚举

专注于范围，以保持测试的聚焦性

迭代侦察：每当获取新信息时，再次执行侦察过程

未得到妥善加固的主机：识别外围网络和主机，组织通常将其精力集中在保护明显的公共系统（如公共 Web 和邮件服务器）上。那些偏离常规的被忽视的主机正是攻击的成熟目标。

使用搜集的网络信息绘制目标组织的网络地图，并了解物理位置之间的关系。

基本信息被输入到漏洞扫描和渗透测试阶段，以识别可利用的缺陷。
侦察提取的用户详细信息（电子邮件地址、电话号码和用户名），可用于暴力密码猜测和社会工程学。

被动信息收集：
    使用被动或间接的信息收集技术，在不接触目标系统的情况下发现信息。

主动信息收集
    直接与系统交互以了解网络服务详情和外围信息，期间可能会被入侵检测系统（IDS）或入侵防御系统（IPS）发现。

隐蔽探测
    组织的保护机制通常只允许外部设备上的某一部分端口子集有流量，若查询非白名单的端口，就会被阻止。最初从一个愿意被阻止或检测到的、可丢弃的 IP 地址范围来探测这种阻止行为。在测试 Web 应用程序时也是如此。达到一定阈值后，Web 应用防火墙会阻止攻击者发出更多请求。

嘈杂探测
    使用与主攻击完全不同的 IP 范围进行非常嘈杂的扫描，以确定组织对攻击者所使用的工具的反应程度。

开源情报（OSINT）是一种情报收集管理形式，涉及从公开可用来源中查找、选择、获取信息，并对其进行分析以产生可操作的情报。
