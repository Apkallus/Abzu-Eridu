## 域名 WHOIS

查询域名注册机构以获取有关目标组织注册的域名信息。

ICANN 和 IANA 的注册机构列表：
- [gTLD 注册机构](https://www.icann.org/en/contracted-parties/registry-operators/resources/listings)
- [ccTLD 注册机构](http://www.iana.org/domains/root/db)

这些注册机构提供以下信息：
- 管理联系人详细信息（姓名、电子邮件地址和电话号码）
- 与目标组织相关的办公地点的邮寄地址
- 每个域名的权威名称服务器详细信息

执行域名 WHOIS 查询的工具：
- `whois` 命令行客户端
- 每个域名注册机构维护的 Web 界面
- Hurricane Electric（HE）、DomainTools 和其他第三方服务

## DNS 分析 - 域名信息

工具：
- 手动
    - `nslookup`
    - `dig`

- 自动
    - `dnsrecon` 普通查询+区域传输
    - `fierce` 区域传输+DNS猜测

DNS 资源记录

    | 记录 | 描述 | 揭示内容 |
    |------|------|----------|
    | SOA | 授权起始 | 创建 DNS 区域的源主机 |
    | NS | 名称服务器 | 给定域的权威 DNS 服务器名称 |
    | A | 地址（IPv4） | 主机名的 IPv4 地址 |
    | AAAA | 地址（IPv6） | 主机名的 IPv6 地址 |
    | PTR | 指针 | 给定 IPv4 或 IPv6 地址的主机名 |
    | CNAME | 规范名称 | CNAME 是其别名的主机名 |
    | MX | 邮件交换 | 给定域的邮件服务器 |
    | HINFO | 主机信息 | 主机的操作系统或其他信息 |
    | SRV | 服务定位器 | 域内的应用服务端点，包括 Kerberos、LDAP、SIP 和 XMPP |
    | TXT | 文本字符串 | 包括 SPF 和 DKIM 字段的材料，根据配置用于提供安全性 |

RFC 1035 详细说明了除 AAAA IPv6 地址和 SRV 服务定位器记录之外的所有记录的底层机制和用法。

RFC 8482 更新了对 any 查询的最小化响应

### 获取 SRV 记录

Nmap 的 `dns-srv-enum` 脚本枚举给定域名的常见 SRV 记录，暴露应用程序使用的内部服务器端点（例如 Microsoft Active Directory、Microsoft Exchange、Kerberos、VoIP 话机和 XMPP 客户端）

```bash
nmap --script dns-srv-enum --script-args dns-srv-enum.domain=ebay.com
```

### DNS 区域传输技术

区域传输通过 TCP 端口 53 执行，以将当前 DNS 区域材料传播到支持该操作的其他名称服务器。

区域文件包含与特定域和 IP 块相关的 DNS 记录。配置错误的服务器会接受公共互联网的传输请求，利用这一点来绘制给定网络的地图。

获取域的权威服务器详细信息后，尝试针对权威名称服务器使用 `dig` 执行正向区域传输。并在完成端口扫描后，针对范围内任何暴露 TCP 端口 53 的名称服务器进行尝试。

    ```bash
    dig nmap.org ns
    ;; ANSWER SECTION:
    nmap.org.               3600    IN      NS      ns1.linode.com.

    dig @ns1.linode.com nmap.org axfr
    ;; no servers could be reached
    ```

若成功则得到主机内部 IP 地址的详细信息，之后使用 IP 块进行反向查询网段（PTR 记录）。示例使用 `dig` 执行 `198.171.79.0/24` 子网的反向区域传输。

    ```bash
    $ dig @glb-ns4.it.verio.net 79.171.198.in-addr.arpa axfr
    ```

### 正向 DNS 猜测

通常名称服务器不允许区域传输，于是进行主动猜测策略来识别有效的 DNS 地址记录，包括：
- 使用 A 记录请求进行字典攻击
- NSEC 和 NSEC3 记录枚举

#### 字典攻击

自动工具：
- fierce
- Nmap
- knockpy
- dnsenum
- dnsmap
- bfdomain.py


fierce 进行正向 DNS 猜测

    ```bash
    fierce -dns academi.com
    ```

使用 dig 半自动针对特定服务器执行正向猜测
- 步骤
    - 识别 `academi.com` 域的权威 DNS 服务器
    - 准备一个主机名字典文件（`academi.txt`）
    - 并使用 `dig` 查询特定服务器（`ns2.dnsbycomodo.com`）。
        
    ```bash
    dig academi.com ns +short
    ns2.dnsbycomodo.net.
    cat /usr/share/fierce/hosts.txt | awk '{printf("%s.academi.com\n",$1);}' > out.txt
    dig @ns2.dnsbycomodo.net -f out.txt +noall +answer
    ```

下级名称服务器通常未加固，测试每个可用的 DNS 服务。

#### NSEC 和 NSEC3 枚举

查询支持 DNSSEC 的名称服务器以揭示有效的主机名。对应 Nmap 脚本 `dns-nsec-enum` 和 `dns-nsec3-enum`

步骤
- 使用 Nmap 脚本得到一批主机名，并输出到文件
- 使用 dig 得到对应 ip 地址

### 反向 DNS 扫描

在构建 IP 网络块列表后，使用反向扫描揭示主机名。

使用 Nmap 执行反向 DNS 扫描，使用 `-sL` 标志

    ```bash
    nmap -sL IP网段
    ```

过程中揭示的新域和子域被反馈到进一步的 Web 搜索和 DNS 查询中，以识别更多的感兴趣系统。通过修改本地 `/etc/resolv.conf` 文件中的名称服务器值，可强制查询特定的 DNS 服务器。


使用 **HE BGP Toolkit**，以获取给定 IP 范围的 DNS 记录。
PTR 记录是权威的且与目标环境相关；然而，A 记录应持保留态度，因为它们可能源于不同 DNS 区域中的错误。

在环境中构建有效主机名列表在稍后测试 Web 服务器端点时特别有用。负载均衡器和反向代理很常见：如果配置错误，攻击者可以通过在 HTTP 1.1 请求中提供有效的 `Host` 头值来访问 Web 应用程序。

### IPv6 主机枚举

通过 AAAA 的 DNS 猜测识别 IPv6 服务器。

使用 `thc-ipv6` 项目中的 `dnsdict6` 程序识别 `ripe.net` 域内可预测主机名的 IPv6 地址。

```bash
dnsdict6 -s ripe.net
```

使用 `thc-ipv6` 项目中的 `dnsrevenum6` 程序查询 IPv6 地址的主机名

```sh
./dnsrevenum6 dns.test.com 2001:db8:42a8::/48
```

## SMTP 探测

邮件网关通过 SMTP 支持跨网络传输邮件。简单地发送一封电子邮件消息到目标域的一个不存在地址，通常通过未送达通知（NDN）揭示有用的内部网络信息

该记录中的以下数据很有用：

- 内部主机名、IP 地址和子域布局
- 邮件服务器运行 Microsoft Exchange Server 2010 SP3
- 使用 Barracuda Networks 设备执行内容过滤

## 枚举技术回顾

以下是对基于互联网的查询技术及其应用的概述：

- **Web 搜索**：
    - 攻：使用 Google、Netcraft、Shodan、LinkedIn、PGP 密钥服务器和其他站点，针对已知域名和 IP 块进行搜索，以识别人员、主机名、域名以及驻留在暴露 Web 服务器上的有用数据。
    - 防：禁用不包含 `index.html` 或类似文件（例如 Microsoft IIS 下的 `default.asp`）的目录索引文件的目录索引。

- **WHOIS 查询**：
    - 攻：查询域名和 IP 注册机构，以检索与目标网络和域名相关的网络块、路由和联系详细信息。IP WHOIS 查询提供有关保留网络块大小（在后续执行侵入性网络扫描时有用）和 AS 号详细信息的信息。
    - 防：在 WHOIS 数据库和 TLS 证书中使用通用的、集中的网络管理联系详细信息，以防止针对 IT 部门的社会工程。

- **BGP 枚举**：
    - 攻：将 AS 号与 BGP 站点交叉引用，以枚举该 AS 下关联的 IP 块，然后将这些详细信息反馈到其他查询路径（如 DNS 或进一步的 WHOIS 查询）。
    - 防：

- **DNS 查询**：
    - 攻：查询可访问的名称服务器以枚举域、子域、主机名和非公开 IP 地址详细信息。配置错误的 DNS 服务器还会提供列出子域、主机名、设备操作系统平台和内部网络详细信息的区域文件。
    - 防：
        - 配置名称服务器以不允许向互联网的主机进行 DNS 区域传输。
        - 精简 DNS 区域文件，使得不必要的信息不被披露（主要是非公开 IP 地址和主机名详细信息），并且 DNS 猜测攻击无效。理想情况下，仅在绝对需要时使用 PTR 记录（用于 SMTP 邮件服务器和其他需要双向解析的关键系统）。

- **SMTP 探测**：
    - 攻：向目标域的不存在账户发送邮件，通过分析邮件系统的响应（包括中继服务器和内容过滤设备）来映射内部网络空间。
    - 防：配置 SMTP 服务器在遇到问题时（例如不存在的邮箱）不发送 NDN，这将防止攻击者枚举内部邮件服务器和配置。

- IPv6 网络和 DNS 配置（如果有）。
