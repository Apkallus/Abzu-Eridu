### 查看公司工程职位的招聘信息

工程职位列表通常会揭示公司使用的技术。例如，看看这样的广告：

> **全栈工程师**
> 精通 Python 和 C/C++
> Linux 经验
> 有 Flask、Django 和 Node.js 经验
> 有 Amazon Web Services 经验，特别是 EC2、ECS、S3 和 RDS

通过阅读这些，你知道公司使用 Flask、Django 和 Node.js 构建其 Web 应用程序。工程师可能还在后端使用 Python、C 和 C++ 和 Linux 机器。最后，他们使用 AWS 来外包其操作和文件存储。

## Whois

注册域名时，需要向域名注册商提供标识信息。然后任何人都可以使用 `whois` 命令查询这些信息，该命令搜索每个已知域的注册人和所有者信息。可能会找到相关的联系信息，例如电子邮件、姓名、地址或电话号码

Whois 的现代版为 RDAP
- https://about.rdap.org/

### 对域名进行正向 WHOIS

查询域名注册机构以获取有关目标组织注册的域名信息。

ICANN 和 IANA 的注册机构列表：
- [gTLD 注册机构](https://www.icann.org/en/contracted-parties/registry-operators/resources/listings)
- [ccTLD 注册机构](http://www.iana.org/domains/root/db)

这些注册机构提供以下信息：
- 每个域名的权威名称服务器详细信息
- 每个域名的 WHOIS 服务器

执行域名 WHOIS 查询的工具：
- `whois` 命令行客户端
- 每个域名的 WHOIS 服务器
- Hurricane Electric（HE）、DomainTools 和其他第三方服务

当注册者使用第三方的域名隐私服务时，查询返回通用的电子邮件、电话号

### 对 IP 进行正向 WHOIS

区域互联网注册机构（RIR）提供与 IP 网络分配相关的信息，包括运维人员姓名、IP 网络块详细信息和物理办公地点。

区域互联网注册机构

    | 名称 | 区域 | 网站 |
    |------|------|------|
    | ARIN | 北美 | http://www.arin.net |
    | RIPE | 欧洲 | http://www.ripe.net |
    | APNIC | 亚太 | http://www.apnic.net |
    | LACNIC | 拉丁美洲和加勒比 | http://www.lacnic.net |
    | AFRNIC | 非洲 | http://www.afrnic.net |

查询 IP WHOIS 数据库的工具：
- `whois` 命令行客户端
- 区域互联网注册机构 WHOIS Web 界面
- 第三方应用程序（例如 DomainTools）

通过检查 `NetRange` 字段查看目标是否有专用的 IP 范围。IP 范围是属于同一组织的 IP 地址块。如果组织有专用的 IP 范围，那么你在该范围内找到的任何 IP 都属于该组织

### 使用域名注册信息进行反向 Whois

使用组织名称、电话号码或电子邮件地址搜索数据库，以找到使用这些信息注册的域名。这样，可以找到属于同一所有者的所有域名。

反向 WHOIS 对于查找未以其他方式向公众披露的冷门或内部域极其有用。

公共反向 WHOIS 工具：
- [ViewDNS.info](https://viewdns.info/reversewhois/)

WHOIS 和反向 WHOIS 将为你提供一组良好的顶级域进行工作。

## BGP 枚举 - 枚举 ASN 和 IP 段

公共互联网中的可路由网络之间的流量通过 BGP 和 AS 号（自治系统编号）进行控制。IANA 将 AS 号分配给 RIR，RIR 再将其分配给 ISP 和组织。

自治系统号（ASN）是分配给组织或公司的唯一标识符。它代表属于该实体的一系列 IP 地址。从枚举角度看，ASN 很重要，因为它能提供公司拥有的 IP 范围信息。

使用 `bgp.he.net` 搜索组织名得到其关联 AS 号。再引用 AS 号，得到 AS 号宣布的 IPv4/6 前缀。

`whois` 命令中的 `-h` 标志设置要从中检索信息的 WHOIS 服务器，`whois.cymru.com` 是一个将 IP 转换为 ASN 的数据库。

## SMTP 探测

邮件网关通过 SMTP 支持跨网络传输邮件。简单地发送一封电子邮件消息到目标域的一个不存在地址，通常通过未送达通知（NDN）揭示有用的内部网络信息

该记录中的以下数据很有用：

- 内部主机名、IP 地址和子域布局
- 邮件服务器运行 Microsoft Exchange Server 2010 SP3
- 使用 Barracuda Networks 设备执行内容过滤


邮件网关电子邮件内容过滤策略的研究：

## 枚举技术回顾

以下是对基于互联网的查询技术及其应用的概述：

- **Web 搜索**：
    - 攻：使用 Google、Netcraft、Shodan、LinkedIn、PGP 密钥服务器和其他站点，针对已知域名和 IP 块进行搜索，以识别人员、主机名、域名以及驻留在暴露 Web 服务器上的有用数据。
    - 防：禁用不包含 `index.html` 或类似文件（例如 Microsoft IIS 下的 `default.asp`）的目录索引文件的目录索引。

- **WHOIS 查询**：
    - 攻：查询域名和 IP 注册机构，以检索与目标网络和域名相关的网络块、路由和联系详细信息。IP WHOIS 查询提供有关保留网络块大小（在后续执行侵入性网络扫描时有用）和 AS 号详细信息的信息。
    - 防：在 WHOIS 数据库和 TLS 证书中使用通用的、集中的网络管理联系详细信息，以防止针对 IT 部门的社会工程。

- **BGP 枚举**：
    - 攻：将 AS 号与 BGP 站点交叉引用，以枚举该 AS 下关联的 IP 块，然后将这些详细信息反馈到其他查询路径（如 DNS 或进一步的 WHOIS 查询）。
    - 防：

- **DNS 查询**：
    - 攻：查询可访问的名称服务器以枚举域、子域、主机名和非公开 IP 地址详细信息。配置错误的 DNS 服务器还会提供列出子域、主机名、设备操作系统平台和内部网络详细信息的区域文件。
    - 防：
        - 配置名称服务器以不允许向互联网的主机进行 DNS 区域传输。
        - 精简 DNS 区域文件，使得不必要的信息不被披露（主要是非公开 IP 地址和主机名详细信息），并且 DNS 猜测攻击无效。理想情况下，仅在绝对需要时使用 PTR 记录（用于 SMTP 邮件服务器和其他需要双向解析的关键系统）。

- **SMTP 探测**：
    - 攻：向目标域的不存在账户发送邮件，通过分析邮件系统的响应（包括中继服务器和内容过滤设备）来映射内部网络空间。
    - 防：配置 SMTP 服务器在遇到问题时（例如不存在的邮箱）不发送 NDN，这将防止攻击者枚举内部邮件服务器和配置。

- IPv6 网络和 DNS 配置（如果有）。
