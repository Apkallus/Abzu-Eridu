# 测试反向标签劫持

|ID          |
|------------|
|WSTG-CLNT-14|

## 概述

[反向标签劫持](https://owasp.org/www-community/attacks/Reverse_Tabnabbing)是一种可用于将用户重定向到钓鱼页面的攻击。这通常是因为 `<a>` 标签的 `target` 属性设置为 `_blank`，导致链接在新标签页中打开。如果在同一个 `<a>` 标签中未使用 `rel='noopener noreferrer'` 属性，则新打开的页面可以影响原始页面，并将其重定向到攻击者控制的域。

由于新标签页打开时用户正在原始域上，他们不太可能注意到页面已更改，尤其是当钓鱼页面与原始域完全相同时。因此，在攻击者控制的域上输入的任何凭据都将落入攻击者手中。

通过 `window.open` JavaScript 函数打开的链接也容易受到此攻击。

_注意：这是一个遗留问题，[现代浏览器](https://caniuse.com/mdn-html_elements_a_implicit_noopener)不受影响。流行浏览器的旧版本（例如 Google Chrome 88 之前的版本）以及 Internet Explorer 容易受到此攻击。_

### 示例

想象一个允许用户在其个人资料中插入 URL 的 Web 应用程序。如果该应用程序容易受到反向标签劫持攻击，恶意用户将能够提供一个指向包含以下代码页面的链接：

```html
<html>
 <body>
  <script>
    window.opener.location = "https://example.org";
  </script>
<b>加载错误...</b>
 </body>
</html>
```

点击该链接将打开一个新标签页，同时原始标签页将重定向到“example.org”。假设“example.org”看起来与易受攻击的 Web 应用程序相似，用户不太可能注意到更改，并且更有可能在该页面上输入敏感信息。

## 如何测试

- 检查应用程序的 HTML 源代码，查看带有 `target="_blank"` 的链接是否在 `rel` 属性中使用了 `noopener` 和 `noreferrer` 关键字。如果没有，则应用程序很可能容易受到反向标签劫持攻击。如果该链接指向已被攻击者破坏的第三方站点，或者由用户控制，则此类链接变得可以利用。
- 检查攻击者可以插入链接的区域，即控制 `<a>` 标签的 `href` 参数。尝试插入一个指向上述示例中源代码页面的链接，并查看原始域是否被重定向。如果其他浏览器无效，此测试可以在 IE 中进行。

## 修复

建议确保所有链接的 `rel` HTML 属性都设置 `noreferrer` 和 `noopener` 关键字。
