# 4.0 引言与目标

本章节描述了 OWASP Web 应用程序安全测试方法论，并解释了如何因安全控制的缺陷而测试应用程序中存在的漏洞证据。

## 什么是 Web 应用程序安全测试？

安全测试是一种通过系统性地验证和确认应用程序安全控制的有效性，来评估计算机系统或网络安全性的方法。Web 应用程序安全测试仅专注于评估 Web 应用程序的安全性。该过程涉及对应用程序进行主动分析，以发现任何弱点、技术缺陷或漏洞。发现的任何安全问题都将连同影响评估、缓解建议或技术解决方案一起提交给系统所有者。

## 什么是漏洞？

漏洞是系统设计、实现、操作或管理中的缺陷或弱点，可能被利用来破坏系统的安全目标。

## 什么是威胁？

威胁是指任何可能利用漏洞损害应用程序所拥有资产（有价值的资源，例如数据库或文件系统中的数据）的事物（恶意外部攻击者、内部用户、系统不稳定等）。

## 什么是测试？

测试是证明应用程序满足其利益相关者安全需求的行为。

## 编写本指南的方法

OWASP 的方法是开放且协作的：

- 开放：每位安全专家都可以用他们的经验参与项目。一切都是免费的。
- 协作：在编写文章之前进行头脑风暴，以便团队分享想法并形成项目的集体愿景。这意味着粗略共识、更广泛的受众和更高的参与度。

这种方法倾向于产生一个定义的测试方法论，该方法论将：

- 一致的
- 可复现的
- 严谨的
- 受质量控制的

需要解决的问题被充分记录和测试。使用各种方法测试所有已知漏洞并记录所有安全测试活动非常重要。

## 什么是 OWASP 测试方法论？

安全测试永远不会是一门精确的科学，无法定义应测试的所有可能问题的完整列表。事实上，安全测试只是在特定情况下测试 Web 应用程序安全性的几种合适技术之一。本项目的目标是收集所有可能的测试技术，解释这些技术，并保持指南的更新。OWASP Web 应用程序安全测试方法论基于黑盒方法。测试人员对要测试的应用程序几乎一无所知。

测试模型包括：

- 测试人员：执行测试活动的人
- 工具和方法论：本测试指南项目的核心
- 应用程序：待测试的黑盒

测试可以分为被动测试和主动测试：

### 被动测试

在被动测试期间，测试人员尝试理解应用程序的逻辑，并像最终用户一样探索应用程序。可以使用工具进行信息收集。例如，可以使用 HTTP(S) 代理来观察所有 HTTP(S) 请求和响应。在此阶段结束时，测试人员通常应了解系统的所有访问点和功能（例如，HTTP 头部、参数、cookies、API、技术使用模式等）。[信息收集](../01-Information_Gathering/README.md)章节解释了如何执行被动测试。

例如，测试人员可能在以下 URL 找到一个页面：`https://www.example.com/login/auth_form`

这可能表示一个身份验证表单，应用程序要求输入用户名和密码。

以下参数表示应用程序的两个访问点：`https://www.example.com/appx?a=1&b=1`

在这种情况下，应用程序有两个访问点（参数 `a` 和 `b`）。在此阶段发现的所有输入点都是测试的目标。在主动测试期间，跟踪应用程序的目录或调用树以及所有访问点可能很有用。

### 主动测试

在主动测试期间，测试人员使用以下章节中描述的方法。

主动测试集分为 12 个类别：

- 信息收集
- 配置和部署管理测试
- 身份管理测试
- 身份验证测试
- 授权测试
- 会话管理测试
- 输入验证测试
- 错误处理测试
- 弱加密测试
- 业务逻辑测试
- 客户端测试
- API 测试
