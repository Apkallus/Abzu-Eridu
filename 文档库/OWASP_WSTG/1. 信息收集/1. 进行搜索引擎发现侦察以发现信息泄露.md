# 进行搜索引擎发现侦察以发现信息泄露

|ID          |
|------------|
|WSTG-INFO-01|

## 概要

搜索引擎的计算机程序（或称为 `机器人`）定期从网络上数十亿个页面中爬取数据。这些程序通过跟随其他页面的链接或查看站点地图来发现 Web 内容和功能。如果网站使用一个名为 `robots.txt` 的特殊文件来列出不希望搜索引擎获取的页面，那么其中列出的页面将被忽略。

测试人员可以利用搜索引擎对网站和 Web 应用程序进行侦察。搜索引擎发现和侦察包含直接和间接两个层面：
- 直接方法涉及搜索索引及缓存中的相关内容
- 间接方法则通过搜索论坛、新闻组和招标网站来获取敏感的设计和配置信息

一旦搜索引擎机器人完成爬取，它会基于标签及相关属性（如 `<TITLE>`）开始对 Web 内容建立索引，以便返回相关的搜索结果。如果在网站生命周期内未更新 `robots.txt` 文件，并且未使用指示机器人不要索引内容的内嵌 HTML meta 标签，那么索引中就可能包含网站所有者不希望被收录的 Web 内容。网站所有者可以使用前面提到的 `robots.txt`、HTML meta 标签、身份验证以及搜索引擎提供的工具来删除此类内容。

## 测试目标

- 识别应用程序、系统或组织的哪些敏感设计和配置信息被直接（在组织网站上）或间接（通过第三方服务）暴露。

## 如何测试

使用搜索引擎搜索潜在的敏感信息。这些信息可能包括：

- 网络拓扑图和配置；
- 管理员或其他关键人员的存档帖子和电子邮件；
- 登录流程和用户名格式；
- 用户名、密码和私钥；
- 第三方或云服务配置文件；
- 泄露的错误消息内容；
- 非公开应用程序（开发、测试、用户验收测试（UAT）及预发布版本的网站）。

### 搜索引擎

不要将测试限制在仅使用一个搜索引擎提供商，因为不同的搜索引擎可能产生不同的结果。搜索引擎结果可能在几个方面有所不同，取决于搜索引擎上次爬取内容的时间以及其确定相关页面的算法。考虑使用以下（按字母顺序排列的）搜索引擎：

- [Baidu](https://www.baidu.com/)，中国[最流行](https://en.wikipedia.org/wiki/Web_search_engine#Market_share)的搜索引擎。
- [Bing](https://www.bing.com/)，由微软拥有和运营的搜索引擎，全球第二[流行](https://en.wikipedia.org/wiki/Web_search_engine#Market_share)。支持[高级搜索关键字](https://help.bing.microsoft.com/#apex/18/en-US/10001/-1)。
- [binsearch.info](https://binsearch.info/)，用于二进制 Usenet 新闻组的搜索引擎。
- [Common Crawl](https://commoncrawl.org/)，“一个开放的 Web 爬取数据存储库，任何人都可以访问和分析。”
- [DuckDuckGo](https://duckduckgo.com/)，一个注重隐私的搜索引擎，从许多不同的[来源](https://help.duckduckgo.com/results/sources/)汇总结果。支持[搜索语法](https://help.duckduckgo.com/duckduckgo-help-pages/results/syntax/)。
- [Google](https://www.google.com/)，提供全球[最流行](https://en.wikipedia.org/wiki/Web_search_engine#Market_share)的搜索引擎，并采用排名系统尝试返回最相关的结果。支持[搜索运算符](https://support.google.com/websearch/answer/2466433)。
- [Internet Archive Wayback Machine](https://archive.org/web/)，“以数字形式构建互联网站点和其他文化文物的数字图书馆。”
- [Censys](https://censys.io) 是一个专注于安全的搜索引擎，索引互联网连接的基础设施，包括服务器、证书和开放服务。它提供有限的免费社区版（每月有信用额度）以及面向企业使用的付费计划。
- [Shodan](https://www.shodan.io/)，一个用于搜索互联网连接设备和服务的服务。使用选项包括有限的免费计划和付费订阅计划。

### 搜索运算符

搜索运算符是一种特殊的关键字或语法，可以扩展常规搜索查询的能力，并有助于获得更具体的结果。它们通常采用 `operator:query` 的形式。以下是一些常见支持的搜索运算符：

- `site:` 将搜索限制在指定的域名内。
- `inurl:` 仅返回 URL 中包含该关键字的搜索结果。
- `intitle:` 仅返回页面标题中包含该关键字的搜索结果。
- `intext:` 或 `inbody:` 仅在页面正文中搜索关键字。
- `filetype:` 仅匹配特定的文件类型，例如 `.png` 或 `.php`。

例如，要查找典型搜索引擎索引的 owasp.org 的 Web 内容，所需语法为：

```text
site:owasp.org
```

#### Internet Archive Wayback Machine

[Internet Archive Wayback Machine](https://archive.org/web/) 是查看网页历史快照的最全面的工具。它维护着可追溯到 1996 年的广泛网页存档。

要查看网站的存档版本，请访问 `https://web.archive.org/web/*/`，后跟目标 URL：

```text
https://web.archive.org/web/*/owasp.org
```

这将显示一个日历视图，展示该网站随时间变化的所有可用快照。

#### 其他缓存内容服务

用于查看缓存或存档网页的其他服务包括：

- [archive.ph](https://archive.ph)（也称为 archive.md）- 按需存档服务，创建永久快照
- [CachedView](https://cachedview.com/) - 汇集来自多个来源的缓存页面，包括 Google 缓存历史数据、Wayback Machine 等

### Google Hacking 或 Dorking

当结合测试人员的创造力时，使用运算符进行搜索可能是一种非常有效的发现技术。运算符可以链式使用，以有效发现特定类型的敏感文件和信息。这种技术被称为 [Google hacking](https://en.wikipedia.org/wiki/Google_hacking) 或 Dorking，只要搜索运算符受支持，也可以使用其他搜索引擎来实现。

一个 dork 数据库，例如 [Google Hacking Database](https://www.exploit-db.com/google-hacking-database)，是一个有用的资源，可以帮助发现特定信息。
AI 辅助查询生成器如 [DorkGPT](https://www.dorkgpt.com) 可以将自然语言提示转换为 Google dork 语法，减少构建复杂搜索运算符的手动工作。
该数据库中的一些 dork 类别包括：

- 立足点
- 包含用户名的文件
- 敏感目录
- Web 服务器检测
- 易受攻击的文件
- 易受攻击的服务器
- 错误消息
- 包含敏感信息的文件
- 包含密码的文件
- 敏感在线购物信息

## 开源情报关联工具

除了单个搜索引擎之外，测试人员还可以使用专用的开源情报框架来关联和可视化发现实体之间的关系：

[Maltego](https://maltego.com) 是一个行业标准的 OSINT 和链接分析平台，通过自动化的数据转换来映射域名、IP 地址、电子邮件地址和组织之间的关系。测试人员用它通过从单个实体进行 pivot 操作来发现相关的基础设施和关联数据点，从而可视化组织的攻击面。非商业用途可免费获得社区版。

## 修复措施

在将设计和配置信息发布到网上之前，仔细考虑其敏感性。

定期审查已发布在网上的现有设计和配置信息的敏感性。
