# 测试路径混淆

|ID          |
|------------|
|WSTG-CONF-13|

## 概述

应用程序路径的正确配置非常重要，因为如果路径配置不当，攻击者可能利用此配置错误在后续阶段利用其他漏洞。

例如，如果路由配置不正确，且目标同时使用了 CDN，攻击者可以利用此配置错误执行 Web 缓存欺骗攻击。

因此，为防止其他攻击，测试人员应对此配置进行评估。

## 测试目标

- 确保应用程序路径配置正确。

## 如何测试

### 黑盒测试

在黑盒测试场景中，测试人员应将所有现有路径替换为不存在的路径，然后检查目标的行为和状态码。

例如，应用程序中有一个仪表盘路径，用于显示用户的账户余额（金钱、游戏积分等）信息。

假设路径为 `https://example.com/user/dashboard`，测试人员应测试开发人员可能为此路径考虑的不同模式。对于 Web 缓存欺骗漏洞，分析人员应考虑诸如 `https://example.com/user/dashboard/non.js` 之类的路径。如果仪表盘信息可见，且目标使用了 CDN（或其他 Web 缓存），则很可能适用 Web 缓存欺骗攻击。

### 白盒测试

检查应用程序路由配置。大多数情况下，开发人员在应用程序路由中使用正则表达式。

在此示例中，在 Django 框架应用程序的 `urls.py` 文件中，我们看到了一个路径混淆的示例。开发人员未使用正确的正则表达式，从而导致漏洞：

```python
    from django.urls import re_path
    from . import views

    urlpatterns = [

        re_path(r'.*^dashboard', views.path_confusion ,name = 'index'),

    ]
```

如果用户也在浏览器中打开路径 `https://example.com/dashboard/none.js`，则可能会显示用户的仪表盘信息，并且如果目标使用 CDN 或 Web 缓存，则可以实现 Web 缓存欺骗攻击。

## 工具

- [Zed Attack Proxy](https://www.zaproxy.org)
- [Burp Suite](https://portswigger.net/burp)

## 修复

- 避免基于文件扩展名或路径进行缓存分类/处理（应利用 content-type）。
- 确保缓存机制遵循应用程序指定的 cache-control 头部。
- 实施符合 RFC 规范的文件未找到处理和重定向。
