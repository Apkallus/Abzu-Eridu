# 客户端模板注入

在本节中，我们将探讨客户端模板注入漏洞以及如何利用它们发起 XSS 攻击。此攻击技术由我们的研究团队首创——更多信息请阅读 [XSS without HTML: Client-Side Template Injection with AngularJS](https://portswigger.net/research/xss-without-html-client-side-template-injection-with-angularjs)。尽管客户端模板注入是一个通用问题，但我们将聚焦于 AngularJS 框架中的示例，因为它最为常见。我们将介绍如何构造逃逸 AngularJS 沙箱的利用方式，以及如何可能利用 AngularJS 特性绕过内容安全策略（CSP）。

## 什么是客户端模板注入？

当使用客户端模板框架的应用程序将用户输入动态嵌入到网页中时，就会出现客户端模板注入漏洞。在渲染页面时，框架会扫描页面中的模板表达式并执行它遇到的任何表达式。攻击者可以通过提供一个恶意模板表达式来利用这一点，从而发起跨站脚本（XSS）攻击。

## 什么是 AngularJS 沙箱？

AngularJS 沙箱是一种机制，用于防止在 AngularJS 模板表达式中访问潜在危险的对象，如 `window` 或 `document`。它还会阻止访问潜在危险的属性，如 `__proto__`。尽管 AngularJS 团队并不将其视为安全边界，但更广泛的开发者社区通常持相反意见。虽然一开始绕过沙箱颇具挑战性，但安全研究人员已经发现了多种方法。因此，它最终在 AngularJS 1.6 版本中被移除。然而，许多遗留应用程序仍在使用旧版本的 AngularJS，因此可能仍然存在漏洞。

## AngularJS 沙箱如何工作？

沙箱的工作原理是解析表达式、重写 JavaScript，然后使用各种函数测试重写后的代码是否包含任何危险对象。例如，`ensureSafeObject()` 函数检查给定对象是否引用自身。这是检测 `window` 对象的一种方式。`Function` 构造器以大致相同的方式被检测，即检查 `constructor` 属性是否引用自身。

`ensureSafeMemberName()` 函数检查对象的每个属性访问，如果包含危险属性（如 `__proto__` 或 `__lookupGetter__`），该对象将被阻止。`ensureSafeFunction()` 函数阻止 `call()`、`apply()`、`bind()` 或 `constructor()` 被调用。

您可以通过访问 [此 fiddle](http://jsfiddle.net/2zs2yv7o/1/) 并在 `angular.js` 文件的第 13275 行设置断点来亲自查看沙箱的运行情况。变量 `fnString` 包含您重写后的代码，因此您可以查看 AngularJS 如何转换它。

## AngularJS 沙箱逃逸如何工作？

沙箱逃逸涉及欺骗沙箱，使其认为恶意表达式是无害的。最著名的逃逸方式是在表达式中全局使用修改后的 `charAt()` 函数：

```javascript
'a'.constructor.prototype.charAt = [].join
```

当它最初被发现时，AngularJS 并未阻止此修改。该攻击通过使用 `[].join` 方法覆盖函数来工作，这导致 `charAt()` 函数返回发送给它的所有字符，而不是特定的单个字符。由于 AngularJS 中 `isIdent()` 函数的逻辑，它会将所认为的单个字符与多个字符进行比较。由于较长字符串的第一个字符在被比较字符的范围内，`isIdent()` 函数总是返回 `true`，如下例所示：

```javascript
isIdent = function(ch) {
    return ('a' <= ch && ch <= 'z' || 'A' <= ch && ch <= 'Z' || '_' === ch || ch === '$');
}
isIdent('x9=9a9l9e9r9t9(919)')
```

一旦 `isIdent()` 函数被欺骗，您就可以注入恶意 JavaScript。例如，诸如 `$eval('x=alert(1)')` 这样的表达式将被允许，因为 AngularJS 将每个字符都视为标识符。请注意，我们需要使用 AngularJS 的 `$eval()` 函数，因为覆盖 `charAt()` 函数只会在沙箱代码执行后生效。此技术将绕过沙箱并允许任意 JavaScript 执行。

### 构造高级 AngularJS 沙箱逃逸

您已经了解了基本沙箱逃逸的工作原理，但您可能会遇到在允许字符方面限制更严格的站点。例如，站点可能阻止您使用双引号或单引号。在这种情况下，您需要使用诸如 `String.fromCharCode()` 之类的函数来生成字符。虽然 AngularJS 阻止在表达式中访问 `String` 构造器，但您可以通过使用字符串的 `constructor` 属性来绕过这一点。这显然需要一个字符串，因此要构造此类攻击，您需要找到一种不使用单引号或双引号来创建字符串的方法。

在标准的沙箱逃逸中，您会使用 `$eval()` 来执行您的 JavaScript 载荷，但在下面的实验中，`$eval()` 函数是未定义的。幸运的是，我们可以使用 `orderBy` 过滤器作为替代。`orderBy` 过滤器的典型语法如下：

```
[123]|orderBy:'Some string'
```

请注意，`|` 运算符的含义与 JavaScript 中不同。通常，这是按位 `OR` 操作，但在 AngularJS 中，它表示过滤器操作。在上面的代码中，我们将左侧的数组 `[123]` 发送给右侧的 `orderBy` 过滤器。冒号表示要发送给过滤器的参数，在此例中是一个字符串。`orderBy` 过滤器通常用于对对象进行排序，但它也接受表达式，这意味着我们可以使用它来传递载荷。

## AngularJS CSP 绕过如何工作？

内容安全策略（CSP）绕过的工作方式与标准沙箱逃逸类似，但通常涉及一些 HTML 注入。当在 AngularJS 中激活 CSP 模式时，它会以不同方式解析模板表达式，并避免使用 `Function` 构造器。这意味着上述标准沙箱逃逸将不再有效。

根据具体策略，CSP 会阻止 JavaScript 事件。然而，AngularJS 定义了自己可用的事件。当在事件内部时，AngularJS 定义了一个特殊的 `$event` 对象，它简单地引用浏览器事件对象。您可以使用此对象执行 CSP 绕过。在 Chrome 上，`$event/event` 对象上有一个特殊属性叫做 `path`。此属性包含一个对象数组，这些对象导致事件被执行。最后一个属性始终是 `window` 对象，我们可以用它来执行沙箱逃逸。通过将此数组传递给 `orderBy` 过滤器，我们可以枚举该数组并使用最后一个元素（`window` 对象）来执行全局函数，如 `alert()`。以下代码演示了这一点：

```html
<input autofocus ng-focus="$event.path|orderBy:'[].constructor.from([1],alert)'">
```

注意，这里使用了 `from()` 函数，它允许您将对象转换为数组，并对该数组的每个元素调用给定的函数（在第二个参数中指定）。在此例中，我们调用了 `alert()` 函数。我们不能直接调用该函数，因为 AngularJS 沙箱会解析代码并检测到正在使用 `window` 对象来调用函数。使用 `from()` 函数可以有效地将 `window` 对象隐藏起来，使其不被沙箱发现，从而允许我们注入恶意代码。

### 结合 AngularJS 沙箱逃逸绕过 CSP

下一个实验采用了长度限制，因此上述向量将不起作用。为了利用该实验，您需要考虑各种将 `window` 对象隐藏起来不让 AngularJS 沙箱发现的方法。其中一种方法是使用 `array.map()` 函数，如下所示：

```javascript
[1].map(alert)
```

`map()` 接受一个函数作为参数，并会对数组中的每个项调用该函数。这将绕过沙箱，因为对 `alert()` 函数的引用是在没有显式引用 `window` 的情况下使用的。要解决该实验，请尝试各种方式来执行 `alert()` 而不触发 AngularJS 的 `window` 检测。

## 如何防止客户端模板注入漏洞

为了防止客户端模板注入漏洞，请避免使用不受信任的用户输入来生成模板或表达式。如果这不可行，请考虑在将用户输入嵌入客户端模板之前，过滤掉其中的模板表达式语法。

请注意，HTML 编码不足以防止客户端模板注入攻击，因为框架会在定位和执行模板表达式之前对相关内容执行 HTML 解码。
