# AngularJS CSP bypass in 56 characters（56 字符的 AngularJS CSP 绕过）

> 作者：**Gareth Heyes**（研究员）  
> Twitter：[@garethheyes](https://twitter.com/garethheyes)

- **发布时间：** 2019 年 10 月 14 日（星期一）13:04 UTC
- **更新日期：** 2023 年 1 月 12 日（星期四）08:36 UTC

在测试网站时，你经常会遇到对输入长度的限制，这就是为什么尽可能缩短向量长度很重要——当然这也很有趣。

当我在 [Allstars](https://ams.globalappsec.org/program/allstars) 进行 [XSS 魔术技巧](https://www.slideshare.net/GarethHeyes/xss-magic-tricks) 演讲时，我展示了一种使用 [AngularJS](/web-security/cross-site-scripting/contexts/client-side-template-injection) 绕过 [CSP](/web-security/cross-site-scripting/content-security-policy) 的技术，长度为 63 个字符：

```html
<input id=x ng-focus=$event.path|orderBy:'CSS&&[1].map(alert)'>
```

其工作原理是利用 Chrome 中的 `path` 属性，它是一个 DOM 对象数组，其中最后一个元素包含 `window` 对象。AngularJS 中的管道字符表示过滤器操作，因此 `path` 数组被发送到 `orderBy` 过滤器，该过滤器接受一个表达式。然后当表达式执行时，其执行上下文是数组的当前元素。然后我使用 `CSS` 属性来检测当前数组元素是否为最后一个元素（即 `window`）。然后使用 `map` 函数执行 `alert` 函数，因为如果作为普通函数调用，沙箱会检测到 `window`。向量如下：

我向观众发起挑战，要求进一步缩短它，[Erlend Oftedal](https://twitter.com/webtonull) 通过将 `CSS` 属性替换为 `x`（注入的 input 元素的 ID）成功做到了。这是因为 `x` 将成为 `window` 的全局属性，因此可以用来检测 `window` 对象。如下所示：

```html
<input id=x ng-focus=$event.path|orderBy:'x&&[1].map(alert)'>
```

我将这个向量发布在 [Slackers](https://www.reddit.com/r/Slackers/comments/d9mwnt/angularjs_csp_bypass_can_you_make_it_shorter/) 上，并发起挑战要求进一步缩短。我自己也接受了挑战，思考如何在不触发 AngularJS `window` 检测的情况下调用 `alert`。我想到了使用数组和 `pop` 方法：

```html
<input id=x ng-focus=$event.path|orderBy:'x&&[alert].pop()(1)'>
```

这个载荷与原始长度相同。但等等——AngularJS 允许你在不抛出异常的情况下静默调用未定义的函数。因此我可以通过移除 `&&` 和属性检查来缩短载荷：

```html
<input id=x ng-focus=$event.path|orderBy:'[alert].pop()(1)'>
```

我仍然决心进一步缩短它，尝试使用不带 `pop` 的数组，通过引用第一个元素并调用它：

```html
<input id=x ng-focus=$event.path|orderBy:'[alert][0](1)'>
```

但这会导致非法操作，因为调用对象不是 `window`，JavaScript 只允许在 `window` 上调用 `alert`。所以我需要一种方法来直接调用 `alert`，同时保持对 `window` 的引用，同时避免 AngularJS 的 `window` 检测。最终我想出了这个简单而优雅的方法——使用赋值来规避 AngularJS 的 `window` 对象检测：

```html
<input id=x ng-focus=$event.path|orderBy:'(y=alert)(1)'>
```

[AngularJS CSP 绕过 PoC](http://portswigger-labs.net/xss/angularjs.php?type=reflected&csp=1&version=1.4.5&x=%3Cinput%20id=x%20ng-focus=$event.path|orderBy:%27(y=alert)(1)%27%3E#x)

## 更新...

自 Chrome 109 起，`path` 属性已被移除。替代方案是使用 `composedPath()`：

```html
<input id=x ng-focus=$event.composedPath()|orderBy:'(y=alert)(1)'>
```
