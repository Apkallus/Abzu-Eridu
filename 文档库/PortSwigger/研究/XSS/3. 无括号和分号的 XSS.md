# XSS without parentheses and semi-colons（无括号和分号的 XSS）

https://portswigger.net/research/xss-without-parentheses-and-semi-colons

> 作者：**Gareth Heyes**（研究员）  
> Twitter：[@garethheyes](https://twitter.com/garethheyes)

- **发布时间：** 2019 年 5 月 15 日（星期三）14:54 UTC
- **更新日期：** 2020 年 3 月 20 日（星期五）07:50 UTC

几年前，我发现了一种使用 `onerror` 和 `throw` 语句[在 JavaScript 中无需括号即可调用函数](http://www.thespanner.co.uk/2012/05/01/xss-technique-without-parentheses/)的技术。它的工作原理是将 `onerror` 处理程序设置为要调用的函数，然后使用 `throw` 语句将参数传递给该函数：

```html
<script>onerror=alert;throw 1337</script>
```

每当 JavaScript 异常被创建时，`onerror` 处理程序都会被调用，而 `throw` 语句允许您创建一个包含表达式的自定义异常，该表达式会被发送到 `onerror` 处理程序。由于 `throw` 是一个语句，您通常需要在 `onerror` 赋值后面加上分号，以便开始一个新语句而不是形成一个表达式。

我遇到了一个过滤括号和分号的网站，我想肯定可以调整这种技术，在没有分号的情况下执行函数。第一种方法相当直接：您可以使用花括号形成一个块语句，在其中放置 `onerror` 赋值。在块语句之后，您可以使用 `throw` 而无需分号（或换行）：

```html
<script>{onerror=alert}throw 1337</script>
```

块语句已经很好了，但我想要一个更酷的替代方案。有趣的是，由于 `throw` 语句接受一个表达式，您可以在 `throw` 语句内部进行 `onerror` 赋值，并且由于表达式的最后一部分被发送到 `onerror` 处理程序，该函数将被调用并传入所选参数。其工作原理如下：

![示例：使用带有表达式的 throw 语句](/cms/images/07/f8/03b498b7f41b-article-onerror.png)

```html
<script>throw onerror=alert,'some string',123,'haha'</script>
```

如果您尝试运行这段代码，您会注意到 Chrome 会在发送到异常处理程序的字符串前加上 "Uncaught" 前缀。

![Chrome 中显示 Uncaught 的 alert 框](/cms/images/26/b9/2e3196af1042-article-uncaught-alert.png)

在我之前的博文中，我展示了如何使用 `eval` 作为异常处理程序并计算字符串。回顾一下，您可以在字符串前加上 `=`，这样 "Uncaught" 字符串就变成了一个变量，从而执行任意 JavaScript。例如：

```html
<script>{onerror=eval}throw'=alert\x281337\x29'</script>
```

发送给 `eval` 的字符串是 `"Uncaught=alert(1337)"`。这在 Chrome 上运行良好，但在 Firefox 上，异常会被加上两个单词的前缀 "uncaught exception"，这当然会在求值时导致语法错误。我开始寻找绕过此问题的方法。

值得注意的是，当从控制台执行 `throw` 时，`onerror/throw` 技巧不起作用。这是因为当 `throw` 语句在控制台中执行时，结果会发送到控制台而不是异常处理程序。

当您在 Firefox 中使用 `Error` 函数创建异常时，它不包含 "uncaught exception" 前缀，而只有字符串 "Error"：

```javascript
throw new Error("My message")//Error: My message
```

我当然无法调用 `Error` 函数，因为它需要括号，但我想也许如果我使用带有 `Error` 原型的对象字面量，可能会模拟这种行为。这没有起作用 —— Firefox 仍然添加了相同的前缀。然后我使用 [Hackability Inspector](http://portswigger-labs.net/hackability/inspector/?input=new%20Error(%27blah%27)) 检查了 Error 对象以查看其属性。我将所有属性添加到对象字面量中，结果成功了！我逐个移除属性，以找到所需的最小属性集合：

```html
<script>{onerror=eval}throw{lineNumber:1,columnNumber:1,fileName:1,message:'alert\x281\x29'}</script>
```

您还可以在 Firefox 上使用 `fileName` 属性来传递第二个参数：

```html
<script>{onerror=prompt}throw{lineNumber:1,columnNumber:1,fileName:'second argument',message:'first argument'}</script>
```

当我在 Twitter 上发布这些内容后，[@terjanq](https://twitter.com/terjanq) 和 [@cgvwzq](https://twitter.com/cgvwzq)（Pepe Vila）跟进了一些很酷的向量。这里 @terjanq 移除了所有字符串字面量：

```html
<script>throw/a/,Uncaught=1,g=alert,a=URL+0,onerror=eval,/1/g+a[12]+[1337]+a[13]</script>
```

Pepe 通过使用类型错误将字符串发送到异常处理程序，完全去除了对 `throw` 语句的需求：

```html
<script>TypeError.prototype.name ='=/',0[onerror=eval]['/-alert(1)//']</script>
```
