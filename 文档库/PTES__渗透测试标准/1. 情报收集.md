# 通用部分

本节定义了渗透测试中情报收集活动的内容。本文档旨在为渗透测试人员针对目标（通常为企业、军事或相关机构）进行侦察提供一套标准。文档详细阐述了渗透测试侦察的思维过程和目标，若使用得当，可帮助读者制定出高度战略性的目标攻击计划。

## 背景概念

对于本文档以及整个 PTES 而言，“级别”是一个重要概念。它类似于渗透测试的成熟度模型。定义级别有助于我们明确在特定现实约束（如时间、精力、信息获取途径等）下的预期输出和活动。

情报收集级别目前分为三个类别，每个类别均给出了典型示例。这些示例将指导本文档后续技术内容的添加。例如，创建 Facebook 个人资料并分析目标社交网络这类高强度活动，适用于更高级的测试场景，并应标注相应级别。具体示例请参见下面的思维导图。

### 级别 1 信息收集
（思考：合规驱动）主要是“一键式”的信息收集过程。此级别的信息几乎完全可以由自动化工具获取。仅满足渗透测试中“进行了情报收集”的最低要求。

Acme Corporation 需要符合 PCI / FISMA / HIPAA 等合规性要求。级别 1 的信息收集工作应足以满足合规性需求。

### 级别 2 信息收集
（思考：最佳实践）此级别可使用级别 1 的自动化工具结合部分手动分析完成。需要深入理解业务，包括物理位置、业务关系、组织架构图等信息。

Widgets Inc 需要符合 PCI 合规性要求，但同时关注其长期安全战略，并且正在收购几家较小的 widget 制造商。级别 2 的信息收集工作应足以满足其需求。

### 级别 3 信息收集
（思考：国家支持型）更高级的渗透测试、红队测试、全范围测试。包含级别 1 和级别 2 的所有信息，并辅以大量手动分析。例如在社交网络上培养关系、深度分析、深刻理解业务关系，通常需要耗费大量工时来完成信息收集和关联分析。

陆军红队被指派分析和攻击某外国陆军网络的一个网段，以寻找可能被外国势力利用的弱点。在此情况下，级别 3 的信息收集工作是合适的。

# 情报收集

## 什么是情报收集
情报收集是对目标进行侦察，尽可能多地收集信息，以便在漏洞评估和漏洞利用阶段用于渗透目标。在此阶段收集的信息越多，未来可能使用的攻击向量就越多。

开源情报（OSINT）是一种情报收集管理形式，涉及从公开可用来源中查找、选择、获取信息，并对其进行分析以产生可操作的情报。[http://en.wikipedia.org/wiki/Open_source_intelligence]

## 为什么要进行情报收集
我们进行开源情报收集是为了确定进入组织的各种入口点。这些入口点可以是物理的、电子的和/或人的。许多公司未能考虑他们公开发布了哪些关于自己的信息，以及这些信息如何被坚定的攻击者利用。此外，许多员工也未能考虑他们公开发布了哪些关于自己的信息，以及这些信息如何被用来攻击他们自己或他们的雇主。

## 情报收集不是什么
OSINT 可能不准确或不是实时的。信息源可能被故意或意外操纵以反映错误数据，信息可能随时间推移而过时，或者仅仅是不完整的。

它不包括垃圾箱搜寻或从现场发现的物理物品中检索公司信息的任何方法。

# 目标选择
## 目标的识别与命名
在接触目标组织时，重要的是要理解公司可能拥有多个不同的顶级域名（TLD）和附属业务。虽然这些信息本应在范围界定阶段发现，但在参与前阶段讨论的初始范围之外识别出额外的服务器、域名和公司也并非罕见。例如，一家公司可能拥有 .com 的 TLD，但也可能拥有 .net、.co 和 .xxx。这些可能需要成为修订后范围的一部分，也可能被禁止测试。无论哪种情况，都需要在测试开始前与客户确认。此外，公司旗下拥有多家子公司也相当常见，例如通用电气和宝洁公司就拥有大量较小的公司。

## 考虑交战规则的任何限制
此时，最好回顾一下交战规则。在测试过程中，这些规则常常被遗忘。有时，作为测试人员，我们过于专注于发现的内容和攻击的可能性，以至于忘记了可以攻击哪些 IP 地址、域名和网络。请始终参考交战规则，以保持测试的聚焦性。这不仅从法律角度很重要，从范围蔓延的角度也同样重要。每次你偏离测试的核心目标，都会耗费时间。长远来看，这可能会让你的公司损失金钱。

## 考虑测试的时间长度
整个测试的时间长度将直接影响可以进行的情报收集量。有些测试总时长为两到三个月。在此类渗透测试中，测试公司会花费大量时间研究公司的每个核心业务部门和个人。然而，对于较短周期的“水晶盒”式测试，目标可能更具战术性。例如，测试一个特定的 Web 应用程序可能不需要你研究公司 CEO 的财务记录。

## 考虑测试的最终目标
每个测试都有一个最终目标——组织认为关键的一个特定资产或流程。牢记最终结果，情报收集阶段应确保包含围绕最终目标的所有次级和三级要素，无论是支持技术、第三方、相关人员等。确保重点放在关键资产上，可以降低其他不太相关的情报要素的优先级并进行归类，以免干扰分析过程。

# 开源情报（OSINT）
开源情报分为三种形式：被动、半被动和主动。

* **被动信息收集**：通常仅在明确要求信息收集活动绝不被目标检测到时才有用。这种剖析在技术上难以执行，因为我们从不向目标组织发送任何流量，也不通过互联网上的“匿名”主机或服务发送。这意味着我们只能使用和收集存档或存储的信息。因此，由于我们仅限于从第三方获取的结果，这些信息可能已过时或不正确。

* **半被动信息收集**：半被动信息收集的目标是使用看起来像正常互联网流量和行为的方法来剖析目标。我们仅查询已发布的名称服务器以获取信息，不执行深入的反向查找或暴力 DNS 请求，不搜索“未发布”的服务器或目录。我们不运行网络级端口扫描或爬虫，仅查看已发布文档和文件中的元数据，而不主动寻找隐藏内容。关键是不引起对我们活动的注意。事后目标可能能够回溯发现侦察活动，但不应能将活动归因于任何人。

* **主动信息收集**：主动信息收集应能被目标检测到，并被视为可疑或恶意行为。在此阶段，我们主动映射网络基础设施（例如完整的端口扫描 `nmap -p1-65535`），主动枚举和/或漏洞扫描开放服务，主动搜索未发布的目录、文件和服务器。这些活动大多属于标准渗透测试中典型的“侦察”或“扫描”活动。

## 企业级
### 物理
#### 位置（L1）
每个位置的完整地址、所有权、相关记录（城市、税务、法律等）、该位置所有物理安全措施的完整列表（摄像头位置、传感器、围栏、岗哨、入口控制、大门、标识类型、供应商入口、基于 IP 块/地理定位服务的物理位置等）。
对于主机/NOC：主机和网络的完整 CIDR 表示法、所有相关资产的完整 DNS 列表、AS 的完整映射、对等路径、CDN 配置、网段所有者（whois 数据）、电子邮件记录（MX + 邮件地址结构）
* 所有者（L1/L2）
* 土地/税务记录（L1/L2）
* 共享/独立（L1/L2）
* 时区（L1/L2）
* 主机 / NOC

#### 普遍性（L1）
目标组织拥有多个不同的物理位置并不罕见。例如，银行会有中心办公室，但也会有许多远程分行。虽然中心位置的物理和技术安全可能非常好，但远程位置的安全控制往往较差。

#### 关系（L1）
商业伙伴、客户、供应商、通过公司网页公开分享的内容进行分析、租赁公司等。此信息可用于更好地理解业务或组织项目。例如，哪些产品和服务对目标组织至关重要？

此外，这些信息也可用于创建成功的社会工程学场景。

* 关系（L2/L3）
    手动分析以验证来自级别 1 的信息，并深入挖掘可能的关系。
* 共享办公空间（L2/L3）
* 共享基础设施（L2/L3）
* 租赁设备（L2/L3）

### 逻辑层
为合作伙伴、客户和竞争对手积累的信息：对于每一个，完整的业务名称、业务地址、关系类型、基本财务信息、基本主机/网络信息列表。
* 业务合作伙伴（L1/L2/L3）
    目标宣传的业务合作伙伴。有时在主网站上公布。
* 业务客户（L1/L2/L3）
    目标宣传的业务客户。有时在主网站上公布。
* 竞争对手（L1/L2/L3）
    目标的竞争对手是谁。这可能很简单，如福特对雪佛兰，或者需要更多分析。

* Touchgraph（L1）
    touchgraph（人与人之间社交联系的可视化表示）将有助于绘制组织内人员之间可能的互动关系，以及如何从外部接触他们（当 touchgraph 包含外部社区且创建深度级别高于 2 时）。
    基本的 touchgraph 应反映迄今为止收集的信息所衍生的组织结构，图的进一步扩展应基于此（因为这通常能更好地反映组织资产的焦点，并使可能的接近路径清晰）。

* Hoovers 资料（L1/L2）
    什么：一个半开源情报资源（通常需付费订阅）。此类资源专注于收集公司相关的业务信息，并提供业务的“标准化”视图。
    为什么：信息包括物理位置、竞争格局、关键人员、财务信息和其他业务相关数据（取决于来源）。这可用于创建更准确的目标资料，并识别可在测试中使用的其他人员和第三方。
    如何：在网站上简单搜索公司名称即可获得公司的完整资料及其所有可用信息。建议使用多个来源进行交叉参考，以确保获取最新信息（付费服务）。

* 产品线（L2/L3）
    目标提供的产品，如果目标也提供服务，则可能需要进一步分析。

* 市场垂直领域（L1）
    目标所在的行业，例如金融、国防、农业、政府等。

* 营销账户（L2/L3）
    营销活动可以提供大量关于目标营销策略的信息。
    评估目标在所有社交媒体网络上的社交形象。
    评估目标过去的营销活动。

* 会议（L2/L3）
    是否发布了会议纪要？
    会议是否对公众开放？

* 公司重要日期（L1/L2/L3）
    董事会会议
    节假日
    周年纪念日
    产品/服务发布

* 职位空缺（L1/L2）
    通过查看组织的职位空缺列表（通常在其网站的“招聘”部分），可以确定组织内使用的技术类型。例如，如果一个组织有“高级 Solaris 系统管理员”的职位空缺，那么很明显该组织正在使用 Solaris 系统。其他职位可能不像职位名称那样明显，但一个“初级网络管理员”的职位可能会提到“CCNA 优先”或“JNCIA 优先”，这告诉你他们正在使用 Cisco 或 Juniper 技术。

* 慈善附属机构（L1/L2/L3）
    目标组织的执行成员与慈善组织有关联是非常常见的。这些信息可用于针对高管制定可靠的社会工程学场景。

* RFP、RFQ 和其他公开招标信息（L1/L2）
    RFP 和 RFQ 通常揭示大量关于公司使用的系统类型的信息，甚至可能揭示其基础设施的差距或问题。
    找出当前的投标赢家可能会揭示正在使用的系统类型或公司资源可能托管在场外的位置。

* 法庭记录（L2/L3）
    法庭记录通常免费提供，有时也需付费。
    诉讼内容可以揭示过去投诉人的信息，包括但不限于前雇员诉讼。
    现任和前任雇员的犯罪记录可能提供社会工程攻击的目标列表。

* 政治捐款（L2/L3）
    绘制政治捐款或其他经济利益图非常重要，以便识别那些可能不处于明显权力位置但拥有既得利益（或他人对其有既得利益）的关键个人。
    政治捐款的映射将根据信息自由程度在不同国家之间有所不同，但通常可以使用可用数据追溯来自其他国家的捐款。

* 专业执照或注册（L2/L3）
    收集目标专业执照和注册列表不仅可以洞察公司的运营方式，还可以了解他们为维持这些执照而遵循的指导方针和法规。一个典型例子是公司的 ISO 标准认证，可以表明公司遵循既定的指导方针和流程。测试人员了解这些流程以及它们可能如何影响对组织进行的测试非常重要。
    公司通常会在其网站上将这些详细信息作为荣誉徽章列出。在其他情况下，可能需要针对特定垂直领域搜索注册表，以查看组织是否是成员。可用信息高度依赖于垂直市场以及公司的地理位置。还应注意，国际公司可能持有不同的执照，并根据所在国家要求注册不同的标准或法律机构。

### 组织架构图（L1）
* 职位识别
    * 组织中的重要人物
    * 需要特别针对的个人
* 人员变动
    * 组织内部变动的映射（晋升、平调）
* 附属机构
    * 与业务相关的附属组织映射

### 电子
#### 文档元数据（L1/L2）
* 它是什么？元数据或元内容提供有关范围内数据/文档的信息。它可以包含作者/创建者姓名、时间和日期、使用/引用的标准、计算机网络中的位置（打印机/文件夹/目录路径等信息）、地理标签等信息。对于图像，其元数据可以包含颜色、深度、分辨率、相机品牌/型号甚至坐标和位置信息。
* 为什么要做？元数据很重要，因为它包含有关内部网络、用户名、电子邮件地址、打印机位置等信息，并有助于创建位置的蓝图。它还包含有关创建相应文档所用软件的信息。这可以使攻击者能够利用内部网络和用户的知识创建个人资料和/或执行定向攻击。
* 怎么做？有多种工具可以从文件（pdf/word/image）中提取元数据，例如 FOCA（基于 GUI）、metagoofil（基于 Python）、meta-extractor、exiftool（基于 Perl）。这些工具能够以 HTML、XML、GUI、JSON 等不同格式提取和显示结果。这些工具的输入大多是从“客户”的公开存在中下载的文档，然后对其进行分析以了解更多信息。而 FOCA 则通过其 GUI 界面帮助你搜索文档、下载和分析。

#### 营销传播（L1/L2）
* 过去的营销活动提供有关可能已退役但仍可访问的项目的信息。
* 当前的营销传播包含设计组件（颜色、字体、图形等），这些组件大部分也在内部使用。
* 额外的联系信息，包括外部营销组织。

### 基础设施资产
#### 拥有的网络块（L1）
* 组织拥有的网络块可以通过执行 whois 搜索被动获取。DNSStuff.com 是获取此类信息的一站式站点。
* 对 IP 地址的开源搜索可能产生有关目标基础设施类型的信息。管理员经常在各种支持网站上以求助帖的形式发布 IP 地址信息。

#### 电子邮件地址（L1）
* 电子邮件地址提供了潜在的有效用户名列表和域结构。
* 电子邮件地址可以从多个来源收集，包括组织的网站。

#### 外部基础设施概况（L1）
* 目标的外部基础设施概况可以提供关于内部使用技术的海量信息。
* 这些信息可以通过被动和主动方式从多个来源收集。
* 该概况应用于制定针对外部基础设施的攻击场景。

#### 使用的技术（L1/L2）
* 通过支持论坛、邮件列表和其他资源进行 OSINT 搜索，可以收集目标使用的技术信息。
* 对已识别的信息技术组织使用社会工程学。
* 对产品供应商使用社会工程学。

#### 采购协议（L1/L2/L3）
* 采购协议包含有关目标处硬件、软件、许可证和其他有形资产的信息。

#### 远程访问（L1/L2）
* 获取有关员工和/或客户如何连接目标进行远程访问的信息，提供了潜在的入口点。
* 远程访问门户的链接通常可以在目标的主页上找到。
* “操作指南”文档揭示了远程用户连接所需的应用程序/流程。

#### 应用程序使用情况（L1/L2）
收集目标组织使用的已知应用程序列表。这通常可以通过从可公开访问的文件中提取元数据来实现（如前所述）。

#### 防御技术（L1/L2/L3）
根据所使用的防御措施，有多种方法可以对防御技术进行指纹识别。

##### 被动指纹识别
* 搜索论坛和可公开访问的信息，其中目标组织的技术人员可能正在讨论问题或寻求有关所使用技术的帮助。
* 搜索目标组织以及流行技术供应商的营销信息。
* 使用 Tin-eye（或其他图像匹配工具）搜索目标组织的标志，查看它是否出现在供应商的参考页面或营销材料上。

##### 主动指纹识别
* 向面向公众的系统发送适当的数据包，以测试封堵模式。有多种工具可用于识别特定 WAF 类型。
* 目标网站响应中的标头信息以及电子邮件中的标头信息，通常会显示不仅有关正在使用的系统的信息，还有已启用的特定保护机制（例如电子邮件网关防病毒扫描器）。

#### 人员能力（L1/L2/L3）
发现目标组织的防御人员能力可能很困难。有几条关键信息可能有助于判断目标组织的安全性。

* 检查是否存在全公司范围的 CERT/CSIRT/PSRT 团队。
* 检查招聘广告，查看安全职位的出现频率。
* 检查招聘广告，查看安全是否被列为非安全职位（例如开发人员）的要求。
* 检查外包协议，查看目标的安全是否部分或全部外包。
* 检查为公司工作且可能活跃于安全社区的特定个人。

* 如何获取：这些信息可在 SEC 的 EDGAR 网站（http://www.sec.gov/edgar.shtml）上获得。特别感兴趣的报告包括 10-K（年度报告）和 10-Q（季度报告）。

## HUMINT

# 足迹分析
它是什么：外部信息收集，也称为足迹分析，是信息收集的一个阶段，包括与目标交互，以从组织外部的视角获取信息。

为什么：通过与目标交互可以收集大量信息。通过探测服务或设备，你通常可以创建能够对其进行指纹识别的场景，或者更简单地，可以获取能识别设备的横幅。这一步对于收集有关目标的更多信息是必要的。在此部分之后，你的目标是获得一个优先级排序的目标列表。

## 外部足迹分析

### 识别客户外部范围
渗透测试期间情报收集的主要目标之一是确定范围内的主机。有多种技术可用于识别系统，包括使用反向 DNS 查找、DNS 暴力破解、对域和范围的 WHOIS 搜索。这些技术及其他技术如下所述。

### 被动侦察
#### WHOIS 查询
对于外部足迹分析，我们首先需要确定哪个 WHOIS 服务器包含我们所需的信息。鉴于我们应该知道目标域的 TLD，我们只需找到目标域注册所在的注册商即可。

WHOIS 信息基于树状层次结构。ICANN（IANA）是所有 TLD 的权威注册机构，是所有手动 WHOIS 查询的绝佳起点。

* ICANN - http://www.icann.org
* IANA - http://www.iana.com
* NRO - http://www.nro.net
* AFRINIC - http://www.afrinic.net
* APNIC - http://www.apnic.net
* ARIN - http://ws.arin.net
* LACNIC - http://www.lacnic.net
* RIPE - http://www.ripe.net

一旦查询了适当的注册商，我们就可以获得注册人信息。有许多网站提供 WHOIS 信息；但是为了文档的准确性，你应仅使用适当的注册商。

* InterNIC - http://www.internic.net/

通常，对 ARIN 进行简单的 whois 查询会将你引导至正确的注册商。

#### BGP 路由透视
对于参与边界网关协议（BGP）的网络，可以识别其自治系统号（ASN）。由于 BGP 路由路径在全球范围内公布，我们可以使用 BGP4 和 BGP6 路由透视来找到它们。

* BGP4 - http://www.bgp4.as/looking-glasses
* BGP6 - http://lg.he.net/

### 主动足迹分析
#### 端口扫描
端口扫描技术将根据测试可用时间和隐蔽性需求而变化。如果对系统一无所知，可以使用快速 ping 扫描来识别系统。此外，应运行不进行 ping 验证的快速扫描（nmap 中的 `-PN`）以检测最常见的开放端口。完成后，可以运行更全面的扫描。一些测试人员只检查开放的 TCP 端口，请确保也检查 UDP。http://nmap.org/nmap_doc.html 文档详细介绍了端口扫描类型。Nmap（“Network Mapper”）是网络审计/扫描的事实标准。Nmap 可在 Linux 和 Windows 上运行。

有关使用 Nmap 实现此目的的更多信息，请参阅 [PTES 技术指南]（http://www.pentest-standard.org/index.php/PTES_Technical_Guidelines#Nmap_.28Windows.2FLinux.29）

Nmap 有数十个可用选项。由于本节涉及端口扫描，我们将重点关注执行此任务所需的命令。需要注意的是，所使用的命令主要取决于时间和要扫描的主机数量。主机越多或完成任务的时间越少，我们对主机的探查就越少。随着我们继续讨论这些选项，这一点将变得明显。

还应测试 IPv6。

#### 横幅抓取
横幅抓取是一种枚举技术，用于收集关于网络上计算机系统及其开放端口上运行的服务的信息。横幅抓取用于识别目标主机上运行的应用程序和操作系统的版本。

横幅抓取通常在超文本传输协议（HTTP）、文件传输协议（FTP）和简单邮件传输协议（SMTP）上执行；即端口 80、21 和 25。常用于执行横幅抓取的工具是 Telnet、nmap 和 Netcat。

#### SNMP 扫描
SNMP 扫描也会被执行，因为它们能提供关于特定系统的大量信息。SNMP 协议是一种无状态、面向数据报的协议。不幸的是，SNMP 服务器不会响应无效团体字符串的请求，并且底层的 UDP 协议不能可靠地报告关闭的 UDP 端口。这意味着来自被探测 IP 地址的“无响应”可能表示以下任何一种情况：

* 机器不可达
* SNMP 服务器未运行
* 无效的团体字符串
* 响应数据报尚未到达

#### 区域传输
DNS 区域传输，也称为 AXFR，是一种 DNS 事务类型。它是一种旨在跨一组 DNS 服务器复制包含 DNS 数据的数据库的机制。区域传输有两种类型：全量（AXFR）和增量（IXFR）。有许多工具可用于测试执行 DNS 区域传输的能力。常用于执行区域传输的工具是 host、dig 和 nmap。

#### SMTP 反弹
SMTP 反弹，也称为未送达报告/回执（NDR）、（失败的）送达状态通知（DSN）消息、未送达通知（NDN）或简称为“bounce”，是邮件系统向另一条消息的发送者发送的自动电子邮件消息，告知其存在投递问题。这可用于帮助攻击者对 SMTP 服务器进行指纹识别，因为 SMTP 服务器信息（包括软件和版本）可能包含在反弹消息中。

这可以通过简单地在目标域内创建一个虚假地址来实现。例如，`asDFADSF_garbage_address@target.com` 可用于测试 target.com。Gmail 提供对邮件头的完全访问权限，使其成为测试人员的简单选择。

#### DNS 发现
可以通过查看域的权威名称服务器的 WHOIS 记录来执行 DNS 发现。此外，应检查主域名的变体，并检查网站是否有对可能由目标控制的其他域的引用。

#### 正向/反向 DNS
反向 DNS 可用于获取组织内使用的有效服务器名称。需要注意的是，必须有 PTR（反向）DNS 记录才能从提供的 IP 地址解析名称。如果解析成功，则返回结果。这通常通过使用各种 IP 地址测试服务器以查看是否返回任何结果来执行。

#### DNS 暴力破解
在识别出与客户域相关的所有信息后，现在开始查询 DNS。由于 DNS 用于将 IP 地址映射到主机名，反之亦然，我们想查看其配置是否不安全。我们将寻求使用 DNS 来揭示有关客户的额外信息。涉及 DNS 的最严重错误配置之一是允许互联网用户执行 DNS 区域传输。有几种工具可用于枚举 DNS，不仅可以检查执行区域传输的能力，还可以潜在地发现不常见的其他主机名。

#### Web 应用程序发现
在渗透测试期间，识别脆弱的 Web 应用程序可能是一项特别富有成效的活动。需要关注的内容包括配置不当的现成（OTS）应用程序、具有插件功能的 OTS 应用程序（插件通常包含比基础应用程序更易受攻击的代码）以及自定义应用程序。Web 应用程序指纹识别器（如 WAFP）可以在此处发挥巨大作用。

#### 虚拟主机检测与枚举
Web 服务器通常托管多个“虚拟”主机，以将功能整合到单个服务器上。如果多个服务器指向相同的 DNS 地址，它们可能托管在同一台服务器上。可以使用诸如 MSN 搜索之类的工具将 IP 地址映射到一组虚拟主机。

### 建立外部目标列表
完成上述活动后，应编制用户、电子邮件、域、应用程序、主机和服务的列表。

#### 版本映射
版本检查是快速识别应用程序信息的一种方法。在某种程度上，可以使用 nmap 对服务版本进行指纹识别，而 Web 应用程序的版本通常可以通过查看任意页面的源代码来收集。

#### 识别补丁级别
为了识别内部服务的补丁级别，可以考虑使用将询问系统以找出版本之间差异的软件。如果客户同意，可以在渗透测试的这个阶段使用凭据。漏洞扫描器在无需凭据的情况下远程识别补丁级别方面特别有效。

#### 寻找脆弱的 Web 应用程序
在渗透测试期间，识别脆弱的 Web 应用程序可能是一项特别富有成效的活动。需要关注的内容包括配置不当的现成（OTS）应用程序、具有插件功能的 OTS 应用程序（插件通常包含比基础应用程序更易受攻击的代码）以及自定义应用程序。Web 应用程序指纹识别器（如 WAFP）可以在此处发挥巨大作用。

#### 识别锁定阈值
识别身份验证服务的锁定阈值将使你能够确保暴力破解攻击不会在测试期间故意锁定有效用户。识别环境中的所有不同身份验证服务，并测试一个单一的无害账户的锁定情况。通常，对有效账户进行 5-10 次尝试足以确定该服务是否会锁定用户。

## 内部足迹分析

### 被动侦察
如果测试人员能够访问内部网络，数据包嗅探可以提供大量信息。使用诸如 p0f 中实现的技术来识别系统。

### 识别客户内部范围
在执行内部测试时，首先枚举本地子网，然后通常可以通过稍微修改地址来推断其他子网。此外，查看内部主机的路由表可能特别能说明问题。以下是可以使用的多种技术。

DHCP 服务器不仅可能是本地信息的潜在来源，也可能是远程 IP 范围和重要主机详细信息的来源。大多数 DHCP 服务器将提供本地 IP 网关地址以及 DNS 和 WINS 服务器的地址。在基于 Windows 的网络中，DNS 服务器往往是 Active Directory 域控制器，因此是感兴趣的目标。

### 主动侦察
内部主动侦察应包含外部主动侦察的所有元素，此外还应侧重于内网功能，例如：
* 目录服务（Active Directory、Novell、Sun 等）
* 提供业务功能的内网站点
* 企业应用程序（ERP、CRM、会计等）
* 识别敏感网段（财务、研发、市场等）
* 到生产网络（数据中心）的访问映射
* VoIP 基础设施
* 身份验证配置（kerberos、cookie 令牌等）
* 代理和互联网访问管理
