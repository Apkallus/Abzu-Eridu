# 情报收集

情报收集是收集数据或“情报”以帮助指导评估行动的阶段。广义上，这种情报收集包括关于员工、设施、产品和计划的信息。在更大的图景中，这些情报可能包含竞争对手潜在的秘密或私密“情报”，或其他与目标相关的信息。

## 开源情报 (OSINT)

简单来说，开源情报（OSINT）是定位并分析公开可用信息源的过程。其关键点在于，该情报收集过程旨在产生对攻击者或竞争对手有价值的、当前且相关的信息。在大多数情况下，OSINT 不仅仅是使用各种来源执行网络搜索。

### 公司信息

### 物理位置

OSINT 的第一步通常是确定目标公司的物理位置。这些信息对于公开已知或公布的地点可能很容易获得，但对于更隐秘的场所则不那么容易。公共站点通常可以通过使用如下搜索引擎来定位：

- Google - [http://www.google.com](http://www.google.com)
- Yahoo - [http://yahoo.com](http://yahoo.com)
- Bing - [http://www.bing.com](http://www.bing.com)
- Ask.com - [http://ask.com](http://ask.com)

#### 位置

#### 共用/独立建筑

在识别物理位置时，重要的是要留意该位置是独立建筑还是大型设施中的一间办公室。尝试识别相邻企业以及公共区域也很重要。

#### 所有权人

一旦确定了物理位置，就有必要确定实际房产的所有者。所有者可以是个人、团体或公司。如果目标公司不拥有该房产，那么他们在实际改善或升级该物理位置方面可能会受到限制。

##### 土地/税务记录

税务记录：  
http://www.naco.org/Counties/Pages/CitySearch.aspx

土地和税务记录通常包含目标的丰富信息，如所有权、占有情况、抵押贷款公司、止赎通知、照片等。记录的信息和透明度水平因司法管辖区而异。美国的土地和税务记录通常在县级管理。

首先，如果您知道目标所在的城市或邮政编码，请使用 http://publicrecords.netronline.com/ 等网站确定它属于哪个县。然后切换到 Google，您可以使用诸如“XX 县税务记录”、“XX 县档案办公室”或“XX 县评估员”等查询，如果存在可搜索的在线数据库，这些查询会引导您找到它。如果不存在，您仍然可以致电县档案办公室，如果您知道自己要找什么，可以要求他们传真特定的记录。

**建筑部门：**

对于某些评估，可能有必要更进一步，向当地建筑部门查询更多信息。根据城市的不同，目标场所可能属于县或市的管辖范围。通常可以通过致电任一实体来确定。

建筑部门通常存档有楼层平面图、旧许可证和当前许可证、租户改进信息以及其他类似信息。这些信息中可能隐藏着建筑公司、工程师、建筑师等的名字。所有这些都可以与 SET 等工具结合使用。在大多数情况下，获取这些信息需要打电话，但大多数建筑部门乐于向任何询问者提供。

以下是一个获取楼层平面图的借口示例：您可以打电话说您是一名建筑顾问，受雇设计该建筑的翻新或扩建工程，如果能拿到原始平面图，将有助于过程更顺利进行。

#### 数据中心位置

通过公司网站、公开备案文件、土地记录或搜索引擎识别目标企业的任何数据中心位置，可以提供额外的潜在目标。

##### 时区

识别目标运营所在的时区，可以提供有关其运营时间的重要信息。了解目标时区与评估团队时区之间的关系也很重要。在进行任何测试时，时区地图通常是有用的参考。

##### 异地聚会

通过公司网站或搜索引擎识别目标近期或未来的异地聚会或派对，可以深入了解目标的企业文化。企业不仅为员工，还为商业伙伴和客户举办异地聚会是常见做法。收集这些数据可以洞察攻击者可能感兴趣的事项。

##### 产品/服务

通过公司网站、新闻稿或搜索引擎识别目标企业的产品以及与产品发布相关的任何重要数据，可以深入了解目标的内部运作。企业通常公开发布此类通知，以吸引宣传并告知现有/新客户。公开信息包括但不限于外语文档、广播和电视广播、互联网站点以及公开演讲。

##### 公司重要日期

重要的公司日期可以洞察员工可能比平时保持更高警惕的日子。这可能是由于潜在的公司会议、董事会会议、投资者会议或公司周年纪念。通常，庆祝各种节假日的企业员工会大幅减少，因此在这些时期进行针对性攻击可能会更加困难。

##### 职位识别

在每个目标中，识别并记录组织内的最高职位至关重要。这是为了确保最终报告针对正确的受众。作为任何渗透测试项目的一部分，至少应识别关键员工。

##### 组织结构图

了解组织结构很重要，不仅是为了了解结构的深度，也是为了了解其广度。如果组织极其庞大，新员工或人员有可能不被察觉。在较小的组织中，这种可能性较小。清晰了解组织结构还可以洞察职能团队。这些信息有助于确定内部目标。

##### 公司通讯

通过公司网站或职位搜索引擎识别公司通讯，可以深入了解目标的内部运作。

###### 市场营销

市场营销通讯通常用于发布关于当前或未来产品发布以及合作伙伴关系的公司公告。

###### 诉讼

关于目标参与诉讼的通讯，可以洞察潜在的威胁行为者或感兴趣的数据。

###### 交易

涉及公司交易的通讯可能是对市场公告或诉讼的间接回应。

##### 职位空缺

通过公司网站或职位搜索引擎搜索当前的职位空缺或招聘信息，可以深入了解目标的内部运作。企业通常会在招聘信息中包含有关当前或未来技术实施的信息。收集这些数据可以洞察攻击者可能感兴趣的事项。存在多个职位搜索引擎，可以针对目标进行查询。

### 关系

识别目标的逻辑关系对于深入了解企业的运营方式至关重要。应利用公开信息来确定目标与供应商、业务伙伴、律师事务所等的关系。这些信息通常可以通过新闻稿、公司网站（目标及供应商的网站）以及潜在的行业相关论坛获得。

#### 慈善附属机构

通过公司网站或搜索引擎识别任何目标企业的慈善附属机构，可以深入了解目标的内部运作和潜在的企业文化。企业向各种组织进行慈善捐赠是常见做法。收集这些数据可以洞察攻击者可能感兴趣的事项。

#### 网络提供商

通过已分配的网段/地址信息、公司网站或搜索引擎识别任何网络供应商或提供商，可以深入了解目标的潜在情况。

#### 业务伙伴

识别业务伙伴对于洞察不仅是目标的企业文化，还有潜在使用的技术至关重要。企业宣布合作协议是常见做法。收集这些数据可以洞察攻击者可能感兴趣的事项。

#### 竞争对手

识别竞争对手可以提供一个了解潜在对手的窗口。竞争对手宣布可能影响目标的消息并不罕见。这些消息可能包括新员工、产品发布甚至合作协议。收集这些数据对于全面理解任何潜在的企业敌意非常重要。

**语气和频率**

识别员工发帖的语气和频率可能是识别心怀不满的员工以及公司对社交网络接受度的关键指标。虽然耗时，但可以确定员工的工作日程和假期时段。

**位置感知**

大多数社交网站都提供在帖子中包含地理定位信息的功能。这些信息可用于确定发帖时该人身处的确切物理位置。此外，上传到社交网站的图片中也可能包含地理定位信息。用户可能有足够知识关闭此功能，但有时只需阅读一条明确说明其所在位置的帖子即可。

#### Cree.py

#### Maltego

Paterva 的 [Maltego](http://www.paterva.com/) 用于自动化信息收集任务。Maltego 是一个开源情报和取证应用程序。本质上，Maltego 是一个数据挖掘和信息收集工具，它将收集到的信息映射成易于理解和操作的格式。它通过自动化诸如电子邮件 harvesting 和子域映射等任务来节省时间。Maltego 的文档相对较少，因此我们在此提供获取所需数据的必要步骤。

启动 Maltego 后，应可见主界面。界面的六个主要区域是：工具栏、面板（Palette）、图谱（视图）区域、概览区域、详细信息区域和属性区域。

首先，找到 Maltego 最左上角，点击“新建图谱”按钮。然后，将“domain”项从面板拖放到图谱上。图谱区域允许您处理转换，并以挖掘视图、动态视图、边权重视图以及实体列表等不同方式查看数据。当您首次将域名图标添加到图谱时，它默认为“paterva.com”，请双击该图标并将其更改为您目标的域名（不含 www 等子域名）。现在您就可以开始挖掘了。

1. 右键单击（或双击）域名图标，从“运行转换”中选择“To Website DNS [using search engine]”。这有望显示出目标的所有子域名。

   > [截图示意图 – 原文件引用图片]

3. 此时，根据目标的大小，您可以选择不同的路径，但合理的下一步是确定网段，因此运行“To Netblock [Using natural boundaries]”转换。

##### 保安

观察保安（或安全官员）通常是评估最明显的威慑措施的第一步。保安身着制服，通过保持高可见性存在来保护财产，以阻止非法和不当行为。通过直接观察保安的移动，可以确定正在使用的程序或建立移动模式。你需要观察保安正在保护什么。可以使用双筒望远镜从安全距离观察任何移动。

一些保安经过培训和许可，可以携带枪支以保护自身安全以及他们受托保护的人员。如果注意到保安携带枪支，这不应感到惊讶。这应在开始渗透测试项目之前记录。如果观察到枪支，请确保除非特别授权并经过培训，否则不要采取任何进一步行动。

##### 徽章使用

徽章使用是指一种使用身份徽章作为访问控制形式的物理安全方法。徽章系统可能连接到物理访问控制系统，或仅用作视觉验证机制。观察个人徽章使用情况很重要。通过观察徽章使用情况，甚至可能复制正在使用的特定徽章。应注意的具体事项包括：徽章是否需要可见或需要出示才能获得进入财产或设施的物理访问权限。应记录徽章使用情况，如果可能，包括观察到的验证程序。

##### 锁具

锁具是一种机械或电子机制，通常用于防止未经授权的进入或离开。这些锁具可以是简单的门锁、插销锁，也可以是复杂的密码锁。通过观察门上锁具的类型和位置，可以确定该门主要用于进入还是离开。你需要观察锁具正在保护什么。所有观察结果应提前记录，如果可能，拍摄照片。

##### 入侵检测系统/警报

观察保安（或安全官员）通常是评估最明显的威慑措施的第一步。保安身着制服…………如果观察到枪支，请确保除非特别授权并经过培训，否则不要采取任何进一步行动。

##### 安保照明

安保照明通常用作物理财产的预防和纠正措施。安保照明可能有助于检测入侵者、威慑入侵者，或者在某些情况下仅仅是增加安全感。安保照明通常是设施环境设计的一个组成部分。安保照明包括泛光灯和低压钠灯。大多数旨在整夜开启的安保照明属于高强度放电灯类型。其他灯具可能由传感器（如被动红外传感器 PIR）激活，仅在有人（或其他哺乳动物）接近时开启。PIR 激活的灯具通常使用白炽灯泡，以便它们可以瞬间启动；节能不太重要，因为它们不会一直亮着。PIR 传感器激活可以增加威慑效果（因为入侵者知道他被发现了）和检测效果（因为人们会被突然增加的光线吸引）。一些 PIR 单元可以设置为发出提示音以及开灯。大多数现代单元有一个光电池，以便它们仅在黑暗时开启。

虽然围绕物理结构部署足够的照明是为了降低入侵风险，但照明的正确实施至关重要，因为布置不当的照明实际上会阻碍对其设计保护的设施的观察。

安保照明可能遭受故意破坏，可能是为了降低其后续入侵尝试的有效性。因此，安保灯具应要么安装得非常高，要么用金属网或坚固的聚碳酸酯防护罩保护。其他灯具可以完全隐藏起来，通过光管或从抛光铝或不锈钢镜面反射光线。出于类似的原因，高安全设施可能为其安保照明提供备用电源。观察并记录正在使用的安保照明的类型、数量和位置。

##### 监控/CCTV 系统

监控/CCTV 系统可用于从中央区域观察设施内外的活动。监控/CCTV 系统可以连续运行，也可以仅在需要监控特定事件时激活。更高级的监控/CCTV 系统使用运动检测设备来激活系统。基于 IP 的监控/CCTV 摄像头可能用于更分散的操作。

监控/CCTV 摄像头可以是显眼的，用作可见威慑，也可以是不显眼的。监控/CCTV 摄像头通常是小巧的高清彩色摄像头，不仅可以聚焦以分辨微小细节，而且通过将摄像头的控制连接到计算机，可以半自动跟踪物体。观察和记录监控/CCTV 系统对于识别覆盖区域至关重要。虽然可能无法确定正在使用的特定摄像头类型甚至覆盖区域，但可以识别出有限覆盖或无覆盖的区域。应注意监控/CCTV 系统是否受到物理保护。如果没有，则需要记录监控/CCTV 摄像头是否容易被故意破坏。此外，物理上不受保护的摄像头可能会因喷洒物质或遮挡镜头而导致图像模糊或阻塞。激光可用于致盲或损坏监控/CCTV 摄像头。对于无线监控/CCTV 系统，以与无线设备相同频率广播信号可能使其受到干扰。

##### 门禁设备

门禁设备允许对给定设施中的区域和/或资源进行访问控制。门禁是指限制未经授权人员进入财产、建筑物或房间的做法。门禁可以通过人员（保安或接待员）、机械手段（如锁和钥匙）或技术手段（如门禁系统，如访问控制前厅）实现。

门禁设备历史上是通过钥匙和锁完成的。电子门禁的使用正在广泛取代机械钥匙。门禁读卡器通常分为基本型、半智能型和智能型。基本型门禁读卡器仅读取卡号或 PIN 并将其转发到控制面板。最流行的门禁读卡器类型包括 RFLOGICS 的 RF Tiny、HID 的 ProxPoint 和 Farpointe Data 的 P300。半智能型读卡器具有控制门硬件（锁、门磁、出门按钮）所需的输入和输出，但不做出任何访问决策。常见的半智能型读卡器包括 CEM Systems 的 InfoProx Lite IPL200 和 Apollo 的 AP-510。智能型读卡器拥有控制门硬件所需的所有输入和输出，同时具有独立做出访问决策所需的内存和处理能力。常见的智能型读卡器包括 CEM Systems 的 InfoProx IPO200、Apollo 的 AP-500、Isonas Security Systems 的 PowerNet IP Reader、Solus 的 ID08（具有内置 Web 服务以使其用户友好）、HID Global 的 Edge ER40 读卡器、ASPiSYS Ltd 的 LogLock 和 UNiLOCK，以及 Suprema Inc. 的 BioEntry Plus 读卡器。

一些读卡器可能具有附加功能，如用于数据收集目的（例如，用于出勤报告的打卡/下班事件）的 LCD 和功能按钮、用于对讲的摄像头/扬声器/麦克风，以及智能卡读/写支持。观察并记录正在使用的门禁设备的类型、数量和位置。

##### 环境设计

环境设计涉及建筑物或设施周围的环境。在物理安全领域，环境设计包括设施的地理、景观、建筑和外部设计。

观察设施及周边区域可以突显潜在关注区域，例如由于地理和景观造成的潜在隐蔽区域。建筑和外部设计会影响保安保护财产的能力，造成低可见性或无可见性区域。此外，围栏、存储容器、保安岗亭、路障和维护区域的布置也可能影响以隐蔽方式在设施周围移动的能力。

#### 员工行为

观察员工通常是最容易执行的步骤之一。员工的行为通常能洞察任何企业行为或可接受的规范。通过观察员工，可以确定正在使用的程序或建立进出流量模式。可以使用双筒望远镜从安全距离观察任何移动。

#### 垃圾搜寻

传统上，大多数目标将垃圾丢弃在垃圾桶或垃圾箱中。这些垃圾可能根据材料的可回收性进行分类。垃圾搜寻是翻找商业或住宅垃圾以找到被所有者丢弃但可能有用的物品的行为。这通常是一个非常脏的过程，但可能产生显著的结果。垃圾箱通常位于私人场所内，因此评估团队可能面临擅入非目标所有财产的风险。尽管法律的执行力度各不相同，但请确保此行为已获得渗透测试项目的授权。垃圾搜寻本身在未被法律明确禁止时通常是合法的。与其从该区域拿走垃圾，通常接受的做法是拍摄获得的材料，然后将其放回原垃圾箱。

##### Kismet-Newcore

Kismet-newcore 是一个用于 802.11 无线局域网的网络探测器、数据包嗅探器和入侵检测系统。Kismet 可以与任何支持原始监控模式的无线网卡配合使用，并且可以嗅探 802.11a、802.11b、802.11g 和 802.11n 流量。

Kismet 通过被动收集数据包、检测标准命名的网络、检测（并给定时间后揭露）隐藏网络，以及通过数据流量推断非信标网络的存在来识别网络。

- **Drones（无人机）：** 捕获无线流量并将其报告给服务器；它们必须手动启动。

- **Client（客户端）：** 图形界面部分，将连接到服务器。

#### 区域传输

DNS 区域传输，也称为 AXFR，是一种 DNS 事务类型。它是一种旨在跨一组 DNS 服务器复制包含 DNS 数据的数据库的机制。区域传输有两种类型：全量（AXFR）和增量（IXFR）。有许多工具可用于测试执行 DNS 区域传输的能力。常用工具包括 `host`、`dig` 和 `nmap`。

##### Dig


#### 反向 DNS

反向 DNS 可用于获取组织内使用的有效服务器名称。但有一个前提：必须存在 PTR（反向）DNS 记录，才能从提供的 IP 地址解析出名称。如果可以解析，则返回结果。这通常通过测试服务器以各种 IP 地址查看是否返回任何结果来执行。

#### SNMP 扫描

进行 SNMP 扫描也是因为 SNMP 可以提供有关特定系统的大量信息。SNMP 协议是一种无状态、面向数据报的协议。遗憾的是，SNMP 服务器不会对带有无效 community 字符串的请求做出响应，并且底层的 UDP 协议不能可靠地报告关闭的 UDP 端口。这意味着，来自被探测 IP 地址的“无响应”可能意味着以下任何一种情况：

- 机器不可达
- SNMP 服务器未运行
- 无效的 community 字符串
- 响应数据报尚未到达

#### SMTP 退回

SMTP 退回，也称为非送达报告/回执（NDR）、（失败的）送达状态通知（DSN）消息、非送达通知（NDN）或简称为退回，是邮件系统向另一条消息的发送者发送的关于投递问题的自动电子邮件消息。这可用于帮助攻击者指纹识别 SMTP 服务器，因为退回消息中可能包含 SMTP 服务器信息，包括软件和版本。

#### 旗帜抓取

旗帜抓取是一种枚举技术，用于收集有关网络上计算机系统及其开放端口上运行的服务的详细信息。旗帜抓取用于识别目标主机运行的应用程序和操作系统的版本。

旗帜抓取通常在超文本传输协议（HTTP）、文件传输协议（FTP）和简单邮件传输协议（SMTP）上执行，分别对应端口 80、21 和 25。常用于旗帜抓取的工具包括 Telnet、nmap 和 Netcat。

#### SNMP 扫描

进行 SNMP 扫描也是因为 SNMP 可以提供有关特定系统的大量信息。SNMP 协议是一种无状态、面向数据报的协议。遗憾的是，SNMP 服务器不会对带有无效 community 字符串的请求做出响应，并且底层的 UDP 协议不能可靠地报告关闭的 UDP 端口。这意味着，来自被探测 IP 地址的“无响应”可能意味着以下任何一种情况：

- 机器不可达
- SNMP 服务器未运行
- 无效的 community 字符串
- 响应数据报尚未到达

#### 区域传输

DNS 区域传输，也称为 AXFR，是一种 DNS 事务类型。它是一种旨在跨一组 DNS 服务器复制包含 DNS 数据的数据库的机制。区域传输有两种类型：全量（AXFR）和增量（IXFR）。有许多工具可用于测试执行 DNS 区域传输的能力。常用工具包括 `host`、`dig` 和 `nmap`。

##### Host

```bash
host <domain> <DNS server>
```

##### Dig

```bash
dig @server domain axfr
```

#### SMTP 退回

SMTP 退回，也称为非送达报告/回执（NDR）、（失败的）送达状态通知（DSN）消息、非送达通知（NDN）或简称为退回，是邮件系统向另一条消息的发送者发送的关于投递问题的自动电子邮件消息。这可用于帮助攻击者指纹识别 SMTP 服务器，因为退回消息中可能包含 SMTP 服务器信息，包括软件和版本。

#### 反向 DNS

反向 DNS 可用于获取组织内使用的有效服务器名称。但有一个前提：必须存在 PTR（反向）DNS 记录，才能从提供的 IP 地址解析出名称。如果可以解析，则返回结果。这通常通过测试服务器以各种 IP 地址查看是否返回任何结果来执行。

#### 旗帜抓取

旗帜抓取是一种枚举技术，用于收集有关网络上计算机系统及其开放端口上运行的服务的详细信息。旗帜抓取用于识别目标主机运行的应用程序和操作系统的版本。

旗帜抓取通常在超文本传输协议（HTTP）、文件传输协议（FTP）和简单邮件传输协议（SMTP）上执行，分别对应端口 80、21 和 25。常用于旗帜抓取的工具包括 Telnet、nmap、netcat 和 netca6（IPv6）。

#### 被动侦察

##### 数据包嗅探

执行数据包嗅探可以收集来自正在分析的数据包流中系统的 IP 地址和 MAC 地址。在大多数情况下，数据包嗅探难以检测，因此这种侦察形式本质上是被动的且非常隐蔽。通过收集和分析大量数据包，可以对给定设备上运行的操作系统和服务进行指纹识别。还可能从数据包流中获取登录信息、密码哈希和其他凭据。Telnet 和旧版本的 SNMP 以明文形式传递凭据，容易通过嗅探被攻破。数据包嗅探还有助于确定哪些服务器充当关键基础设施，从而成为攻击者感兴趣的目标。
