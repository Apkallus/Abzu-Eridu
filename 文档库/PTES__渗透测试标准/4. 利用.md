# 目的

渗透测试的利用阶段完全侧重于通过绕过安全限制来建立对系统或资源的访问权限。如果上一阶段（漏洞分析）执行得当，本阶段应经过周密规划，是一次精确打击。主要重点是确定进入组织的主要入口点以及识别高价值目标资产。

如果漏洞分析阶段顺利完成，应已整理出一份高价值目标列表。最终的攻击向量应考虑成功概率以及对组织的最大影响。

# 对策

对策定义为阻碍成功完成利用途径的预防性技术或控制措施。这些技术可以是基于主机的入侵防御系统、安全网关、Web 应用防火墙或其他预防性方法。在执行利用时，应考虑几个因素。如果存在预防性技术，应考虑规避技术。在无法规避的情况下，应考虑替代的利用方法。

总体而言，攻击组织时要保持隐蔽，如果触发警报，评估的级别可能会降低。如果可能，应在触发利用之前枚举对策。这可以通过进行攻击演练或枚举技术来实现。

## 反病毒

反病毒是一种旨在防止恶意软件部署到系统上的技术。作为渗透测试人员，我们应能识别这些类型的反病毒技术并能够对其进行防护。反病毒只是所有不同预防措施中的一小部分，例如基于主机的入侵防御系统、Web 应用防火墙和其他预防性技术。

### 编码

编码是一种混淆数据的方法，使得部署的代码片段看起来不一样。通过编码，混淆通常是通过打乱信息和重新排列来实现的，以隐藏应用程序实际执行的操作。

### 加壳

加壳在某种意义上类似于编码，它试图重新排列数据以压缩应用程序或“打包”它。希望这样做的目的是，被传递的可执行文件或代码片段以某种方式混淆，从而不会被反病毒技术检测到。

### 加密

加密，像编码和加壳一样，是另一种操纵预期可运行代码的方法，使其无法被识别或检查。只有在内存中解密后（使用类似于加壳的方法），实际代码才首次暴露出来——希望是在安全机制允许其通过之后，并在解密后立即执行。

### 白名单绕过

白名单技术利用基于信任的模型，针对给定系统在某个时间点出现过的应用程序。该技术对系统进行基线化，识别系统上正常运行的内容与外来内容。渗透测试人员应能够绕过白名单技术。最常见的方法之一是通过直接内存访问。白名单没有能力实时监控内存，如果一个内存驻留程序正在运行且不触及磁盘，它可以不被该技术检测到。

### 进程注入

进程注入就是注入到已经运行的进程中的方法。通过注入到进程中，应用程序的信息可以隐藏在一个通常会被信任的进程中。预防性技术很难检查正在运行的进程，攻击者几乎总是可以隐藏在一个应用程序认为可信的不同进程中。

### 纯内存驻留

内存驻留攻击通常是最受欢迎的，因为大多数技术不检查内存。作为攻击者，找到一种纯粹驻留在内存中的方式是最理想的。当写入磁盘时，大多数应用程序会进行扫描、基线化和其他潜在恶意软件的识别。写入磁盘时被检测到的可能性显著增加。

## 人为因素

在执行利用时，通过直接利用或应用程序漏洞并不总是最佳途径。有时人为因素可能是攻击组织的更好方式。理解正确的攻击途径并确保我们采用的方法是最佳途径至关重要。

## 数据执行保护 (DEP)

在执行利用时，许多预防措施会发挥作用。数据执行保护是大多数操作系统中实现的一种防御措施，当内存中发生覆盖时，阻止执行权限。DEP 背后的思路是阻止攻击者重写内存然后执行该代码。有多种方法可以绕过数据执行保护，将在 PTES 的利用阶段后面讨论。

## 地址空间布局随机化 (ASLR)

在缓冲区溢出漏洞（或任何我们控制内存的情况下）中，内存地址是硬编码的，以便将执行流重定向到我们的 shellcode。在 ASLR 的情况下，某些字节被随机化，以防止攻击者预测他们总是可以到哪里去执行 shellcode。

## Web 应用防火墙 (WAF)

Web 应用防火墙是一种与应用程序串联部署的技术，用于保护免受基于 Web 的应用程序攻击。Web 应用防火墙试图识别针对给定 Web 应用程序的潜在危险或畸形攻击，并阻止它们。有许多绕过 Web 应用防火墙的技术，应在渗透测试中进行测试。

# 规避

规避是在渗透测试期间用于逃避检测的技术。这可以是绕过摄像系统以免被警卫看到，混淆有效载荷以逃避入侵检测系统 (IDS) 或入侵防御系统 (IPS)，或对请求/响应进行编码以绕过 Web 应用防火墙。总体而言，应在利用之前制定一个低风险的规避技术或人员策略。

# 精确打击

渗透测试的主要焦点是模拟攻击者，以代表对组织进行模拟攻击。渗透测试带来的价值通常不是通过“ smash and grab”技术（即攻击本身嘈杂，尝试每一种漏洞）来实现的。这种方法在渗透测试结束时可能特别有用，以衡量组织的事件响应水平，但在大多数情况下，利用阶段是对目标进行特定研究的积累。

# 定制的利用途径

每次攻击的利用途径通常不会相同。为了在本阶段取得成功，攻击应根据场景进行定制和调整。例如，如果进行无线渗透测试，并且使用了特定技术，则需要识别这些技术并根据已部署的技术进行攻击。清晰理解每个场景和漏洞利用的适用性是渗透测试本阶段最重要的方面之一。

# 定制化的漏洞利用

在许多情况下，互联网上公开的漏洞利用可能需要一些修改才能成功完成。在大多数情况下，如果漏洞利用是为 Windows XP SP2 设计的，则需要对漏洞利用进行特定修改，以便通过 Windows XP SP3 成功进行攻击。渗透测试人员应具备定制漏洞利用的能力，并能够随时更改以成功完成攻击。

## 漏洞利用定制化

在攻击过程中，通常需要模拟受害者的基础设施，以确保利用阶段能够成功。信息收集阶段所利用的技术总是有助于此，但是，拥有一个可用的基础设施和系统将使利用阶段变得容易得多。在定制化漏洞利用的情况下，渗透测试人员应能够定制已公开的漏洞利用，以成功攻击系统。漏洞利用的一个常见主题是针对特定版本的操作系统或应用程序。原因是由于服务包和/或操作系统的新版本导致内存地址发生变化。测试人员应能够定制这些漏洞利用，以成功部署到不同的操作系统并成功攻陷系统。

# 零日角度

在大多数情况下，零日角度通常是大多数渗透测试人员的最后手段。这种类型的攻击通常代表了一个高度复杂的组织，可以通过正常的攻击方法对组织进行集中攻击。在某些情况下，可能需要进行研究以进行逆向工程、模糊测试或执行高级漏洞发现，以发现尚未发现的漏洞。如果这种类型的攻击适用，请确保尽可能重现环境，包括对策技术。

为了使零日漏洞利用（或任何此类利用）成功，拥有相同的操作系统、补丁和对策对成功非常重要。有时，根据已发生的访问或枚举级别，这些信息可能不可用。

## 模糊测试

模糊测试是能够重新创建协议或应用程序，并尝试向应用程序发送数据，希望识别漏洞的能力。通常，模糊测试的目的是识别应用程序中的崩溃，并从中制作特定的漏洞利用。在模糊测试的情况下，攻击者试图从以前未被发现的事物中创建特定的漏洞。作为渗透测试的一部分，如果在参与过程中没有识别出任何途径，或者参与要求进行零日研究，则应利用模糊测试技术来识别潜在的漏洞暴露。

## 源代码分析

渗透测试人员可用的其他途径是，如果源代码可用或是开源的。如果测试人员能够查看源代码并识别应用程序中的缺陷，也可以通过这些方法识别零日漏洞。

## 漏洞利用类型

在渗透测试期间可以识别出几种类型的漏洞利用，这些漏洞利用可被归类为零日。本节列出了一些。

### 缓冲区溢出

缓冲区溢出的发生是由于不正确的编码技术。具体来说，这通常发生在程序将数据写入缓冲区，然后超出缓冲区的边界并开始覆盖内存部分时。在缓冲区溢出漏洞利用中，攻击者的目标是控制崩溃并在给定系统上获得代码执行。在缓冲区溢出漏洞利用中，一种更常见的技术是覆盖给定寄存器并“跳转”到 shellcode。

### SEH 覆盖

SEH 覆盖发生在结构化异常处理程序开始优雅地关闭应用程序时。攻击者可以操纵 SEH 的工作方式，覆盖 SEH 处理程序的基础地址，并通过 SEH 获得执行流的控制权。这是一种常见的攻击，与缓冲区溢出漏洞和已使用 SEH 编译的应用程序结合使用。

### 面向返回的编程 (ROP)

面向返回的编程 (ROP) 是一种技术，在用户控制执行流但可能存在数据执行保护 (DEP) 或其他阻止防御机制的阶段使用。在启用 DEP 的情况下，攻击者无法直接执行特定的汇编指令，因此攻击者构建 ROP 小工具以准备某些 Windows API 调用或技术来禁用 DEP 或绕过 DEP。一种常见的方法是利用 WriteProcessMemory 调用将数据从栈复制到可写内存空间，然后可以执行该空间。

## 流量分析

流量分析是识别正在发送的信息类型以及理解和操纵该流量的能力。渗透测试人员应能够理解协议的工作原理以及如何操纵它以发起攻击。

## 物理访问

渗透测试期间的物理访问可以是一种可行的攻击方法，用于尝试绕过物理安全控制并获得未授权访问。在渗透测试期间，评估者应能够识别可能存在缺陷的物理安全控制，并尝试在范围内进入设施。

### 人为角度

在物理渗透测试期间，最明显的方式之一是通过社会工程学进入设施并获得访问权限。这需要深入了解组织的业务运作方式，以及您从情报收集阶段学到的所有知识。

### PC 访问

如果获得了对 PC 的物理访问权限，渗透测试人员应能够攻击该 PC 并通过多种允许访问系统的方法获得访问权限。

## 近距离访问 (WiFi)

无线通信是攻击者通过射频类型通信获得访问权限的一种途径。渗透测试人员应查看 FCC 无线电频率列表，以确定目标是否已注册使用的频谱频率。

### WiFi 攻击

无论协议如何，都有许多可用的攻击方式针对 WEP、WPA、WPA2、EAP-FAST、EAP-LEAP 和其他途径。攻击者应熟悉各种加密协议和标准，并能够有效地测试已部署控制措施的实施情况。

### 攻击用户

利用恶意接入点来攻击受害者通常是一种有益且可行的攻击方法。在进行无线评估时，应利用恶意接入点引诱受害者，以利用漏洞或窃取敏感信息。有几种常见的技术用于此目的，但最常见的是攻击者设置一个具有相同名称或诱人名称的无线接入点，以便受害者连接。

# 攻击途径示例

在任何场景中，攻击应根据参与范围内的场景进行定制。以下是根据场景需要考虑的几个攻击途径列表，但绝不是详尽列表。

- Web 应用程序攻击
- 社会工程学
- 物理攻击途径
- 基于内存的漏洞利用（例如缓冲区/堆溢出、内存损坏、释放后使用）
- 中间人攻击
- VLAN 跳转
- USB/闪存驱动器部署
- 逆向工程
- 零日角度
- 攻击用户
- 加密破解
- 图形处理单元 (GPU) 破解
- 流量分析
- 火线接口 (Firewire)
- 路由协议
- 带有预文本的网络钓鱼
- 员工冒充

再次强调，这些示例只是基于您为组织执行的场景的基本攻击途径。渗透测试的价值来自于创造力以及识别暴露点并以精确方式利用它们的能力。

# 总体目标

在与客户进行参与前交互阶段，应已明确沟通渗透测试的总体目标。在利用阶段，最大的挑战是识别进入组织的最少阻力路径，且不被发现，并对组织创收能力产生最大影响。

通过正确执行前面的阶段，应该相对清楚地了解组织的运作方式和盈利模式。从利用阶段到后利用阶段，攻击向量应完全依赖于绕过安全控制的任务，以展示组织如何通过针对性的攻击遭受重大损失。
