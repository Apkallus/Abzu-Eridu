## TCP FIN、NULL 和 Xmas 扫描（`-sF`、`-sN`、`-sX`）

这三种扫描类型（下一节描述的 `--scanflags` 选项甚至允许更多组合）利用了 [TCP RFC](http://www.rfc-editor.org/rfc/rfc793.txt) 中的一个细微漏洞来区分 `open` 和 `closed` 端口。RFC 793 第 65 页指出：“如果[目标]端口状态为 CLOSED .... 一个不包含 RST 的 incoming segment 会导致发送一个 RST 作为响应。”下一页讨论发送到开放端口且未设置 SYN、RST 或 ACK 位的数据包，指出：“你不太可能到达这里，但如果到达了，请丢弃该 segment 并返回。”

在扫描符合此 RFC 文本的系统时，任何不包含 SYN、RST 或 ACK 位的数据包，如果端口关闭将返回一个 RST，如果端口开放则根本没有任何响应。只要不包含这三个位中的任何一个，其他三个位（FIN、PSH 和 URG）的任何组合都是允许的。Nmap 利用这一点实现了三种扫描类型：

- **Null 扫描（`-sN`）**：不设置任何位（TCP 标志头为 0）
- **FIN 扫描（`-sF`）**：仅设置 TCP FIN 位。
- **Xmas 扫描（`-sX`）**：设置 FIN、PSH 和 URG 标志，像圣诞树一样点亮数据包。

这三种扫描类型的行为完全相同，只是探测数据包中设置的 TCP 标志不同。响应按表 5.4 所示处理。

**表 5.4 Nmap 如何解释对 NULL、FIN 或 Xmas 扫描探测的响应**

| 探测响应 | 分配的状态 |
| :--- | :--- |
| 未收到响应（即使重传后） | `open|filtered` |
| TCP RST 数据包 | `closed` |
| ICMP 不可达错误（类型 3，代码 1、2、3、9、10 或 13） | `filtered` |

这些扫描类型的主要优势是它们能够穿过某些无状态防火墙和数据包过滤路由器。此类防火墙试图通过阻止任何设置了 SYN 位且清除 ACK 位的 TCP 数据包来防止入站 TCP 连接（同时允许出站连接）。这种配置非常常见，以至于 Linux iptables 防火墙命令提供了一个特殊的 `--syn` 选项来实现它。NULL、FIN 和 Xmas 扫描清除了 SYN 位，因此可以直接穿过这些规则。

另一个优势是这些扫描类型甚至比 SYN 扫描更隐蔽一点。不过不要依赖这一点——大多数现代 IDS 产品都可以配置为检测它们。

最大的缺点是并非所有系统都严格遵守 [RFC 793](http://www.rfc-editor.org/rfc/rfc793.txt)。许多系统无论端口是否开放，都会对探测发送 RST 响应。这会导致所有端口都被标记为 `closed`。这样做的主要操作系统是 Microsoft Windows、许多 Cisco 设备和 IBM OS/400。不过，这种扫描确实适用于大多数基于 Unix 的系统。由于 Nmap 的操作系统检测会测试这种怪癖，您可以通过查看 `nmap-os-db` 文件来了解扫描是否适用于特定类型的系统。测试 T2 向开放端口发送一个 NULL 数据包。因此，如果您看到像 `T2(R=N)` 这样的行，该系统似乎支持 RFC，这些扫描之一应该对其有效。如果 T2 行较长，则系统违反了 RFC 并发送了响应，这些扫描将不起作用。第 8 章《远程操作系统检测》更详细地解释了操作系统指纹识别。

这些扫描的另一个缺点是无法区分开放端口和某些被过滤的端口。如果数据包过滤器发送 ICMP 禁止错误，Nmap 知道端口被过滤。但大多数过滤器只是简单地丢弃被禁止的探测而不作任何响应，使端口看起来是开放的。由于 Nmap 无法确定是哪种情况，它将无响应的端口标记为 `open|filtered`。添加版本检测（`-sV`）可以像 UDP 扫描那样消除歧义，但这会破坏该扫描的大部分隐蔽性。如果您愿意并且能够连接到这些端口，那么不妨使用 SYN 扫描。

使用这些扫描方法很简单。只需添加 `-sN`、`-sF` 或 `-sX` 选项来指定扫描类型。示例 5.10 显示了两个示例。第一个是对 Para 的 FIN 扫描，将所有五个开放端口识别为 `open|filtered`。第二次执行是对 scanme.nmap.org 的 Xmas 扫描，效果不太好。它检测到了关闭端口，但无法区分 995 个过滤端口和四个开放端口，所有 999 个都被列为 `open|filtered`。这说明了为什么 Nmap 提供如此多的扫描方法。没有一种技术在所有情况下都是优选的。Ereet 将不得不尝试另一种方法来了解更多关于 Scanme 的信息。
