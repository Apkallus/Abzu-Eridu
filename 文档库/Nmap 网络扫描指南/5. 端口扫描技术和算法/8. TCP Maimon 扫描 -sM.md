## TCP Maimon 扫描（`-sM`）

Maimon 扫描以其发现者 Uriel Maimon 的名字命名。他在《Phrack》杂志第 49 期（1996 年 11 月）中描述了这项技术。Nmap 在随后的两期版本中加入了这项技术。该技术与 NULL、FIN 和 Xmas 扫描完全相同，区别在于探测包是 FIN/ACK。根据 [RFC 793](http://www.rfc-editor.org/rfc/rfc793.txt)（TCP），无论端口是开放还是关闭，收到此类探测都应生成一个 RST 数据包。然而，Uriel 注意到许多 BSD 衍生的系统在端口开放时会直接丢弃该数据包。Nmap 利用这一点来确定开放端口，如表 5.7 所示。

**表 5.7 Nmap 如何解释对 Maimon 扫描探测的响应**

| 探测响应 | 分配的状态 |
| :--- | :--- |
| 未收到响应（即使重传后） | `open|filtered` |
| TCP RST 数据包 | `closed` |
| ICMP 不可达错误（类型 3，代码 1、2、3、9、10 或 13） | `filtered` |

Maimon 扫描的 Nmap 标志是 `-sM`。虽然该选项在 1996 年非常有用，但现代系统很少表现出这个漏洞。它们对所有端口都返回 RST，使得每个端口看起来都是关闭的。示例 5.18 显示了这一结果。

**示例 5.18 一次失败的 Maimon 扫描**

```screen
# nmap -sM -T4 para

Starting Nmap ( https://nmap.org )
All 1000 scanned ports on para (192.168.10.191) are: closed
MAC Address: 00:60:1D:38:32:90 (Lucent Technologies)

Nmap done: 1 IP address (1 host up) scanned in 4.19 seconds
```
