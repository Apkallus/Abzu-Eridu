## 使用 `--scanflags` 自定义扫描类型

真正高级的 Nmap 用户无需将自己局限于预设的扫描类型。`--scanflags` 选项允许您通过指定任意 TCP 标志来设计自己的扫描。释放您的创造力，同时绕过那些厂商仅仅翻阅 Nmap 手册页添加特定规则的入侵检测系统！

`--scanflags` 的参数可以是一个数字标志值（例如 9 表示 PSH 和 FIN），但使用符号名称更容易。只需将 `URG`、`ACK`、`PSH`、`RST`、`SYN` 和 `FIN` 的任意组合拼在一起即可。例如，`--scanflags URGACKPSHRSTSYNFIN` 设置了所有标志，但这对于扫描来说没什么用。这些标志的顺序无关紧要。

除了指定所需的标志，您还可以指定一个 TCP 扫描类型（例如 `-sA` 或 `-sF`）。该基础类型告诉 Nmap 如何解释响应。例如，SYN 扫描将无响应视为 `filtered` 端口，而 FIN 扫描将同样的情况视为 `open|filtered`。Nmap 的行为将与基础扫描类型相同，只是它会使用您指定的 TCP 标志。如果您不指定基础类型，则使用 SYN 扫描。

### 自定义 SYN/FIN 扫描

一种有趣的自定义扫描类型是 SYN/FIN 扫描。有时防火墙管理员或设备制造商试图通过类似“丢弃任何仅设置了 SYN 标志的入站数据包”的规则来阻止入站连接。他们将其限制为 **仅** SYN 标志，因为他们不想阻止作为出站连接第二步返回的 SYN/ACK 数据包。

这种方法的问题在于，大多数终端系统也会接受包含其他（非 ACK）标志的初始 SYN 数据包。例如，Nmap 的操作系统指纹识别系统会向开放端口发送一个 SYN/FIN/URG/PSH 数据包。数据库中超过一半的指纹以 SYN/ACK 响应。因此它们允许使用此数据包进行端口扫描，并且通常也允许建立完整的 TCP 连接。甚至已知某些系统会对 SYN/RST 数据包响应 SYN/ACK！[TCP RFC](http://www.rfc-editor.org/rfc/rfc793.txt) 对于初始 SYN 数据包中哪些标志是可接受的规定并不明确，尽管 SYN/RST 显然看起来是伪造的。

示例 5.13 显示 Ereet 成功对 Google 进行了 SYN/FIN 扫描。显然他开始对 scanme.nmap.org 感到厌倦了。

**示例 5.13 对 Google 的 SYN/FIN 扫描**

```screen
krad# nmap -sS --scanflags SYNFIN -T4 www.google.com

Starting Nmap ( https://nmap.org )
Warning: Hostname www.google.com resolves to 4 IPs. Using 74.125.19.99.
Nmap scan report for cf-in-f99.google.com (74.125.19.99)
Not shown: 996 filtered ports
PORT    STATE  SERVICE
80/tcp  open   http
113/tcp closed auth
179/tcp closed bgp
443/tcp open   https

Nmap done: 1 IP address (1 host up) scanned in 7.58 seconds
```

类似的扫描类型，如 SYN/URG 或 SYN/PSH/URG/FIN，通常也有效。如果您无法穿透，不要忘记前面提到的 SYN/RST 选项。

### PSH 扫描

[第 5.4 节 “TCP FIN、NULL 和 Xmas 扫描（`-sF`、`-sN`、`-sX`）”](scan-methods-null-fin-xmas-scan.html) 指出，符合 RFC 的系统允许使用 FIN、PSH 和 URG 标志的任意组合来扫描端口。虽然有八种可能的排列，但 Nmap 只提供了三种预设模式（NULL、FIN 和 Xmas）。尝试使用 PSH/URG 或 FIN/PSH 扫描来展示一些个人风格。结果很少与三种预设模式不同，但有可能规避扫描检测系统。

要执行这样的扫描，只需使用 `--scanflags` 指定所需的标志，并指定 FIN 扫描（`-sF`）作为基础类型（选择 NULL 或 Xmas 没有区别）。示例 5.14 演示了对本地网络上的一台 Linux 机器进行的 PSH 扫描。

**示例 5.14 自定义 PSH 扫描**

```screen
krad# nmap -sF --scanflags PSH  para

Starting Nmap ( https://nmap.org )
Nmap scan report for para (192.168.10.191)
(The 995 ports scanned but not shown below are in state: closed)
PORT     STATE         SERVICE
22/tcp   open|filtered ssh
53/tcp   open|filtered domain
111/tcp  open|filtered rpcbind
515/tcp  open|filtered printer
6000/tcp open|filtered X11
MAC Address: 00:60:1D:38:32:90 (Lucent Technologies)

Nmap done: 1 IP address (1 host up) scanned in 5.95 seconds
```

因为这些扫描都以相同的方式工作，我可以只保留 `-sF`、`-sN` 和 `-sX` 选项中的一个，让用户使用 `--scanflags` 来模拟其他选项。目前没有这样做的计划，因为快捷选项更容易记住和使用。您仍然可以尝试模拟的方法来展示您的 Nmap 技能。执行 `nmap -sF --scanflags FINPSHURG target` 而不是更普通的 `nmap -sX target`。
