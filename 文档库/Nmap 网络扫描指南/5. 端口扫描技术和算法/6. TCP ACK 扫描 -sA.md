## TCP ACK 扫描（`-sA`）

这种扫描与前面讨论的其他扫描不同，它从不确定 `open`（甚至 `open|filtered`）端口。它用于映射防火墙规则集，确定防火墙是否具有状态以及哪些端口被过滤。

通过指定 `-sA` 选项启用 ACK 扫描。其探测数据包仅设置了 ACK 标志（除非使用 `--scanflags`）。在扫描未经过滤的系统时，`open` 和 `closed` 端口都会返回 RST 数据包。Nmap 随后将它们标记为 `unfiltered`，这意味着 ACK 数据包可以到达这些端口，但无法确定它们是 `open` 还是 `closed`。没有响应或发回某些 ICMP 错误消息的端口被标记为 `filtered`。表 5.5 提供了完整的详细信息。

**表 5.5 Nmap 如何解释对 ACK 扫描探测的响应**

| 探测响应 | 分配的状态 |
| :--- | :--- |
| TCP RST 响应 | `unfiltered` |
| 未收到响应（即使重传后） | `filtered` |
| ICMP 不可达错误（类型 3，代码 1、2、3、9、10 或 13） | `filtered` |

ACK 扫描的使用与大多数其他扫描类型类似，只需添加一个选项标志即可，这里就是 `-sA`。示例 5.15 显示了对 Scanme 的 ACK 扫描。

**示例 5.15 一个典型的 ACK 扫描**

```screen
krad# nmap -sA -T4 scanme.nmap.org

Starting Nmap ( https://nmap.org )
Nmap scan report for scanme.nmap.org (64.13.134.52)
Not shown: 994 filtered ports
PORT    STATE      SERVICE
22/tcp  unfiltered ssh
25/tcp  unfiltered smtp
53/tcp  unfiltered domain
70/tcp  unfiltered gopher
80/tcp  unfiltered http
113/tcp unfiltered auth

Nmap done: 1 IP address (1 host up) scanned in 4.01 seconds
```

ACK 扫描最有趣的用途之一是区分有状态防火墙和无状态防火墙。有关如何执行此操作以及为什么要这样做的信息，请参见 [“ACK 扫描”一节](determining-firewall-rules.html#defeating-firewalls-ids-ackscan)。

有时可以通过组合多种扫描类型来从系统中获取额外信息。例如，首先回顾一下 [示例 5.12“对 Docsrv 的 FIN 扫描”](scan-methods-null-fin-xmas-scan.html#scan-methods-ex-sco-fin-scan)。在该例中，Nmap 发现了关闭的端口，但有 39 个端口被列为 `open|filtered`，因为 FIN 扫描无法区分这两种状态。现在看一下对同一主机的 ACK 扫描（示例 5.16“对 Docsrv 的 ACK 扫描”）。之前未识别的 39 个端口中有 2 个被显示为 `filtered`。其他 37 个（基于表上方默认端口行）处于 `unfiltered` 状态。这意味着它们要么是 `open` 要么是 `closed`。如果一种扫描类型将端口标识为 `open` 或 `filtered`，而另一种将其标识为 `open` 或 `closed`，逻辑上可以推断它一定是 `open`。通过结合两种扫描类型，我们得知 Docsrv 上有 37 个端口是 `open`，2 个是 `filtered`，961 个是 `closed`。虽然在这里逻辑推理很好地确定了端口状态，但这种技术并不总是可靠的。它假设不同的扫描类型对同一端口总是返回一致的状态，而这并不准确。防火墙和 TCP 协议栈的特性可能导致对同一台机器的不同扫描产生显著差异。对于 Docsrv，我们看到 SYN 扫描认为 SSH 端口（`tcp/22`）是 `filtered`，而 ACK 扫描认为它是 `unfiltered`。在探索边界条件和配置奇怪的网络时，解释 Nmap 结果是一门艺术，需要经验和直觉。

**示例 5.16 对 Docsrv 的 ACK 扫描**

```screen
# nmap -sA -T4 docsrv.caldera.com

Starting Nmap ( https://nmap.org )
Nmap scan report for docsrv.caldera.com (216.250.128.247)
Not shown: 998 unfiltered ports
PORT     STATE    SERVICE
135/tcp  filtered msrpc
1434/tcp filtered ms-sql-m

Nmap done: 1 IP address (1 host up) scanned in 7.20 seconds
```
