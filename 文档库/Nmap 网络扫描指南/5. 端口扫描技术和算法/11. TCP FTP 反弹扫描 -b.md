## TCP FTP 反弹扫描（`-b`）

FTP 协议（[RFC 959](http://www.rfc-editor.org/rfc/rfc959.txt)）一个有趣的特性是支持所谓的代理 FTP 连接。这允许用户连接到一台 FTP 服务器，然后请求将文件发送到第三方服务器。这种特性在多个层面上都很容易被滥用，因此大多数服务器已停止支持它。该特性允许的滥用之一就是让 FTP 服务器对其他主机进行端口扫描。只需让 FTP 服务器依次向目标主机的每个感兴趣的端口发送一个文件。错误消息将描述端口是否开放。这是绕过防火墙的好方法，因为组织中的 FTP 服务器通常被放置在比普通互联网主机对其他内部主机有更多访问权限的位置。Nmap 通过 `-b` 选项支持 FTP 反弹扫描。它接受以下格式的参数：`<username>:<password>@<server>:<port>`。`<server>` 是易受攻击的 FTP 服务器的名称或 IP 地址。与普通 URL 一样，您可以省略 `<username>:<password>`，此时使用匿名登录凭据（用户名：`anonymous` 密码：`-wwwuser@`）。端口号（及其前面的冒号）也可以省略，此时使用 `<server>` 上的默认 FTP 端口（21）。

示例 5.21 显示了一次通过微软主 FTP 服务器反弹来扫描 google.com 的尝试。

**示例 5.21 尝试 FTP 反弹扫描**

```screen
# nmap -Pn -b ftp.microsoft.com google.com

Starting Nmap ( https://nmap.org )
Your FTP bounce server doesn't allow privileged ports, skipping them.
Your FTP bounce server sucks, it won't let us feed bogus ports!
```

FTP 反弹扫描的频繁使用者最好习惯这个错误消息。这个漏洞在 1997 年 Nmap 发布时很普遍，但现在已基本修复。易受攻击的服务器仍然存在，所以在其他方法都失败时值得一试。如果您的目标是绕过防火墙，请扫描目标网络的开放端口 21（或者如果您使用版本检测扫描所有端口，则扫描任何 FTP 服务），然后尝试使用每个服务进行反弹扫描。Nmap 会告诉您主机是否易受攻击。如果您只是想隐藏自己的踪迹，您不需要（事实上也不应该）将自己限制在目标网络上的主机。在您开始扫描随机互联网地址以寻找易受攻击的 FTP 服务器之前，请考虑系统管理员可能不喜欢您以这种方式滥用他们的服务器。

示例 5.22 显示了对 Scanme 上几个有趣端口的成功反弹扫描。给出了详细选项（`-v`）以提供额外细节。给定的服务器类型“JD FTP Server”表示这是一台 HP JetDirect 打印服务器。
