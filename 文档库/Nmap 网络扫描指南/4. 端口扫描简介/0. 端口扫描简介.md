## 端口扫描简介

虽然 Nmap 的功能多年来不断增长，但它最初是一个高效的端口扫描器，这仍然是其核心功能。简单的命令 `nmap <target>` 会扫描主机 `<target>` 上最常用的 1,000 个 TCP 端口，并将每个端口分类为以下状态之一：`open`、`closed`、`filtered`、`unfiltered`、`open|filtered` 或 `closed|filtered`。

### 究竟什么是端口？

端口只是一种软件抽象，用于区分不同的通信通道。类似于 IP 地址用于标识网络上的机器，端口用于标识单台机器上正在使用的特定应用程序。例如，您的 Web 浏览器默认会连接到 HTTP URL 中机器的 TCP 端口 80。如果您改为指定安全的 HTTPS 协议，浏览器将默认尝试端口 443。

Nmap 使用两种基于端口的协议：TCP 和 UDP。每种协议的连接由四个元素唯一标识：源 IP 地址、目标 IP 地址以及相应的源端口和目标端口。所有这些元素都是放置在主机之间发送的每个数据包报头中的数字。协议是一个 8 位字段，用于指定 IP 数据（有效载荷）部分中包含的数据包类型。例如，TCP 是协议号 6，UDP 是 17。IPv4 地址长度为 32 位，而端口长度为 16 位。IPv6 地址长度为 128 位。有关 IP、TCP 和 UDP 报头布局的更多详细信息，请参见“TCP/IP 参考”一节。

由于大多数流行服务都注册了众所周知的端口号，人们通常可以猜测开放端口代表什么服务。Nmap 包含一个 [`nmap-services`](https://svn.nmap.org/nmap/nmap-services) 文件，其中包含注册端口和协议号的众所周知服务，以及木马后门和其他懒得向互联网号码分配机构（IANA）注册的应用程序的常见端口。Nmap 会打印此服务名称以及端口号以供参考。

由于端口号字段是 16 位宽，因此值可以达到 65,535。最小可能值零是无效的。Berkeley sockets API（定义了网络通信程序的通常编写方式）不允许将端口零用作此类。相反，它将端口零请求解释为通配符，意味着程序员不关心使用哪个端口。然后系统会选择一个可用的端口号。例如，程序员很少关心传出连接使用哪个源端口号，因此他们将其设置为零，让操作系统选择一个。

虽然端口零无效，但没有什么能阻止有人在报头字段中指定它。一些恶意的木马后门会在被入侵系统的端口零上监听，作为一种隐蔽的方式提供非法访问，而不会出现在大多数端口扫描中。为了应对这种情况，Nmap 允许在明确指定时扫描端口零（例如 `-p0-65535`）。

第一类有效端口，即 1 到 1,023 号，被称为保留端口。Unix 系统（与 Windows 不同）要求应用程序具有特殊的（root）特权才能绑定并监听这些端口。其想法是允许远程用户信任他们正在连接到由管理员启动的有效服务，而不是某个恶意的非特权用户。如果 SSH 的注册端口是 2,222 而不是 22，恶意用户可以在该端口上启动一个 rogue SSH 守护进程，从任何连接的人那里收集密码。由于大多数常见的服务器应用程序监听保留端口，因此这些端口通常是最值得扫描的。

临时端口范围是另一类端口。系统提供此端口池以供按需分配。当应用程序指定端口零（意思是“任何端口”）时，系统会从此范围内选择一个端口。该范围因操作系统而异，并且通常是可配置的。它应至少包含几千个端口，以避免在打开许多并发连接时耗尽。Nmap 连接扫描在扫描每个目标机器上的每个指定端口时，一次可以使用数百个端口。在 Linux 上，您可以使用文件 `/proc/sys/net/ipv4/ip_local_port_range` 查看或设置该范围。示例 4.1 显示，在我的 Linux 系统上，范围是 32,768 到 61,000。如此大的范围在几乎所有情况下都应该足够了，但我将其扩展只是为了演示如何操作。

**示例 4.1 查看并增加 Linux 上的临时端口范围**

```screen
felix/# cat /proc/sys/net/ipv4/ip_local_port_range 
32768   61000
felix/# echo "10000 65000" > /proc/sys/net/ipv4/ip_local_port_range
felix/# cat /proc/sys/net/ipv4/ip_local_port_range
10000   65000
felix/#
```

SunRPC 端口通常在临时范围内找到。其他应用程序会临时打开临时端口以进行文件传输或其他事件。FTP 客户端在请求主动模式传输时经常这样做。一些 P2P 和即时通讯客户端也会这样做。

IANA 有自己的端口分类方案，与本书的术语略有不同。他们在 [`http://www.iana.org/assignments/port-numbers`](http://www.iana.org/assignments/port-numbers) 上的权威端口列表将空间分为以下三类：

- **well-known ports（众所周知端口）**  
  这些是保留端口（范围为 1 到 1,023，如上所述），已向 IANA 注册用于特定服务。熟悉的例子是服务 SSH、SMTP 和 HTTP 的端口 22、25 和 80。

- **registered ports（注册端口）**  
  这些端口在 1,024 到 49,151 范围内，并且以与众所周知端口相同的方式向 IANA 注册。其中大多数不像众所周知端口那样常用。关键区别在于非特权用户可以绑定到这些端口，从而在其注册端口上运行服务。对于众所周知端口，用户无法在大多数平台上这样做，因为它们位于保留端口范围内。

- **dynamic and/or private ports（动态和/或私有端口）**  
  IANA 将 49,152 到 65,535 的端口号保留用于动态用途，例如临时端口部分讨论的那些。仅在公司内部使用的专有服务也可以使用这些端口。

当本书在没有提及 IANA 的情况下提到注册或众所周知的端口时，它通常指的是在 `nmap-services` 文件中向 Nmap 注册的端口，无论它们是否属于保留端口范围。

Nmap 的端口注册文件（`nmap-services`）包含关于每个 TCP 或 UDP 端口被发现开放的频率的经验数据。默认情况下，Nmap 扫描它被要求扫描的每个协议中最流行的 1,000 个端口。有许多选项可以指定备选端口集（按频率或显式列出它们），如 [“选择要扫描的端口”](port-scanning-options.html#port-scanning-options-ports) 一节所述。

### 哪些是最流行的端口？

我在 2008 年夏天扫描了数千万个互联网主机，并从企业收集数据，以确定每个端口号被发现开放的频率。熟悉最常见的服务端口很重要，看看哪些端口进入了列表也很有趣。以下两个列表提供了根据我们的经验扫描数据确定的顶级 TCP 和 UDP 端口。列出的服务是在我们的 `nmap-services` 文件中找到的服务。我们尝试列出每个端口最常见的服务，尽管一个端口可能被用于不同的事情。

**最常见的 20 个 TCP 端口**

1. **端口 80（HTTP）** — 如果你连这个服务都不知道，那你读错书了。这占我们发现开放端口的 14% 以上。
2. **端口 23（Telnet）** — Telnet 仍然存在（特别是在路由器和智能交换机等设备上作为管理端口），尽管它是不安全的（未加密）。
3. **端口 443（HTTPS）** — SSL 加密的 Web 服务器默认使用此端口。
4. **端口 21（FTP）** — FTP 像 Telnet 一样，是另一种应该消亡的不安全协议。即使是匿名 FTP（避免了身份验证嗅探的担忧），数据传输仍然容易被篡改。
5. **端口 22（SSH）** — Secure Shell，Telnet（以及在某些情况下的 FTP）的加密替代品。
6. **端口 25（SMTP）** — 简单邮件传输协议（也不安全）。
7. **端口 3389（ms-term-server）** — Microsoft 终端服务管理端口。
8. **端口 110（POP3）** — 用于电子邮件检索的邮局协议版本 3（不安全）。
9. **端口 445（Microsoft-DS）** — 用于通过 IP 与 MS Windows 服务（如文件/打印机共享）进行 SMB 通信。
10. **端口 139（NetBIOS-SSN）** — 用于与 MS Windows 服务（如文件/打印机共享）通信的 NetBIOS 会话服务。这在 Windows 机器上得到支持的时间比 445 更长。
11. **端口 143（IMAP）** — 互联网消息访问协议版本 2。一种不安全的电子邮件检索协议。
12. **端口 53（Domain）** — 域名系统（DNS），一种用于主机/域名和 IP 地址之间转换的不安全系统。
13. **端口 135（MSRPC）** — MS Windows 服务的另一个常见端口。
14. **端口 3306（MySQL）** — 用于与 MySQL 数据库通信。
15. **端口 8080（HTTP-Proxy）** — 通常用于 HTTP 代理或作为普通 Web 服务器的备用端口（例如，当另一个服务器已在端口 80 上监听时，或者由只能绑定到高端口的非特权 UNIX 用户运行时）。
16. **端口 1723（PPTP）** — 点对点隧道协议（一种实现 VPN 的方法，通常是宽带连接到 ISP 所必需的）。
17. **端口 111（RPCBind）** — 将 SunRPC 程序号映射到它们当前的 TCP 或 UDP 端口号。
18. **端口 995（POP3S）** — 添加 SSL 以提高安全性的 POP3。
19. **端口 993（IMAPS）** — 添加 SSL 以提高安全性的 IMAPv2。
20. **端口 5900（VNC）** — 一种图形桌面共享系统（不安全）。

**最常见的 20 个 UDP 端口**

1. **端口 631（IPP）** — 互联网打印协议。
2. **端口 161（SNMP）** — 简单网络管理协议。
3. **端口 137（NETBIOS-NS）** — 用于 Windows 服务（如文件和打印机共享）的众多 UDP 端口之一。
4. **端口 123（NTP）** — 网络时间协议。
5. **端口 138（NETBIOS-DGM）** — 另一个 Windows 服务。
6. **端口 1434（MS-SQL-DS）** — Microsoft SQL Server。
7. **端口 445（Microsoft-DS）** — 另一个 Windows 服务端口。
8. **端口 135（MSRPC）** — 另一个 Windows 服务端口。
9. **端口 67（DHCPS）** — 动态主机配置协议服务器（当客户端加入网络时向其分配 IP 地址）。
10. **端口 53（Domain）** — 域名系统（DNS）服务器。
11. **端口 139（NETBIOS-SSN）** — 另一个 Windows 服务端口。
12. **端口 500（ISAKMP）** — 互联网安全协会和密钥管理协议用于设置 IPsec VPN。
13. **端口 68（DHCPC）** — DHCP 客户端端口。
14. **端口 520（Route）** — 路由信息协议（RIP）。
15. **端口 1900（UPNP）** — Microsoft 简单服务发现协议，用于发现通用即插即用设备。
16. **端口 4500（nat-t-ike）** — 在发起 IPsec 连接（在互联网密钥交换期间）时协商网络地址转换遍历。
17. **端口 514（Syslog）** — 标准的 UNIX 日志守护进程。
18. **端口 49152（Varies）** — IANA 指定的动态/私有端口中的第一个。从该端口开始直到端口范围结束（65536），不得注册官方端口。某些系统将其临时端口范围用于此范围，因此绑定端口而不请求特定号码的服务如果第一个这样做，通常会分配 49152。
19. **端口 162（SNMPTrap）** — 简单网络管理协议陷阱端口（SNMP 代理通常使用 161，而 SNMP 管理器通常使用 162）。
20. **端口 69（TFTP）** — 简单文件传输协议。

### 什么是端口扫描？

端口扫描是远程测试众多端口以确定它们处于什么状态的行为。最有趣的状态通常是 `open`，意味着有一个应用程序正在监听并在该端口上接受连接。有多种技术可用于进行此类扫描。第 5 章 《端口扫描技术与算法》解释了每种技术最适用的场景。

虽然传统上许多端口扫描器将所有端口归类为开放或关闭，但 Nmap 更加细粒度。它将端口分为六种状态。这些状态不是端口本身的内在属性，而是描述 Nmap 如何看待它们。例如，从与目标相同的网络进行的 Nmap 扫描可能显示端口 `135/tcp` 是开放的，而同时从互联网另一端使用相同选项进行的扫描可能显示该端口被过滤。

**Nmap 识别的六种端口状态**

- **open**  
  应用程序在此端口上主动接受 TCP 连接或 UDP 数据包。找到这些通常是端口扫描的主要目标。注重安全的人知道每个开放端口都是攻击的途径。攻击者和渗透测试人员希望利用开放端口，而管理员试图在不阻碍合法用户的情况下关闭或使用防火墙保护它们。开放端口对于非安全扫描也很有趣，因为它们显示了网络上可用的服务。在你对开放端口过于兴奋之前，请注意应用程序可能受 TCP wrapper（`tcpd`）保护，或者应用程序本身配置为仅服务批准的客户端 IP 地址。与关闭的端口相比，这种情况仍然留下更多的攻击面。

- **closed**  
  关闭的端口是可访问的（它接收并响应 Nmap 探测数据包），但没有应用程序在监听。它们有助于显示主机在线并使用 IP 地址（主机发现或 ping 扫描），并作为操作系统检测的一部分。由于关闭的端口是可到达的，它们可能值得稍后扫描，以防某些端口开放。管理员可能希望考虑使用防火墙阻止此类端口，使它们呈现为过滤状态（如下所述）。

- **filtered**  
  由于数据包过滤阻止 Nmap 的探测到达端口，Nmap 无法确定端口是否开放。过滤可能来自专用防火墙设备、路由器规则或基于主机的防火墙软件。这些端口让攻击者感到沮丧，因为它们提供的信息很少。有时它们会响应 ICMP 错误消息，例如类型 3 代码 13（目标不可达：管理禁止通信），但更常见的是直接丢弃探测而不响应的过滤器。这迫使 Nmap 重试几次，以防探测是由于网络拥塞而不是过滤而丢失。这种过滤会显著降低扫描速度。

- **unfiltered**  
  unfiltered 状态意味着端口是可访问的，但 Nmap 无法确定它是开放还是关闭。只有 ACK 扫描（用于映射防火墙规则集）将端口分类为此状态。使用其他扫描类型（如 Window 扫描、SYN 扫描或 FIN 扫描）扫描 unfiltered 端口可能有助于确定端口是否开放。

- **open|filtered**  
  当 Nmap 无法确定端口是开放还是被过滤时，它会将端口置于此状态。这发生在开放端口不给出响应的扫描类型中。没有响应也可能意味着数据包过滤器丢弃了探测或其引发的任何响应。因此 Nmap 无法确定端口是开放还是被过滤。UDP、IP 协议、FIN、NULL 和 Xmas 扫描以这种方式对端口进行分类。

- **closed|filtered**  
  此状态在 Nmap 无法确定端口是关闭还是被过滤时使用。它仅用于 “TCP 空闲扫描（`-sI`）” 一节中讨论的 IP ID 空闲扫描。

虽然 Nmap 试图产生准确的结果，但请记住，其所有洞察都基于目标机器（或其前面的防火墙）返回的数据包。此类主机可能是不可信的，并发送旨在混淆或误导 Nmap 的响应。更常见的是不符合 RFC 的主机，它们对 Nmap 探测的响应不符合预期。FIN、NULL 和 Xmas 扫描特别容易受到此问题的影响。此类问题特定于某些扫描类型，因此在第 5 章 《端口扫描技术与算法》 的相关部分中讨论。

### 为什么要扫描端口？

端口扫描不仅是为了娱乐和消遣。定期扫描您的网络有许多实际好处。其中最重要的是安全性。网络安全的核心原则之一是，减少所提供的服务的数量和复杂性会减少攻击者入侵的机会。大多数远程网络入侵来自利用监听在 TCP 或 UDP 端口上的服务器应用程序。在许多情况下，被利用的应用程序甚至没有被目标组织使用，而是在设置机器时默认启用的。如果该服务被禁用或受防火墙保护，攻击就会被阻止。

意识到每个开放端口都是入侵的机会，攻击者定期扫描目标，列出所有开放端口的清单。他们将这个监听服务列表与他们的易受攻击软件的漏洞利用列表进行比较。只需要一个匹配就可以入侵一台机器，创建一个通常用于感染整个网络的立足点。那些对他们攻击目标不那么挑剔的攻击者通常会只扫描可被利用的应用程序的默认端口。这比扫描每个端口快得多，尽管当服务在非默认端口上运行时会被遗漏。此类攻击者常被嘲笑为“脚本小子”，因为他们对安全的了解通常只比运行由更熟练的人编写的漏洞利用脚本多一点。在许多组织中，这样的攻击者必然会发现易受攻击的主机。他们可能相当讨厌，尽管他们庞大的数量和不断对可访问互联网的机器的猛烈攻击常常驱使人们快速修补系统。这降低了更严重的针对性攻击成功的可能性。

抵御这些破解者的一个重要防御措施是系统管理员定期使用 Nmap 等工具扫描自己的网络。获取开放端口列表，并关闭任何未使用的服务。确保那些必须保持可用的服务已完全修补，并且你已加入供应商的安全通知列表。应在可能的情况下添加防火墙规则，将访问限制为仅合法用户。大多数流行应用程序的加固说明都可以在网上找到，从而进一步减少破解者的机会。Nmap 不能为你完成大部分工作，但它会创建可用服务的列表作为起点。一些管理员尝试使用 `netstat` 代替，但这不能很好地扩展。它需要访问每台机器，而一些移动机器很容易被遗漏。此外，你不能在普通的无线接入点、VoIP 电话或打印机上运行 `netstat`。此外，总存在被入侵的机器可能有一个被植入木马的 `netstat`，给出虚假信息的风险。攻击者安装的大多数现代 rootkit 都包含此功能。仅仅依赖 Nmap 也是一个错误。结合仔细的设计、配置审计和定期扫描是非常可取的。

虽然安全是端口扫描最常见的原因，但管理员经常发现它也适用于其他目的。创建机器及其所提供服务的清单对于资产跟踪、网络设计、策略合规性检查、软件许可证跟踪、可用性测试、网络调试等非常有用。
