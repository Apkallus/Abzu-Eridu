## IPv6 扫描（`-6`）

自 2002 年以来，Nmap 为其最流行的功能提供了 IPv6 支持。特别是，ping 扫描（仅 TCP）、连接扫描和版本检测都支持 IPv6。命令语法与往常相同，只是需要添加 `-6` 选项。当然，如果您指定的是地址而不是主机名，则必须使用 IPv6 语法。一个地址可能看起来像 `3ffe:7501:4819:2000:210:f3ff:fe03:14d0`，因此建议使用主机名。[示例 4.4](#port-scanning-ex-ipv6) 展示了一个典型的端口扫描会话。输出看起来与平常相同，“interesting ports”行上的 IPv6 地址是唯一表明使用 IPv6 的地方。

**示例 4.4 一个简单的 IPv6 扫描**

```screen
# nmap -6 -sV www.eurov6.org

Starting Nmap ( https://nmap.org )
Nmap scan report for ns1.euro6ix.com (2001:800:40:2a03::3)
Not shown: 996 closed ports
PORT   STATE SERVICE VERSION
21/tcp open  ftp     Pure-FTPd
22/tcp open  ssh     OpenSSH 3.5p1 (protocol 2.0)
53/tcp open  domain  ISC BIND 9.2.1
80/tcp open  http    Apache httpd

Nmap done: 1 IP address (1 host up) scanned in 56.78 seconds
```

虽然 IPv6 尚未席卷全球，但它在一些国家得到了大量使用，并且大多数现代操作系统都支持它。要使用 Nmap 进行 IPv6 扫描，扫描的源和目标都必须配置 IPv6。如果您的 ISP（像大多数 ISP 一样）没有为您分配 IPv6 地址，则广泛提供免费的隧道代理，并且它们可以与 Nmap 很好地配合使用。我使用 [http://www.tunnelbroker.net](http://www.tunnelbroker.net) 上的免费 IPv6 隧道代理服务。其他隧道代理在 [Wikipedia 上列出](http://en.wikipedia.org/wiki/List_of_IPv6_tunnel_brokers)。6to4 隧道是另一种流行的免费方法。

支持 IPv6 的系统并不总是使其 IPv4 和 IPv6 防火墙规则保持同步。[“IPv6 攻击”](firewall-subversion.html#defeating-firewalls-ipv6)一节展示了一个实际示例，通过 IPv6 可以访问在 IPv4 中被过滤的端口。
