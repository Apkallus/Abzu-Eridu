## 命令行标志

虽然教程展示了执行 Nmap 端口扫描可以多么简单，但有几十个命令行标志可用，使系统更加强大和灵活。本节仅涵盖与端口扫描相关的选项，并且通常只描述这些选项中与端口扫描有关的功能。有关选项标志及其所有功能的完整列表，请参见第 15 章《Nmap 参考指南》。

### 选择扫描技术

考虑端口扫描时，首要问题之一是决定使用哪些技术。Nmap 提供了大约十几种此类方法，本节对它们进行了简要总结。完整介绍在下一章。一次只能使用一种扫描方法，但 UDP 扫描（`-sU`）可以与任意一种 TCP 扫描类型结合使用。作为记忆辅助，端口扫描类型选项的形式为 `-s<C>`，其中 `<C>` 是扫描名称中突出的字符，通常是第一个字母。唯一的例外是已弃用的 FTP 反弹扫描（`-b`）。默认情况下，Nmap 执行 SYN 扫描，但如果用户没有发送原始数据包所需的适当权限（在 Unix 上需要 root 访问权限）或指定了 IPv6 目标，则会替换为连接扫描。

**Nmap 支持的端口扫描方法**

- **TCP SYN（隐蔽）扫描（`-sS`）**  
  这绝对是迄今为止最流行的扫描类型，因为它是扫描最流行协议（TCP）端口的最快方式。它比连接扫描更隐蔽，并且适用于所有功能性 TCP 协议栈（与某些专用扫描如 FIN 扫描不同）。

- **TCP 连接扫描（`-sT`）**  
  连接扫描使用同名的系统调用来扫描机器，而不是像大多数其他方法那样依赖原始数据包。它通常由非特权 Unix 用户和针对 IPv6 目标使用，因为在这些情况下 SYN 扫描不起作用。

- **UDP 扫描（`-sU`）**  
  不要忘记 UDP 端口——它们也提供了大量的安全漏洞。

- **TCP FIN、NULL 和 Xmas 扫描（`-sF`、`-sN`、`-sX`）**  
  这些专用扫描类型擅长绕过防火墙以探索其背后的系统。不幸的是，它们依赖于某些系统（特别是 Windows 变种）不表现出的目标行为。

- **TCP ACK 扫描（`-sA`）**  
  ACK 扫描通常用于映射防火墙规则集。特别是，它有助于理解防火墙规则是否是有状态的。缺点是无法区分开放端口和关闭端口。

- **TCP 窗口扫描（`-sW`）**  
  窗口扫描类似于 ACK 扫描，但它能够针对某些机器检测开放端口与关闭端口。

- **TCP Maimon 扫描（`-sM`）**  
  这种晦涩的防火墙规避扫描类型类似于 FIN 扫描，但也包含 ACK 标志。这使其能够通过更多的数据包过滤防火墙，缺点是它能够生效的系统比 FIN 扫描还要少。

- **TCP 空闲扫描（`-sI <zombie host>`）**  
  空闲扫描是所有扫描类型中最隐蔽的，有时可以利用受信任的 IP 地址关系。不幸的是，它也很慢且复杂。

- **IP 协议扫描（`-sO`）**  
  协议扫描确定目标机器支持哪些 IP 协议（TCP、ICMP、IGMP 等）。从技术上讲，这不是端口扫描，因为它循环遍历 IP 协议号而不是 TCP 或 UDP 端口号。但它仍然使用 `-p` 选项来选择扫描的协议号，以正常的端口表格式报告结果，甚至使用与真正端口扫描方法相同的底层扫描引擎。因此它与端口扫描足够接近，属于这里。

- **TCP FTP 反弹扫描（`-b <FTP bounce proxy>`）**  
  这种已弃用的扫描类型欺骗 FTP 服务器代理执行端口扫描。大多数 FTP 服务器现在已经打了补丁来防止这种情况，但在它起作用时，它是穿过限制性防火墙的好方法。

### 选择要扫描的端口

Nmap 的端口注册文件（`nmap-services`）包含关于每个 TCP 或 UDP 端口被发现开放的频率的经验数据。这些数据是通过扫描数千万个互联网地址，然后将这些结果与大型企业贡献的内部扫描数据相结合而收集的。默认情况下，Nmap 扫描每个被要求扫描的协议中最流行的 1,000 个端口。或者，您可以指定 `-F`（快速）选项来仅扫描每个协议中最常见的 100 个端口，或者使用 `--top-ports` 来指定任意数量的端口进行扫描。

当这些预设端口集都不适合您的需求时，可以使用 `-p` 选项在命令行上指定任意的端口号列表。`-p` 选项的语法可能很复杂，最好用示例来说明。

**使用 `-p` 选项选择端口的示例**

- `-p 22`：扫描单个端口（此处为端口 22），只需将该数字作为 `-p` 参数。
- `-p ssh`：可以使用端口名称而不是数字。注意一个名称可能匹配多个端口。
- `-p 22,25,80`：多个端口可以用逗号分隔。注意没有指定协议，因此这些端口号将用于命令行上指定的任何扫描方法。如果指定了 TCP 扫描（如 SYN 扫描 `-sS`），则扫描 TCP 端口 22、25 和 80。它们分别对应服务 SSH、SMTP 和 HTTP。如果选择了 UDP 扫描（`-sU`），则扫描这三个 UDP 端口。如果两者都指定，则每个协议扫描这三个端口，总共六个扫描端口。对于 IP 协议扫描（`-sO`），则扫描这三个 IP 协议（对应 XNS IDP、Leaf-1 和 ISO-IP）。
- `-p80-85,443,8000-8005,8080-8085`：端口范围可以通过用连字符分隔起始和结束端口来指定。多个范围或单个端口可以用逗号指定。此选项扫描端口 80、81、82、83、84、85、443、8000 等。根据端口号，该用户可能正在扫描 TCP 并寻找 Web 服务器。
- `-p-100,60000-`：可以省略范围的开始表示端口 1，或省略结束表示最后一个可能的端口（TCP 和 UDP 为 65535，协议扫描为 255）。此示例扫描端口 1 到 100，以及所有大于或等于 60,000 的端口。
- `-p-`：省略开始和结束数字以扫描整个范围（不包括零）。
- `-pT:21,23,110,U:53,111,137,161`：可以通过在列表前加上 T:（用于 TCP）或 U: 来分别指定 TCP 和 UDP 端口列表。此示例扫描三个 TCP 端口（FTP、Telnet 和 POP3）和四个 UDP 服务（DNS、rpcbind、NetBIOS 和 SNMP）。只有同时告诉 Nmap 进行 UDP 扫描（`-sU`）和一种 TCP 扫描方法（如 `-sS`、`-sA` 或 `-sF`）时，同时指定 TCP 和 UDP 端口才有意义。
- `-p http*`：可以使用通配符匹配名称相似的端口。此表达式匹配八个端口号，包括 http（80）、http-mgmt（280）、https（443）和 http-proxy（8080）。根据您的命令 shell，您可能需要转义星号，以免被当作文件名通配符处理。
- `-p 1-1023,[1024-]`：将范围括在方括号中，意味着仅当这些端口号在 `nmap-services` 中注册时才进行扫描。在此示例中，扫描所有保留端口（1–1,023），加上在 `nmap-services` 中注册的所有更高端口。这是在 `nmap-services` 被增加开放端口频率数据以进行更精确选择之前的 Nmap 默认行为。

### 时序相关选项

端口扫描通常是 Nmap 扫描中最耗时的部分（还可能包括操作系统检测、版本检测和 NSE 脚本）。虽然 Nmap 默认尝试快速高效，但手动优化通常会有所帮助。Nmap 提供了几十个选项来调整扫描强度和速度以满足您的确切需求。本节列出了优化端口扫描时间最重要的选项。接受时间值的选项默认以秒为单位，或者您可以在值后附加 `ms`（毫秒）、`s`（秒）、`m`（分钟）或 `h`（小时）。有关这些选项的任何进一步细节，请参见“时序与性能”一节。关于提高 Nmap 性能的示例和最佳实践的更详尽处理，请参见第 6 章《优化 Nmap 性能》。

**顶级端口扫描性能选项**

- **`-T0` 到 `-T5`**：这些时序模板影响许多变量，提供了一种从非常慢（`-T0`）到极其激进（`-T5`）的调整 Nmap 整体速度的简单方法。时序模板可以与下面描述的更细粒度选项结合使用，并且最细粒度的选项优先。
- **`--min-rtt-timeout`、`--max-rtt-timeout`、`--initial-rtt-timeout`**：Nmap 等待端口扫描探测响应的最小、最大和初始时间量。
- **`--host-timeout`**：要求 Nmap 放弃扫描时间超过给定值的主机。
- **`--min-rate`、`--max-rate`**：分别设置 Nmap 每秒发送探测数据包数量的下限和上限。
- **`--max-retries`**：指定对单个端口进行端口扫描探测重传的最大次数。
- **`--min-hostgroup`、`--max-hostgroup`**：设置 Nmap 并行进行端口扫描的主机的最小和最大数量。
- **`--min-parallelism`、`--max-parallelism`**：限制 Nmap 可能未完成的端口扫描探测（跨所有并发扫描的主机）的最小或最大数量。
- **`--scan-delay`、`--max-scan-delay`**：要求 Nmap 在向任何单个主机发送探测之间至少等待给定的时间量。扫描延迟会随着 Nmap 检测到数据包丢失而增长，因此可以使用 `--max-scan-delay` 指定最大值。

### 输出格式和详细程度选项

Nmap 能够以其标准格式、简单的面向行的“grepable”格式或 XML 写入报告。这些报告通过 `-oN`（正常）、`-oG`（grepable）和 `-oX`（XML）选项启用。每个选项接受一个文件名，并且它们可以组合以同时输出多种格式。还有一些选项可用于增加输出详细程度。本节列出了与输出相关的最重要选项以及它们如何应用于端口扫描。有关这些选项的任何进一步细节，请参见“输出”一节。关于输出选项和格式的更详尽处理（包含许多示例），请参见第 13 章《Nmap 输出格式》。

**适用于端口扫描的顶级 Nmap 输出选项**

- **`-v`**：增加详细程度级别，使 Nmap 打印更多关于进行中扫描的信息。开放端口在发现时显示，并且当 Nmap 认为扫描将花费超过几分钟时提供完成时间估计。使用两次或更多次以获得更高的详细程度。
- **`-d`**：增加调试级别，使 Nmap 打印出其操作的详细信息，这对于跟踪错误或仅仅是理解其工作原理很有用。更高的级别会产生大量数据。使用一次该选项将调试级别设置为 1，每增加一个 `-d` 就递增一次。或者您可以在 `-d` 后跟所需级别，如 `-d5`。如果您没有看到足够的信息，请尝试更高的级别。最大有效级别是 9。如果您的屏幕被过多调试数据淹没，请降低级别。减少扫描强度（例如扫描的端口或目标数量以及使用的功能）也有助于仅隔离您想要的调试消息。
- **`--packet-trace`**：使 Nmap 打印发送或接收的每个数据包的摘要。这通常用于调试，也是新用户准确理解 Nmap 在幕后所做事情的一种有价值的方式。为了避免打印数千行，您可能希望指定有限数量的端口进行扫描，例如 `-p20-30`。
- **`-oN <filename>`（正常输出）**：以 Nmap 的正常格式将输出写入 `<filename>`。此格式大致与 Nmap 运行时打印的标准交互式输出相同。
- **`-oX <filename>`（XML 输出）**：以 Nmap 的 XML 格式将输出写入 `<filename>`。正常（人类可读）输出仍将打印到 stdout，除非您通过指定 `<filename>` 为 `-` 来要求将 XML 定向到那里。这是用于处理 Nmap 结果的脚本和程序的首选格式。
- **`-oG <filename>`（grepable 格式输出）**：以 Nmap 所谓的 grepable 格式将输出写入 `<filename>`。这种表格格式将每个主机的输出放在一行上，便于使用 grep 查找开放端口、某些操作系统、应用程序名称或其他数据。正常输出仍将打印到 stdout，除非您通过指定 `<filename>` 为 `-` 来要求将 grepable 输出定向到那里。虽然这种格式适用于使用简单的 grep 和 awk 命令行进行解析，但重要的脚本和程序应改用 XML 输出。XML 格式包含大量 grepable 格式无法容纳的信息，并且可扩展性使得在不破坏依赖它的工具的情况下更容易用新信息更新 XML。
- **`-oA <basename>`（输出到所有格式）**：为方便起见，您可以指定 `-oA <basename>` 将扫描结果同时存储在正常、XML 和 grepable 格式中。它们分别存储在 `<basename>.nmap`、`<basename>.xml` 和 `<basename>.gnmap` 中。与大多数程序一样，您可以在文件名前加上目录路径，例如 Unix 上的 `~/nmaplogs/foocorp/` 或 Windows 上的 `c:\hacking\sco`。
- **`--resume <filename>`**：通过指定在不幸的扫描期间创建的正常（`-oN`）或 grepable（`-oG`）输出文件，恢复中止的扫描。不要使用除 `--resume` 以外的任何选项，因为 Nmap 将使用输出文件中指定的选项。然后它解析文件，并在前一次 Nmap 执行停止时正在处理的主机上恢复扫描（并记录到文件）。
- **`--append-output`**：告诉 Nmap 将扫描结果附加到指定的任何输出文件（使用诸如 `-oN` 或 `-oX` 之类的参数），而不是覆盖它们。
- **`--open`**：仅显示具有开放端口的主机，并且仅显示这些主机的开放端口。在这里，“开放端口”是指任何可能开放的状态，包括 `open`、`open|filtered` 和 `unfiltered`。

### 防火墙和 IDS 规避选项

Nmap 提供了许多选项，用于不被检测地绕过 IDS 或规避防火墙规则。有关概述，请参见“防火墙/IDS 规避和欺骗”一节。关于防火墙和 IDS 规避技术以及实际示例的全面介绍，请参见第 10 章《检测和破坏防火墙与入侵检测系统》。

### 指定目标

要扫描单个主机（或几个主机），只需将它们的名称或 IP 地址添加到 Nmap 命令行的末尾。Nmap 还具有结构化语法，使扫描大型网络变得容易。您可以给 Nmap 一个列出目标的文件，甚至要求 Nmap 随机生成它们。所有这些都在“指定目标主机和网络”一节中描述。

### 其他选项

以下是一些非常方便的选项，尽管它们不属于特定类别。描述集中在每个选项与端口扫描的关系上。有关每个选项的更全面覆盖，请参见第 15 章《Nmap 参考指南》。

- **`-6`**：要求 Nmap 使用 IPv6 协议扫描目标。此过程在“IPv6 扫描（`-6`）”一节中描述。
- **`-r`**：Nmap 默认随机化端口扫描顺序以使检测稍微困难一些。`-r` 选项使端口按数字顺序扫描。
- **`-Pn`**：告诉 Nmap 跳过 ping 测试，直接扫描提供的每个目标主机。控制主机发现的其他选项在第 3 章《主机发现（“Ping 扫描”）》中描述。
- **`--reason`**：向有趣的端口表添加一列，描述 Nmap 为什么将端口分类为那样。
