## 快速端口扫描教程

我开发 Nmap 的目标之一是保持最常见用法的简单性，同时保留自定义和高级扫描的灵活性。这是通过命令行界面实现的：提供几十个选项，但在未指定时选择合理的默认值。新手可以使用像 `nmap <target>` 这样简单的命令开始。与此同时，高级用户有时会指定太多选项，以至于终端行都绕回来了。

命令输出也必须达到类似的平衡。最重要的结果应该即使是从未阅读过手册页的偶尔用户也能一眼看出。然而，输出应该足够全面和简洁，以满足每天针对数千台机器运行 Nmap 的专业渗透测试人员。聪明到阅读本书或 Nmap 源代码的用户将从对扫描仪的更强控制以及对 Nmap 输出真正含义的深入了解中受益。

本教程演示了一些常见的 Nmap 端口扫描场景并解释了输出。目标不是面面俱到，而只是让新用户足够熟悉，以便理解本章的其余部分。

最简单的 Nmap 命令就是单独的 `nmap`。这会打印常见 Nmap 选项和语法的速查表。一个更有趣的命令是 `nmap <target>`，它执行以下操作：

1. 使用 DNS 将 `<target>` 从主机名转换为 IPv4 地址。如果指定了 IP 地址而不是主机名，则跳过此查找。
2. 对主机进行 ping 测试，默认使用一个 ICMP 回显请求数据包和一个到端口 80 的 TCP ACK 数据包，以确定它是否在线并运行。如果不在线，Nmap 会报告该事实并退出。我可以指定 `-Pn` 来跳过此测试。参见第 3 章《主机发现（“Ping 扫描”）》.
3. 使用反向 DNS 查询将目标 IP 地址转换回名称。由于 DNS 的工作方式，反向名称可能与命令行上指定的 `<target>` 不同。可以使用 `-n` 选项跳过此查询，以提高速度和隐蔽性。
4. 对 `nmap-services` 中列出的最流行的 1,000 个端口启动 TCP 端口扫描。通常使用 SYN 隐蔽扫描，但对于缺乏发送原始数据包所需权限的非 root Unix 用户，会替代使用连接扫描。
5. 以正常的人类可读格式将结果打印到标准输出，然后退出。可以指定其他输出格式和位置（文件），如第 13 章《Nmap 输出格式》所述。示例 4.2 显示了当使用 scanme.nmap.org 作为 `<target>` 时的结果。

**示例 4.2 简单扫描：nmap scanme.nmap.org**

```screen
# nmap scanme.nmap.org

Starting Nmap ( https://nmap.org )
Nmap scan report for scanme.nmap.org (64.13.134.52)
Not shown: 994 filtered ports
PORT    STATE  SERVICE
22/tcp  open   ssh
25/tcp  closed smtp
53/tcp  open   domain
70/tcp  closed gopher
80/tcp  open   http
113/tcp closed auth

Nmap done: 1 IP address (1 host up) scanned in 4.99 seconds
```

示例 4.2 中的第一行输出简单地给出了下载 Nmap 的 URL（[https://nmap.org](https://nmap.org)）。通常还会提供 Nmap 启动时间和版本号，但为了保持一致性和避免换行，本书中通常删除了这些内容。

下一行提供了目标 IP 地址（此处为 IPv4），以及反向 DNS 名称（也称为 PTR 记录）（如果可用）。Nmap 承诺显示“有趣的端口”，尽管所有扫描的端口都会被说明。那些因为开放或处于该主机罕见状态而被认为最有趣的端口会逐条列出。当许多端口处于单一的非开放状态时，它们被视为默认状态，并汇总到一行中，以避免用数千个无趣的条目稀释结果。在本例中，Nmap 指出 994 个端口被过滤。

接下来是有趣端口表，提供了关键的扫描结果。列根据使用的选项而变化，但在本例中，每个端口都提供了端口号与协议、状态和服务协议。这里的服务只是通过查找 `nmap-services` 文件中的端口来猜测的。如果任何端口在该文件中没有注册名称，服务将被列为 `unknown`。其中三个端口是开放的，三个是关闭的。

最后，Nmap 在退出前报告一些基本的计时统计信息。这些统计信息是指定的目标数量、ping 扫描发现在线的主机数量以及所花费的总时间。

虽然这个简单的命令通常就足够了，但高级用户往往会走得更远。在示例 4.3 中，使用四个选项修改了扫描。`-p0-` 要求 Nmap 扫描每一个可能的 TCP 端口，`-v` 要求 Nmap 提供详细输出，`-A` 启用积极测试，如远程操作系统检测、服务/版本检测和 Nmap 脚本引擎（NSE）。最后，`-T4` 启用更积极的时序策略以加快扫描速度。

在示例 4.3 中，Nmap 当然提供了所要求的详细输出！幸运的是，额外的输出很容易理解。前 13 行新内容是运行时信息，让用户在她满怀期待地盯着终端、希望得到好消息时知道发生了什么。什么是好消息取决于她是一个需要解决问题的系统管理员，一个需要报告一些问题的渗透测试人员，还是一个试图利用它们的黑帽破解者。为简洁起见，删除了大约十几行类似的输出。“发现开放端口”行提供了开放端口的实时通知，这样她甚至可以在扫描完成之前就开始攻击它们。“扫描计时”行提供了完成时间估计，这样她就知道是继续盯着屏幕还是去吃午饭。由于网络条件（延迟、拥塞、带宽等）和数据包过滤规则变化很大，相同的扫描选项可能针对一台主机需要 30 秒完成，而针对另一台需要 45 分钟。如果您想在扫描时获取当前时间估计，只需按回车键。

端口表没有显示新端口。所有额外扫描的端口都处于过滤状态，使过滤端口总数从 994 增加到 65,530。虽然没有新的逐条列出的端口，但条目发生了变化。新的 `VERSION` 列提供了监听服务的应用程序名称和版本详细信息。这来自服务检测，这是 `-A` 选项启用的功能之一。服务检测的另一个特点是 `SERVICE` 列中的所有服务协议实际上都已被验证。在之前的扫描中，它们是基于 `nmap-services` 端口号查找的相对不可靠的启发式方法。这次表查找碰巧是正确的，但并不总是如此。

`-A` 添加的另一个功能是 Nmap 脚本引擎（NSE），这在第 9 章《Nmap 脚本引擎》中深入讨论。这里只显示了一个脚本 `HTML title`。还有几十个其他脚本存在，但没有一个为此机器找到有用的输出。traceroute 结果也是由 `-A` 添加的。这个选项比大多数 traceroute 程序更高效、更强大，因为探测是并行执行的，而且 Nmap 使用扫描结果来确定有利的探测类型（本例中是发往端口 80 的 TCP 数据包）。

其余的新行大多来自操作系统检测（也由 `-A` 启用），这将在第 8 章《远程操作系统检测》中深入讨论。最后一行显示所有这些额外信息都是有代价的——扫描花费的时间几乎是示例 4.2“简单扫描：nmap scanme.nmap.org”的 100 倍（477 秒对比 5 秒）。
