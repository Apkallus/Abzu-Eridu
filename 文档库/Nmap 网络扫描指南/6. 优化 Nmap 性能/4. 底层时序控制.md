## 底层时序控制

Nmap 提供了许多细粒度选项来控制扫描速度。大多数人使用这些选项来加速 Nmap，但它们也可用于减慢 Nmap 速度。人们这样做是为了规避 IDS 系统、减少网络负载，甚至在网络状况极差以至于 Nmap 的保守默认值都过于激进时提高准确性。

[表 6.2](#performance-tbl-lowlevel-timing) 按功能列出了每个底层时序控制选项。有关每个选项的详细使用信息，请阅读 [“时序与性能”](man-performance.html) 一节。假设读者已经熟悉 [“扫描代码与算法”](port-scanning-algorithms.html) 一节中描述的 Nmap 扫描算法。

**表 6.2 按功能划分的底层时序控制**

| 功能 | 选项 |
| :--- | :--- |
| 主机组（并发扫描的主机批次）大小 | `--min-hostgroup`、`--max-hostgroup` |
| 并行发起的探测数量 | `--min-parallelism`、`--max-parallelism` |
| 探测超时值 | `--min-rtt-timeout`、`--max-rtt-timeout`、`--initial-rtt-timeout` |
| 允许的最大探测重传次数 | `--max-retries` |
| 放弃整个主机前的最大时间 | `--host-timeout` |
| 对单个主机每次探测之间插入的延迟控制 | `--scan-delay`、`--max-scan-delay` |
| 每秒发送探测数据包的速率 | `--min-rate`、`--max-rate` |
| 克服目标主机的 RST 数据包响应速率限制 | `--defeat-rst-ratelimit` |
