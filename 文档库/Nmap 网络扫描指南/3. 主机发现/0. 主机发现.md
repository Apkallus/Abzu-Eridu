# 第 3 章 主机发现（“Ping 扫描”）

## 引言

在任何网络侦察任务中，最初几步之一就是将（有时是庞大的）IP 地址范围缩小为一份活跃或有趣的主机列表。扫描每一个 IP 地址的每一个端口既缓慢又通常没有必要。当然，什么使主机变得“有趣”很大程度上取决于扫描目的。网络管理员可能只关心运行特定服务的主机，而安全审计员可能关心每一个拥有 IP 地址的设备。管理员可能仅使用 ICMP ping 就能在内部网络上定位主机，而外部渗透测试人员可能会使用几十种不同的探测来尝试绕过防火墙限制。

由于主机发现的需求多种多样，Nmap 提供了丰富的选项来自定义所使用的技术。尽管名为 “ping 扫描”，它远远超出了普通 `ping` 工具所关联的简单 ICMP 回显请求数据包。用户可以通过列表扫描（`-sL`）或完全禁用 ping（`-Pn`）来跳过 ping 步骤，也可以使用任意组合的多端口 TCP SYN/ACK、UDP 和 ICMP 探测来与网络交互。这些探测的目的是引出响应，证明某个 IP 地址确实处于活跃状态（正在被主机或网络设备使用）。在许多网络上，任何时候只有一小部分 IP 地址是活跃的。这在私有地址空间（如 10.0.0.0/8）中尤其常见。该网络拥有 1680 万个 IP，但我见过一些公司使用它却只有不到一千台机器。主机发现可以在稀疏的 IP 地址海洋中找到这些机器。

本章首先讨论 Nmap ping 扫描的整体工作方式以及高层控制选项。然后介绍具体技术，包括它们的工作原理以及每种技术最适合的场景。Nmap 提供了许多 ping 技术，因为通常需要精心构造的组合才能穿过到达目标网络的一系列防火墙和路由器过滤器。接下来讨论有效的整体 ping 扫描策略，最后从底层角度介绍所使用的算法。
