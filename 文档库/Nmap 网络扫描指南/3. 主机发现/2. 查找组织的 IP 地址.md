## 查找组织的 IP 地址

Nmap 自动化了网络扫描的许多方面，但您仍然需要告诉它要扫描哪些网络。我猜您可以指定 `-iR` 并希望 Nmap 随机命中您目标公司，或者尝试蛮力方法，指定 `0.0.0.0/0` 来扫描整个互联网。但这两种选项都可能需要数月或数年，并且可能给您带来麻烦。因此，在扫描目标网络块之前仔细研究它们非常重要。即使您正在进行合法的渗透测试，并且客户给了您他们的网络块列表，双重检查也很重要。客户有时拥有过时的记录，或者只是写错了。如果您意外闯入错误的公司，客户签署的授权书也无济于事。

在许多情况下，您最初只有一个公司的域名。本节演示了一些最常见且有效的方法，将其转换为目标公司拥有、运营或关联的网络块列表。演示了典型的 Linux 命令行工具，但其他平台也有类似的工具。

在 2006 年的 ShmooCon 会议上，有个人走过来向我抱怨，Nmap 文档中指定了许多扫描 `target.com` 的示例。他指出 ICANN 已为此目的保留了域名 `example.com`，并敦促我相应地修订手册。虽然他在技术上是正确的，但这是一件很奇怪的事情。当他递给我他的名片时，他的动机就变得清晰了：

### DNS 技巧

DNS 的主要用途是将域名解析为 IP 地址，因此它是一个合乎逻辑的起点。在示例 3.1 中，我使用 Linux 的 `host` 命令查询一些常见的 DNS 记录类型。

**示例 3.1 使用 host 命令查询常见的 DNS 记录类型**

```screen
> host -t ns target.com
target.com name server ns4.target.com.
target.com name server ns3.target.com.
target.com name server ns1-auth.sprintlink.net.
target.com name server ns2-auth.sprintlink.net.
target.com name server ns3-auth.sprintlink.net.
> host -t a target.com
target.com has address 161.225.130.163
target.com has address 161.225.136.0
> host -t aaaa target.com
target.com has no AAAA record
> host -t mx target.com
target.com mail is handled by 50 smtp02.target.com.
target.com mail is handled by 5 smtp01.target.com.
> host -t soa target.com
target.com has SOA record extdns02.target.com. hostmaster.target.com.
```

接下来，我解析上述主机名的 IP 地址（再次使用 host），并尝试一些常见的子域名，例如 `www.target.com` 和 `ftp.target.com`。从像 `ns3.target.com` 和 `smtp01.target.com` 这样的名称开始，我尝试更改数字以找到新的机器。所有这些让我得到了以下 target.com 名称和地址：

**表 3.1 首次列出 target.com 的 IP**

| Hostname | IP Addresses |
| :--- | :--- |
| ns3.target.com | 161.225.130.130 |
| ns4.target.com | 161.225.136.136 |
| ns5.target.com | 161.225.130.150 |
| target.com | 161.225.136.0, 161.225.130.163 |
| smtp01.target.com | 161.225.140.120 |

虽然可以用这种方式生成大量的主机名列表，但主机名的最大来源来自区域传输。现在大多数 DNS 服务器拒绝区域传输请求，但值得一试，因为许多服务器仍然允许。请确保尝试您通过域名 NS 记录和扫描公司 IP 范围找到的每个 DNS 服务器。到目前为止，我们已经找到了七个 Target 名称服务器：`ns3.target.com`、`ns4.target.com`、`ns5.target.com`、`ns1-auth.sprintlink.net`、`ns2-auth.sprintlink.net`、`ns3-auth.sprintlink.net` 和 `extdns02.target.com`。不幸的是，所有这些服务器要么拒绝传输，要么不支持区域传输所需的 TCP DNS 连接。示例 3.2 显示了使用常用工具 `dig` 对 `target.com` 进行区域传输尝试失败，接着是对一个无关组织（`cpsr.org`）成功尝试。

### 向 IP 注册机构进行 Whois 查询

在发现一组初始“种子”IP 后，必须对它们进行研究，以确保它们属于您预期的公司，并确定它们属于哪些网络块。一个小公司可能只有 1-16 个 IP 地址的小分配，而较大的公司通常有数千个。这些信息保存在区域数据库中，例如北美的 ARIN（美国 Internet 号码注册机构）以及欧洲和中东的 RIPE。现代的 whois 工具接受一个 IP 地址，并自动查询适当的注册机构。

中小型公司通常不会有像 ARIN 这样的机构分配的 IP 空间。相反，它们从 ISP 获得委派的网络块。有时您可以从 IP 查询中获得 ISP 信息。这通常会让您得到一个大的网络块，而您不知道其中哪一部分分配给了您的目标。幸运的是，许多 ISP 现在使用共享 Whois（SWIP）或转介 Whois（RWhois）将客户范围再次委派。如果 ISP 这样做了，您就可以知道客户的确切网络块大小。

之前为 target.com 发现的一个 IP 地址是 `161.225.130.163`。示例 3.5 演示了一个 whois 查询（自动定向到 ARIN），以确定此 IP 的所有者和 IP 分配信息。

毫不奇怪，Target 拥有一个巨大的 B 类网络块，覆盖从 161.225.0.0 到 161.225.255.255 的所有 65,536 个 IP。由于 OrgName 是 Target，这不是我们看到来自其 ISP 结果的情况。

下一步是类似地查找所有先前发现的不在此范围内的 IP。然后，您可以开始更高级的查询。命令 `whois -h whois.arin.net \?` 给出 ARIN 查询语法。如果能够搜索匹配给定地址、OrgID 或 OrgTechEmail 的所有网络块会很好，但 IP 注册机构通常不允许这样。不过，许多其他有用的查询是允许的。例如，`whois -h whois.arin.net @target.com` 显示所有电子邮件地址为 target.com 的 ARIN 联系人。查询 `whois -h whois.arin.net "n target*"` 显示所有以 `target` 开头的网络块句柄。它不区分大小写。类似地，`whois -h whois.arin.net "o target*"` 显示所有以 `target` 开头的组织名称。您可以查找与每个条目关联的地址、电话号码和联系电子邮件，以确定它们是否属于您要扫描的公司。通常它们是恰好有类似名称的第三方。

### 互联网路由信息

互联网的核心路由协议是边界网关协议（BGP）。在扫描中型和大型组织时，BGP 路由表可以帮助您找到他们在世界各地的 IP 子网。例如，假设您要扫描属于 Microsoft Corporation 的 IP 地址。对 `microsoft.com` 的 DNS 查找提供了 IP 地址 `207.46.196.115`。上一节讨论的 whois 查询显示整个 207.46.0.0/16 块属于 Microsoft，地址在雷德蒙德的“One Microsoft Way”。这提供了 65,536 个 IP 地址进行扫描，但 BGP 表暴露了更多。

诸如 Microsoft 之类的实体出于路由目的被分配了自治系统（AS）编号。一个用于确定给定 IP 地址通告的 AS 编号的便捷工具可以在 `http://asn.cymru.com/` 找到。在此表单中输入 `207.46.0.0` 会提供 Microsoft 的 AS 编号 8075。接下来，我想找到所有路由到此 AS 的 IP 前缀。一个方便的工具可以在 `http://www.robtex.com/as/` 找到。在该页面输入 `AS8075` 并点击“Go”，会进入一个摘要屏幕，显示找到的 42 个前缀。这些前缀代表 339,456 个 IP 地址，可以通过单击“BGP”选项卡来枚举。

虽然从这样的预先打包的 Web 表单获取 BGP 信息很方便，但从实际路由器获取路由数据更有趣，并且可能允许更强大的自定义查询。一些组织提供这样的服务。例如，telnet 到 `route-views.routeviews.org` 或访问 `http://routeviews.org`。当然，这些服务提供对数据的只读访问。如果您需要操纵全局路由表作为接管互联网的邪恶计划的一部分，这超出了本书的范围。
