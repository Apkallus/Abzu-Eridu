## DNS 解析

Nmap 主机发现的主要焦点是确定网络上有哪些主机处于活动状态并能响应。这可以缩小目标范围，因为你无法入侵一个不存在的主机。但不要让发现就此止步。你不会仅仅因为某个人在呼吸就与他约会，选择网络上的机器进行渗透也同样需要特别小心。一个很好的信息源（关于网络主机，而非潜在约会对象）是 DNS，即域名系统。即使是对安全敏感的组织，也常常会分配泄露其系统功能的名称。常见的情况包括：无线接入点命名为 `wap` 或 `wireless`，防火墙命名为 `fw`、`firewall` 或 `fw-1`，以及包含尚未发布内容的开发 Web 服务器命名为 `dev`、`staging`、`www-int` 或 `beta`。位置或部门名称也经常被泄露，例如某公司芝加哥办公室的防火墙命名为 `fw.chi`。

默认情况下，Nmap 会对响应主机发现探测（即在线）的每个 IP 执行反向 DNS 解析。如果通过 `-Pn` 跳过了主机发现，则会对所有 IP 执行解析。Nmap 没有使用缓慢的标准 DNS 解析库，而是使用自定义的存根解析器，并行执行数十个请求。

虽然默认设置通常效果不错，但 Nmap 提供了四个选项来控制 DNS 解析。它们可以显著影响扫描速度和收集到的信息量。

- **`-n`（不进行 DNS 解析）**  
  告诉 Nmap 永远不要对其发现的活跃 IP 地址进行反向 DNS 解析。由于即使使用 Nmap 内置的并行存根解析器，DNS 也可能很慢，因此此选项可以减少扫描时间。

- **`-R`（对所有目标进行 DNS 解析）**  
  告诉 Nmap 始终对目标 IP 地址进行反向 DNS 解析。通常情况下，反向 DNS 仅针对有响应的（在线）主机执行。

- **`--system-dns`（使用系统 DNS 解析器）**  
  默认情况下，Nmap 通过直接向您主机上配置的名称服务器发送查询，然后侦听响应来解析 IP 地址。为提高性能，会并行执行许多请求（通常数十个）。指定此选项将改用系统解析器（通过 `getnameinfo` 调用一次一个 IP）。这种方式很慢，除非您发现 Nmap 并行解析器中的错误（如果发现请告诉我们），否则很少有用。系统解析器始终用于 IPv6 扫描。

- **`--dns-servers <server1>[,<server2>[, ...]]`（用于反向 DNS 查询的服务器）**  
  默认情况下，Nmap 从您的 `resolv.conf` 文件（Unix）或注册表（Win32）中确定用于反向 DNS 解析的 DNS 服务器。或者，您可以使用此选项指定备用服务器。如果您正在使用 `--system-dns` 或进行 IPv6 扫描，则此选项无效。使用多个 DNS 服务器通常更快，特别是当您为目标的 IP 空间选择权威服务器时。此选项还可以提高隐蔽性，因为您的请求可以通过互联网上几乎任何递归 DNS 服务器进行转发。

  在扫描私有网络时，此选项也很有用。有时只有少数名称服务器提供正确的反向 DNS 信息，您甚至可能不知道它们在哪里。您可以扫描网络的 53 端口（可能使用版本检测），然后使用 `--dns-servers` 逐一指定每个名称服务器进行 Nmap 列表扫描（`-sL`），直到找到可用的服务器。
