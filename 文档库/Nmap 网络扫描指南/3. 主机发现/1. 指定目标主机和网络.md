## 指定目标主机和网络

Nmap 命令行中不属于选项（或选项参数）的所有内容都将被视为目标主机规范。最简单的情况是指定一个目标 IP 地址或主机名进行扫描。

有时您希望扫描一个相邻主机的整个网络。为此，Nmap 支持 CIDR 风格的寻址。您可以在 IPv4 地址或主机名后附加 `/<numbits>`，Nmap 将扫描所有前 `<numbits>` 位与给定参考 IP 或主机名相同的主机。例如，`192.168.10.0/24` 将扫描 192.168.10.0 到 192.168.10.255（含）之间的 256 个主机。`192.168.10.40/24` 将扫描完全相同的主机。假设主机 scanme.nmap.org 的 IP 地址是 64.13.134.52，那么规范 `scanme.nmap.org/16` 将扫描 64.13.0.0 到 64.13.255.255 之间的 65,536 个 IP 地址。允许的最小值是 `/0`，它将扫描整个互联网。最大值是 `/32`，由于所有地址位都已固定，因此只扫描指定主机或 IP 地址。

CIDR 表示法简短，但并不总是足够灵活。例如，您可能想扫描 192.168.0.0/16，但跳过任何以 .0 或 .255 结尾的 IP，因为它们可能被用作子网网络地址和广播地址。Nmap 通过八位组范围寻址支持此功能。您可以不指定普通的 IP 地址，而是为每个八位组指定一个以逗号分隔的数字或范围列表。例如，`192.168.0-255.1-254` 将跳过范围中以 .0 或 .255 结尾的所有地址，而 `192.168.3-5,7.1` 将扫描四个地址：192.168.3.1、192.168.4.1、192.168.5.1 和 192.168.7.1。范围的任何一侧都可以省略；左侧默认值为 0，右侧默认值为 255。单独使用 `-` 等同于 `0-255`，但请记住在第一个八位组中使用 `0-`，以使目标规范看起来不像命令行选项。范围不必限制在最后的八位组：规范 `0-255.0-255.13.37` 将对所有以 13.37 结尾的 IP 地址执行全网扫描。这种广泛的采样对于互联网调查和研究很有用。

IPv6 地址只能通过其完整的 IPv6 地址或主机名来指定。IPv6 不支持 CIDR 和八位组范围，因为它们很少有用。

Nmap 在命令行上接受多个主机规范，并且它们不必是同一类型。命令 `nmap scanme.nmap.org 192.168.0.0/8 10.0.0,1,3-7.-` 会执行您所期望的操作。

### 从列表输入（`-iL`）

在命令行上传递一大串主机列表通常很麻烦，但这是一个常见需求。例如，您的 DHCP 服务器可能导出了一个包含 10,000 个当前租约的列表，您希望扫描这些租约。或者，您可能想要扫描除这些地址之外的所有 IP 地址，以定位使用未经授权的静态 IP 地址的主机。只需生成要扫描的主机列表，并将该文件名作为 `-iL` 选项的参数传递给 Nmap。条目可以是 Nmap 在命令行上接受的任何格式（IP 地址、主机名、CIDR、IPv6 或八位组范围）。每个条目必须由一个或多个空格、制表符或换行符分隔。如果您希望 Nmap 从标准输入而不是实际文件读取主机，可以将连字符（`-`）指定为文件名。

### 随机选择目标（`-iR <numtargets>`）

对于互联网范围内的调查和其他研究，您可能希望随机选择目标。这可以通过 `-iR` 选项完成，该选项的参数是要生成的 IP 数量。Nmap 会自动跳过某些不良 IP，例如私有地址、多播地址或未分配的地址范围。可以指定参数 `0` 来进行永不停止的扫描。请记住，某些网络管理员对未经授权的网络扫描感到不满。在使用 `-iR` 之前，请仔细阅读“法律问题”一节。

如果您在一个下雨的下午感到非常无聊，可以尝试命令 `nmap -sS -PS80 -iR 0 -p 80` 来随机查找可浏览的 Web 服务器。

### 排除目标（`--exclude`, `--excludefile <filename>`）

通常会有一些机器在任何情况下您都不希望扫描。这些机器可能至关重要，您不愿意承担任何不良反应的风险。即使 Nmap 扫描与故障无关，您也可能因为偶发的停机而被指责。或者，某些旧硬件在扫描时已知会崩溃，但您尚未修复或更换它。也许某些 IP 范围代表子公司、客户或合作伙伴，您无权扫描它们。顾问通常不希望自己的机器包含在客户网络的扫描中。无论出于何种原因，您都可以使用 `--exclude` 选项排除主机或整个网络。只需使用标准的 Nmap 语法，将排除的目标和网络块的逗号分隔列表传递给该选项。或者，您可以创建一个排除主机/网络的文件，并通过 `--excludefile` 选项传递给 Nmap。`--exclude` 选项不能与使用逗号的 IP 范围（如 `192.168.0.10,20,30`）混合使用，因为 `--exclude` 本身使用逗号。在这些情况下，请使用 `--excludefile`。

### 实际示例

虽然有些工具具有简单的界面，只允许列出一组主机或指定范围的起始和结束 IP 地址，但 Nmap 更强大、更灵活。然而，Nmap 也可能更难学习——扫描错误的 IP 地址有时会带来灾难性后果。幸运的是，Nmap 提供了列表扫描（`-sL` 选项）来进行预演。只需执行 `nmap -sL -n <targets>`，即可在实际扫描前查看哪些 IP 会被扫描。

示例可能是教授 Nmap 主机规范语法最有效的方式。本节提供一些示例，从最简单的开始。

- **`nmap scanme.nmap.org`**、**`nmap scanme.nmap.org/32`**、**`nmap 64.13.134.52`**  
  这三个命令的作用相同，假设 scanme.nmap.org 解析为 64.13.134.52。它们扫描那一个 IP 然后退出。

- **`nmap scanme.nmap.org/24`**、**`nmap 64.13.134.52/24`**、**`nmap 64.13.134.-`**、**`nmap 64.13.134.0-255`**  
  这四个命令都要求 Nmap 扫描从 64.13.134.0 到 64.13.134.255 之间的 256 个 IP 地址。换句话说，它们要求扫描 scanme.nmap.org 周围的 C 类大小地址空间。

- **`nmap 64.13.134.52/24 --exclude scanme.nmap.org,insecure.org`**  
  告诉 Nmap 扫描 64.13.134.52 周围的 C 类地址范围，但如果 scanme.nmap.org 和 insecure.org 位于该地址范围内，则跳过它们。

- **`nmap 10.0.0.0/8 --exclude 10.6.0.0/16,ultra-sensitive-host.company.com`**  
  告诉 Nmap 扫描整个私有 10 范围，但必须跳过所有以 10.6 开头的地址以及 ultra-sensitive-host.company.com。

- **`egrep '^lease' /var/lib/dhcp/dhcpd.leases | awk '{print $2}' | nmap -iL -`**  
  获取已分配的 DHCP IP 地址列表，并将其直接输入 Nmap 进行扫描。注意，向 `-iL` 传递了一个连字符，以从标准输入读取。

- **`nmap -6 2001:800:40:2a03::3`**  
  扫描 IPv6 主机地址 2001:800:40:2a03::3。
