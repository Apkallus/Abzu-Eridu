## 用法与示例

在深入探讨版本检测实现的技术细节之前，这里有一些示例演示了它的用法和功能。要启用版本检测，只需在你通常使用的任何 Nmap 标志中添加 `-sV`。或者使用 `-A` 选项，它会打开版本检测以及后面的其他高级和积极功能。就这么简单，如[示例 7.2](#ex-version-detection-scan2) 所示。

**示例 7.2 对 www.microsoft.com 的版本检测**

```screen
# nmap -A -T4 -F www.microsoft.com

Starting Nmap ( https://nmap.org )
Nmap scan report for 80.67.68.30
(The 1208 ports scanned but not shown below are in state: closed)
PORT    STATE    SERVICE     VERSION
22/tcp  open     ssh         Akamai-I SSH (protocol 1.5)
80/tcp  open     http        AkamaiGHost (Akamai's HTTP Acceleration service)
443/tcp open     ssl/http    AkamaiGHost (Akamai's HTTP Acceleration service)
Device type: general purpose
Running: Linux 2.1.X|2.2.X
OS details: Linux 2.1.19 - 2.2.25

Nmap finished: 1 IP address (1 host up) scanned in 19.223 seconds
```

上述扫描展示了几件事。首先，看到 www.Microsoft.Com 由 Akamai 的 Linux 机器提供服务是令人欣慰的。与本章更相关的是，端口 443 列出的服务是 `ssl/http`。这意味着服务检测首先发现该端口是 SSL，然后加载 OpenSSL，并通过 SSL 连接再次执行服务检测，发现在加密后面运行着一个 AkamaiGHost Web 服务器。回想一下，`-T4` 使 Nmap 运行更快（更积极的时序），而 `-F` 告诉 Nmap 只扫描 `nmap-services` 中注册的端口。

[示例 7.3](#ex-version-detection-scan3) 是一个更长且更多样化的示例。

**示例 7.3 复杂的版本检测**

```screen
# nmap -A -T4 localhost 

Starting Nmap ( https://nmap.org )
Nmap scan report for felix (127.0.0.1)
(The 1640 ports scanned but not shown below are in state: closed)
PORT     STATE SERVICE    VERSION
21/tcp   open  ftp        WU-FTPD wu-2.6.1-20
22/tcp   open  ssh        OpenSSH 3.1p1 (protocol 1.99)
53/tcp   open  domain     ISC BIND 9.2.1
79/tcp   open  finger     Linux fingerd
111/tcp  open  rpcbind    2 (rpc #100000)
443/tcp  open  ssl/http   Apache httpd 2.0.39 ((Unix) mod_perl/1.99_04-dev)
515/tcp  open  printer
631/tcp  open  ipp        CUPS 1.1
953/tcp  open  rndc?
5000/tcp open  ssl/ftp    WU-FTPD wu-2.6.1-20
5001/tcp open  ssl/ssh    OpenSSH 3.1p1 (protocol 1.99)
5002/tcp open  ssl/domain ISC BIND 9.2.1
5003/tcp open  ssl/finger Linux fingerd
6000/tcp open  X11        (access denied)
8000/tcp open  http-proxy Junkbuster webproxy
8080/tcp open  http       Apache httpd 2.0.39 ((Unix) mod_perl/1.99_04-dev)
8081/tcp open  http       Apache httpd 2.0.39 ((Unix) mod_perl/1.99_04-dev)
Device type: general purpose
Running: Linux 2.4.X|2.5.X
OS details: Linux Kernel 2.4.0 - 2.5.20

Nmap finished: 1 IP address (1 host up) scanned in 42.494 seconds
```

你可以在这里看到处理 RPC 服务的方式：使用暴力 RPC 扫描器来确定端口 111 是 rpcbind 版本 2。你还可以注意到端口 515 给出的服务是 `printer`，但版本字段为空。Nmap 通过探测确定了服务名称，但无法确定其他任何信息。另一方面，端口 953 给出的服务是 `rndc?`。问号告诉我们 Nmap 甚至无法通过探测确定服务名称。作为后备，提到 rndc 是因为该端口在 `nmap-services` 中注册为 rndc。不幸的是，Nmap 的任何探测都没有从 rndc 引发任何响应。如果它们有响应，Nmap 会打印一个服务指纹和一个提交 URL，以便在下一个版本中能够被识别。实际上，Nmap 需要一个特殊的探测。到你读到这时，可能已经有一个可用的探测了。[“社区贡献”](vscan-community.html) 一节提供了编写自己探测的详细信息。

另外值得注意的是，一些服务提供的信息远不止版本号。上面的示例包括 X11 是否允许连接、SSH 协议号以及 Apache 模块版本列表。有些 Apache 模块甚至不得不从输出中截断以适应该页面。

一些早期审阅者质疑通过 SSL 运行诸如 SSH 和 finger 等服务是否合理。这实际上只是使用 [stunnel](http://www.stunnel.org/) 做的一个趣味实验，部分是为了确保并行 SSL 扫描确实有效。
