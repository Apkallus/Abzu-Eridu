## 使用与示例

虽然 NSE 的实现为了效率而较为复杂，但使用起来却出奇地简单。只需指定 `-sC` 即可启用最常见的脚本。或者使用 `--script` 选项，通过提供类别、脚本文件名或包含要执行脚本的目录名来选择要执行的脚本。您可以通过 `--script-args` 和 `--script-args-file` 选项为某些脚本提供参数。`--script-help` 选项显示每个选中脚本的描述。剩下的两个选项 `--script-trace` 和 `--script-updatedb` 通常仅用于脚本调试和开发。脚本扫描也是 `-A`（积极扫描）选项的一部分。

脚本扫描通常与端口扫描结合进行，因为脚本是否运行取决于扫描发现的端口状态。使用 `-sn` 选项，可以在没有端口扫描的情况下运行脚本扫描，仅进行主机发现。此时只有主机脚本有资格运行。要运行既没有主机发现也没有端口扫描的脚本扫描，请将 `-Pn -sn` 选项与 `-sC` 或 `--script` 一起使用。每个主机都将被假定为在线，但仍然只运行主机脚本。此技术对于像 `whois-ip` 这样仅使用远程系统地址而不需要其在线状态的脚本非常有用。

脚本不在沙箱中运行，因此可能意外或恶意地损坏您的系统或侵犯您的隐私。除非您信任作者或自己仔细审查过脚本，否则永远不要运行第三方脚本。

### 脚本类别

NSE 脚本定义了它们所属的类别列表。当前定义的类别有：`auth`、`broadcast`、`brute`、`default`、`discovery`、`dos`、`exploit`、`external`、`fuzzer`、`intrusive`、`malware`、`safe`、`version` 和 `vuln`。类别名称不区分大小写。以下列表描述了每个类别。

- **`auth`**  
  这些脚本处理目标系统上的认证凭证（或绕过它们）。示例包括 `x11-access`、`ftp-anon` 和 `oracle-enum-users`。使用暴力攻击确定凭证的脚本被放置在 `brute` 类别中。

- **`broadcast`**  
  此类别中的脚本通常通过在本地网络上广播来发现未在命令行中列出的主机。使用 `newtargets` 脚本参数允许这些脚本自动将它们发现的主机添加到 Nmap 扫描队列中。

- **`brute`**  
  这些脚本使用暴力攻击来猜测远程服务器的认证凭证。Nmap 包含了针对数十种协议的暴力脚本，包括 `http-brute`、`oracle-brute`、`snmp-brute` 等。

- **`default`**  
  这些脚本是默认集合，在使用 `-sC` 或 `-A` 选项而不是用 `--script` 列出脚本时运行。此类别也可以像任何其他类别一样使用 `--script=default` 显式指定。在决定脚本是否应默认运行时，会考虑许多因素：
  - **速度**：默认扫描必须快速完成，这排除了暴力认证破解、Web 爬虫以及任何可能花费数分钟或数小时扫描单个服务的脚本。
  - **有用性**：默认扫描需要产生有价值且可操作的信息。如果连脚本作者都难以解释为什么普通的网络或安全专业人员会觉得输出有价值，那么该脚本不应默认运行。
  - **冗长性**：Nmap 输出用于多种目的，需要可读且简洁。经常产生页满输出的脚本不应添加到 `default` 类别。当没有重要信息可报告时，NSE 脚本（尤其是默认脚本）应该什么都不返回。
  - **可靠性**：许多脚本使用启发式和模糊签名匹配来得出关于目标主机或服务的结论。如果脚本经常出错，它就不属于 `default` 类别，以免混淆或误导普通用户。直接指定脚本或类别的用户通常更高级，可能知道脚本如何工作，或者至少知道在哪里找到其文档。
  - **侵入性**：某些脚本非常具有侵入性，因为它们会占用远程系统的大量资源，可能使系统或服务崩溃，或者可能被远程管理员视为攻击。脚本越具有侵入性，就越不适合 `default` 类别。默认脚本几乎也总是在 `safe` 类别中，尽管我们偶尔会在脚本仅具有轻微侵入性且在其他因素上得分较高时，默认允许 `intrusive` 脚本。
  - **隐私**：某些脚本，尤其是下面描述的 `external` 类别中的脚本，其性质决定了它们会向第三方泄露信息。例如，`whois` 脚本必须将目标 IP 地址泄露给区域 whois 注册机构。我们还考虑过（但决定不）添加检查目标 SSH 和 SSL 密钥指纹与互联网弱密钥数据库的脚本。脚本对隐私的侵犯越强，就越不适合包含在 `default` 类别中。
  
  我们对这些标准没有确切的阈值，其中许多是主观的。在决定是否将脚本提升到 `default` 类别时，会综合考虑所有这些因素。一些默认脚本包括 `identd-owners`（通过 identd 确定运行远程服务的用户名）、`http-auth`（获取需要身份验证的网站的身份验证方案和域）和 `ftp-anon`（测试 FTP 服务器是否允许匿名访问）。

- **`discovery`**  
  这些脚本试图通过查询公共注册机构、启用 SNMP 的设备、目录服务等来主动发现更多关于网络的信息。示例包括 `html-title`（获取网站根路径的标题）、`smb-enum-shares`（枚举 Windows 共享）和 `snmp-sysdescr`（通过 SNMP 提取系统详细信息）。

- **`dos`**  
  此类别中的脚本可能导致拒绝服务。有时这样做是为了测试对拒绝服务方法的脆弱性，但更常见的是，这是测试传统漏洞时不必要的副作用。这些测试有时会使易受攻击的服务崩溃。

- **`exploit`**  
  这些脚本旨在主动利用某些漏洞。示例包括 `jdwp-exec` 和 `http-shellshock`。

- **`external`**  
  此类别中的脚本可能将数据发送到第三方数据库或其他网络资源。一个例子是 `whois-ip`，它连接到 whois 服务器以了解目标的地址。第三方数据库的操作者有可能记录您发送给他们的任何内容，在许多情况下，这将包括您的 IP 地址和目标地址。大多数脚本的流量严格限于扫描计算机和客户端之间；任何不在此范围内的脚本都被归入此类别。

- **`fuzzer`**  
  此类别包含旨在向服务器软件发送意外或随机化字段的脚本。虽然这种技术可用于发现软件中未发现的错误和漏洞，但它既缓慢又消耗带宽。此类别中的一个脚本示例是 `dns-fuzz`，它会用稍微有缺陷的域名请求轰炸 DNS 服务器，直到服务器崩溃或用户指定的时间限制到期。

- **`intrusive`**  
  这些脚本不能被归类为 `safe`，因为风险太高，可能使目标系统崩溃、占用目标主机上的大量资源（如带宽或 CPU 时间），或者被目标的系统管理员视为恶意。例如 `http-open-proxy`（试图将目标服务器用作 HTTP 代理）和 `snmp-brute`（通过发送常见值如 `public`、`private` 和 `cisco` 来猜测设备的 SNMP 团体字符串）。除非脚本在特殊的 `version` 类别中，否则它应被归类为 `safe` 或 `intrusive`。

- **`malware`**  
  这些脚本测试目标平台是否感染了恶意软件或后门。示例包括 `smtp-strangeport`（监视在不寻常端口号上运行的 SMTP 服务器）和 `auth-spoof`（检测在收到查询之前就提供虚假答案的 identd 欺骗守护进程）。这两种行为通常都与恶意软件感染有关。

- **`safe`**  
  并非设计为崩溃服务、使用大量网络带宽或其他资源、或利用安全漏洞的脚本被归类为 `safe`。这些脚本不太可能冒犯远程管理员，尽管（与所有其他 Nmap 功能一样）我们不能保证它们永远不会引起不良反应。大多数执行常规网络发现。示例包括 `ssh-hostkey`（检索 SSH 主机密钥）和 `html-title`（从网页获取标题）。`version` 类别中的脚本不按安全性分类，但任何不在 `safe` 中的其他脚本都应放在 `intrusive` 中。

- **`version`**  
  此特殊类别中的脚本是版本检测功能的扩展，不能显式选择。仅当请求版本检测（`-sV`）时才选择运行它们。它们的输出不能与版本检测输出区分开，并且它们不产生服务或主机脚本结果。示例包括 `skypev2-version`、`pptp-version` 和 `iax2-version`。

- **`vuln`**  
  这些脚本检查特定的已知漏洞，通常仅在发现漏洞时报告结果。示例包括 `realvnc-auth-bypass` 和 `afp-path-vuln`。

### 脚本类型与阶段

NSE 支持四种类型的脚本，通过它们接受的目标类型和运行的扫描阶段来区分。单个脚本可能支持多种操作类型。

- **Prerule 脚本**  
  这些脚本在任何 Nmap 扫描阶段之前运行，因此 Nmap 尚未收集有关其目标的任何信息。它们对于不依赖于特定扫描目标的任务很有用，例如执行网络广播请求以查询 DHCP 和 DNS SD 服务器。其中一些脚本可以为 Nmap 生成新目标进行扫描（仅当您指定 `newtargets` NSE 参数时）。例如，`dns-zone-transfer` 可以通过区域传输请求获取域中的 IP 列表，然后自动将它们添加到 Nmap 的扫描目标列表中。Prerule 脚本可以通过包含 `prerule` 函数来识别（参见[“规则”](nse-script-format.html#nse-format-rules)一节）。

- **Host 脚本**  
  此阶段的脚本在 Nmap 的正常扫描过程中运行，在 Nmap 对目标主机执行了主机发现、端口扫描、版本检测和操作系统检测之后。此类脚本对每个匹配其 `hostrule` 函数的目标主机调用一次。示例包括 `whois-ip`（查找目标 IP 的所有权信息）和 `path-mtu`（尝试确定能够到达目标而无需分片的最大 IP 数据包大小）。

- **Service 脚本**  
  这些脚本针对目标主机上监听的特定服务运行。例如，Nmap 包含超过 15 个针对 Web 服务器的 http 服务脚本。如果一个主机在多个端口上运行 Web 服务器，这些脚本可能会运行多次（每个端口一次）。这是最常见的 Nmap 脚本类型，通过包含 `portrule` 函数来区分，该函数决定脚本应针对哪些检测到的服务运行。

- **Postrule 脚本**  
  这些脚本在 Nmap 扫描完所有目标后运行。它们对于格式化和呈现 Nmap 输出很有用。例如，`ssh-hostkey` 最出名的是它的服务（portrule）脚本，该脚本连接到 SSH 服务器，发现其公钥并打印出来。但它还包括一个 postrule，用于检查所有扫描主机之间的重复密钥，然后打印任何找到的重复密钥。postrule 脚本的另一个潜在用途是打印 Nmap 输出的反向索引——显示哪些主机运行特定服务，而不是仅列出每个主机上的服务。Postrule 脚本通过包含 `postrule` 函数来识别。

许多脚本可能既可以作为 prerule 也可以作为 postrule 脚本运行。在这些情况下，我们建议为了保持一致性而使用 prerule。

### 命令行参数

以下是特定于脚本扫描的五个命令行参数：

- **`-sC`**  
  使用默认脚本集执行脚本扫描。它等同于 `--script=default`。此 `default` 类别中的某些脚本被认为具有侵入性，未经许可不应针对目标网络运行。

- **`--script <filename>|<category>|<directory>/|<expression>[, ...]`**  
  使用逗号分隔的文件名、脚本类别和目录列表运行脚本扫描。列表中的每个元素也可以是一个布尔表达式，描述一组更复杂的脚本。每个元素首先被解释为表达式，然后解释为类别，最后解释为文件或目录名。特殊参数 `all` 使 Nmap 脚本数据库中的每个脚本都有资格运行。使用 `all` 参数应谨慎，因为 NSE 可能包含危险的脚本，包括漏洞利用、暴力认证破解和拒绝服务攻击。

  脚本表达式列表中的每个元素都可以用 `+` 字符作为前缀，以强制给定的脚本运行，无论其 `portrule` 或 `hostrule` 函数中的条件如何。这通常仅由高级用户在特殊情况下进行。例如，您可能希望对一组 MS SQL 服务器进行配置审查，其中一些运行在非标准端口上。与其通过运行广泛的版本检测（`-sV --version-all`）来减慢 Nmap 扫描速度，以便 Nmap 识别 `ms-sql` 服务，不如通过指定 `--script +ms-sql-config` 强制 `ms-sql-config` 脚本针对所有目标主机和端口运行。

  文件名和目录名可以是相对的或绝对的。绝对名称直接使用。相对路径按顺序在以下每个位置的 `scripts` 子目录中搜索，直到找到为止：`--datadir`、`$NMAPDIR`、`~/.nmap`（Windows 上不搜索）、`<APPDATA>\nmap`（仅限 Windows）、包含 `nmap` 可执行文件的目录、包含 `nmap` 可执行文件的目录后跟 `../share/nmap`（Windows 上不搜索）、`NMAPDATADIR`（Windows 上不搜索）、当前目录。

  当给出以 `/` 结尾的目录名时，Nmap 会加载目录中所有以 `.nse` 结尾的文件。所有其他文件被忽略，目录不会被递归搜索。当给出文件名时，它不必具有 `.nse` 扩展名；如果需要，将自动添加。

  有关 `--script` 选项的示例和完整说明，请参见[“脚本选择”](#nse-script-selection)一节。

  Nmap 脚本默认存储在 Nmap 数据目录的 `scripts` 子目录中（参见[第 14 章《理解与自定义 Nmap 数据文件》](data-files.html)）。为了提高效率，脚本在存储在 `scripts/script.db` 中的数据库中被索引，该数据库列出了每个脚本所属的一个或多个类别。参数 `all` 将执行 Nmap 脚本数据库中的所有脚本，但应谨慎使用，因为 Nmap 可能包含漏洞利用、拒绝服务攻击和其他危险脚本。

- **`--script-args <args>`**  
  为脚本提供参数。有关详细说明，请参见[“脚本参数”](#nse-args)一节。

- **`--script-args-file <filename>`**  
  此选项与 `--script-args` 相同，只是您在文件中传递参数而不是在命令行上。有关详细说明，请参见[“脚本参数”](#nse-args)一节。

- **`--script-help <filename>|<category>|<directory>|<expression>|all[,...]`**  
  显示关于脚本的帮助。对于匹配给定规范的每个脚本，Nmap 打印脚本名称、其类别及其描述。规范与 `--script` 接受的相同；例如，如果您想要关于 `ssl-enum-ciphers` 脚本的帮助，您将运行 `nmap --script-help ssl-enum-ciphers`。脚本帮助的示例如[示例 9.2“脚本帮助”](#nse-script-help)所示。

  **示例 9.2 脚本帮助**
  ```screen
  $ nmap --script-help "afp-* and discovery"

  Starting Nmap 7.40 ( https://nmap.org ) at 2017-04-21 14:15 UTC

  afp-ls
  Categories: discovery safe
  https://nmap.org/nsedoc/scripts/afp-ls.html
    Attempts to get useful information about files from AFP volumes.
    The output is intended to resemble the output of `ls`.

  afp-serverinfo
  Categories: default discovery safe
  https://nmap.org/nsedoc/scripts/afp-serverinfo.html
    Shows AFP server information. This information includes the server's
    hostname, IPv4 and IPv6 addresses, and hardware type (for example
    `Macmini` or `MacBookPro`).

  afp-showmount
  Categories: discovery safe
  https://nmap.org/nsedoc/scripts/afp-showmount.html
    Shows AFP shares and ACLs.
  ```

  如果使用了 `-oX` 选项，脚本帮助的 XML 表示将被写入给定文件。

- **`--script-trace`**  
  此选项类似于 `--packet-trace`，但工作在应用程序级别而不是逐包级别。如果指定了此选项，脚本执行的所有传入和传出通信都将被打印。显示的信息包括通信协议、源和目标地址以及传输的数据。如果超过 5% 的传输数据不可打印，则显示十六进制转储。指定 `--packet-trace` 也会启用脚本跟踪。

- **`--script-updatedb`**  
  此选项更新位于 `scripts/script.db` 中的脚本数据库，Nmap 使用该数据库来确定可用的默认脚本和类别。仅当您添加或删除了默认 `scripts` 目录中的 NSE 脚本，或者更改了任何脚本的类别时，才需要更新数据库。此选项单独使用，不带参数：`nmap --script-updatedb`。

其他一些 Nmap 选项对脚本扫描也有影响。其中最主要的是 `-sV`。版本扫描会自动执行 `version` 类别中的脚本。此类别中的脚本与其他脚本略有不同，因为它们的输出与版本扫描结果混合在一起，并且不会向屏幕产生任何脚本扫描输出。如果使用了 `-oX` 选项，典型的脚本输出仍然可以在 XML 输出文件中找到。

另一个影响脚本引擎的选项是 `-A`。积极的 Nmap 模式隐含了 `-sC` 选项。

### 脚本选择

`--script` 选项接受一个由类别、文件名和目录名组成的逗号分隔列表。一些简单的使用示例：

- `nmap --script default,safe`  
  加载 `default` 和 `safe` 类别中的所有脚本。

- `nmap --script smb-os-discovery`  
  仅加载 `smb-os-discovery` 脚本。注意 `.nse` 扩展名是可选的。

- `nmap --script default,banner,/home/user/customscripts`  
  加载 `default` 类别中的脚本、`banner` 脚本以及 `/home/user/customscripts` 目录中的所有 `.nse` 文件。

当按名称引用 `script.db` 中的脚本时，可以使用类似 shell 的 `*` 通配符。

- `nmap --script "http-*"`  
  加载所有以 `http-` 开头的脚本，例如 `http-auth` 和 `http-open-proxy`。必须将 `--script` 的参数用引号括起来，以防止 shell 解析通配符。

可以使用 `and`、`or` 和 `not` 运算符构建布尔表达式，从而进行更复杂的脚本选择。运算符的优先级与 Lua 中相同：`not` 最高，然后是 `and`，最后是 `or`。您可以使用括号改变优先级。由于表达式包含空格字符，因此需要将它们引用起来。

- `nmap --script "not intrusive"`  
  加载除 `intrusive` 类别之外的所有脚本。

- `nmap --script "default or safe"`  
  这在功能上等同于 `nmap --script "default,safe"`。它加载所有在 `default` 类别或 `safe` 类别（或两者）中的脚本。

- `nmap --script "default and safe"`  
  加载那些同时在 `default` 和 `safe` 类别中的脚本。

- `nmap --script "(default or safe or intrusive) and not http-*"`  
  加载 `default`、`safe` 或 `intrusive` 类别中的脚本，但排除那些名称以 `http-` 开头的脚本。

布尔表达式中的名称可以是类别、`script.db` 中的文件名或 `all`。名称是不包含空格、逗号、括号或分号的任何字符序列，但作为运算符的序列 `and`、`or` 和 `not` 除外。

### 脚本参数

可以使用 `--script-args` 选项向 NSE 脚本传递参数。这些参数描述了一个键值对表，可能还有数组值。参数作为一个名为 `nmap.registry.args` 的表提供给脚本，尽管通常通过 `stdnse.get_script_args` 函数访问它们。

脚本参数的语法类似于 Lua 的表构造语法。参数是逗号分隔的 `name=value` 对列表。名称和值可以是字符串，不包含空格或字符 `{`、`}`、`=` 或 `,`。要包含这些字符之一，请将字符串括在单引号或双引号中。在带引号的字符串中，`\` 转义引号。反斜杠仅用于在这种特殊情况下转义引号；在所有其他情况下，反斜杠按字面解释。

值也可以是包含在 `{}` 中的表，就像在 Lua 中一样。表可以包含简单的字符串值（例如代理主机列表），或者更多的键值对，包括嵌套表。

脚本参数通常用相关脚本的名称进行限定，以便用户不会因为使用单个通用名称而意外影响多个脚本。例如，您可以通过将 `broadcast-ping.timeout` 设置为您愿意等待的时间量，来设置 `broadcast-ping` 脚本（且仅该脚本）的响应超时。但有时您希望脚本参数更广泛地应用。如果您移除限定并仅指定 `timeout=250ms`，您将除了 `broadcast-ping` 之外，还为十多个脚本设置了该值。您甚至可以结合限定和未限定的参数，最具体的匹配优先。例如，您可以指定 `rlogin-brute.timeout=20s,timeout=250ms`。在这种情况下，`rlogin-brute` 脚本的超时将为 20 秒，而支持此变量的所有其他脚本（`broadcast-ping`、`lltd-discovery` 等）的超时将为 250 毫秒。

与其在命令行上使用 `--script-args` 传递参数，不如将它们存储在一个文件中（用逗号或换行符分隔），并使用 `--script-args-file` 仅指定文件名。在命令行上使用 `--script-args` 指定的选项优先于文件中给出的选项。文件名可以作为绝对路径或相对于 Nmap 的常规搜索路径（NMAPDIR 等）给出。

这是一个带有脚本参数的典型 Nmap 调用：

```bash
nmap -sC --script-args 'user=foo,pass=",{}=bar",paths={/admin,/cgi-bin},xmpp-info.server_name=localhost'
```

请注意，脚本参数用单引号括起来。对于 Bash shell，这可以防止 shell 解释双引号并进行自动字符串连接。当然，不同的 shell 可能需要您转义引号或使用不同的引号。请参阅您的相关手册。该命令产生以下 Lua 表：

```lua
nmap.registry.args = {
  user = "foo",
  pass = ",{}=bar",
  paths = {
    "/admin",
    "/cgi-bin"
  },
  xmpp-info.server_name="localhost"
}
```

虽然您可以直接从 `nmap.registry.args` 访问值，但通常最好使用 `stdnse.get_script_args` 函数，如下所示：

```lua
local server_name = stdnse.get_script_args("xmpp-info.server_name")
```

所有脚本参数共享一个全局命名空间，即 `nmap.registry.args` 表。因此，不推荐使用像 `user` 这样的短名称或模糊名称。某些脚本用其脚本名称作为参数的前缀，例如 `smtp-open-relay.domain`。由库使用的参数（可能会影响许多脚本）通常以库名称开头，例如 `smbuser` 和 `creds.snmp`。

在线 NSE 文档门户网站 [https://nmap.org/nsedoc/](https://nmap.org/nsedoc/) 列出了每个脚本接受的参数，包括可能影响脚本的任何库参数。

### 完整示例

- `nmap -sC example.com`  
  使用默认脚本集进行简单的脚本扫描。

- `nmap -sn -sC example.com`  
  没有端口扫描的脚本扫描；只有主机脚本有资格运行。

- `nmap -Pn -sn -sC example.com`  
  没有主机发现或端口扫描的脚本扫描。所有主机都被假定为在线，只有主机脚本有资格运行。

- `nmap --script smb-os-discovery --script-trace example.com`  
  执行特定脚本并启用脚本跟踪。

- `nmap --script snmp-sysdescr --script-args creds.snmp=admin example.com`  
  运行一个接受脚本参数的单独脚本。

- `nmap --script mycustomscripts,safe example.com`  
  执行 `mycustomscripts` 目录中的所有脚本以及 `safe` 类别中的所有脚本。
