## 引言

Nmap 脚本引擎（NSE）是 Nmap 最强大、最灵活的功能之一。它允许用户编写（并分享）简单的脚本，以自动化各种网络任务。这些脚本随后会以 Nmap 固有的速度和效率并行执行。用户可以依赖 Nmap 分发的不断增长且多样化的脚本集，也可以编写自己的脚本来满足自定义需求。

我们设计 NSE 时考虑到了多功能性，主要面向以下任务：

- **网络发现**  
  这是 Nmap 的核心能力。示例包括：根据目标域查询 whois 数据、针对目标 IP 查询 ARIN/RIPE/APNIC 以确定所有权、对开放端口执行 identd 查询、SNMP 查询以及列出可用的 NFS/SMB/RPC 共享和服务。

- **更复杂的版本检测**  
  Nmap 的版本检测系统（第 7 章《服务和应用程序版本检测》）能够通过其基于探测和正则表达式签名的匹配系统识别数千种不同的服务，但无法识别所有服务。例如，识别 Skype v2 服务需要两个独立的探测，而版本检测的灵活性不足以处理这种情况。如果 Nmap 尝试通过暴力方式尝试几百个不同的团体名称，它还可以识别更多的 SNMP 服务。这些任务都不适合传统的 Nmap 版本检测，但都可以通过 NSE 轻松完成。由于这些原因，版本检测现在默认调用 NSE 来处理一些棘手的服务。这在[“使用 NSE 进行版本检测”](nse-vscan.html)一节中进行了描述。

- **漏洞检测**  
  当发现新漏洞时，您通常希望快速扫描网络，在攻击者之前识别易受攻击的系统。虽然 Nmap 不是一个全面的[漏洞扫描器](https://sectools.org/vuln-scanners.html)，但 NSE 足够强大，甚至可以处理要求苛刻的漏洞检查。当 Heartbleed 漏洞影响全球数十万个系统时，Nmap 开发者在 2 天内就用 `ssl-heartbleed` 检测脚本做出了响应。许多漏洞检测脚本已经可用，我们计划随着它们的编写而发布更多。

- **后门检测**  
  许多攻击者以及一些自动化的蠕虫会留下后门以便日后重新进入。其中一些可以通过 Nmap 基于正则表达式的版本检测来发现，但更复杂的蠕虫和后门需要 NSE 的高级功能才能可靠地检测。NSE 已被用于检测 SMB 中的 Double Pulsar NSA 后门以及 UnrealIRCd、vsftpd 和 ProFTPd 的被植入后门的版本。

- **漏洞利用**  
  作为一种通用脚本语言，NSE 甚至可以用来利用漏洞，而不仅仅是发现它们。添加自定义利用脚本的能力对某些人（尤其是渗透测试人员）可能很有价值，尽管我们不打算将 Nmap 变成一个像 [Metasploit](http://www.metasploit.com) 那样的利用框架。

上面列出的项目是我们的初始目标，我们期望 Nmap 用户能为 NSE 提出更多创造性的用途。

脚本是用嵌入式 [Lua 编程语言](https://lua.org/)（5.4 版）编写的。该语言本身在《[Programming in Lua, Fourth Edition](http://www.amazon.com/dp/8590379868?tag=secbks-20)》和《[Lua 5.2 Reference Manual](http://www.amazon.com/dp/9888381229?tag=secbks-20)》中有很好的文档记录。针对 Lua 5.4 更新的参考手册也可[在线免费获取](https://lua.org/manual/5.4/)，《[Programming in Lua](https://lua.org/pil/)》第一版也是如此。鉴于这些优秀的通用 Lua 编程参考资料，本文档仅涵盖 Nmap 脚本引擎特有的方面和扩展。

NSE 通过 `-sC` 选项（或 `--script` 选项，如果您希望指定自定义脚本集）激活，结果会集成到 Nmap 的正常输出和 XML 输出中。

一个典型的脚本扫描如[示例 9.1](#nse-ex1) 所示。在此示例中产生输出的服务脚本有 `ssh-hostkey`（提供系统的 RSA 和 DSA SSH 密钥）和 `rpcinfo`（查询 portmapper 以枚举可用服务）。产生输出的唯一主机脚本是 `smb-os-discovery`，它从 SMB 服务器收集各种信息。Nmap 在三分之一秒内就发现了所有这些信息。
