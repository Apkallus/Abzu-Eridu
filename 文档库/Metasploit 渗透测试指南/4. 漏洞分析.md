# 漏洞分析

漏洞扫描器是一种自动化程序，旨在寻找计算机系统、网络和应用程序中的弱点。该程序通过网络向系统发送数据来探测系统。然后将响应与漏洞数据库中的样本进行比较，匹配结果用于枚举目标上的漏洞。

由于网络实现不同，操作系统在接收网络探测时往往会产生不同的响应。这些独特的响应构成了不同版本操作系统的指纹。扫描器使用这些指纹来确定操作系统的身份甚至其补丁级别。漏洞扫描器还可以使用一组给定的用户凭证登录远程系统，枚举软件和服务以确定它们是否打了补丁。根据获得的结果，扫描器会生成一份报告，概述系统上检测到的任何漏洞，这对网络管理员和渗透测试人员都很有用。

漏洞扫描器通常会在网络上产生大量流量，因此在以保持不被发现为目标的隐蔽渗透测试中通常不会使用。然而，如果你正在进行公开渗透测试且隐蔽性不是问题，漏洞扫描器可以让你免于手动探测系统以确定其补丁级别和漏洞。

无论你使用自动化扫描器还是手动进行，扫描都是渗透测试过程中最重要的步骤之一；如果做得彻底，它可以为客户提供最大的价值。通常，结合手动和自动化测试会给你带来最佳结果。

在本章中，我们将讨论几种漏洞扫描器以及它们如何集成到 Metasploit 中。我们还将重点介绍 Metasploit 框架中的一些辅助模块，这些模块可以定位远程系统中的特定漏洞。

## 注

要学习本章内容，请尝试使用示例中显示的命令扫描你的家庭网络或附录 A 中提到的任何机器。

## 基本漏洞扫描

让我们看看漏洞扫描在最基本的层面是如何工作的。在下面的示例中，我们使用 Netcat 从目标 192.168.1.203 抓取一个横幅。横幅抓取是连接到远程网络服务并读取返回的服务标识（横幅）的行为。许多网络服务，如 Web、文件传输和邮件服务器，要么在连接时立即返回其横幅，要么响应特定命令时返回。在这里，我们连接到 TCP 端口 80 上的 Web 服务器，并发出一个格式错误的 GET HTTP 请求，这使我们能够查看远程服务器响应我们请求返回的错误消息中的标头信息：

```text
kali@kali:~$ sudo nc 192.168.1.203 80
GET HTTP 1/1
HTTP/1.1 400 Bad Request
Content-Type: text/html; charset=us-ascii
Server: nginx/1.21.0
```

“Server:”之后返回的信息告诉我们，端口 80 上运行的进程是一个基于 Nginx 1.21 的 Web 服务器。有了这些信息，我们可以使用漏洞扫描器（例如图 4-1 中所示的 Nexpose 扫描器）来确定此版本的 Nginx 是否存在任何已知漏洞以及此服务器是否已打补丁。

<center>图 4-1：网络的漏洞扫描结果</center>

当然，在实践中，事情并没有那么简单。由于系统和应用程序配置的细微差异，漏洞扫描通常包含许多误报（报告存在漏洞但实际上不存在）和漏报（未能记录存在的漏洞）。此外，漏洞扫描器的创建者有动机报告阳性结果：漏洞扫描器发现的“命中”越多，对潜在买家来说它看起来就越好。漏洞扫描器的好坏取决于其漏洞数据库，并且它们很容易被误导性的横幅或不一致的配置所欺骗。

让我们看看一些更有用的漏洞扫描器，包括 Nexpose、Nessus 和一些专用扫描器。

## 使用 Nexpose 扫描

Nexpose 是 Rapid7 的漏洞扫描器。它扫描网络以识别其上运行的设备，并执行检查以识别操作系统和应用程序中的安全弱点。然后分析扫描数据并将其处理以包含在各种报告中。我们将使用 Nexpose 的 30 天免费试用版，你可以从 https://www.rapid7.com/products/nexpose/ 下载。如果你打算商业使用 Nexpose，请参阅 Rapid7 网站了解其功能和定价信息。

我们将首先对我们的目标执行一次基本的公开扫描，并将漏洞扫描结果导入 Metasploit。然后，我们将直接从 MSFconsole 运行 Nexpose 漏洞扫描，而不是使用基于 Web 的 GUI，从而无需导入扫描报告。

## 配置 Nexpose

安装 Nexpose 后，打开 Web 浏览器并导航到 https://<你的IP地址>:3780（记住使用 HTTPS）。服务启动可能需要大约五分钟。接受 Nexpose 自签名证书，并使用你在设置期间创建的凭据登录。你应该会看到一个类似于图 4-2 所示的界面。（Nexpose 的完整安装说明可以在 Rapid7 网站上找到。）

<center>图 4-2：Nexpose 主屏幕</center>

在 Nexpose 主页面上，你会注意到界面左侧有几个选项卡：

- **Assets（资产）** 选项卡：显示扫描后网络上计算机和其他设备的详细信息。
- **Vulnerabilities（漏洞）** 选项卡：提供有关扫描期间发现的任何漏洞的信息。
- **Automated Actions（自动化操作）** 选项卡：显示每当发生某些事件（例如新设备加入网络或新的漏洞更新）时自动执行的操作列表。
- **Policies（策略）** 选项卡：显示策略扫描列表。你的组织可能需要符合政府或组织标准，因此你可以配置 Nexpose 来扫描机器以确保它们符合这些标准。
- **Reports（报告）** 选项卡：列出生成后的漏洞扫描报告。
- **Tickets（工单）** 选项卡：显示活动工单。
- **Administration（管理）** 选项卡：允许你配置各种选项。

页面主体中的按钮允许你执行常见任务，例如创建新站点或设置新的漏洞扫描。

## 新建站点向导

在使用 Nexpose 运行漏洞扫描之前，你需要配置一个站点：设备的逻辑集合，例如特定的子网、服务器集合，甚至单个工作站。然后 Nexpose 将扫描这些站点，并且可以为特定站点定义不同的扫描类型。

要创建站点，请单击 Nexpose 主页上的“Create Site”，为你的站点输入名称和简短描述，然后单击“Next”。在“Assets”选项卡中，你可以在定义目标时具有相当大的粒度。你可以添加单个 IP 地址、地址范围、主机名等。我们喜欢使用 CIDR 表示法指定 IP 地址范围；例如 10.0.1.1/24。

你还可以声明要从扫描中排除的设备，例如打印机。（打印机通常不喜欢被扫描。我们见过一些实例，一次简单的漏洞扫描导致超过一百万页纯黑色墨水被放入打印队列！）

在“Authentication”选项卡中，如果你有凭据，可以为你想要扫描的站点添加凭据。凭据可以通过对目标上已安装的软件和系统策略执行深入枚举来帮助创建更准确、更完整的结果。

在“Templates”选项卡中，你可以从几种不同的扫描模板中进行选择，例如“Discovery Scan”和“Full Audit Scan”。选择你想要使用的扫描引擎。对于此初步演示，保持默认选择。最后，单击“Save and Scan”完成新建站点向导并返回到“Home”选项卡，该选项卡应显示你新添加的扫描。

Nexpose 应在扫描进行时动态刷新页面。等待“Scan Progress”和“Discovered Assets”的状态均显示“Completed”。在“Scan Progress”部分下，你可以看到扫描检测到的漏洞数量，在“Discovered Assets”下，你将获得有关目标的更多信息，例如设备名称及其操作系统。现在单击“Reports”选项卡。

## 新建报告

如果这是你第一次运行 Nexpose 并且只完成了一次扫描，“Reports”选项卡应显示你尚未生成任何报告。单击“Create a Report”启动“New Report”向导。输入一个友好的名称，然后在“Report format”字段中，单击模板并选择“Nexpose Simple XML Export”，以便你能够将扫描结果导入 Metasploit。你可以选择不同的报告模板，如果你碰巧在外地进行渗透测试，还可以配置时区。

通过单击“Select Sites”添加你的站点，将你想要包含在报告中的扫描或设备添加进去。

回到“Report Configuration”向导，单击“Save and Run”接受报告剩余的默认值。“Reports”选项卡现在应列出新创建的报告。（请确保保存报告文件，以便将其与报告一起使用。）

## 将报告导入 Metasploit

完成使用 Nexpose 的完整漏洞扫描后，你需要将结果导入 Metasploit。但是，在导入之前，你必须检查 MSFconsole 中数据库连接的状态，运行 `db_status` 命令。如果数据库未运行，请打开一个新终端并运行以下命令以启动数据库和 Metasploit 框架：

```sh
kali@kali:~$ sudo msfdb run
```

启动数据库后，你将使用 `db_import` 命令导入 Nexpose XML 文件。Metasploit 会自动检测到该文件来自 Nexpose 并导入已扫描的主机。然后，你可以通过运行 `hosts` 命令来验证导入是否成功。这些步骤如以下列表所示：

```bash
msf > db_import ~/Downloads/host_195.xml
[*] Importing 'Nexpose Simple XML' data
[*] Importing host 192.168.1.195
[*] Successfully imported /tmp/host_195.xml

msf > hosts -c address,svcs,vulns

Hosts
=====
address         Svcs  Vulns  Workspace
----            ----  -----  ---------
192.168.1.195   8     35     default
```

现在 Metasploit 知道了你的扫描发现的漏洞。要显示导入 Metasploit 的漏洞的完整详细信息，包括通用漏洞披露（CVE）编号和其他参考信息，请运行以下命令：

```bash
msf > vulns
```

使用完整凭据运行公开漏洞扫描可以提供惊人的信息量，在这种情况下，甚至可以发现漏洞。但是，当然，这是一次非常嘈杂的扫描，很可能会引起很多注意。这类漏洞扫描最好在不需要隐蔽性的渗透测试中使用。

## 在 MSFconsole 中运行 Nexpose

从 Web GUI 运行 Nexpose 非常适合微调漏洞扫描和生成报告，但如果你更愿意留在 MSFconsole 中，你仍然可以使用 Metasploit 中包含的 Nexpose 插件运行完整的漏洞扫描。

为了演示有凭据和无凭据扫描结果之间的差异，我们将从 Metasploit 运行一次扫描，而不指定目标系统的用户名和密码。在开始之前，请使用 `workspace` 命令创建一个新数据库并切换到该数据库。然后，使用 `load nexpose` 加载 Nexpose 插件：

```bash
msf > workspace -a nexpose-no-creds 
[*] Added workspace: nexpose-no-creds 
msf > workspace nexpose-no-creds 
[*] Workspace: nexpose-no-creds
msf > load nexpose
[*] Nexpose integration has been activated
[*] Successfully loaded plugin: nexpose
```

加载 Nexpose 插件后，输入 `help` 命令查看专门用于漏洞扫描器的命令：

```bash
msf > help
```

你应该会在列表顶部看到一系列特定于运行 Nexpose 的新命令。

在从 MSFconsole 运行第一次扫描之前，你需要连接到你的 Nexpose 安装。输入 `nexpose_connect -h` 以显示连接所需的用法。添加你的用户名、密码和主机地址，并在连接字符串末尾添加 `ok` 以接受 SSL 证书警告：

```bash
msf > nexpose_connect -h
[*] Usage:
[*]   nexpose_connect username:password@host[:port] <ssl-confirm...>
[*] -OR-
[*]   nexpose_connect username password host port <ssl-confirm...>

msf > nexpose_connect username:password@192.168.1.206 ok
[*] Connecting to Nexpose instance at 192.168.1.206:3780 with username...
```

输入 `nexpose_scan` 后跟目标 IP 地址以启动扫描。在此示例中，我们扫描单个 IP 地址，但你也可以将一系列主机传递给扫描器（例如 192.168.1.1-254）或 CIDR 表示法的子网（例如 192.168.1.0/24）：

```bash
msf > nexpose_scan 192.168.1.195
[*] Scanning 1 addresses with template pentest-audit in sets of 32
[*] Completed the scan of 1 address
msf >
```

Nexpose 扫描完成后，你之前创建的数据库应包含漏洞扫描的结果。要查看结果，请输入以下内容（在此示例中，输出已通过对地址、服务和漏洞列进行过滤而精简）：

```bash
msf > hosts -c address,svcs,vulns

Hosts
=====
address         Svcs  Vulns  Workspace
----            ----  -----  ---------
192.168.1.195   8     7      default
msf >
```

Nexpose 发现了 7 个漏洞。运行以下命令以显示发现的漏洞：

```bash
msf > vulns
```

因为我们没有提供与目标机关联的凭据，所以这次扫描发现的漏洞数量（7 个）远少于我们通过 GUI 使用凭据使用 Nexpose 发现的 35 个漏洞。尽管如此，这里你应该有足够的漏洞来为漏洞利用打下良好的基础。

## 使用 Nessus 扫描

由 Tenable Security（https://www.tenable.com/products/nessus）创建的 Nessus Essentials 是使用最广泛的漏洞扫描器之一。Metasploit 的 Nessus 插件允许你通过控制台启动扫描并从 Nessus 提取信息，但在下面的示例中，我们将首先独立导入 Nessus 扫描结果。在渗透测试的早期阶段，你能使用越多的工具来微调未来的攻击就越好。然后，我们将讨论直接从框架运行 Nessus 扫描。

## 配置 Nessus

下载并安装 Nessus 后，通过运行以下命令启动 Nessus 服务：

```bash
kali@kali:~$ sudo systemctl start nessusd.service
```

服务启动后，打开你的 Web 浏览器并导航到 https://localhost:8834，接受证书警告，然后使用你在安装期间创建的凭据登录 Nessus。登录后，你应该会看到 Nessus 主窗口。这是 Nessus Essentials 欢迎屏幕，你可以在其中添加要扫描的 IP 地址。你也可以使用 CIDR 表示法指定子网。

单击“Submit”，确认要扫描的 IP 地址，然后单击“Run Scan”。你将进入“Reports”部分（图 4-3），在那里你会看到结果。单击一个主机以查看有关其漏洞的更多详细信息。

<center>图 4-3：Nessus 扫描结果</center>

在界面侧面，你应该会看到“Scans”选项卡，你可以在其中创建和查看扫描任务，以及“Policies”选项卡，你可以在其中配置 Nessus 以包含你想要在扫描中使用的各种插件。

## 创建扫描

我们之前运行的扫描使用了 Nessus 的“Basic Network Scan”策略，但你可以创建自己的策略。在“Policies”选项卡上，单击右上角的“New Scan”以打开“Scan Templates”。你会看到许多可用选项，所有这些选项都可以在 Nessus 文档中找到。

选择“Advanced Network Scan”，并为扫描输入一个名称。在我们的示例中，我们使用名称“The Works”。接下来，选择要扫描的目标。你可以选择扫描一个主机，但也可以输入 CIDR 表示法的 IP 地址范围，甚至上传包含要扫描的目标地址的文件。

我们将配置此扫描使用 Windows 登录凭据，以获取目标系统上存在漏洞的更完整信息。Nessus 也支持 SSH 凭据。单击“Credentials”选项卡，输入目标系统的登录凭据，然后单击“Next”。

在“Plugins”页面上，你可以从大量用于 Windows、Linux、BSD 等的 Nessus 插件中进行选择。例如，如果在扫描期间你知道只会扫描基于 Windows 的系统，你可以在第一次运行时取消选择许多这些插件。现在，单击右上角的“Enable All”，然后单击“Save”。

这里有很多设置超出了我们的范围，但 Nessus 团队有很好的文档，如果你想了解更多相关信息。自动化扫描器并不完美，有时可能会错误地预测漏洞，导致误报。你可以通过在“Settings/Assessment”选项卡中选择“Avoid potential false alarms”来减少这些误报的数量。

创建扫描后，通过单击“My Scans”选项卡中扫描旁边的三角形“播放”图标来运行扫描。

## 创建扫描策略

你也可以通过定义扫描策略来创建自己的扫描模板。单击“Policies”选项卡，然后单击“New Policy”。设置新策略的过程与配置新扫描几乎相同。事实上，当你设置第一次扫描时，Nessus 会引导你完成策略过程。像之前一样，你可以定义扫描哪些机器、提供什么凭据以及启用哪些插件。

你可能会根据扫描是否为了遵守法律政策（例如支付卡行业数据安全标准）而做出不同的选择。Nessus 包含针对某些审计的策略模板，你可以设计自己的模板以遵守其他法律政策。

创建新策略后，你可以在“Scan Templates”中的“User Defined”选项卡中看到它。通过创建使用该策略的扫描来运行你的新策略。

## 查看报告

扫描完成后，扫描旁边会出现一个对勾。单击对勾打开报告摘要页面，显示发现漏洞的严重级别，如图 4-4 所示。

<center>图 4-4：Nessus 扫描报告摘要</center>

请记住，因为此扫描是使用 Windows 凭据运行的，Nessus 将发现比匿名扫描多得多的漏洞。

## 将结果导入 Metasploit

现在让我们将结果导入框架。在“Reports”屏幕上单击“Export”将 Nessus 结果保存到你的硬盘。Nessus 报告的默认文件格式 .nessus 可以被 Metasploit 解析，因此在提示选择默认格式时，单击“Nessus”，而不是“Nessus DB”。

加载 MSFconsole，使用 `workspace` 创建一个新的工作区，然后通过输入 `db_import` 后跟报告文件名来导入 Nessus 结果文件：

```bash
msf > workspace -a nessus
[*] Added workspace: nessus
msf > workspace nessus
[*] Workspace: nessus
msf > db_import /tmp/nessus_report_Host_195.nessus
[*] Importing 'Nessus XML (v2)' data
[*] Importing host 10.0.1.19
[*] Importing host 10.0.1.18
```

要验证扫描的主机和漏洞数据是否已正确导入，请使用 `hosts` 命令，如下所示。这将输出一个简短的列表，包含目标 IP 地址、检测到的服务数量以及 Nessus 发现的漏洞数量：

```bash
msf > hosts -c address,svcs,vulns

Hosts
=====
address       svcs  vulns
-----         ----  -----
10.0.1.1      11    32
10.0.1.2      1     12
10.0.1.4      1     12
```

要获取导入 Metasploit 的漏洞数据的完整列表，请输入以下内容：

```bash
msf > vulns
```

在渗透测试结束时，当为客户编写报告时，拥有这些参考信息会非常有帮助。

## Metasploit 中的 Nessus 扫描

当那些你不想离开舒适的命令行的时候，你可以使用 Metasploit 中由 Zate 编写的 Nessus Bridge 插件。Nessus Bridge 允许你通过 Metasploit 框架完全控制 Nessus，运行扫描，解释结果，并基于 Nessus 识别的漏洞发起攻击。

如同前面的示例，首先使用 `workspace` 命令创建并切换到一个新的数据库工作区。然后，加载 Nessus 插件：

```bash
msf > workspace -a nessus2
[*] Added workspace: nessus2
msf > workspace nessus2
[*] Workspace: nessus2
msf > load nessus
[*] Nessus Bridge for Metasploit
[+] Type nessus_help for a command listing
[*] Successfully loaded plugin: nessus
```

运行 `nessus_help` 将显示插件支持的所有命令。Nessus Bridge 会定期开发和更新，因此定期检查帮助输出以查看是否添加了新功能是个好主意。

在使用 Nessus Bridge 开始扫描之前，你首先需要使用 `nessus_connect` 对你的 Nessus 服务器进行身份验证。将用户名和密码占位符替换为你在安装期间设置的用户名和密码：

```bash
msf > nessus_connect username:password@192.168.1.101:8834 ok
[*] Connecting to https://192.168.1.101:8834/ as username
[*] Authenticated
```

与 Nessus 的 GUI 版本一样，通过指定策略的 UUID 来使用已定义的策略启动扫描。要列出服务器上可用的扫描策略，请输入 `nessus_policy_list`：

```bash
msf > nessus_policy_list
Policy ID  Name         Policy UUID
13         The Works    731a8e52-3ea6-a291-ec0a...
```

记下你想要用于扫描的策略 UUID，然后使用 `nessus_scan_new` 启动一个新的扫描，后跟策略 UUID、扫描名称、扫描描述和目标 IP 地址：

```bash
msf > nessus_scan_new
[*] Usage:
[*]   nessus_scan_new <Policy UUID> <scan name> <Description> <targets>
[*]   use nessus_policy_list to list all available policies

msf > nessus_scan_new 731a8e52... bridge_scan scan_description 10.0.1.19
[*] Creating scan from policy number 2, called "bridge_scan" and scanning 10.0.1.19
[*] Creating scan from policy number 731a8e52-3ea6-a291-ec0a-d2ff0619c19d7bd788d6be818b65, called bridge_scan - scan_description and scanning 10.0.1.12
[*] New scan added
[*] Use nessus_scan_launch 19 to launch the scan
Scan ID  Scanner ID  Policy ID  Targets     Owner
-------  ----------  ---------  -------     -----
21       1           18         10.0.1.12   bot
```

创建扫描后，你可以使用 `nessus_scan_launch` 命令配合扫描 ID 来启动它：

```bash
msf > nessus_scan_launch 21

[+] Scan ID 21 successfully launched. The Scan UUID is b6225dc2-f612-3c5d-88a9-882b9681e47413a67750d24d6da8
```

当扫描正在进行时，你可以通过运行 `nessus_scan_list` 命令查看其状态：

```bash
msf > nessus_scan_list

Scan ID  Name                     Owner  Status      Folder
-------  ----                     -----  ------      ------
5        My Host Discovery Scan   bot    canceled    3
11       My Basic Network Scan    bot    completed   3
13       The Works                bot    canceled    3
21       bridge_scan              bot    running     3
```

扫描完成后，你可以使用 `nessus_scan_export <scan ID> <export format>` 命令导出它。可用的格式有 Nessus、HTML、PDF、CSV 和 DB。在此示例中，我们选择导出为 Nessus 格式：

```bash
msf > nessus_scan_export 21 Nessus
[+] The export file ID for scan ID 21 is 2007425285
[*] Checking export status...
[*] Export status: loading
[*] Export status: ready
The status of scan ID 21 export is ready
```

导出将生成一个文件 ID，例如本例中的 2007425285。现在我们可以使用 `nessus_report_download <Scan ID>` 命令下载报告到本地：

```sh
msf > nessus_report_download 21 2007425285 
[*] Report downloaded to /home/userA/.msf6/local directory
```

如果你想直接将扫描结果导入 Metasploit 数据库，可以使用 `nessus_db_import <Scan ID>` 命令：

```shell
msf > nessus_db_import 21
[*] Exporting scan ID 21 is Nessus format...
[+] The export file ID for scan ID 21 is 282706699
[*] Checking export status...
[*] Export status: loading
[*] Export status: ready
[*] The status of scan ID 21 export is ready
[*] Importing scan results to the database...
[*] Importing data of 10.0.1.1
[+] Done
```

与本章演示的其他导入功能一样，你可以使用 `hosts` 命令来验证扫描数据是否已成功导入：

```shell
msf > hosts -c address,svcs,vulns

Hosts
=====
address    svcs  vulns
-------    ----  -----
10.0.1.1   18    45
```

现在你已经看到了两种不同产品扫描结果的差异，应该更能体会使用多种工具来满足扫描需求的价值。毕竟，如何解读这些自动化工具的结果并转化为可操作的情报，仍然取决于你自己。

## 专用漏洞扫描器

尽管市场上有许多商用漏洞扫描器，但你并不局限于使用它们。当你想在网络中针对特定漏洞运行扫描时，Metasploit 的许多辅助模块可以提供帮助。让我们看几个例子。利用你的实验室，尽可能多地探测和探索它们。

## 验证 SMB 登录

要检查 SMB 用户名和密码组合的有效性，请使用 SMB Login Check Scanner 连接到一个范围的主机。正如你可能预料的那样，这种扫描是吵闹且引人注目的，并且每次登录尝试都会出现在它遇到的每个 Windows 机器的事件日志中。

使用 `use` 选择 `smb_login` 模块后，你可以运行 `show_options` 查看“Required”下列出的设置。Metasploit 允许你指定单个用户名和密码对、用户名和密码列表，或者多个用户名和密码对的列表。在下一个示例中，`RHOSTS` 设置为一个小范围的 IP 地址，并为 Metasploit 配置了一个用户名和密码，以便对所有地址进行尝试：

```bash
msf > use auxiliary/scanner/smb/smb_login
msf auxiliary(scanner/smb/smb_login) > show options

Module options (auxiliary/scanner/smb/smb_login):

   Name                 Current Setting  Required  Description
   ----                 ---------------  --------  -----------
   ABORT_ON_LOCKOUT     false            yes       Abort the run when an...
   BLANK_PASSWORDS      false            no        Try blank passwords for...
   --snip--

msf auxiliary(smb_login) > set RHOSTS 192.168.1.150-155
RHOSTS => 192.168.1.150-192.168.1.155
msf auxiliary(smb_login) > set SMBUser Administrator
SMBUser => Administrator
msf auxiliary(smb_login) > set SMBPass s3cr3t
SMBPass => s3cr3t
msf auxiliary(smb_login) > run

[+] 192.168.1.155 - SUCCESSFUL LOGIN 'Administrator': 's3cr3t'
[*] Scanned 4 of 6 hosts (066% complete)
[*] Scanned 5 of 6 hosts (083% complete)
[*] Scanned 6 of 6 hosts (100% complete)
[*] Auxiliary module execution completed
msf auxiliary(smb_login) >
```

你可以看到使用用户 `Administrator` 和密码 `s3cr3t` 成功登录。因为在许多企业环境中，工作站都是从同一个镜像克隆并通过企业部署的，所以管理员密码很可能在所有工作站上都相同，这让你可以访问网络上的每个工作站。

## 查找针对近期漏洞的扫描器

搜索新扫描器的最佳地点之一是 Rapid7 的 Vulnerability and Exploit Database（https://www.rapid7.com/db/）。在你阅读本书的时候，很可能有人最近发现了一个新漏洞，并编写了一个 Metasploit 扫描器来帮助你找到它。你可以通过在搜索框中输入关键字 `scanner` 来查找最新的扫描器，因为扫描漏洞的模块名称通常以此关键字结尾。

当你选择感兴趣的扫描器时，你应该会看到关于如何运行它的逐步说明。例如，让我们看看 Apache Traversal RCE 扫描器。该扫描器识别运行 Apache 2.4.49 和 2.4.50 的机器，这些机器存在远程代码执行漏洞，允许我们远程控制机器。

我们首先选择模块和我们想要执行的操作。在这里，我们选择 `CHECK_RCE`，因为我们想要检查 RCE 漏洞：

```bash
msf > use auxiliary/scanner/http/apache_normalize_path
msf auxiliary(scanner/http/apache_normalize_path) > show actions

Auxiliary actions:

   Name             Description
   ----             -----------
   CHECK_RCE        Check for RCE (if mod_cgi is enabled).
   CHECK_TRAVERSAL  Check for vulnerability.
   READ_FILE        Read file on the remote server.

msf auxiliary(apache_normalize_path) > set ACTION CHECK_RCE
```

然后，我们像使用其他模块一样显示和设置选项。请记住，我们需要将 `RHOSTS` 设置为我们想要扫描的机器：

```bash
msf auxiliary(apache_normalize_path) > show options
msf auxiliary(apache_normalize_path) > run
```

将来你阅读本书时，可能会选择不同的扫描器，但运行它的一般过程应该是相同的。
