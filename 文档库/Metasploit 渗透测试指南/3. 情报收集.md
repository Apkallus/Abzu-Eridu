## 情报收集

情报收集是渗透测试的第二步，紧随参与前活动之后。在此阶段，你的目标是在不暴露自身存在的情况下获取关于目标的准确信息，了解组织的运作方式，并确定最佳的入侵途径。如果你没有彻底执行这些任务，可能会错过易受攻击的系统及可行的攻击向量。需要时间和耐心来梳理网页、执行 Google 黑客攻击、映射系统，以完全理解特定目标的基础设施。你还需要仔细规划、研究，以及最重要的是，能够像攻击者一样思考。

在开始情报收集之前，考虑如何记录你的操作和获得的结果。大多数安全专业人士很快会意识到，详细的笔记可能意味着成功与失败的区别。正如科学家必须获得可复现的结果一样，其他有经验的渗透测试人员应该能够仅凭你的文档复现你的工作。

## 警告

如果你遵循本书中的步骤，可能会损坏你的系统和目标，因此请务必在测试环境中操作（有关帮助，请参见附录 A）。这些章节中的许多示例具有破坏性，可能使目标系统无法使用。如果由怀有恶意的人进行，其中一些活动甚至可能被视为非法，所以请遵守规则，不要做蠢事。

大多数人发现自己渴望利用系统并获得 root 权限，但你需要先学会走路才能跑步。

## 被动信息收集

通过使用被动或间接的信息收集技术，你可以在不接触目标系统的情况下发现有关目标的细节。例如，你可以使用这些技术定位网络边界、识别网络维护者，甚至了解目标网络上运行的操作系统和 Web 服务器软件。

开源情报（OSINT）是一种利用公开或易于获取的信息来查找、选择和获取目标细节的情报收集形式。有几种工具使被动信息收集几乎毫不费力，包括 Yeti 和 Whois 等复杂软件。在本节中，我们将探讨被动信息收集的过程以及你可能会在此步骤中使用的工具。

例如，假设对 https://www.trustedsec.com 发起攻击。作为渗透测试的一部分，我们的目标是确定公司拥有哪些系统以及我们可以攻击哪些系统。某些系统可能不属于该公司，因此可能被视为超出范围而无法攻击。

## Whois 查询

Whois 是一个允许你搜索域名和互联网基础设施信息的工具。让我们首先使用 Kali Linux 的 Whois 查询来查找 trustedsec.com 的域名服务器名称：

```bash
msf > whois trustedsec.com
[*] exec: whois trustedsec.com
--snip--
Domain Name: TRUSTEDSEC.COM
Domain servers in listed order:
   GLEN.NS.CLOUDFLARE.COM
   LEIA.NS.CLOUDFLARE.COM
```

我们得知域名系统（DNS）服务器由第三方 Cloudflare 托管，因此我们不应将这些系统包含在渗透测试中，因为我们无权攻击它们。然而，在大多数大型组织中，DNS 服务器位于公司内部，是可行的攻击向量。区域传输和类似的 DNS 攻击通常可用于从外部和内部更多地了解网络。但在这种情况下，我们应该转向不同的攻击向量。

## Netcraft

Netcraft（https://searchdns.netcraft.com）是一个基于 Web 的工具，我们可以用它来查找托管特定网站的服务器 IP 地址，如图 3-1 所示。

```bash
msf > whois 104.26.15.63
```

反向代理试图隐藏原始 IP 地址。然而，攻击者仍可能使用其他策略恢复 IP 地址。详细介绍其中一些技术的文章可在 https://citadelo.com/en/blog/cloudflare-how-to-do-it-right-and-do-not-reveal-your-real-ip 找到。其中许多策略已被整合到 Metasploit 的云查找和绕过模块中。

## DNS 分析

DNS 服务器包含有关域的信息。为了获取更多域信息，我们将使用 `dig`（大多数 Unix 操作系统内置的工具）向 DNS 服务器查询有关 trustedsec.com 的信息。其他一些用于 DNS 分析的好工具是 `fierce` 和 `dnsrecon`。

在以下示例中，我们使用 `dig` 查找域的邮件交换（MX）记录。MX 记录包含用于处理该域电子邮件的服务器信息：

```bash
kali@kali:~$ sudo dig mx trustedsec.com
;; QUESTION SECTION:
;trustedsec.com.            IN  MX

;; ANSWER SECTION:
trustedsec.com.     5   IN  MX  20 mx2-us1.ppe-hosted.com.
trustedsec.com.     5   IN  MX  10 mx1-us1.ppe-hosted.com.
```

我们看到邮件服务器指向 mx2-us1.ppe-hosted.com 和 mx1-us1.ppe-hosted.com。快速调查告诉我们这些网站由第三方托管，这使它们脱离了我们渗透测试的范围。

此时，我们已经收集了一些可能有用的有价值信息。但最终，我们可能不得不诉诸主动信息收集技术来获取更多细节。

## 注

被动信息收集的艺术并非仅靠几页讨论就能轻易掌握。请参阅 PTES（http://www.pentest-standard.org）和 Cyber Detective 的 OSINT 工具集（https://github.com/cipher387/osint_stuff_tool_collection），获取执行额外被动情报收集的潜在方法列表。

## 主动信息收集

在主动信息收集中，我们直接与系统交互以了解更多信息。例如，我们可能会进行扫描以发现目标上的开放端口或确定正在运行哪些服务。我们发现的每个正在运行的系统或服务都为我们提供了另一个漏洞利用的机会。但请注意：如果在主动信息收集期间粗心大意，你可能会被入侵检测系统（IDS）或入侵防御系统（IPS）发现——这对隐蔽的渗透测试人员来说可不是好结果。

## 使用 Nmap 进行端口扫描

通过被动信息收集确定了目标 IP 范围和 trustedsec.com 的 IP 地址后，我们可以开始通过端口扫描来寻找目标上的开放端口。端口扫描是一个我们精心连接远程主机上的端口以识别哪些端口处于活动状态的过程。（在大型企业中，我们会有多个 IP 范围可供攻击，而不仅仅是一个 IP 地址。）

Nmap 是目前最流行的端口扫描工具。它与 Metasploit 的集成非常优雅，将扫描输出存储在后端数据库中以供后续使用。Nmap 允许你扫描主机以识别每个主机上运行的服务，其中任何一个都可能提供入侵途径。

在本例中，让我们暂时离开 trustedsec.com，转而使用 scanme.nmap.org（45.33.32.156），这是 Nmap 团队维护的一台服务器。如果你更愿意扫描自己的机器，请使用附录 A 中描述的虚拟机之一。在开始之前，通过在 Kali 机器上的命令行输入 `nmap` 快速查看基本的 Nmap 语法。你会立即看到它有多个选项，但你可能只会使用其中几个。

最有用的 Nmap 选项之一是 `-sS`，它运行一个隐蔽的 TCP 扫描，确定特定的基于 TCP 的端口是否开放。另一个首选选项是 `-Pn`，它告诉 Nmap 不要使用 ping 来确定系统是否正在运行；相反，它将所有主机视为“活动”。如果你正在执行基于互联网的渗透测试，应使用此标志，因为大多数网络不允许 Internet 控制消息协议（ICMP），即 ping 使用的协议。如果你在内部执行此扫描，则可以忽略此标志。

## 执行 TCP 空闲扫描

一种更高级的 Nmap 扫描方法——TCP 空闲扫描，允许我们通过欺骗网络上另一台主机的 IP 地址来隐蔽地扫描目标。要使这种扫描生效，我们首先需要定位一个使用增量 IP ID（用于跟踪数据包顺序）的空闲主机。当空闲系统使用增量 IP ID 时，这些 ID 变得可预测，从而使我们能够计算出下一个 ID。每当 IP ID 序列的可预测性出现中断时，我们就知道发现了一个开放端口。要了解有关 IP ID 序列和此模块的更多信息，请访问 https://nmap.org/book/idlescan.html 和 https://www.metasploit.com/modules/auxiliary/scanner/ip/ipidseq。

然而，许多操作系统通过随机化 IP ID 来防范此类攻击。使用框架的 `scanner/ip/ipidseq` 模块扫描符合 TCP 空闲扫描要求的主机：

```bash
msf > use auxiliary/scanner/ip/ipidseq
msf auxiliary(ipidseq) > show options

Module options:

   Name       Current Setting  Required  Description
   INTERFACE                   no        The name of the interface
   RHOSTS                      yes       The target address range or CIDR...
   RPORT      80               yes       The target port
   SNAPLEN    65535            yes       The number of bytes to capture
   THREADS    1                yes       The number of concurrent threads
   TIMEOUT    500              yes       The reply read timeout in milliseconds
```

此列表显示 ipidseq 扫描所需的选项。一个值得注意的选项 `RHOSTS` 可以接受 IP 范围（例如 192.168.1.20 到 192.168.1.30）；无类域间路由（CIDR）范围（例如 192.168.1.0/24）；用逗号分隔的多个范围（例如 192.168.1.0/24,192.168.3.0/24）；或每行一个主机的文本文件（例如 file:/tmp/hostlist.txt）。所有这些选项使我们在指定目标时具有灵活性。

`THREADS` 值设置扫描时使用的并发线程数。默认情况下，所有扫描器模块的 `THREADS` 值初始设置为 1。我们可以提高此值以加快扫描速度，或降低它以减少网络流量。

```bash
msf auxiliary(ipidseq) > set RHOSTS 192.168.1.0/24
RHOSTS => 192.168.1.0/24
msf auxiliary(ipidseq) > set THREADS 50
THREADS => 50
msf auxiliary(ipidseq) > run
```

## 使用 Metasploit 进行端口扫描

除了能够使用第三方扫描器外，Metasploit 在其辅助模块中还内置了几个端口扫描器，这些扫描器直接与框架的大部分方面集成。在后续章节中，我们将利用被攻陷的系统来扫描和攻击其他系统；这个过程通常称为“ pivoting”，它允许我们使用内部连接的系统将流量路由到原本无法访问的网络。

例如，假设你攻陷了一个防火墙后面的系统，该系统使用网络地址转换（NAT）。NAT 防火墙后面的系统使用私有 IP 地址，你无法直接从互联网联系到它们。如果你使用 Metasploit 攻陷了 NAT 防火墙后面的系统，你也许能够利用那个被攻陷的内部系统将流量传递（或 pivoting）到内部托管且基于私有 IP 的系统，从而渗透到防火墙更远的网络内部。

要查看框架提供的端口扫描工具列表，请输入：

```bash
msf > search portscan
```

让我们使用 Metasploit 的 SYN 端口扫描器对单个主机执行示例扫描。在下面的列表中，我们将 `RHOSTS` 设置为 192.168.1.155，将 `THREADS` 设置为 50，然后运行扫描：

```bash

RHOSTS => 192.168.1.155
msf auxiliary(syn) > set THREADS 50

msf auxiliary(syn) > run

[*] TCP OPEN 192.168.1.155:445
[*] Scanned 1 of 1 hosts (100% complete)

msf auxiliary(syn) >
```

从结果中可以看到，IP 地址 192.168.1.155 上的端口 135、139 和 445 处于开放状态。

## 定向扫描

在进行渗透测试时，寻找捷径并不可耻。定向扫描会查找已知可利用且能提供进入目标网络的便捷途径的特定操作系统、服务、程序版本或配置。Rapid7 维护了一个经过验证的扫描器和漏洞利用模块仓库（https://www.rapid7.com/db/?q=&type=metasploit）。从最新的扫描器开始是一个好主意。

## 扫描服务器消息块（SMB）

Metasploit 可以扫描网络并尝试使用其 `smb_version` 模块识别 Microsoft Windows 的版本。此扫描器依赖于检测服务器消息块（SMB），一种常见的文件共享协议。

## 注

如果你不熟悉 SMB，请在继续之前稍微学习一下。Microsoft 团队提供的这个资源很好地介绍了 SMB 的一些基础知识：https://docs.microsoft.com/en-us/windows/win32/fileio/microsoft-smb-protocol-and-cifs-protocol-overview。

我们运行该模块，列出选项，设置 `RHOSTS`，并开始扫描：

```bash
msf > use auxiliary/scanner/smb/smb_version
...
```

`smb_version` 扫描器已检测到该机器上运行的 SMB 服务的首选方言、加密能力和其他属性。由于我们只扫描一个系统，因此将 `THREADS` 保留为 1。如果扫描多个系统（例如一个 C 类子网范围），我们可能会考虑使用 `set THREADS number` 选项来增加 `THREADS`。

## 寻找配置不当的 Microsoft SQL Server

配置不当的 Microsoft SQL Server（MS SQL）安装可能提供进入目标网络的初始途径。事实上，一些系统管理员甚至不知道他们的工作站上安装了 MS SQL 服务器，因为该服务是作为某些常见软件（如 Microsoft Visual Studio）的先决条件安装的。这些安装可能未被使用、未打补丁，甚至从未配置过。

当 MS SQL 安装后，它默认监听 TCP 端口 1433 或一个随机的动态 TCP 端口。如果 MS SQL 正在监听动态端口，只需查询 UDP 端口 1434 即可发现是哪个。当然，Metasploit 有一个可以利用此功能的模块：`mssql_ping`。

由于 `mssql_ping` 使用 UDP，由于超时问题，跨多个子网运行可能会非常慢。但在本地 LAN 上，将 `THREADS` 设置为 255 将大大加快扫描速度。当 Metasploit 发现 MS SQL 服务器时，它应显示它能提取的所有详细信息，包括（也许最重要的）服务器正在监听的 TCP 端口。

## 扫描简单网络管理协议（SNMP）

简单网络管理协议（SNMP）通常用于网络设备中，以报告带宽利用率和冲突率等信息。然而，一些操作系统也有 SNMP 服务器，可以提供 CPU 利用率、可用内存和其他特定于系统的详细信息。

系统管理员的便利可能成为渗透测试人员的金矿，可访问的 SNMP 服务器可以提供有关特定系统的大量信息，甚至可能使远程设备被攻陷。例如，如果你能获得 Cisco 路由器的读/写 SNMP 社区字符串，你就可以下载路由器的整个配置、修改它，然后将其上传回路由器。（社区字符串本质上是用于查询设备信息或将配置信息写入设备的密码。）

Metasploit 框架包含一个名为 `snmp_enum` 的内置辅助模块，专门用于 SNMP 扫描。在开始扫描之前，请记住只读（RO）和读/写（RW）社区字符串将在你能够从给定设备提取的信息类型中发挥重要作用。在配置了 SNMP 的基于 Windows 的设备上，你通常可以使用 RO 或 RW 社区字符串来提取补丁级别、运行的服务、用户名、正常运行时间、路由和其他信息，这些信息可以在渗透测试期间为你提供极大便利。

要获得对交换机的访问权限，你首先需要尝试找到其社区字符串。在你猜出社区字符串后，某些版本的 SNMP 将允许从过度的信息披露到完全系统入侵的各种行为。SNMPv1 和 v2 本身存在缺陷的协议。SNMPv3 融合了加密和更好的检查机制，因此更加安全。

框架的 `snmp_login` 模块将尝试通过将一个字典中的条目发送到一个 IP 地址或一个 IP 地址范围来猜测社区字符串：

```bash
msf > use auxiliary/scanner/snmp/snmp_login
msf auxiliary(snmp_login) > set RHOSTS 192.168.1.0/24
RHOSTS => 192.168.1.0/24
msf auxiliary(snmp_login) > set THREADS 50
THREADS => 50
msf auxiliary(snmp_login) > run
[*] >> progress (192.168.1.0-192.168.1.255) 0/30208...
[*] 192.168.1.2 'public' 'GSM7224 L2 Managed Gigabit Switch'
[*] 192.168.1.2 'private' 'GSM7224 L2 Managed Gigabit Switc
h'
[*] Auxiliary module execution completed
msf auxiliary(snmp_login) >
```

在输出结果中快速搜索引擎一下 "GSM7224"，我们会发现扫描器已经找到了 NETGEAR 交换机的 public 和 private 团体字符串。信不信由你，这个结果并非是为本书特意安排的。这正是该交换机的出厂默认设置。

在渗透测试的职业生涯中，你会遇到许多像这样令人瞠目结舌的情况，因为很多管理员只是简单地将设备接入网络，而保留所有默认设置。当你在某个大型企业中发现这些设备可从互联网访问时，情况就更可怕了。

## 编写自定义扫描器

在安全评估期间，编写自己的扫描器可能会很有用，因为许多应用程序和服务在 Metasploit 中缺少现成的扫描模块。值得庆幸的是，Metasploit 框架提供了许多功能来帮助你构建自定义扫描器，包括对代理、安全套接字层 (SSL) 协议以及多线程的支持。

Metasploit 框架的扫描模块通常通过使用各种 `mixin` 来包含功能，mixin 是带有预定义函数的部分代码。`Auxiliary::Scanner` mixin 会重载辅助模块的 `run` 方法，调用 `run_host(ip)`、`run_range(range)` 或 `run_batch(batch)` 方法，然后处理你指定要扫描的 IP 地址。虽然我们将在第 11 章中更详细地介绍辅助模块，但在此我们将演示如何利用 `Auxiliary::Scanner` 来调用额外的内置 Metasploit 功能。让我们写一些代码。

下面是一个用于简单 TCP 扫描器的 Ruby 脚本，它通过默认端口 12345 连接到远程主机，连接后发送消息 "HELLO SERVER"，接收服务器响应，并将其与服务器的 IP 地址一起打印出来：

```ruby
require 'msf/core'

class MetasploitModule < Msf::Auxiliary
  include Msf::Exploit::Remote::Tcp
  include Msf::Auxiliary::Scanner

  def initialize
    super(
      'Name'        => 'My custom TCP scan',
      'Version'     => '$Revision: 1 $',
      'Description' => 'My quick scanner',
      'Author'      => 'Your name here',
      'License'     => MSF_LICENSE
    )
    register_options(
      [
        Opt::RPORT(12345)
      ], self.class)
  end

  def run_host(ip)
    connect()
    sock.puts('HELLO SERVER')
    data = sock.recv(1024)
    print_status("Received: #{data} from #{ip}")
    disconnect()
  end
end
```

如果你不熟悉 Ruby，你可能需要花一些时间熟悉该语言，然后再回到本节。

`include Msf::Exploit::Remote::Tcp` 处理 TCP 网络。`Msf::Auxiliary::Scanner` 混入（mixin）公开了框架内扫描器所需的各种设置。此扫描器配置为使用默认端口 12345，在连接到服务器时，它会发送一条消息，接收来自服务器的回复，然后将其与服务器 IP 地址一起打印到屏幕上。

我们将此自定义脚本保存在 `modules/auxiliary/scanner/` 下，命名为 `simple_tcp.rb`。在 Metasploit 中，保存位置很重要。例如，如果模块保存在 `modules/auxiliary/scanner/http/` 下，它将在模块列表中显示为 `scanner/http/simple_tcp`。

为了测试这个初级扫描器，我们在端口 12345 上设置了一个 Netcat 监听器，并传入一个文本文件作为服务器响应：

```bash
kali@kali:/$ echo "Hello Metasploit" > banner.txt
kali@kali:/$ nc -lvnp 12345 < banner.txt
listening on [any] 12345 ...
```

接下来，我们加载 MSFconsole，选择我们的扫描器模块，设置其参数，并运行它看看是否有效：

```bash
msf > use auxiliary/scanner/simple_tcp
msf auxiliary(simple_tcp) > show options

Module options:

   Name       Current Setting  Required  Description
   ----       ---------------  --------  -----------
   RHOSTS                      yes       The target address range or CIDR identifier
   RPORT      12345            yes       The target port
   THREADS    1                yes       The number of concurrent threads

msf auxiliary(simple_tcp) > set RHOSTS 192.168.1.101
RHOSTS => 192.168.1.101
msf auxiliary(simple_tcp) > run

[*] Received: Hello Metasploit from 192.168.1.101
[*] Scanned 1 of 1 hosts (100% complete)
[*] Auxiliary module execution completed
msf auxiliary(simple_tcp) >
```

尽管这只是一个简单的例子，但当你需要在渗透测试过程中快速启动并运行一些自定义代码时，Metasploit 框架提供的多功能性可能会提供很大的帮助。希望这个示例能展示框架和模块化代码的强大功能。
