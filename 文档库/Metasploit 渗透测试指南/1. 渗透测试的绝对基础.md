# 渗透测试的绝对基础

渗透测试是一种模拟攻击者可能用来绕过安全控制并获取组织系统访问权限的方法。它不仅仅涉及运行扫描器和自动化工具然后编写报告。你也不可能一夜之间成为渗透测试专家；精通它需要多年的实践和真实世界的经验。

多年来，渗透测试执行标准（PTES）通过标准化术语和方法论重新定义了安全行业，安全社区的几位领先成员已采纳该标准为其测试建立基准。本章概述了 PTES 的基本原则。

## PTES 的各阶段

PTES 的各阶段定义了一次渗透测试。它们还向客户组织保证，任何进行此类评估的人员都将投入一致水平的努力。该标准分为七个类别，根据被测组织的不同，每个类别需要不同级别的工作量。

## 参与前交互

参与前交互通常发生在与客户讨论渗透测试的范围和条款时。在此阶段，传达测试的目标至关重要。这个阶段也是一个机会，可以向客户说明他们应该从一次彻底、全范围的渗透测试中期待什么：即在测试期间对可以且将要测试的内容不加限制。

## 情报收集

在情报收集阶段，你将使用社交媒体网络、Google 黑客、足迹分析等方法收集关于目标组织的任何信息。渗透测试人员最重要的技能之一就是能够了解目标，包括其行为方式、运作方式以及最终如何被攻击。

在情报收集过程中，你通过开始缓慢探测目标系统，试图识别目标上存在哪些保护机制。例如，组织通常只允许外部设备上的某一部分端口子集有流量，如果你查询的不是允许列表中的端口，就会被阻止。通常，最好最初从一个你愿意被阻止或检测到的、可丢弃的 IP 地址来探测这种阻止行为。在测试 Web 应用程序时也是如此。达到一定阈值后，Web 应用防火墙会阻止你发出更多请求。

为了保持不被发现，你可以从无法追溯到你及你团队的 IP 地址范围进行初始扫描。通常，在互联网上有外部存在的组织每天都遭受攻击，你的初步探测很可能成为背景噪音中未被发现的一部分。

在某些情况下，使用与主攻击完全不同的 IP 范围进行非常嘈杂的扫描可能是合理的。这将帮助你确定组织对你所使用的工具的反应程度。

## 威胁建模

威胁建模利用你在情报收集阶段获取的信息来识别目标系统上存在的任何漏洞。在执行威胁建模时，你将确定最有效的攻击方法、你追求的信息类型以及组织可能如何被攻击。威胁建模涉及以对手的身份看待组织，并像攻击者一样试图利用弱点。

## 漏洞分析

在确定了最可行的攻击方法之后，下一步是考虑如何访问目标。在漏洞分析期间，你将结合之前阶段学到的信息，并利用这些信息来理解哪些攻击是可行的。除其他事项外，漏洞分析评估端口和漏洞扫描、通过横幅抓取收集的数据以及在情报收集期间收集的信息。

## 漏洞利用

漏洞利用可能是渗透测试中最具魅力的部分之一，但它通常是靠蛮力而非精准度完成的。只有当你知道漏洞利用几乎肯定会成功时，才应该执行它。当然，目标上可能存在无法预见的防护措施，阻止特定漏洞利用生效。此外，盲目地发射大量漏洞利用并祈祷获得一个 shell 是无效的；它很嘈杂，并且对你和你的客户几乎没有价值。先做好功课，然后启动经过充分研究、很可能成功的漏洞利用。

## 后渗透利用

后渗透利用阶段在你攻陷一个或多个系统后开始。但是，即便如此，离完成还差得远。

后渗透利用是任何渗透测试中的关键组成部分。这是你通过访问有价值信息来区别于普通黑客的地方。后渗透利用针对特定系统，识别关键基础设施，并关注公司最重视并试图保护的信息或数据。当你一个接一个地利用系统时，你试图展示会产生最大业务影响的攻击。

在后渗透利用中攻击系统时，你应该花时间确定各个系统的功能以及它们不同的用户角色。例如，假设你攻陷了一个域基础设施系统，并且以企业管理员身份运行或拥有域管理员级别的权限。你可能成为域的主宰，但是与 Active Directory 通信的系统呢？用于支付员工工资的主要财务应用程序呢？你能攻陷该系统，然后在下一个支付周期，让它将所有资金转出公司并进入一个离岸账户吗？目标的知识产权又如何？

例如，假设你的客户是一家大型软件开发公司，向客户提供用于制造环境的定制应用程序。你能给其源代码植入后门，从而危害其所有客户吗？那会对其品牌信誉造成什么损害？

后渗透利用是一个棘手的场景，你必须花时间了解哪些信息可用，然后利用这些信息为你服务。真实的攻击者通常会在被攻陷的系统中花费大量时间做同样的事情。像恶意攻击者一样思考：要有创造力，快速适应，依靠智慧而不是自动化工具。

## 报告编写

报告编写是渗透测试中迄今为止最重要的元素。你将使用报告来沟通你做了什么、如何做的，以及最重要的是，组织应如何修复在渗透测试期间发现的漏洞。

在执行渗透测试时，你是在从攻击者的角度工作，这是组织很少看到的视角。你在测试期间获得的信息对于组织信息安全管理计划的成功以及阻止未来攻击至关重要。在整理和报告你的发现时，思考组织如何利用它们来提高安全意识、修复发现的问题以及改善整体安全性，而不仅仅是修补技术漏洞。

至少，将你的报告分为执行摘要、执行演示文稿和技术发现，客户将使用技术发现来修复安全漏洞。例如，如果你在客户基于 Web 的应用程序中发现 SQL 注入漏洞，你可能会建议客户对所有用户输入进行清理、使用参数化 SQL 查询、以受限用户账户运行 SQL 并启用自定义错误消息。

在客户实施你的建议并修复了那一个特定的 SQL 注入漏洞后，它是否就免受 SQL 注入了？不是。潜在的问题，例如未能确保第三方应用程序安全，很可能是首先导致 SQL 注入漏洞的原因。这类问题也需要修复。

## 渗透测试的类型

现在你已经对 PTES 的七个类别有了基本了解，让我们来看看两种主要的渗透测试类型：公开测试和隐蔽测试。公开测试（或白盒测试）是在组织完全知情的情况下进行的；隐蔽测试旨在模拟一个未知且未宣布的攻击者的行动。两种测试都有优缺点。

## 公开测试

使用公开渗透测试，你与组织合作，在组织的 IT 或安全团队向你展示其系统时，识别潜在的安全威胁。公开测试的一个主要好处是你有内幕知识，可以在不被阻止的情况下发起攻击。公开测试的一个潜在缺点是它可能无法有效测试客户的事件响应程序，也无法识别安全管理程序检测特定攻击的效果如何。当时间有限且情报收集等 PTES 步骤不在范围内时，公开测试可能是你最好的选择。

## 隐蔽测试

与公开测试不同，经过授权的隐蔽渗透测试旨在模拟攻击者的行动，并且在组织内大多数人员不知情的情况下执行。隐蔽测试挑战内部安全团队检测和响应攻击的能力。

隐蔽测试可能成本高昂且耗时，并且需要比公开测试更高的技能。在渗透测试人员眼中，隐蔽场景通常更受青睐，因为它最接近模拟真实攻击。隐蔽攻击依赖于你通过侦察获取信息的能力。因此，作为隐蔽测试人员，你通常不会试图在目标上发现大量漏洞。相反，你只会寻找最简单的方式，在保持不被发现的情况下获得系统访问权限。

## 漏洞扫描器

漏洞扫描器是用于识别影响给定系统或应用程序的安全缺陷的自动化工具。漏洞扫描器通常通过指纹识别目标的操作系统（即识别其版本和类型）以及任何正在运行的服务来工作。一旦你获取了目标操作系统的指纹，你就使用漏洞扫描器执行特定的检查，以确定是否存在漏洞。

当然，这些检查的好坏取决于其创建者，并且与任何完全自动化的解决方案一样，它们有时可能会遗漏或误报系统上的漏洞。然而，大多数现代漏洞扫描器在最小化误报方面做得非常出色，许多组织使用它们来识别过时的系统或攻击者可能利用的潜在新暴露点。

漏洞扫描器在渗透测试中扮演着非常重要的角色，尤其是在公开测试中，你可以发起多次攻击而无需担心规避检测。从漏洞扫描器中获得的知识可能是无价的，但注意不要过分依赖它们。在大多数情况下，当你成为经验丰富的渗透测试人员后，你会很少使用漏洞扫描器，而是依靠自己的知识和专业能力来攻陷系统。

## 安装 Kali、Metasploit 和 Metasploitable

Metasploit 预装在 Kali Linux、Parrot OS、BlackArch 和 BackBox 等渗透测试发行版中。如果你已经在运行这些发行版之一，Metasploit 很可能已经安装好了。

在本书中，示例将假设你是在 Kali 虚拟机中运行所有命令，不过你也可以直接在 Windows、macOS 或 Linux 操作系统上安装 Metasploit 框架。Metasploit 框架项目由 Rapid7 的一个团队管理，他们还维护了一个包含详细安装说明的网页：https://docs.rapid7.com/metasploit/installing-the-metasploit-framework/。

一旦你安装了 Metasploit 框架，你可能想练习使用它。Rapid7 创建了一个名为 Metasploitable 的易受攻击的虚拟机，其中包含多个你可以使用 Metasploit 练习利用的漏洞。在本书中，我们将多次攻击 Metasploitable 机器。

关于搭建包含 Kali 和 Metasploitable 的虚拟测试环境的分步指南，请参见附录 A。

## 总结

如果你是渗透测试的新手，或者尚未采用正式的方法论，请学习 PTES。与任何实验一样，在执行渗透测试时，确保你拥有一个精细、可适应且可重复的流程。你还需要完善情报收集和漏洞分析技能。这些技能将帮助你适应新的场景。
