# 第11章：辅助模块

当大多数人想到 Metasploit 时，他们会想到漏洞利用。漏洞利用很有用，因为它们可以让你获得远程访问权限，但有时你需要其他东西。在本章中，我们将讨论辅助模块，它们涵盖了广泛的功能。

## 探索辅助模块

除了提供有价值的侦察工具（如端口扫描器和服务指纹识别器）之外，像 `ssh_login` 这样的辅助模块还可以获取已知的用户名和密码列表，然后尝试通过暴力破解登录整个目标网络。辅助模块中还包含各种协议模糊测试器，例如 `ftp_pre_post`、`http_get_uri_long`、`smtp_fuzzer` 和 `ssh_version_corrupt` 等等。你可以针对目标服务启动这些模糊测试器，以期发现自己的可利用漏洞。

Metasploit 将这些模块安装在 `/modules/auxiliary` 目录中，并根据它们提供的功能进行分类。如果你是从源码安装 Metasploit 而不是通过 Kali，你会在 `/opt/metasploit-framework/embedded/framework/tools/modules` 下找到这些模块。如果你想创建自己的模块或编辑现有的模块以适应特定目的，你可以在相应的目录中找到示例。例如，如果你需要开发一个模糊测试器来发现自己的漏洞，你可以在 `/fuzzers` 目录中找到预先存在的模糊测试模块。

要列出 Metasploit 中所有可用的辅助模块，可以在 MSFconsole 中执行 `show auxiliary` 命令。如果你将上面的目录列表与那里显示的模块名称进行比较，你会注意到模块的命名取决于底层的目录结构：

```sh
msf > show auxiliary
...

    reset
9   auxiliary/admin/chromecast/chromecast_    normal  No   Chromecast YouTube Remote
    youtube
...
```

辅助模块按类别组织。你可以使用 DNS 枚举模块、Wi-Fi 模糊测试器，甚至还有一个用于定位和滥用 Energizer USB 电池充电器中包含的特洛伊木马后门的模块。

使用辅助模块就像使用框架中的任何漏洞一样：只需输入 `use` 命令后跟模块名称。例如，要使用 `webdav_scanner` 模块，请运行以下命令：

```sh
msf > use auxiliary/scanner/http/webdav_scanner
```

## 搜索 HTTP 模块

辅助模块令人兴奋，因为它们可以以多种方式用于许多事情。如果你找不到完美的辅助模块，修改一个以适应你的特定需求也很容易。

考虑一个常见示例：假设你正在进行远程渗透测试，扫描网络后发现了几台 Web 服务器，但没有其他太多东西。你的攻击面有限，必须利用可用的东西。当你寻找可以发起攻击的“低垂果实”时，辅助 scanner/http 模块现在将变得非常有帮助。要搜索所有可用的 HTTP 扫描器，请运行 `search`：

## 创建辅助模块

让我们看看一个当前不在 Metasploit 仓库中的辅助模块的结构。这个例子将演示如何将大量编程工作轻松转移给框架，让我们专注于模块的具体细节。

Chris Gates 编写了一个辅助模块，给他的 Twitter 粉丝留下了他不知何故发明了一种以光速行进的设备的印象。你可以在 https://github.com/carnal0wnage/Metasploit-Code/blob/master/modules/auxiliary/admin/foursquare.rb 找到 Chris 的原始程序。

尽管 Chris 在示例中使用的 API 已被弃用，但它对于创建提交 HTTP 请求的模块来说是一个很好的参考。在这里，我们将使用 Chris 的模块作为模板来构建我们自己的模块，该模块通过查询 Google 的安全浏览 API 来检查 URL 是否与已知的恶意软件或网络钓鱼攻击相关联。你可以在此处访问完整的模块代码：https://github.com/Metasploit-Book/Code-By-Chapter/blob/main/Chapter_09/safebrowse.rb。

Metasploit 框架允许你加载此模块：

```bash
kali@kali:~$ cd /usr/share/metasploit-framework/modules/auxiliary/safebrowse
kali@kali:/usr/share/metasploit-framework/modules/auxiliary/safebrowse$ sudo wget https://github.com/Metasploit-Book/Code-By-Chapter/blob/main/Chapter_09/safebrowse.rb
```

我们将其放在 auxiliary 目录中，以便 Metasploit 可以使用它。但在使用此模块之前，让我们看看它的 Ruby 代码并分解各个部分，以准确了解模块包含的内容。

## 编写模块

模块首先扩展 auxiliary 类并导入 `msf/core` gem（Ruby 中的库术语）：

```ruby
require 'msf/core'

class MetasploitModule < Msf::Auxiliary

  # Exploit mixins should be called first
  include Msf::Exploit::Remote::HttpClient
  include Msf::Auxiliary::Report
```

接下来，它通过包含 `httpClient` mixin 使 HTTP 客户端函数在模块内可用：

```ruby
  def initialize
    super(
      'Name'        => 'Safe Browsing API Check',
      'Version'     => 'Revision: $5$',
      'Description' => "Checks Google's safe browsing list",
      'Author'      => ['Daniel Graham'],
      'License'     => MSF_LICENSE,
      'References'  => [
        [ 'URL', 'https://developers.google.com/safe-browsing/v4/lookup-api' ],
        [ 'URL', 'https://console.cloud.google.com' ]
      ]
    )
    register_options(
      [
        Opt::RHOST('safebrowsing.googleapis.com'),
        Opt::RPORT('443'),
        OptBool.new('SSL', [true, 'Use SSL', true]),
        OptString.new('TARGET_URL', [true, 'URL to Check', '']),
        OptString.new('API_KEY', [true, 'API Key', '']),
        OptString.new('PLATFORM', [false, 'Threat Types', 'WINDOWS']),
      ], self.class)
  end
```

在初始化构造函数中，我们定义了大量信息，当用户在 MSFconsole 中执行 `info` 命令时，Metasploit 会报告这些信息。我们定义了各种选项以及它们是否是必需的。

Google 安全浏览 API 需要加密连接，因此使用 `OptBool.new` 函数将 SSL 选项设置为 true。该函数接受两个参数：第一个是运行 `options` 命令时显示的名称，第二个是一个数组，用于配置选项的三个方面的内容：即是否需要、选项的描述以及是否有默认值。我们将 required 值设置为 true，描述设置为 "Use SSL"，默认值设置为 true。这个 Google API 需要一个 API 密钥，你可以从 Google API 控制台获得。

现在我们已经定义了选项，让我们实现 `run` 方法。当用户输入 `run` 或 `exploit` 时，会调用该方法：

```ruby
  def run
    url = datastore['TARGET_URL']
    apiKey = datastore['API_KEY']
    platform = datastore['PLATFORM']

    postrequest = %{
      "client": {
        "clientId": "Metasploit Framework",
        "clientVersion": "1.x.x"
      },
      "threatInfo": {
        "threatTypes": ["MALWARE", "SOCIAL_ENGINEERING"],
        "platformTypes": ["#{platform}"],
        "threatEntryTypes": ["URL"],
        "threatEntries": [
          {"url": "#{url}"}
        ]
      }
    }

    res = send_request_cgi({
      'uri'     => "/v4/threatMatches:find?key=#{apiKey}",
      'version' => "1.1",
      'method'  => 'POST',
      'data'    => postrequest,
      'headers' => {
        'Content-Type' => 'application/json',
      }
    })
```

首先，我们将提供的选项赋值给局部变量。然后，我们通过调用 `send_request_cgi` 方法创建一个对象，该方法是从 `lib/msf/core/exploit/http.rb` 导入到脚本中的。这个方法连接到 API 并提交请求。一旦 API 处理完请求，该方法返回响应。我们将响应存储在名为 `res` 的变量中。现在我们可以打印结果，并向用户报告任何错误：

```ruby
    print_status("#{res}") # it's nice to see what's going on.
    # ... (错误处理略)
  end
end
```

你需要一个有效的 Google 安全浏览 API 密钥才能成功运行此模块。记住你可以从 Google API 控制台获得一个。我们定义要扫描的目标 URL，设置 API 密钥，最后运行模块。

API 请求成功：Google 服务确认并返回一个对象。在我们的例子中，该对象为空。这意味着该 URL 与任何恶意活动无关。如果匹配，该对象将包含 URL 及其相关信息。作为练习，尝试扩展该模块，使其爬取域以搜索其链接与已知恶意活动相关联的页面。

## 调试模块

调试模块可能具有挑战性，因为如果模块包含错误，Metasploit 框架将不会加载它。有时它会在终端中显示错误，但其他时候会将它们存储在 `framework.log` 文件中。在开发模块时，最好检查这个日志文件，你可以使用 `locate` 命令找到它：

```bash
kali@kali:~$ locate framework.log
```

找到文件后，使用 `tail` 命令查看最近添加的日志：

```bash
kali@kali:~$ tail -20 /home/kali/.msf6/logs/framework.log
```

当你修复了一个错误后，你需要重新启动 Metasploit 框架才能使更改生效。对于更高级的调试，请尝试 `pry-byebug`（https://github.com/deivid-rodriguez/pry-byebug）。Rapid7 有一些关于使用 `pry-byebug` 调试 Metasploit 模块的优秀文章。

## 总结

虽然你可以轻松创建自定义辅助模块，但不要忽视框架中现有的辅助模块。这些模块可能正是你访问额外信息、攻击向量或漏洞所需的工具。请记住，如果你创建了一个有用的辅助模块，请将你的模块上传到 Metasploit 仓库与社区分享。
