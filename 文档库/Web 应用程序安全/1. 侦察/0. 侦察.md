# 第一部分：信息收集（Recon）

与本书其他多处提供的技术概述不同，我将从哲学层面的概述开始这一部分。

要高效地利用 Web 应用，需要广泛的技能。一方面，黑客需要了解网络协议、软件开发技术以及各类应用中常见的漏洞。但另一方面，黑客还需要理解他们所针对的应用。这种知识越深入，就越有效、越适用。

黑客应从功能角度理解应用的目的。它的用户是谁？应用如何产生收入？用户为何选择该应用而非竞争对手？竞争对手有哪些？应用中包含哪些功能？

如果不从非技术角度深入理解目标应用，实际上很难确定哪些数据和功能是重要的。例如，一个用于汽车销售的 Web 应用可能将代表待售汽车的对象（价格、库存等）视为关键任务数据。但一个汽车爱好者可以发布和分享自己车辆改装方案的爱好网站，可能认为用户账户比用户资料中列出的库存更有价值。

在谈论功能而不仅仅是数据时也是如此。许多 Web 应用通过多种方式产生收入，而不是仅依赖一种收入来源。

例如，一个媒体分享平台可能提供月度订阅、投放广告并提供付费下载。其中哪一项对公司最有价值？从可用性角度看，这些变现功能的使用有何不同？有多少用户为每个收入流贡献收入？

最终，Web 应用信息收集是关于收集数据并构建一个模型，该模型将 Web 应用的技术细节和功能细节结合起来，使您能够全面理解 Web 应用的目的和用法。缺少任何一个方面，黑客都无法恰当地针对其攻击。因此，从哲学上讲，Web 应用信息收集是为了生成对目标 Web 应用更深层的理解。在这种哲学模型中，信息是关键——无论其本质上是否是技术性的。

由于这是一本技术书籍，我们的大部分重点将放在从技术角度查找和分析 Web 应用的组件上。然而，我们也会讨论功能分析的重要性以及一些信息组织技术。除此之外，我恳请您在未来遇到信息收集机会时，自行进行非技术性的研究。
