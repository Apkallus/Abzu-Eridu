# 第 8 章：第一部分总结

至此，您应该对 Web 应用侦察的目的有了扎实的基础性理解，并掌握了几种可用于启动侦察工具包的技术。

侦察技术不断发展，准确判断哪些技术优于其他技术可能很困难。因此，您应始终关注新的、有趣的侦察技术——尤其是那些可以快速执行并能自动化以节省重复手动操作宝贵时间的技术。

有时，您的旧技术可能会过时，您可能需要开发新技术来替代它们。例如，随着时间推移，Web 服务器包的安全性不断提高，现在它们会尽最大努力防止任何泄露 Web 服务器软件和版本号的状态信息。

您侦察工具包中的基本技能可能永远不会完全过时，但您可能会发现新技术不断涌现。除了理解当前时代和遗留技术外，您还需要开发映射新技术的方法。

在第一部分中，我强调了记录和组织侦察发现的重要性。但我也建议您记录和整理您的侦察技术。最终，您的侦察工具包将扩展到涵盖许多独特的技术、框架、版本和方法。

有效地记录和组织您的侦察技术，将使您将来更容易将其转化为自动化工具，或者如果您处于指导者位置，也更容易将它们分发和传授给他人。强大的侦察技术常常被当作机构知识保守秘密。如果您开发了有效的新侦察技术，请考虑与更大的安全社区分享。您发现的技术不仅会帮助渗透测试人员，也可能推动应用安全的进步。

最终，您选择如何积累、记录和分发这些技术取决于您自己。我希望本书中奠定的基础能成为您侦察工具包的基石，并在您未来应用安全领域的冒险中为您提供良好服务。
