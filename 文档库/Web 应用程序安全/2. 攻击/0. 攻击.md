# 第二部分：攻击（Offense）

在本书的第一部分，我们探讨了调查和记录 Web 应用结构与功能的多种方法。我们评估了在服务器上查找 API 的方法，包括那些存在于子域名而不仅仅是顶级域名的 API。我们考虑了枚举这些 API 暴露的端点及其接受的 HTTP 动词的方法。

在构建了子域名、API 和 HTTP 动词的地图之后，我们研究了确定每个端点接受何种请求和响应载荷的方法。我们从通用角度出发，也通过寻找开放规范（可以更快地引导我们找到载荷结构）的方法来处理。

在研究了绘制应用 API 结构地图的方法之后，我们开始讨论第三方依赖，并评估了在第一方应用上检测第三方集成的各种方法。通过这次调查，我们学会了如何检测 SPA 框架、数据库和 Web 服务器，并学习了通用技术（如指纹识别）以识别其他依赖的版本。

最后，我们通过讨论可能导致功能保护薄弱的架构缺陷，结束了关于侦察的讨论。通过评估几种常见的不安全 Web 应用架构形式，我们深入了解了匆忙开发的 Web 应用所面临的风险。

现在，在第二部分中，我们将开始学习黑客用于入侵现代 Web 应用的常用技术。在开始第二部分之前，先理解第一部分的技术是有用的。

后续页面中介绍的许多攻击功能强大，有时甚至易于部署，但它们并非适用于所有 API 端点、所有 HTML 表单或所有 Web 链接。在寻找将第二部分中的利用方式应用到现实 Web 应用的方法时，我们可以利用第一部分的侦察技术。

在这里，我们将学习源于不安全 API 端点、UI 中不安全的 Web 表单、设计不当的浏览器标准、配置不当的服务端解析器等的攻击。

通过应用第一部分的概念，我们可以找到 API 端点并判断它们是否编写不安全。我们也可以评估客户端（浏览器）代码，看它是否正确或不安全地处理 DOM 操作。指纹识别客户端框架对于发现应用 UI 中的弱点很有用，因为客户端代码存储在本地且易于评估。如您所见，本书中的技术是相互构建的。

在接下来的几章中，您将学习如何通过多种强大且常见的利用技术来攻击 Web 应用。当您学习这些技术时，请思考前面部分的教训，并尝试头脑风暴，这些侦察技术如何帮助您发现应用中可能应用您即将学到的利用方式的弱点。
