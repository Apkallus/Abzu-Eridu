# 第 9 章：Web 应用程序黑客入门

在第二部分中，我们将建立在侦察技能的基础上，学习可用于利用 Web 应用漏洞的特定攻击方法。在这里，您将学习如何扮演黑客的角色。

我们将使用各种各样的攻击手段来攻击第一部分中介绍的假设 Web 应用 mega-bank.com，所有这些攻击都非常常见，并且在当今许多 Web 应用中经常被发现。从第二部分获得的技能可以轻松迁移到其他地方，只要您同时应用第一部分的技能和技术。到第二部分结束时，您将同时具备发现应用中可被利用漏洞所需的侦察技能，以及构建和部署利用这些安全漏洞的载荷所需的进攻性黑客技能。

## 黑客思维

成为一名成功的黑客需要的不仅仅是一套可客观衡量的技能和知识——它还需要一种非常特殊的思维方式。

软件工程师通过为现有代码库增加功能或改进来衡量的价值增值。软件工程师可能会说：“我添加了 x 和 y 功能；因此，今天是美好的一天。”或者，他们可能会说：“我将 a 和 b 功能的性能提高了 10%”，这暗示了软件工程师的工作虽然与传统职业相比难以衡量，但仍然是可以量化衡量的。

黑客衡量生产力的方式则更难辨别和衡量。这是因为黑客的大部分工作实际上是数据收集和分析。通常，这个过程充斥着误报，在一个未受过教育的旁观者看来可能像是在浪费时间。

大多数黑客不会解构或修改软件，而是分析软件以便与现有代码库协同工作——寻找入口点而不是制造入口点。通常，在寻找入口点时用于分析应用的技能，与本书第一部分中介绍的技能相似（如果不完全相同的话）。

任何给定的代码库都充满了可能被利用的错误。优秀的黑客不断寻找可能导致发现漏洞的线索。

不幸的是，这项工作的性质意味着，即使是优秀的黑客也可能在很长一段时间内没有重大突破。完全有可能花费数周甚至数月的时间分析一个 Web 应用，然后才能找到合适的入口点并设计和交付攻击载荷。

作为黑客，您需要不断强化发现和交付载荷的重要性。除此之外，您还必须仔细记录之前的尝试以及从中吸取的教训。当您从探索小型应用转向攻击大型应用（尤其是以关键功能或数据为目标的应用）时，记录先前工作时对细节的关注将至关重要。

正如我们在软件安全历史中所见，黑客还必须不断提高自己的技能，否则他们将被那些意图阻止黑客进入其软件的人击败。这意味着黑客还必须不断学习，因为随着 Web 的发展，旧技术可能会变得不那么有价值。

黑客首先是一位侦探。优秀的黑客是组织得当的侦探，而伟大的黑客是恰好也拥有出色技术知识和技能的优秀黑客。大师级的黑客具备以上所有特质，并且不断学习和调整自己的技能，因为那些试图阻止他们的人也在提升自己的技能。

## 应用侦察

在第一部分中，我们学习了如何侦察 Web 应用，在此过程中了解其底层技术和结构的各种信息。这一部分是关于利用同一应用中的安全漏洞。

然而，第一部分的经验教训不应被遗忘。这些经验教训在后续将至关重要，您很快就会明白为什么。

在第一部分中，您学习了如何确定应用使用什么类型的 API 向客户端（在我们示例中是浏览器）提供数据。我们了解到大多数现代 Web 应用使用 REST API 来实现这一点。后续章节中的示例将主要涉及通过 REST API 发送载荷。因此，能够确定您试图攻击的应用的 API 类型在此将非常重要。

此外，我们结合使用了公共记录和网络脚本来发现未记录的 API 端点。在本章中，我们开发的攻击方法将适用于许多不同的 Web 应用。正如我们在第一部分中了解的，有时尝试对同一所有者的多个应用使用相同的攻击方法可能很有价值。由于代码复用，您很可能发现针对单个 Web 应用的攻击方法，并将其复制到通过前面章节讨论的技术发现的内部 Web 应用中。

关于端点发现的主题同样有益，因为您可能会遇到多个接受结构相似载荷的 API 端点。也许对 `/users/1234/friends` 的攻击不会返回任何敏感的非公开数据，但 `/users/1234/settings` 可以。

了解如何确定 Web 应用所使用的认证方案也至关重要。如今大多数 Web 应用会向已认证用户提供访客功能的超集。这意味着您可以使用认证令牌攻击的 API 数量更多，并且由于这些请求所运行进程被赋予的权限可能也更大。

在第一部分中，我们还学习了如何识别应用中的第三方依赖（通常是 OSS）。在这一部分中，我们将学习如何查找和定制针对第三方依赖的公开记录的攻击方法。有时我们甚至可能发现由于自定义代码和第三方代码之间集成而产生的安全漏洞。

我们围绕应用架构的讨论和分析在此将很有价值，因为我们可能会发现，虽然应用 A 无法被利用，但应用 B 可以。如果我们无法直接将攻击方法部署到应用 B，我们可以转而研究应用 A 与应用 B 的通信方式，以尝试找到一种将载荷传递到应用 A 的方法，然后由应用 A 再将其传递给应用 B。

总而言之，再次指出，前面章节的侦察技能和后面章节的黑客技能是相辅相成的。黑客和侦察本身都是复杂而有趣的技能，但将它们结合起来，价值会显著增加。
