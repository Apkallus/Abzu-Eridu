# 第 1 章：软件安全的历史

在深入探讨实际的进攻性和防御性安全技术之前，至少了解一些软件安全漫长而有趣的历史是重要的。对过去一百年间重大安全事件的简要概述，应足以让您理解当今 Web 应用所依赖的基础技术。此外，它还将展示安全机制的发展与富有前瞻性的黑客（他们时刻寻找突破或绕过这些机制的机会）之间持续演进的关系。

## 黑客的起源

在过去的二十年里，黑客获得的公众关注和恶名比以往任何时候都多。因此，没有相应背景的人很容易认为黑客概念与互联网紧密相关，且大多数黑客出现在最近 20 年。

但这只是部分事实。虽然全球黑客数量确实随着万维网的兴起而爆炸式增长，但黑客自 20 世纪中叶就已存在——甚至可能更早，具体取决于您对“黑客”的定义。许多专家对现代黑客真正起源的十年存在争议，因为 20 世纪初的一些重大事件与您今天看到的黑客行为有显著相似之处。

例如，在 1910 年代和 1920 年代，发生了一些很可能符合黑客定义的孤立事件，其中大多数涉及篡改莫尔斯电码发送器和接收器，或干扰无线电波传输。然而，虽然这些事件确实发生过，但并不常见，也很难找出因此类技术滥用而中断的大规模行动。

同样重要的是，我并非历史学家。我是一名安全专业人士，背景是解决企业软件中深层次的架构和代码级安全问题。在此之前，我曾多年担任软件工程师，使用各种语言和框架编写 Web 应用。如今我仍在编写软件，形式包括安全自动化，并作为业余爱好为各种项目贡献代码。这意味着我无意争论细节或探讨替代起源故事。相反，本节内容基于多年独立研究汇编而成，重点在于我们能从这些事件中汲取并可应用于今天的教训。

由于本章并非旨在提供全面概述，而是作为关键历史事件的参考，我们的时间线从 20 世纪 30 年代初开始。现在，不再中断，让我们审视一系列塑造了今天黑客与工程师之间关系的历史事件。

## 恩尼格玛密码机，约 1930 年

恩尼格玛密码机使用电力驱动的机械转子来加密和解密通过无线电波发送的基于文本的消息（见图 1-1）。该设备源自德国，并在第二次世界大战期间成为一项重要的技术发展。

图 1-1. 恩尼格玛密码机

该设备看起来像一个大型方形或矩形的机械打字机。每次按键时，转子会移动并记录一个看似随机的字符，然后该字符会被传输到附近所有恩尼格玛密码机。然而，这些字符并非随机；它们由转子的旋转以及设备上可随时修改的多种配置选项决定。任何具有特定配置的恩尼格玛密码机都能读取或“解密”来自具有相同配置的另一台机器发送的消息。这使得恩尼格玛密码机在发送关键消息同时避免截获方面极具价值。

虽然很难确定该机器所使用的旋转加密机制的唯一发明者，但该技术由一家位于德国的名为 Chiffriermaschinen AG 的两人公司推广。在 20 世纪 20 年代，Chiffriermaschinen AG 在德国各地展示该技术，导致德国军方于 1928 年采用它来保护传输中的绝密军事信息。

能够避免远程消息被截获是一项前所未有的根本性发展。在今天的软件世界中，消息截获仍然是黑客试图采用的一种流行技术，通常称为中间人攻击。当今软件使用与一百年前恩尼格玛密码机类似（但强大得多）的技术来防御此类攻击。

尽管恩尼格玛密码机在其时代是一项极其令人印象深刻的技术，但它并非没有缺陷。由于截获和解密的唯一条件是拥有一台与发送方配置相同的恩尼格玛密码机，单个泄露的配置日志（用今天的话说就是私钥）可能使整个恩尼格玛密码机网络变得毫无用处。

为了解决这个问题，任何通过恩尼格玛密码机发送消息的群组都会定期更改其配置设置。重新配置恩尼格玛密码机是一个耗时的过程。首先，配置日志必须亲自交换，因为当时尚不存在远程安全共享方式。在两台机器和两名操作员之间共享配置日志可能并不困难。但对于更大的网络，比如 20 台机器，则需要多名信使来传递配置日志——每个信使都增加了配置日志被截获和窃取，甚至泄露或出售的可能性。

共享配置日志的第二个问题是，要使恩尼格玛密码机能够读取、加密和解密来自其他恩尼格玛密码机的新消息，需要对机器本身进行手动调整。这意味着必须有一名经过专门培训的专职人员在场，以备需要配置更新时使用。这一切都发生在软件出现之前的时代，因此这些配置调整需要篡改硬件并调整插线板的物理布局和接线。调整者需要具备电子学背景，这在 20 世纪初非常罕见。

由于更新这些机器既困难又耗时，更新通常每月进行一次——对于关键任务通信线路则每日进行。如果密钥被截获或泄露，当月剩余时间内的所有传输都可能被恶意行为者（相当于今天的黑客）截获。

这些恩尼格玛密码机使用的加密类型现在被称为对称密钥算法，这是一种特殊的密码，允许使用单个加密密钥对消息进行加密和解密。这类加密至今仍在软件中使用，以保护传输中（发送方和接收方之间）的数据安全，但在经典模型（因恩尼格玛密码机而普及）的基础上有了许多改进。

在软件中，密钥可以变得复杂得多。现代密钥生成算法产生的密钥非常复杂，即使用最快的现代硬件尝试所有可能的组合（暴力破解或暴力攻击）也轻松需要超过一百万年。此外，与过去的恩尼格玛密码机不同，软件密钥可以快速更改。

根据用例，密钥可以在每个用户会话（每次登录）、每次网络请求或按计划间隔重新生成。当在软件中使用此类加密时，泄露的密钥可能在每次请求重新生成的情况下仅暴露单个网络请求，或者在每次登录（每会话）重新生成的最坏情况下，暴露几个小时。

如果您将现代密码学的谱系追溯得很远，最终会到达 20 世纪 30 年代的第二次世界大战。可以肯定地说，恩尼格玛密码机是保护远程通信的一个重要里程碑。由此，我们可以得出结论，恩尼格玛密码机是后来成为软件安全领域的关键发展。

恩尼格玛密码机对于那些最终被称为“黑客”的人来说也是一项重要的技术发展。轴心国在第二次世界大战期间采用恩尼格玛密码机，给盟军带来了开发破解加密技术的巨大压力。德怀特·艾森豪威尔将军本人声称，这样做对于战胜纳粹至关重要。

1932 年 9 月，一位名叫马里安·雷耶夫斯基的波兰数学家获得了一台被偷来的恩尼格玛密码机。与此同时，一位名叫汉斯-蒂洛·施密特的法国间谍能够向他提供 1932 年 9 月和 10 月的有效配置。这使得马里安能够截获消息，并从中开始分析恩尼格玛密码机加密的奥秘。

马里安试图确定该机器在机械和数学上是如何工作的。他想了解机器硬件的特定配置如何导致输出完全不同的加密消息。

马里安的破解尝试基于关于何种机器配置会导致特定输出的若干理论。通过分析加密消息中的模式，并根据机器机械原理提出理论，马里安和两位同事杰日·罗日茨基以及亨里克·扎加尔斯基最终逆向工程了整个系统。凭借团队对恩尼格玛转子机械结构和面板配置的深入理解，他们能够对哪些配置会导致哪些加密模式做出有根据的猜测。然后他们可以以相当高的准确度重新配置面板，经过几次尝试后，开始读取加密的无线电流量。到 1933 年，该团队每天都在截获和解密恩尼格玛密码机的通信流量。

与今天的黑客非常相似，马里安和他的团队截获并逆向工程加密方案，以获取由他们自身以外来源生成的有价值数据。基于这些原因，我认为马里安·雷耶夫斯基及其协助团队是世界上最早期的黑客之一。

在接下来的几年里，德国不断增加其恩尼格玛密码机加密的复杂性。这是通过逐渐增加加密一个字符所需的转子数量来实现的。最终，逆向工程配置的复杂性变得太大，马里安的团队无法在合理时间内破解。这一发展也很重要，因为它展示了黑客与试图阻止黑客行为者之间不断演变的关系。

这种关系今天仍在继续，富有创造力的黑客不断迭代和改进他们入侵软件系统的技术。而另一方面，聪明的工程师也在不断开发新技术来防御最具创新性的黑客。

## 自动破解恩尼格玛密码，约 1940 年

艾伦·图灵是一位英国数学家，以开发如今被称为“图灵测试”的测试而闻名。图灵测试旨在根据将机器生成的对话与真实人类对话区分开的难度来评估机器生成的对话。该测试通常被认为是人工智能领域的奠基哲学之一。

虽然艾伦·图灵以在人工智能方面的工作而闻名，但他也是密码学和自动化领域的先驱。事实上，在第二次世界大战之前和期间，艾伦的研究重点主要是密码学而非人工智能。从 1938 年 9 月开始，艾伦在政府代码与密码学校（GC&CS）兼职工作。GC&CS 是英国陆军资助的一个研究和情报机构，位于英格兰布莱切利公园。

艾伦的研究主要集中在恩尼格玛密码机的分析上。在布莱切利公园，艾伦与当时经验丰富的密码学家、他的导师迪利·诺克斯一起研究恩尼格玛密码机的密码学。

与之前的波兰数学家非常相似，艾伦和迪利想找到一种方法来破解（现在显著更强大的）德国恩尼格玛密码机的加密。由于他们与波兰密码局的合作，两人获得了马里安团队近十年前产生的所有研究资料。这意味着他们已经对机器有了深入了解。他们理解转子和布线之间的关系，也知道设备配置与输出加密之间的关系（图 1-2）。

图 1-2. 一对用于校准恩尼格玛密码机传输配置的恩尼格玛转子，相当于更改数字密码主密钥的模拟方式

马里安的团队能够在加密中找到模式，从而对机器的配置做出有根据的猜测。但如今机器的转子数量已增加多达十倍，这种方法无法扩展。在尝试所有潜在组合所需的时间内，新的配置早已发布。因此，艾伦和迪利在寻找一种不同类型的解决方案；一种可扩展、并能用于破解新型加密的解决方案。他们想要一个通用解决方案，而非高度专业化的方案。

炸弹机（bombe）是一种电力驱动的机械装置，它试图根据对从恩尼格玛密码机发送的消息的机械分析，自动逆向工程出恩尼格玛密码机中机械转子的位置（见图 1-3）。

第一批炸弹机由波兰人建造，试图自动化马里安的工作。不幸的是，这些设备设计用于确定具有非常特定硬件的恩尼格玛密码机的配置。特别是，它们对具有超过三个转子的机器无效。由于波兰炸弹机无法扩展以应对更复杂的恩尼格玛密码机的发展，波兰密码学家最终回到使用手动方法试图解读德国战时消息。

图 1-3. 第二次世界大战期间使用的早期布莱切利公园炸弹机（注意用于快速执行恩尼格玛配置解密的许多排转子）

艾伦·图灵认为，原始机器失败是因为它们不是以通用方式编写的。为了开发一种能够破解任何恩尼格玛配置（无论转子数量多少）的机器，他从一个简单的假设开始：为了正确设计解密加密消息的算法，您必须首先知道该消息中存在的一个单词或短语及其位置。

幸运的是，艾伦，德国军方有非常严格的通信标准。每天，一条包含详细区域天气报告的消息会通过加密的恩尼格玛无线电波发送。这就是德国军方确保所有部队了解天气状况，同时又不向任何收听无线电的人公开的方式。德国人不知道艾伦的团队能够逆向工程这些报告的目的和位置。

了解通过正确配置的恩尼格玛密码机发送的输入（天气数据），使得算法确定输出变得容易得多。艾伦利用这一新发现的知识，确定了一种炸弹机配置，该配置可以独立于其试图破解的恩尼格玛密码机所依赖的转子数量而工作。

艾伦申请预算建造一台炸弹机，该机器能够准确检测截获和读取德国恩尼格玛密码机加密消息所需的配置要求。预算获批后，艾伦建造了一台由 108 个鼓组成的炸弹机，这些鼓可以以高达 120 RPM 的速度旋转。这台机器可以在短短 20 分钟内运行近 20,000 种可能的恩尼格玛密码机配置。这意味着任何新配置都可以被迅速破解。恩尼格玛加密不再是安全的通信手段。

今天，艾伦的逆向工程策略被称为已知明文攻击（KPA）。这是一种通过提供先前的输入/输出数据而变得更高效的算法。现代黑客使用类似技术来破解软件中存储或使用的数据加密。艾伦建造的机器标志着历史上的一个重要时刻，因为它是有史以来最早建造的自动化黑客工具之一。

## 电话“飞客”（Phreaking），约 1950 年

在 20 世纪 30 年代恩尼格玛密码机兴起以及主要世界大国之间发生的密码学之战之后，电话的引入是我们时间线中的下一个重大事件。电话使普通民众能够远距离快速相互通信。随着电话网络的发展，它们需要自动化才能大规模运行。

在 20 世纪 50 年代末，AT&T 等电信公司开始实施新电话，这些电话可以根据电话机发出的音频信号自动路由到目的地号码。按下电话键盘上的按键会发出特定的音频频率，该频率通过线路传输并由交换中心的机器解释。交换机器将这些声音转换为数字，并将呼叫路由到适当的接收方。

该系统被称为音频拨号（tone dialing），它是大规模电话网络不可或缺的重要发展。音频拨号大大降低了运行电话网络的开销，因为网络不再需要接线员手动连接每个呼叫。相反，一个监督网络的接线员可以在同一时间内管理数百个呼叫，而以前只能处理一个呼叫。

在短时间内，一小群人开始意识到任何基于音频音调解释构建的系统都可能被轻易操纵。只需学习如何在电话听筒旁重现相同的音频频率，就可能干扰设备的预期功能。热衷于操纵这项技术的业余爱好者最终被称为“飞客”（phreakers）——一种专门破坏或操纵电话网络的早期黑客。“Phreaking”一词的真正起源不得而知，尽管它有几个普遍接受的可能的起源。最常见的说法是它源自两个词：“freaking”和“phone”。

还有一个我认为更合理的替代起源说法。我认为“phreaking”一词源自“audio frequency”（音频频率），以回应当时电话使用的音频信令语言。我认为这个解释更合理，因为该术语的起源在时间上非常接近 AT&T 原始音频拨号系统的发布。在音频拨号之前，电话呼叫更难被篡改，因为每个呼叫都需要接线员连接两条线路。

我们可以将飞客追溯到几个事件，但早期飞客最臭名昭著的案例是 2600 Hz 音调的发现和利用。AT&T 内部使用 2600 Hz 音频频率来信号表示呼叫已结束。它本质上是原始音频拨号系统中内置的“管理命令”。发射 2600 Hz 音调会阻止电信交换系统意识到呼叫仍处于打开状态（将呼叫记录为已结束，尽管它仍在进行中）。这使得昂贵的国际呼叫可以被拨打，而不会产生账单或发送给呼叫者。

2600 Hz 音调的发现通常归因于两个事件。首先，一个名叫乔·恩格瑞西亚的小男孩以能吹出 2600 Hz 的音调而闻名，据报道他会向朋友炫耀吹出能阻止电话拨号的音调。有些人认为乔是原始的飞客之一，尽管他的发现是偶然的。

后来，乔·恩格瑞西亚的一个朋友约翰·德雷珀发现，Cap'n Crunch 麦片盒中附带的玩具哨子能模仿 2600 Hz 的音调。小心使用这个哨子也可以用同样的技术产生免费的长途电话。这些技术的知识传遍了西方世界，最终导致了能够通过按键匹配特定音频频率的硬件产生。

这些硬件设备中的第一个被称为蓝盒子（blue box）。蓝盒子能播放近乎完美的 2600 Hz 信号，使任何拥有它的人都能利用电信交换系统中固有的免费通话漏洞。蓝盒子只是自动化飞客硬件的开始，后来的飞客们继续篡改公用电话、在不使用 2600 Hz 信号的情况下阻止计费周期开始、模拟军事通信信号，甚至伪造来电显示。

由此我们看到，早期电话网络的设计者只考虑了正常人和他们的通信目标。在今天的软件世界中，这被称为“最佳情况”设计。基于此进行设计是一个致命缺陷，但它成为一个至今仍然重要的教训：在设计复杂系统时，始终首先考虑最坏情况。

最终，音频拨号系统固有弱点的知识变得更为广泛传播，这导致分配预算来开发对策，以保护电信利润和呼叫完整性免受飞客攻击。

## 反飞客技术，约 1960 年

在 20 世纪 60 年代，电话配备了称为双音多频（DTMF）信令的新技术。DTMF 是一种基于音频的信令语言，由贝尔系统开发，并以更常见的商标“Touch Tones”注册专利。DTMF 与我们今天所知的由三列四行数字组成的电话拨号布局有着内在联系。DTMF 电话上的每个按键会发出两个非常具体的音频频率，而不是像原始音频拨号系统那样发出单个频率。

下表表示旧式电话按键时发出的“Touch Tones”或声音（单位：赫兹）：

| 按键 | 列频率 |
|------|--------|
| 行频率 | 1209 Hz | 1336 Hz | 1477 Hz |
| 697 Hz | 1 | 2 | 3 |
| 770 Hz | 4 | 5 | 6 |
| 852 Hz | 7 | 8 | 9 |
| 941 Hz | * | 0 | # |

DTMF 的发展很大程度上是因为飞客们利用了音频拨号系统，因为那些系统太容易被逆向工程。贝尔系统认为，由于 DTMF 系统同时使用两种截然不同的音调，恶意行为者将更难利用它。

DTMF 音调不能轻易被人的声音或哨子复制，这意味着该技术比其前身安全得多。DTMF 是为对抗当时黑客（飞客）而引入的成功安全开发的一个典型例子。

DTMF 音调生成的机制非常简单。每个按键后面都有一个开关，向内部扬声器发出信号，发射两种频率：一种基于按键的行频率，一种基于按键的列频率。因此使用了“双音”一词。

DTMF 被国际电信联盟（ITU）采纳为标准，后来也用于有线电视（指定商业广告休息时间）以及电话。

DTMF 是一项重要的技术发展，因为它表明如果进行适当规划，系统可以设计得更难被滥用。注意这些 DTMF 音调最终也被复制了，但所需的努力要大得多。最终，交换中心转向数字（而非模拟）输入，几乎消除了所有飞客行为。

## 计算机黑客的起源，约 1980 年

1976 年，苹果公司发布了 Apple 1 个人电脑。这台电脑并非开箱即用的预配置整机，而是需要购买者自行提供多个组件并将其连接到主板上。仅制造并售出了几百台该设备。

1982 年，Commodore International 发布了其竞争产品 Commodore 64，这是一台开箱即用完全配置好的个人电脑。它自带键盘，支持音频，甚至可以用于彩色显示器。

Commodore 64 在 20 世纪 90 年代初之前每月售出近 50 万台。自此以后，个人电脑的销售趋势在接下来的几十年里逐年持续增长。计算机很快成为家庭和企业中的常用工具，并接管了常见的重复性任务，如财务管理、人力资源、会计和销售。

1983 年，美国计算机科学家弗雷德·科恩创建了第一个计算机病毒。他编写的病毒能够自我复制，并很容易通过软盘从一台个人电脑传播到另一台。他能够将病毒存储在一个合法程序中，使其对任何没有源代码访问权限的人隐藏。弗雷德·科恩后来成为软件安全的先驱，证明了用算法从合法软件中检测病毒几乎是不可能的。

几年后，即 1988 年，另一位美国计算机科学家罗伯特·莫里斯成为第一个部署病毒感染研究实验室以外计算机的人。该病毒被称为“莫里斯蠕虫”，其中“蠕虫”是一个新词，用来描述自我复制的计算机病毒。莫里斯蠕虫在其发布的第一天就传播到约 15,000 台联网计算机。

历史上第一次，美国政府介入考虑针对黑客行为的官方法规。美国政府问责局（GAO）估计该病毒造成的损失为 10,000,000 美元。罗伯特被判处三年缓刑、四百小时社区服务和 10,050 美元罚款。这使他成为美国第一个被定罪的黑客。

如今，大多数黑客不再构建感染操作系统的病毒，而是将目标瞄准 Web 浏览器。现代浏览器提供了极其强大的沙箱机制，使得网站很难在未经用户明确许可的情况下，在浏览器外部（针对主机操作系统）运行可执行代码。

虽然今天的黑客主要瞄准可通过 Web 浏览器访问的用户和数据，但与针对操作系统的黑客有许多相似之处。可扩展性（从一个用户跳到另一个用户）和伪装（将恶意代码隐藏在合法程序中）是攻击 Web 浏览器所采用的技术。

如今，攻击通常通过电子邮件、社交媒体或即时消息分发来实现扩展。一些黑客甚至建立合法网站的合法网络来推广单个恶意网站。

恶意代码通常隐藏在看起来合法的界面后面。网络钓鱼（凭证窃取）攻击发生在看起来和感觉与社交媒体或银行网站完全相同的网站上。浏览器插件经常被发现窃取数据，有时黑客甚至找到在他们不拥有的网站上运行自己代码的方法。

## 万维网的兴起，约 2000 年

万维网（WWW）在 20 世纪 90 年代兴起，但其流行度在 90 年代末和 21 世纪初开始爆炸式增长。

在 90 年代，Web 几乎完全被用作共享 HTML 编写文档的方式。网站不关注用户体验，且很少有网站允许用户向服务器发送任何输入以修改网站流程。图 1-4 显示了 1997 年的 Apple.com 网站，仅包含信息性数据。

图 1-4. Apple.com 网站，1997 年 7 月；所呈现的数据纯粹是信息性的，用户无法注册、登录、评论或在会话之间持久化任何数据

21 世纪初标志着互联网的新时代，因为网站开始存储用户提交的数据，并根据用户输入修改网站功能。这是一个关键发展，后来被称为 Web 2.0。Web 2.0 网站允许用户通过超文本传输协议（HTTP）将输入提交到 Web 服务器，然后服务器存储输入并在请求时与其他用户共享，从而实现用户之间的协作。

这种构建网站的新理念催生了我们今天所知的社交媒体。Web 2.0 实现了博客、维基、媒体共享网站等等。

这种 Web 理念的根本性变化导致 Web 从文档共享平台转变为应用分发平台。图 1-5 显示了 2007 年的 Apple.com 商店前台，您可以在此购买商品。注意右上角的账户链接，表明该网站支持用户账户和数据持久化。该账户链接在 2000 年代 Apple 网站的先前版本中就存在，但在 2007 年被提升到用户体验的右上角，而不是底部的链接。此前可能是实验性或未被充分利用。

图 1-5. Apple.com，2007 年 10 月，展示了一个可以在线购买商品的商店前台

网站架构设计方向的巨大转变也改变了黑客针对 Web 应用的方式。到那时，已经为保护服务器和网络安全做出了重大努力——这两个是过去十年黑客的主要攻击向量。随着类应用网站的出现，用户成为了黑客的完美目标。

这是一个完美的布局。用户很快就能通过 Web 访问关键功能。军事通信、银行转账等最终都将通过 Web 应用（一个像桌面应用一样运作的网站）完成。不幸的是，当时几乎没有安全控制措施来保护用户免受针对他们的攻击。此外，关于黑客或互联网运行机制的教育非常匮乏。21 世纪初的早期互联网用户中，很少有人能开始理解为他们工作的底层技术。

在 21 世纪初，第一次被广泛报道的拒绝服务（DoS）攻击导致 Yahoo!、Amazon、eBay 和其他流行网站瘫痪。2002 年，微软用于浏览器的 ActiveX 插件出现了一个漏洞，允许怀有恶意的网站调用远程文件上传和下载。到 2000 年代中期，黑客经常利用“网络钓鱼”网站窃取凭证。当时没有控制措施来保护用户免受这些网站的侵害。

跨站脚本（XSS）漏洞允许黑客的代码在合法网站内以用户浏览器会话运行，在此期间在 Web 上横行霸道，因为浏览器厂商尚未为这类攻击建立防御。21 世纪的许多黑客尝试源于驱动 Web 的技术是为单一用户（网站所有者）设计的。当这些技术被用于构建允许众多用户之间共享数据的系统时，它们就会崩溃。

## 现代黑客（约 2015 年至今）

讨论以往时代的黑客是为了为我们在本书中的旅程奠定基础。通过分析 20 世纪 30 年代恩尼格玛密码机的发展和密码分析，我们深入了解了安全的重要性以及他人为破坏安全所付出的努力。

在 20 世纪 40 年代，我们看到了安全自动化的早期用例。这个特定案例是由攻击者和防御者之间持续的战斗所驱动的。在此案例中，恩尼格玛密码机技术已经进步到无法再通过手动密码分析技术可靠破解的程度。艾伦·图灵转向自动化以击败安全改进。

20 世纪 50 年代和 60 年代向我们展示了黑客与修补匠有很多共同点。我们还了解到，设计时未考虑恶意用户的技术最终会被攻破。在设计和部署大规模、面向广泛用户群的技术时，我们始终必须考虑最坏情况。

在 20 世纪 80 年代，个人电脑开始流行。大约在这个时候，我们开始看到我们今天所认可的黑客出现。这些黑客利用软件赋予的能力，将病毒伪装在合法应用程序中，并利用网络快速传播病毒。

最后，万维网的引入和快速普及导致了 Web 2.0 的发展，这改变了我们对互联网的思考方式。互联网不再仅仅是共享文档的媒介，它变成了共享应用的媒介。因此，出现了新型利用方式，它们利用用户而非网络或服务器。这是一个根本性变化，今天仍然适用，因为如今大多数黑客已转向通过浏览器攻击 Web 应用，而非桌面软件和操作系统。

让我们跳转到 2019 年，我开始撰写本书第一版的那一年。在撰写本文时，Web 上有成千上万个网站，背后是价值数百万甚至数十亿美元的公司。事实上，许多公司的全部收入都来自其网站（例如 Google、Facebook、Yahoo!、Reddit 等）。

YouTube 允许用户之间以及用户与应用本身进行交互（见图 1-6）。评论、视频上传和图像上传均受支持。所有这些上传都有可变的权限，允许上传者决定内容对谁可见。许多托管数据永久持久并在会话之间保留，几个功能的变化在近实时（通过通知）在用户之间反映。此外，大量关键功能被卸载到客户端（浏览器），而非驻留在服务器上。

图 1-6. YouTube.com（现归 Google 所有）是 Web 2.0 网站的绝佳示例

一些传统的桌面软件公司现在正试图将其产品线迁移到 Web，也就是如今所谓的“云”，这只是一个复杂的服务器网络。例如 Adobe 的 Creative Cloud，这是一项订阅服务，通过 Web 提供 Photoshop 和其他 Adobe 工具；还有 Microsoft Office，它提供 Word 和 Excel，但现在作为 Web 应用。

由于大量资金沉淀在 Web 应用中，风险前所未有之高。这意味着当今 Web 上的应用正适合被利用，而利用它们的回报极高。

对于黑客和注重安全的工程师来说，这确实是最佳时代之一。两者的工作需求都很高，且在法律两边都有机会。

浏览器已经比 10 年前显著先进。伴随着这一进步出现了大量新的安全特性。我们用来访问互联网的网络协议也有所改进。

当今的浏览器在不同源（origin）的网站之间提供了非常强大的隔离，遵循称为同源策略（SOP）的安全规范。这意味着即使网站 A 和 B 同时打开，或者一个作为 iframe 嵌入到另一个中，网站 B 也无法访问网站 A。

浏览器还接受一种称为内容安全策略（CSP）的新安全配置。CSP 允许网站开发者指定各种安全级别，例如脚本是否应该能够在（HTML 中）内联执行。这使 Web 开发者能够进一步保护其应用免受常见威胁。

HTTP（用于发送 Web 流量的主要协议）也从安全角度得到了改进。HTTP 采用了 SSL 和 TLS 等协议，对网络中传输的任何数据强制执行严格加密。这使得中间人攻击很难成功实施。

由于浏览器安全的这些进步，如今许多最成功的黑客实际上瞄准的是由开发者编写并在其 Web 应用中运行的逻辑。通过利用应用代码中的缺陷来成功攻破网站，比攻击浏览器本身要容易得多。对黑客来说幸运的是，如今的 Web 应用比过去的 Web 应用大得多且复杂得多。

如今，一个知名 Web 应用通常可以有数百个开源依赖项、与其他网站的集成、多种类型的多个数据库，并且可以从多个位置的多个 Web 服务器提供。这些类型的 Web 应用是您最有可能成功利用的，也是我们在本书中重点关注的类型。

总而言之，当今的 Web 应用比其前辈更大、更复杂。作为黑客，您现在可以专注于通过利用应用代码中的逻辑错误来攻破 Web 应用。这些错误往往是 Web 应用中高级用户交互的副作用。

过去十年的黑客将大量时间花在攻破服务器、网络和浏览器上。现代黑客则将大部分时间花在利用代码中存在的漏洞攻破 Web 应用上。

## 小结

软件安全的起源以及试图绕过该安全的黑客的起源至少可追溯约一百年。当今软件建立在从过去技术中汲取的经验教训之上，其安全性也同样如此。

过去的黑客针对应用的方式与今天不同。当应用堆栈的一部分变得越来越安全时，黑客就会转向针对新兴技术。这些新技术通常不具备同等水平的内置安全控制，只有通过反复试验，工程师才能设计和实施适当的安全控制。

类似于过去简单的网站充斥着安全漏洞（尤其是在服务器和网络层面），现代 Web 应用为攻击者带来了新的攻击面，并且这些面正在被积极利用。这个简短的历史背景很重要，因为它突显了当今对 Web 应用的安全关切只是循环过程中的一个阶段。未来的 Web 应用将更加安全，而黑客很可能会转向新的攻击面（例如实时通信或 WebSockets）。

每种新技术都带有其独特的攻击面和漏洞。成为优秀黑客的一种方法是始终紧跟最新技术——这些技术通常带有尚未公开或在网上发现的漏洞。

与此同时，本书将向您展示如何攻破和防御现代 Web 应用。但现代的攻防安全技术只是您应从本书中获取的学习内容之一。最终，能够找到自己对安全问题的解决方案是您作为安全专业人士所能拥有的最有价值的技能。如果您能从后续章节中培养与安全相关的批判性思维和解决问题的技能，那么当发现新的或不同寻常的利用方式时，或者当遇到前所未有的安全机制阻碍您时，您将能脱颖而出。
