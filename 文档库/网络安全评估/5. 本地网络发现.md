# 第 5 章 本地网络发现

本章描述了用于评估本地网络配置的策略。目标包括枚举可用资源以及利用弱点访问数据。

这里描述的大多数协议是不可路由的（使用数据链路层和本地广播地址），因此你只能从本地网络对其进行评估。在测试过程中，你可能会发现自己处于两种情况之一：要么你在现场并拥有对网络的物理访问，要么你已经获得了对其他地方系统的远程访问。这里讨论的一些攻击需要物理网络访问，但大多数不需要。

## 数据链路协议

以太网被广泛用作底层的物理层和数据链路层格式，由 IEEE 802.3 和 802.2 标准工作组定义。环境中通常采用许多增强功能，这些功能由 IEEE 802.1 工作组批准：

- **802.1D**（生成树协议）
- **802.1Q**（VLAN 桥接）
- **802.1X**（基于端口的网络访问控制）

还存在许多专有扩展，由包括 Cisco 在内的供应商定义和使用。802.3、802.2 和 802.1 标准、专有协议、IP 和 OSI 模型之间的关系如图 5-1 所示。尽管其他数据链路标准（例如 802.11 WiFi）超出了本书的范围，但这里描述的许多攻击策略仍然适用。

### 802.3 以太网测试

以太网容易受到被动网络嗅探和主动攻击（主要通过 ARP 缓存投毒和 CAM 表溢出）的影响，导致对等方之间的流量被破坏。

在制造过程中，每个网络适配器都被编程了一个唯一的 48 位 MAC 地址。这些地址被 IEEE 802 网络（包括 802.3 以太网和 802.11 WiFi）中的系统用来相互寻址。因此，它们的接口会处理发往它们的数据。你可以通过启用**混杂模式**来移除网络适配器的 MAC 过滤器——结果所有接收到的帧，无论目的地如何，都会被处理。包括 **Wireshark**[^1] 和 **Cain & Abel**[^2] 在内的工具启用混杂模式并显示捕获的网络流量。

[^1]: 参见 https://www.wireshark.org。
[^2]: 参见 http://www.oxid.it/cain.html。

#### 被动网络嗅探

在本地网络评估工作中，一个很好的起点是运行嗅探器并评估本地网络暴露的材料。图 5-2 显示了在以太网接口上运行的 Wireshark。

在这个例子中，我们看到 Wireshark 记录了以下内容：

- Wellfleet Breath of Life（BOFL）帧
- 简单服务发现协议（SSDP）广播包
- Microsoft 计算机浏览器服务公告
- 地址解析协议（ARP）请求和应答
- 一个 802.1X EAPOL 开始帧
- Dropbox 发现广播

**图 5-2. 使用 Wireshark 嗅探本地流量**

通过查看捕获的帧和数据包，我们可以了解 IP 范围、子网大小、MAC 地址和主机名的详细信息。如果网络配置错误或交换结构处于压力之下，攻击者可以通过被动网络嗅探捕获敏感材料。

如果交换式以太网配置正确，你将只看到广播帧和发往你 MAC 地址的数据。要破坏其他主机之间发送的流量，你必须考虑主动攻击策略，如下所述。

#### ARP 缓存投毒

ARP 在本地网络中用于将 IPv4 地址映射到底层 MAC 地址。示例 5-1 演示了由 `tcpdump` 捕获的正常 ARP 操作。在这种情况下，`192.168.0.1` 解析并向 `192.168.0.10` 发送 ICMP 回显请求（ping）。首先，一个 ARP `who-has` 消息被广播到网络。接下来，目标主机响应（使用 ARP `is-at` 应答，提供其 MAC 和 IP 地址），然后 ICMP 操作通过 IP 完成。

**示例 5-1. 使用 tcpdump 捕获的 ARP 和 ICMP 流量**

```bash
root@kali:~# tcpdump -ennqti eth0 \( arp or icmp \)
tcpdump: listening on eth0
0:80:c8:f8:4a:51 ff:ff:ff:ff:ff:ff : arp who-has 192.168.0.10 tell 192.168.0.1
0:80:c8:f8:5c:73 0:80:c8:f8:4a:51 : arp reply 192.168.0.10 is-at 0:80:c8:f8:5c:73
0:80:c8:f8:4a:51 0:80:c8:f8:5c:73 : 192.168.0.1 > 192.168.0.10: icmp: echo request
0:80:c8:f8:5c:73 0:80:c8:f8:4a:51 : 192.168.0.10 > 192.168.0.1: icmp: echo reply
```

每个主机维护一个最近映射的 IP 和 MAC 地址对的缓存。你可以使用大多数操作系统中的 `arp -a` 命令本地查看缓存内容，如示例 5-2 所示。

**示例 5-2. 显示本地 ARP 缓存内容**

```bash
root@kali:~# arp -a
? (192.168.0.1) at 0:80:c8:f8:4a:51 [ether] on eth0
? (192.168.0.10) at 0:80:c8:f8:5c:73 [ether] on eth0
? (192.168.0.35) at 4:f1:3e:e1:a7:c9 [ether] on eth0
? (192.168.0.53) at 78:fd:94:1d:2f:aa [ether] on eth0
? (192.168.0.180) at 34:2:86:7c:63:b1 [ether] on eth0
```

ARP 是无状态的且缺乏身份验证。因此，该协议容易通过发送未经请求的 ARP 应答而被投毒。通过将其 MAC 地址注入受害者系统的 ARP 缓存，攻击者可以实现中间人（MITM），如图 5-3 所示。在这种情况下，系统 A 和 B 的缓存都被注入了 E 的 MAC 地址。

**图 5-3. ARP 缓存投毒**

为了破坏对等方之间（例如主机与其本地网关之间）双向的流量，首先在攻击者系统上启用 IP 转发，然后发送未经请求的 ARP 应答以投毒双方系统。包括 **Cain & Abel** 和 **Ettercap**[^3] 在内的工具自动化了这一过程。实现 MITM 后，攻击者可以使用许多实用程序获取数据和秘密，包括：

[^3]: 参见 GitHub 上的 Ettercap。[链接](https://github.com/Ettercap/ettercap)

- **sslstrip**[^4]：降级 HTTPS 会话
- **easy-creds**[^5]：用于从 Ettercap、sslstrip 等中收集凭据
- Laurent Gaffié 的 **Responder**[^6]：响应名称解析请求
- **Evilgrade**[^7]：通过服务冒充向受害者提供恶意内容
- **Metasploit**：执行服务冒充和进一步攻击

[^5]: 参见 GitHub 上的 easy-creds。[链接](https://github.com/brav0hax/easy-creds)

[^7]: 参见 GitHub 上的 Evilgrade。[链接](https://github.com/infobyte/evilgrade)

在 Microsoft Windows 环境中，通过 MS15-011 组策略 SMB MITM 也可以实现命令执行[^8]。

[^8]: Nicolas Economou，“MS15-011 — Microsoft Windows Group Policy Real Exploitation via a SMB MiTM Attack”，Core Security Blog，2015 年 5 月 18 日。[链接](https://www.coresecurity.com/core-labs/articles/ms15-011-microsoft-windows-group-policy-real-exploitation-via-a-smb-mitm-attack)

ARP 不用于 IPv6 网络中的解析。相反，邻居发现协议（NDP）通过链路层使用多播 ICMPv6 数据包，本章后面将对此进行描述。

#### CAM 表溢出

以太网交换机使用**内容可寻址内存（CAM）表**将 MAC 地址和 VLAN 分配映射到各个端口，以便网络帧被正确交付。攻击者可以使用 **macof** 实用程序（Dug Song 的 **dsniff** 软件包的一部分[^9]）向交换机泛洪随机的以太网帧和 IP 数据包，导致 CAM 表溢出。由于无法将入站帧映射到其目的地，交换机将**失败开放**并将它们广播到所有端口（变成一个集线器）。

[^9]: 参见 Monkey.org 上的 dsniff。[链接](https://www.monkey.org/~dugsong/dsniff/)

示例 5-3 演示了 `macof` 的使用；使用 Kali Linux 中的 `eth1` 网络接口泛洪本地交换机。一个未加固的交换机的 CAM 表将在几分钟内溢出，然后可以使用网络嗅探器捕获泄露的帧。

**示例 5-3. 使用 macof 进行 CAM 表溢出**

```bash
root@kali:~# macof -i eth1
```

### 802.1Q VLAN

VLAN 在企业中用于划分网络（例如数据、语音和管理）并创建单独的广播域。除了减少不必要的广播流量外，802.1Q 标记还限制了 ARP 缓存投毒和其他本地攻击的范围。管理员定义任意的 VLAN ID 值（0-4095），用于标记以太网帧并建立网段。图 5-4 展示了一个典型环境——在交换机之间创建中继以分发本征 VLAN 上的所有流量，并将各个端口分配给其他 VLAN。

**图 5-4. 使用 802.1Q VLAN 标记的典型网络**

VLAN 跨越交换机并不常见，因为应该使用路由器在网段之间中继流量，并避免可能影响延迟的数据链路层问题（例如与广播帧相关的泛洪）。

以下是 802.1Q 实现的一些常见风险：

- 滥用动态中继以破坏 VLAN 和数据（交换机欺骗）
- 双标记帧以向其他 VLAN 发送数据
- 绕过私有 VLAN 端口隔离的第 3 层攻击[^10]

[^10]: 参见 CVE-2005-4441。

这些攻击将在以下各节中描述。

#### 动态中继

在加固的环境中，你的端口将具有静态分配，将你限制在特定的 VLAN 中。然而，许多交换机默认支持**动态中继协议（DTP）**，攻击者可以滥用该协议来模拟交换机并接收所有 VLAN 上的流量，如图 5-5 所示。

**图 5-5. 使用 DTP 在本地端口上启用中继**

Cisco 交换机支持的五种端口模式列于表 5-1 中。

**表 5-1. Cisco 交换机使用的 802.1Q 端口模式**

| 模式 | 描述 |
|------|------|
| Access | 将端口置于永久非中继模式 |
| Trunk | 将端口置于永久中继模式 |
| Dynamic auto | 如果相邻端口协商中继或 dynamic desirable 连接，端口可能将链路转换为中继（默认模式） |
| Dynamic desirable | 端口主动尝试将其链路转换为中继，如果相邻端口协商中继、dynamic desirable 或 dynamic auto 模式，则成为中继端口 |
| Nonegotiate | 完全禁用端口的动态中继 |

#### 802.1Q 双标记

如果用于在交换机之间形成中继的本征 VLAN 暴露给攻击者，则该攻击者可以双标记帧并将内容发送到其他网络，如图 5-6 所示。帧被标记了本征 VLAN（1）和目标 VLAN（20）的标签。

**图 5-6. 双标记 VLAN 帧**

传递内容需要一个有效的目标 MAC 和 IP 地址。大多数实际攻击是单向的，并使用无连接协议（例如 SNMP）；然而，如果受害者可以与攻击者控制的 IP 通信，则有可能建立 TCP 连接（如图 5-7 所示）。Andrew Vladimirov 在 Full Disclosure 邮件列表中详细介绍了该策略[^12]，Steve Rouiller 的论文[^13]在其附录（`vlan-de-1-2.c`）中包含此攻击的参考代码。

[^12]: Andrew A. Vladimirov，“Making Unidirectional VLAN and PVLAN Jumping Bidirectional”，电子邮件发送至 Full Disclosure，2005 年 12 月 19 日。[链接](https://seclists.org/fulldisclosure/2005/Dec/1042)
[^13]: Steve A. Rouiller，“Virtual LAN Security: Weaknesses and Countermeasures”，SANS Institute，2003。[链接](https://www.sans.org/reading-room/whitepapers/networkdevs/virtual-lan-security-weaknesses-countermeasures-1090)

你还可以使用这种技术对受害者进行端口扫描，通过向每个端口发送使用伪造源 IP（受害者可以到达的攻击者控制的系统）的双标记 TCP SYN 数据包，并监控行为（例如，来自开放端口的 TCP SYN/ACK 响应，如第 6 章所述）。

双标记攻击的缓解措施涉及使本征 VLAN 仅可被特权端口和设备访问，如图 5-4 所示。通过强制执行正确的 VLAN 边界，攻击者无法双标记并向其他网段发送帧。

#### 第 3 层私有 VLAN 绕过

在访客无线网络和其他环境中，**私有 VLAN**[^14]（也称为端口隔离）设置用于防止对等方之间进行交互（即客户端连接到无线接入点但无法相互寻址）。根据网络 ACL（或缺乏 ACL），可能可以将 IP 数据包发送到路由器，然后路由器将其转发回相邻对等方。图 5-8 演示了该攻击，攻击者发送一个带有精心构造的目标 MAC（路由器）和 IP 地址（目标）的网络帧。

[^14]: 参见 RFC 5517。

**图 5-8. 私有 VLAN 攻击**

网关接收帧并将数据包转发到目的地。此攻击可以与双标记相同的方式被利用（扩展 Steve Rouiller 的 `pvlan.c`）——如果受害者可以连接到攻击者控制的 IP，则可以与监听端口建立 TCP 会话。

### 802.1X PNAC

**基于端口的网络访问控制（PNAC）** 是一种允许连接到局域网的设备进行身份验证的机制。图 5-9 显示了可扩展身份验证协议（EAP）消息如何在请求者、认证者和身份验证服务器之间发送。未经身份验证的请求者不能参与网络；然而，它们可以发起与后端 RADIUS 服务器的 EAP 对话。

**图 5-9. 802.1X 身份验证**

大多数 802.1X 实现支持表 5-2 中列出的常见 EAP 方法。**EAP-MD5** 支持尤其成问题，因为该方法容易受到离线暴力密码猜测的攻击。**EAP-TLS** 和**受保护的 EAP（PEAP）** 通过 TLS 提供身份验证和传输安全，这减轻了凭据的暴露。然而，在每种情况下，请求者身份都在明文 EAPOL 开始消息中广播。

**表 5-2. 常见的 802.1X 身份验证方法**

| 方法 | 身份验证过程 |
|------|--------------|
| EAP-MD5 | 身份验证服务器向请求者发送一个挑战，请求者以 MD5 哈希的形式发送响应（使用标识符、挑战值和用户密码计算）。服务器执行相同的计算，如果哈希匹配，请求者通过身份验证。 |
| EAP-TLS | 身份验证服务器和请求者建立 TLS 会话。请求者验证身份验证服务器的 X.509 证书，反之亦然。通过使用证书进行相互身份验证，不使用密码。TLS 身份验证在第 11 章中详述。 |
| PEAP | 请求者和身份验证服务器建立一个经过身份验证的 TLS 隧道。MS-CHAPv2 挑战/响应通过 TLS 使用用户凭据（即用户名和密码）对请求者进行身份验证。 |

以下是一些可用于攻击 802.1X 实现的攻击策略：

- 通过 EAP 进行主动暴力密码猜测
- 使用格式错误的 EAP 内容攻击 RADIUS 服务器[^15]
- EAP 消息捕获和离线密码破解（EAP-MD5 和 PEAP）
- 强制 EAP-MD5 身份验证以绕过 TLS 证书验证
- 在使用集线器或类似设备进行身份验证后注入恶意网络流量[^16]

[^15]: 参见 CVE-2008-2441 和 CVE-2013-3466。
[^16]: 这可以通过采用 802.1AE（MACsec）来缓解。

#### 强制 EAP-MD5 身份验证

Microsoft IAS、Cisco ACS 和其他身份验证服务器支持 **EAP-MD5**，攻击者可以使用窃取的凭据（即用户名/密码）滥用该协议进行身份验证，而无需提供客户端证书。这可以在企业环境中被利用，例如使用不属于 Windows 域的系统访问网络。

### 802.1D STP

**生成树协议（STP）** 是一种用于维护无环网络拓扑的机制。交换机使用**网桥协议数据单元（BPDU）** 帧进行自组织，通过将冗余上行链路端口设置为称为**阻塞**的状态来关闭环路。图 5-11 展示了一个典型配置，表 5-5 描述了五种端口状态。

**图 5-11. 使用 STP 的 802.3 以太网网络**

**表 5-5. STP 网络中的端口状态**

| 状态 | 描述 |
|------|------|
| Disabled | 电气上处于非活动状态，直到启用 |
| Blocking | 端口丢弃除 BPDU 帧之外的所有数据 |
| Listening | 交换机监听 BPDU 以构建无环树 |
| Learning | 交换机使用帧的源 MAC 地址构建转发表 |
| Forwarding | 端口完全运行，向/从交换机转发流量 |

在选举根桥后，交换机选择根端口和指定端口。根端口是提供到根桥最佳路径的端口，其他端口成为指定端口。有关 BPDU 选举过程和端口配置的详细信息，请参阅 Kevin Lauerman 和 Jeff King 的论文[^21]，该论文提供了许多底层示例和详细的拓扑图。

## 本地 IP 协议

前一节描述的协议使用以太网多播并在数据链路层上运行。本节描述的协议通过传输层广播地址在 IPv4 和 IPv6 上提供网络发现和配置服务。常见的本地名称解析和配置机制包括以下内容：

- **动态主机配置协议（DHCP）**
- **预启动执行环境（PXE）**
- **本地名称解析协议（LLMNR、NBT-NS 和 mDNS）**
- **Web 代理自动发现（WPAD）**
- **内部路由协议（例如 HSRP、VRRP、EIGRP 和 OSPF）**
- **IPv6 网络发现协议**

许多专有系统使用 IP 广播和多播请求执行主机发现和自动配置。在撰写本文时，Nmap 的 broadcast NSE 类别包括 40 个单独的脚本，涵盖了多种协议。

通过第 2 层 MITM 攻击拦截流量后，你可以使用 Ettercap[^22] 伪造 DNS 响应并将用户引导到任意目的地。

[^22]: 有关 Ettercap 的更多信息，请参见 Lakshmanan Ganapathy，“Ettercap Tutorial: DNS Spoofing & ARP Poisoning Examples”，The Geek Stuff Blog，2012 年 5 月 10 日。[链接](https://www.thegeekstuff.com/2012/05/ettercap-tutorial/)

### DHCP

**动态主机配置协议（DHCP）** 通过 UDP 运行，使用表 5-6 中描述的消息。该协议用于配置本地系统并向客户端提供详细信息，包括 IP 地址、子网和默认网关详细信息。

**表 5-6. DHCP 消息类型**

| 消息 | 发送方 | 描述 |
|------|--------|------|
| DHCPDISCOVER | 客户端 | 广播以从网络请求 DHCP 提供，并可选地提供最后使用的 IP 地址（请求租约） |
| DHCPOFFER | 服务器 | 收到请求后，服务器首先为客户端预留一个 IP 地址，然后准备并发送一个提供，包括地址、子网掩码和 DHCP 选项（例如 DNS 服务器和路由器详细信息） |
| DHCPREQUEST | 客户端 | 客户端可以接受来自多个服务器的 DHCP 提供，因此此消息用于正式请求特定地址 |
| DHCPACK | 服务器 | 服务器的最终确认，包括租约持续时间和额外的 DHCP 选项（例如 WPAD 服务器） |
| DHCPNAK | 服务器 | 指示客户端的网络地址概念不正确（例如客户端已更改子网或其租约已过期） |
| DHCPDECLINE | 客户端 | 指示客户端网络地址已在使用中 |
| DHCPRELEASE | 客户端 | 取消租约并放弃网络地址 |
| DHCPINFORM | 客户端 | 在已配置其本地 IP 地址和子网后，请求额外的网络配置参数 |

一种流行的攻击 DHCP 的主动策略涉及设置恶意服务器（通常涉及拒绝服务以使合法服务器过载）。最终结果是向客户端提供恶意的默认网关、DNS 服务器和 WPAD 设置。

### PXE

**预启动执行环境（PXE）** 是一种系统从本地网络加载操作系统映像的方法。图 5-13 演示了该协议：DHCP 用于提供网络配置，TFTP 用于提供初始启动映像，文件服务协议（通常是 NFS 或 SMB）用于部署操作系统。

#### RIP

存在三个版本的**路由信息协议（RIP）**——RIP[^48]、RIPv2[^49] 和 RIPng[^50]。RIP 和 RIPv2 通过端口 520 向对等方发送 UDP 数据报，而 RIPng 通过 IPv6 多播将数据报广播到 UDP 端口 521。RIPv2 引入了 MD5 身份验证支持。RIPng 不包含本地身份验证；相反，它依赖于 IPv6 中可选的 IPsec AH 和 ESP 头[^51]。

[^48]: 参见 RFC 1058。
[^49]: 参见 RFC 2453。
[^50]: 参见 RFC 2080。
[^51]: IPsec 安全特性在第 10 章中描述。

在许多情况下，RIP 对等方会处理未经请求的路由通告。考虑图 5-16 所示的环境。使用 Nemesis 和 Scapy，我们可以在受害者主机（`10.0.0.5`）上注入一条路由，将目标服务器（`10.2.0.10`）的新网关指定为 `10.0.0.100`。

## 本地网络发现回顾

用于攻击本地协议的个别策略因网络而异。在高层次上，有两个迭代阶段，如图 5-22 所示。主动攻击策略（例如 VLAN 跳跃和 ARP 缓存投毒）用于改变网络状态，而被动嗅探则产生网络拓扑信息和敏感数据。

**图 5-22. 本地网络测试的迭代方法**

常见的第 2 层主动攻击方法如下：

- **ARP 缓存投毒**：操纵本地主机之间的流量
- **CAM 表溢出**：导致交换机将帧广播到所有端口
- **通过伪造的 BPDU 帧修改 802.1D STP 流量**：改变流量流
- **通过动态中继或双标记访问特权 VLAN**
- **802.1X 凭据的破坏和破解**（需要网络访问）
- **CDP 泛洪**：导致拒绝服务
- **CDP 欺骗**：通告虚假设备，诱导管理软件连接（例如 SNMP）并破坏敏感数据

第 3 层主动攻击策略包括：

- 使用恶意的 DHCP 和 WPAD 服务器修改客户端配置
- 通过 PXE 提供恶意启动映像，以破坏用户凭据并对易受攻击的主机发起硬件层攻击（例如 BIOS 攻击）
- **LLMNR、NBT-NS 和 mDNS 欺骗**：将用户引导到恶意服务
- 通过 **HSRP、EIGRP、OSPF** 和其他协议模拟路由器
- **IPv6 发现协议欺骗**：导致 MITM
- **识别支持 IP 转发的主机**（用于将流量路由到其他地方）

## 本地网络攻击对抗措施

可以考虑采取多种对抗措施来提高本地网络的安全性。针对 MITM 和恶意服务器攻击的一个非常有效的广泛缓解措施是强制实施传输安全（通过 IPsec 或 TLS）以及通过证书验证进行强身份验证。

许多 802.1X 攻击在请求者侧（即客户端系统）得到缓解：

- 始终验证认证者的 X.509 证书
- 指定有效认证者（RADIUS 服务器）的 CN 值
- 通过不在安全异常时提示最终用户来实现故障安全

要考虑的 Cisco 特定数据链路安全特性包括：

- 启用端口安全以限制分配给一个端口的 MAC 地址数量
- 禁用 CDP 支持以防止针对交换机的拒绝服务攻击
- 启用 `bpdu-guard` 和 `root guard` 以缓解 STP 攻击
- 使用未知流量泛洪控制功能来限制第 2 层广播攻击[^63]
- 通过在不信任的端口上设置 ACL 以丢弃发往 UDP 端口 520 的流量和使用 IP 协议 88 和 89 的数据包，避免因 OSPF、EIGRP 和 RIP 数据包具有优先级而导致 CPU 耗尽

[^63]: 更多信息，请参见“Catalyst 6500 Release 12.25X Software Configuration Guide”，Cisco.com，2013 年 11 月 17 日。[链接](https://www.cisco.com/c/en/us/td/docs/switches/lan/catalyst6500/ios/12-2SX/configuration/guide/book/layer2.html)

通用的数据链路攻击缓解措施如下：

- 将交换机端口设置为接入模式并禁用动态中继
- 建立 VLAN，防止不受信任的用户获得对敏感系统（例如 IT 运维人员使用的服务器和工作站）的第 2 层访问
- 禁用未使用的以太网端口并将其置于隔离 VLAN 中
- 始终为中继端口使用专用的 VLAN ID
- 尽可能避免使用默认的 VLAN ID 值 “1”
- 尽可能使用私有 VLAN（端口隔离）功能，以防止客户端系统相互交互

网络层和应用层对抗措施：

- 如果未明确需要 IPv6，请禁用以防止覆盖网络攻击
- 禁用 ICMP 重定向支持以缓解 MITM[^64]
- 在 Windows 中禁用多播名称解析和基于 TCP/IP 的 NetBIOS
- 在 Apple OS X 和 Linux 中禁用 Bonjour/零配置功能
- 在未使用隔离的端口上建立 ACL，使得私有 VLAN 攻击（通过网关将流量路由到隔离端口）无效
- 在你的 Web 应用程序中使用 HSTS 以缓解降级 HTTPS 到 HTTP 的 MITM 攻击（例如 `sslstrip`）
- 审查客户端代理设置，使其不通过 WPAD 自动设置[^65]
