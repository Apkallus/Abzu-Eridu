# 第 2 章 评估工作流与工具

本章概述我的渗透测试方法，并描述一个有效的测试环境。许多评估工具运行在 Linux 平台上，而在攻击 Microsoft 系统时需要 Windows 特定的实用程序。一个灵活、虚拟化的平台是关键。在 Matta，我们运行了一个名为 Sentinel 的程序，通过它评估了金融服务行业客户的第三方测试供应商。每个供应商根据我们在准备好的系统中发现的漏洞进行排名。在一次涉及 10 家供应商的测试中，我们发现：

- 有两家未能扫描全部 65,536 个 TCP 端口
- 有五家未能报告 MySQL 服务的 root 密码为 "password"

有些供应商被评估了多次。似乎缺乏对严格测试方法的遵守，测试结果（最终报告）因参与的顾问而异。

在测试过程中，切记你应该遵循一套完整的方法论。工程师和顾问常常钻入所谓的“兔子洞”，而忽略了环境中的关键领域。

同样重要的是，要快速识别网络中的重大漏洞。因此，该方法论具有两个特点：

1. **全面性**，以便你能够一致地发现重大缺陷
2. **灵活性**，以便你能够优先安排工作并最大化回报

## 网络安全评估方法论

坚定的攻击者和安全顾问使用的最佳实践评估方法包括四个不同的步骤：

- **侦察**：识别感兴趣的网络、主机和用户
- **漏洞扫描**：识别可能被利用的条件
- **漏洞调查**：手动进一步探测
- **漏洞利用**：利用漏洞并绕过安全机制

该方法适用于在仅有有限目标信息（如域名）的情况下对互联网网络进行盲测。如果顾问被聘请评估特定的 IP 地址段，她可以跳过网络枚举，直接开始批量网络扫描和漏洞调查。

本地网络测试涉及不可路由协议和 OSI 第二层特性（例如 802.1X 和 802.1Q）的评估。你可以使用 VLAN 跳跃和网络嗅探来破坏数据和系统，详见第 5 章。

### 侦察

你可以采用不同的策略来识别感兴趣的主机、网络和用户。攻击者通过使用开放源（例如网络搜索引擎、WHOIS 数据库和 DNS 服务器）来映射目标环境，而不通过端口扫描进行直接网络交互。

侦察常常发现未得到妥善加固的主机。坚定的攻击者会投入时间识别外围网络和主机，而组织通常将其精力集中在保护明显的公共系统（如公共 Web 和邮件服务器）上。那些偏离常规的被忽视的主机正是攻击的成熟目标。

通过侦察收集的有用信息包括基于互联网的网络块和内部 IP 地址的详细信息。通过 DNS 和 WHOIS 查询，你可以绘制目标组织的网络地图，并了解物理位置之间的关系。

这些信息被输入到漏洞扫描和渗透测试阶段，以识别可利用的缺陷。进一步的侦察涉及提取用户详细信息（例如电子邮件地址、电话号码和用户名），这些信息可用于暴力密码猜测和社会工程学阶段。

### 漏洞扫描

在识别出感兴趣的 IP 地址段后，攻击者会进行批量扫描，以识别可访问的网络服务，这些服务随后可被利用来实现特定目标——无论是代码执行、信息泄露还是拒绝服务。网络扫描工具（例如 Nmap、Nessus、Rapid7 Nexpose 和 QualysGuard）执行服务指纹识别、探测和已知问题的测试。

通过漏洞扫描收集的有用信息包括暴露的网络服务详情和外围信息（例如主机响应的 ICMP 消息，以及对防火墙 ACL 的洞察）。已知的弱点和暴露也会被扫描工具报告，然后你可以进一步调查。

### 漏洞调查

软件中的漏洞有时会通过公共互联网邮件列表和论坛披露，但越来越多地被出售给私营组织，例如零日计划（ZDI），该计划反过来负责任地向供应商披露问题并通知付费订阅者。根据 Immunity Inc. 的数据，平均而言，一个零日漏洞在供应商发布补丁之前有 348 天的生命周期。

许多经纪人不会将漏洞通知产品供应商，有些则向其客户提供商业支持的零日漏洞利用。在负责任披露、零日滥用和对已知缺陷的公开讨论之间，漏洞详细信息是分散的，如图 2-1 所示。

为了有效地报告环境中的漏洞，安全专业人员需要了解私人和公共领域中的缺陷。许多人无法访问私人信息源，只能基于公共知识并结合自己的研究进行评估。

#### 公共漏洞来源

以下开放来源在调查潜在漏洞时非常有用：

- NIST 国家漏洞数据库
- Offensive-Security 漏洞利用数据库
- Full Disclosure 邮件列表
- HackerOne 互联网漏洞赏金
- SecurityFocus
- Packet Storm
- CERT 漏洞备注

ZDI 和 Google 的 Project Zero 运营着可公开访问的漏洞追踪器，详细说明即将披露的漏洞和未修补的漏洞[^1]。包括 OpenSSL 和 Linux 内核在内的开源项目也拥有公共漏洞追踪器，可揭示未修补缺陷的有用细节。在测试期间，值得同时查阅漏洞追踪器和发布说明，以了解软件包中的已知弱点。

[^1]: 分别参见 Zero Day Initiative 的即将发布的公告（https://www.zerodayinitiative.com/advisories/upcoming/）和 Chromium 漏洞（https://bugs.chromium.org/p/chromium/issues/list）。

### 漏洞利用

攻击者可以利用暴露逻辑中的缺陷来获利。根据他们的目标，他们可能会利用漏洞获得特权网络访问、持久性或获取敏感信息。

在渗透测试期间，漏洞的确认通常涉及利用。稳健的商业支持框架提供了对给定环境中易受攻击组件的灵活定位（支持不同的操作系统和配置），允许你定义特定的漏洞利用负载。第三方开发的模块扩展了这些框架，为监控与数据采集（SCADA）和其他技术提供支持。流行的利用框架如下：

- Rapid7 Metasploit
- CORE Impact
- Immunity CANVAS

在 NIST SP 800-115 中，利用任务属于“目标漏洞验证技术”类别。作为测试人员（具有正确的授权），你可能会进行密码破解和社会工程学，并通过使用利用框架和工具来确认漏洞。

## 你的测试平台

为了支持基于 Linux 和 Windows 的测试工具，最好使用虚拟化软件（例如适用于 Apple OS X 的 VMware Fusion 或适用于 Windows 的 VMware Workstation）。Linux 中有用的原生应用程序包括 `showmount`、`dig` 和 `snmpwalk`。你还可以构建特定的实用程序来攻击不常见的网络协议和应用程序。

Kali Linux 是一个渗透测试发行版，你可以轻松地在虚拟化环境中运行它。Kali 包含本书中的许多实用程序，包括 Metasploit、Nmap、Burp Suite 和 Nikto。Kali Linux 文档详细介绍了安装过程，最近出版的两本书描述了各个工具，如下所示：

- *Kali Linux：通过渗透测试确保安全*，作者 Tedi Heriyanto、Lee Allen 和 Shakeel Ali（Packt Publishing，2014）
- *精通 Kali Linux 高级渗透测试*，作者 Robert W. Beggs（Packt Publishing，2014）

例如，如果你使用 Apple OS X，在 VMware Fusion 中运行 Microsoft Windows 和 Kali Linux 足以满足大多数测试需求。Offensive Security 维护着自定义的 Kali Linux VMware 和 ARM 镜像，这些镜像支持 VMware Tools（使你能够在虚拟化客户机与主机系统之间复制和粘贴）以及 ARM 平台（如 Chromebook）。

### 更新 Kali Linux

安装 Kali Linux 后，使用以下命令更新发行版和软件包（Nmap 和 Metasploit 的新版本包含应积极维护的组件）：

```bash
apt-get update
apt-get dist-upgrade
updatedb
```

## 部署易受攻击的服务器

要在受控环境中测试实用程序，请使用易受攻击的服务器镜像，以确保每个工具都能正常工作。Rapid7 准备了一个名为 Metasploitable 2 的虚拟机镜像[^3]，它可以通过多种途径被入侵，包括：

- 软件包中的后门，包括 FTP 和 IRC
- 易受攻击的 Unix RPC 服务
- SMB 权限提升
- 弱用户密码
- Web 应用服务器问题（通过 Apache HTTP Server 和 Tomcat）
- Web 应用程序漏洞（例如 phpMyAdmin 和 TWiki）

在线提供教程和演练，描述可用于评估和利用虚拟机中缺陷的实用程序、Metasploit 模块和命令。以下是两个有用的资源：

- Pentest Lab — Metasploitable 2
- Computer Security Student（导航至 Security Tools → Metasploitable Project → Exploits）

你可以在 VulnHub 上找到许多其他可利用的虚拟机镜像。出于 Web 应用程序测试的目的，请特别关注 OWASP 易受攻击的 Web 应用程序目录和 PentesterLab。

[^3]: 更多信息，请参阅《Metasploitable 2 Exploitability Guide》。[链接](https://docs.rapid7.com/metasploit/metasploitable-2-exploitability-guide)
