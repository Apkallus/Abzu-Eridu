# 标准的高层组织

渗透测试执行标准由七个（7）主要部分组成。这些部分涵盖了与渗透测试相关的所有内容——从初步沟通和进行渗透测试的缘由开始，经过情报收集和威胁建模阶段，测试人员在这些阶段中于幕后工作，以更好地了解被测试组织；接着是漏洞研究、漏洞利用和后漏洞利用阶段，测试人员的技术安全专长在此发挥作用，并与对项目的业务理解相结合；最后是报告环节，它以一种对客户有意义且能为其提供最大价值的方式记录整个过程。

本版本可视为 v1.0，因为标准的核心要素已经稳固，并在业界经过了一年多的“实战检验”。v2.0 即将推出，将在“级别”（即渗透测试各环节的执行强度级别）方面提供更细致的工作。由于没有两次渗透测试是完全相同的，测试范围可能涵盖从较为普通的 Web 应用或网络测试，到全面的红队演练，上述级别将使组织能够界定其预期对手可能展现的复杂程度，并让测试人员在组织最需要的关键领域提升强度。有关“级别”的初步工作可在情报收集部分看到。
