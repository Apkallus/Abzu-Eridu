# 第4章 环境设置与流量拦截

如果你在一个运转良好的实验室中寻找漏洞，可以省去很多时间和麻烦。在本章中，我将逐步指导你设置自己的黑客环境。你将配置浏览器以配合 Burp Suite 工作，Burp Suite 是一个 Web 代理，可以让你查看和修改浏览器与 Web 服务器之间发送的 HTTP 请求和响应。你将学习使用 Burp 的功能来拦截 Web 流量、发送自动化和重复请求、解码编码内容以及比较请求。我还将讨论如何做好漏洞赏金笔记。

本章重点仅在于设置 Web 黑客环境。如果你的目标是攻击移动应用，则需要额外的设置和工具。我们将在第 23 章讨论移动黑客技术时介绍这些内容。

## 选择操作系统

在我们继续之前，你需要做的第一件事是选择一个操作系统。你的操作系统会限制你可用的黑客工具。我建议使用基于 Unix 的系统，如 Kali Linux 或 macOS，因为许多开源黑客工具都是为这些系统编写的。Kali Linux 是一个专为数字取证和黑客攻击设计的 Linux 发行版。它包含许多有用的漏洞赏金工具，例如 Burp Suite、DirBuster 和 Gobuster 等侦察工具，以及 Wfuzz 等模糊测试器。你可以从 [https://www.kali.org/downloads/](https://www.kali.org/downloads/) 下载 Kali Linux。

如果你无法使用这些选项，也可以随意使用其他操作系统进行黑客攻击。请记住，你可能必须学习使用与本书中提到的不同的工具。

## 设置基本工具：浏览器和代理

接下来，你需要一个 Web 浏览器和一个 Web 代理。你将使用浏览器来检查目标应用程序的功能。我建议使用 Firefox，因为它与代理的设置最简单。你还可以在黑客攻击时使用两个不同的浏览器：一个用于浏览目标，另一个用于在互联网上研究漏洞。这样，你可以轻松隔离目标应用程序的流量以供进一步检查。

代理是位于客户端和服务器之间的软件；在这里，它位于你的浏览器和你与之交互的 Web 服务器之间。它在将请求传递到服务器之前拦截它们，并在将响应传递给你之前拦截服务器的响应，如下所示：

`浏览器 <--------------> 代理 <--------------> 服务器`

使用代理在漏洞赏金狩猎中至关重要。代理使你能够查看和修改发往服务器的请求以及进入浏览器的响应，我将在本章后面解释。如果没有代理，浏览器和服务器将自动交换消息，而你对此一无所知，你唯一看到的就是最终生成的网页。而代理会在消息到达预定接收者之前捕获所有消息。

因此，代理允许你通过检查和分析与服务器之间的流量来执行侦察。它们还允许你检查有趣的请求以寻找潜在漏洞，并通过篡改请求来利用这些漏洞。

例如，假设你访问电子邮件收件箱，并使用代理拦截将返回你电子邮件的请求。这是一个包含你用户 ID 的 GET 请求。你还注意到请求中包含了一个带有你用户 ID 的 cookie：

```
GET /emails/USER_ID HTTP/1.1
Host: example.com
Cookie: user_id=USER_ID
```

在这种情况下，你可以尝试将 URL 和 Cookie 头中的 `USER_ID` 更改为另一个用户的 ID，看看是否能访问其他用户的电子邮件。

有两种代理在漏洞赏金猎人中特别受欢迎：Burp Suite 和 Zed Attack Proxy（ZAP）。本节将向你展示如何设置 Burp，但你可以自由选择使用 ZAP。

### 打开嵌入式浏览器

Burp Suite 和 ZAP 都带有嵌入式浏览器。如果你选择使用这些嵌入式浏览器进行测试，可以跳过接下来的两个步骤。

要使用 Burp Suite 的嵌入式浏览器，在启动 Burp 后，单击其“Proxy”选项卡中的“Open browser”（图 4-1）。此嵌入式浏览器的流量将自动通过 Burp 路由，无需任何额外设置。

**图 4-1：你可以使用 Burp 的嵌入式浏览器而不是自己的外部浏览器进行测试。**

### 设置 Firefox

Burp 的嵌入式浏览器提供了一种方便的方式，可以用最少的设置开始漏洞狩猎。但是，如果你像我一样，更喜欢用自己习惯的浏览器进行测试，你可以设置 Burp 与你的浏览器一起工作。让我们设置 Burp 与 Firefox 一起工作。

首先下载并安装你的浏览器和代理。你可以从 [https://www.mozilla.org/firefox/new/](https://www.mozilla.org/firefox/new/) 下载 Firefox 浏览器，并从 [https://portswigger.net/burp/](https://portswigger.net/burp/) 下载 Burp Suite。

漏洞赏金猎人使用两个版本的 Burp Suite：Professional 或 Community。你需要购买许可证才能使用 Burp Suite Professional，而 Community 版本是免费的。Burp Suite Pro 包含一个漏洞扫描器和其他便利功能，例如保存工作会话以便稍后恢复。它还提供完整版本的 Burp Intruder，而 Community 版本仅包含有限版本。在本书中，我将介绍如何使用 Community 版本来寻找漏洞。

现在，你需要配置浏览器，使其通过代理路由流量。本节教你如何配置 Firefox 以配合 Burp Suite 工作。如果你使用的是其他浏览器-代理组合，请查阅其官方文档以获取教程。

启动 Firefox。然后通过选择“Preferences” → “General” → “Network Settings”打开“Connection Settings”页面。你可以从 Firefox 右上角的菜单访问“Preferences”选项卡（图 4-2）。

**图 4-2：你可以在 Firefox 的右上角找到“Preferences”选项。**

“Connection Settings”页面应如图 4-3 所示。选择“Manual proxy configuration”，并为所有协议类型输入 IP 地址 `127.0.0.1` 和端口 `8080`。这将告诉 Firefox 使用你机器上端口 `8080` 上运行的服务作为其所有流量的代理。`127.0.0.1` 是本地主机 IP 地址。它标识你当前的计算机，因此你可以使用它来访问你机器上运行的网络服务。由于 Burp 默认在端口 `8080` 上运行，此设置告诉 Firefox 将所有流量路由到 Burp。单击“OK”完成设置。现在 Firefox 将通过 Burp 路由所有流量。

**图 4-3：在“Connection Settings”页面上配置 Firefox 的代理设置。**

### 设置 Burp

下载 Burp Suite 后，打开它并单击“Next”，然后单击“Start Burp”。你应该会看到一个如图 4-4 所示的窗口。

**图 4-4：Burp Suite Community Edition 启动窗口**

现在让我们配置 Burp，使其能够与 HTTPS 流量一起工作。HTTPS 通过加密你的流量来保护数据的隐私，确保通信中的两方（你的浏览器和服务器）才能解密它。这也意味着你的 Burp 代理将无法拦截进出浏览器的 HTTPS 流量。为了解决这个问题，你需要通过安装其证书颁发机构（CA）证书来向 Firefox 表明你的 Burp 代理是可信方。

让我们在 Firefox 上安装 Burp 的证书，以便你可以处理 HTTPS 流量。在 Burp 打开并运行，并且代理设置设置为 `127.0.0.1:8080` 的情况下，在浏览器中访问 `http://burp/`。你应该会看到一个 Burp 欢迎页面（图 4-5）。单击右上角的“CA Certificate”下载证书文件；然后单击“Save File”将其保存在安全位置。

**图 4-5：访问 http://burp/ 下载 Burp 的 CA 证书。**

接下来，在 Firefox 中，单击“Preferences” → “Privacy & Security” → “Certificates” → “View Certificates” → “Authorities”。单击“Import”并选择你刚刚保存的文件，然后单击“Open”。按照对话框的说明信任证书以识别网站（图 4-6）。

**图 4-6：在 Firefox 的对话框中选择“Trust this CA to identify websites”选项。**

重启 Firefox。现在你应该可以拦截 HTTP 和 HTTPS 流量了。

让我们执行一个测试以确保 Burp 正常工作。切换到 Burp 中的“Proxy”选项卡，通过单击“Intercept is off”打开流量拦截。该按钮现在应显示“Intercept is on”（图 4-7）。这意味着你现在正在拦截来自 Firefox 或嵌入式浏览器的流量。

**图 4-7：“Intercept is on”表示你正在拦截流量。**

然后打开 Firefox 并访问 `https://www.google.com/`。在 Burp 的代理中，你应该会看到主窗口开始填充各个请求。Burp Proxy 中的“Forward”按钮会将当前请求发送到指定的服务器。继续单击“Forward”，直到看到主机名为 `www.google.com` 的请求。如果你看到此请求，则 Burp 正在正确拦截 Firefox 的流量。它应该以如下内容开头：

```
GET / HTTP/1.1
Host: www.google.com
```

单击“Forward”将请求发送到 Google 的服务器。你应该会看到 Google 的主页出现在你的 Firefox 窗口中。

如果你在 Burp 的窗口中没有看到请求，则可能没有正确安装 Burp 的 CA 证书。按照本章中的步骤重新安装证书。此外，检查你是否在 Firefox 的“Connection Settings”中正确设置了代理设置为 `127.0.0.1:8080`。

## 使用 Burp

除了 Web 代理之外，Burp Suite 还有各种有用的功能。Burp Suite 还包括一个用于自动化攻击的 Intruder、一个用于操作单个请求的 Repeater、一个用于解码编码内容的 Decoder，以及一个用于比较请求和响应的 Comparer 工具。在 Burp 的所有功能中，这些对漏洞赏金狩猎最有用，因此我们将在此处进行探讨。

### 代理

让我们看看如何使用 Burp 代理来检查请求、修改它们，并将它们转发到 Burp 的其他模块。打开 Burp 并切换到“Proxy”选项卡，开始探索它的功能！要开始拦截流量，请确保“Intercept”按钮显示“Intercept is on”（图 4-8）。

**图 4-8：Burp Proxy 选项卡显示“Intercept is on”。**

当你使用 Firefox 或 Burp 的嵌入式浏览器浏览网站时，你应该会看到一个 HTTP/HTTPS 请求出现在主窗口中。当拦截打开时，你的浏览器发送的每个请求都会经过 Burp，除非你在代理窗口中单击“Forward”，否则 Burp 不会将它们发送到服务器。你可以利用这个机会在将请求发送到服务器之前对其进行修改，或者将其转发到 Burp 中的其他模块。

你还可以使用窗口底部的搜索栏在请求或响应中搜索字符串。

要将请求转发到另一个 Burp 模块，请右键单击请求并选择“Send to Module”（图 4-9）。

**图 4-9：你可以通过右键单击将请求或响应转发到不同的 Burp 模块。**

让我们练习使用 Burp Proxy 拦截和修改流量！转到 Burp Proxy 并打开流量拦截。然后打开 Firefox 或 Burp 的嵌入式浏览器并访问 `https://www.google.com/`。正如你在上一节中所做的那样，单击“Forward”，直到看到主机名为 `www.google.com` 的请求。你应该会看到如下所示的请求：

```
GET / HTTP/1.1
Host: www.google.com
User-Agent: Mozilla/5.0
Accept-Language: en-US
Accept-Encoding: gzip, deflate
Connection: close
```

让我们在发送之前修改此请求。将 `Accept-Language` 头的值更改为 `de`。

```
GET / HTTP/1.1
Host: www.google.com
User-Agent: Mozilla/5.0
Accept-Language: de
Accept-Encoding: gzip, deflate
Connection: close
```

单击“Forward”将请求发送到 Google 的服务器。你应该会在浏览器窗口中看到德语的 Google 主页（图 4-10）。

**图 4-10：德语的 Google 主页**

如果你会说德语，可以反向测试：将 `Accept-Language` 头的值从 `de` 切换为 `en`。你应该会看到英文的 Google 主页。恭喜！你现在已成功通过代理拦截、修改和转发了 HTTP 请求。

### Intruder

Burp Intruder 工具可自动发送请求。如果你使用的是 Burp Community 版本，你的 Intruder 将是有限的试用版。尽管如此，它允许你执行诸如暴力破解之类的攻击，即攻击者使用预定义值列表向服务器提交许多请求，并查看服务器是否有不同的响应。例如，获得常用密码列表的黑客可以尝试通过反复提交包含所有常用密码的登录请求来入侵你的帐户。你可以通过在代理窗口中右键单击请求并选择“Send to intruder”将请求发送到 Intruder。

Intruder 选项卡中的“Target”屏幕允许你指定要攻击的主机和端口（图 4-11）。如果你从代理转发请求，则主机和端口将为你预填。

**图 4-11：你可以在“Target”屏幕上指定要攻击的主机和端口。**

Intruder 提供了几种自定义攻击的方法。对于每个请求，你可以选择要使用的载荷和载荷位置。载荷是你要插入到请求中特定位置的数据。载荷位置指定请求的哪些部分将被你选择的载荷替换。例如，假设用户通过向 `example.com/login` 发送 POST 请求来登录 `example.com`。在 Burp 中，此请求可能如下所示：

```
POST /login HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0
Accept: text/html,application/xhtml+xml,application/xml
Accept-Language: en-US
Accept-Encoding: gzip, deflate
Connection: close

username=vickie&password=abc123
```

POST 请求体包含两个参数：`username` 和 `password`。如果你试图暴力破解用户的帐户，你可以更改请求的密码字段并保持其他所有内容不变。为此，在“Positions”屏幕中指定载荷位置（图 4-12）。要将请求的一部分添加到载荷位置，请高亮文本并单击右侧的“Add”。

**图 4-12：你可以在“Positions”屏幕中指定载荷位置。**

然后，切换到“Payloads”屏幕（图 4-13）。在这里，你可以选择要插入到请求中的载荷。要暴力破解登录密码，你可以在此处添加常用密码列表。例如，你也可以使用数字列表来暴力破解请求中的 ID，或者使用从互联网下载的攻击载荷列表。

**图 4-13：在“Payloads”屏幕上选择你的载荷列表。**

重复使用他人分享的攻击载荷可以帮助你更快地找到漏洞。我们将在第 25 章中详细讨论如何使用重用载荷来寻找漏洞。

一旦你指定了这些，单击“Start attack”按钮开始自动化测试。Intruder 将为你列出的每个载荷发送一个请求，并记录所有响应。然后，你可以查看响应和响应代码，寻找有趣的结果。

### Repeater

Repeater 可能是你使用最频繁的工具（图 4-14）。你可以使用它来修改请求并详细检查服务器响应。你也可以用它来标记有趣的请求，以便稍后返回查看。

尽管 Repeater 和 Intruder 都允许你操作请求，但这两个工具的用途截然不同。Intruder 通过自动发送程序化修改的请求来自动化攻击。Repeater 用于对单个请求进行手动的、详细的修改。

通过右键单击请求并选择“Send to repeater”将请求发送到 Repeater。

在 Repeater 屏幕的左侧是请求。你可以在此处修改请求，并通过单击顶部的“Send”将修改后的请求发送到服务器。服务器返回的相应响应将显示在右侧。

Repeater 非常适合手动利用漏洞、尝试绕过过滤器以及测试针对同一端点的不同攻击方法。

**图 4-14：Repeater 非常适合仔细检查请求和手动利用。**

### Decoder

Burp Decoder 是一种方便的方式，可以对你在请求和响应中发现的数据进行编码和解码（图 4-15）。大多数情况下，我使用它来解码、操作和重新编码应用程序数据，然后再将其转发给应用程序。

**图 4-15：你可以使用 Decoder 解码应用程序数据以读取或操作其明文。**

通过在任何请求或响应中高亮文本块，然后右键单击并选择“Send to decoder”将数据发送到 Decoder。使用右侧的下拉菜单指定用于编码或解码消息的算法。如果你不确定消息是用哪种算法编码的，可以尝试“Smart decode”。Burp 将尝试检测编码，并相应地解码消息。

### Comparer

Comparer 是一种比较请求或响应的方法（图 4-16）。它会高亮显示两个文本块之间的差异。例如，你可以使用它来检查参数差异如何影响从服务器获得的响应。

通过在任何请求或响应中高亮文本块，然后右键单击并选择“Send to comparer”将数据发送到 Comparer。

**图 4-16：Comparer 将高亮显示两个文本块之间的差异。**

### 保存 Burp 请求

你也可以在 Burp 上保存请求和响应。只需右键单击任何请求，然后选择“Copy URL”、“Copy as curl command”或“Copy to file”即可将这些结果存储到该目标的笔记文件夹中。“Copy URL”选项复制请求的 URL。“Copy as curl command”将整个请求（包括请求方法、URL、头和体）复制为 curl 命令。“Copy to file”将整个请求保存到一个单独的文件中。

## 关于记笔记的最后说明

在你开始下一章寻找漏洞之前，有一点建议：如果你想在漏洞赏金中取得成功，组织能力至关重要。当你处理范围很大的目标或同时攻击多个目标时，从目标收集的信息可能会激增并变得难以管理。

通常，你不会立即找到漏洞。相反，你会发现许多奇怪的行为和配置错误，这些目前无法利用，但你以后可以在攻击中与其他行为结合。你需要做好笔记，记录你发现的任何新功能、配置错误、小漏洞和可疑端点，以便能够快速返回并使用它们。

笔记还可以帮助你规划攻击。你可以跟踪你的黑客进展、你测试过的功能以及你仍需检查的功能。这可以防止你一遍又一遍地测试相同的功能而浪费时间。

笔记的另一个好用途是记录你学到的漏洞信息。记录每个漏洞的详细信息，例如其理论概念、潜在影响、利用步骤和示例概念验证代码。随着时间的推移，这将增强你的技术技能，并建立一个技术知识库，以便在需要时重新查阅。

由于这些笔记往往会数量激增并变得非常混乱，因此最好从一开始就保持其组织有序。我喜欢使用 Sublime Text（[https://www.sublimetext.com/](https://www.sublimetext.com/)）以纯文本文件形式记笔记，并通过将它们分类到目录中来组织它们，为每个目标和主题创建子目录。

例如，你可以为你正在处理的每个目标（如 Facebook、Google 或 Verizon）创建一个文件夹。然后，在每个文件夹中，创建文件来记录有趣的端点、新的和隐藏的功能、侦察结果、报告草稿和 POC。

找到适合你的笔记记录和组织策略。例如，如果你像我一样喜欢以纯文本形式存储笔记，你可以搜索让你感觉最舒适的集成开发环境（IDE）或文本编辑器。有些人更喜欢使用 Markdown 格式记笔记。在这种情况下，Obsidian（[https://obsidian.md/](https://obsidian.md/)）是一个很好的工具，它能够以有组织的方式显示你的笔记。如果你喜欢使用思维导图来组织你的想法，可以试试思维导图工具 XMind（[https://www.xmind.net/](https://www.xmind.net/)）。

将你的漏洞赏金笔记保存在一个中心位置，例如外部硬盘驱动器或 Google Drive、Dropbox 等云存储服务，并且不要忘记定期备份你的笔记！

总结一下，这里有一些帮助你做好笔记的建议：

*   记录任何奇怪的行为、新功能、配置错误、小漏洞和可疑端点，以跟踪潜在的漏洞。
*   记录以跟踪你的黑客进展、你测试过的功能以及你仍需检查的功能。
