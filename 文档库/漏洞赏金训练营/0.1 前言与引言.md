# 前言

二十年前甚至十年前，像我这样的黑客如果试图行善，是会被逮捕的。而如今，我们被一些世界上最具影响力的组织聘用。

如果你还在犹豫自己是否错过了漏洞赏金这趟列车，那么请知道，你正赶上这个行业历史上最激动人心的时刻之一。这个社区的发展速度比以往任何时候都快：政府开始要求公司举办漏洞披露计划，财富500强公司正在大量制定此类政策，而由黑客驱动的安全应用场景也在与日俱增。人类双眼的价值对于抵御不断演变的威胁将永远至关重要，而世界正在认识到我们就是提供这种价值的人。

漏洞赏金世界的美妙之处在于，与典型的朝九晚五工作或咨询工作不同，它允许你随时随地参与，并且可以针对任何你感兴趣的资产类型！你所需要的只是一个体面的网络连接、一杯好咖啡（或你选择的饮料）、一些好奇心，以及一种热爱破坏事物的热情。而且，它不仅给你自由让你按自己的时间表工作，而且威胁的演变速度快于创新的速度，为学习、提升技能和成为新领域的专家提供了充足的机会。

如果你有兴趣获得真实世界的黑客经验，漏洞赏金市场通过提供由 Facebook、Google 或 Apple 等巨无霸公司拥有的无穷无尽的目标，使这成为可能！我并不是说在这些公司找到漏洞是件容易的事；然而，漏洞赏金项目提供了狩猎的平台，而漏洞赏金社区则推动你学习更多关于新漏洞类型的知识，增长你的技能，并在遇到困难时继续尝试。

与大多数实验室和夺旗赛（CTF）不同，漏洞赏金项目没有解决方案，也没有保证可被利用的漏洞。相反，你会一直问自己某个功能是否存在漏洞，或者它是否能迫使应用程序或其功能做不该做的事情。这种不确定性可能令人望而生畏，但它让发现漏洞时的激动感更加甜蜜。

在这本书中，Vickie 探索了各种不同的漏洞类型，以提升你对 Web 应用程序黑客技术的理解。她涵盖了能让你成为一名成功的漏洞赏金猎人的技能，包括关于如何为你自己选择合适的项目、执行适当的侦察以及撰写有力报告的逐步分析。她解释了诸如跨站脚本、SQL 注入、模板注入以及你工具包中几乎任何其他需要成功利用的攻击。之后，她还会带你超越 Web 应用程序的基础，介绍诸如代码审查、API 黑客技术、自动化工作流和模糊测试等主题。

对于任何愿意付出努力的人，《Bug Bounty Bootcamp》为你提供了在漏洞赏金领域取得成功所需的基础。

# 引言

我仍然记得第一次发现高影响漏洞时的情景。在我当时测试的应用程序中，我已经找到了几个低影响的漏洞，包括一个 CSRF、一个 IDOR 和一些信息泄露。最终，我设法将这些漏洞链式利用，最终完全接管了网站上的任何账户：我可以以任何人的身份登录，读取任何人的数据，并随心所欲地修改它。有那么一瞬间，我感觉自己拥有了超能力。

我将问题报告给了公司，公司迅速修复了该漏洞。黑客可能是我在现实世界中遇到的最接近超级英雄的事物。他们用自己的技能克服限制，让软件程序做出远超其设计初衷的事情，这就是我热爱 Web 应用程序黑客技术的原因：它关乎创造性思考、挑战自我，以及做到比看起来可能的事情更多。

也像超级英雄一样，白帽黑客帮助维护社会安全。仅在美国，每年就有数千起数据泄露事件发生。通过了解漏洞及其成因，你可以善用你的知识来帮助防止恶意攻击、保护应用程序和用户，并使互联网成为一个更安全的地方。

不久前，黑客活动和尝试 Web 应用程序还是非法的。但现在，多亏了漏洞赏金计划，你可以合法地进行黑客活动；公司设立漏洞赏金计划，以奖励安全研究人员在其应用程序中发现漏洞。《Bug Bounty Bootcamp》教你如何入侵 Web 应用程序，以及如何通过参与这些计划来合法地做到这一点。你将学习如何熟悉漏洞赏金计划、对目标执行侦察，以及识别和利用漏洞。

## 本书的读者对象

本书将帮助任何人从零开始学习 Web 黑客技术和漏洞赏金狩猎。你可能是一个想进入 Web 安全领域的学生，一个想了解网站安全性的 Web 开发者，或者一个想了解如何攻击 Web 应用程序的经验丰富的黑客。如果你对 Web 黑客技术和 Web 安全感到好奇，那么这本书就是为你准备的。理解和掌握本书材料无需技术背景。然而，理解基础编程将会对你有所帮助。

尽管本书是为初学者编写的，但高级黑客也可能发现它是一本有用的参考书。特别是，我会讨论高级利用技术以及我在过程中学到的有用技巧和窍门。

## 本书内容

《Bug Bounty Bootcamp》涵盖了开始攻击 Web 应用程序和参与漏洞赏金计划所需的一切。本书分为四个部分：行业概况、入门、Web 漏洞和专家技术。

### 第一部分：行业概况

本书的第一部分聚焦于漏洞赏金行业。**第1章：选择漏洞赏金计划**解释了各种类型的漏洞赏金计划以及如何选择适合你兴趣和经验水平的项目。**第2章：保持你的成功**教会你在漏洞赏金行业取得成功所需的非技术技能，比如撰写好报告、建立职业关系以及处理冲突和挫折。

### 第二部分：入门

本书的第二部分为你做好 Web 黑客攻击的准备，并向你介绍成功狩猎漏洞所需的基础技术和工具。

**第3章：互联网的工作原理**解释了互联网技术的基础。它还介绍了你将遇到的互联网安全机制，例如会话管理、基于令牌的认证和同源策略。

**第4章：环境设置与流量拦截**向你展示如何设置你的黑客环境、配置 Burp Suite，并有效利用 Burp Suite 的各个模块来拦截流量和狩猎漏洞。

**第5章：Web 黑客侦察**详细介绍了你可以用来收集目标信息的侦察策略。它还包括对 bash 脚本编写的介绍，并向你展示如何从头开始创建一个自动化的侦察工具。

### 第三部分：Web 漏洞

然后，我们开始黑客攻击！这部分是本书的核心，深入探讨了特定漏洞的细节。每章都专门讨论一个漏洞，并解释是什么导致了该漏洞、如何预防它，以及如何发现、利用和升级它以获得最大影响。

**第6章到第18章**讨论了你在真实应用程序中可能遇到的常见漏洞，包括跨站脚本（XSS）、开放重定向、点击劫持、跨站请求伪造（CSRF）、不安全的直接对象引用（IDOR）、SQL 注入、竞争条件、服务端请求伪造（SSRF）、不安全的反序列化、XML 外部实体漏洞（XXE）、模板注入、应用逻辑错误与访问控制失效，以及远程代码执行（RCE）。

**第19章：同源策略漏洞**深入探讨了现代互联网的一项基础防御机制：同源策略。你将了解开发人员在构建应用程序以绕过同源策略时所犯的错误，以及黑客如何利用这些错误。

**第20章：单点登录安全问题**讨论了应用程序实现单点登录功能的最常见方式、每种方法的潜在弱点，以及你如何利用这些弱点。

最后，**第21章：信息泄露**讨论了从 Web 应用程序中提取敏感信息的几种方法。

### 第四部分：专家技术

本书的最后一部分为经验丰富的黑客介绍了深入的技术。一旦你理解了第三部分涵盖的基础知识，本节将帮助你提升技能。

**第22章：进行代码审查**教你如何识别源代码中的漏洞。你还将有机会练习审查几段代码。

**第23章：Android 应用黑客技术**教你如何设置移动黑客环境以及在 Android 应用程序中发现漏洞。

**第24章：API 黑客技术**讨论了应用程序编程接口（API），这是许多现代应用程序的重要组成部分。我将讨论 API 的类型以及如何猎取其中显现的漏洞。

**第25章：使用模糊测试自动发现漏洞**通过向你展示如何使用一种称为模糊测试的方法来自动狩猎漏洞，为本书画上句号。你将使用一个开源模糊测试器练习模糊测试一个 Web 应用程序。
