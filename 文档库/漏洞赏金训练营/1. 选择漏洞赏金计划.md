# 第1章 选择漏洞赏金计划

漏洞赏金计划：它们都一样吗？找到合适的目标计划是成为一名成功的漏洞赏金猎人的第一步。在过去几年中涌现了许多计划，很难弄清楚哪些能提供最好的金钱奖励、经验和学习机会。

漏洞赏金计划是一项公司邀请黑客攻击其产品和服务的倡议。但是，你应该如何选择计划？又该如何对其不同的指标进行优先级排序，例如涉及的资产类型、计划是否托管在平台上、是公开还是私有、计划的范围、奖励金额以及响应时间？

在本章中，我们将探讨不同类型的漏洞赏金计划，分析各自的优缺点，并找出你应该选择哪一个。

## 行业现状

目前，漏洞赏金是组织接收安全漏洞反馈最流行的方法之一。像 PayPal 和 Facebook 这样的大公司，以及像美国国防部这样的政府机构，都接受了这一理念。然而，就在不久前，向公司报告漏洞更可能让你锒铛入狱，而不是获得奖励。

1995 年，Netscape 推出了有史以来第一个漏洞赏金计划。该公司鼓励用户报告在其全新浏览器 Netscape Navigator 2.0 中发现的漏洞，将众包安全测试的理念引入互联网世界。九年后，也就是 2004 年，Mozilla 推出了下一个企业漏洞赏金计划，邀请用户识别 Firefox 浏览器中的漏洞。

但直到 2010 年代，提供漏洞赏金才成为一种普遍做法。那一年，Google 推出了自己的计划，Facebook 也在 2011 年效仿。这两个计划开启了使用漏洞赏金来增强公司内部安全基础设施的趋势。

随着漏洞赏金成为一种更为人熟知的策略，漏洞赏金即服务平台应运而生。这些平台帮助公司建立和运营他们的计划。例如，它们为公司提供托管其计划的地方、处理奖励支付的方式，以及与漏洞赏金猎人沟通的集中场所。

其中最大的两个平台 HackerOne 和 Bugcrowd 都于 2012 年推出。之后，又有几个平台如 Synack、Cobalt 和 Intigriti 进入市场。这些平台和托管式漏洞赏金服务使即使是资源有限的公司也能运行安全计划。如今，大公司、小型初创企业、非营利组织乃至政府机构都已采用漏洞赏金作为额外的安全措施及其安全政策的基本组成部分。你可以在 [https://en.wikipedia.org/wiki/Bug_bounty_program](https://en.wikipedia.org/wiki/Bug_bounty_program) 上阅读更多关于漏洞赏金计划历史的信息。

术语“安全计划”通常指更广泛的信息安全行业中的信息安全政策、程序、指南和标准。在本书中，我使用“计划”或“漏洞赏金计划”来指代公司的漏洞赏金运营。如今，存在大量的计划，每个都有其独特的特征、优点和缺点。让我们来审视一下这些。

## 资产类型

在漏洞赏金计划的上下文中，资产是指你可以入侵的应用程序、网站或产品。存在不同类型的资产，每种都有其自身的特征、要求以及优缺点。考虑完这些差异后，你应该选择一个资产类型能发挥你优势的计划，基于你的技能组合、经验水平和偏好。

### 社交网站与应用

任何带有“社交”标签的东西都有很大的漏洞潜力，因为这些应用程序往往很复杂，并且涉及大量用户之间以及用户与服务器之间的交互。这就是我们要讨论的第一类漏洞赏金计划——针对社交网站和应用程序的原因。术语“社交应用”指任何允许用户相互交互的网站。许多计划属于这一类别：例如 HackerOne 的漏洞赏金计划，以及 Facebook、Twitter、GitHub 和 LINE 的计划。

社交应用需要管理用户之间的交互，以及每个用户的角色、权限和账户完整性。它们通常充满了严重的 Web 漏洞潜力，例如不安全的直接对象引用（IDOR）、信息泄露和账户接管。当许多用户在一个平台上，并且应用程序管理用户信息不当时，就会出现这些漏洞；当应用程序未能正确验证用户身份时，恶意用户就可以冒充他人身份。

这些复杂的应用程序也常常提供大量的用户输入机会。如果输入验证执行不当，这些应用程序就容易出现注入漏洞，如 SQL 注入（SQLi）或跨站脚本（XSS）。

如果你是漏洞赏金的新手，我建议你从社交网站开始。当今大量的社交应用程序意味着，如果你以社交网站为目标，你将有很多计划可供选择。此外，社交网站的复杂性意味着你将遇到一个广阔的“攻击面”可以进行实验。（应用程序的攻击面指的是攻击者可以尝试利用的所有不同点。）最后，这些站点上出现的各种漏洞意味着你将能够快速建立深厚的 Web 安全知识。

入侵社交计划所需的技能包括使用代理的能力（如第 4 章介绍的 Burp Suite 代理），以及关于 XSS 和 IDOR 等 Web 漏洞的知识。你可以在第 6 章和第 10 章中了解更多关于这些漏洞的信息。拥有一些 JavaScript 编程技能和 Web 开发知识也会有所帮助。然而，这些技能并非成为黑客所必需的。

但这些计划有一个主要的缺点。由于其产品的受欢迎程度和低准入门槛，它们通常竞争非常激烈，有很多黑客在上面狩猎。像 Facebook 和 Twitter 这样的社交媒体平台是目标最多的计划。

### 通用 Web 应用

通用 Web 应用对初学者来说也是不错的靶子。这里，我指的是任何不涉及用户间交互的 Web 应用程序。相反，用户与服务器交互以访问应用程序的功能。属于这些类别的目标可以包括静态网站、云应用、像银行站点这样的消费者服务，以及物联网（IoT）设备或其他连接硬件的 Web 门户。与社交站点一样，它们也相当多样化，并且很好地适应各种技能水平。例如 Google、美国国防部和 Credit Karma 的计划。

也就是说，根据我的经验，它们往往比社交应用更难入侵，而且攻击面更小。如果你在寻找账户接管和信息泄露漏洞，你可能没那么幸运，因为没有太多机会让用户与他人互动并可能窃取他们的信息。你在这些应用程序中发现的漏洞类型略有不同。你需要寻找服务端漏洞以及特定于应用程序技术栈的漏洞。你也可以寻找常见的网络漏洞，如子域名接管。这意味着你需要了解客户端和服务端的 Web 漏洞，并且应该能够使用代理。了解一些 Web 开发和编程知识也会有所帮助。

这些计划的受欢迎程度各不相同。然而，大多数计划的准入门槛较低，因此你很可能可以立即开始黑客活动！

### 移动应用（Android、iOS 和 Windows）

在掌握了 Web 应用黑客技术之后，你可以选择专注于移动应用。移动计划正变得越来越普遍；毕竟，如今大多数 Web 应用都有对应的移动版本。它们包括 Facebook Messenger、Twitter 应用、LINE 移动应用、Yelp 应用和 Gmail 应用的计划。

入侵移动应用需要你从入侵 Web 应用中积累的技能，以及关于移动应用结构的相关平台编程技术的额外知识。你应该理解诸如证书固定绕过、移动逆向工程和密码学等攻击和分析策略。

入侵移动应用也比入侵 Web 应用需要更多的设置，因为你需要拥有一台可以用于实验的移动设备。一个好的移动测试实验室由一台普通设备、一台 root 设备以及 Android 和 iOS 的设备模拟器组成。Root 设备是指你拥有管理员权限的设备。它可以让你更自由地进行实验，因为你可以绕过移动系统的安全约束。模拟器是在计算机上运行的移动环境的虚拟仿真。它允许你在无需为每种设置都购买一台设备的情况下，运行多种设备版本和操作系统。

由于这些原因，移动应用在漏洞赏金猎人中不如 Web 应用受欢迎。然而，较高的准入门槛对那些确实参与的人来说是一个优势。这些计划竞争较少，因此相对容易找到漏洞。

### API

应用程序编程接口（API）是定义其他应用程序如何与组织的资产（例如检索或更改其数据）进行交互的规范。例如，另一个应用程序可能能够通过向某个端点发送超文本传输协议（HTTP）消息来检索一个应用程序的数据，而该应用程序将返回可扩展标记语言（XML）或 JavaScript 对象表示法（JSON）消息格式的数据。

如果某些计划正在推出其 API 的新版本，它们可能会在其漏洞赏金计划中高度关注 API 漏洞。安全的 API 实现是防止数据泄露和保护客户数据的关键。入侵 API 需要许多与入侵 Web 应用、移动应用和物联网应用相同的技能。但是在测试 API 时，你应该关注常见的 API 漏洞，如数据泄露和注入缺陷。

### 源代码与可执行文件

如果你有更高级的编程和逆向工程技能，可以尝试源代码和可执行程序。这些计划通过直接向黑客提供开源代码库或二进制可执行文件，鼓励黑客在组织的软件中寻找漏洞。例如互联网漏洞赏金计划、PHP 语言计划和 WordPress 计划。

入侵这些计划可能需要分析开源项目的源代码以查找 Web 漏洞，并对二进制文件进行模糊测试以发现潜在的利用点。要在此获得成功，你通常必须理解编程和计算机科学概念。你需要了解 Web 漏洞、与项目代码库相关的编程技能以及代码分析技能。密码学、软件开发和逆向工程技能会有所帮助。

源代码计划可能听起来令人生畏，但请记住，它们种类繁多，你有很多选择。你不需要成为编程大师才能入侵这些计划；相反，目标是扎实理解项目的技术栈和底层架构。由于这些计划通常需要更多技能，它们的竞争较小，只有一小部分黑客会尝试它们。

### 硬件与物联网

最后但同样重要的是硬件和物联网计划。这些计划要求你入侵汽车、智能电视和恒温器等设备。例如特斯拉和福特汽车公司的漏洞赏金计划。

你需要高度专业的技能来入侵这些计划：除了理解常见的物联网漏洞外，你通常还必须对你正在入侵的设备类型有深入的了解。你应该了解 Web 漏洞、编程、代码分析和逆向工程。此外，还要学习物联网概念和行业标准，如数字签名和非对称加密方案。最后，密码学、无线黑客和软件开发技能也会有所帮助。

尽管有些计划会向你提供免费设备进行入侵，但这通常只适用于那些已经与公司建立了特定关系的少数黑客。要开始入侵这些计划，你可能需要资金自己购买设备。

由于这些计划需要专业技能和设备，它们往往是竞争最小的。

## 漏洞赏金平台

公司可以通过两种方式托管漏洞赏金计划：漏洞赏金平台和独立托管的网站。

漏洞赏金平台是许多公司托管其计划的网站。通常，平台会根据黑客的成果直接授予他们声望点和金钱。一些最大的漏洞赏金平台是 HackerOne、Bugcrowd、Intigriti、Synack 和 Cobalt。

漏洞赏金平台是黑客与安全团队之间的中介。它们为公司的支付和沟通等后勤任务提供协助。它们还经常通过为公司过滤、去重和分类漏洞报告来帮助管理收到的报告。最后，这些平台为公司提供了一种通过黑客统计数据和声望来评估黑客技能水平的方式。这使得不希望被低质量报告淹没的公司可以邀请经验丰富的黑客加入其私有计划。这些平台中的一些在允许黑客入侵计划之前也会筛选或面试黑客。

从黑客的角度来看，漏洞赏金平台提供了一个集中的地方来提交报告。它们还为你的发现获得认可和报酬提供了一种无缝的方式。

另一方面，许多组织在没有平台帮助的情况下托管和管理其漏洞赏金计划。像 Google、Facebook、Apple 和 Medium 这样的公司都是这样做的。你可以通过访问他们的网站或在线搜索“公司名 bug bounty program”来找到他们的漏洞赏金政策页面。

作为一个漏洞赏金猎人，你应该在漏洞赏金平台上进行黑客活动吗？还是应该寻找公司独立托管的计划？

### 优点……

漏洞赏金平台最好的地方在于它们提供了公司流程的大量透明度，因为它们会公开已披露的报告、有关计划分类率、奖励金额和响应时间的指标。独立托管的计划通常缺乏这种透明度。在漏洞赏金领域，“分类”指确认漏洞。

你也不必每次提交漏洞报告时都担心给安全团队发邮件的后勤问题、跟进报告以及提供支付和税务信息。漏洞赏金计划通常也有声望系统，允许你展示你的经验，从而获得仅限邀请的漏洞赏金计划访问权限。

漏洞赏金平台的另一个优点是，它们经常作为第三方介入提供冲突解决和法律保护。如果你向非平台计划提交报告，在最终的赏金决定中你将没有追索权。归根结底，在行业现状下你不能总是期望公司支付报酬或解决报告，但平台提供的黑客间反馈系统是有帮助的。

### ……与缺点

然而，一些黑客避免使用漏洞赏金平台，因为他们不喜欢这些平台处理报告的方式。提交给平台管理的漏洞赏金计划的报告通常由分类员处理，这些第三方员工通常不熟悉公司产品的所有安全细节。关于分类员不当处理报告的抱怨很常见。

平台上的计划也切断了黑客与开发者之间的直接联系。通过直接计划，你经常可以与公司的安全工程师讨论漏洞，这是一次很棒的学习经历。

最后，漏洞赏金平台上的公开计划通常很拥挤，因为平台给予了它们额外的曝光度。另一方面，许多私有托管的计划没有得到黑客太多的关注，因此竞争较少。而对于许多不与漏洞赏金平台签约的公司来说，如果你想参与他们的计划，你别无选择，只能离开平台。

## 范围、奖励与响应时间

在选择计划时，除了资产类型和平台之外，你还应该考虑哪些其他指标？在每个漏洞赏金计划的页面上，通常会列出一些指标来帮助你评估该计划。这些指标可以让你深入了解你找到漏洞的难易程度、可能获得的报酬以及计划的运作情况。

### 计划范围

首先，考虑范围。计划在其政策页面上指定的范围界定了你被允许入侵的内容和方式。有两种类型的范围：资产范围和漏洞范围。资产范围告诉你可以入侵哪些子域、产品和应用程序。漏洞范围则指定公司会接受哪些漏洞作为有效漏洞。

例如，公司可能会列出其网站的子域，哪些在范围内，哪些不在范围内：

**范围内的资产**
- a.example.com
- b.example.com
- c.example.com

**范围外的资产**
- dev.example.com
- test.example.com
- users.example.com
- landing.example.com

被列为“在范围内”的资产是允许你入侵的。另一方面，被列为“范围外”的资产是漏洞赏金猎人的禁区。要格外小心并遵守规则！入侵范围外的资产是非法的。

公司通常也会列出他们认为有效漏洞的漏洞：

**范围内的漏洞**
- 除列为范围外的所有漏洞

**范围外的漏洞**
- 自 XSS
- 点击劫持
- 缺少 HTTP 头和其他没有直接安全影响的最佳实践
- 拒绝服务攻击
- 使用已知易受攻击的库，无可利用性证明
- 自动化扫描器的结果，无可利用性证明

你在本示例中看到的范围外漏洞是你在漏洞赏金计划中通常会遇到的典型情况。请注意，许多计划认为不可利用的问题（如违反最佳实践）属于范围外。

任何具有较大资产范围和漏洞范围的计划都是初学者起步的好地方。资产范围越大，你可以查看的目标应用程序和网页数量就越多。当一个计划有较大的资产范围时，你通常可以找到被其他黑客忽视的冷门应用程序。这通常意味着报告漏洞时竞争较少。

漏洞范围越大，组织愿意听取报告的漏洞类型就越多。在这些计划中更容易找到漏洞，因为你有更多的机会，因此可以发挥你的优势。

### 奖励金额

你应该考虑的下一个指标是计划的奖励金额。有两种类型的支付计划：漏洞披露计划（VDP）和漏洞赏金计划。

VDP 是仅有声望的计划，意味着它们不支付发现漏洞的报酬，但通常提供诸如声望点和纪念品等奖励。如果赚钱不是你的主要目标，它们是学习黑客技术的好方法。由于它们不支付报酬，竞争较少，因此更容易找到漏洞。你可以用它们来练习发现常见漏洞以及与安全工程师沟通。

另一方面，漏洞赏金计划为你的发现提供不同金额的金钱奖励。通常，漏洞越严重，报告获得的报酬就越高。但不同的计划对每个严重性级别有不同的平均支付额。你可以在其漏洞赏金页面上找到计划的支付信息，通常列在称为“支付表”的部分。通常，低影响问题支付约 50 到 500 美元，而严重问题可能支付超过 10,000 美元。然而，漏洞赏金行业正在发展，高影响漏洞的奖励金额正在增加。例如，Apple 现在为最严重的漏洞提供高达 100 万美元的奖励。

### 响应时间

最后，考虑计划的平均响应时间。有些公司会在几天内处理并解决你的报告，而其他公司可能需要数周甚至数月才能完成修复。延迟通常是由于安全团队的内部限制造成的，例如缺乏处理报告的人员、发布安全补丁的延迟，以及缺乏资金及时奖励研究人员。有时，延迟是因为研究人员发送了没有清晰重现步骤的糟糕报告。

优先选择响应时间快的计划。等待公司的回复可能是一种令人沮丧的经历，而且当你刚开始时，你会犯很多错误。你可能误判漏洞的严重性、写出不清晰的解释，或在报告中出现技术错误。来自安全团队的快速反馈将帮助你改进，并更快地将你培养成一名合格的黑客。

## 私有计划

大多数漏洞赏金平台区分公开计划和私有计划。公开计划是对所有人开放的；只要遵守法律和漏洞赏金计划的政策，任何人都可以入侵并向这些计划提交漏洞。

另一方面，私有计划仅对受邀的黑客开放。对于这些计划，公司会邀请具有一定经验水平和良好记录的黑客攻击公司并向其提交漏洞。由于参与的黑客数量有限，私有计划比公开计划竞争少得多。因此，在它们中更容易找到漏洞。私有计划通常也有更快的响应时间，因为它们平均收到的报告较少。

公司会以某种方式向其能力得到证明的黑客发送私有邀请，因此一旦你找到几个漏洞，获得私有计划邀请并不难。不同的漏洞赏金平台会有不同的算法来决定谁获得邀请，但这里有一些提示可以帮助你达到目标。

首先，向公开计划提交一些漏洞。要获得私有邀请，你通常需要在平台上获得一定数量的声望点，而开始赚取这些声望的唯一方法是向公开计划提交有效的漏洞。你还应该专注于提交高影响的漏洞。这些漏洞通常会奖励你更高的声望点，并帮助你更快地获得私有邀请。在本书第二部分各章中，我会就如何升级你发现的问题以策划最高影响的攻击提出建议。在某些漏洞赏金平台上，如 HackerOne，你还可以通过完成教程或解决夺旗赛（CTF）挑战来获得私有邀请。

其次，不要发送垃圾信息。提交非问题通常会导致声望点下降。大多数漏洞赏金平台将私有邀请限制在声望点高于某个阈值的黑客。

最后，与安全团队沟通时要有礼貌和体贴。对安全团队粗鲁或辱骂很可能会让你被计划封禁，并阻止你获得其他公司的私有邀请。

## 选择正确的计划

漏洞赏金是获得网络安全经验和赚取额外收入的好方法。但这个行业的竞争越来越激烈。随着越来越多的人发现这些计划并参与其中，初学者入门变得越来越困难。这就是为什么从一开始就选择一个你能成功的计划非常重要。

在你培养出漏洞猎人的直觉之前，你通常必须依赖“低垂的果实”和众所周知的技术。这意味着许多其他黑客也能找到同样的漏洞，而且往往比你快得多。因此，选择一个更有经验的漏洞赏金猎人会忽略的计划以避免竞争是个好主意。你可以通过两种方式找到这些人口较少的计划：寻找无偿计划，或者选择范围大的计划。

首先尝试漏洞披露计划。无偿计划通常被有经验的漏洞猎人忽略，因为它们不支付金钱奖励。但它们仍然能为你赢得积分和认可！而这种认可可能正是你获得私有、有偿计划邀请所需要的。

选择一个范围大的计划意味着你将能够查看大量的目标应用程序和网页。这稀释了竞争，因为更少的黑客会报告任何单一的资产或漏洞类型。选择响应时间快的计划，以防止沮丧并尽快获得反馈。

你可以纳入决策过程的最后一点是计划的口碑。如果可能，通过其已披露的报告收集有关公司流程的信息，并从其他黑客的经验中学习。公司善待其报告者吗？他们尊重且支持吗？他们帮助你学习吗？在你还在学习的时候，选择那些会支持你的计划，以及那些会为你提供的价值给予奖励的计划。

选择适合你技能组合的正确计划对于你进入漏洞赏金世界至关重要。本章应该已经帮助你梳理了你可能感兴趣的各种计划。快乐黑客！
