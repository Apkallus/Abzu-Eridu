# 第5章 Web 黑客侦察

攻击任何目标的第一步是进行侦察，或者说，收集关于目标的信息。侦察之所以重要，是因为它帮助你找出应用程序的攻击面。为了最有效地寻找漏洞，你需要发现所有可能攻击目标的方式，然后再决定最有效的方法。

例如，如果一个应用程序不使用 PHP，就没有理由测试它的 PHP 漏洞；如果组织不使用亚马逊网络服务（AWS），你就不应浪费时间尝试破解其存储桶。通过了解目标的工作原理，你可以为发现漏洞奠定坚实的基础。侦察技能是区分优秀黑客与低效黑客的关键。

在本章中，我将介绍对漏洞赏金猎人最有用的侦察技术。然后，我将带你了解编写 bash 脚本的基础知识，以自动化侦察任务并提高效率。Bash 是 macOS 和 Linux 系统上可用的 shell 解释器。尽管本章假设你使用的是 Linux 系统，但你也应该能够在其他操作系统上安装许多这些工具。在使用本章讨论的一些工具之前，你需要先安装它们。我在本章末尾附上了所有工具的链接。

在继续之前，请确认你被允许对目标进行侵入性侦察，然后再尝试任何主动与其交互的技术。特别是，端口扫描、爬取和目录暴力破解等活动可能会在网站上产生大量不必要的流量，可能不受组织欢迎。

## 手动浏览目标

在我们深入其他内容之前，首先手动浏览应用程序以了解更多信息会很有帮助。尝试通过浏览每个页面、点击每个链接来发现应用程序中用户可以访问的每个功能。访问那些你不常使用的功能。

例如，如果你在入侵 Facebook，尝试创建一个活动、玩一个游戏，如果你以前从未使用过支付功能，也尝试使用它。在每个权限级别注册帐户，以揭示应用程序的所有功能。例如，在 Slack 上，你可以创建工作区的所有者、管理员和成员。还要创建属于同一工作区下不同频道的成员用户。这样，你就可以看到应用程序在不同用户面前的样子。

这应该会让你对攻击面（攻击者可以尝试利用应用程序的所有不同点）的样子、数据入口点在哪里以及不同用户如何相互交互有一个大致的了解。然后你可以开始更深入的侦察过程：找出应用程序的技术和结构。

## Google Dorking

在寻找漏洞时，你经常需要研究漏洞的细节。如果你正在利用一个潜在的跨站脚本（XSS）漏洞，你可能想找到你在 GitHub 上看到的某个特定载荷。高级搜索引擎技能将帮助你快速准确地找到所需的资源。

事实上，高级 Google 搜索是黑客经常用来执行侦察的强大技术。黑客称之为 Google dorking。对于普通用户来说，Google 只是一个用于查找图片、视频和网页的文本搜索工具。但对于黑客来说，Google 可以成为发现有价值信息的手段，例如隐藏的管理门户、未锁定的密码文件以及泄露的认证密钥。

Google 的搜索引擎有自己的内置查询语言，可以帮助你过滤搜索。以下是一些最有用的运算符，可以与任何 Google 搜索一起使用：

- **`site`**：告诉 Google 只显示来自某个站点的结果。这可以帮助你快速找到你正在研究的主题的最权威来源。例如，如果你想搜索 Python 的 `print()` 函数的语法，你可以将结果限制在官方 Python 文档中：`print site:python.org`。
- **`inurl`**：搜索 URL 与搜索字符串匹配的页面。这是一种在特定网站上搜索易受攻击页面的强大方法。假设你读到一篇博客文章，提到网站上存在一个名为 `/course/jumpto.php` 的页面可能表明它容易受到远程代码执行攻击。你可以通过搜索 `inurl:"/course/jumpto.php" site:example.com` 来检查你的目标是否存在该漏洞。
- **`intitle`**：在页面标题中查找特定字符串。这很有用，因为它允许你找到包含特定类型内容的页面。例如，Web 服务器上的文件列表页面通常在其标题中包含 `index of`。你可以使用此查询搜索网站上的目录页面：`intitle:"index of" site:example.com`。
- **`link`**：搜索包含指向指定 URL 的链接的网页。你可以使用它来查找关于不常见技术或漏洞的文档。例如，假设你正在研究不常见的正则表达式拒绝服务（ReDoS）漏洞。你可以轻松地在网上找到它的定义，但可能很难找到示例。`link` 运算符可以发现引用该漏洞维基百科页面的页面，从而找到关于同一主题的讨论：`link:"https://en.wikipedia.org/wiki/ReDoS"`。
- **`filetype`**：搜索具有特定文件扩展名的页面。这是黑客的绝佳工具；黑客经常使用它来定位目标站点上可能敏感的文件，例如日志和密码文件。例如，此查询在目标站点上搜索通常具有 `.log` 文件扩展名的日志文件：`filetype:log site:example.com`。
- **通配符 (`*`)**：你可以在搜索中使用通配符 `*` 来表示任何字符或字符序列。例如，以下查询将返回任何以 `how to hack` 开头并以 `using Google` 结尾的字符串。它将匹配诸如 `how to hack websites using Google`、`how to hack applications using Google` 等字符串：`"how to hack * using Google"`。
- **引号 (`" "`)**：在搜索词周围加上引号强制进行精确匹配。例如，此查询将搜索包含短语 `how to hack` 的页面：`"how to hack"`。而此查询将搜索包含 `how`、`to` 和 `hack` 这些词的页面，但不一定连续：`how to hack`。
- **或 (`|`)**：或运算符用竖线 `|` 表示，可用于搜索一个搜索词或另一个搜索词，或者同时搜索两者。竖线字符必须被空格包围。例如，此查询将在 Reddit 或 Stack Overflow 上搜索 `how to hack`：`"how to hack" site:(reddit.com | stackoverflow.com)`。此查询将搜索提及 `SQL Injection` 或 `SQLi` 的网页：`(SQL Injection | SQLi)`。SQLi 是通常用来指代 SQL 注入攻击的首字母缩略词，我们将在第 11 章讨论。
- **减号 (`-`)**：减号运算符 `-` 排除某些搜索结果。例如，假设你有兴趣了解讨论黑客攻击的网站，但不包括讨论 PHP 黑客攻击的网站。此查询将搜索包含 `how to hack websites` 但不包含 `php` 的页面：`"how to hack websites" -php`。

你可以使用更多高级搜索引擎选项来提高工作效率。你甚至可以搜索 `Google search operators` 一词来发现更多内容。这些运算符可能比你想象的更有用。例如，通过如下搜索查找公司的所有子域：

```
site:*.example.com
```

你还可以寻找可能导致漏洞的特殊端点。Kibana 是一个数据可视化工具，用于显示服务器操作数据，如服务器日志、调试消息和服务器状态。被入侵的 Kibana 实例可以允许攻击者收集有关站点操作的大量信息。许多 Kibana 仪表板在路径 `app/kibana` 下运行，因此此查询将揭示目标是否具有 Kibana 仪表板。然后你可以尝试访问仪表板，看看它是否未受保护：

```
site:example.com inurl:app/kibana
```

Google 可以找到公司在线托管的第三方资源，例如 Amazon S3 存储桶（我们将在第 74 页的“第三方托管”中更详细地讨论这些）：

```
site:s3.amazonaws.com COMPANY_NAME
```

寻找可能表示敏感文件的特殊扩展名。除了通常表示日志文件的 `.log` 之外，还可以搜索 `.php`、`.cfm`、`.asp`、`.jsp` 和 `.pl`，这些是脚本文件常用的扩展名：

```
site:example.com ext:php
site:example.com ext:log
```

最后，你还可以组合搜索词以获得更精确的搜索。例如，此查询在站点 `example.com` 中搜索包含 `password` 的文本文件：

```
site:example.com ext:txt password
```

除了构建自己的查询外，还可以查看 Google Hacking Database（[https://www.exploit-db.com/google-hacking-database/](https://www.exploit-db.com/google-hacking-database/)），这是一个黑客和安全从业者用来共享 Google 搜索查询以查找安全相关信息的网站。它包含许多在侦察过程中可能对你有帮助的搜索查询。例如，你可以找到查找包含密码的文件、管理门户的常见 URL 或使用易受攻击软件构建的页面的查询。

当你使用 Google 搜索进行侦察时，请记住，如果你发送大量搜索查询，Google 将开始要求来自你网络的访问者在进行更多搜索之前完成 CAPTCHA 挑战。这可能会给你网络上的其他人带来困扰，因此我不建议在公司或共享网络上进行 Google dorking。

## 范围发现

现在让我们深入侦察本身。首先，始终验证目标的范围。计划在其政策页面上的范围指定了允许你攻击哪些子域、产品和应用程序。仔细验证公司的哪些资产在范围内，以避免在侦察和黑客过程中越界。例如，如果 `example.com` 的政策指定 `dev.example.com` 和 `test.example.com` 超出范围，则你不应对这些子域执行任何侦察或攻击。

一旦验证了这一点，就发现实际在范围内的内容。你可以攻击哪些域、子域和 IP 地址？组织在这些机器上托管了哪些公司资产？

### WHOIS 和反向 WHOIS

当公司或个人注册域名时，他们需要向域名注册商提供标识信息，例如邮寄地址、电话号码和电子邮件地址。然后任何人都可以使用 `whois` 命令查询这些信息，该命令搜索每个已知域的注册人和所有者信息。你可能会找到相关的联系信息，例如电子邮件、姓名、地址或电话号码：

```bash
$ whois facebook.com
```

此信息并非总是可用，因为一些组织和个人使用称为域名隐私的服务，其中第三方服务提供商将用户的信息替换为转发服务的信息。

然后你可以进行反向 WHOIS 搜索，使用组织名称、电话号码或电子邮件地址搜索数据库，以找到使用这些信息注册的域名。这样，你可以找到属于同一所有者的所有域名。反向 WHOIS 对于查找未以其他方式向公众披露的冷门或内部域极其有用。使用公共反向 WHOIS 工具，如 ViewDNS.info（[https://viewdns.info/reversewhois/](https://viewdns.info/reversewhois/)），进行此搜索。WHOIS 和反向 WHOIS 将为你提供一组良好的顶级域进行工作。

### IP 地址

发现目标顶级域的另一种方法是定位 IP 地址。通过运行 `nslookup` 命令找到你已知域名的 IP 地址。你可以在此处看到 `facebook.com` 位于 `157.240.2.35`：

```bash
$ nslookup facebook.com
Server: 192.168.0.1
Address: 192.168.0.1#53

Non-authoritative answer:
Name:   facebook.com
Address: 157.240.2.35
```

一旦找到已知域的 IP 地址，执行反向 IP 查找。反向 IP 搜索根据给定的 IP 或域查找托管在同一服务器上的域。你也可以使用 ViewDNS.info 进行此操作。

另一种找到范围内 IP 地址的方法是查看自治系统，它们是公共互联网中的可路由网络。自治系统编号（ASN）标识这些网络的所有者。通过检查两个 IP 地址是否共享一个 ASN，你可以确定这些 IP 是否属于同一所有者。

要判断公司是否拥有专用的 IP 范围，运行几次 IP 到 ASN 的转换，看看这些 IP 地址是否映射到单个 ASN。如果一个范围内的许多地址属于同一个 ASN，那么该组织可能拥有专用的 IP 范围。从以下输出中，我们可以推断出 `157.240.2.21` 到 `157.240.2.34` 范围内的任何 IP 可能都属于 Facebook：

```bash
$ whois -h whois.cymru.com 157.240.2.20
AS       | IP           | AS Name
32934    | 157.240.2.20 | FACEBOOK, US

$ whois -h whois.cymru.com 157.240.2.27
AS       | IP           | AS Name
32934    | 157.240.2.27 | FACEBOOK, US

$ whois -h whois.cymru.com 157.240.2.35
AS       | IP           | AS Name
32934    | 157.240.2.35 | FACEBOOK, US
```

`whois` 命令中的 `-h` 标志设置要从中检索信息的 WHOIS 服务器，`whois.cymru.com` 是一个将 IP 转换为 ASN 的数据库。如果公司有专用的 IP 范围并且没有将这些地址标记为超出范围，你可以计划攻击该范围内的每个 IP。

### 证书解析

另一种寻找主机的方法是利用用于加密 Web 流量的安全套接层（SSL）证书。SSL 证书的“主题备用名称”字段允许证书所有者指定使用同一证书的其他主机名，因此你可以通过解析此字段找到这些主机名。使用在线数据库如 crt.sh、Censys 和 Cert Spotter 查找域的证书。

例如，通过对 `facebook.com` 使用 crt.sh 运行证书搜索，我们可以找到 Facebook 的 SSL 证书。你会看到列出了许多属于 Facebook 的其他域名：

```
X509v3 Subject Alternative Name:
    DNS:*.facebook.com
    DNS:*.facebook.net
    DNS:*.fbcdn.net
    DNS:*.fbsbx.com
    DNS:*.messenger.com
    DNS:facebook.com
    DNS:messenger.com
    DNS:*.m.facebook.com
    DNS:*.xx.fbcdn.net
    DNS:*.xy.fbcdn.net
    DNS:*.xz.fbcdn.net
```

crt.sh 网站还有一个有用的实用程序，允许你以 JSON 格式（而不是 HTML）检索信息，以便于解析。只需将 URL 参数 `output=json` 添加到请求 URL：`https://crt.sh/?q=facebook.com&output=json`。

### 子域枚举

在找到目标上尽可能多的域之后，尽可能多地找到这些域上的子域。每个子域都代表攻击网络的一个新角度。枚举子域的最佳方法是使用自动化。

像 Sublist3r、SubBrute、Amass 和 Gobuster 这样的工具可以使用各种单词列表和策略自动枚举子域。例如，Sublist3r 通过查询搜索引擎和在线子域数据库工作，而 SubBrute 是一个暴力破解工具，猜测可能的子域直到找到真实的子域。Amass 结合了 DNS 区域传输、证书解析、搜索引擎和子域数据库来查找子域。你可以构建一个组合多个工具结果的工具以获得最佳效果。我们将在第 80 页的“编写自己的侦察脚本”中讨论如何做到这一点。

要使用许多子域枚举工具，你需要向程序提供一个可能出现在子域中的术语单词列表。你可以在网上找到其他黑客制作的一些好的单词列表。Daniel Miessler 的 SecLists（[https://github.com/danielmiessler/SecLists/](https://github.com/danielmiessler/SecLists/)）是一个相当全面的列表。你也可以使用单词列表生成工具，如 Commonspeak2（[https://github.com/assetnote/commonspeak2](https://github.com/assetnote/commonspeak2)），根据最新的互联网数据生成单词列表。最后，你可以组合几个在网上找到的或你自己生成的单词列表，以获得最全面的结果。以下是从两个单词列表中删除重复项的简单命令：

```bash
sort -u wordlist1.txt wordlist2.txt
```

`sort` 命令行工具对文本文件的行进行排序。当给定多个文件时，它将对所有文件进行排序并将输出写入终端。`-u` 选项告诉 `sort` 仅返回排序列表中的唯一项。

Gobuster 是一个用于暴力破解发现目标 Web 服务器上的子域、目录和文件的工具。其 DNS 模式用于子域暴力破解。在此模式下，你可以使用 `-d` 标志指定要暴力破解的域，使用 `-w` 指定要使用的单词列表：

```bash
gobuster dns -d target_domain -w wordlist
```

一旦你找到了相当数量的子域，你可以通过识别模式来发现更多子域。例如，如果你找到 `example.com` 的两个子域 `1.example.com` 和 `3.example.com`，你可以猜测 `2.example.com` 可能也是一个有效的子域。一个用于自动化此过程的好工具是 Altdns（[https://github.com/infosec-au/altdns/](https://github.com/infosec-au/altdns/)），它通过排列其他子域名称来发现子域。

此外，你可以基于对公司技术栈的了解找到更多子域。例如，如果你已经知道 `example.com` 使用 Jenkins，你可以检查 `jenkins.example.com` 是否是一个有效的子域。

还要寻找子域的子域。例如，在你找到 `dev.example.com` 之后，你可能会找到像 `1.dev.example.com` 这样的子域。你可以通过递归运行枚举工具来找到子域的子域：将第一次运行的结果添加到“已知域”列表中，然后再次运行该工具。

### 服务枚举

接下来，枚举你在找到的机器上托管的服务。由于服务通常在默认端口上运行，查找它们的一个好方法是使用主动或被动扫描对机器进行端口扫描。

在主动扫描中，你直接与服务器交互。主动扫描工具发送请求以连接目标机器的端口，以查找开放的端口。你可以使用 Nmap 或 Masscan 等工具进行主动扫描。例如，这个简单的 Nmap 命令揭示了 `scanme.nmap.org` 上的开放端口：

```bash
$ nmap scanme.nmap.org
Nmap scan report for scanme.nmap.org (45.33.32.156)
Host is up (0.086s latency).
Other addresses for scanme.nmap.org (not scanned): 2600:3c01::f03c:91ff:fe18:bb2f
Not shown: 993 closed ports
PORT     STATE    SERVICE
22/tcp   open     ssh
25/tcp   filtered smtp
80/tcp   open     http
135/tcp  filtered msrpc
445/tcp  filtered microsoft-ds
9929/tcp open     nping-echo
31337/tcp open     Elite

Nmap done: 1 IP address (1 host up) scanned in 230.83 seconds
```

另一方面，在被动扫描中，你使用第三方资源来了解机器的端口，而无需与服务器交互。被动扫描更隐蔽，有助于攻击者避免检测。要找到机器上的服务而不主动扫描它，你可以使用 Shodan，一个允许用户找到连接到互联网的机器的搜索引擎。

使用 Shodan，你可以根据主机名或 IP 地址等条件发现网络摄像头、Web 服务器甚至发电厂的存在。例如，如果你对 `scanme.nmap.org` 的 IP 地址 `45.33.32.156` 运行 Shodan 搜索，你会得到图 5-1 中的结果。你可以看到该搜索产生了与我们的端口扫描不同的数据，并提供了关于服务器的额外信息。

**图 5-1：scanme.nmap.org 的 Shodan 结果页面**

Shodan 的替代品包括 Censys 和 Project Sonar。结合从不同数据库收集的信息以获得最佳效果。通过这些数据库，你还可以找到目标的 IP 地址、证书和软件版本。

### 目录暴力破解

接下来，你可以通过暴力破解找到的 Web 服务器的目录来发现更多站点的攻击面。在服务器上找到目录很有价值，因为通过它们，你可能会发现隐藏的管理面板、配置文件、密码文件、过时的功能、数据库副本和源代码文件。目录暴力破解有时可以让你直接接管服务器！

即使你找不到任何直接的漏洞，目录信息也常常能告诉你应用程序的结构和技术。例如，包含 `phpmyadmin` 的路径名通常意味着应用程序是用 PHP 构建的。

你可以使用 Dirsearch 或 Gobuster 进行目录暴力破解。这些工具使用单词列表构建 URL，然后向 Web 服务器请求这些 URL。如果服务器响应 200 范围内的状态码，则该目录或文件存在。这意味着你可以浏览到该页面并查看应用程序在那里托管了什么。状态码 404 表示目录或文件不存在，而 403 表示它存在但受保护。仔细检查 403 页面，看看是否可以绕过保护以访问内容。

```bash
$ ./dirsearch.py -u scanme.nmap.org -e php
```

Gobuster 的 `dir` 模式用于在特定域或子域上查找其他内容。这包括隐藏的目录和文件。在此模式下，你可以使用 `-u` 标志指定要暴力破解的域或子域，使用 `-w` 指定要使用的单词列表：

```bash
gobuster dir -u target_url -w wordlist
```

手动访问所有通过暴力破解找到的页面可能很耗时。相反，使用截图工具，如 EyeWitness（[https://github.com/FortyNorthSecurity/EyeWitness/](https://github.com/FortyNorthSecurity/EyeWitness/)）或 Snapper（[https://github.com/dxa4481/Snapper/](https://github.com/dxa4481/Snapper/)），自动验证每个位置是否托管了页面。EyeWitness 接受一个 URL 列表，并截取每个页面的屏幕截图。在照片库应用中，你可以快速浏览这些截图，找到看起来有趣的页面。留意隐藏的服务，例如开发者或管理员面板、目录列表页面、分析页面以及看起来过时且维护不善的页面。这些都是漏洞常见的藏身之处。

### 第三方托管

查看公司的第三方托管足迹。例如，寻找组织的 S3 存储桶。S3 代表简单存储服务，是亚马逊的在线存储产品。组织可以付费将资源存储在存储桶中，以便在其 Web 应用程序中使用，或者他们可以使用 S3 存储桶作为备份或存储位置。如果组织使用 Amazon S3，其 S3 存储桶可能包含隐藏的端点、日志、凭证、用户信息、源代码和其他可能对你有用的信息。

如何找到组织的存储桶？一种方法是通过前面提到的 Google dorking。大多数存储桶使用 URL 格式 `BUCKET.s3.amazonaws.com` 或 `s3.amazonaws.com/BUCKET`，因此以下搜索词很可能找到结果：

```
site:s3.amazonaws.com COMPANY_NAME
site:amazonaws.com COMPANY_NAME
```

如果公司为其 S3 存储桶使用自定义 URL，则尝试更灵活的搜索词。公司通常仍会在其自定义存储桶 URL 中放置像 `aws` 和 `s3` 这样的关键词，因此尝试这些搜索：

```
amazonaws s3 COMPANY_NAME
amazonaws bucket COMPANY_NAME
amazonaws COMPANY_NAME
s3 COMPANY_NAME
```

另一种找到存储桶的方法是搜索公司的公共 GitHub 仓库中的 S3 URL。尝试在这些仓库中搜索 `s3` 一词。我们将在下一页的“GitHub 侦察”中讨论使用 GitHub 进行侦察。

GrayhatWarfare（[https://buckets.grayhatwarfare.com/](https://buckets.grayhatwarfare.com/)）是一个在线搜索引擎，你可以用它来查找公开暴露的 S3 存储桶（图 5-7）。它允许你使用关键词搜索存储桶。提供与你的目标相关的关键词，例如应用程序、项目或组织名称，以找到相关的存储桶。

**图 5-7：GrayhatWarfare 主页**

最后，你可以尝试使用关键词暴力破解存储桶。Lazys3（[https://github.com/nahamsec/lazys3/](https://github.com/nahamsec/lazys3/)）是一个可以帮助你做到这一点的工具。它依靠一个单词列表来猜测常见存储桶名称排列的存储桶。另一个好工具是 Bucket Stream（[https://github.com/eth0izzle/bucket-stream/](https://github.com/eth0izzle/bucket-stream/)），它解析属于组织的证书，并根据证书上找到的域名排列来查找 S3 存储桶。Bucket Stream 还会自动检查存储桶是否可访问，因此可以节省你的时间。

一旦你找到几个属于目标组织的存储桶，使用 AWS 命令行工具查看你是否可以访问一个。使用以下命令安装该工具：

```bash
pip install awscli
```

然后按照 Amazon 的文档（[https://docs.aws.amazon.com/cli/latest/userguide/cli-chap-configure.html](https://docs.aws.amazon.com/cli/latest/userguide/cli-chap-configure.html)）配置它以与 AWS 一起工作。

现在你应该能够直接从终端通过 `aws s3` 命令访问存储桶。尝试列出你找到的存储桶的内容：

```bash
aws s3 ls s3://BUCKET_NAME/
```

如果有效，看看你是否可以通过将文件复制到本地机器来读取任何有趣文件的内容：

```bash
aws s3 cp s3://BUCKET_NAME/FILE_NAME /path/to/local/directory
```

收集通过存储桶泄露的任何有用信息，并将其用于未来的利用！如果组织泄露了诸如活动 API 密钥或个人信息等信息，你应该立即报告。仅暴露的 S3 存储桶本身就常被视为一个漏洞。你还可以尝试将新文件上传到存储桶或从存储桶中删除文件。如果你可以弄乱其内容，你可能能够篡改 Web 应用程序的操作或破坏公司数据。例如，此命令将把你本地名为 `TEST_FILE` 的文件复制到目标的 S3 存储桶中：

```bash
aws s3 cp TEST_FILE s3://BUCKET_NAME/
```

此命令将删除你刚刚上传的 `TEST_FILE`：

```bash
aws s3 rm s3://BUCKET_NAME/TEST_FILE
```

这些命令是在不实际篡改目标公司文件的情况下证明你对存储桶具有写访问权限的无害方式。始终上传和删除你自己的测试文件。除非你愿意承担昂贵的诉讼，否则不要在测试期间冒险删除重要的公司资源。

### GitHub 侦察

搜索组织的 GitHub 仓库，查找意外提交的敏感数据，或可能导致发现漏洞的信息。

首先，找到与你的目标相关的 GitHub 用户名。你应该能够通过 GitHub 的搜索栏搜索组织名称或产品名称，或通过检查已知员工的 GitHub 帐户来找到这些用户名。

当你找到要审计的用户名后，访问他们的页面。找到与你正在测试的项目相关的仓库并记录下来，同时记录组织的主要贡献者的用户名，这可以帮助你找到更多相关的仓库。

然后深入代码。对于每个仓库，特别注意“Issues”和“Commits”部分。这些部分充满了潜在的信息泄露：它们可能指向未解决的错误、有问题的代码以及最新的代码修复和安全补丁。最近未经时间考验的代码更改更可能包含错误。查看实现的任何保护机制，看看是否可以绕过它们。你还可以在“Code”部分搜索潜在易受攻击的代码片段。一旦你找到感兴趣的文件，检查文件页面右上角的“Blame”和“History”部分，以了解它是如何开发的（图 5-8）。

**图 5-8：History 和 Blame 部分**

我们将在第 22 章更深入地讨论审查源代码，但在侦察阶段，寻找硬编码的秘密，如 API 密钥、加密密钥和数据库密码。搜索组织的仓库中的诸如 `key`、`secret` 和 `password` 等词，以找到可用于访问内部系统的硬编码用户凭证。找到泄露的凭证后，你可以使用 KeyHacks（[https://github.com/streaak/keyhacks/](https://github.com/streaak/keyhacks/)）检查凭证是否有效，并学习如何使用它们访问目标的服务。

你还应该在项目中搜索敏感功能。查看是否有任何源代码处理重要功能，如认证、密码重置、状态更改操作或私有信息读取。注意处理用户输入的代码，例如 HTTP 请求参数、HTTP 头、HTTP 请求路径、数据库条目、文件读取和文件上传，因为它们为攻击者提供了利用应用程序漏洞的潜在入口点。查找任何配置文件，因为它们允许你收集有关基础设施的更多信息。此外，搜索你可以攻击的旧端点和 S3 存储桶 URL。记录这些文件，以备将来进一步审查。

过时的依赖项和危险函数的不加检查的使用也是漏洞的重要来源。注意正在使用的依赖项和导入，并浏览版本列表以查看它们是否过时。记录任何过时的依赖项。你以后可以使用此信息查找公开披露的、可能对目标有效的漏洞。

像 Gitrob 和 TruffleHog 这样的工具可以自动化 GitHub 侦察过程。Gitrob（[https://github.com/michenriksen/gitrob/](https://github.com/michenriksen/gitrob/)）定位推送到 GitHub 公共仓库的潜在敏感文件。TruffleHog（[https://github.com/trufflesecurity/truffleHog/](https://github.com/trufflesecurity/truffleHog/)）专门通过进行正则表达式搜索和扫描高熵字符串来查找公共 GitHub 仓库中的秘密。

### 其他隐秘的 OSINT 技术

到目前为止，我讨论的许多策略都是开源情报（OSINT）的例子，即从公共信息来源收集情报的做法。本节详述了你可以用来提取有价值信息的其他 OSINT 来源。

首先，查看公司工程职位的招聘信息。工程职位列表通常会揭示公司使用的技术。例如，看看这样的广告：

> **全栈工程师**
> **最低要求：**
> 精通 Python 和 C/C++
> Linux 经验
> 有 Flask、Django 和 Node.js 经验
> 有 Amazon Web Services 经验，特别是 EC2、ECS、S3 和 RDS

通过阅读这些，你知道公司使用 Flask、Django 和 Node.js 构建其 Web 应用程序。工程师可能还在后端使用 Python、C 和 C++ 和 Linux 机器。最后，他们使用 AWS 来外包其操作和文件存储。

如果你找不到相关的招聘信息，可以在 LinkedIn 上搜索员工的个人资料，阅读员工的个人博客或他们在 Stack Overflow 和 Quora 等论坛上的工程问题。公司顶尖员工的专业知识往往反映了开发中使用的技术。

另一个信息来源是员工的 Google 日历。人们的工作日历通常包含会议记录、幻灯片，有时甚至包含登录凭据。如果员工不小心将日历与公众共享，你可能能够访问这些内容。组织或其员工的社交媒体页面也可能泄露有价值的信息。例如，黑客实际上在办公室自拍照背景中的便利贴上发现过有效的凭证！

如果公司有工程邮件列表，请订阅以深入了解公司的技术和开发过程。还可以查看公司的 SlideShare 或 Pastebin 帐户。有时，当组织在会议上发言或举行内部会议时，他们会将幻灯片上传到 SlideShare 以供参考。你可能能够找到有关公司面临的技术栈和安全挑战的信息。

Pastebin（[https://pastebin.com/](https://pastebin.com/)）是一个用于在线粘贴和存储文本短时间的网站。人们使用它在不同机器之间或与他人共享文本。工程师有时使用它与同事共享源代码或服务器日志以供查看或协作，因此它可能是一个很好的信息来源。你还可以找到上传的凭证和开发注释。访问 Pastebin，搜索目标的组织名称，看看会发生什么！你还可以使用自动化工具，如 PasteHunter（[https://github.com/kevthehermit/PasteHunter/](https://github.com/kevthehermit/PasteHunter/)），扫描公共粘贴的数据。

最后，查阅像 Wayback Machine（[https://archive.org/web/](https://archive.org/web/)）这样的存档网站，这是互联网内容的数字记录（图 5-9）。它记录了站点在不同时间点的内容。使用 Wayback Machine，你可以找到过时但仍在使用的旧端点、目录列表、被遗忘的子域、URL 和文件。Tomnomnom 的工具 Waybackurls（[https://github.com/tomnomnom/waybackurls/](https://github.com/tomnomnom/waybackurls/)）可以从 Wayback Machine 自动提取端点和 URL。

**图 5-9：Wayback Machine 存档互联网，允许你查看已被网站删除的页面。**

## 技术栈指纹识别

指纹识别技术可以帮助你更好地理解目标应用程序。指纹识别是识别机器或应用程序使用的软件品牌和版本的过程。这些信息允许你对应用程序执行有针对性的攻击，因为你可以搜索与特定版本相关的任何已知的配置错误和公开披露的漏洞。例如，如果你知道服务器正在使用可能受已披露漏洞影响的旧版 Apache，你可以立即尝试使用该漏洞攻击服务器。

安全社区将已知漏洞分类为公共漏洞和暴露（CVE），并为每个 CVE 分配一个编号以供参考。在 CVE 数据库（[https://cve.mitre.org/cve/search_cve_list.html](https://cve.mitre.org/cve/search_cve_list.html)）上搜索它们。

对应用程序进行指纹识别的最简单方法是直接与应用程序交互。首先，在机器上运行带有 `-sV` 标志的 Nmap，以在端口扫描上启用版本检测。在这里，你可以看到 Nmap 尝试为我们指纹识别目标主机上运行的一些软件：

```bash
$ nmap scanme.nmap.org -sV
Starting Nmap 7.60 ( https://nmap.org )
Nmap scan report for scanme.nmap.org (45.33.32.156)
Host is up (0.065s latency).
Other addresses for scanme.nmap.org (not scanned): 2600:3c01::f03c:91ff:fe18:bb2f
Not shown: 992 closed ports
PORT     STATE    SERVICE     VERSION
22/tcp   open     ssh         OpenSSH 6.6.1p1 Ubuntu 2ubuntu2.13 (Ubuntu Linux; protocol 2.0)
25/tcp   filtered smtp
80/tcp   open     http        Apache httpd 2.4.7 ((Ubuntu))
135/tcp  filtered msrpc
139/tcp  filtered netbios-ssn
445/tcp  filtered microsoft-ds
9929/tcp open     nping-echo  Nping echo
31337/tcp open     tcpwrapped
Service Info: OS: Linux; CPE: cpe:/o:linux:linux_kernel

Service detection performed. Please report any incorrect results at https://nmap.org/submit/ .
Nmap done: 1 IP address (1 host up) scanned in 9.19 seconds
```

接下来，在 Burp 中，向服务器发送一个 HTTP 请求，检查用于深入了解技术栈的 HTTP 头。服务器可能泄露许多有助于指纹识别其技术的信息：

```
Server: Apache/2.0.6 (Ubuntu)
X-Powered-By: PHP/5.0.1
X-Generator: Drupal 8
X-Drupal-Dynamic-Cache: UNCACHEABLE
Set-Cookie: PHPSESSID=abcde;
```

像 `Server` 和 `X-Powered-By` 这样的 HTTP 头是技术栈的良好指标。`Server` 头通常揭示服务器上运行的软件版本。`X-Powered-By` 揭示所使用的服务器或脚本语言。此外，某些头仅由特定技术使用。例如，只有 Drupal 使用 `X-Generator` 和 `X-Drupal-Dynamic-Cache`。特定于技术的 cookie，如 `PHPSESSID`，也是线索；如果服务器发回一个名为 `PHPSESSID` 的 cookie，它很可能是用 PHP 开发的。

网页的 HTML 源代码也可以提供线索。许多 Web 框架或其他技术会在源代码中嵌入签名。右键单击页面，选择“查看源代码”，然后按 `CTRL-F` 搜索诸如 `powered by`、`built with` 和 `running` 等短语。例如，你可能会在源代码中找到 `Powered by: WordPress 3.3.2`。

检查特定于技术的文件扩展名、文件名、文件夹和目录。例如，根目录下名为 `phpmyadmin` 的文件，如 `https://example.com/phpmyadmin`，意味着应用程序运行 PHP。包含模板的名为 `jinja2` 的目录意味着该站点可能使用 Django 和 Jinja2。你可以通过访问特定技术的文档来找到有关其文件系统特征的更多信息。

## 编写自己的侦察脚本

你现在可能已经意识到，良好的侦察是一个广泛的过程。但它不必耗时或难以管理。我们已经讨论了几种利用自动化功能使过程更轻松的工具。

有时，你会发现编写自己的脚本很方便。脚本是设计为由程序执行的命令列表。它们用于自动化任务，如数据分析、网页生成和系统管理。对于我们漏洞赏金猎人来说，脚本是一种快速有效地执行侦察、测试和利用的方式。例如，你可以编写一个脚本来扫描目标的新子域，或枚举服务器上潜在敏感的文件和目录。一旦你学会了如何编写脚本，可能性是无限的。

本节特别涵盖 bash 脚本——它们是什么以及为什么你应该使用它们。你将学习如何使用 bash 简化你的侦察过程，甚至编写自己的工具。我将假设你对编程语言的工作原理有基本了解，包括变量、条件语句、循环和函数，所以如果你不熟悉这些概念，请参加 Codecademy（[https://www.codecademy.com/](https://www.codecademy.com/)）的编程入门课程或阅读一本编程书籍。

Bash 脚本，或任何类型的 shell 脚本，对于管理复杂性和自动化重复性任务非常有用。如果你的命令涉及多个输入参数，或者一个命令的输入依赖于另一个命令的输出，手动输入所有内容可能会很快变得复杂并增加编程错误的机会。另一方面，你可能有一堆想要多次执行的命令。脚本在这里很有用，因为它们为你节省了一遍又一遍地输入相同命令的麻烦。每次只需运行脚本，就完成了。

### 理解 Bash 脚本基础

让我们编写第一个脚本。打开任何文本编辑器进行跟随。你编写的每个 shell 脚本的第一行应该是 shebang 行。它以井号（`#`）和感叹号（`!`）开头，并声明脚本要使用的解释器。这允许像二进制文件一样执行纯文本文件。我们将用它来表明我们正在使用 bash。

假设我们想编写一个执行两个命令的脚本；它应该在目标上运行 Nmap 然后运行 Dirsearch。我们可以像这样将命令放入脚本中：

```bash
#!/bin/bash
nmap scanme.nmap.org
/PATH/TO/dirsearch.py -u scanme.nmap.org -e php
```

这个脚本不是很有用；它只能扫描一个站点，`scanme.nmap.org`。相反，我们应该让用户向 bash 脚本提供输入参数，以便他们可以选择要扫描的站点。在 bash 语法中，`$1` 代表传入的第一个参数，`$2` 是第二个参数，依此类推。此外，`$@` 代表所有传入的参数，而 `$#` 代表参数的总数。让我们允许用户使用分配给变量 `$1` 的第一个输入参数来指定他们的目标：

```bash
#!/bin/bash
nmap $1
/PATH/TO/dirsearch.py -u $1 -e php
```

现在，这些命令将针对用户作为第一个参数传入的任何域执行。

请注意，脚本的第三行包含 `/PATH/TO/dirsearch.py`。你应该将 `/PATH/TO/` 替换为你存储 Dirsearch 脚本的目录的绝对路径。如果你不指定其位置，你的计算机将尝试在当前目录中查找它，除非你将 Dirsearch 文件存储在与你的 shell 脚本相同的目录中，否则 bash 将找不到它。

确保脚本能够找到要使用的命令的另一种方法是通过 `PATH` 变量，这是 Unix 系统中的一个环境变量，指定了在哪里找到可执行的二进制文件。如果你运行此命令将 Dirsearch 的目录添加到你的 `PATH` 中，你可以从任何地方运行该工具，而无需指定其绝对路径：

```bash
export PATH="PATH_TO_DIRSEARCH:$PATH"
```

执行此命令后，你应该能够直接使用 Dirsearch：

```bash
#!/bin/bash
nmap $1
dirsearch.py -u $1 -e php
```

请注意，你需要在重启终端后再次运行 `export` 命令，以便你的 `PATH` 包含 Dirsearch 的路径。如果你不想一遍又一遍地导出 `PATH`，可以将 `export` 命令添加到你的 `~/.bash_profile` 文件中，该文件存储你的 bash 偏好和配置。你可以通过用你喜欢的文本编辑器打开 `~/.bash_profile` 并将 `export` 命令添加到文件底部来做到这一点。

脚本完成了！将其保存到当前目录中，文件名为 `recon.sh`。`.sh` 扩展名是 shell 脚本的传统扩展名。通过运行命令 `cd /location/of/your/script`，确保你的终端工作目录与存储脚本的目录相同。使用以下命令在终端中执行脚本：

```bash
$ ./recon.sh
```

你可能会看到如下消息：

```
permission denied: ./recon.sh
```

这是因为当前用户没有执行脚本的权限。出于安全原因，大多数文件默认不可执行。你可以通过在终端中运行以下命令来为所有用户添加执行权限来纠正此行为：

```bash
$ chmod +x recon.sh
```

`chmod` 命令编辑文件的权限，`+x` 表示我们想要为所有用户添加执行权限。如果你只想为脚本的所有者授予执行权限，请改用此命令：

```bash
$ chmod 700 recon.sh
```

### 将工具输出保存到文件

为了以后分析侦察结果，你可能希望将脚本的输出保存在单独的文件中。这就是输入和输出重定向发挥作用的地方。输入重定向是使用文件的内容或另一个程序的输出作为脚本的输入。输出重定向是将程序的输出重定向到另一个位置，例如文件或另一个程序。

以下是一些最有用的重定向运算符：

- `PROGRAM > FILENAME`：将程序的输出写入到具有该名称的文件中。（它会先清除文件中的任何内容。如果文件不存在，它还会创建该文件。）
- `PROGRAM >> FILENAME`：将程序的输出追加到文件的末尾，而不会清除文件的原始内容。
- `PROGRAM < FILENAME`：从文件中读取并将其内容用作程序输入。
- `PROGRAM1 | PROGRAM2`：使用 `PROGRAM1` 的输出作为 `PROGRAM2` 的输入。

例如，我们可以将 Nmap 和 Dirsearch 扫描的结果写入不同的文件：

```bash
#!/bin/bash
echo "Creating directory $1_recon."   # 1
mkdir $1_recon                        # 2
nmap $1 > $1_recon/nmap               # 3
echo "The results of nmap scan are stored in $1_recon/nmap."
/PATH/TO/dirsearch.py -u $1 -e php --simple-report=$1_recon/dirsearch   # 4
echo "The results of dirsearch scan are stored in $1_recon/dirsearch."
```

`echo` 命令 ➊ 向终端打印一条消息。接下来，`mkdir` 创建一个名为 `DOMAIN_recon` 的目录 ➋。我们将 nmap 的结果存储在新创建的目录中名为 `nmap` 的文件中 ➌。Dirsearch 的 `simple-report` 标志 ➍ 在指定位置生成报告。我们将 Dirsearch 的结果存储在新目录中名为 `dirsearch` 的文件中。

你可以通过引入变量来引用文件、名称和值，使脚本更易于管理。bash 中的变量可以使用以下语法分配：`VARIABLE_NAME=VARIABLE_VALUE`。请注意，等号周围不应有空格。引用变量的语法是 `$VARIABLE_NAME`。让我们将这些实现到脚本中：

```bash
#!/bin/bash
PATH_TO_DIRSEARCH="/Users/vickieli/tools/dirsearch"
DOMAIN=$1
DIRECTORY=${DOMAIN}_recon      # 1
echo "Creating directory $DIRECTORY."
mkdir $DIRECTORY
nmap $DOMAIN > $DIRECTORY/nmap
echo "The results of nmap scan are stored in $DIRECTORY/nmap."
$PATH_TO_DIRSEARCH/dirsearch.py -u $DOMAIN -e php --simple-report=$DIRECTORY/dirsearch   # 2
echo "The results of dirsearch scan are stored in $DIRECTORY/dirsearch."
```

我们使用 `${DOMAIN}_recon` 而不是 `$DOMAIN_recon` ➊，因为否则 bash 会将 `DOMAIN_recon` 整体识别为变量名。花括号告诉 bash `DOMAIN` 是变量名，而 `_recon` 是我们附加到其上的纯文本。注意，我们还将 Dirsearch 的路径存储在一个变量中，以便将来易于更改 ➋。

使用重定向，你现在可以编写 shell 脚本，用单个命令运行许多工具并将其输出保存在单独的文件中。

### 将扫描日期添加到输出

假设你想将当前日期添加到脚本的输出中，或者选择运行哪些扫描，而不是总是同时运行 Nmap 和 Dirsearch。如果你想编写具有此类更多功能的工具，你必须了解一些高级 shell 脚本概念。

例如，一个有用的是命令替换，即对命令的输出进行操作。使用 `$()` 告诉 Unix 执行括号中的命令，并将其输出分配给变量的值。让我们练习使用这种语法：

```bash
#!/bin/bash
PATH_TO_DIRSEARCH="/Users/vickieli/tools/dirsearch"
TODAY=$(date)                 # 1
echo "This scan was created on $TODAY"   # 2
DOMAIN=$1
DIRECTORY=${DOMAIN}_recon
echo "Creating directory $DIRECTORY."
mkdir $DIRECTORY
nmap $DOMAIN > $DIRECTORY/nmap
echo "The results of nmap scan are stored in $DIRECTORY/nmap."
$PATH_TO_DIRSEARCH/dirsearch.py -u $DOMAIN -e php --simple-report=$DIRECTORY/dirsearch
echo "The results of dirsearch scan are stored in $DIRECTORY/dirsearch."
```

在 ➊，我们将 `date` 命令的输出分配给变量 `TODAY`。`date` 命令显示当前日期和时间。这使我们能够输出一条消息，指示我们执行扫描的日期 ➋。

### 添加选项以选择要运行的工具

现在，为了选择性地只运行某些工具，你需要使用条件语句。在 bash 中，`if` 语句的语法如下。请注意，条件语句以 `fi` 关键字结束，这是 `if` 的倒写：

```bash
if [ condition 1 ]
then
   # 如果满足 condition 1 则执行
elif [ condition 2 ]
then
   # 如果满足 condition 2 且不满足 condition 1 则执行
else
   # 如果两个条件都不满足则执行其他操作
fi
```

假设我们希望用户能够指定扫描 `MODE`，如下所示：

```bash
$ ./recon.sh scanme.nmap.org MODE
```

我们可以这样实现这个功能：

```bash
#!/bin/bash
PATH_TO_DIRSEARCH="/Users/vickieli/tools/dirsearch"
TODAY=$(date)
echo "This scan was created on $TODAY"
DIRECTORY=${DOMAIN}_recon
echo "Creating directory $DIRECTORY."
mkdir $DIRECTORY
if [ $2 == "nmap-only" ]          # 1
then
    nmap $DOMAIN > $DIRECTORY/nmap   # 2
    echo "The results of nmap scan are stored in $DIRECTORY/nmap."
elif [ $2 == "dirsearch-only" ]    # 3
then
    $PATH_TO_DIRSEARCH/dirsearch.py -u $DOMAIN -e php --simple-report=$DIRECTORY/dirsearch   # 4
    echo "The results of dirsearch scan are stored in $DIRECTORY/dirsearch."
else                              # 5
    nmap $DOMAIN > $DIRECTORY/nmap   # 6
    echo "The results of nmap scan are stored in $DIRECTORY/nmap."
    $PATH_TO_DIRSEARCH/dirsearch.py -u $DOMAIN -e php --simple-report=$DIRECTORY/dirsearch
    echo "The results of dirsearch scan are stored in $DIRECTORY/dirsearch."
fi
```

如果用户指定 `nmap-only` ➊，我们只运行 nmap 并将结果存储到名为 `nmap` 的文件中 ➋。如果用户指定 `dirsearch-only` ➌，我们只执行 Dirsearch 并存储其结果 ➍。如果用户两者都不指定 ➎，我们同时运行两个扫描 ➏。

现在，你可以通过指定以下命令之一来使你的工具仅运行 Nmap 或 Dirsearch 命令：

```bash
$ ./recon.sh scanme.nmap.org nmap-only
$ ./recon.sh scanme.nmap.org dirsearch-only
```

### 运行其他工具

如果你还想选择从 crt.sh 工具检索信息呢？例如，你想在这三种模式之间切换，或者一次运行所有三个侦察工具：

```bash
$ ./recon.sh scanme.nmap.org nmap-only
$ ./recon.sh scanme.nmap.org dirsearch-only
$ ./recon.sh scanme.nmap.org crt-only
```

我们可以重写 `if-else` 语句以处理三个选项：首先，我们检查 `MODE` 是否为 `nmap-only`。然后我们检查 `MODE` 是否为 `dirsearch-only`，最后检查 `MODE` 是否为 `crt-only`。但那将是很多 `if-else` 语句，使代码复杂。

相反，让我们使用 bash 的 `case` 语句，它允许你针对一个变量匹配多个值，而无需通过一长串 `if-else` 语句。`case` 语句的语法如下所示。请注意，该语句以 `esac` 结束，即 `case` 的倒写：

```bash
case $VARIABLE_NAME in
    case1)
        Do something
        ;;
    case2)
        Do something
        ;;
    caseN)
        Do something
        ;;
    *)
        Default case, this case is executed if no other case matches.
        ;;
esac
```

我们可以通过使用 `case` 语句而不是多个 `if-else` 语句来实现功能，从而改进我们的脚本：

```bash
#!/bin/bash
PATH_TO_DIRSEARCH="/Users/vickieli/tools/dirsearch"
TODAY=$(date)
echo "This scan was created on $TODAY"
DOMAIN=$1
DIRECTORY=${DOMAIN}_recon
echo "Creating directory $DIRECTORY."
mkdir $DIRECTORY
case $2 in
    nmap-only)
        nmap $DOMAIN > $DIRECTORY/nmap
        echo "The results of nmap scan are stored in $DIRECTORY/nmap."
        ;;
    dirsearch-only)
        $PATH_TO_DIRSEARCH/dirsearch.py -u $DOMAIN -e php --simple-report=$DIRECTORY/dirsearch
        echo "The results of dirsearch scan are stored in $DIRECTORY/dirsearch."
        ;;
    crt-only)
        curl "https://crt.sh/?q=$DOMAIN&output=json" -o $DIRECTORY/crt   # 1
        echo "The results of cert parsing is stored in $DIRECTORY/crt."
        ;;
    *)
        nmap $DOMAIN > $DIRECTORY/nmap
        echo "The results of nmap scan are stored in $DIRECTORY/nmap."
        $PATH_TO_DIRSEARCH/dirsearch.py -u $DOMAIN -e php --simple-report=$DIRECTORY/dirsearch
        echo "The results of dirsearch scan are stored in $DIRECTORY/dirsearch."
        curl "https://crt.sh/?q=$DOMAIN&output=json" -o $DIRECTORY/crt
        echo "The results of cert parsing is stored in $DIRECTORY/crt."
        ;;
esac
```

`curl` 命令 ➊ 下载页面的内容。我们在这里使用它从 crt.sh 下载数据。`curl` 的 `-o` 选项允许你指定输出文件。但是请注意，我们的代码有很多重复！运行每种扫描类型的代码部分重复了两次。让我们尝试通过使用函数来减少重复。bash 函数的语法如下所示：

```bash
FUNCTION_NAME() {
    DO_SOMETHING
}
```

声明函数后，你可以像脚本中的任何其他 shell 命令一样调用它。让我们向脚本添加函数：

```bash
#!/bin/bash
PATH_TO_DIRSEARCH="/Users/vickieli/tools/dirsearch"
TODAY=$(date)
echo "This scan was created on $TODAY"
DOMAIN=$1
DIRECTORY=${DOMAIN}_recon
echo "Creating directory $DIRECTORY."
mkdir $DIRECTORY

nmap_scan() {                     # 1
    nmap $DOMAIN > $DIRECTORY/nmap
    echo "The results of nmap scan are stored in $DIRECTORY/nmap."
}

dirsearch_scan() {                # 2
    $PATH_TO_DIRSEARCH/dirsearch.py -u $DOMAIN -e php --simple-report=$DIRECTORY/dirsearch
    echo "The results of dirsearch scan are stored in $DIRECTORY/dirsearch."
}

crt_scan() {                      # 3
    curl "https://crt.sh/?q=$DOMAIN&output=json" -o $DIRECTORY/crt
    echo "The results of cert parsing is stored in $DIRECTORY/crt."
}

case $2 in                        # 4
    nmap-only)
        nmap_scan
        ;;
    dirsearch-only)
        dirsearch_scan
        ;;
    crt-only)
        crt_scan
        ;;
    *)
        nmap_scan
        dirsearch_scan
        crt_scan
        ;;
esac
```

你可以看到我们简化了代码。我们创建了三个函数：`nmap_scan` ➊、`dirsearch_scan` ➋ 和 `crt_scan` ➌。我们将扫描和 `echo` 命令放在这些函数中，以便我们可以重复调用它们而无需一遍又一遍地编写相同的代码 ➍。这种简化在这里可能看起来不多，但是当你编写更复杂的程序时，使用函数重用代码将为你省去很多麻烦。

请记住，除了像 `$1`、`$2` 和 `$3` 这样的输入参数外，所有 bash 变量都是全局的。这意味着像 `$DOMAIN`、`$DIRECTORY` 和 `$PATH_TO_DIRSEARCH` 这样的变量在声明后在整个脚本中都可用，即使它们在函数内声明。另一方面，像 `$1`、`$2` 和 `$3` 这样的参数值只能引用调用函数时传递的值，因此你不能在函数内使用脚本的输入参数，像这样：

```bash
nmap_scan() {
    nmap $1 > $DIRECTORY/nmap
    echo "The results of nmap scan are stored in $DIRECTORY/nmap."
}
nmap_scan
```

在这里，函数中的 `$1` 指的是调用 `nmap_scan` 时传递的第一个参数，而不是我们的 `recon.sh` 脚本调用时的参数。由于 `nmap_scan` 没有传递任何参数，`$1` 是空的。

### 解析结果

现在我们有了一个执行三种类型扫描并将结果存储到文件中的工具。但是扫描之后，我们仍然必须手动阅读和理解复杂的输出文件。有没有办法也加速这个过程？

假设你想在输出文件中搜索某条信息。你可以使用全局正则表达式打印（`grep`）来做到这一点。这个命令行工具用于在文本、文件和命令输出中执行搜索。一个简单的 `grep` 命令如下所示：

```bash
grep password file.txt
```

这告诉 `grep` 在文件 `file.txt` 中搜索字符串 `password`，然后在标准输出中打印匹配的行。例如，我们可以快速搜索 Nmap 输出文件，看看目标是否开放了 80 端口：

```bash
$ grep 80 TARGET_DIRECTORY/nmap
80/tcp open http
```

你还可以通过在搜索字符串中使用正则表达式使搜索更灵活。正则表达式（regex）是一个描述搜索模式的特殊字符串。它可以帮助你仅显示输出的特定部分。例如，你可能已经注意到 Nmap 命令的输出如下所示：

```log

Nmap scan report for scanme.nmap.org (45.33.32.156)
Host is up (0.065s latency).
Other addresses for scanme.nmap.org (not scanned): 2600:3c01::f03c:91ff:fe18:bb2f
Not shown: 992 closed ports
PORT     STATE    SERVICE
22/tcp   open     ssh
25/tcp   filtered smtp
80/tcp   open     http
135/tcp  filtered msrpc
139/tcp  filtered netbios-ssn
445/tcp  filtered microsoft-ds
9929/tcp open     nping-echo
31337/tcp open     Elite

```

你可能希望从文件中修剪无关的消息，使其看起来更像这样：

```
PORT     STATE    SERVICE
22/tcp   open     ssh
25/tcp   filtered smtp
80/tcp   open     http
135/tcp  filtered msrpc
139/tcp  filtered netbios-ssn
445/tcp  filtered microsoft-ds
9929/tcp open     nping-echo
31337/tcp open     Elite
```

使用此命令过滤掉 Nmap 输出开头和结尾的消息，仅保留报告的基本部分：

```bash
grep -E "^\S+\s+\S+\s+\S+$" DIRECTORY/nmap > DIRECTORY/nmap_cleaned
```

`-E` 标志告诉 `grep` 你正在使用正则表达式。正则表达式由两部分组成：常量和运算符。常量是字符串集，而运算符是表示对这些字符串进行操作的符号。这两个元素共同使正则表达式成为强大的模式匹配工具。以下是表示字符的正则表达式运算符的快速概述：

- `\d` 匹配任何数字。
- `\w` 匹配任何字符。
- `\s` 匹配任何空白字符，`\S` 匹配任何非空白字符。
- `.` 匹配任何单个字符。
- `\` 转义一个特殊字符。
- `^` 匹配字符串或行的开头。
- `$` 匹配字符串或行的结尾。

还有几个运算符指定要匹配的字符数：

- `*` 匹配前一个字符零次或多次。
- `+` 匹配前一个字符一次或多次。
- `{3}` 匹配前一个字符三次。
- `{1,3}` 匹配前一个字符一至三次。
- `{1,}` 匹配前一个字符一次或多次。
- `[abc]` 匹配括号内的一个字符。
- `[a-z]` 匹配 a 到 z 范围内的一个字符。
- `(a|b|c)` 匹配 a 或 b 或 c。

让我们再看一下这里的正则表达式。还记得 `\s` 匹配任何空白字符，`\S` 匹配任何非空白字符吗？这意味着 `\s+` 会匹配一个或多个字符长的任何空白字符，而 `\S+` 会匹配一个或多个字符长的任何非空白字符。此正则表达式模式指定我们应该提取包含由两个空白字符分隔的三个字符串的行：

```
"^\S+\s+\S+\s+\S+$"
```

过滤后的输出将如下所示：

```
PORT     STATE    SERVICE
22/tcp   open     ssh
25/tcp   filtered smtp
80/tcp   open     http
135/tcp  filtered msrpc
139/tcp  filtered netbios-ssn
445/tcp  filtered microsoft-ds
9929/tcp open     nping-echo
31337/tcp open     Elite
```

为了考虑到命令输出中可能存在的额外空白字符，让我们在搜索字符串周围再添加两个可选空格：

```
"^\s*\S+\s+\S+\s+\S+\s*$"
```

你可以使用更多高级正则表达式功能来执行更复杂的匹配。然而，这组简单的运算符足以满足我们的目的。有关正则表达式语法的完整指南，请阅读 RexEgg 的速查表（[https://www.rexegg.com/regex-quickstart.html](https://www.rexegg.com/regex-quickstart.html)）。

### 构建主报告

如果你想从所有三个输出文件生成主报告怎么办？你需要解析来自 crt.sh 的 JSON 文件。你可以使用 `jq` 来做到这一点，`jq` 是一个处理 JSON 的命令行实用程序。如果我们检查来自 crt.sh 的 JSON 输出文件，我们可以看到我们需要提取每个证书项的 `name_value` 字段来提取域名。此命令正是这样做的：

```bash
$ jq -r ".[] | .name_value" $DOMAIN/crt
```

`-r` 标志告诉 `jq` 将输出直接写入标准输出，而不是将其格式化为 JSON 字符串。`.[]` 遍历 JSON 文件中的数组，而 `.name_value` 提取每个项的 `name_value` 字段。最后，`$DOMAIN/crt` 是 `jq` 命令的输入文件。要了解有关 `jq` 如何工作的更多信息，请阅读其手册（[https://stedolan.github.io/jq/manual/](https://stedolan.github.io/jq/manual/)）。

要将所有输出文件组合成一个主报告，编写如下脚本：

```bash
#!/bin/bash
PATH_TO_DIRSEARCH="/Users/vickieli/tools/dirsearch"
DOMAIN=$1
DIRECTORY=${DOMAIN}_recon
echo "Creating directory $DIRECTORY."
mkdir $DIRECTORY

nmap_scan() {
    nmap $DOMAIN > $DIRECTORY/nmap
    echo "The results of nmap scan are stored in $DIRECTORY/nmap."
}

dirsearch_scan() {
    $PATH_TO_DIRSEARCH/dirsearch.py -u $DOMAIN -e php --simple-report=$DIRECTORY/dirsearch
    echo "The results of dirsearch scan are stored in $DIRECTORY/dirsearch."
}

crt_scan() {
    curl "https://crt.sh/?q=$DOMAIN&output=json" -o $DIRECTORY/crt
    echo "The results of cert parsing is stored in $DIRECTORY/crt."
}

case $2 in
    nmap-only)
        nmap_scan
        ;;
    dirsearch-only)
        dirsearch_scan
        ;;
    crt-only)
        crt_scan
        ;;
    *)
        nmap_scan
        dirsearch_scan
        crt_scan
        ;;
esac

echo "Generating recon report from output files..."
TODAY=$(date)
echo "This scan was created on $TODAY" > $DIRECTORY/report          # 1
echo "Results for Nmap:" >> $DIRECTORY/report
grep -E "^\s*\S+\s+\S+\s+\S+\s*$" $DIRECTORY/nmap >> $DIRECTORY/report   # 2
echo "Results for Dirsearch:" >> $DIRECTORY/report
cat $DIRECTORY/dirsearch >> $DIRECTORY/report                       # 3
echo "Results for crt.sh:" >> $DIRECTORY/report
jq -r ".[] | .name_value" $DIRECTORY/crt >> $DIRECTORY/report       # 4
```

首先，我们创建一个名为 `report` 的新文件，并将今天的日期写入其中 ➊，以便跟踪报告生成的时间。然后我们将 nmap 和 dirsearch 命令的结果附加到报告文件中 ➋。`cat` 命令将文件的内容打印到标准输出，但我们也可以使用它将文件的内容重定向到另一个文件 ➌。最后，我们从 crt.sh 报告中提取域名并将其附加到报告文件的末尾 ➍。

### 扫描多个域

如果我们想一次扫描多个域怎么办？在侦察目标时，我们可能会从组织的几个域名开始。例如，我们知道 Facebook 拥有 `facebook.com` 和 `fbcdn.net`。但是我们当前的脚本一次只允许我们扫描一个域。我们需要编写一个工具，可以用单个命令扫描多个域，如下所示：

```bash
./recon.sh facebook.com fbcdn.net nmap-only
```

当我们像这样扫描多个域时，我们需要一种方法来区分哪些参数指定扫描 `MODE`，哪些指定目标域。正如你已经从我介绍的工具中看到的，大多数工具允许用户通过使用命令行选项或标志（如 `-u` 和 `--simple-report`）来修改工具的行为。

`getopts` 工具通过使用单字符标志从命令行解析选项。其语法如下，其中 `OPTSTRING` 指定 `getopts` 应识别的选项字母。例如，如果它应该识别选项 `-m` 和 `-i`，你应该指定 `mi`。如果你希望一个选项包含参数值，则该字母后应跟一个冒号，如下所示：`m:i`。`NAME` 参数指定存储选项字母的变量名。

```bash
getopts OPTSTRING NAME
```

为了实现我们的多域扫描功能，我们可以让用户使用 `-m` 标志指定扫描模式，并假设所有其他参数都是域。在这里，我们告诉 `getopts` 如果选项标志是 `-m` 并且此选项应包含一个输入值，则识别该选项。`getopts` 工具还会自动将任何选项的值存储到 `$OPTARG` 变量中。我们可以将该值存储到名为 `MODE` 的变量中：

```bash
getopts "m:" OPTION
MODE=$OPTARG
```

现在，如果你使用 `-m` 标志运行 shell 脚本，脚本将知道你正在指定扫描 `MODE`！请注意，`getopts` 在遇到不以 `-` 字符开头的参数时会停止解析参数，因此你在运行脚本时需要将扫描模式放在域参数之前：

```bash
./recon.sh -m nmap-only facebook.com fbcdn.net
```

接下来，我们需要一种方法来读取每个域参数并对其执行扫描。让我们使用循环！Bash 有两种类型的循环：`for` 循环和 `while` 循环。`for` 循环更适合我们的目的，因为我们已经知道要循环遍历的值的数量。通常，当你已经有一个要遍历的值列表时，应该使用 `for` 循环。当你不确定要循环遍历多少个值，但希望指定执行应停止的条件时，应该使用 `while` 循环。

以下是 bash 中 `for` 循环的语法。对于 `LIST_OF_VALUES` 中的每个项，bash 将执行 `do` 和 `done` 之间的代码一次：

```bash
for i in LIST_OF_VALUES
do
    DO SOMETHING
done
```

现在让我们使用 `for` 循环实现我们的功能：

```bash
for i in "${@:$OPTIND:$#}"   # 1
do
    # 对 $i 执行扫描
done
```

我们创建一个数组 ➊，其中包含除已被 `getopts` 解析的参数之外的每个命令行参数，`getopts` 将解析的选项之后的第一个参数的索引存储到名为 `$OPTIND` 的变量中。字符 `$@` 表示包含所有输入参数的数组，而 `$#` 是传入的命令行参数的数量。`"${@:OPTIND:}"` 对数组进行切片，以便它删除 `MODE` 参数（如 `nmap-only`），确保我们只遍历输入中的域部分。数组切片是从数组中提取项子集的一种方法。在 bash 中，你可以使用以下语法对数组进行切片（请注意，命令周围的引号是必需的）：

```bash
"${INPUT_ARRAY:START_INDEX:END_INDEX}"
```

`$i` 变量表示参数数组中的当前项。然后我们可以将循环包裹在代码周围：

```bash
#!/bin/bash
PATH_TO_DIRSEARCH="/Users/vickieli/tools/dirsearch"

nmap_scan() {
    nmap $DOMAIN > $DIRECTORY/nmap
    echo "The results of nmap scan are stored in $DIRECTORY/nmap."
}

dirsearch_scan() {
    $PATH_TO_DIRSEARCH/dirsearch.py -u $DOMAIN -e php --simple-report=$DIRECTORY/dirsearch
    echo "The results of dirsearch scan are stored in $DIRECTORY/dirsearch."
}

crt_scan() {
    curl "https://crt.sh/?q=$DOMAIN&output=json" -o $DIRECTORY/crt
    echo "The results of cert parsing is stored in $DIRECTORY/crt."
}

getopts "m:" OPTION
MODE=$OPTARG

for i in "${@:$OPTIND:$#}"   # 1
do
    DOMAIN=$i
    DIRECTORY=${DOMAIN}_recon
    echo "Creating directory $DIRECTORY."
    mkdir $DIRECTORY

    case $MODE in
        nmap-only)
            nmap_scan
            ;;
        dirsearch-only)
            dirsearch_scan
            ;;
        crt-only)
            crt_scan
            ;;
        *)
            nmap_scan
            dirsearch_scan
            crt_scan
            ;;
    esac

    echo "Generating recon report for $DOMAIN..."
    TODAY=$(date)
    echo "This scan was created on $TODAY" > $DIRECTORY/report
    if [ -f $DIRECTORY/nmap ];then       # 2
        echo "Results for Nmap:" >> $DIRECTORY/report
        grep -E "^\s*\S+\s+\S+\s+\S+\s*$" $DIRECTORY/nmap >> $DIRECTORY/report
    fi
    if [ -f $DIRECTORY/dirsearch ];then   # 3
        echo "Results for Dirsearch:" >> $DIRECTORY/report
        cat $DIRECTORY/dirsearch >> $DIRECTORY/report
    fi
    if [ -f $DIRECTORY/crt ];then         # 4
        echo "Results for crt.sh:" >> $DIRECTORY/report
        jq -r ".[] | .name_value" $DIRECTORY/crt >> $DIRECTORY/report
    fi
done   # 5
```

`for` 循环以 `for` 关键字开始 ➊，以 `done` 关键字结束 ➎。注意，我们还在报告部分添加了几行，以查看是否需要生成每种类型的报告。我们检查 Nmap 扫描、Dirsearch 扫描或 crt.sh 扫描的输出文件是否存在，以便确定是否需要为该扫描类型生成报告 ➋ ➌ ➍。

条件周围的中括号意味着我们将内容传递给 `test` 命令：`[ -f $DIRECTORY/nmap ]` 等价于 `test -f $DIRECTORY/nmap`。

`test` 命令评估条件并输出 `true` 或 `false`。`-f` 标志测试文件是否存在。但你可以测试更多条件！让我们介绍一些有用的测试条件。`-eq` 和 `-ne` 标志分别测试相等和不相等。如果 `$3` 等于 1，则返回 `true`：

```bash
if [ $3 -eq 1 ]
```

如果 `$3` 不等于 1，则返回 `true`：

```bash
if [ $3 -ne 1 ]
```

`-gt`、`-ge`、`-lt` 和 `-le` 标志分别测试大于、大于或等于、小于、小于或等于：

```bash
if [ $3 -gt 1 ]
if [ $3 -ge 1 ]
if [ $3 -lt 1 ]
if [ $3 -le 1 ]
```

`-z` 和 `-n` 标志测试字符串是否为空。这两个条件都为 `true`：

```bash
if [ -z "" ]
if [ -n "abc" ]
```

`-d`、`-f`、`-r`、`-w` 和 `-x` 标志检查目录和文件状态。你可以在 shell 脚本对文件进行操作之前使用它们检查文件的存在和权限。例如，如果 `/bin` 是一个存在的目录，此命令返回 `true`：

```bash
if [ -d /bin ]
```

如果 `/bin/bash` 是一个存在的文件，此命令返回 `true`：

```bash
if [ -f /bin/bash ]
```

如果 `/bin/bash` 是一个可读文件，此命令返回 `true`：

```bash
if [ -r /bin/bash ]
```

或者是可写文件：

```bash
if [ -w /bin/bash ]
```

或者是可执行文件：

```bash
if [ -x /bin/bash ]
```

你还可以使用 `&&` 和 `||` 来组合测试表达式。如果两个表达式都为 `true`，此命令返回 `true`：

```bash
if [ $3 -gt 1 ] && [ $3 -lt 3 ]
```

如果至少有一个为 `true`，此命令返回 `true`：

```bash
if [ $3 -gt 1 ] || [ $3 -lt 0 ]
```

你可以在 `test` 命令的手册中找到更多比较标志，方法是运行 `man test`。（如果你不确定你正在使用的命令，你总是可以在终端中输入 `man` 后跟命令名称来访问命令的手册文件。）

### 构建交互式程序

如果你想构建一个在执行期间接受用户输入的交互式程序怎么办？假设如果用户输入命令行选项 `-i`，你希望程序进入交互模式，允许你随时指定要扫描的域：

```bash
./recon.sh -i -m nmap-only
```

为此，你可以使用 `read`。此命令读取用户输入并将输入字符串存储到一个变量中：

```bash
echo "Please enter a domain!"
read DOMAIN
```

这些命令将提示用户输入一个域，然后将输入存储在名为 `$DOMAIN` 的变量中。

要反复提示用户，我们需要使用 `while` 循环，它将不断打印提示，要求输入域，直到用户退出程序。以下是 `while` 循环的语法。只要 `CONDITION` 为 `true`，`while` 循环将重复执行 `do` 和 `done` 之间的代码：

```bash
while CONDITION
do
    DO SOMETHING
done
```

我们可以使用 `while` 循环反复提示用户输入域，直到用户输入 `quit`：

```bash
while [ $INPUT != "quit" ];do
    echo "Please enter a domain!"
    read INPUT
    if [ $INPUT != "quit" ];then
        scan_domain $INPUT
        report_domain $INPUT
    fi
done
```

我们还需要一种方法让用户实际调用 `-i` 选项，而我们当前的 `getopts` 命令并没有这样做。我们可以通过重复使用 `getopts` 来使用 `while` 循环解析选项：

```bash
while getopts "m:i" OPTION; do
    case $OPTION in
        m)
            MODE=$OPTARG
            ;;
        i)
            INTERACTIVE=true
            ;;
    esac
done
```

在这里，我们指定一个 `while` 循环，重复获取命令行选项。如果选项标志是 `-m`，我们将 `MODE` 变量设置为用户指定的扫描模式。如果选项标志是 `-i`，我们将 `$INTERACTIVE` 变量设置为 `true`。

然后，在脚本后面，我们可以通过检查 `$INTERACTIVE` 变量的值来决定是否调用交互模式。将它们放在一起，我们得到最终的脚本：

```bash
#!/bin/bash
source ./scan.lib

while getopts "m:i" OPTION; do
    case $OPTION in
        m)
            MODE=$OPTARG
            ;;
        i)
            INTERACTIVE=true
            ;;
    esac
done

scan_domain(){
    DOMAIN=$1
    DIRECTORY=${DOMAIN}_recon
    echo "Creating directory $DIRECTORY."
    mkdir $DIRECTORY
    case $MODE in
        nmap-only)
            nmap_scan
            ;;
        dirsearch-only)
            dirsearch_scan
            ;;
        crt-only)
            crt_scan
            ;;
        *)
            nmap_scan
            dirsearch_scan
            crt_scan
            ;;
    esac
}

report_domain(){
    DOMAIN=$1
    DIRECTORY=${DOMAIN}_recon
    echo "Generating recon report for $DOMAIN..."
    TODAY=$(date)
    echo "This scan was created on $TODAY" > $DIRECTORY/report
    if [ -f $DIRECTORY/nmap ];then
        echo "Results for Nmap:" >> $DIRECTORY/report
        grep -E "^\s*\S+\s+\S+\s+\S+\s*$" $DIRECTORY/nmap >> $DIRECTORY/report
    fi
    if [ -f $DIRECTORY/dirsearch ];then
        echo "Results for Dirsearch:" >> $DIRECTORY/report
        cat $DIRECTORY/dirsearch >> $DIRECTORY/report
    fi
    if [ -f $DIRECTORY/crt ];then
        echo "Results for crt.sh:" >> $DIRECTORY/report
        jq -r ".[] | .name_value" $DIRECTORY/crt >> $DIRECTORY/report
    fi
}

if [ $INTERACTIVE ];then               # 1
    INPUT="BLANK"
    while [ $INPUT != "quit" ];do      # 2
        echo "Please enter a domain!"
        read INPUT
        if [ $INPUT != "quit" ];then   # 3
            scan_domain $INPUT
            report_domain $INPUT
        fi
    done
else
    for i in "${@:$OPTIND:$#}";do
        scan_domain $i
        report_domain $i
    done
fi
```

在这个程序中，我们首先通过检查用户是否指定了 `-i` 选项来检查用户是否选择了交互模式 ➊。然后我们使用 `while` 循环反复提示用户输入域 ➋。如果用户输入不是关键字 `quit`，我们假设他们输入了一个目标域，因此我们扫描并为该域生成报告 ➌。`while` 循环将继续运行，并向用户询问域，直到用户输入 `quit`，这将导致 `while` 循环退出，程序终止。

交互式工具可以帮助你的工作流程更顺畅地运行。例如，你可以构建测试工具，让你根据初步结果选择如何继续。

### 使用特殊变量和字符

你现在具备了足够的 bash 知识，可以构建许多多功能工具。本节提供了更多关于 shell 脚本特殊性的提示。

在 Unix 中，命令成功时返回 0，失败时返回正整数。变量 `$?` 包含最后执行的命令的退出值。你可以使用它们来测试执行成功和失败：

```bash
#!/bin/sh
chmod 777 script.sh
if [ "$?" -ne "0" ]; then
    echo "Chmod failed. You might not have permissions to do that!"
fi
```

另一个特殊变量是 `$$`，它包含当前进程的 ID。当你需要为脚本创建临时文件时，这很有用。如果同一脚本或程序的多个实例同时运行，每个实例可能需要自己的临时文件。在这种情况下，你可以为每个实例创建名为 `/tmp/script_name_$$` 的临时文件。

还记得我们之前在本章中讨论的 shell 脚本中的变量作用域吗？不是输入参数的变量对整个脚本是全局的。如果你希望其他程序也使用该变量，你需要导出该变量：

```bash
export VARIABLE_NAME=VARIABLE_VALUE
```

假设在你的一个脚本中，你设置了变量 `VAR`：

```bash
VAR="hello!"
```

如果你不导出它或在另一个脚本中引入它，该值会在脚本退出后被销毁。但是，如果你在第一个脚本中导出 `VAR` 并在运行第二个脚本之前运行第一个脚本，则第二个脚本将能够读取 `VAR` 的值。

你还应该注意 bash 中的特殊字符。在 Unix 中，通配符 `*` 代表所有。例如，此命令将打印当前目录中所有具有 `.txt` 文件扩展名的文件名：

```bash
$ ls *.txt
```

反引号（`` ` ``）表示命令替换。你可以同时使用反引号和前面提到的 `$()` 命令替换语法达到相同的目的。此 `echo` 命令将打印 `whoami` 命令的输出：

```bash
echo `whoami`
```

大多数特殊字符，如通配符或单引号，当放在双引号中时，不会被解释为特殊字符。相反，它们被视为字符串的一部分。例如，此命令将回显字符串 `"abc '*' 123"`：

```bash
$ echo "abc '*' 123"
```

另一个重要的特殊字符是反斜杠（`\`），bash 中的转义字符。它告诉 bash 某个字符应按字面意思解释，而不是作为特殊字符。

某些特殊字符，如双引号、美元符号、反引号和反斜杠，即使在双引号内也保持特殊，因此如果你希望 bash 按字面意思处理它们，你必须使用反斜杠来转义它们：

```bash
$ echo "\" is a double quote. \$ is a dollar sign. \` is a backtick. \\ is a backslash."
```

此命令将回显：

```
" is a double quote. $ is a dollar sign. ` is a backtick. \ is a backslash.
```

你也可以在换行符前使用反斜杠来表示代码行尚未结束。例如，此命令

```bash
chmod 777 \
script.sh
```

与以下命令相同：

```bash
chmod 777 script.sh
```

恭喜！你现在可以编写 bash 脚本了。Bash 脚本起初可能看起来令人生畏，但是一旦你掌握了它，它将是你黑客武器库中的一个强大补充。你将能够执行更好的侦察，进行更有效的测试，并拥有更有结构的黑客工作流程。

如果你计划实现大量自动化，那么从一开始就组织好你的脚本是个好主意。建立一个脚本目录，并按功能对脚本进行分类。这将是你开发自己的黑客方法的开始。当你收集了一些你经常使用的脚本时，你可以使用脚本来自动运行它们。例如，你可以将脚本分类为侦察脚本、模糊测试脚本、自动化报告等。这样，每次你找到喜欢的脚本或工具时，都可以以有组织的方式快速将其整合到你的工作流程中。

### 安排自动扫描

现在，让我们通过构建一个警报系统将你的自动化提升到新的水平，该系统将在扫描中出现有趣的内容时通知我们。这样我们就不必手动运行命令并一遍又一遍地梳理结果。

我们可以使用 `cron` 作业来安排我们的扫描。Cron 是基于 Unix 的操作系统上的作业调度程序。它允许你安排定期运行的作业。例如，你可以每天在相同时间运行一个脚本，检查特定站点上的新端点。或者你可以每天运行一个扫描器，检查同一目标上的漏洞。这样，你可以监控应用程序行为的变化并找到利用它的方法。

你可以通过编辑名为 crontab 的文件来配置 Cron 的行为。Unix 为每个用户保留不同的 crontab 副本。通过运行以下命令编辑你自己的用户的 crontab：

```bash
crontab -e
```

所有 crontab 都遵循相同的语法：

```
A B C D E command_to_be_executed
A: 分钟 (0 – 59)
B: 小时 (0 – 23)
C: 日 (1 – 31)
D: 月 (1 – 12)
E: 星期几 (0 – 7) (星期日是 0 或 7，星期一是 1...)
```

每行使用五个数字指定要运行的命令以及运行时间。第一个数字从 0 到 59，指定命令应运行的分钟。第二个数字指定小时，范围从 0 到 23。第三和第四个数字是命令应运行的日和月。最后一个数字是命令应运行的星期几，范围从 0 到 7。0 和 7 都表示命令应在星期日运行；1 表示命令应在星期一运行；依此类推。

例如，你可以将此行添加到你的 crontab 中，以每天下午 9:30 运行你的侦察脚本：

```
30 21 * * * ./scan.sh
```

你还可以批量运行目录中的脚本。crontab 中的 `run-parts` 命令告诉 Cron 运行存储在目录中的所有脚本。例如，你可以将所有侦察工具存储在一个目录中，并定期扫描你的目标。以下行告诉 Cron 每天下午 9:30 运行我 `security` 目录中的所有脚本：

```
30 21 * * * run-parts /Users/vickie/scripts/security
```

接下来，`git diff` 是一个输出两个文件之间差异的命令。你需要安装 Git 程序才能使用它。你可以使用 `git diff` 比较不同时间的扫描结果，这可以让你快速查看自上次扫描以来目标是否发生了变化：

```bash
git diff SCAN_1 SCAN_2
```

这将帮助你识别目标的新域、子域、端点和其他新资产。你可以编写如下脚本，每天通知你目标的新变化：

```bash
#!/bin/bash
DOMAIN=$1
DIRECTORY=${DOMAIN}_recon
echo "Checking for new changes about the target: $DOMAIN.\n Found these new things."
git diff <SCAN AT TIME 1> <SCAN AT TIME 2>
```

并使用 Cron 安排它：

```bash
30 21 * * * ./scan_diff.sh facebook.com
```

这些自动化技术帮助我快速找到目标上的新 JavaScript 文件、端点和功能。我特别喜欢使用这种技术来自动发现子域名接管漏洞。我们将在第 20 章讨论子域名接管。

或者，你可以使用 GitHub 来跟踪更改。在 [https://github.com/new/](https://github.com/new/) 上设置一个仓库来存储你的扫描结果。GitHub 有一个通知功能，当仓库上发生重大事件时会通知你。它位于每个仓库页面的“Settings” → “Notifications”。向 GitHub 提供一个电子邮件地址，它将使用该地址通知你更改。然后，在你存储扫描结果的目录中，运行以下命令以在该目录中初始化 git：

```bash
git init
git remote add origin https://PATH_TO_THE_REPOSITORY
```

最后，使用 Cron 定期扫描目标并将文件上传到 GitHub：

```bash
30 21 * * * ./recon.sh facebook.com
40 21 * * * git add *; git commit -m "new scan"; git push -u origin master
```

然后，GitHub 将向你发送一封关于新扫描中更改的文件的电子邮件。

## 关于侦察 API 的说明

本章提到的许多工具都有 API，允许你将它们的服务集成到你的应用程序和脚本中。我们将在第 24 章更多地讨论 API，但就目前而言，你可以将 API 视为你可以用来查询服务数据库的端点。使用这些 API，你可以从脚本中查询侦察工具，并将结果添加到你的侦察报告中，而无需手动访问它们的站点。

例如，Shodan 有一个 API（[https://developer.shodan.io/](https://developer.shodan.io/)），允许你查询其数据库。你可以通过访问此 URL 来访问主机的扫描结果：`https://api.shodan.io/shodan/host/{ip}?key={YOUR_API_KEY}`。你可以配置你的 bash 脚本向此 URL 发送请求并解析结果。LinkedIn 也有一个 API（[https://www.linkedin.com/developers/](https://www.linkedin.com/developers/)），允许你查询其数据库。例如，你可以使用此 URL 访问 LinkedIn 上用户的信息：`https://api.linkedin.com/v2/people/{PERSON ID}`。Censys API（[https://censys.io/api](https://censys.io/api)）允许你通过查询端点 `https://censys.io/api/v1` 来访问证书。

本章提到的其他工具，如 BuiltWith、Google 搜索和 GitHub 搜索，都有自己的 API 服务。这些 API 可以通过将第三方工具集成到你的侦察脚本中，帮助你更有效地发现资产和内容。请注意，大多数 API 服务要求你在其网站上创建一个帐户以获取 API 密钥，这是大多数 API 服务认证其用户的方式。你可以在 [https://github.com/lanmaster53/recon-ng-marketplace/wiki/API-Keys/](https://github.com/lanmaster53/recon-ng-marketplace/wiki/API-Keys/) 找到有关如何获取流行侦察服务的 API 密钥的信息。
