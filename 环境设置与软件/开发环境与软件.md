##### WSL 安装 Linux 分发版

[官网文档](https://learn.microsoft.com/zh-cn/windows/wsl/)
1. ```wsl.exe --install```，在 管理员 PowerShell 中输入此命令，安装 Linux 分发版（默认为 Ubuntu）。
2. 设置 Linux 用户名和密码
- ```wsl.exe -d Ubuntu``` 启动此分发
- [忘记密码指引](https://learn.microsoft.com/zh-cn/windows/wsl/setup/environment#set-up-your-linux-username-and-password)
- ```--unregister <Distro>```
    取消注册分发版并删除根文件系统。

#### Docker

[Docker Desktop 下载](https://www.docker.com/products/docker-desktop/)
- 遇到 WSL 依赖项下载缓慢```wsl --install```时，访问其Github项目，使用 [Github加速网站](https://github.akams.cn/)下载 .msi 安装文件
- 运行```docker pull```命令时网络连接失败时：
    1. 访问[镜像检测项目](https://github.com/dongyubin/DockerHub)获得最新可用镜像
    2. 在 Docker 设置中选择 Docker Engine 项，在 json 中添加获得的镜像地址：
        ```json
        "registry-mirrors": [
            "https://registry.cyou",
            "https://docker.1ms.run"
        ]
        ```
    - 自动解析宿主机IP地址```host.docker.internal```
    - 设置环境变量```-e 环境变量```
- 概念
    - Docker 通过读取 Dockerfile 中的指令来构建镜像。
    - 使用 Docker Compose，您可以使用 YAML 配置文件（称为 Compose 文件 ）来配置应用程序的服务，然后使用 Compose CLI 从配置中创建和启动所有服务。
        - Compose 文件的默认路径是 compose.yaml （首选）或位于工作目录中的 compose.yml 。为了向后兼容早期版本，Compose 还支持 docker-compose.yaml 和 docker-compose.yml 。如果这两个文件都存在，Compose 会优先使用标准的 compose.yaml 文件。
- 命令
    <https://docs.docker.com/reference/cli/docker/container/>
    - 查看容器名称/ID ```docker ps``` 
    - docker-compose build 建立容器镜像
    - docker-compose up 启动容器
    - docker-compose down 关闭容器
    - docker volume 共享目录


#### VMware Workstation

不存在 VirtualBox 的键盘输入丢失，并且启动速度快，运行虚拟机的性能更强。

[techspot 下载站](https://www.techspot.com/downloads/189-vmware-workstation-for-windows.html)等网站下载，跳过博通账户注册

linux 安装
- 安装依赖项 `sudo apt install build-essential linux-headers-$(uname -r)`

由于取消安装或其他原因导致在之后安装时出现异常： `需要 VMware install disk上的文件 'vmnetbridge.dll'` 提示框
- 显示的来源对应之前的安装路径，于是手动选择当前的安装路径即可
- 尝试 `Win + R` 输入 `regedit` 搜索并手动删除注册表，但遇到无权限或拒绝时 [找到 CSDN 文章](https://blog.csdn.net/RookieSiegeLion/article/details/127187844)：
    1. 选择注册表的 `权限 - 高级 - （所有者）更改`
    2. 在 `输入要选择的对象名称` 填入 “Everyone” 点击`检查`并`确定`
    3. 此时可授权从而修改或删除注册表

[官网文档](https://techdocs.broadcom.com/us/en/vmware-cis/desktop-hypervisors/workstation-pro/25H2.html)，在 `preparing-to-create-a-new-virtual-machine`这一节拥有安装与配置虚拟机的说明

连接 USB 设备到虚拟机

##### 虚拟机设置

- 在 Ubuntu 安装 VMware 工具的 [openvmtools](https://knowledge.broadcom.com/external/article/313456/vmware-support-for-openvmtools.html)，以启用3D加速与双向粘贴板
    进入虚拟机后命令行输入 
    ```sh
    sudo apt update
    sudo apt install open-vm-tools open-vm-tools-desktop
    reboot
    ```
- 启用共享文件夹：
    `VM - 设置 - 选项 - 共享文件夹` 启用并选择文件夹位置
    [博通 vmware-tools 命令介绍文章](https://knowledge.broadcom.com/external/article/313233/how-to-configure-vmware-tools-shared-fol.html)
    - 对于 VMware Tools - 共享文件夹用户模式的 FUSE 客户端，请使用以下挂载命令：    
    `sudo /usr/bin/vmhgfs-fuse .host:/ /mnt/hgfs -o subtype=vmhgfs-fuse,allow_other`
